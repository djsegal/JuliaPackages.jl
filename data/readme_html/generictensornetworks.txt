<p dir="auto">This package implements generic tensor networks to compute <em>solution space properties</em> of a class of hard combinatorial optimization problems.
The <em>solution space properties</em> include</p>
<ul dir="auto">
<li>The maximum/minimum solution sizes,</li>
<li>The number of solutions at certain sizes,</li>
<li>The enumeration/sampling of solutions at certain sizes.</li>
</ul>
<p dir="auto">The types of problems that can be solved using this package include <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/IndependentSet/" rel="nofollow">Independent set problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/MaximalIS/" rel="nofollow">Maximal independent set problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/SpinGlass/" rel="nofollow">Spin-glass problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/MaxCut/" rel="nofollow">Cutting problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/Matching/" rel="nofollow">Vertex matching problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/PaintShop/" rel="nofollow">Binary paint shop problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/Coloring/" rel="nofollow">Coloring problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/DominatingSet/" rel="nofollow">Dominating set problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/SetPacking/" rel="nofollow">Set packing problem</a>, <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/Satisfiability/" rel="nofollow">Satisfiability problem</a> and <a href="https://queracomputing.github.io/GenericTensorNetworks.jl/dev/generated/SetCovering/" rel="nofollow">Set covering problem</a>.</p>

<p dir="auto">We recommend that you use <strong>Julia version &gt;= 1.7</strong>; otherwise, your program may suffer from significant (exponential in the tensor dimension) overheads when permuting the dimensions of a large tensor.
If you have to use an older version of Julia, you can overwrite the <code>LinearAlgebra.permutedims!</code> by adding the following patch to your own project.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="# only required when your Julia version is &lt; 1.7

function LinearAlgebra.permutedims!(C::Array{T,N}, A::StridedArray{T,N}, perm) where {T,N}
    if isbitstype(T)
        TensorOperations.tensorcopy!(A, ntuple(identity,N), C, perm)
    else
        invoke(permutedims!, Tuple{Any,AbstractArray,Any}, C, A, perm)
    end
end"><pre><span class="pl-c"><span class="pl-c">#</span> only required when your Julia version is &lt; 1.7</span>

<p dir="auto">A graphical editor for graph-like structures based on <a href="https://github.com/AlgebraicJulia/Catlab.jl">Catlab</a>.</p>
<p dir="auto">Legacy version built with typescript is in the <code>legacy</code> branch, and will not receive updates; new version with scala is now in the <code>main</code> branch.</p>
<h2 dir="auto"><a id="user-content-how-to-run" class="anchor" aria-hidden="true" href="#how-to-run"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>How to run</h2>
<p dir="auto">The core of Semagrams is just a library; in order to make it do things, one needs to create an "app" that uses it. Currently, the only app that is being developed is a Petri net editor, though this will soon change.</p>
<p dir="auto">In order to run the Petri net editor standalone, install <a href="https://github.com/com-lihaoyi/mill">Mill</a> and <a href="https://www.npmjs.com/" rel="nofollow">npm</a>, and then in one terminal in <code>scala/</code> run <code>mill --watch apps.petri.fullLinkJS</code> and in another terminal in <code>scala/</code> run <code>npm run dev</code>. The second command should print out a url that you can click on. You may have to run <code>npm install</code> before running <code>npm run dev</code>.</p>
<p dir="auto">You can look at <code>scala/build.sc</code> to figure out how to add another app, though you will have to change <code>scala/main.js</code> to point to the compiled js of the new file.</p>
<p dir="auto">Although Semagrams is a library, it is not a particularly stable library. Thus, in the beginning phases, we are working on similar development principles as the linux kernel; nothing that is not in this repository will be supported. If you would like to work on an app using Semagrams and would like it to not go out of date, you will have to add it to this repository so that it can be refactored when Semagrams is refactored.</p>

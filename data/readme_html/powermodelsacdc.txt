<p dir="auto">PowerModelsACDC.jl is a Julia/JuMP/PowerModels package with models for DC lines, meshed DC networks, and AC DC converters.
Building upon the PowerModels architecture, the code is engineered to decouple problem specifications (e.g. Power Flow, Optimal Power Flow, ...) from the power network formulations (e.g. AC, DC-approximation, SOC-relaxation, ...).</p>
<p dir="auto"><strong>Installation</strong>
The latest stable release of PowerModelACDC can be installed using the Julia package manager with</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="Pkg.add(&quot;PowerModelsACDC&quot;)"><pre>Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>PowerModelsACDC<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto"><strong>Core Problem Specifications</strong></p>
<ul dir="auto">
<li>Optimal Power Flow with both point-to-point and meshed and dc grid support</li>
<li>Power Flow with both point-to-point and meshed ac and dc grid support</li>
<li>TNEP problem for point-to-point and meashed ac and dc grids</li>
</ul>
<p dir="auto"><strong>Core Formulations</strong>
All AC formulations of PowerModels are re-used.
Therefore, the core formulations in this package are twofold: those for the DC network and those for the AC/DC converters.</p>
<p dir="auto">DC network connecting dc nodes:</p>
<ul dir="auto">
<li>DC nonlinear nonconvex formulation (NLP)</li>
<li>Convexified (SOC) bus injection model and branch flow model for the DC grid (which can be used with both the SDP and SOC convex relaxation formulations for the AC side)</li>
<li>Linearized (LP) active power only formulation, extending the linearized 'DC' approximation of AC grids to DC grids</li>
</ul>
<p dir="auto">AC/DC converter stations, connecting ac nodes and dc nodes, are composed of a transformer, filter, phase reactor and LCC/VSC converter. The passive components can be removed/disabled. Convex relaxation and linearized models for the passive components have been described, therefore, the converter station formulation is categorized by converter model complexity. The converter model includes constant losses and losses proportional to the current magnitude as well as current magnitude squared.</p>
<ul dir="auto">
<li>Nonlinear nonconvex formulation (NLP)</li>
<li>Convexified formulation (SOC)</li>
<li>Linearized formulation (LP)</li>
</ul>
<p dir="auto"><strong>Network Data Formats</strong></p>
<ul dir="auto">
<li>MatACDC-style ".m" files (matpower ".m"-derived).</li>
<li>Matpower-style ".m" files, including matpower's dcline extenstions.</li>
<li>PTI ".raw" files, using PowerModels.jl parser</li>
</ul>
<p dir="auto">Note that the matpower-style <code>dcline</code> is transformed internally to two converters + a dcline connecting them. Such a transformation is exact for the 'dc'-style linearized models, but not for the AC models.</p>
<p dir="auto">For further information, consult the PowerModels <a href="https://lanl-ansi.github.io/PowerModels.jl/stable/" rel="nofollow">documentation</a>.</p>

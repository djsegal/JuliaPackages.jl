<p dir="auto">Templates for <a href="https://github.com/tlienart/Franklin.jl">Franklin</a>, the static-site generator in Julia.</p>
<p dir="auto">Most of these templates are adapted from existing, popular templates with minor modifications to accommodate Franklin's content.</p>
<p dir="auto"><strong>NOTE</strong>: these templates should be seen as <em>starting points</em>, they are far from perfect.
PRs to help improve them will be very welcome, thanks!
Most importantly they are designed to be simple to adjust to your needs.</p>

<h2 dir="auto"><a id="user-content-modifying-or-adding-a-template" class="anchor" aria-hidden="true" href="#modifying-or-adding-a-template"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Modifying or adding a template</h2>
<p dir="auto">The package contains a few utils to make it easier to modify or add templates.
Now if it was just a bunch of fixes to an existing template, you can just push those changes to your fork and open a PR.
If it's a new template that you're working on, you can also do that but there's a few extra things you need to do:</p>
<ol dir="auto">
<li>in <code>FranklinTemplates/src/FranklinTemplates.jl</code> add the name of your template in the list</li>
<li>in <code>FranklinTemplates/docs/make.jl</code> add the name of your template with a description in the list</li>
<li>in <code>FranklinTemplates/docs/thumb</code> add a screenshot of your template in <code>png</code> format with <strong>exactly</strong> an 850x850 dimension</li>
<li>in <code>FranklinTemplates/docs/index_head.html</code> add a CSS block following the other examples</li>
</ol>
<p dir="auto">To locally see changes quickly, use <a href="#user-content-changing-a-single-template">Changing a single template</a>.
To change all templates at the same time, use <a href="#user-content-changing-multiple-templates">Changing multiple templates</a>.</p>
<h3 dir="auto"><a id="user-content-changing-a-single-template" class="anchor" aria-hidden="true" href="#changing-a-single-template"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Changing a single template</h3>
<ol dir="auto">
<li>clone a fork of this package wherever you usually do things, typically <code>~/.julia/dev/</code></li>
<li>checkout the package in development mode with <code>] dev FranklinTemplates</code></li>
<li><code>cd</code> to a sensible workspace and do one of
<ol dir="auto">
<li><code>using FranklinTemplates; newsite("newTemplate")</code> to start working on <code>newTemplate</code> more or less from scratch,</li>
<li><code>using FranklinTemplates; newsite("newTemplate", template="jemdoc")</code> to start working on <code>newTemplate</code> using some other template as starting point,</li>
<li><code>using FranklinTemplates; modify("jemdoc")</code> to quickly start working on an existing template in order to fix it.</li>
</ol>
</li>
<li>change things, fix things, etc.</li>
<li>bring your changes into your fork with <code>addtemplate("path/to/your/work")</code>
<ol dir="auto">
<li>if the template doesn't exist, it will just add the folder removing things that are duplicate from <code>templates/common/</code>.</li>
<li>if the template exists, it will just adjust what needs to be adjusted.</li>
</ol>
</li>
</ol>

<p dir="auto">This package contains a copy of the relevant KaTeX files and highlight.js files;</p>
<ul dir="auto">
<li>the KaTeX files are basically provided "as is", completely unmodified; you could download your own version of the files from the <a href="https://github.com/KaTeX/KaTeX">original repo</a> and replace the files in <code>_libs/katex</code>,</li>
<li>the Highlight.js files are <em>essentially</em> provided "as is" for a set of languages, there is a small modification in the <code>highlight.pack.js</code> file to highlight julia shell and pkg prompt (see next section). You can also download your own version of files from the <a href="https://highlightjs.org" rel="nofollow">original source</a> where you might want to
<ul dir="auto">
<li>specify languages you want to highlight if other than the default list above</li>
<li>specify the "style" (we use github but you could use another sheet)</li>
</ul>
</li>
</ul>
<p dir="auto"><strong>Note</strong>: in Franklin's <code>optimize</code> pass, by default the <strong>full library</strong> <code>highlight.js</code> is called to pre-render highlighting; this bypasses the <code>highlight.pack.js</code> file and, in particular, supports highlighting for <strong>all</strong> languages. In other words, the <code>highlight.pack.js</code> file is relevant only when you preview your site locally with <code>serve()</code> or if you don't intend to apply the prerendering step.</p>

<p>Templates for <a href="https://github.com/tlienart/Franklin.jl">Franklin</a>, the static-site generator in Julia.</p>
<p>Most of these templates are adapted from existing, popular templates with minor modifications to accommodate Franklin's content.</p>
<p><strong>NOTE</strong>: these templates should be seen as <em>starting points</em>, they are far from perfect. PRs to help improve them will be very welcome, thanks! Most importantly they are designed to be simple to adjust to your needs.</p>

<h2><a id="user-content-fixingadding-a-template" class="anchor" aria-hidden="true" href="#fixingadding-a-template"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Fixing/Adding a template</h2>
<p>The package now contains a few utils to make it easier to add / fix templates.</p>
<ol>
<li>clone a fork of this package wherever you usually do things, typically <code>~/.julia/dev/</code></li>
<li>checkout the package in development mode with <code>] dev FranklinTemplates</code></li>
<li><code>cd</code> to a sensible workspace and do one of
<ol>
<li><code>using FranklinTemplates; newsite("newTemplate")</code> to start working on <code>newTemplate</code> more or less from scratch,</li>
<li><code>using FranklinTemplates; newsite("newTemplate", template="jemdoc")</code> to start working on <code>newTemplate</code> using some other template as starting point,</li>
<li><code>using FranklinTemplates; modify("jemdoc")</code> to quickly start working on an existing template in order to fix it.</li>
</ol>
</li>
<li>change things, fix things, etc.</li>
<li>bring your changes into your fork with <code>addtemplate("path/to/your/work")</code>
<ol>
<li>if the template doesn't exist, it will just add the folder removing things that are duplicate from <code>templates/common/</code>.</li>
<li>if the template exists, it will just adjust what needs to be adjusted.</li>
</ol>
</li>
</ol>
<p>Now if it was just a bunch of fixes to an existing template, you can just push those changes to your fork and open a PR.</p>
<p>If it's a new template that you're working on, you can also do that but there's a few extra things you need to do:</p>
<ol>
<li>in <code>FranklinTemplates/src/FranklinTemplates.jl</code> add the name of your template in the list</li>
<li>in <code>FranklinTemplates/docs/make.jl</code> add the name of your template with a description in the list</li>
<li>in <code>FranklinTemplates/docs/thumb</code> add a screenshot of your template in <code>png</code> format with <strong>exactly</strong> an 850x850 dimension</li>
<li>in <code>FranklinTemplates/docs/index_head.html</code> add a CSS block following the other examples</li>
</ol>
<p>Thanks!!</p>

<h2><a id="user-content-notes" class="anchor" aria-hidden="true" href="#notes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Notes:</h2>
<p>This package contains a copy of the relevant KaTeX files and highlight.js files;</p>
<ul>
<li>the KaTeX files are basically provided "as is", completely unmodified; you could download your own version of the files from the <a href="https://github.com/KaTeX/KaTeX">original repo</a> and replace the files in <code>_libs/katex</code>,</li>
<li>the Highlight.js files are <em>essentially</em> provided "as is" for a set of languages, there is a small modification in the <code>highlight.pack.js</code> file to highlight julia shell and pkg prompt (see next section). You can also download your own version of files from the <a href="https://highlightjs.org" rel="nofollow">original source</a> where you might want to
<ul>
<li>specify languages you want to highlight if other than the default list above</li>
<li>specify the "style" (we use github but you could use another sheet)</li>
</ul>
</li>
</ul>
<p><strong>Note</strong>: in Franklin's <code>optimize</code> pass, by default the <strong>full library</strong> <code>highlight.js</code> is called to pre-render highlighting; this bypasses the <code>highlight.pack.js</code> file and, in particular, supports highlighting for <strong>all</strong> languages. In other words, the <code>highlight.pack.js</code> file is relevant only when you preview your site locally with <code>serve()</code> or if you don't intend to apply the prerendering step.</p>

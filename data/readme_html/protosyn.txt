<p dir="auto"><em>ProtoSyn.jl is a platform for molecular manipulation and simulation, with an emphasis on peptide design.</em></p>
<p dir="auto">The main goal of ProtoSyn.jl is to be a basis on top of which new tools and protocols can be experimented and prototyped. Taking advantage of Julia‚Äôs environment, ProtoSyn has been built with emergent technologies in mind, such as distributed computing, GPU/SIMD acceleration and machine learning models usage. ProtoSyn.jl intends to be a playground for trying out new emergent models and algorithms in a <em>plug-and-play</em> environment, completly open-source and with curated &amp; up-do-date documentation, examples and tutorials.</p>

<ul dir="auto">
<li>Edit a peptide structure by removing, adding and mutating any number of residues</li>
<li>Create peptides from scratch by providing the desired sequence</li>
<li>Copy parts of other molecules and graft them together to create something new</li>
<li>Explore Ramachandran maps to introduce secondary structure variations</li>
<li>Run Monte Carlo simulations to optimize a structure or a sequence</li>
<li>Calculate energies and forces using native potentials (such as SASA/GB, Coulomb or simple harmonics) or commonly used tools now integrated in Julia (such as <a href="https://github.com/aiqm/torchani">TorchANI</a> or <a href="https://www.rosettacommons.org/docs/latest/rosetta_basics/scoring/score-types" rel="nofollow">REF15 from PyRosetta</a>).</li>
<li>Perform Steepest Descent optimizations</li>
<li>Explore rotamer libraries to optimize sidechain packaging</li>
<li>Select residues based on name, index, distance and other parameters, with a rich combinatory selection syntax</li>
<li>Perform rigid body docking of ligands</li>
<li>Include ramified carbohydrates and glycoproteins in your simulations, with support for sugar residues</li>
<li>Include non-canonical aminoacids (NCAAs) and post-translational modifications, such as methylation or phosporylation</li>
<li>Generate functionalized carbon models, including multi-layer &amp; pore generation support</li>
</ul>

<p dir="auto">After setting up your Julia installation, open a new REPL and add ProtoSyn.jl using the package manager:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; ] add https://github.com/sergio-santos-group/ProtoSyn.jl.git"><pre lang="@julia" class="notranslate"><code>julia&gt; ] add https://github.com/sergio-santos-group/ProtoSyn.jl.git
</code></pre></div>
<p dir="auto">All Julia-based dependencies should be automatically installed/updated. Some functionalities of ProtoSyn.jl interact with external packages or tools (such as Python packages. For example, in order to use TorchANI's energy function, you'll also need to install <a href="https://www.python.org/downloads/" rel="nofollow">Python</a> with both <a href="https://pytorch.org/get-started/locally/" rel="nofollow">Torch</a> and <a href="https://aiqm.github.io/torchani/start.html" rel="nofollow">TorchANI</a> libraries). In either case, upon using ProtoSyn.jl for the first time, users should be prompted to install any additional tool, if they so choose. These warning can be supressed at any time by adding <code>export JULIA_PROTOSYN_WARN_NON_AVALIABLE_EFC=false</code> in the user's <code>.bashrc</code> file.</p>
<p dir="auto">For trying out new features, consider using the development branch of ProtoSyn.jl instead. Be aware that some bugs and missing documentation are to be expected when using the latest versions.</p>

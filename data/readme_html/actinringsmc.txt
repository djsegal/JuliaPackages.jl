<p dir="auto">A Julia simulation package with functions for running adaptive umbrella sampling Monte Carlo simulations of crosslinked actin rings.</p>
<p dir="auto"><a href="https://github.com/cumberworth/ActinRingsMC.jl">Repository</a></p>
<p dir="auto"><a href="http://www.cumberworth.org/ActinRingsMC.jl" rel="nofollow">Documentation</a></p>
<p dir="auto">This package implements the simulation protocol outlined in the Supplemental Material of <a href="#references">Ref. 1</a>.</p>

<p dir="auto">In <code>examples</code> directory of the repository is a script for umbrella sampling.
It has both an initial run and a continuation run.
The commands can be run in the Julia REPL, or, when in the same directory as the script, it can be run with</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia run-umbrella-sampling.jl"><pre class="notranslate"><code>julia run-umbrella-sampling.jl
</code></pre></div>
<p dir="auto">To achieve good sampling, one should experiment with the number of steps per iteration and the number of iterations.</p>

<p dir="auto">The simulations output two data file types per simulation, or in the case of an umbrella sampling run, per iteration.
The file type with an <code>.ops</code> extension contains order parameters saved at steps determined by the write interval and can be read in as a dataframe for analysis and plotting.
Currently the order parameters include the energy, lattice height, and ring radius.
The file type with a <code>.vtf</code> extension is able to be read by the molecular visualization program, <a href="https://www.ks.uiuc.edu/Research/vmd/" rel="nofollow">VMD</a>, such that the configurations of the simulations can be spatially visualized.
When viewing in VMD, it is recommended to use "Chain" for "Coloring Method" and "VDW" for "Drawing Method".
The simulations also output a file containing all the system and simulation parameters used in the run to a <code>.parms</code> file.</p>
<p dir="auto">Umbrella sampling runs output additional file types.
The output of these file is based on every step, not only those determined by the write frequency.
The top row gives the lattice height (or bin, although binning has not been tested).
Each subsequent row contains data from an iteration.
<code>.counts</code> gives the number of times each lattice height was visited, <code>.freqs</code> gives a normalized version of this.
<code>.biases</code> includes the biases that were used to for that iteration (in contrast to those that would be calculated from its data).</p>
<p dir="auto">A related python package, <a href="https://github.com/cumberworth/actinrings">actinrings</a>, includes code for analyzing and plotting the output from these simulations, including free energies and ring constriction forces; see its documentation for details.</p>

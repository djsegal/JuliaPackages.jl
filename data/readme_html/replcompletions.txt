<p dir="auto">The interface is very simple. There are two exported methods provided:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="	completions(string,pos)
	shell_completions(string,pos)"><pre>	<span class="pl-c1">completions</span>(string,pos)
	<span class="pl-c1">shell_completions</span>(string,pos)</pre></div>
<p dir="auto">The former provides completion of julia expression while the latter provides completion
of julia shell syntax (<code>;</code> at the standard REPL or inside backticks).</p>
<p dir="auto">Both functions have the same return format:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="	results, range = completions(string,pos)"><pre>	results, range <span class="pl-k">=</span> <span class="pl-c1">completions</span>(string,pos)</pre></div>
<p dir="auto">The first return value is an array of possible completions, while the second return value
is a range specifying the part of the string that was matched. In particular this means
that to execute a completion, you should replace <code>string[range]</code> by <code>completion[i]</code> for some <code>i</code>.</p>

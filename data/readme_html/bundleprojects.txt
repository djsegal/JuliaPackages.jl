<p dir="auto">This package makes it easy to take a Julia project, and bundle a specific list of packages that are
used in this project into a sub folder. This can be useful for example in the preparation of replication archives for journal submissions.</p>
<h2 dir="auto"><a id="user-content-example" class="anchor" aria-hidden="true" href="#example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Example</h2>
<p dir="auto">To use, make sure there is a Julia project in your working directory, and then run</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using FilePaths, BundleProjects

bundle(joinpath(home(), &quot;outputpathforbundle&quot;), packages=[&quot;PackageA&quot;, &quot;PackageB&quot;])"><pre><span class="pl-k">using</span> FilePaths, BundleProjects

<span class="pl-c1">bundle</span>(<span class="pl-c1">joinpath</span>(<span class="pl-c1">home</span>(), <span class="pl-s"><span class="pl-pds">"</span>outputpathforbundle<span class="pl-pds">"</span></span>), packages<span class="pl-k">=</span>[<span class="pl-s"><span class="pl-pds">"</span>PackageA<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>PackageB<span class="pl-pds">"</span></span>])</pre></div>
<p dir="auto">This will copy the content of the current folder to <code>~/outputpathforbundle</code>, then create a folder called
<code>~/outputpathforbundle/packages</code> (you can change the name of this with the keyword argument <code>packages_dir</code>)
and download the exact versions of PackageA and PackageB that are recorded in the <code>Manifest.toml</code> into this
new packages folder.</p>

<p dir="auto">Many of methods in the Influence Explorer API accept similar arguments. To reduce redundancy, these are documented once, below. Not all methods accept all common parameters - be sure to check the function signature in the docs further down.</p>
<ul dir="auto">
<li><code>auth</code>: Your Sunlight API key (<a href="http://sunlightfoundation.com/api/" rel="nofollow">get one here</a>).</li>
<li><code>entity_id</code>: The transparencydata ID of the entity that you'd like to look up. Most methods will accept the ID as a <code>String</code>, however, sometimes this can cause ambiguity (for instance,<code>fec_summary</code> is defined for both Organizations and Politicians). For these cases the <code>Org</code> (Organization), <code>Ind</code> (Individual), and <code>Pol</code> (Politician) types are defined to wrap IDs (i.e. <code>Org(entity_id)</code>).</li>
<li><code>cycle</code>: Use to limit results by cycle(s).</li>
<li><code>limit</code>: Use to limit the number of results to be returned.</li>
</ul>

</ul>
</li>

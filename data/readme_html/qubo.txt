<p dir="auto"><code>QUBO.jl</code> is an all-in-one package for working with QUBO formulations in <a href="https://github.com/jump-dev/JuMP.jl">JuMP</a> and interfacing with QUBO solvers. This project aggregates three complementary packages: <a href="https://github.com/psrenergy/QUBO.jl"><code>QUBO.jl</code></a>, <a href="https://github.com/psrenergy/QUBODrivers.jl"><code>QUBODrivers.jl</code></a> and <a href="https://github.com/psrenergy/QUBOTools.jl"><code>QUBOTools.jl</code></a>.</p>
<h2 dir="auto"><a id="user-content-qubo" class="anchor" aria-hidden="true" href="#qubo"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>QUBO?</h2>
<p dir="auto">QUBO is an acronym for <em>Quadratic Unconstrained Binary Optimization</em>. So every QUBO problem is comprised of:</p>
<ul dir="auto">
<li>a linear or quadratic objective function</li>
<li>no constraints</li>
<li>binary variables</li>
</ul>
<p dir="auto">We can represent such problem as follows:</p>
<math-renderer class="js-display-math" style="display: block" data-static-url="https://github.githubassets.com/static" data-run-id="2f457e72783624dc7371274e9c4bb67c">$$\begin{array}{rl}
   \min          &amp; \mathbf{x}' Q\,\mathbf{x} \\\
   \textrm{s.t.} &amp; \mathbf{x} \in \mathbb{B}^{n}
\end{array}$$</math-renderer>
<p dir="auto">QUBOs are suited for representing non-convex global optimization problems.
With that said, the significant advances in computing systems and algorithms specialized for sampling QUBOs have contributed to their popularity.</p>
<p dir="auto">Some of the paradigms that stand out for running QUBOs are quantum gate-based optimization algorithms (QAOA and VQE), quantum annealers and hardware-accelerated platforms (Coherent Ising Machines and Simulated Bifurcation Machines).</p>
<h2 dir="auto">
<a id="user-content-qubojl-features" class="anchor" aria-hidden="true" href="#qubojl-features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>QUBO.jl</code> features</h2>
<p dir="auto"><code>QUBO.Jl</code> main features are spreaded into its three subpackages:</p>
<ul dir="auto">
<li>
<p dir="auto"><code>QUBO.jl</code>:  reformulate general JuMP problems into the QUBO format.</p>
</li>
<li>
<p dir="auto"><code>QUBODrivers.jl</code>: define a simple interface to connect with these solvers using a <a href="https://github.com/jump-dev/MathOptInterface.jl">MOI</a>-compliant API.</p>
</li>
<li>
<p dir="auto"><code>QUBOTools.jl</code>:   a set of methods to work with different formats for QUBO.</p>
</li>
</ul>
<p dir="auto">More features are available in the documentation.</p>

<p>An experiment carried out over a lazy confined sunday.</p>
<p>This wraps some functionality of Jax in julia, attempting to make Jax able to trace through native Julia functions, and compute their gradients. The package also attempts to play nicely with Flux.</p>
<p>To install it you must have jax and jaxlib installed throuh pip/conda in PyCall's python environment.</p>

<p>The main type defined by this package is <code>JaxArray</code> which wraps a python object. You can cast any dense array to <code>JaxArray</code>, and if you have a Flux model you can use the <code>|&gt; tojax</code> function much like you'd use <code>|&gt; gpu</code>.</p>
<p>Code that normally works on Julia Arrays/CuArrays should work out of the box with JaxArrays and (hopefully) yield the same results.</p>
<p>An important note is that, since Jax does not support Fortran memory ordering (like julia), all arrays are transposed when passed to Jax, to allow to perform operations efficiently. Likewise, all (wrapped) reduction operations will transpose the axis of the reduction. This should be transparent when you use it.</p>
<p>Julia's broadcasting is overloaded, in order to call the correct Jax (python) operations. In order for this to work, if you define some functions that you want to broadcast you must redefine them with the <code>@jaxfunc</code> macro, similarly to what you would do with <code>CuArrays</code>.</p>

<p>Of course the performance of all this will be quite low because the code will keep jumping between Python and Julia. However, you (again, hopefully) should be able to jit the resulting code, which will make it so that it will jump all those hoops only once, and the subsequent times it will run the compiled operations.</p>
<p>Also vmap is supported.
Check <code>jax.jit</code> and <code>jax.vmap</code>.</p>

<p dir="auto">What is the motivation of this repo?</p>
<ol dir="auto">
<li>The language server doesn't understand defining methods with macros like <code>@eval</code> or <code>ForwardDiff.@define_binary_dual_op</code>. Defining methods within the module you're working on will result in them all being marked as undefined references, but if a dependency defines them, the LSP will be able see them. Therefore, it's nice to silo off all macro-defined methods into a separate repo.</li>
<li>Sometimes it's nice to work with and do math with tuples.</li>
</ol>

<p dir="auto">This is <code>QEDbase.jl</code>, a julia package which provides the general data structures for calculations in relativistic particle physics.</p>
<p dir="auto">This package is part of the <code>QED.jl</code> library. For the description of the interoperability with other packages of <code>QED.jl</code> see <a href="www.docs-to-qed.jl">docs</a>.</p>
<h2 dir="auto"><a id="user-content-current-features" class="anchor" aria-hidden="true" href="#current-features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Current features</h2>
<ul dir="auto">
<li>Generic interface for Lorentz vectors</li>
<li>concrete implementations of static and mutable Lorentz-vector/four-momentum types</li>
<li>general Dirac bi-spinors, its adjoint counterpart as well as Dirac matrices</li>
<li>particle spinors, i.e. solutions of Dirac's equation in momentum space</li>
<li>Dirac's gamma matrices</li>
</ul>

<h4 dir="auto"><a id="user-content-four-momentum" class="anchor" aria-hidden="true" href="#four-momentum"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Four momentum</h4>
<p dir="auto">One can define a static four momentum component wise:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using QEDbase

juila&gt; mass = rand()*10

julia&gt; px,py,pz = rand(3)

julia&gt; E = sqrt(px^2 + py^2 + pz^2 + mass^2) # on-shell condition

julia&gt; P = SFourMomentum(E,px,py,pz)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> QEDbase

juila<span class="pl-k">&gt;</span> mass <span class="pl-k">=</span> <span class="pl-c1">rand</span>()<span class="pl-k">*</span><span class="pl-c1">10</span>

julia<span class="pl-k">&gt;</span> px,py,pz <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">3</span>)

julia<span class="pl-k">&gt;</span> E <span class="pl-k">=</span> <span class="pl-c1">sqrt</span>(px<span class="pl-k">^</span><span class="pl-c1">2</span> <span class="pl-k">+</span> py<span class="pl-k">^</span><span class="pl-c1">2</span> <span class="pl-k">+</span> pz<span class="pl-k">^</span><span class="pl-c1">2</span> <span class="pl-k">+</span> mass<span class="pl-k">^</span><span class="pl-c1">2</span>) <span class="pl-c"><span class="pl-c">#</span> on-shell condition</span>

julia<span class="pl-k">&gt;</span> P <span class="pl-k">=</span> <span class="pl-c1">SFourMomentum</span>(E,px,py,pz)</pre></div>
<p dir="auto">Such <code>SFourMomentum</code> behaves like a four element static array (with all the standard arithmetics), but with the <code>dot</code> product exchanged with the Minkowski product</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; @assert dot(P,P) == P*P == getMass2(P) == P[1]^2 - sum(P[1:].^2)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">@assert</span> <span class="pl-c1">dot</span>(P,P) <span class="pl-k">==</span> P<span class="pl-k">*</span>P <span class="pl-k">==</span> <span class="pl-c1">getMass2</span>(P) <span class="pl-k">==</span> P[<span class="pl-c1">1</span>]<span class="pl-k">^</span><span class="pl-c1">2</span> <span class="pl-k">-</span> <span class="pl-c1">sum</span>(P[<span class="pl-c1">1</span><span class="pl-k">:</span>]<span class="pl-k">.</span><span class="pl-k">^</span><span class="pl-c1">2</span>)</pre></div>
<p dir="auto">Furthermore, the Lorentz-vector interface provides a lot of properties for such a <code>SFourMomentum</code>, e.g.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="
julia&gt; @assert isapprox(getRapidity(mom), 0.5*log((E+pz)/(E-pz)))
julia&gt; @assert isapprox(getPlus(mom), 0.5*(E+pz))
julia&gt; @assert isapprox(getPerp(mom), px^2 + py^2)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">@assert</span> <span class="pl-c1">isapprox</span>(<span class="pl-c1">getRapidity</span>(mom), <span class="pl-c1">0.5</span><span class="pl-k">*</span><span class="pl-c1">log</span>((E<span class="pl-k">+</span>pz)<span class="pl-k">/</span>(E<span class="pl-k">-</span>pz)))
julia<span class="pl-k">&gt;</span> <span class="pl-c1">@assert</span> <span class="pl-c1">isapprox</span>(<span class="pl-c1">getPlus</span>(mom), <span class="pl-c1">0.5</span><span class="pl-k">*</span>(E<span class="pl-k">+</span>pz))
julia<span class="pl-k">&gt;</span> <span class="pl-c1">@assert</span> <span class="pl-c1">isapprox</span>(<span class="pl-c1">getPerp</span>(mom), px<span class="pl-k">^</span><span class="pl-c1">2</span> <span class="pl-k">+</span> py<span class="pl-k">^</span><span class="pl-c1">2</span>)</pre></div>
<p dir="auto">and a lot more (see <a href="www.docs-to-the-lorentz-interface-getter.jl">here</a> for a complete list). There is also a mutable version of a four vector in <code>QEDbase.jl</code>, where the Lorentz-vector interface provides setters to different properties as well (see <a href="www.docs-to-the-lorentz-interface-setter.jl">here</a> for details).</p>

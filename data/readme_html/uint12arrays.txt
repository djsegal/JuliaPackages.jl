<p dir="auto">UInt12Arrays.jl is a Julia package to support the use of arrays of 12-bit
unsigned integers and their elements. This package's main purpose is to
provide a <code>UInt12Array</code> type that allows for the indexing into arrays of packed
<code>UInt12</code>s.</p>
<h2 dir="auto"><a id="user-content-background" class="anchor" aria-hidden="true" href="#background"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Background</h2>
<p dir="auto">Dealing with 12-bit integers is challenging because the the underlying LLVM
integer support is based on integers that are a multiple of a byte (8-bits).
For integer types that are a multiple of a byte, see
<a href="https://github.com/rfourquet/BitIntegers.jl">BitIntegers.jl</a>.</p>
<p dir="auto">To use unsigned 12-bit integers (<code>UInt12</code>) in computations, it is easiest to
load them as unsigned 16-bit integers (<code>UInt16</code>). However, to conserve memory
it may be advantageous to keep the 12-bit integers packed into a dense array.</p>
<h2 dir="auto"><a id="user-content-what-this-package-does" class="anchor" aria-hidden="true" href="#what-this-package-does"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>What this package does</h2>
<ol dir="auto">
<li>Provide a <code>UInt12Array</code> that allows for indexing of arrays of packed <code>UInt12</code>s.</li>
<li>Allow access of 12-bit integers as type <code>UInt16</code> (default element type of <code>UInt12Array</code>)</li>
<li>Provides a prototype <code>UInt12</code> type that boxes a <code>UInt16</code> and implement 12-bit arithmetic</li>
<li>Provides lookup table (LUT) and single instruction multiple data (SIMD) methods for unpacking 12-bit data</li>
</ol>
<h2 dir="auto"><a id="user-content-why-is-the-default-element-type-a-uint16-rather-than-uint12" class="anchor" aria-hidden="true" href="#why-is-the-default-element-type-a-uint16-rather-than-uint12"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why is the default element type a <code>UInt16</code> rather than <code>UInt12</code>?</h2>
<p dir="auto">As mentioned in the background, there is better hardware and compiler support
for <code>UInt16</code>. Additionally the supporting code for <code>UInt12</code> is under
development with the main implementation being a boxed <code>UInt16</code>.</p>

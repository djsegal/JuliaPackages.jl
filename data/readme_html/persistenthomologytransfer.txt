<p dir="auto">Persistent Homology Transform is produced and maintained by <br>
Yossi Bokor and Katharine Turner <br>
<a href="mailto:yossi.bokor@anu.edu.au">yossi.bokor@anu.edu.au</a> and <a href="mailto:katharine.turner@anu.edu.au">katharine.turner@anu.edu.au</a></p>
<p dir="auto">This package provides an implementation of the Persistent Homology Transform, as defined in <a href="https://arxiv.org/abs/1310.1030" rel="nofollow">Persistent Homology Transform for Modeling Shapes and Surfaces</a>. It also does Rank Functions of Persistence Diagrams, and implements <a href="https://www.sciencedirect.com/science/article/pii/S0167278916000476" rel="nofollow">Principal Component Analysis of Rank functions</a>.</p>

<h3 dir="auto">
<a id="user-content-persistent-homology-transform" class="anchor" aria-hidden="true" href="#persistent-homology-transform"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Persistent Homology Transform</h3>
<p dir="auto">Given an <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$m \times 2$</math-renderer> matrix of ordered points sampled from a simple, closed curve <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$C \subset \mathbb{R}^2$</math-renderer> (in either a clockwise or anti-clockwise direction), calculate the Persistent Homology Transform for a set of directions. You can either specify the directions explicity as a <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$n \times 2$</math-renderer> array (<code>directions::Array{Float64}(n,2)</code>), or specify an integer (<code>directions::Int64</code>) and then the directions used will be generated by</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="angles = [n*pi/(directions/2) for n in 1:directions]
directions = [[cos(x), sin(x)] for x in angles]"><pre>angles <span class="pl-k">=</span> [n<span class="pl-k">*</span><span class="pl-c1">pi</span><span class="pl-k">/</span>(directions<span class="pl-k">/</span><span class="pl-c1">2</span>) <span class="pl-k">for</span> n <span class="pl-k">in</span> <span class="pl-c1">1</span><span class="pl-k">:</span>directions]
directions <span class="pl-k">=</span> [[<span class="pl-c1">cos</span>(x), <span class="pl-c1">sin</span>(x)] <span class="pl-k">for</span> x <span class="pl-k">in</span> angles]</pre></div>
<p dir="auto">To perform the Persistent Homology Transfer for the directions, run</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="PHT(points, directions)"><pre><span class="pl-c1">PHT</span>(points, directions)</pre></div>
<p dir="auto">This outputs an array of <a href="https://github.com/Eetion/Eirene.jl">Eirene</a> Persistence Diagrams, one for each direction.</p>
<h3 dir="auto">
<a id="user-content-rank-functions" class="anchor" aria-hidden="true" href="#rank-functions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Rank Functions</h3>
<p dir="auto">Given an <a href="https://github.com/Eetion/Eirene.jl">Eirene</a> Persistence Diagram <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$D$</math-renderer>, PersistentHomologyTransfer can calculate the Rank Function <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$r_D$</math-renderer> either exactly, or given a grid of points, calculate a discretised version. Recall that <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$D$</math-renderer> is an <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$n \times 2$</math-renderer> array of points, and hence the function <code>Total_Rank_Exact</code> accepts an <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$n \times 2$</math-renderer> array of points, and returns a list of points critical points of the Rank function and the value at each of these points. Running</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="rk = Total_Rank_Exact(barcode)"><pre>rk <span class="pl-k">=</span> <span class="pl-c1">Total_Rank_Exact</span>(barcode)</pre></div>
<p dir="auto">we obtain the critical points via</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="rk[1]"><pre>rk[<span class="pl-c1">1</span>]</pre></div>
<p dir="auto">which returns an array of points in <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$\mathbb{R}^2$</math-renderer>, and the values through</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="rk[2]"><pre>rk[<span class="pl-c1">2</span>]</pre></div>
<p dir="auto">wich returns an array of integers.</p>
<p dir="auto">To obtain a discrete approximation of a Rank Function over a persistence diagram <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$D$</math-renderer>, use <code>Total_Rank_Grid</code>, which acceps as input an <a href="https://github.com/Eetion/Eirene.jl">Eirene</a> Persistence Diagram <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$D$</math-renderer>, an increasing <code>StepRange</code> for <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$x$</math-renderer>-coordinates <code>x_g</code>, and a decreasing <code>StepRange</code> for <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$y$</math-renderer>-coordinates <code>y_g</code>. The <code>StepRanges</code> are obtained by running</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="x_g = lb:delta:ub
x_g = ub:-delta:lb"><pre>x_g <span class="pl-k">=</span> lb<span class="pl-k">:</span>delta<span class="pl-k">:</span>ub
x_g <span class="pl-k">=</span> ub<span class="pl-k">:</span><span class="pl-k">-</span>delta<span class="pl-k">:</span>lb</pre></div>
<p dir="auto">with <code>lb</code> being the lower bound so that <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$(lb, lb)$</math-renderer> is the lower left corner of the grid, and <code>ub</code> being the upper bound so that <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$(ub,ub)$</math-renderer> is the top right corner of the grid, and <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$delta$</math-renderer> is the step size.</p>
<p dir="auto">Finally, the rank is obtained by</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="rk = Total_Rank_Grid(D, x_g, y_g)"><pre>rk <span class="pl-k">=</span> <span class="pl-c1">Total_Rank_Grid</span>(D, x_g, y_g)</pre></div>
<p dir="auto">which returns an array or values.</p>
<h3 dir="auto">
<a id="user-content-pca-of-rank-functions" class="anchor" aria-hidden="true" href="#pca-of-rank-functions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>PCA of Rank Functions</h3>
<p dir="auto">Given a set of rank functions, we can perform principal component analysis on them. The easiest way to do this is to use the wrapper function <code>PCA</code> which has inputs an array of rank functions evaluated at the same points (best to use <code>Total_Rank_Grid</code> to obtain them), an dimension <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$d$</math-renderer> and an array of weights <code>weights</code>, where the weights correspond to the grid points used in <code>Total_Rank_Grid</code>.</p>
<p dir="auto">To perform Principal Component Analysis and obtain the scores run</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="scores = PCA(ranks, d, weights)"><pre>scores <span class="pl-k">=</span> <span class="pl-c1">PCA</span>(ranks, d, weights)</pre></div>
<p dir="auto">which returns the scores in <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="2c4148c6220806902ee1de6421fb313d">$d$</math-renderer>-dimensions.</p>

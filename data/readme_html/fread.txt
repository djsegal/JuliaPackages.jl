<p>This packages allows you to use <a href="https://github.com/Rdatatable/data.table">R' {data.table}</a>'s excellent <code>fread</code> function to read CSVs</p>
<h2><a id="user-content-use-csvjl-instead" class="anchor" aria-hidden="true" href="#use-csvjl-instead"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Use <a href="https://github.com/JuliaData/CSV.jl">CSV.jl</a> instead</h2>
<p>You should really be using <a href="https://github.com/JuliaData/CSV.jl">CSV.jl</a> because it performs quite well. I only use Fread.jl for converting data from parquet to feature etc now and not for reading CSVs as CSV.jl is actually really good.</p>

<h2><a id="user-content-how-does-it-work-internally" class="anchor" aria-hidden="true" href="#how-does-it-work-internally"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How does it work internally?</h2>
<p>The function <code>fread</code> does a few of things</p>
<ol>
<li>Reads the CSV using <code>data.table::fread</code></li>
<li>Saves the <code>data.frame</code> in feather format</li>
<li>Loads the feather file into Julia as a <code>DataFrame</code></li>
</ol>
<p>Step 2 creates a feather file which you can set the location of by using a 2nd unnamed argument .e.g.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">fread</span>(path_to_csv, <span class="pl-s"><span class="pl-pds">"</span>path/to/out.feather<span class="pl-pds">"</span></span>)</pre></div>
<p>by default the <code>feather</code> output path is <code>path_to_csv*".feather</code> i.e. with the feather extension attached to the input file.</p>

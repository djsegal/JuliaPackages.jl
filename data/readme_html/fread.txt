<h2 dir="auto"><a id="user-content-how-does-it-work-internally" class="anchor" aria-hidden="true" href="#how-does-it-work-internally"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>How does it work internally?</h2>
<p dir="auto">The function <code>fread</code> does a few of things</p>
<ol dir="auto">
<li>Reads the CSV using <code>data.table::fread</code></li>
<li>Saves the <code>data.frame</code> in feather format</li>
<li>Loads the feather file into Julia as a <code>DataFrame</code></li>
</ol>
<p dir="auto">Step 2 creates a feather file which you can set the location of by using a 2nd unnamed argument .e.g.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="fread(path_to_csv, &quot;path/to/out.feather&quot;)"><pre><span class="pl-c1">fread</span>(path_to_csv, <span class="pl-s"><span class="pl-pds">"</span>path/to/out.feather<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">by default the <code>feather</code> output path is <code>path_to_csv*".feather</code> i.e. with the feather extension attached to the input file.</p>

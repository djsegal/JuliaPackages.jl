<p dir="auto"><a href="https://fncbook.github.io/FundamentalsNumericalComputation.jl/stable" rel="nofollow"><img src="https://camo.githubusercontent.com/c97f0a5f2ae95755f64a27f1aa8d9a17462941fd3d6c907c7630abd5d3e60acf/68747470733a2f2f696d672e736869656c64732e696f2f62616467652f646f63732d737461626c652d626c75652e737667" alt="" data-canonical-src="https://img.shields.io/badge/docs-stable-blue.svg" style="max-width: 100%;"></a></p>
<p dir="auto">These are core functions for the text <em>Fundamentals of Numerical Computation</em> by T. A. Driscoll and R. J. Braun (<a href="http://tobydriscoll.net/unlinked/fnc-preview/" rel="nofollow">preview site</a>). They are a companion to the Julia (2nd) edition, to be published in 2022.</p>
<p dir="auto"><strong>For Julia versions of the functions accompanying the MATLAB (first) edition, go to <a href="https://github.com/fncbook/fnc">https://github.com/fncbook/fnc</a> instead.</strong></p>
<h2 dir="auto"><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Installation</h2>
<ol dir="auto">
<li>
<p dir="auto"><a href="https://julialang.org/downloads/" rel="nofollow">Install Julia.</a> Any of the available methods for the latest stable version 1.x should be fine. The Julia Pro version comes with an integrated editor and many preinstalled packages, and it might be the best choice for those not comfortable with their command line shell.</p>
</li>
<li>
<p dir="auto">Start Julia on your machine. Look for the <code>julia&gt;</code> prompt.</p>
</li>
<li>
<p dir="auto">At the prompt type the <code>]</code> character. This will turn the prompt a different color and say <code>pkg&gt;</code>, indicating that you can give commands to the package manager.</p>
</li>
<li>
<p dir="auto">At the <code>pkg</code> prompt, type</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="add FundamentalsNumericalComputation"><pre>add FundamentalsNumericalComputation</pre></div>
<p dir="auto">The process will take a while. In order to flatten the learning curve, this package loads many other standard numerical and plotting packages and makes them available, so there is a lot of code to install and compile.</p>
</li>
<li>
<p dir="auto">Hit the backspace (delete on Mac) key to go back to the main Julia prompt. <strong>Steps 3-5 should only need to be done once per Julia installation.</strong></p>
</li>
</ol>
<h2 dir="auto"><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Usage</h2>
<p dir="auto">In order to use the functions, in each new Julia session you must enter</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using FundamentalsNumericalComputation"><pre><span class="pl-k">using</span> FundamentalsNumericalComputation</pre></div>
<p dir="auto">None of the functions are exported, so they must all be prefixed with the package name. However, the constant <code>FNC</code> is set as an alias to the package name to make typing more reasonable, e.g., <code>FNC.lufact</code>, <code>FNC.rk23</code>, etc.</p>
<p dir="auto">There is a bare-bones <a href="https://fncbook.github.io/FundamentalsNumericalComputation.jl/stable" rel="nofollow">documentation site</a>, but it only gives summaries of the documentation strings that are found in the text. The textbook is meant to be the real guide.</p>
<h2 dir="auto"><a id="user-content-startup-speed" class="anchor" aria-hidden="true" href="#startup-speed"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Startup speed</h2>
<p dir="auto">After installation, importing the package with a <code>using</code> statement should only take 5-10 seconds in Julia 1.6 or later. The first plot created in a Julia session may take 20-60 seconds to appear. This is a well-known irritant in the Julia ecosystem, and progress is steadily being made toward reducing the lag. In the meantime, there are a few options available for power users.</p>
<h3 dir="auto"><a id="user-content-packagecompiler" class="anchor" aria-hidden="true" href="#packagecompiler"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>PackageCompiler</h3>
<p dir="auto">The <a href="https://julialang.github.io/PackageCompiler.jl/" rel="nofollow"><code>PackageCompiler</code></a> package allows you to compile a new version of the Julia binary to get better startup performance. It's not too hard to use, but it helps if you are comfortable with command line/terminal interfaces.</p>

<p dir="auto">You can speed up loading time by installing a special branch of this package. At the prompt, type the <code>]</code> character to enter package mode. If you already installed the default version, enter</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="rm FundamentalsNumericalComputation"><pre>rm FundamentalsNumericalComputation</pre></div>
<p dir="auto">Then enter</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="add https://github.com/fncbook/FundamentalsNumericalComputation.jl#fast-load"><pre>add https<span class="pl-k">:</span><span class="pl-k">//</span>github<span class="pl-k">.</span>com<span class="pl-k">/</span>fncbook<span class="pl-k">/</span>FundamentalsNumericalComputation<span class="pl-k">.</span>jl<span class="pl-c"><span class="pl-c">#</span>fast-load</span></pre></div>
<p dir="auto">Now, each import of this package should take ten seconds or less. However, you will need to manually install and then load packages in order to run the demos and solve many of the exercises in the book. (This is how Julia is normally used in practice.) In particular, only <code>LinearAlgebra</code>, <code>Polynomials</code>, <code>SparseArrays</code>, and a few packages for loading files and displaying output are provided. Other packages used in the text, and their key dependents, are given in the following table.</p>

<p dir="auto">This package provides a Julia interface to Bitcoin's JSON-RPC API.
Only <a href="https://developer.bitcoin.org/reference/rpc/index.html" rel="nofollow">blockchain and mining RPCs</a>
have been implemented for now. They should be enough for performing on-chain analytics, which is
the primary motivation here. Remaining ones will be added later if necessary. Pull requests are
always welcome!</p>

<p dir="auto">You will need to run a full node using either the GUI from
<a href="https://bitcoin.org/en/bitcoin-core/" rel="nofollow">Bitcoin Core</a> or the headless version called <code>bitcoind</code>.
See instructions <a href="https://en.bitcoinwiki.org/wiki/Running_Bitcoind" rel="nofollow">here.</a> The daemon can also be
started with a <code>bitcoin.conf</code> configuration file, which can be generated
<a href="https://jlopp.github.io/bitcoin-core-config-generator/" rel="nofollow">here.</a></p>
<p dir="auto">The configuration file should have the RPC options enabled, which will allow our package to
communicate with the node's RPC server. Setting a username and password is a must, as those
credentials will be used for authentication. An example is shown below:</p>

<p dir="auto">Once the GUI or daemon is started, the software will download a copy of the entire blockchain onto
your disk. This means that you will have data starting from the first block (mined in 2009), also
known as the genesis block. Default download location is usually at ~/.bitcoin/, but you are free
to select another one. Keep in mind that you need ~ 424 GB space (as of 11–04–2022) in case you
want the most recent blocks. This is the recommended way to get authentic and reliable data, as it
is fetched from a trusted set of peers, which are usually other nodes running on computers around
the world. It's wise not to trust any other sources.</p>

<p dir="auto">Bio3DView.jl provides a viewer for molecular structures in <a href="https://julialang.org" rel="nofollow">Julia</a>.
It is a wrapper round the excellent <a href="http://3dmol.csb.pitt.edu" rel="nofollow">3Dmol.js</a> package [1].
When used from the REPL or a file, the viewer shows in a popup using <a href="https://github.com/JunoLab/Blink.jl">Blink.jl</a>.
When used from <a href="https://github.com/JuliaLang/IJulia.jl">IJulia</a> running in a <a href="http://jupyter.org" rel="nofollow">Jupyter</a> notebook or JupyterLab, or from <a href="https://github.com/fonsp/Pluto.jl">Pluto.jl</a>, the viewer shows in the output cell.
You can also use Bio3DView.jl to generate standalone HTML, e.g. for use in a web page.</p>
<p dir="auto">[1] Nicholas Rego and David Koes,
3Dmol.js: molecular visualization with WebGL,
Bioinformatics (2015) 31(8): 1322-1324 - <a href="http://doi.org/10.1093/bioinformatics/btu829" rel="nofollow">link</a></p>
<p dir="auto">Contributions and bug reports are welcome.</p>

<p dir="auto">Julia v1.0 or later is required.
Install Bio3DView from the package mode of the Julia REPL (press <code>]</code>):</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="add Bio3DView"><pre class="notranslate"><code>add Bio3DView
</code></pre></div>
<p dir="auto">Bio3DView uses <a href="https://github.com/MikeInnes/Requires.jl">Requires.jl</a> to minimise dependencies so you will also need to install either Blink, IJulia or Pluto to make this package useful, though you can generate HTML without them.
If you are using Blink, you will need to run <code>using Blink</code> before the Bio3DView functions work.
To view structural objects from <a href="https://github.com/BioJulia/BioStructures.jl">BioStructures.jl</a> you will need to have BioStructures.jl installed.</p>

<p>Basic bindings to <code>libogg</code> to read Ogg bitstreams.  Basic operation is to use <code>load()</code> to read in an array of packets which can then be decoded by whatever higher-level codec can use them (such as <a href="https://github.com/staticfloat/Opus.jl"><code>Opus.jl</code></a>), or use <code>save()</code> to write out a set of packets and their respective granule positions.  Manual use of this package is unusual, however if you are curious as to how <code>.ogg</code> files work, this package can act as a nice debugging tool.</p>
<p>To look into details of an <code>.ogg</code> file such as its actual pages, you must keep track of the <code>OggDecoder</code> object so you can inspect its internal fields <code>pages</code> and <code>packets</code>.  The definition of <code>load()</code> is roughly equivalent to:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="dec = OggDecoder()
Ogg.decode_all_pages(dec, fio)
Ogg.decode_all_packets(dec, fio)
"><pre>dec <span class="pl-k">=</span> <span class="pl-c1">OggDecoder</span>()
Ogg<span class="pl-k">.</span><span class="pl-c1">decode_all_pages</span>(dec, fio)
Ogg<span class="pl-k">.</span><span class="pl-c1">decode_all_packets</span>(dec, fio)</pre></div>
<p>Where <code>fio</code> is an <code>IO</code> object you wish to decode.  The fields <code>dec.pages</code> and <code>dec.packets</code> now contains much information about the <code>.ogg</code> file you have just decoded.</p>

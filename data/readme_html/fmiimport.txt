<p dir="auto"><a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a> implements the import functionalities of the FMI-standard (<a href="http://fmi-standard.org/" rel="nofollow">fmi-standard.org</a>) for the Julia programming language.
<a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a> provides the foundation for the Julia packages <a href="https://github.com/ThummeTo/FMI.jl"><em>FMI.jl</em></a> and <a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a>.</p>

<p dir="auto"><a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a> is part of <a href="https://github.com/ThummeTo/FMI.jl"><em>FMI.jl</em></a>. However, if you only need the import functionality without anything around and want to keep the dependencies as small as possible, <a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a> might be the right way to go. You can install it via:</p>
<p dir="auto">1. Open a Julia-REPL, switch to package mode using <code>]</code>, activate your preferred environment.</p>
<p dir="auto">2. Install <a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a>:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="(@v1.6) pkg&gt; add FMIImport"><pre lang="julia-repl" class="notranslate"><code>(@v1.6) pkg&gt; add FMIImport
</code></pre></div>
<p dir="auto">3. If you want to check that everything works correctly, you can run the tests bundled with <a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a>:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="(@v1.6) pkg&gt; test FMIImport"><pre lang="julia-repl" class="notranslate"><code>(@v1.6) pkg&gt; test FMIImport
</code></pre></div>
<p dir="auto">4. Have a look inside the <a href="https://github.com/ThummeTo/FMI.jl/tree/examples/examples">examples folder</a> in the examples branch or the <a href="https://thummeto.github.io/FMI.jl/dev/examples/overview/" rel="nofollow">examples section</a> of the documentation of the <a href="https://github.com/ThummeTo/FMI.jl"><em>FMI.jl</em></a> package. All examples are available as Julia-Script (<em>.jl</em>), Jupyter-Notebook (<em>.ipynb</em>) and Markdown (<em>.md</em>).</p>

<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="https://github.com/ThummeTo/FMI.jl/blob/main/docs/src/assets/FMI_JL_family.png?raw=true"><img src="https://github.com/ThummeTo/FMI.jl/raw/main/docs/src/assets/FMI_JL_family.png?raw=true" alt="FMI.jl Family" title="FMI.jl Family" style="max-width: 100%;"></a>
To keep dependencies nice and clean, the original package <a href="https://github.com/ThummeTo/FMI.jl"><em>FMI.jl</em></a> had been split into new packages:</p>
<ul dir="auto">
<li><a href="https://github.com/ThummeTo/FMI.jl"><em>FMI.jl</em></a>: High level loading, manipulating, saving or building entire FMUs from scratch</li>
<li><a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a>: Importing FMUs into Julia</li>
<li><a href="https://github.com/ThummeTo/FMIExport.jl"><em>FMIExport.jl</em></a>: Exporting stand-alone FMUs from Julia Code</li>
<li><a href="https://github.com/ThummeTo/FMICore.jl"><em>FMICore.jl</em></a>: C-code wrapper for the FMI-standard</li>
<li><a href="https://github.com/ThummeTo/FMIBuild.jl"><em>FMIBuild.jl</em></a>: Compiler/Compilation dependencies for FMIExport.jl</li>
<li><a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a>: Machine Learning with FMUs (differentiation over FMUs)</li>
<li><a href="https://github.com/ThummeTo/FMIZoo.jl"><em>FMIZoo.jl</em></a>: A collection of testing and example FMUs</li>
</ul>

<p dir="auto"><a href="https://github.com/ThummeTo/FMIImport.jl">FMIImport.jl</a> is tested (and testing) under Julia Versions <em>1.6 LTS</em> and <em>latest</em> on Windows <em>latest</em> and Ubuntu <em>latest</em>. <code>x64</code> architectures are tested. Mac and x86-architectures might work, but are not tested.</p>

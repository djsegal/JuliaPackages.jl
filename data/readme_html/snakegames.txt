<h2 dir="auto"><a id="user-content-inner-representation" class="anchor" aria-hidden="true" href="#inner-representation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Inner Representation</h2>
<p dir="auto">By default, a vector of <code>2*n_snakes+2</code> bits is used to represent the current state of each grid.</p>
<ul dir="auto">
<li>The first <code>n_snakes</code> bits are used to mark which snakes' head occupy the grid.</li>
<li>The following up <code>n_snakes</code> bits are used to mark which snakes' body occupy the grid.</li>
<li>The last two bits are used to mark whether this grid is occupied by wall/food or not.</li>
</ul>
<p dir="auto">You can access it via <code>game.board</code> and use it in your own algorithms.</p>

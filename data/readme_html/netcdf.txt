<p dir="auto">NetCDF support for the Julia programming language, there is a high-level and a medium-level interface for writing and reading netcdf files.</p>

<p dir="auto">First, load the library:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using NetCDF"><pre><span class="pl-k">using</span> NetCDF</pre></div>
<p dir="auto">The high-level interface is quite similar to the Matlab NetCDF interface, reading files is done by:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="x = ncread(&quot;myfile.nc&quot;, &quot;Radiation&quot;)"><pre>x <span class="pl-k">=</span> <span class="pl-c1">ncread</span>(<span class="pl-s"><span class="pl-pds">"</span>myfile.nc<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>Radiation<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">which will read the variable called "Radiation" from the file "myfile.nc". General information can be gained by using</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="ncinfo(filename)"><pre><span class="pl-c1">ncinfo</span>(filename)</pre></div>
<p dir="auto">which gives an overview of the dimensions, variables and attributes stored in the file.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="filename = &quot;myfile.nc&quot;
varname  = &quot;var1&quot;
attribs  = Dict(&quot;units&quot;   =&gt; &quot;mm/d&quot;,
                &quot;data_min&quot; =&gt; 0.0,
                &quot;data_max&quot; =&gt; 87.0)"><pre>filename <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>myfile.nc<span class="pl-pds">"</span></span>
varname  <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>var1<span class="pl-pds">"</span></span>
attribs  <span class="pl-k">=</span> <span class="pl-c1">Dict</span>(<span class="pl-s"><span class="pl-pds">"</span>units<span class="pl-pds">"</span></span>   <span class="pl-k">=&gt;</span> <span class="pl-s"><span class="pl-pds">"</span>mm/d<span class="pl-pds">"</span></span>,
                <span class="pl-s"><span class="pl-pds">"</span>data_min<span class="pl-pds">"</span></span> <span class="pl-k">=&gt;</span> <span class="pl-c1">0.0</span>,
                <span class="pl-s"><span class="pl-pds">"</span>data_max<span class="pl-pds">"</span></span> <span class="pl-k">=&gt;</span> <span class="pl-c1">87.0</span>)</pre></div>
<p dir="auto">Creating variables and files is done by using the nccreate command:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="nccreate(filename, varname, &quot;x1&quot;, collect(11:20), &quot;t&quot;, 20, Dict(&quot;units&quot;=&gt;&quot;s&quot;), atts=attribs)"><pre><span class="pl-c1">nccreate</span>(filename, varname, <span class="pl-s"><span class="pl-pds">"</span>x1<span class="pl-pds">"</span></span>, <span class="pl-c1">collect</span>(<span class="pl-c1">11</span><span class="pl-k">:</span><span class="pl-c1">20</span>), <span class="pl-s"><span class="pl-pds">"</span>t<span class="pl-pds">"</span></span>, <span class="pl-c1">20</span>, <span class="pl-c1">Dict</span>(<span class="pl-s"><span class="pl-pds">"</span>units<span class="pl-pds">"</span></span><span class="pl-k">=&gt;</span><span class="pl-s"><span class="pl-pds">"</span>s<span class="pl-pds">"</span></span>), atts<span class="pl-k">=</span>attribs)</pre></div>
<p dir="auto">This will create the variable called var1 in the file myfile.nc. The attributes defined in the Dict attribs are written to the file and are associated with the
newly created variable. The dimensions "x1" and "t" of the variable are called "x1" and "t" in this example. If the dimensions do not exist yet in the file,
they will be created. The dimension "x1" will be of length 10 and have the values 11..20, and the dimension "t" will have length 20 and the attribute "units"
with the value "s".</p>
<p dir="auto">Now we can write data to the file:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="d = rand(10, 20)
ncwrite(d, filename, varname)"><pre>d <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">10</span>, <span class="pl-c1">20</span>)
<span class="pl-c1">ncwrite</span>(d, filename, varname)</pre></div>
<p dir="auto">The full documentation can be found <a href="https://JuliaGeo.github.io/NetCDF.jl/dev" rel="nofollow">here</a></p>
<p dir="auto">An alternative interface for reading NetCDF files can be found here: <a href="https://github.com/Alexander-Barth/NCDatasets.jl">https://github.com/Alexander-Barth/NCDatasets.jl</a></p>

<p dir="auto">As of version 0.9 we provide an interface to DiskArrays.jl, which lets you treat NetCDF variables as Julia AbstractArrays. Special methods for accessing, reading slices and reductions and broadcasting are implemented, so that working with the arrays should be efficient.</p>
<p dir="auto">So the following returns a lazy AbstractArray container referencing the data in the variable.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="v = NetCDF.open(filename, varname)"><pre>v <span class="pl-k">=</span> NetCDF<span class="pl-k">.</span><span class="pl-c1">open</span>(filename, varname)</pre></div>

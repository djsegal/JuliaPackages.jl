<p dir="auto"><strong>Trixi.jl</strong> is a numerical simulation framework for hyperbolic conservation
laws written in <a href="https://julialang.org" rel="nofollow">Julia</a>. A key objective for the
framework is to be useful to both scientists and students. Therefore, next to
having an extensible design with a fast implementation, Trixi.jl is
focused on being easy to use for new or inexperienced users, including the
installation and postprocessing procedures. Its features include:</p>
<ul dir="auto">
<li>1D, 2D, and 3D simulations on <a href="https://trixi-framework.github.io/Trixi.jl/stable/overview/#Semidiscretizations" rel="nofollow">line/quad/hex/simplex meshes</a>
<ul dir="auto">
<li>Cartesian and curvilinear meshes</li>
<li>Conforming and non-conforming meshes</li>
<li>Structured and unstructured meshes</li>
<li>Hierarchical quadtree/octree grid with adaptive mesh refinement</li>
<li>Forests of quadtrees/octrees with <a href="https://github.com/cburstedde/p4est">p4est</a> via <a href="https://github.com/trixi-framework/P4est.jl">P4est.jl</a></li>
</ul>
</li>
<li>High-order accuracy in space and time</li>
<li>Discontinuous Galerkin methods
<ul dir="auto">
<li>Kinetic energy-preserving and entropy-stable methods based on flux differencing</li>
<li>Entropy-stable shock capturing</li>
<li>Positivity-preserving limiting</li>
<li><a href="https://github.com/ranocha/SummationByPartsOperators.jl">Finite difference summation by parts (SBP) methods</a></li>
</ul>
</li>
<li>Compatible with the <a href="https://diffeq.sciml.ai/latest/" rel="nofollow">SciML ecosystem for ordinary differential equations</a>
<ul dir="auto">
<li><a href="https://diffeq.sciml.ai/latest/solvers/ode_solve/#Low-Storage-Methods" rel="nofollow">Explicit low-storage Runge-Kutta time integration</a></li>
<li><a href="https://diffeq.sciml.ai/latest/solvers/ode_solve/#Explicit-Strong-Stability-Preserving-Runge-Kutta-Methods-for-Hyperbolic-PDEs-(Conservation-Laws)" rel="nofollow">Strong stability preserving methods</a></li>
<li>CFL-based and error-based time step control</li>
</ul>
</li>
<li>Native support for differentiable programming
<ul dir="auto">
<li>Forward mode automatic differentiation via <a href="https://github.com/JuliaDiff/ForwardDiff.jl">ForwardDiff.jl</a></li>
</ul>
</li>
<li>Periodic and weakly-enforced boundary conditions</li>
<li>Multiple governing equations:
<ul dir="auto">
<li>Compressible Euler equations</li>
<li>Magnetohydrodynamics (MHD) equations</li>
<li>Multi-component compressible Euler and MHD equations</li>
<li>Linearized Euler and acoustic perturbation equations</li>
<li>Hyperbolic diffusion equations for elliptic problems</li>
<li>Lattice-Boltzmann equations (D2Q9 and D3Q27 schemes)</li>
<li>Shallow water equations</li>
<li>Several scalar conservation laws (e.g., linear advection, Burgers' equation)</li>
</ul>
</li>
<li>Multi-physics simulations
<ul dir="auto">
<li><a href="https://github.com/trixi-framework/paper-self-gravitating-gas-dynamics">Self-gravitating gas dynamics</a></li>
</ul>
</li>
<li>Shared-memory parallelization via multithreading</li>
<li>Visualization and postprocessing of the results
<ul dir="auto">
<li>In-situ and a posteriori visualization with <a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a></li>
<li>Interactive visualization with <a href="https://makie.juliaplots.org/" rel="nofollow">Makie.jl</a></li>
<li>Postprocessing with ParaView/VisIt via <a href="https://github.com/trixi-framework/Trixi2Vtk.jl">Trixi2Vtk</a></li>
</ul>
</li>
</ul>

<p dir="auto">If you have not yet installed Julia, please <a href="https://julialang.org/downloads/platform/" rel="nofollow">follow the instructions for your

with Julia v1.8.</p>

<p dir="auto">Trixi.jl and its related tools are registered Julia packages. Hence, you
can install Trixi.jl, the visualization tool
<a href="https://github.com/trixi-framework/Trixi2Vtk.jl">Trixi2Vtk</a>,
<a href="https://github.com/SciML/OrdinaryDiffEq.jl">OrdinaryDiffEq.jl</a>, and
<a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a>
by executing the following commands in the Julia REPL:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Pkg

julia&gt; Pkg.add([&quot;Trixi&quot;, &quot;Trixi2Vtk&quot;, &quot;OrdinaryDiffEq&quot;, &quot;Plots&quot;])"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Pkg

julia<span class="pl-k">&gt;</span> Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>([<span class="pl-s"><span class="pl-pds">"</span>Trixi<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>Trixi2Vtk<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>OrdinaryDiffEq<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>Plots<span class="pl-pds">"</span></span>])</pre></div>
<p dir="auto">You can copy and paste all commands to the REPL <em>including</em> the leading
<code>julia&gt;</code> prompts - they will automatically be stripped away by Julia.
The package <a href="https://github.com/SciML/OrdinaryDiffEq.jl">OrdinaryDiffEq.jl</a>
provides time integration schemes used by Trixi.jl, while
<a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a> can be used to directly
visualize Trixi.jl's results from the REPL.</p>
<p dir="auto"><em>Note on package versions:</em> If some of the examples for how to use Trixi.jl do not
work, verify that you are using a recent Trixi.jl release by comparing the
installed Trixi.jl version from</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Pkg; Pkg.update(&quot;Trixi&quot;); Pkg.status(&quot;Trixi&quot;)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Pkg; Pkg<span class="pl-k">.</span><span class="pl-c1">update</span>(<span class="pl-s"><span class="pl-pds">"</span>Trixi<span class="pl-pds">"</span></span>); Pkg<span class="pl-k">.</span><span class="pl-c1">status</span>(<span class="pl-s"><span class="pl-pds">"</span>Trixi<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">to the <a href="https://github.com/trixi-framework/Trixi.jl/releases/latest">latest release</a>.
If the installed version does not match the current release, please check the
<em>Troubleshooting</em> section in the <a href="#documentation">documentation</a>.</p>
<p dir="auto">The commands above can also be used to update Trixi.jl. A brief list of notable
changes to Trixi.jl is available in <a href="NEWS.md"><code>NEWS.md</code></a>.</p>

<p dir="auto">In the Julia REPL, first load the package Trixi.jl</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Trixi"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Trixi</pre></div>
<p dir="auto">Then start a simulation by executing</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; trixi_include(default_example())"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">trixi_include</span>(<span class="pl-c1">default_example</span>())</pre></div>
<p dir="auto">Please be patient since Julia will compile the code just before running it.
To visualize the results, load the package Plots</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Plots"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Plots</pre></div>
<p dir="auto">and generate a heatmap plot of the results with</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; plot(sol) # No trailing semicolon, otherwise no plot is shown"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">plot</span>(sol) <span class="pl-c"><span class="pl-c">#</span> No trailing semicolon, otherwise no plot is shown</span></pre></div>
<p dir="auto">This will open a new window with a 2D visualization of the final solution:</p>
<p align="center" dir="auto">
  <a target="_blank" rel="noopener noreferrer nofollow" href="https://user-images.githubusercontent.com/26361975/177492363-74cee347-7abe-4522-8b2d-0dfadc317f7e.png"><img width="300px" src="https://user-images.githubusercontent.com/26361975/177492363-74cee347-7abe-4522-8b2d-0dfadc317f7e.png" style="max-width: 100%;"></a>
</p>
<p dir="auto">The method <code>trixi_include(...)</code> expects a single string argument with the path to a
Trixi.jl elixir, i.e., a text file containing Julia code necessary to set up and run a
simulation. To quickly see Trixi.jl in action, <code>default_example()</code>
returns the path to an example elixir with a short, two-dimensional
problem setup. A list of all example elixirs packaged with Trixi.jl can be
obtained by running <code>get_examples()</code>. Alternatively, you can also browse the
<a href="examples/"><code>examples/</code></a> subdirectory.
If you want to modify one of the elixirs to set up your own simulation,
download it to your machine, edit the configuration, and pass the file path to
<code>trixi_include(...)</code>.</p>
<p dir="auto"><em>Note on performance:</em> Julia uses just-in-time compilation to transform its
source code to native, optimized machine code at the <em>time of execution</em> and
caches the compiled methods for further use. That means that the first execution
of a Julia method is typically slow, with subsequent runs being much faster. For
instance, in the example above the first execution of <code>trixi_include</code> takes about
20 seconds, while subsequent runs require less than 60 <em>milli</em>seconds.</p>

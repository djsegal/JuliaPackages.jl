<p><a href="https://travis-ci.org/ayush1999/ONNX.jl.svg?branch=master" rel="nofollow"><img src="https://camo.githubusercontent.com/188bc33ea3d9e0a0eb291d96bb583abf2a855fa6/68747470733a2f2f7472617669732d63692e6f72672f6179757368313939392f4f4e4e582e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/ayush1999/ONNX.jl.svg?branch=master" style="max-width:100%;"></a></p>
<p>ONNX.jl : Read <a href="https://onnx.ai/" rel="nofollow">ONNX</a> graphs and load these models in Julia. ONNX.jl provides an instance of transfer learning into Julia, by reading pretrained models from ONNX format to <a href="https://github.com/FluxML/Flux.jl">Flux.jl</a>. This is done by generating the DataFlow graph from the model, and then reading it as Julia code.</p>
<h2><a id="user-content-loading-models" class="anchor" aria-hidden="true" href="#loading-models"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Loading models</h2>
<p>You need to have the <code>model.onnx</code> ( or in some cases <code>model.pb</code> ) file,  which will be read. Several pretrained ONNX model files can also be downloaded from <a href="https://github.com/onnx/models">here</a>.  Now that we have the <code>model.onnx</code> file, we can read it into Flux as :</p>
<pre><code>julia&gt; using Flux, ONNX                             # Import the required packages.
julia&gt; ONNX.load_model("model.onnx")                # If you are in some other directory, specify the entire path.
                                                    # This creates two files: model.jl and weights.bson.
julia&gt; weights = ONNX.load_weights("weights.bson")  # Read the weights from the binary serialized file.
julia&gt; model = include("model.jl")                  # Loads the model from the model.jl file.
</code></pre>
<p>And <code>model</code> is the corresponding model in Flux!</p>
<p>This package is currently under development, don't tell us we didn't warn you!</p>
<h2><a id="user-content-running-the-tests" class="anchor" aria-hidden="true" href="#running-the-tests"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Running the tests</h2>
<p>It's always better to run the tests before moving on to importing a model. The operator tests ensure that all ops are working. Use <code>]test ONNX</code> to run the tests.</p>
<ul>
<li>Running these tests may take some time, as it may initially download the test files if you don't already have them.(You need to have git preinstalled in order to download the tests)</li>
</ul>
<p>In order to read more about these tests and run model specific tests, please go through the docs in the <code>test</code> directory.</p>

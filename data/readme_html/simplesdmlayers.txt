<p dir="auto"><em><strong>MAINTENANCE MODE</strong></em> - this repo is being moved to <a href="https://github.com/PoisotLab/SpeciesDistributionToolkit.jl">a monorepo with better integration between features</a></p>
<p dir="auto"><strong>What is this package</strong>? <code>SimpleSDMLayers</code> offers a series of types, methods,
and additional helper functions to build species distribution models. It does
<em>not</em> implement any species distribution models, although there are a few
examples of how this can be done in the documentation.</p>
<p dir="auto"><strong>Who is developping this package</strong>? This package is primarily maintained by the
<a href="https://poisotlab.io/" rel="nofollow">Quantitative &amp; Computational Ecology</a> group at Université de Montréal, and
is part of the broader EcoJulia organisation.</p>
<p dir="auto"><strong>Can I sponsor this project</strong>? Sure! There is a link in the sidebar on the right.
Any money raised this way will go towards the snacks and coffee fund for students,
or any charitable cause we like to support.</p>
<p dir="auto"><strong>How can I cite this package</strong>? This repository itself can be cited through its
Zenodo archive (<a href="https://zenodo.org/record/4902317" rel="nofollow"><code>4902317</code></a>; this will generate a DOI for every
release), and there is a manuscript in <em>Journal of Open Science Software</em>
describing the package as well (<a href="https://joss.theoj.org/papers/10.21105/joss.02872" rel="nofollow"><code>10.21105/joss.02872</code></a>).</p>
<p dir="auto"><strong>Is there a manual to help with the package</strong>? Yes. You can read the
documentation for the current <a href="https://ecojulia.github.io/SimpleSDMLayers.jl/stable/" rel="nofollow">stable release</a>, which includes help on
the functions, as well as a series of tutorials and vignettes ranging from
simple analyses to full-fledged mini-studies.</p>
<p dir="auto"><strong>Don't you have some swanky badges to display</strong>? We do. They are listed at the
very end of this README.</p>
<p dir="auto"><strong>Can I contribute to this project</strong>? Absolutely. The most immediate way to
contribute is to <em>use</em> the package, see what breaks, or where the documentation
is incomplete, and <a href="https://github.com/EcoJulia/SimpleSDMLayers.jl/issues">open an issue</a>. If you have a more general question, you can
also <a href="https://github.com/EcoJulia/SimpleSDMLayers.jl/discussions">start a discussion</a>. Please read the <a href="https://github.com/EcoJulia/SimpleSDMLayers.jl/blob/master/CODE_OF_CONDUCT.md">Code of Conduct</a> and the
<a href="https://github.com/EcoJulia/SimpleSDMLayers.jl/blob/master/CONTRIBUTING.md">contributing guidelines</a>.</p>
<p dir="auto"><strong>How do I install the package</strong>? The latest tagged released can be installed
just like any Julia package: <code>]add SimpleSDMLayers</code>. To get the most of it, we
strongly suggest to also add <code>StatsPlots</code> and <code>GBIF</code>.</p>
<p dir="auto"><strong>Why are there no code examples in this README</strong>? In short, because keeping the
code in the README up to date with what the package actually does is tedious;
the documentation is built around many case studies, with richer text, and with
a more narrative style. This is where you will find the code examples and the
figures you are looking for!</p>

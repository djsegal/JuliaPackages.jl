<p>Registrator is a GitHub app that automates creation of registration pull requests for your julia packages to the <a href="https://github.com/JuliaRegistries/General">General</a> registry. Install the app below!</p>

<h2><a id="user-content-how-to-use" class="anchor" aria-hidden="true" href="#how-to-use"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>How to Use</h2>
<p>There are two ways to use Registrator: a web interface and a GitHub app.</p>
<h3><a id="user-content-via-the-web-interface" class="anchor" aria-hidden="true" href="#via-the-web-interface"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Via the Web Interface</h3>
<p>This workflow supports repositories hosted on either GitHub or GitLab.</p>
<p>Go to <a href="https://juliahub.com" rel="nofollow">https://juliahub.com</a> and log in using your GitHub or GitLab account. Then click on "Register packages" on the left menu.
There are also more detailed instructions <a href="https://juliaregistries.github.io/Registrator.jl/stable/webui/#Usage-(For-Package-Maintainers)-1" rel="nofollow">here</a>.</p>
<h3><a id="user-content-via-the-github-app" class="anchor" aria-hidden="true" href="#via-the-github-app"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Via the GitHub App</h3>
<p>Unsurprisingly, this method only works for packages whose repositories are hosted on GitHub.</p>
<p>First, install the app on your package(s) as mentioned above.  The procedure for registering a new package is the same as for releasing a new version.</p>
<ol>
<li>Set the <a href="Project.toml"><code>(Julia)Project.toml</code></a> version field in your repository to your new desired <code>version</code>.</li>
<li>Comment <code>@JuliaRegistrator register</code> on the commit/branch you want to register (e.g. like <a href="https://github.com/JuliaRegistries/Registrator.jl/issues/61#issuecomment-483486641">here</a> or <a href="https://github.com/chakravala/Grassmann.jl/commit/3c3a92610ebc8885619f561fe988b0d985852fce#commitcomment-33233149">here</a>).</li>
<li>If something is incorrect, adjust, and redo step 2.</li>
<li>If the automatic tests pass, but a moderator makes suggestions (e.g., manually updating your <code>(Julia)Project.toml</code> to include a [compat] section with version requirements for dependancies), then incorporate suggestions as you see fit into a new commit, and redo step 2 <em>for the new commit</em>.  You don't need to do anything to close out the old request.</li>
<li>Finally, either rely on the <a href="https://github.com/marketplace/actions/julia-tagbot">TagBot GitHub Action</a> to tag and make a github release or alternatively tag the release manually.</li>
</ol>
<p>Registrator will look for the project file in the master branch by default, and will use the version set in the <code>(Julia)Project.toml</code> file via, for example, <code>version = "0.1.0"</code>. To use a custom branch comment with:</p>
<pre><code>@JuliaRegistrator register branch=name-of-your-branch
</code></pre>
<p>The old pseudo-Julia syntax is also still supported:</p>
<pre><code>@JuliaRegistrator register(branch="foo")
</code></pre>
<h3><a id="user-content-transitioning-from-require-to-projecttoml" class="anchor" aria-hidden="true" href="#transitioning-from-require-to-projecttoml"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Transitioning from REQUIRE to Project.toml</h3>
<p>Download <a href="https://github.com/JuliaLang/Pkg.jl/blob/934f8b71eb436da6d2bdb30ccfc80e5e11891c5b/bin/gen_project.jl">gen_project.jl</a>, enter in your package directory and run <code>julia gen_project.jl</code>, resulting in a <code>Project.toml</code> file. You may need to do minor modifications (license, current version, description, etc.) and then remove the REQUIRE file, since it is only used for packages supporting Julia 0.6 and is otherwise irrelevant now.</p>
<p>Check that your package conforms to the required <code>Project.toml</code> structure found in the <a href="https://julialang.github.io/Pkg.jl/v1/creating-packages/" rel="nofollow">general package guidelines</a>.</p>
<h3><a id="user-content-details-for-triggering-juliaregistrator-for-step-2-above" class="anchor" aria-hidden="true" href="#details-for-triggering-juliaregistrator-for-step-2-above"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Details for triggering JuliaRegistrator (for step 2 above)</h3>
<p>Either:</p>
<ol>
<li>Open an issue and add <code> @JuliaRegistrator register</code> as a comment.  You can re-trigger the registrator by commenting <code> @JuliaRegistrator register</code> again (in case registrator reports an error or to make changes).</li>
<li>Add a comment to a commit and say <code> @JuliaRegistrator register</code>.</li>
</ol>
<p><em>Note</em>: Only <em>collaborators</em> on the package repository and <em>public members</em> on the organization the package is under are allowed to register. If you are not a collaborator, you can request a collaborator trigger registrator in a GitHub issue or a comment on a commit.</p>
<p>If you want to register as a private member you should host your own instance of Registrator, see the <a href="https://juliaregistries.github.io/Registrator.jl/stable/hosting/" rel="nofollow">documentation</a>.</p>
<h3><a id="user-content-release-notes" class="anchor" aria-hidden="true" href="#release-notes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Release notes</h3>
<p>If you have enabled TagBot on your repositories, then you may write your release notes in the same place that you trigger Registrator, or allow them to be automatically generated from closed issues and merged pull requests instead.
These can later be edited via the GitHub releases interface.</p>
<p>To write your release notes, add a section labeled "Release notes:" or "Patch notes:" to your Registrator trigger issue/comment, after the <code>@JuliaRegistrator</code> trigger. For example,</p>
<pre><code>@JuliaRegistrator register

Release notes:

Check out my new features!
</code></pre>
<p>Note that if you have not enabled TagBot, no release will be made at all, and so any release notes you write will not be used.</p>
<h3><a id="user-content-note-on-git-tags-and-github-releases" class="anchor" aria-hidden="true" href="#note-on-git-tags-and-github-releases"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Note on git tags and GitHub releases</h3>
<p>The Julia package manager <strong>does not</strong> rely on git tags and GitHub releases. However, Registrator will generate a <code>git tag</code> command for you to optionally create a corresponding tag with your package version, or you can use TagBot as is mentioned above.</p>
<h3><a id="user-content-note-on-documentation-build" class="anchor" aria-hidden="true" href="#note-on-documentation-build"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Note on documentation build</h3>
<p>The docs for your project will be automatically built by <a href="https://github.com/JuliaDocs/DocumentationGenerator.jl">DocumentationGenerator.jl</a>. Please see that repo for details. Your docs should show up at pkg.julialang.org/docs.</p>
<p>By default, <code>docs/make.jl</code> will be run to build the docs. If that is missing, the <code>README.md</code> will be used instead.</p>
<h3><a id="user-content-registering-a-package-in-a-subdirectory" class="anchor" aria-hidden="true" href="#registering-a-package-in-a-subdirectory"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Registering a package in a subdirectory</h3>
<p>If the package you want to register is in a subdirectory of your git repository, you can tell Registrator to register it by adding the <code>subdir</code> argument to your trigger, e.g. <code>@JuliaRegistrator register subdir=path/to/my/package</code>.</p>
<h2><a id="user-content-approving-pull-requests-on-the-registry" class="anchor" aria-hidden="true" href="#approving-pull-requests-on-the-registry"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Approving pull requests on the registry</h2>
<p>Pull requests that comply with the <a href="https://github.com/JuliaRegistries/RegistryCI.jl#automatic-merging-guidelines">automatic merging guidelines</a> will be merged without human intervention periodically. On other cases, a registry maintainer will manually merge the pull request made by Registrator.</p>
<h2><a id="user-content-private-packages-and-registries" class="anchor" aria-hidden="true" href="#private-packages-and-registries"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Private packages and registries</h2>
<p>Private packages will be ignored by the current running instance of Registrator.
Please see the <a href="https://juliaregistries.github.io/Registrator.jl/stable/hosting/" rel="nofollow">documentation</a> on how to host your own Registrator for private packages.</p>

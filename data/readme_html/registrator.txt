<p dir="auto">Registrator is a GitHub app that automates creation of registration pull requests for your julia packages to the <a href="https://github.com/JuliaRegistries/General">General</a> registry. Install the app below!</p>
<p dir="auto">Note: if you are registering a new package or new version in the General registry, please make sure that you have read the <a href="https://github.com/JuliaRegistries/General/blob/master/README.md">General registry README</a>.</p>

<p dir="auto">There are two ways to use Registrator: a web interface and a GitHub app.</p>
<h3 dir="auto"><a id="user-content-via-the-web-interface" class="anchor" aria-hidden="true" href="#via-the-web-interface"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Via the Web Interface</h3>
<p dir="auto">This workflow supports repositories hosted on either GitHub or GitLab.</p>
<p dir="auto">Go to <a href="https://juliahub.com" rel="nofollow">https://juliahub.com</a> and log in using your GitHub or GitLab account. Then click on "Register packages" on the left menu.
There are also more detailed instructions <a href="https://juliaregistries.github.io/Registrator.jl/stable/webui/#Usage-(For-Package-Maintainers)-1" rel="nofollow">here</a>.</p>
<h3 dir="auto"><a id="user-content-via-the-github-app" class="anchor" aria-hidden="true" href="#via-the-github-app"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Via the GitHub App</h3>
<p dir="auto">Unsurprisingly, this method only works for packages whose repositories are hosted on GitHub.</p>
<p dir="auto">The procedure for registering a new package is the same as for releasing a new version.</p>
<p dir="auto">If the registration bot is not added to the repository, <code>@JuliaRegistrator register</code> will not result in package registration.</p>
<ol dir="auto">
<li>Click on the "install" button above to add the registration bot to your repository</li>
<li>Set the <a href="Project.toml"><code>(Julia)Project.toml</code></a> version field in your repository to your new desired <code>version</code>.</li>
<li>Comment <code>@JuliaRegistrator register</code> on the commit/branch you want to register (e.g. like <a href="https://github.com/JuliaRegistries/Registrator.jl/issues/61#issuecomment-483486641" data-hovercard-type="issue" data-hovercard-url="/JuliaRegistries/Registrator.jl/issues/61/hovercard">here</a> or <a href="https://github.com/chakravala/Grassmann.jl/commit/3c3a92610ebc8885619f561fe988b0d985852fce#commitcomment-33233149">here</a>).</li>
<li>If something is incorrect, adjust, and redo step 2.</li>
<li>If the automatic tests pass, but a moderator makes suggestions (e.g., manually updating your <code>(Julia)Project.toml</code> to include a [compat] section with version requirements for dependencies), then incorporate suggestions as you see fit into a new commit, and redo step 2 <em>for the new commit</em>.  You don't need to do anything to close out the old request.</li>
<li>Finally, either rely on the <a href="https://github.com/marketplace/actions/julia-tagbot">TagBot GitHub Action</a> to tag and make a github release or alternatively tag the release manually.</li>
</ol>
<p dir="auto">Registrator will look for the project file in the master branch by default, and will use the version set in the <code>(Julia)Project.toml</code> file via, for example, <code>version = "0.1.0"</code>. To use a custom branch comment with:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="@JuliaRegistrator register branch=name-of-your-branch"><pre class="notranslate"><code>@JuliaRegistrator register branch=name-of-your-branch
</code></pre></div>
<p dir="auto">The old pseudo-Julia syntax is also still supported:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="@JuliaRegistrator register(branch=&quot;foo&quot;)"><pre class="notranslate"><code>@JuliaRegistrator register(branch="foo")
</code></pre></div>
<h3 dir="auto"><a id="user-content-triggering-new-releases-from-the-main-branch-via-the-github-app" class="anchor" aria-hidden="true" href="#triggering-new-releases-from-the-main-branch-via-the-github-app"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Triggering New Releases from the Main Branch Via the GitHub App</h3>
<p dir="auto">If TagBot has been installed as above, and the initial package registration is complete, new versions can be tagged directly on your repository.  The simplest way to trigger a change is to</p>
<ol dir="auto">
<li>Increment the <code>version = "X.X.X"</code> line in the <code>Project.toml</code> file of your package, and merge that commit to the main branch.</li>
<li>Open that commit on github and add in a comment with <code>@JuliaRegistrator register</code>.  TagBot will respond with a comment on that same commit.</li>
</ol>
<p dir="auto">If that operation is successful, a new version will be tagged and merged automatically in the general registry within approximately 15 minutes.</p>
<h3 dir="auto"><a id="user-content-transitioning-from-require-to-projecttoml" class="anchor" aria-hidden="true" href="#transitioning-from-require-to-projecttoml"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Transitioning from REQUIRE to Project.toml</h3>
<p dir="auto">Download <a href="https://github.com/JuliaLang/Pkg.jl/blob/934f8b71eb436da6d2bdb30ccfc80e5e11891c5b/bin/gen_project.jl">gen_project.jl</a>, enter in your package directory and run <code>julia gen_project.jl</code>, resulting in a <code>Project.toml</code> file. You may need to do minor modifications (license, current version, description, etc.) and then remove the REQUIRE file, since it is only used for packages supporting Julia 0.6 and is otherwise irrelevant now.</p>
<p dir="auto">Check that your package conforms to the required <code>Project.toml</code> structure found in the <a href="https://julialang.github.io/Pkg.jl/v1/creating-packages/" rel="nofollow">general package guidelines</a>.</p>
<h3 dir="auto"><a id="user-content-details-for-triggering-juliaregistrator-for-step-2-above" class="anchor" aria-hidden="true" href="#details-for-triggering-juliaregistrator-for-step-2-above"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Details for triggering JuliaRegistrator (for step 2 above)</h3>
<p dir="auto">Either:</p>
<ol dir="auto">
<li>Open an issue and add <code> @JuliaRegistrator register</code> as a comment.  You can re-trigger the registrator by commenting <code> @JuliaRegistrator register</code> again (in case registrator reports an error or to make changes).</li>
<li>Add a comment to a commit and say <code> @JuliaRegistrator register</code>.</li>
</ol>
<p dir="auto"><em>Note</em>: Only <em>collaborators</em> on the package repository and <em>public members</em> on the organization the package is under are allowed to register. If you are not a collaborator, you can request a collaborator trigger registrator in a GitHub issue or a comment on a commit.</p>
<p dir="auto">If you want to register as a private member you should host your own instance of Registrator, see the <a href="https://juliaregistries.github.io/Registrator.jl/stable/hosting/" rel="nofollow">documentation</a>.</p>
<h3 dir="auto"><a id="user-content-release-notes" class="anchor" aria-hidden="true" href="#release-notes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Release notes</h3>
<p dir="auto">If you have enabled TagBot on your repositories, then you may write your release notes in the same place that you trigger Registrator, or allow them to be automatically generated from closed issues and merged pull requests instead.
These can later be edited via the GitHub releases interface.</p>
<p dir="auto">To write your release notes, add a section labeled "Release notes:" or "Patch notes:" to your Registrator trigger issue/comment, after the <code>@JuliaRegistrator</code> trigger. For example,</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="@JuliaRegistrator register

Release notes:

Check out my new features!"><pre class="notranslate"><code>@JuliaRegistrator register

Release notes:

Check out my new features!
</code></pre></div>
<p dir="auto">Note that if you have not enabled TagBot, no release will be made at all, and so any release notes you write will not be used.</p>

<p dir="auto">The Julia package manager <strong>does not</strong> rely on git tags and GitHub releases. However, Registrator will generate a <code>git tag</code> command for you to optionally create a corresponding tag with your package version, or you can use TagBot as is mentioned above.</p>
<h3 dir="auto"><a id="user-content-note-on-documentation-build" class="anchor" aria-hidden="true" href="#note-on-documentation-build"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Note on documentation build</h3>
<p dir="auto">The docs for your project will be automatically built by <a href="https://github.com/JuliaDocs/DocumentationGenerator.jl">DocumentationGenerator.jl</a>. Please see that repo for details. Your docs should show up at pkg.julialang.org/docs.</p>
<p dir="auto">By default, <code>docs/make.jl</code> will be run to build the docs. If that is missing, the <code>README.md</code> will be used instead.</p>

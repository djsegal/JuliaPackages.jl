<p dir="auto">The first step is to load the module <code>UDUnits</code> and to initialize the unit system <code>sys</code> which correspond to the SI unit system.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using UDUnits
sys = System()"><pre><span class="pl-k">using</span> UDUnits
sys <span class="pl-k">=</span> <span class="pl-c1">System</span>()</pre></div>
<h2 dir="auto"><a id="user-content-units" class="anchor" aria-hidden="true" href="#units"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Units</h2>
<p dir="auto">The unit objects can be created for <code>m</code> and <code>cm</code> using either their symbol or their full name by indexing the <code>sys</code> object as if <code>sys</code> is a dictionary.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="m = sys[&quot;meter&quot;]
cm = sys[&quot;cm&quot;]"><pre>m <span class="pl-k">=</span> sys[<span class="pl-s"><span class="pl-pds">"</span>meter<span class="pl-pds">"</span></span>]
cm <span class="pl-k">=</span> sys[<span class="pl-s"><span class="pl-pds">"</span>cm<span class="pl-pds">"</span></span>]</pre></div>
<p dir="auto">Similarily to a dictionary, the function <code>haskey</code> is defined to determine if a unit is valid:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="haskey(sys,&quot;μm&quot;) # returns true since UDUnits knows about micrometers"><pre><span class="pl-c1">haskey</span>(sys,<span class="pl-s"><span class="pl-pds">"</span>μm<span class="pl-pds">"</span></span>) <span class="pl-c"><span class="pl-c">#</span> returns true since UDUnits knows about micrometers</span></pre></div>

<p dir="auto">Units can be derived using the usual mathemical operators. Of course, <code>m_per_s</code> could have been also create simply by using <code>sys["m/s"]</code>.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="m = sys[&quot;m&quot;]
km = sys[&quot;km&quot;]

h = sys[&quot;h&quot;]
m_per_s = m/s
km_per_h = km/h"><pre>m <span class="pl-k">=</span> sys[<span class="pl-s"><span class="pl-pds">"</span>m<span class="pl-pds">"</span></span>]
km <span class="pl-k">=</span> sys[<span class="pl-s"><span class="pl-pds">"</span>km<span class="pl-pds">"</span></span>]
s <span class="pl-k">=</span> sys[<span class="pl-s"><span class="pl-pds">"</span>s<span class="pl-pds">"</span></span>]

m_per_s <span class="pl-k">=</span> m<span class="pl-k">/</span>s
km_per_h <span class="pl-k">=</span> km<span class="pl-k">/</span>h</pre></div>
<h1 dir="auto"><a id="user-content-converting-data" class="anchor" aria-hidden="true" href="#converting-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Converting data</h1>
<p dir="auto">The function <code>areconvertible</code> returns <code>true</code> if two units are convertible:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="@show areconvertible(m_per_s,km_per_h)"><pre><span class="pl-c1">@show</span> <span class="pl-c1">areconvertible</span>(m_per_s,km_per_h)</pre></div>
<p dir="auto">To convert units, create a converter object and then apply the object to some data.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="conv = Converter(m_per_s,km_per_h)
speed_in_m_per_s = 100.
speed_in_km_per_h = conv(speed_in_m_per_s)"><pre>conv <span class="pl-k">=</span> <span class="pl-c1">Converter</span>(m_per_s,km_per_h)
speed_in_m_per_s <span class="pl-k">=</span> <span class="pl-c1">100.</span>
speed_in_km_per_h <span class="pl-k">=</span> <span class="pl-c1">conv</span>(speed_in_m_per_s)</pre></div>
<p dir="auto">The dot syntax can be used for an array as input:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="speed_in_m_per_s = [100.,110.,120.]
speed_in_km_per_h = conv.(speed_in_m_per_s)"><pre>speed_in_m_per_s <span class="pl-k">=</span> [<span class="pl-c1">100.</span>,<span class="pl-c1">110.</span>,<span class="pl-c1">120.</span>]
speed_in_km_per_h <span class="pl-k">=</span> <span class="pl-c1">conv</span>.(speed_in_m_per_s)</pre></div>

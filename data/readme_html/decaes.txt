<p dir="auto">DECAES is a <em>fast</em> Julia implementation of the <a href="https://mriresearch.med.ubc.ca/news-projects/myelin-water-fraction/" rel="nofollow">MATLAB toolbox</a> from the <a href="https://mriresearch.med.ubc.ca/" rel="nofollow">UBC MRI Research Centre</a> for computing voxelwise <a href="https://doi.org/10.1016/0022-2364(89)90011-5" rel="nofollow">T2-distributions</a> from multi spin-echo MRI images using the extended phase graph algorithm with stimulated echo corrections.
Post-processing of these T2-distributions allows for the computation of measures such as the <a href="https://doi.org/10.1002/mrm.1910310614" rel="nofollow">myelin water fraction (MWF)</a> or the <a href="https://doi.org/10.1148/radiol.2017161687" rel="nofollow">luminal water fraction (LWF)</a>.</p>
<p dir="auto">DECAES is written in the open-source <a href="https://julialang.org/" rel="nofollow">Julia programming language</a>.
Julia and command line interfaces are available through this package.
The <a href="https://github.com/jondeuce/mwiexamples">examples repository</a> additionally provides a MATLAB interface via the MATLAB function <a href="https://github.com/jondeuce/DECAES.jl/blob/master/api/decaes.m"><code>decaes.m</code></a>, as well as a Python interface via the <a href="https://github.com/jondeuce/DECAES.jl/blob/master/api/decaes.py"><code>decaes.py</code></a> module.
If you use DECAES in your research, please <a href="CITATION.bib">cite</a> our work.</p>

<h2 dir="auto"><a id="user-content-command-line-interface" class="anchor" aria-hidden="true" href="#command-line-interface"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Command Line Interface</h2>
<p dir="auto">This toolbox provides a command line interface (CLI) for processing from the terminal.
The CLI takes image files as inputs and performs one or both of T2-distribution computation and T2-parts analysis, the latter of which performs post-processing of the T2-distribution to calculate parameters such as the MWF or LWF.
The input image must be one of the following file types:</p>
<ol dir="auto">
<li><a href="https://nifti.nimh.nih.gov/" rel="nofollow">NIfTI file</a> with extension <code>.nii</code>, or <a href="https://www.gzip.org/" rel="nofollow">gzip</a> compressed NIfTI file with extension <code>.nii.gz</code></li>
<li><a href="https://www.mathworks.com/help/matlab/import_export/mat-file-versions.html" rel="nofollow">MATLAB file</a> with extension <code>.mat</code></li>
<li>Philips <a href="https://www.nitrc.org/plugins/mwiki/index.php/dcm2nii:MainPage#Philips_PAR.2FREC_Images" rel="nofollow">PAR/REC</a> file pair with extensions <code>.par</code> and <code>.rec</code> (or <code>.PAR</code> and <code>.REC</code>)</li>
<li>Philips XML/REC file pair with extensions <code>.xml</code> and <code>.rec</code> (or <code>.XML</code> and <code>.REC</code>)</li>
</ol>
<p dir="auto">All output files are saved as <code>.mat</code> files in format <code>v7.3</code>.</p>
<ul dir="auto">
<li><strong>Note:</strong> if your data is in DICOM format, the <a href="https://www.nitrc.org/plugins/mwiki/index.php/dcm2nii:MainPage" rel="nofollow">freely available <code>dcm2niix</code> tool</a> is able to convert <a href="https://www.nitrc.org/plugins/mwiki/index.php/dcm2nii:MainPage#General_Usage" rel="nofollow">DICOM</a> files into NIfTI format.</li>
</ul>

<p dir="auto">There are two equivalent ways to use the CLI, assuming DECAES is already installed:</p>
<p dir="auto"><strong>1. Helper script:</strong> Create a script called e.g. <code>decaes.jl</code> with the following contents (or, download the script located <a href="https://github.com/jondeuce/DECAES.jl/blob/master/api/decaes.jl">here</a>):</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using DECAES # load the package
main() # call CLI entrypoint function"><pre><span class="pl-k">using</span> DECAES <span class="pl-c"><span class="pl-c">#</span> load the package</span>
<span class="pl-c1">main</span>() <span class="pl-c"><span class="pl-c">#</span> call CLI entrypoint function</span></pre></div>
<p dir="auto">This script can then be invoked from the command line as follows:</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="$ julia --threads=auto decaes.jl -- &lt;COMMAND LINE ARGS&gt; # --threads=auto enables parallel processing"><pre>$ julia --threads=auto decaes.jl -- <span class="pl-k">&lt;</span>COMMAND LINE ARGS<span class="pl-k">&gt;</span> <span class="pl-c"><span class="pl-c">#</span> --threads=auto enables parallel processing</span></pre></div>
<p dir="auto"><strong>2. Julia <code>-e</code> flag:</strong> The contents of the above script can be written directly at the command line using the <code>-e</code> (for "evaluate") flag:</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="$ julia --threads=auto -e 'using DECAES; main()' -- &lt;COMMAND LINE ARGS&gt; # --threads=auto enables parallel processing"><pre>$ julia --threads=auto -e <span class="pl-s"><span class="pl-pds">'</span>using DECAES; main()<span class="pl-pds">'</span></span> -- <span class="pl-k">&lt;</span>COMMAND LINE ARGS<span class="pl-k">&gt;</span> <span class="pl-c"><span class="pl-c">#</span> --threads=auto enables parallel processing</span></pre></div>

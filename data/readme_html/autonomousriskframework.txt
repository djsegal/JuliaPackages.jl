<h4 dir="auto"><a id="user-content-python-version-conflicts" class="anchor" aria-hidden="true" href="#python-version-conflicts"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Python version conflicts</h4>
<p dir="auto">Some versions of Python e.g., 3.9 are incompatible with the framework as they do not support packages such as <code>pytorch</code> that are needed.
It is possible to switch to a working version of Python as follows:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Conda
Conda.add(&quot;python=3.7.5&quot;)"><pre><span class="pl-k">using</span> Conda
Conda<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>python=3.7.5<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">However, note that if you were using an incompatible of Python before, you might have installed Python packages of versions
that can conflict with the new compatible version of Python as the packages remain under the Conda directory.
You may see error messages like the following if this is the case:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="Package enum34 conflicts for:

pyopenssl -&gt; cryptography[version='&gt;=2.8'] -&gt; enum34

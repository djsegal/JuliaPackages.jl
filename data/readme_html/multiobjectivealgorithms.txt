(MOA) is a collection of algorithms for multi-objective optimization.</p>

<span class="pl-c1">set_attribute</span>(model, MOA<span class="pl-k">.</span><span class="pl-c1">Algorithm</span>(), MOA<span class="pl-k">.</span><span class="pl-c1">Dichotomy</span>())
<span class="pl-c1">set_attribute</span>(model, MOA<span class="pl-k">.</span><span class="pl-c1">SolutionLimit</span>(), <span class="pl-c1">4</span>)</pre></div>
<p dir="auto">Replace <code>HiGHS.Optimizer</code> with an optimizer capable of solving a
single-objective instance of your optimization problem.</p>
<p dir="auto">You may set additional optimizer attributes, the supported attributes depend on
the choice of solution algorithm.</p>

<p dir="auto">Set the algorithm using the <code>MOA.Algorithm()</code> attribute.</p>
<p dir="auto">The value must be one of the algorithms supported by MOA:</p>
<ul dir="auto">
<li><code>MOA.Chalmet()</code></li>
<li><code>MOA.Dichotomy()</code></li>
<li><code>MOA.DominguezRios()</code></li>
<li><code>MOA.EpsilonConstraint()</code></li>
<li><code>MOA.Hierarchical()</code></li>
<li><code>MOA.KirlikSayin()</code></li>
<li><code>MOA.Lexicographic()</code> [default]</li>
<li><code>MOA.TambyVanderpooten()</code></li>
</ul>
<p dir="auto">Consult their docstrings for details.</p>

<p dir="auto">There are a number of optimizer attributes supported by the algorithms in MOA.</p>
<p dir="auto">Each algorithm supports only a subset of the attributes. Consult the algorithm's
docstring for details on which attributes it supports, and how it uses them in
the solution process.</p>
<ul dir="auto">
<li><code>MOA.EpsilonConstraintStep()</code></li>
<li><code>MOA.LexicographicAllPermutations()</code></li>
<li><code>MOA.ObjectiveAbsoluteTolerance(index::Int)</code></li>
<li><code>MOA.ObjectivePriority(index::Int)</code></li>
<li><code>MOA.ObjectiveRelativeTolerance(index::Int)</code></li>
<li><code>MOA.ObjectiveWeight(index::Int)</code></li>
<li><code>MOA.SolutionLimit()</code></li>
<li><code>MOI.TimeLimitSec()</code></li>
</ul>

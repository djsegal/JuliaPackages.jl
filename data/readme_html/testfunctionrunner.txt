<p dir="auto">A rather hacky solution to define tests in a package.  In particular, this
package takes care of some ugly details for dealing with:</p>
<ul dir="auto">
<li>doctest (or any <code>eval</code>-based testing)</li>
<li>tests that require loading test code for Distributed.jl</li>
</ul>
<p dir="auto">This package is aiming at a clear separation of test <em>definition</em> and test
<em>execution</em> phases.</p>
<h2 dir="auto"><a id="user-content-quick-start" class="anchor" aria-hidden="true" href="#quick-start"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Quick start</h2>
<p dir="auto">To create a <code>Pkg.test</code>-compatible test entry point <code>test/runtests.jl</code>:</p>
<ol dir="auto">
<li>
<p dir="auto">Create a package at <code>test/$TestPackageName/</code> (i.e., there are
<code>test/$TestPackageName/Project.toml</code> and
<code>test/$TestPackageName/src/$TestPackageName.jl</code>).</p>
</li>
<li>
<p dir="auto">Write tests as functions that takes no argument and the name starting with
<code>test_</code>.  Use <code>Test.@test</code> etc. as usual to assert the result.</p>
</li>
<li>
<p dir="auto">Create <code>test/runtests.jl</code> with the following code:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using TestFunctionRunner
TestFunctionRunner.@run"><pre><span class="pl-k">using</span> TestFunctionRunner
TestFunctionRunner<span class="pl-k">.</span><span class="pl-c1">@run</span></pre></div>
</li>
</ol>
<p dir="auto"><strong>Note:</strong> Test dependencies must be listed in both <code>test/Project.toml</code> and
<code>test/$TestPackageName/Project.toml</code>.</p>
<p dir="auto">Test package or its sub-modules can also be executed using
<code>TestFunctionRunner.run(module)</code>.</p>

<p dir="auto">Timeout (in seconds) for <strong>each</strong> test function. If the function takes more time
than the specified limit, <strong>entire <code>julia</code> process</strong> that is running the test
will be terminated. TestFunctionRunner.jl tries to print stack trace if
possible.</p>
<p dir="auto">Note: This works only in Unix systems.</p>
<p dir="auto">Environment variable <code>TEST_FUNCTION_RUNNER_JL_TIMEOUT</code> can be used for setting
the default value of <code>timeout</code>. Function and macro arguments overrides the

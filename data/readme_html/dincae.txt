<h1 dir="auto"><a id="user-content-dincaejl" class="anchor" aria-hidden="true" href="#dincaejl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>DINCAE.jl</h1>
<p dir="auto">DINCAE (Data-Interpolating Convolutional Auto-Encoder) is a neural network to reconstruct missing data in satellite observations.
This repository contains the Julia port of DINCAE. The original <a href="https://github.com/gher-uliege/DINCAE">Python code</a> is no longer maintained.</p>
<p dir="auto">Utilities (for plotting and data preparation) are available in a separate repository
<a href="https://github.com/gher-uliege/DINCAE_utils.jl">https://github.com/gher-uliege/DINCAE_utils.jl</a></p>
<p dir="auto">The method is described in the following articles:</p>
<ul dir="auto">
<li>Barth, A., Alvera-Azcárate, A., Licer, M., &amp; Beckers, J.-M. (2020). DINCAE 1.0: a convolutional neural network with error estimates to reconstruct sea surface temperature satellite observations. Geoscientific Model Development, 13(3), 1609–1622. <a href="https://doi.org/10.5194/gmd-13-1609-2020" rel="nofollow">https://doi.org/10.5194/gmd-13-1609-2020</a></li>
<li>Barth, A., Alvera-Azcárate, A., Troupin, C., &amp; Beckers, J.-M. (2022). DINCAE 2.0: multivariate convolutional neural network with error estimates to reconstruct sea surface temperature satellite and altimetry observations. Geoscientific Model Development, 15(5), 2183–2196. <a href="https://doi.org/10.5194/gmd-15-2183-2022" rel="nofollow">https://doi.org/10.5194/gmd-15-2183-2022</a></li>
</ul>
<p dir="auto">(click <a href="CITATION.bib">here</a> for the BibTeX entry).</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="examples/Fig/data-avg_2001-09-12.png"><img src="examples/Fig/data-avg_2001-09-12.png" alt="" style="max-width: 100%;"></a>
Panel (a) is the original data where we have added clouds (panel (b)). The reconstuction based on the data in panel (b) is shown in panel (c) together
with its expected standard deviation error (panel (d))</p>
<p dir="auto">DINCAE is indented to be used with a <a href="https://en.wikipedia.org/wiki/Graphics_processing_unit" rel="nofollow">GPU</a> with <a href="https://en.wikipedia.org/wiki/CUDA" rel="nofollow">CUDA</a> support (NVIDIA GPU). The code can also run on a <a href="https://en.wikipedia.org/wiki/Central_processing_unit" rel="nofollow">CPU</a> but which will be quite slow.</p>

<p dir="auto">You need <a href="https://julialang.org/downloads" rel="nofollow">Julia</a> (version 1.7 or later) to run <code>DINCAE</code>. The command line interface of Julia is sufficient for <code>DINCAE</code>.
If you are using Linux (on a x86_64 CPU), installing and running Julia 1.8.5 is as easy as running these shell commands:</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="curl https://julialang-s3.julialang.org/bin/linux/x64/1.8/julia-1.8.5-linux-x86_64.tar.gz | tar -xzf -
julia-1.8.5/bin/julia"><pre>curl https://julialang-s3.julialang.org/bin/linux/x64/1.8/julia-1.8.5-linux-x86_64.tar.gz <span class="pl-k">|</span> tar -xzf -
julia-1.8.5/bin/julia</pre></div>
<p dir="auto">This installs Julia in the current directory under the folder <code>julia-1.8.5</code>.
For more information and other platforms, please see <a href="https://julialang.org/downloads/platform/" rel="nofollow">platform specific instructions</a> for further installation instructions.
You can check the latest available version of Julia at <a href="https://julialang.org/downloads" rel="nofollow">https://julialang.org/downloads</a> but avoid beta releases and release candidates if you are new to Julia.</p>
<p dir="auto">Inside a Julia terminal, you can download and install <code>DINCAE</code> and <code>DINCAE_utils</code> by issuing these commands:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Pkg
Pkg.add(url=&quot;https://github.com/gher-uliege/DINCAE.jl&quot;, rev=&quot;main&quot;)

Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(url<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>https://github.com/gher-uliege/DINCAE_utils.jl<span class="pl-pds">"</span></span>, rev<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>main<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto"><code>DINCAE.jl</code> depends on <code>Knet.jl</code> and <code>CUDA.jl</code> which will automatically be installed. More information is available at <a href="https://denizyuret.github.io/Knet.jl/latest/install/" rel="nofollow">https://denizyuret.github.io/Knet.jl/latest/install/</a> and <a href="https://cuda.juliagpu.org/stable/" rel="nofollow">https://cuda.juliagpu.org/stable/</a>.</p>
<p dir="auto">After this, you should be able to load <code>DINCAE</code> with:</p>

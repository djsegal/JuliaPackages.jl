<p>Type-based static analysis for the Julia programming language.</p>
<p>There are three main checks you can run: <code>checkreturntypes</code>, <code>checklooptypes</code>, and <code>checkmethodcalls</code>.
Running a check on a function checks each method; running a check on a module checks each function (by checking each method of each function).</p>
<h3><a id="user-content-installing-this-package" class="anchor" aria-hidden="true" href="#installing-this-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installing This Package</h3>
<p>To use any of these functions, you'll need to <code>Pkg.add("TypeCheck")</code> once to install the package on your computer and then import it using <code>using TypeCheck</code>. You'll need to re-import every time you restart the REPL.</p>
<h3><a id="user-content-checkreturntypes-do-the-return-types-of-your-functions-depend-on-the-types-not-the-values-of-your-arguments" class="anchor" aria-hidden="true" href="#checkreturntypes-do-the-return-types-of-your-functions-depend-on-the-types-not-the-values-of-your-arguments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>checkreturntypes</code>: do the return types of your functions depend on the types, not the values of your arguments?</h3>
<p>It is considered good style in Julia to have the return type of functions depend only on their argument types, not on the argument values.
This function tries to check that you did so.</p>
<p>You can run this on a generic function or on a module:</p>
<ul>
<li><code>checkreturntypes(istext)</code></li>
<li><code>checkreturntypes(Base)</code></li>
</ul>
<p>It is only effective at catching functions with annotated argument types.</p>
<p>It will catch things like:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="julia&gt; foo1(x::Int) = isprime(x) ? x: false
foo1 (generic function with 1 method)

julia&gt; checkreturntypes(foo1)
foo1(Int64)::Union(Bool,Int64)
"><pre><code>julia&gt; foo1(x::Int) = isprime(x) ? x: false
foo1 (generic function with 1 method)

julia&gt; checkreturntypes(foo1)
foo1(Int64)::Union(Bool,Int64)
</code></pre></div>
<p>However, it will not catch:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="julia&gt; foo2(x) = isprime(x) ? x : false
foo2 (generic function with 1 method)

julia&gt; checkreturntypes(foo2)

"><pre><code>julia&gt; foo2(x) = isprime(x) ? x : false
foo2 (generic function with 1 method)

julia&gt; checkreturntypes(foo2)

</code></pre></div>
<p>Additionally, it does a check to see if the return type of the function depends on a function call in the return statement.
This prevents the analysis from complaining about every function that calls a "bad" function.
However, it's possible that this silences too many alerts.</p>
<h3><a id="user-content-checklooptypes-do-the-variables-in-your-loops-have-stable-types" class="anchor" aria-hidden="true" href="#checklooptypes-do-the-variables-in-your-loops-have-stable-types"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>checklooptypes</code>: do the variables in your loops have stable types?</h3>
<p>A common performance problem is having unstable (numeric) variable types in an important loop.
Having stable types within loops allows Julia's JIT compiler to output code as fast as a static compiler;
having unstable types means resorting to slower, dynamic code.</p>
<p>You can run this on a generic function or on a module:</p>
<ul>
<li><code>checklooptypes(sort)</code></li>
<li><code>checklooptypes(Base)</code></li>
</ul>
<p>It will complain about:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="julia&gt; function barr1()

	x::Union(Float64,Int64)

<h3><a id="user-content-checkmethodcalls-could-your-functions-have-run-time-nomethoderrors" class="anchor" aria-hidden="true" href="#checkmethodcalls-could-your-functions-have-run-time-nomethoderrors"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>checkmethodcalls</code>: could your functions have run-time NoMethodErrors?</h3>
<p><code>NoMethodError</code>s are probably the most common error in Julia. This is an attempt to find them statically.</p>
<p>You can run this on a generic function or on a module:</p>
<ul>
<li><code>checkmethodcalls(sort)</code></li>
<li><code>checkmethodcalls(Base)</code></li>
</ul>
<p>This functionality is still clearly imperfect. I'm working on refining it to be more useful.</p>

<p>####<code>methodswithdescendants(t::DataType;onlyleaves::Bool=false,lim::Int=10)</code></p>

It returns a list of (Symbol,Float64) tuples, where the Symbol is the name of a function and the Float64 is the percentage of subtypes whose <code>methodswith</code> shows a result for that function.</p>

<p dir="auto">The purpose of this package is to provide a <code>DistributedDiscreteModel</code> implementation (a parallel mesh data structure, see <code>GridapDistributed.jl</code> for more details) able to handle forests of quadtrees/octrees of the computational domain. To this end, it leverages the <a href="https://p4est.github.io/" rel="nofollow"><code>p4est</code> software library</a> meshing engine under the hood (click <a href="https://github.com/gridap/GridapP4est.jl/blob/main/test/UniformlyRefinedForestOfOctreesDiscreteModelsTests.jl">here</a> for an example). At present, it only supports uniformly refined meshes, but the plan is that it also supports locally adapted meshes with hanging nodes in the future.</p>

<p dir="auto">Before using <code>GridapP4est.jl</code>, we have to build <code>MPI.jl</code> and

<h2><a id="user-content-dressing-up-your-algorithms-with-documentation-driven-development" class="anchor" aria-hidden="true" href="#dressing-up-your-algorithms-with-documentation-driven-development"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Dressing up your algorithms with documentation-driven development</h2>
<p>Implementing technical and scientific algorithms in code starts with a thorough understanding of the problem
that is being solved and how a particular algorithm solves that problem. Tuxedo provides a framework for
transforming theory found in publications and technical papers into documented, testable, and usable code.</p>
<p>A proper and well-dressed algorithm implementation starts with naming it and documenting its purpose. Tuxedo
provides a macro in the Julia environment that scaffolds documentation, function definition and unit tests.</p>

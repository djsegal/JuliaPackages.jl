<p dir="auto">Create and maintain local registries for Julia packages. This package
is intended to provide a simple workflow for maintaining local
registries (private or public) without making any assumptions about
how the registry or the packages are hosted.</p>
<p dir="auto">For registration in the General registry, see
<a href="https://github.com/JuliaComputing/Registrator.jl">Registrator</a>.</p>
<p dir="auto">For serving local packages through a <a href="https://github.com/JuliaLang/Pkg.jl/issues/1377" data-hovercard-type="issue" data-hovercard-url="/JuliaLang/Pkg.jl/issues/1377/hovercard">Package
Server</a> see the
companion package
<a href="https://github.com/GunnarFarneback/LocalPackageServer.jl">LocalPackageServer</a>.</p>

<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using LocalRegistry
create_registry(name, repository_url, description = &quot;My private registry&quot;)"><pre class="notranslate"><code>using LocalRegistry
create_registry(name, repository_url, description = "My private registry")
</code></pre></div>
<p dir="auto">This prepares a registry with the given name in the standard
location for registries. Review the result and <code>git push</code> it
manually. When created in this way the registry is automatically
activated and the next section can be skipped.</p>
<p dir="auto">The repository at <code>repository_url</code> must be able to be pushed to,
for example by being a bare repository. <code>repository_url</code> itself
can be an URL, or a path to a local git repository.</p>
<p dir="auto">The registry can also be created at a specified path, with a specified
branch, or being automatically pushed. See the documentation string
for details.</p>

<p dir="auto">To activate the registry, do</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using Pkg

<p dir="auto">This only needs to be done once per Julia installation.

<h2 dir="auto"><a id="user-content-register-a-package" class="anchor" aria-hidden="true" href="#register-a-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Register a Package</h2>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using LocalRegistry
register(package; registry = registry)"><pre class="notranslate"><code>using LocalRegistry
register(package; registry = registry)
</code></pre></div>
<p dir="auto">Register the new <code>package</code> in the registry <code>registry</code>. The version
number and other information is obtained from the package's
<code>Project.toml</code>. The easiest way to specify <code>package</code> and <code>registry</code> is
by name as strings. See the documentation string for more options.</p>
<p dir="auto">Notes:</p>
<ul dir="auto">
<li>You need to have a clean working copy of your registry.</li>
<li>The package must be stored as a git working copy, e.g. having been
cloned with <code>Pkg.develop</code>.</li>
<li>The package must be available in the current <code>Pkg</code> environment.</li>
<li>The package must have a <code>Project.toml</code> or <code>JuliaProject.toml</code> file.</li>
<li>There is no checking that the dependencies are available in any
registry.</li>
<li>If you have exactly one installed registry beside the <code>General</code>
registry, it is not necessary to specify <code>registry</code>.</li>
<li>By default the registry changes are <code>git push</code>ed to the upstream
registry repository.</li>
</ul>

<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using LocalRegistry
register(package)"><pre class="notranslate"><code>using LocalRegistry

<p dir="auto">When adding a new version of a package, the registry can be
omitted. The new version number is obtained from the <code>version</code> field
of the package's <code>Project.toml</code> file.</p>

<p dir="auto">If you start Julia with the <code>--project</code> flag or use <code>Pkg.activate</code> to
activate a developed package, this package can be registered simply by</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using LocalRegistry
register()"><pre class="notranslate"><code>using LocalRegistry
register()
</code></pre></div>
<p dir="auto">This is also sufficient for registering a new package, provided that
you have exactly one installed registry beside the <code>General</code> registry.</p>
<p dir="auto">If you run <code>register()</code> but the active project is not a package, it
will look for a package in the current directory.</p>

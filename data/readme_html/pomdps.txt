<p>This package provides a core interface for working with <a href="https://en.wikipedia.org/wiki/Markov_decision_process" rel="nofollow">Markov decision processes (MDPs)</a> and <a href="https://en.wikipedia.org/wiki/Partially_observable_Markov_decision_process" rel="nofollow">partially observable Markov decision processes (POMDPs)</a>. For examples, please see <a href="https://github.com/JuliaPOMDP/POMDPExamples.jl">POMDPExamples</a>, <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl">QuickPOMDPs</a>, and the <a href="https://github.com/JuliaPOMDP/POMDPGallery.jl">Gallery</a>.</p>
<p>Our goal is to provide a common programming vocabulary for:</p>
<ol>
<li>Expressing problems as MDPs and POMDPs.</li>
<li>Writing solver software.</li>
<li>Running simulations efficiently.</li>
</ol>
<p>There are <a href="http://juliapomdp.github.io/POMDPs.jl/stable/def_pomdp" rel="nofollow">nested interfaces for expressing and interacting with (PO)MDPs</a>: When the <em><a href="http://juliapomdp.github.io/POMDPs.jl/stable/explicit" rel="nofollow">explicit</a></em> interface is used, the transition and observation probabilities are explicitly defined using api <a href="http://juliapomdp.github.io/POMDPs.jl/stable/explicit/#functional-form-explicit-pomdp" rel="nofollow">functions</a>; when the <em><a href="http://juliapomdp.github.io/POMDPs.jl/stable/generative" rel="nofollow">generative</a></em> interface is used, only a single step simulator (e.g. (s', o, r) = G(s,a)) needs to be defined. Problems may also be defined with probability <a href="http://juliapomdp.github.io/POMDPs.jl/stable/explicit/#tabular-form-explicit-pomdp" rel="nofollow">tables</a>, or with the simplified <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl">QuickPOMDPs interfaces</a>.</p>
<p><strong>Python</strong> can be used to define and solve MDPs and POMDPs via the QuickPOMDPs or tabular interfaces and <a href="https://github.com/JuliaPy/pyjulia">pyjulia</a> (Example: <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl/blob/master/examples/tiger.py">tiger.py</a>).</p>
<p>For help, please post to the <a href="https://groups.google.com/forum/#!forum/pomdps-users" rel="nofollow">Google group</a>, or on <a href="https://gitter.im/JuliaPOMDP" rel="nofollow">gitter</a>. Check <a href="https://github.com/JuliaPOMDP/POMDPs.jl/releases">releases</a> for information on changes. POMDPs.jl and all packages in the JuliaPOMDP project are fully supported on Linux and OS X. Windows is supported for all native solvers*, and most non-native solvers should work, but may require additional configuration.</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<p>To install POMDPs.jl, run the following in the Julia REPL:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Pkg; <span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg</span>"</span>add POMDPs<span class="pl-pds">"</span></span></pre></div>
<p>To install supported JuliaPOMDP packages including various solvers, first add the JuliaPOMDP registry:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Pkg; <span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg</span>"</span>registry add https://github.com/JuliaPOMDP/Registry<span class="pl-pds">"</span></span></pre></div>
<p>Note: to use this registry, <a href="https://juliacomputing.com/products/juliapro" rel="nofollow">JuliaPro</a> users must also run <code>edit(normpath(Sys.BINDIR,"..","etc","julia","startup.jl"))</code>, comment out the line <code>ENV["DISABLE_FALLBACK"] = "true"</code>, save the file, and restart JuliaPro as described in <a href="https://github.com/JuliaPOMDP/POMDPs.jl/issues/249">this issue</a>.</p>
<p>You can then list packages with <code>POMDPs.available()</code> and install a solver (say <code>SARSOP.jl</code>) with</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Pkg; <span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg</span>"</span>add SARSOP<span class="pl-pds">"</span></span></pre></div>
<h2><a id="user-content-quick-start" class="anchor" aria-hidden="true" href="#quick-start"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Quick Start</h2>
<p>To run a simple simulation of the classic <a href="https://www.cs.rutgers.edu/~mlittman/papers/aij98-pomdp.pdf" rel="nofollow">Tiger POMDP</a> using a policy created by the QMDP solver, you can use the following code (Note that this uses the simplified <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl">Discrete Explicit interface from QuickPOMDPs.jl</a>; the <a href="https://juliapomdp.github.io/POMDPs.jl/stable/def_pomdp/" rel="nofollow">main interface</a> and the <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl">Quick Interface</a> have much more expressive power):</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> POMDPs, QuickPOMDPs, POMDPSimulators, QMDP

S <span class="pl-k">=</span> [<span class="pl-c1">:left</span>, <span class="pl-c1">:right</span>]
A <span class="pl-k">=</span> [<span class="pl-c1">:left</span>, <span class="pl-c1">:right</span>, <span class="pl-c1">:listen</span>]
O <span class="pl-k">=</span> [<span class="pl-c1">:left</span>, <span class="pl-c1">:right</span>]
γ <span class="pl-k">=</span> <span class="pl-c1">0.95</span>

<span class="pl-k">function</span> <span class="pl-en">T</span>(s, a, sp)
    <span class="pl-k">if</span> a <span class="pl-k">==</span> <span class="pl-c1">:listen</span>
        <span class="pl-k">return</span> s <span class="pl-k">==</span> sp
    <span class="pl-k">else</span> <span class="pl-c"><span class="pl-c">#</span> a door is opened</span>
        <span class="pl-k">return</span> <span class="pl-c1">0.5</span> <span class="pl-c"><span class="pl-c">#</span>reset</span>
    <span class="pl-k">end</span>
<span class="pl-k">end</span>

<span class="pl-k">function</span> <span class="pl-en">Z</span>(a, sp, o)
    <span class="pl-k">if</span> a <span class="pl-k">==</span> <span class="pl-c1">:listen</span>
        <span class="pl-k">if</span> o <span class="pl-k">==</span> sp
            <span class="pl-k">return</span> <span class="pl-c1">0.85</span>
        <span class="pl-k">else</span>
            <span class="pl-k">return</span> <span class="pl-c1">0.15</span>
        <span class="pl-k">end</span>
    <span class="pl-k">else</span>
        <span class="pl-k">return</span> <span class="pl-c1">0.5</span>
    <span class="pl-k">end</span>
<span class="pl-k">end</span>

<span class="pl-k">function</span> <span class="pl-en">R</span>(s, a)
    <span class="pl-k">if</span> a <span class="pl-k">==</span> <span class="pl-c1">:listen</span>  
        <span class="pl-k">return</span> <span class="pl-k">-</span><span class="pl-c1">1.0</span>
    <span class="pl-k">elseif</span> s <span class="pl-k">==</span> a <span class="pl-c"><span class="pl-c">#</span> the tiger was found</span>
        <span class="pl-k">return</span> <span class="pl-k">-</span><span class="pl-c1">100.0</span>
    <span class="pl-k">else</span> <span class="pl-c"><span class="pl-c">#</span> the tiger was escaped</span>
        <span class="pl-k">return</span> <span class="pl-c1">10.0</span>
    <span class="pl-k">end</span>
<span class="pl-k">end</span>

m <span class="pl-k">=</span> <span class="pl-c1">DiscreteExplicitPOMDP</span>(S,A,O,T,Z,R,γ)

solver <span class="pl-k">=</span> <span class="pl-c1">QMDPSolver</span>()
policy <span class="pl-k">=</span> <span class="pl-c1">solve</span>(solver, m)

rsum <span class="pl-k">=</span> <span class="pl-c1">0.0</span>
<span class="pl-k">for</span> (s,b,a,o,r) <span class="pl-k">in</span> <span class="pl-c1">stepthrough</span>(m, policy, <span class="pl-s"><span class="pl-pds">"</span>s,b,a,o,r<span class="pl-pds">"</span></span>, max_steps<span class="pl-k">=</span><span class="pl-c1">10</span>)
    <span class="pl-c1">println</span>(<span class="pl-s"><span class="pl-pds">"</span>s: <span class="pl-v">$s</span>, b: <span class="pl-v">$([<span class="pl-c1">pdf</span>(b,s) <span class="pl-k">for</span> s <span class="pl-k">in</span> S])</span>, a: <span class="pl-v">$a</span>, o: <span class="pl-v">$o</span><span class="pl-pds">"</span></span>)
    <span class="pl-k">global</span> rsum <span class="pl-k">+=</span> r
<span class="pl-k">end</span>
<span class="pl-c1">println</span>(<span class="pl-s"><span class="pl-pds">"</span>Undiscounted reward was <span class="pl-v">$rsum</span>.<span class="pl-pds">"</span></span>)</pre></div>
<p>For more examples with visualization see <a href="https://github.com/JuliaPOMDP/POMDPGallery.jl">POMDPGallery.jl</a>.</p>

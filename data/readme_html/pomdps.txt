<p>This package provides a core interface for working with <a href="https://en.wikipedia.org/wiki/Markov_decision_process" rel="nofollow">Markov decision processes (MDPs)</a> and <a href="https://en.wikipedia.org/wiki/Partially_observable_Markov_decision_process" rel="nofollow">partially observable Markov decision processes (POMDPs)</a>. For examples, please see <a href="https://github.com/JuliaPOMDP/POMDPExamples.jl">POMDPExamples</a>, <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl">QuickPOMDPs</a>, and the <a href="https://github.com/JuliaPOMDP/POMDPGallery.jl">Gallery</a>.</p>
<p>Our goal is to provide a common programming vocabulary for:</p>
<ol>
<li>Expressing problems as MDPs and POMDPs.</li>
<li>Writing solver software.</li>
<li>Running simulations efficiently.</li>
</ol>
<p>There are <a href="http://juliapomdp.github.io/POMDPs.jl/stable/def_pomdp" rel="nofollow">several ways to define (PO)MDPs</a>:</p>
<ul>
<li>Transition and observation distributions and rewards can be defined separately with explicit or <a href="https://juliapomdp.github.io/POMDPModelTools.jl/stable/distributions/#Implicit" rel="nofollow">implicitly sampled</a> distributions.</li>
<li>All of the dynamics can be defined in a single <a href="https://en.wikipedia.org/wiki/Markov_decision_process#Simulator_models" rel="nofollow">generative model</a> function: (s', o, r) = G(s,a).</li>
<li>Problems may be defined with probability <a href="https://github.com/JuliaPOMDP/POMDPExamples.jl/blob/master/notebooks/Defining-a-tabular-POMDP.ipynb">tables</a>.</li>
<li>The <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl">QuickPOMDPs interfaces</a> make defining simple problems easier.</li>
</ul>
<p>POMDPs.jl integrates with other ecosystems:</p>
<ul>
<li>The <a href="https://github.com/JuliaPOMDP/POMDPModelTools.jl">POMDPModelTools package</a> provides <a href="https://juliapomdp.github.io/POMDPModelTools.jl/stable/common_rl/#CommonRLInterface-Integration" rel="nofollow">two-way integration</a> with <a href="https://github.com/JuliaReinforcementLearning/CommonRLInterface.jl">CommonRLInterface</a> and therefore with the <a href="https://github.com/JuliaReinforcementLearning/ReinforcementLearning.jl">JuliaReinforcementLearning packages</a>.</li>
<li>Python can be used to define and solve MDPs and POMDPs via the QuickPOMDPs or tabular interfaces and <a href="https://github.com/JuliaPy/pyjulia">pyjulia</a> (Example: <a href="https://github.com/JuliaPOMDP/QuickPOMDPs.jl/blob/master/examples/tiger.py">tiger.py</a>).</li>
</ul>
<p>For help, please post in <a href="https://github.com/JuliaPOMDP/POMDPs.jl/discussions">GitHub Discussions tab</a>. We welcome contributions from anyone! See <a href="/CONTRIBUTING.md">CONTRIBUTING.md</a> for information about contributing.</p>

<p>POMDPs.jl and associated solver packages can be installed using <a href="https://docs.julialang.org/en/v1/stdlib/Pkg/" rel="nofollow">Julia's package manager</a>. For example, to install POMDPs.jl and the QMDP solver package, type the following in the Julia REPL:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using Pkg; Pkg.add(&quot;POMDPs&quot;); Pkg.add(&quot;QMDP&quot;)
"><pre><span class="pl-k">using</span> Pkg; Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>POMDPs<span class="pl-pds">"</span></span>); Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>QMDP<span class="pl-pds">"</span></span>)</pre></div>
<p>Some auxiliary packages and older versions of solvers may be found in the JuliaPOMDP registry. To install this registry, see the <a href="https://juliapomdp.github.io/POMDPs.jl/stable/install/" rel="nofollow">installation instructions</a>.</p>
<h2><a id="user-content-quick-start" class="anchor" aria-hidden="true" href="#quick-start"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Quick Start</h2>
<p>To run a simple simulation of the classic <a href="https://people.csail.mit.edu/lpk/papers/aij98-pomdp.pdf" rel="nofollow">Tiger POMDP</a> using a policy created by the QMDP solver, you can use the following code (note that POMDPs.jl is not limited to discrete problems with explicitly-defined distributions like this):</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using POMDPs, QuickPOMDPs, POMDPModelTools, POMDPSimulators, QMDP

m = QuickPOMDP(
    states = [&quot;left&quot;, &quot;right&quot;],
    actions = [&quot;left&quot;, &quot;right&quot;, &quot;listen&quot;],
    observations = [&quot;left&quot;, &quot;right&quot;],
    initialstate = Uniform([&quot;left&quot;, &quot;right&quot;]),
    discount = 0.95,

    transition = function (s, a)
        if a == &quot;listen&quot;
            return Deterministic(s) # tiger stays behind the same door
        else # a door is opened
            return Uniform([&quot;left&quot;, &quot;right&quot;]) # reset
        end
    end,

    observation = function (s, a, sp)
        if a == &quot;listen&quot;
            if sp == &quot;left&quot;
                return SparseCat([&quot;left&quot;, &quot;right&quot;], [0.85, 0.15]) # sparse categorical distribution
            else
                return SparseCat([&quot;right&quot;, &quot;left&quot;], [0.85, 0.15])
            end
        else
            return Uniform([&quot;left&quot;, &quot;right&quot;])
        end
    end,

    reward = function (s, a)
        if a == &quot;listen&quot;
            return -1.0
        elseif s == a # the tiger was found
            return -100.0
        else # the tiger was escaped
            return 10.0
        end
    end
)

solver = QMDPSolver()
policy = solve(solver, m)

rsum = 0.0
for (s,b,a,o,r) in stepthrough(m, policy, &quot;s,b,a,o,r&quot;, max_steps=10)
    println(&quot;s: $s, b: $([pdf(b,s) for s in states(m)]), a: $a, o: $o&quot;)
    global rsum += r
end
println(&quot;Undiscounted reward was $rsum.&quot;)

<p>Many packages use the POMDPs.jl interface, including MDP and POMDP solvers, support tools, and extensions to the POMDPs.jl interface. POMDPs.jl and all packages in the JuliaPOMDP project are fully supported on Linux and OS X. Windows is supported for all native solvers*, and most non-native solvers should work, but may require additional configuration.</p>

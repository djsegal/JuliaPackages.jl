<p dir="auto">The following camera calibration / projection models were found and consolidated in to a common <code>CameraCalibration</code> type with many helper functions to facility downstream adoption of this package:</p>
<ul dir="auto">
<li><code>JuliaRobotics/SensorFeatureTracking.jl</code>: <code>PinholeCamera</code>, <code>CameraIntrinsics</code>, <code>CameraExtrinsics</code>,<code>CameraModelFull</code></li>
<li><code>yakir12/CameraModels</code>: <code>Pinhole</code></li>
<li><code>JuliaImages/CameraGeometry.jl</code></li>
<li>More models to consolidate: <a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="759324696" data-permission-text="Title is private" data-url="https://github.com/JuliaRobotics/CameraModels.jl/issues/1" data-hovercard-type="issue" data-hovercard-url="/JuliaRobotics/CameraModels.jl/issues/1/hovercard?comment_id=1144740912&amp;comment_type=issue_comment" href="https://github.com/JuliaRobotics/CameraModels.jl/issues/1#issuecomment-1144740912">#1 (comment)</a></li>

<p dir="auto">The image convention is intended to be the best compromise between <a href="https://github.com/JuliaImages/Images.jl">JuliaImages's unified Array stack</a>, and established <a href="https://opencv.org/" rel="nofollow">OpenCV</a> conventions.  Also see <a href="https://evizero.github.io/Augmentor.jl/stable/images/#Working-with-Images-in-Julia" rel="nofollow">Augmentor.jl's Images description</a>, and open issues to help resolve miscommunication or convention issues.  The following figure is hosted in this repo, with raw image for easy update via <a href="docs/imgs/JuliaRoboticsImgConv.excalidraw">Excalidraw.com</a>.</p>

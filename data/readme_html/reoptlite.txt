PLEASE USE <a href="https://github.com/nrel/reopt.jl">REopt.jl</a>]</h1>
<p dir="auto">This package is currently under development and only has a subset of capabilities of the REopt Lite model used in the <a href="https://github.com/NREL/REopt_Lite_API">REopt Lite API</a>. For example, the CHP model and thermal loads are not yet modeled in this code. However this code is easier to use than the API (only dependencies are Julia and a solver) and has a novel model for uncertain outages.</p>
<p dir="auto">Note: this package has been tested with Julia 1.4, 1.5, and 1.6 and may not be compatible with older versions.</p>
<p dir="auto">For more information please see the documentation:</p>

<p dir="auto">The full details of the model will be published in <em>Laws et al. 2021, Co-Optimizing Distributed Energy Resources for Grid-connected Benefits and Resilience Benefits Under Uncertain Grid Reliability, [Submitted]</em>. In brief, the model is set up to minimize the maximum expected outage cost (while minimizing the lifecycle cost of energy including utility tariff costs), where the maximum is taken over outage start times, and the expectation is taken over outage durations.</p>

<p dir="auto">Evaluating only <code>PV</code> and <code>Storage</code> requires a linear program solver. Adding a generator and/or multiple outages makes the problem mixed-integer linear, and thus requires a MILP solver. See <a href="https://jump.dev/JuMP.jl/stable/installation/" rel="nofollow">https://jump.dev/JuMP.jl/stable/installation/</a> for a list of solvers. The REopt Lite package has been tested with <code>Xpress</code>, <code>Cbc</code> and <code>CPLEX</code>.</p>

<h2 dir="auto"><a id="user-content-basic-principles" class="anchor" aria-hidden="true" href="#basic-principles"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Basic principles</h2>
<p dir="auto">Names of GMPE functions looks like: <code>gmpe_{Name_of_gmpe_function}</code>. For example: <code>gmpe_as2008</code>, where <code>as2008</code> is Abrahamson and Silva 2008 GMPE Model. The configuration for a model (see <code>examples/*.conf</code>) has <code>ground_motion_type</code> that can be <code>PGA</code>,<code>PGV</code>,<code>PSA</code> and define the type of output data points.</p>
<p dir="auto">Each GMPE function has at least 2 methods: for calculation based on input VS30-grid or without any grid.</p>
<h3 dir="auto"><a id="user-content-grid-case" class="anchor" aria-hidden="true" href="#grid-case"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>GRID case</h3>
<p dir="auto">The GMPE function for each grid's point calculates <code>{pga/pgv/psa}</code> values using <code>latitude</code>, <code>longitude</code> [degrees for WGS84 ellipsoid] and <code>VS30</code> [m/s]. The output data has return in custom type (depends by config) where latitude and longitude are copy from input grid and <code>pga/pgv/pgd/psa</code> is calculated by function.</p>
<p dir="auto">For example: the function <code>gmpe_as2008</code> with parameters</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pga_as2008(eq::Earthquake,
           config_as2008::Params_as2008,
           grid::Array{Point_vs30};
           min_val::Number)"><pre><span class="pl-c1">pga_as2008</span>(eq<span class="pl-k">::</span><span class="pl-c1">Earthquake</span>,
           config_as2008<span class="pl-k">::</span><span class="pl-c1">Params_as2008</span>,
           grid<span class="pl-k">::</span><span class="pl-c1">Array{Point_vs30}</span>;
           min_val<span class="pl-k">::</span><span class="pl-c1">Number</span>)</pre></div>
<p dir="auto">where <code>ground_motion_type = "PGA"</code> at <code>config</code>, returns 1-d is <code>Array{Point_pga_out}</code> with points based on input grid and <code>pga &gt; min_val</code> (<code>pga</code> is Acceleration of gravity in percent (%g) rounded to <code>ggg.gg</code>).</p>
<h3 dir="auto"><a id="user-content-without-grid" class="anchor" aria-hidden="true" href="#without-grid"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Without grid</h3>
<p dir="auto">In case of without any grid GMPE functions return simple 1-d <code>Array{Float64}</code> with <code>{pga/pgv/pgd/psa}</code> data. It calculates from epicenter to <code>distance</code> with <code>1</code> [km] step perpendicularly to the epicenter.</p>
<p dir="auto">Example:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pga_as2008(eq::Earthquake,
           config::Params_as2008;
           VS30::Number=350,
           distance::Int64=1000)"><pre><span class="pl-c1">pga_as2008</span>(eq<span class="pl-k">::</span><span class="pl-c1">Earthquake</span>,
           config<span class="pl-k">::</span><span class="pl-c1">Params_as2008</span>;
           VS30<span class="pl-k">::</span><span class="pl-c1">Number</span><span class="pl-k">=</span><span class="pl-c1">350</span>,

<p dir="auto">where <code>ground_motion_type = "PGA"</code> at <code>config</code>, return is <code>Array{Float64}</code> with <code>1:distance</code> values of <code>pga</code> (also rounded to <code>ggg.gg</code>).</p>

<p dir="auto"><strong>About Dl variable</strong></p>
<p dir="auto">The <code>Dl</code> is the top depth to the layer whose S-wave velocity is <code>l</code> (in <code>[m/s]</code>) at the site. Actually it should be another one grid with <code>Dl</code> depths on each grid point (<code>Point_vs30_dl</code> type). If you pass grid without <code>Dl</code>, then <code>Dl</code> variable pass to GMPE functions as a constant.</p>

<p dir="auto"><code>StationXML</code> is mainly designed to be used by other modules such
as <a href="https://github.com/anowacki/Seis.jl">Seis</a> to process
station information.  Therefore, relatively few convenience functions
are defined for working with <code>FDSNStationXML</code> objects.  However,
the package aims to comprehensively document all objects and functions
used.</p>
<p dir="auto">The basic type exported by StationXML.jl is <code>FDSNStationXML</code>.</p>
<h3 dir="auto"><a id="user-content-reading-fdsn-stationxml-data" class="anchor" aria-hidden="true" href="#reading-fdsn-stationxml-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Reading FDSN StationXML data</h3>
<p dir="auto">Two unexported functions are available for use in creating <code>FDSNStationXML</code> objects:</p>
<ul dir="auto">
<li><code>StationXML.read(filename)</code>: Read from a file on disk.</li>
<li><code>StationXML.readstring(string)</code>: Parse from a <code>String</code>.</li>
</ul>

<p dir="auto">Simply call <code>write</code> on an <code>FDSNStationXML</code> object to write it to disk or other
<code>IO</code> stream:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; write(&quot;output_file.xml&quot;, sxml)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">write</span>(<span class="pl-s"><span class="pl-pds">"</span>output_file.xml<span class="pl-pds">"</span></span>, sxml)</pre></div>
<p dir="auto">StationXML.jl <strong>always</strong> writes files according to the v1.1 schema.</p>
<p dir="auto">Note that v1.1 removed a small number of fields from the specification.  Therefore, if
you are writing an <code>FDSNStationXML</code> object read from a v1.0 file, there is the potential
that information may be lost.  If you are worried, pass the <code>warn=true</code> keyword argument
to <code>write</code> to enable warnings for the presence of any fields which will not be written.</p>

<h4 dir="auto"><a id="user-content-merge" class="anchor" aria-hidden="true" href="#merge"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>merge[!]</code></h4>
<p dir="auto">You can merge together mulitiple <code>FDSNStationXML</code> objects with <code>merge</code> (returining
a copy and not modifying the originals), or <code>merge!</code>, which updates the first
object given.  In situations where it is obvious that two networks, stations
or channels are the same, these will not be duplicated in the final merged
object.  Cases which are ambiguous are not merged and the user is warned by
default.</p>

<p dir="auto">As an alternative to <code>merge!</code>, one can simply <code>append!</code> two objects together.
This simply has the effect of copying everything in one <code>FDSNStationXML</code>
object into another, and duplication is not avoided.</p>

<p dir="auto"><code>StationXML</code> represents the XML as laid out in the StationXML schema.
Therefore, it contains all the information in a StationXML file which
is part of the StationXML standard.  Elements and attributes of the XML
are fields within structures nested several layers deep.</p>

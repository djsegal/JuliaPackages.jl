<p><strong>Debugging Tip</strong> <em>Homebrew and other installers tend to install very old versions, for some reason.  The only way to effectively check that you are running the most recent version of Julia is to visit the downloads page (<a href="https://julialang.org/downloads/" rel="nofollow">https://julialang.org/downloads/</a>) and compare with your current install.</em></p>
<p>Installing Julia is simple.  Download the most recent version of Julia at <a href="https://julialang.org/downloads/" rel="nofollow">https://julialang.org/downloads/</a>.  Open a Julia shell and type <code>]</code> to enter the Package management environment.  Then run  <code>add Eirene</code>:</p>
<pre><code>(v1.1) pkg&gt; add Eirene
</code></pre>
<p>It's fairly common to get error messages when you do this.  Generally they will want you to install additional (supporting) packages.  Read these messages, and follow the instructions.  When in doubt, <em>FIRST</em> check that you are running the latest version of Julia, and use</p>
<pre><code>(v1.1) pkg&gt; update
</code></pre>
<p>to ensure that you are using the latest versions of the supporting packages.  <em>SECOND</em>, close the Julia shell.  Some updates will only take effect after re-opening.</p>
<p><strong>Debugging Tip</strong> <em>Pay special attention to red error messages.  You can solve nine out of ten installation problems by copy/pasting text from these directly into the REPL.</em></p>
<p><strong>Debugging Tip</strong> <em>Wait times of an hour or more have been reported.  The install will finish eventually!</em></p>

<h2><a id="user-content-keyword-arguments" class="anchor" aria-hidden="true" href="#keyword-arguments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Keyword arguments</h2>
<p>Consider the following line of code</p>
<pre><code>julia&gt; C = eirene(x, model = "pc")
</code></pre>
<p>In this expression, <code>model = "pc"</code> tells Eirene that the columns of <code>x</code> should be treated as a collection of points in Euclidean space. The expression <code>model</code> is a <em>keyword argument</em>, and the expression <code>"pc"</code> is the <em>value</em> of keyword <code>model</code>.  Every keyword comes with a default value; you only have to declare a value if you want something other than a default.</p>
<p><code>model = "pc", "vr", "complex"</code></p>
<ul>
<li>States format of the data.</li>
<li>Possible values: <code>"pc"</code> (point cloud), <code>"vr"</code> (vietoris-rips), <code>"complex"</code>.   Default value: <code>"vr"</code>.</li>
<li>See 'Complexes' below for special</li>
</ul>
<p><code>maxdim = k</code></p>
<ul>
<li>Compute persistent homology in dimensions 0, ..., k.</li>
<li>Default value: <code>1</code>.</li>
</ul>
<p><code>minrad = t</code></p>
<ul>
<li>Compute homology from time <code>t</code> onward.</li>
<li>Default value: <code>-Inf</code>.</li>
</ul>
<p><code>maxrad = t</code></p>
<ul>
<li>Stop computing homology after time <code>t</code>.</li>
<li>Devalue value <code>Inf</code>.</li>
</ul>
<p><code>numrad = N</code></p>
<ul>
<li>Divide the interval from <code>minrad</code> to <code>maxrad</code> into <code>N</code> equally spaced steps, and compute the homology of each step.  If the value of numrad is set to <code>Inf</code>, then homology will computed at every time point.</li>
<li>Possible values: any positive integer, or <code>Inf</code>.  Default value: <code>Inf</code>.</li>
<li>This keyword argument is currently only available for <code>model = "vr"</code>.</li>
</ul>
<p><code>record = "cyclerep", "none"</code></p>
<ul>
<li>Determines wether or not to compute generators.</li>
<li>Possible values: <code>"cyclerep"</code>, <code>"none"</code>.  Default value: <code>"cyclerep"</code>.</li>
</ul>

<h3><a id="user-content-vocabulary-the-faces-of-a-cell" class="anchor" aria-hidden="true" href="#vocabulary-the-faces-of-a-cell"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Vocabulary: the faces of a cell</h3>
<p>We'll talk a lot about cell complexes.  Roughly speaking, a cell complex is an object constructed by "gluing" together simple building blocks, called <em>cells</em>.  For us, the details aren't important.  What <em>is</em> important is that each cell has a dimension.   A simplicial complex is a cell complex built from vertices (dimension 0 cells), edges (dimension 1 cells), triangles (dimension 2 cells), tetrahedra (dimension 3 cells), and so on.  A cubical complex is built from vertices (dimension 0 cells), edges (dimension 1 cells), squares (dimension 2 cells), cubes (dimension 3 cells), etc.  The <strong>faces</strong> of a cell are the cells of lower dimension that "touch" it.  For example, the faces of an edge e in a simple graph are the two vertices that touch e.  The faces of a triangle, A, in a simplicial complex are the edges and vertices that touch A.  We say that x is a <strong>codimension-1 face</strong> of Y if x touches Y, and dim(x) = dim(Y)-1.  Thus, while both vertices and edges can be faces of a triangle in a simplicial complex, only <em>edges</em> can be codimension-1 faces.</p>
<h3><a id="user-content-example-1" class="anchor" aria-hidden="true" href="#example-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Example</h3>
<p>This section gives a practical overview on customized (filtered) topological spaces.  For complete details, see the following sections.  As an example of a (filtered) topological space, we'll use an undirected  graph with with two vertices, <code>v1</code> and <code>v2</code>, and one edge, <code>e1</code>.  To begin, let's assign unique ID numbers to the cells of <code>G</code> (see table below).</p>
<p><strong>Debugging  tip</strong> ID numbers must be assigned in ascending order with respect to the dimension of cells (e.g. cells of dimension 0 receive lower id numbers than cells of dimension 1). **</p>
<p><strong>Debugging tip</strong> Numbering must start at 1, not 0.**</p>
<pre><code>i = cell id number (user-assigned)   |   new cell label    |   original cell label
----------------------------------------------------------------------------------
1                                    |   c1                |   v1
2                                    |   c2                |   v2
3                                    |   c3                |   e1
0                                    |   n/a               |   n/a
</code></pre>
<p>These ID numbers determine the entries of the boundary matrix <code>D</code>.  For example, <code>D[1,3] = 1</code> because <code>c1</code> is a face of <code>c3</code>, and <code>D[1,2] = 0</code> because <code>c1</code> is not a face of <code>c2</code>.</p>
<pre><code>julia&gt; D = [0 0 1; 0 0 1; 0 0 0]
3×3 Array{Int64,2}:
 0  0  1
 0  0  1
 0  0  0
</code></pre>
<p>Suppose that <em>G</em> is the last in a nested sequence of graphs <code>G1, G2, G3</code> where <code>G1</code> is just the vertex <code>v1</code> and <code>G2</code> is the (unconnected) pair of vertices <code>v1, v2</code>.  Suppose that <code>G1</code>, <code>G2</code>, and <code>G3</code> appear at times <code>t1 =0.01</code>, <code>t2 =0.02</code>, and <code>t3 =0.03</code>, respectively.  Then</p>
<pre><code>i = cell id number   |   fv[i]   |   dv[i]   |   ev[i]   |   dp[i]   |   rv[i]   |   cp[i]   |
----------------------------------------------------------------------------------------------
0                      undefined   undefined   undefined   undefined   undefined   undefined
1                        0.01        0           2           1           1           1
2                        0.02        0           1           3           2           1
3                        0.03        1           0           4         undefined     3
4                      undefined   undefined   undefined     4         undefined   undefined
5                      undefined   undefined   undefined   undefined   undefined   undefined
</code></pre>
<p>That is,</p>
<pre><code>fv = [0.01,0.02,0.03]
dv = [0,0,1]
ev = [2,1,0]
dp = [1,2,4,4]
rv = [1,2]
cp = [1,1,3]
</code></pre>

General formats</h3>
<p><strong>Debugging Tip</strong> Julia is <em><strong>1-indexed</strong></em>.  For an example, <code>v[1]</code> is the first entry of a list or vector <code>v</code>.  Thus, when assigning unique ID numbers to cells, numbering should start at 1, not 0.</p>
<p>There are several ways to format the data of a cell complex <code>E</code>.  Here are the main ingredients.</p>
<ul>
<li>number the cells <code>1, ..., N</code> in ascending order, according to dimension</li>
<li>let <code>D</code> be the <code>N x N</code> zero/one matrix with <code>D[i,j] = 1</code> iff <code>i</code> is a codimension-1 face of cell <code>j</code></li>
<li>define vectors <code>rv</code> and <code>cp</code> by the following procedure (see Julia documentation for a more direct interpretation)</li>
</ul>
<pre><code>julia&gt; S  = sparse(D)
julia&gt; rv = S.rowval
julia&gt; cp = S.colptr
</code></pre>
<p>Define vectors <code>dv</code>, <code>ev</code>, and <code>dp</code> such that</p>
<ul>
<li><code>dv[i]</code> is the dimension of cell <code>i</code></li>
<li><code>ev[k]</code> is the total number of cells with dimension <code>k-1</code></li>
<li><code>dp[k]</code> is <em>1 plus</em> the number of cells of dimension <em>strictly less than</em> <code>k-1</code></li>
</ul>
<p>If in addition we have a nested sequence of complexes <code>E_0 ≤ ... ≤ E_n = E</code>, then let <code>fv</code> be the vector such that</p>
<ul>
<li><code>fv[i]</code> is the birthtime of cell <code>i</code></li>
</ul>
<h3><a id="user-content-2-sparse-column-format" class="anchor" aria-hidden="true" href="#2-sparse-column-format"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>2. Sparse column format</h3>
<p>Eirene has a keyword argument for every vector defined in the general formatting section.  In the example below, you can substitute either <code>ev = ev</code> or <code>dp=dp</code> in place of <code>dv=dv</code>.  Eirene only needs one of the three.</p>
<p><em><strong>Example computation</strong></em></p>
<pre><code>julia&gt; C = eirene(rv=rv,cp=cp,dv=dv,fv=fv)
</code></pre>

Simple format (from file)</h3>
<h4><a id="user-content-ideal-for-beginners---well-suited-to-small-hand-written-data--slow-for-big-data" class="anchor" aria-hidden="true" href="#ideal-for-beginners---well-suited-to-small-hand-written-data--slow-for-big-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>ideal for beginners //  well suited to small, hand-written data // slow for big data</h4>
<p>This format is handy if you want to write files by hand, or if you need to visually proof-read your data to make sure it's formatted correctly.  Large files with this format can take a long time to load (in fact, much longer than the persistence computation itself), so if you experiene performance issues, please consider another input format.  Recall that rows must be arranged in <strong>sorted according to dimension</strong>.  In practice, this means that the entries in the first column should increase from top to bottom.</p>
<pre><code>dv[1], fv[1], &lt;id #'s for the codimension-1 faces of cell 1, separated by commas&gt;
dv[2], fv[2], &lt;id #'s for the codimension-1 faces of cell 2, separated by commas&gt;
...
dv[N], fv[N], &lt;id #'s for the codimension-1 faces of cell N, separated by commas&gt;
</code></pre>
<p><em><strong>Example</strong></em>  For our example space, the file would take form:</p>
<pre><code>0, 0.01
0, 0.02
1, 0.03, 1, 2
</code></pre>
<p>If this file is saved as <code>Users/Adam/ez.csv</code>, then to compute PH call</p>
<pre><code>julia&gt; C = eirene("Users/Adam/complex.csv",model="complex",entryformat="sp")
</code></pre>
<h3><a id="user-content-4-sparse-column-format-from-file" class="anchor" aria-hidden="true" href="#4-sparse-column-format-from-file"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>4. Sparse column format (from file)</h3>
<p>You can store <code>E</code> as a comma separated .txt or .csv file with four lines</p>
<pre><code>dp_1, ..., dp_m
fv_1, ..., fv_n
rv_1, ..., rv_p
cp_1, ..., cp_q
</code></pre>
<p>where line one is <code>dp</code>, line 2 is <code>fv</code>, line 3 is <code>rv</code>, and line 4 is <code>cp</code>.</p>
<ul>
<li>If you know <code>D</code> and want to figure out what <code>rv</code> and <code>cp</code> should be, you can either view the Julia docs on sparse matrices or google "sparse column format".</li>
<li>Instead of <code>dp</code>, you can put either <code>dv</code> or <code>ev</code> in the first line.</li>
</ul>
<p><em><strong>Example files</strong></em></p>
<p>Recall that</p>
<pre><code>fv = [0.01,0.02,0.03]
dv = [0,0,1]
ev = [2,1,0]
dp = [1,2,4,4]
rv = [1,2]
cp = [1,1,3]
</code></pre>
<p>To use the <code>dp</code> format option, you could write a file like so.</p>
<pre><code>1,2,4,4
0.01, 0.02, 0.03
1,2
1,1,3
</code></pre>
<p>To use the <code>ev</code> format option, replace the top row with <code>ev</code>:</p>
<pre><code>2,1,0
0.01, 0.02, 0.03
1,2
1,1,3
</code></pre>
<p><em><strong>Example computations</strong></em></p>
<p>Suppose the first file is saved as <code>desktop/DP.csv</code>.  To compute persistent homology, run:</p>
<pre><code>julia&gt; C =eirene("desktop/DP.csv",model="complex",  entryformat=  "dp")
</code></pre>
<p>Suppose the second file is saved as <code>desktop/EV.csv</code>.  To compute persistent homology, run:</p>
<pre><code>julia&gt; C =eirene("desktop/EV.csv",model="complex",  entryformat=  "ev")
</code></pre>
<h2><a id="user-content-barcodes-betti-curves-and-persistence-diagrams" class="anchor" aria-hidden="true" href="#barcodes-betti-curves-and-persistence-diagrams"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Barcodes, Betti Curves, and Persistence Diagrams</h2>
<br>
Eirene stores a list of every persistent homology class (that is, every bar) in the output variable `C`.  To find the birth and death times of each class, run
<pre><code>julia&gt; A = barcode(C, dim=k)
</code></pre>
<p>Variable <code>A</code> will be an <code>n x 2</code> matrix, where <code>n</code> is the number of persistent homology classes of dimension <code>k</code>.  The pth persistent homology class in Eirene's list is born at time <code>A[p,1]</code> and dies at time <code>A[p,2]</code>.  To obtain the betti curve, run</p>
<pre><code>julia&gt; B = betticurve(C, dim=k)
</code></pre>
<p>Variable <code>B</code> is an <code>n x 2</code> array for which <code>B[j,2]</code> is the <code>k</code>th betti number of the space at time <code>t = B[j,1]</code>.</p>
<p>To plot the persistence diagram, betti curve, and barcode in dimension <code>k</code>, run the following.  Note  that <code>_pjs</code> simply refers to PlotlyJS, with is the Julia package used for graphing.</p>
<pre><code>julia&gt; plotpersistencediagram_pjs(C, dim=k)`
julia&gt; plotbetticurve_pjs(C, dim=k)
julia&gt; plotbarcode_pjs(C, dim=k)
</code></pre>
<ul>
<li>Keyword <code>dim</code> defaults to <code>1</code>.</li>
<li>A class that never dies will appear as red dot on the diagonal at the time of its birth.</li>
<li>Hovering the cursor over a point in the persistence diagram will show a message with its precise Euclidean coordinates (birth and death times) and two additional pieces of information:  <code>class</code> and <code>size</code>.  Variable <code>class</code> is an integer.  If <code>class = p</code>, then the point where the cursor is hovering represents the birth/death time of <code>p</code>th persistent homology class in Eirene's list.  It is born at time <code>A[p,1]</code> and dies at time <code>A[p,2]</code>.  Variable <code>size</code> is also an integer.  It is the number of cells in the cycle representative Eirene computed for this persistent homology class (since Eirene computes homology over the two element field, we always refer to vectors by their support).</li>
<li><code>class</code> and <code>size</code> can be displayed permanently, without the need to hover a cursor, via the keyword argument <code>showlabels = true</code>.</li>
</ul>
<h3><a id="user-content-qp-wasserstein-distances-between-diagrams" class="anchor" aria-hidden="true" href="#qp-wasserstein-distances-between-diagrams"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>q,p</code>-Wasserstein distances between diagrams</h3>
<p>Given two persistence diagrams<code>X</code> and <code>Y</code>, the <code>q,p</code>-Wasserstein Distance <code>W_{q,p}(X,Y)</code> between them is defined as</p>
<pre><code>W_{q,p}(X,Y) := (\inf_{bijections s: X \to Y} \Sigma_{x \in X} ||x - s(x)||_p^q)^(1/q).
</code></pre>
<p>To compute the q,p-Wasserstein distance between a pair of persistence diagrams, dgm1 and dgm2, run:</p>
<pre><code>julia&gt; wasserstein_distance(dgm1, dgm2, q=q, p=p)
</code></pre>
<ul>
<li>Keyword <code>p</code> defaults to 2.</li>
<li>Keyword <code>q</code> defaults to p.</li>
</ul>
<h3><a id="user-content-representatives" class="anchor" aria-hidden="true" href="#representatives"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Representatives</h3>
<p>To compute a cycle representative for the <code>p</code>th persistent homology class in Eirene's list for dimension <code>k</code>, run</p>
<pre><code>julia&gt; S = classrep(C, class=p, dim=k)
</code></pre>
<p>By default <code>class = 1</code> and <code>dim = 1</code>.  Recall that, since Eirene computes homology over the two element field, we can always specify a cycle representative by listing the cells that support it.  If <code>C</code> was generated from a point cloud or Vietoris-Rips complex, then the cycle representative <code>S</code> will be encoded as an array with <code>k+1</code> rows.  Each column of <code>S</code> represents the set of vertices that make up a <code>k</code>-dimensional simplex.  If <code>C</code> was generated from an <code>nxn</code> distance matrix <code>M</code>, then the entries of <code>S</code> correspond to the rows/columns of <code>M</code>.  If <code>C</code> is generated from a matrix <code>M</code> whose columns represent points in a point cloud, then the elements of <code>S</code> refer to columns of <code>M</code>.  If <code>C</code> was generated with <code>model = "complex"</code>, then <code>S</code> will be a vector of integers.  These numbers refer to cells in the complex; more precisely, the number <code>q</code> refers to the <code>q</code>th  dimension-<code>k</code> cell stored in memory.</p>
<p>Plotting for class representatives is currently supported only for point clouds and Vietoris-Rips complexes.  If the input data is a point cloud, coordinates for each vertex will be inferred automatically.  If the input data is not a point cloud, or if you would like to overwrite the existing coordinates, set  <code>C["inputdata"]["pointlabels"] = N</code> where <code>N</code> is a numeric array whose columns correspond to sample points.  Alternatively, you can pass <code>N</code> directly to the plotting function using keyword <code>coords = N</code> (this will not overwrite the existing coordinate data in <code>C</code>).</p>
<p>There are several ways to assign text labels to the points in your figure.  If labels were extracted from the input data or file, they are made available automatically for plotting.  To create or overwrite existing text labels, set <code>C["inputdata"]["pointlables"] = U</code>, where <code>U</code> is an array of type <code>Any</code> with all string entries.  <code>U</code> may also be passed directly to a plotting function using keyword <code>textlabels = U</code>.  If no other information is available, Eirene will label data points 1 through n, according to their order in the input array.  As in the case of persistence diagrams, both wrappers accept the <code>showlabels</code> keyword argument.  Hover-tags in PlotlyJS will still show text labels when <code>showlabels = false</code>.  A third option, <code>showlabels = "cycle"</code>, will cause <code>plotclassrep_pjs</code> to display the labels of only those vertices incident to the cycle representative being plotted.</p>
<h3><a id="user-content-visualizing-non-euclidean-data" class="anchor" aria-hidden="true" href="#visualizing-non-euclidean-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Visualizing non-Euclidean Data</h3>
<p>Multidimensional scaling (MDS) is a general approach to generating Euclidean embeddings of non-Euclidean or
high-dimensional data which (to the extent possible) faithfully represent distances between points. Most implementations of
the MDS algorithm accept a symmetric <code>m x m</code> matrix <code>S</code> and return an ordered set of <code>m</code> points in R<sup>n</sup>, with point <code>i</code> corresponding
to the <code>i</code>th column of <code>S</code>. These points determine an <code>m x m</code> matrix <code>D</code>, with <code>D[i,j]</code> equal to the Euclidean distance between points <code>i</code>
and <code>j</code>. The algorithm aims to to select the points that will make <code>D</code> as similar as possible to <code>S</code>. To use multidmensional scaling with
Eirene, pass <code>coords = "mds"</code> to the <code>plotclassrep_pjs</code> function. To specify an embedding of dimension <code>n</code>, use <code>embeddingdim = n</code>.</p>
<p>By default, the input matrix <code>S</code> will be the distance matrix used to generate the filtered complex. However in the special case
where only the vertices of a specific class representative are to be shown, that is, when the user specifies <code>showcloud = false</code>, there an additional option. The class to be represented consists of a finite family of simplices, or faces, and the
dimension-1 simplices incident to these faces form a graph. The hop distance on this graph yields a new distance matrix,
which the user may specify with <code>embeddingobj = "hop"</code>. This option often creates cleaner representations.</p>

<h4><a id="user-content-point-clouds-and-vietoris-rips-complexes" class="anchor" aria-hidden="true" href="#point-clouds-and-vietoris-rips-complexes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Point clouds and Vietoris Rips complexes</h4>
<p>PlotlyJS plots may be exported to the Plotly web API for sharing, storage, and advanced editing. Every function that generates
a PlotlyJS visual in the Eirene library returns a PlotlyJS object which may be uploaded via the Plotly. post function. See the
documentation for Plotly.jl to learn more.</p>
<p>Arrays can be read from (and written to) text flies using the native <code>readdlm</code> and <code>writedlm</code> (see the Julia
wikibooks chapter for text flies). The Julia base includes a number of additional functions for importing data from text flies,
and for convenience several of these have been combined in the Eirene wrapper <code>ezread(&lt;pathtofile&gt;)</code>.</p>
<p>Import from and export to Matlab flies are available from the packages MATLAB and MAT. Import from and export to Numpy
flies is managed by NPZ.</p>
<p>To save an entire dictionary <code>C</code>, the most convenient option by far is the Julia JLD package.</p>

<p>The marginal time and memory cost of computing explicit cycle representatives is often small relative to that of computing
homological barcodes alone (usually around 10%, for Euclidean point cloud data). Setting the <code>record = "none"</code>
may yield significant savings for other spaces, however.</p>
<h3><a id="user-content-comments" class="anchor" aria-hidden="true" href="#comments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Comments</h3>
<p>It is common among persistence solvers to round the entries of a user-provided distance matrix to a fixed number of distinct
values or significant digits. This is not the default behavior with Eirene, so care must be taken when comparing outputs.</p>
<p>Multidimensional scaling is a blue-chip method in data visualization, but no embedding algorithm is perfect. Particular care must be taken when <code>embeddingobj = "hop"</code> , as performance often decreases appreciably with the number of connected
components in the underlying graph. Remember that you can always inspect class representatives directly with the
<code>classrep</code> function.</p>
<h3><a id="user-content-advanced-usage" class="anchor" aria-hidden="true" href="#advanced-usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Advanced Usage</h3>
<h5><a id="user-content-keyword-arguments-1" class="anchor" aria-hidden="true" href="#keyword-arguments-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Keyword arguments</h5>
<p>The following are optional keyword arguments for the main funciton, <code>eirene</code>.</p>
<p><code>pointlabels = A</code></p>
<ul>
<li>If <code>A</code> is a nonempty array, then assign the pth vertex in the complex a label of <code>A[p]</code>, for all p.</li>
<li>If <code>A</code> is empty, assign the pth vertex a label of <code>p</code>.</li>
<li>Possible values: <code>A</code> may be any array of numbers or strings.  Default value: <code>[]</code>.</li>
</ul>
<p><code>fastop = true, false</code></p>
<ul>
<li>A value of <code>true</code> allows Eirene to stop computing homology at a given time point <code>t</code> if it can be determined that the homology of the complex does not change after time <code>t</code>.  There is no reason to change this setting under normal circumstances.</li>
<li>Possible values: <code>true</code>, <code>false</code>.  Default value: <code>true</code>.</li>
</ul>
<h5><a id="user-content-complex-formats" class="anchor" aria-hidden="true" href="#complex-formats"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Complex formats</h5>
<p>Let us say that for any vector <code>v</code>, the symbol <code>v{k}</code> denotes the set of all integers <code>j</code> such that <code>v[k] ≤ j &lt; v[k+1]</code> (you might recall that we used this convention when we explained how to format data for an arbitrary complex).  Now let us suppose we have a matrix <code>M</code> with 0 - 1 entries.  To save <code>M</code> in computer memory, it suffices to record which rows of each column have nonzero entries.  In particular, suppose we are given two vectors, <code>rv</code> (row values) and <code>cp</code> (column pattern).  If for all <code>p</code> the set <code>rv[cp{p}]</code> equals the set of rows where column <code>p</code> is nonzero, then we can (essentially) record <code>M</code> in computer memory by saving <code>rv</code> and <code>cp</code>.  We call this the <strong>column sparse format</strong> of <code>M</code>.</p>
<p>When we run the command</p>
<pre><code>C = eirene(rv=rv, cp=cp, fv=fv, dp=dp, &lt;keyword arguments&gt;)
</code></pre>
<p>we are passing <code>eirene</code> a pair of vectors (<code>rv</code> and <code>cp</code>) that record the <em>total boundary matrix</em>; this matrix has a row and column for every cell of every dimension.  The function of the vector <code>dp</code> is to record the dimension of each cell, and the function of <code>fv</code> is to specify the birth time of each cell.</p>
<p>When Eirene processes these data, they are broken down by dimension.  Eirene creates <em>arrays</em> <code>RV</code>, <code>CP</code>, and <code>FV</code> such that (1) <code>RV[p]</code> and <code>CP[p]</code> are column sparse formats for the boundary matrix with columns indexed by the cells of dimension <code>p</code> and rows indexed by cells of dimension <code>p-1</code>, and (2) <code>FV[p][q]</code> is the birth time of the <code>q</code>th <code>p-1</code>-dimensional cell.  If you already have access to <code>RV</code>, <code>CP</code>, and <code>FV</code> you can pass them to Eirene directly with the command</p>
<pre><code>C = eirene(rv=RV, cp=CP, fv=FV, dp=[], &lt;keyword arguments&gt;)
</code></pre>
<h3><a id="user-content-common-issues" class="anchor" aria-hidden="true" href="#common-issues"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Common Issues</h3>
<p>The most common source of technical difficulties in working with Eirene is software update. The best way to check that you are running the most recent version of Julia is to log onto the Julia homepage. To ensure that you have the most recent
version of the the various support packages, enter <code>Pkg. update()</code> in a Julia REPL.</p>
<p>The second most common source of technical difficulties rregards operating systems. The fewest issues by far have been
reported for Mac OS X. If you have already visited the Julia homepage to make sure your Julia install is up-to-date, it may be worthwhile switching to Mac to see if your issue persists. Windows users have reported encountering an error message that includes the phrase "invalid escape sequence" in reference to file paths. We found that replacing the unary operator <code>\</code> with <code>\\</code> generally resolves this issue, e.g. <code>C:\\Users</code> versus <code>C:\Users</code>.</p>
<p>Calling the include function twice with the same input will frequently produce a large number of error messages. This is a well documented issue in the current Julia distribution, and does not impact performance.</p>

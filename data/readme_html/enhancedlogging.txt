<p dir="auto"><a href="https://travis-ci.com/adamslc/EnhancedLogging.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/db7dc383858a37f898dec05bd1332ff27a7372943d9b82a3ba7952e95125692f/68747470733a2f2f7472617669732d63692e636f6d2f6164616d736c632f456e68616e6365644c6f6767696e672e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.com/adamslc/EnhancedLogging.jl.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">Extends and improves the Julia language's built in <code>ConsoleLogger</code>.
Additionally adds a <code>WorkerLogger</code> that passes log records to the main process
before processing them.</p>

<p dir="auto">To setup enhanced logging on all running processes, put the following <em>after</em>
all worker processes have been started:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="@everywhere using Logging, EnhancedLogging
global_logger(EnhancedConsoleLogger())
@everywhere global_logger(WorkerLogger(global_logger()))"><pre><span class="pl-c1">@everywhere</span> <span class="pl-k">using</span> Logging, EnhancedLogging
<span class="pl-c1">global_logger</span>(<span class="pl-c1">EnhancedConsoleLogger</span>())
<span class="pl-c1">@everywhere</span> <span class="pl-c1">global_logger</span>(<span class="pl-c1">WorkerLogger</span>(<span class="pl-c1">global_logger</span>()))</pre></div>
<p dir="auto">This will setup an <code>EnhancedLogger</code> on the master process, and <code>WorkerLogger</code>s
on the workers.</p>

<p dir="auto">Currently, the worker logging only works with global loggers, i.e. using
<code>with_logger</code> will prevent the log message from being forwarded to the master
process. This is mostly due to a limitation in how pointers are serialized for
interprocess communication. If you have ideas about how to circumvent this
problem, please open an issue so that we can discuss.</p>

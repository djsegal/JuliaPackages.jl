<p dir="auto"><a href="https://travis-ci.org/JuliaStrings/TinySegmenter.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/260045641f45a822f5538c5a08d7275223f55b662fdb4d68c522892f763ff8d8/68747470733a2f2f7472617669732d63692e6f72672f4a756c6961537472696e67732f54696e795365676d656e7465722e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/JuliaStrings/TinySegmenter.jl.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">TinySegmenter.jl is a Julia version of <a href="http://chasen.org/~taku/software/TinySegmenter/" rel="nofollow">TinySegmenter</a>, which is an extremely compact Japanese tokenizer
originally written in JavaScript by Mr. Taku Kudo.</p>

<p dir="auto">The return value of <code>tokenize</code> is an array of substrings of the string input,
giving the locations of the tokens in the text.  (Substrings are represented
by the <code>SubString</code> Julia type.)</p>

<p dir="auto">The following are times in seconds for a benchmark (see <a href="benchmark/README.md">benchmark/README.md</a>) of TinySegmenter
implementations in different languages tokenizing a large (243kB) Japanese text:</p>
<p dir="auto">|Ruby | C++ | Perl | JavaScript(Node.js) | Go | Python | Julia |
|---|---|---|---|---|---|---|---|
|132.98 | 48 | 134 |105.31 | 10.50 | 111.85 | 11.70 |</p>
<p dir="auto">The benchmark was performed on the following machine:</p>
<ul dir="auto">
<li>Intel Core i5-3210M CPU at 2.50GHz</li>
<li>8GB RAM (1600MHz DDR3)</li>
<li>MacBook Pro (Retina, 13-inch, Late 2012), MacOS 10.11 ("El Capitan")</li>
</ul>

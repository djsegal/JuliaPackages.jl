<h1><a id="user-content-gadjetjl" class="anchor" aria-hidden="true" href="#gadjetjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>GadJet.jl</h1>
<p>This package provides some basic functionality to work with the SPH code "Gadget" by Volker Springel (doi:10.1111/j.1365-2966.2005.09655.x).</p>
<p>These functionalities are: reading and writing data in snapshot format 1+2, reading the subfind output, basic mapping of sph data to a grid.
Additionally I provide some exact riemann solvers for shocktube tests, unit conversion and other utility.
This list will extend over time.</p>
<p>Documentation can be found <a href="https://gadjetjl.readthedocs.io/en/latest/index.html" rel="nofollow">here</a>.</p>
<p>Please note that I am not affiliated with Volker Springel. This project was started because I needed to work with Gadget for university and couldn't find any Julia implementations to work with the data.
I'm now working with Klaus Dolag and others on a development version of Gadget3, so you will find some functionality in this project that won't be useful for you if you don't have access to that version. Sorry!</p>
<p>Any help and contribution is greatly appreciated, as this is still a work in progress. Please see the section on contributing.</p>

<h2><a id="user-content-reading-data" class="anchor" aria-hidden="true" href="#reading-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Reading Data</h2>
<p>If you want to read a simulation snapshot into memory with GadJet.jl, it's as easy as this:</p>
<div class="highlight highlight-source-julia"><pre>    data <span class="pl-k">=</span> <span class="pl-c1">read_snap</span>(filename)</pre></div>
<p>This will return a dictionary with the header information in <code>data["Header"]</code> and the blocks sorted by particle type.</p>
<p>As an example, this is how you would access the positions of the gas particles:</p>
<div class="highlight highlight-source-julia"><pre>    data[<span class="pl-s"><span class="pl-pds">"</span>Parttype0<span class="pl-pds">"</span></span>][<span class="pl-s"><span class="pl-pds">"</span>POS<span class="pl-pds">"</span></span>]</pre></div>
<p>If you only want to read a specific block for a single particle type (e.g. positions of gas particles) you can use the function with a specified blockname and particle type like so:</p>
<div class="highlight highlight-source-julia"><pre>    pos <span class="pl-k">=</span> <span class="pl-c1">read_snap</span>(filename, <span class="pl-s"><span class="pl-pds">"</span>POS<span class="pl-pds">"</span></span>, <span class="pl-c1">0</span>)</pre></div>
<p>This will return an array of the datatype of your simulation, usually Float32.</p>
<h2><a id="user-content-quick-visualisation" class="anchor" aria-hidden="true" href="#quick-visualisation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Quick Visualisation</h2>
<p>For a quick glimpse at your data you can use the glimpse function (yes, I thought hard about this one...)</p>
<div class="highlight highlight-source-julia"><pre>    image <span class="pl-k">=</span> <span class="pl-c1">glimpse</span>(filename)</pre></div>
<p>This will return a 500x500 pixel image of the whole box, centered on the center of mass.</p>
<p>If you want to look at a specific range you can provide an array with the center coordinates as <code>center_pos = [x, y, z]</code> and the extent in x, y and z direction with <code>dx, dy, dz</code>.</p>
<div class="highlight highlight-source-julia"><pre>    image <span class="pl-k">=</span> <span class="pl-c1">glimpse</span>(filename, center_pos, dx, dy, dz)</pre></div>

<p dir="auto">Kaleido has for long been a part of PlotlyBase but has recently been removed, due to
non-availability on some platforms and due to extra loadtime (more information, e.g.
<a href="https://github.com/sglyon/PlotlyBase.jl/issues/52" data-hovercard-type="issue" data-hovercard-url="/sglyon/PlotlyBase.jl/issues/52/hovercard">here</a>). In order not to interfere with
the current version of PlotlyJS this package exports <code>save</code> from FileIO with inverted order
of filename and plot compared to savefig. If you want to use old scripts with existing
savefig commands, you can import it via:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="import PlotlySave.savefig"><pre><span class="pl-k">import</span> PlotlySave<span class="pl-k">.</span>savefig</pre></div>

<p dir="auto">There is a branch
<a href="https://github.com/hhaensel/PlotlySave#hh-abstractplot"><code>hh-abstractplot</code></a> that plays
together with respective branches in my forks of
<a href="https://github.com/hhaensel/PlotlyBase#hh-abstractplot"><code>PlotlyBase</code></a> and
<a href="https://github.com/hhaensel/PlotlyJS#hh-abstractplot"><code>PlotlyJS</code></a> to introduce an
<code>AbstractPlot</code> type that both <code>Plot</code> and <code>SyncPlot</code> are subtypes of. That way many
multiple definitions can be avoided and other projects could derive their own subtypes.</p>
<p dir="auto">For this to happen <code>PlotlySave</code> would probably need to move to
<a href="https://github.com/JuliaPlots/PlotlyJS"><code>JuliaPlots</code></a>.</p>
<p dir="auto">Feel free to use the <a href="https://github.com/hhaensel/PlotlySave/issues">issue pages</a> to
comment or improve this approach.</p>

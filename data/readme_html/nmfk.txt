<div dir="auto">
    <a target="_blank" rel="noopener noreferrer" href="logo/nmfk-logo.jpg"><img src="logo/nmfk-logo.jpg" alt="nmfk" width="50%" style="max-width: 100%;"></a>
</div>
<p dir="auto"><strong>NMFk</strong> is one of the tools in the <strong>SmartTensors</strong> ML framework (<a href="https://smarttensors.com" rel="nofollow">smarttensors.com</a>).</p>
<div dir="auto">
    <a target="_blank" rel="noopener noreferrer" href="logo/SmartTensorsNewSmall.png"><img src="logo/SmartTensorsNewSmall.png" alt="SmartTensors" width="25%" style="max-width: 100%;"></a>
</div>
<p dir="auto"><strong>NMFk</strong> is a novel unsupervised machine learning methodology that allows for automatic identification of the optimal number of features (signals/signatures) present in the data.</p>
<p dir="auto">Classical <strong>NMF</strong> approaches do not allow for automatic estimation of the number of features.</p>
<p dir="auto"><strong>NMFk</strong> estimates the number of features <code>k</code> through <em>k</em>-means clustering coupled with regularization constraints (sparsity, physical, mathematical, etc.).</p>
<p dir="auto"><strong>NMFk</strong> can be applied to perform:</p>
<ul dir="auto">
<li>Feature extraction (<strong>FE</strong>)</li>
<li>Blind source separation (<strong>BSS</strong>)</li>
<li>Detection of disruptions / anomalies</li>
<li>Image recognition</li>
<li>Text mining</li>
<li>Data classification</li>
<li>Separation (deconstruction) of co-occurring (physics) processes</li>
<li>Discovery of unknown dependencies and phenomena</li>
<li>Development of reduced-order/surrogate models</li>
<li>Identification of dependencies between model inputs and outputs</li>
<li>Guiding the development of physics models representing the ML analyzed data</li>
<li>Blind predictions</li>
<li>Optimization of data acquisition (optimal experimental design)</li>
<li>Labeling of datasets for supervised ML analyses</li>
</ul>
<p dir="auto"><strong>NMFk</strong> provides high-performance computing capabilities to solve problems with Shared and Distributed Arrays in parallel.
The parallelization allows for utilization of multi-core / multi-processor environments.
GPU and TPU accelerations are available through existing Julia packages.</p>
<p dir="auto"><strong>NMFk</strong> provides advanced tools for data visualization, pre- and post-processing.
These tools substantially facilitate utilization of the package in various real-world applications.</p>
<p dir="auto"><strong>NMFk</strong> methodology and applications are discussed in the research papers and presentations listed below.</p>
<p dir="auto"><strong>NMFk</strong> is demonstrated with a series of examples and test problems provided here.</p>

<p dir="auto">A simple problem demonstrating <strong>NMFk</strong> can be executed as follows.
First, generate 3 random signals in a matrix <code>W</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="a = rand(15)
b = rand(15)
c = rand(15)
W = [a b c]"><pre>a <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">15</span>)
b <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">15</span>)
c <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">15</span>)
W <span class="pl-k">=</span> [a b c]</pre></div>
<p dir="auto">Then, mix the signals to produce a data matrix <code>X</code> of 5 sensors observing the mixed signals as follows:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="X = [a+c*3 a*10+b b b*5+c a+b*2+c*5]"><pre>X <span class="pl-k">=</span> [a<span class="pl-k">+</span>c<span class="pl-k">*</span><span class="pl-c1">3</span> a<span class="pl-k">*</span><span class="pl-c1">10</span><span class="pl-k">+</span>b b b<span class="pl-k">*</span><span class="pl-c1">5</span><span class="pl-k">+</span>c a<span class="pl-k">+</span>b<span class="pl-k">*</span><span class="pl-c1">2</span><span class="pl-k">+</span>c<span class="pl-k">*</span><span class="pl-c1">5</span>]</pre></div>
<p dir="auto">This is equivalent to generating a mixing matrix <code>H</code> and obtain <code>X</code> by multiplying <code>W</code> and <code>H</code></p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="H = [1 10 0 0 1; 0 1 1 5 2; 3 0 0 1 5]
X = W * H"><pre>H <span class="pl-k">=</span> [<span class="pl-c1">1</span> <span class="pl-c1">10</span> <span class="pl-c1">0</span> <span class="pl-c1">0</span> <span class="pl-c1">1</span>; <span class="pl-c1">0</span> <span class="pl-c1">1</span> <span class="pl-c1">1</span> <span class="pl-c1">5</span> <span class="pl-c1">2</span>; <span class="pl-c1">3</span> <span class="pl-c1">0</span> <span class="pl-c1">0</span> <span class="pl-c1">1</span> <span class="pl-c1">5</span>]
X <span class="pl-k">=</span> W <span class="pl-k">*</span> H</pre></div>
<p dir="auto">After that, execute <strong>NMFk</strong> to estimate the number of unknown mixed signals based only on the information in <code>X</code>.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="import NMFk

<p dir="auto">The code also returns estimates of matrices <code>W</code> and <code>H</code>.</p>
<p dir="auto">It can be easily verified that estimated <code>We[kopt]</code> and <code>He[kopt]</code> are scaled versions of the original <code>W</code> and <code>H</code> matrices.</p>
<p dir="auto">Note that the order of columns ('signals') in <code>W</code> and  <code>We[kopt]</code> are not expected to match.
The order of rows ('sensors') in <code>H</code> and  <code>He[kopt]</code> are also not expected to match.
The estimated orders will be different every time the code is executed.</p>

<p dir="auto">Hypatia is a highly-customizable open source interior point solver for generic conic optimization problems, written in <a href="https://julialang.org/" rel="nofollow">Julia</a>.
It is licensed under the MIT License (see <a href="https://github.com/chriscoey/Hypatia.jl/blob/master/LICENSE.md">LICENSE</a>).</p>
<p dir="auto">For more information on Hypatia, please see:</p>
<ul dir="auto">
<li><a href="https://chriscoey.github.io/Hypatia.jl/dev" rel="nofollow">documentation</a> for Hypatia's conic form, predefined cones, and interfaces</li>
<li><a href="https://github.com/chriscoey/Hypatia.jl/wiki/files/coneref.pdf">cones reference</a> for cone definitions and oracles</li>
<li><a href="https://github.com/chriscoey/Hypatia.jl/tree/master/examples">examples folder</a> for applied examples and instances</li>
<li><a href="https://github.com/chriscoey/Hypatia.jl/tree/master/benchmarks">benchmarks folder</a> for scripts used to run and analyze various computational benchmarks</li>
</ul>
<p dir="auto">and preprints of our papers:</p>
<ul dir="auto">
<li><a href="https://arxiv.org/abs/2005.01136" rel="nofollow">Solving natural conic formulations with Hypatia.jl</a> for computational arguments for expanding the class of cones recognized by conic solvers</li>
<li><a href="https://arxiv.org/abs/2107.04262" rel="nofollow">Performance enhancements for a generic conic interior point algorithm</a> for a description of Hypatia's algorithm and our enhanced stepping procedures</li>
<li><a href="https://arxiv.org/abs/2103.11499" rel="nofollow">Sum of squares generalizations for conic sets</a> for barriers and computational techniques for our generalized polynomial sum of squares cones</li>
<li><a href="https://arxiv.org/abs/2103.04104" rel="nofollow">Conic optimization with spectral functions on Euclidean Jordan algebras</a> for barriers and computational techniques for many of our epigraph/hypograph cones</li>
</ul>
<p dir="auto">and corresponding <a href="https://github.com/chriscoey/Hypatia.jl/wiki">raw results CSV files</a> generated by our run scripts in the benchmarks folder.</p>

<p dir="auto">To use Hypatia, install <a href="https://julialang.org/downloads/" rel="nofollow">Julia</a>, then at the Julia REPL, type:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Hypatia
using Pkg
Pkg.add(&quot;Hypatia&quot;)"><pre><span class="pl-k">using</span> Hypatia
<span class="pl-k">using</span> Pkg
Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>Hypatia<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">Hypatia is an experimental solver and a work in progress, and may not run with older releases of Julia.
Default options/parameters are not well-tuned, so we encourage you to experiment with these.</p>

<p dir="auto">Hypatia can be accessed through a low-level native Julia interface or through open-source modeling tools such as <a href="https://github.com/jump-dev/JuMP.jl">JuMP</a> and <a href="https://github.com/jump-dev/Convex.jl">Convex.jl</a>.
The native interface is more expressive, allowing Hypatia to solve conic models expressed with generic real floating point types and structured matrices or linear operators, for example.
However, it is typically sufficient and more convenient to use JuMP.</p>
<p dir="auto">Using JuMP, we can model a simple D-optimal experiment design problem and call Hypatia:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using LinearAlgebra

# setup JuMP model
opt = Hypatia.Optimizer(verbose = false)
model = Model(() -&gt; opt)
@variable(model, x[1:3] &gt;= 0)
@constraint(model, sum(x) == 5)
@variable(model, hypo)
@objective(model, Max, hypo)
V = rand(2, 3)
Q = V * diagm(x) * V'
aff = vcat(hypo, [Q[i, j] for i in 1:2 for j in 1:i]...)
@constraint(model, aff in MOI.RootDetConeTriangle(2))

# solve and query solution
optimize!(model)
termination_status(model)
objective_value(model)
value.(x)"><pre><span class="pl-k">using</span> LinearAlgebra

<p dir="auto">Comments, questions, suggestions, and improvements/extensions to the code or documentation are welcomed.
Please reach out on <a href="https://discourse.julialang.org/c/domain/opt" rel="nofollow">Discourse</a>, or submit an issue or contribute a PR on our <a href="https://github.com/chriscoey/Hypatia.jl">Github repo</a>.
If contributing code, try to maintain consistent style and add docstrings or comments for clarity.
New examples are welcomed and should be implemented similarly to the <a href="https://github.com/chriscoey/Hypatia.jl/tree/master/examples">existing examples</a>.</p>

<p dir="auto">You can start lattice QCD in 5 steps!</p>
<p dir="auto">1.Download a Julia binary from <a href="https://julialang.org/downloads/" rel="nofollow">Julialang.org</a>. Set a path to the binary of Julia.
Julia 1.6 (or higher) is supported.</p>
<p dir="auto">2.In Julia REPL, push "]" key to enter the package mode and type</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="add LatticeQCD"><pre class="notranslate"><code>add LatticeQCD
</code></pre></div>
<p dir="auto">and "return" key. Press "backspace" key ( "delete" key for mac) to exit the package mode.
You can get the latest version via <code>add LatticeQCD#master</code>.</p>
<p dir="auto">(All dependence will be solved automatically)</p>
<p dir="auto">3.Include the package with</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using LatticeQCD"><pre class="notranslate"><code>using LatticeQCD
</code></pre></div>
<p dir="auto">4.Make a parameter file with wizard,</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="run_wizard()"><pre class="notranslate"><code>run_wizard()
</code></pre></div>
<p dir="auto">Choose parameters as you want!</p>
<p dir="auto">5.Start simulation with created your parameter file!</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content=" run_LQCD(&quot;my_parameters.toml&quot;)"><pre class="notranslate"><code> run_LQCD("my_parameters.toml")
</code></pre></div>
<p dir="auto">You'll get results!</p>
<p dir="auto">Of cource, you can write/modify a parameter file by yourself.</p>
<p dir="auto">Enjoy life with lattice QCD.</p>
<p dir="auto">To see our demonstration above, execute,</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Plots
using LatticeQCD 
demo()"><pre><span class="pl-k">using</span> Plots
<span class="pl-k">using</span> LatticeQCD 
<span class="pl-c1">demo</span>()</pre></div>
<p dir="auto">it takes time before showing up the window.</p>
<h1 dir="auto"><a id="user-content-what-is-supported" class="anchor" aria-hidden="true" href="#what-is-supported"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>What is supported?</h1>
<p dir="auto">We support lattice gauge theory in 4 dimensional euclidean spacetime.</p>
<ul dir="auto">
<li>Wizard for parameter files</li>
<li>Gauge fields
<ul dir="auto">
<li>Optimized SU(2), SU(3)</li>
<li>General SU(N)</li>
<li>General gauge action = plaquette + rect + etc action</li>
</ul>
</li>
<li>Fermions
<ul dir="auto">
<li>Wilson (2 flavor)</li>
<li>Staggered fermion (1-8 tastes ~ flavor)</li>
<li>Standard Domain-wall (2 flavor, experimental)</li>
</ul>
</li>
<li>Configuration generation algorithms
<ul dir="auto">
<li>Cold/Hot start for SU(N). One instanton configuration for SU(2)</li>
<li>Heatbath for SU(N) &amp; overelaxation for a general gauge action</li>
<li>Even-odd heatbath for the plaquette action</li>
<li>Quenched HMC with SU(N) for a general gauge action</li>
<li>HMC (2 flavor Wilson) with SU(N) with a general gauge action</li>
<li>HMC (4 taste staggered fermions) with SU(N) with a general gauge action</li>
<li>RHMC (any flavor staggered) with SU(N) for a general gauge action*</li>
<li>SU(N) stout smeared dynamical fermions (experimental)</li>
<li>Self-learning HMC with the plaquette action</li>
</ul>
</li>
<li>Measurements
<ul dir="auto">
<li>Plaquette</li>
<li>Polyakov loop</li>
<li>Chiral condensates (Wilson, staggered)</li>
<li>Momentum projected pion correlator (Wilson fermion, staggered)</li>
<li>RxT Wilson loop</li>
<li>Energy density</li>
<li>Topological charge (plaquette, clover and O(a^2) improved definition)</li>
<li>Load &amp; measurement mode (load and measure all configurations in a directory)</li>
</ul>
</li>
<li>Smearing
<ul dir="auto">
<li>Stout</li>
<li>Gradient flow for a generic action (experimental)</li>
</ul>
</li>
<li>I/O for gauge configurations
<ul dir="auto">
<li>ILDG format (Binary)</li>
<li>JLD format (Default binary file for Julia, one of HDF5)</li>
<li>Text file for Bridge++ (Bridgetext)</li>
</ul>
</li>
</ul>
<p dir="auto">*If you specified other than Nf=4, 8 with the staggered fermion HMC, RHMC is automatically used. For a machine with the apple silicon, Nf=1-8 is avalable.</p>
<p dir="auto">To use following functions, please use v 0.1.2</p>
<ul dir="auto">
<li>Fermion integrated HMC with a general gauge action</li>
<li>Self-learning Monte-Carlo with a general action (ref. <a href="https://arxiv.org/abs/2010.11900" rel="nofollow">this paper</a> )</li>
</ul>
<p dir="auto">*Version below 1.0, it uses <code>***.jl</code> as a default parameter file, and now it uses <code>***.toml</code>. Version 1.0 supports both parameter file formats.</p>
<p dir="auto">*Parallelazation is supported by <a href="https://github.com/akio-tomiya/LatticeDiracOperators.jl">LatticeDiracOperators.jl</a>. See below.</p>

<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="LQCDjl_block.png"><img src="LQCDjl_block.png" width="300" style="max-width: 100%;"></a></p>
<p dir="auto">LatticeQCD.jl is basically a wrapper of the following packages</p>
<ul dir="auto">
<li><a href="https://github.com/akio-tomiya/Wilsonloop.jl">Wilsonloop.jl</a> :Wilsonloop.jl helps us to treat with the Wilson loops and generic Wilson lines in any Nc and dimensions. Wilson lines can be defined in symbolly.</li>
<li><a href="https://github.com/akio-tomiya/Gaugefields.jl">Gaugefields.jl</a>: Gaugefields.jl is a package for lattice QCD codes. Treating gauge fields (links), gauge actions with MPI and autograd. This can generate quenched configurations.</li>
<li><a href="https://github.com/akio-tomiya/LatticeDiracOperators.jl">LatticeDiracOperators.jl</a>: LatticeDiracOperators.jl is a package for lattice QCD codes. Treating pseudo-femrion fields with various lattice Dirac operators, fermion actions with MPI. This can generate configurations with dynamical fermions.</li>
</ul>

<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using LatticeQCD

</code></pre></div>
<p dir="auto">You write the following code (&amp; save as <code>run.jl</code>):</p>

<p dir="auto">We develop this code to achive following things:</p>
<ol dir="auto">
<li>Good portability (If one has Julia, this code is runnable. All dependences are under control.)</li>
<li>Easy to start/ pedagogical (start in 10 minutes)</li>
<li>Suite (configuration generation with and without fermions, and measurements)</li>
<li>Easy to modify (Good for prototyping)</li>
<li>Compatitive speed with Fortran 90 codes</li>
</ol>
<p dir="auto">This is the first open source Julia code for lattice QCD. High performance is out of our scope.</p>

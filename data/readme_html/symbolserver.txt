<p>SymbolServer is a helper package for LanguageServer.jl that provides information about internal and exported variables of packages (without loading them). A package's symbol information is initially loaded in an external process but then stored on disc for (quick loading) future use.</p>

<div class="highlight highlight-source-julia"><pre><span class="pl-c1">SymbolServerInstance</span>(path_to_depot, path_to_store)</pre></div>
<p>Creates a new symbol server instance that works on a given Julia depot. This symbol server instance can be long lived, i.e. one can re-use it for different environments etc. If <code>path_to_store</code> is specified, cache files will be stored there, otherwise a standard location will be used.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">getstore</span>(ssi<span class="pl-k">::</span><span class="pl-c1">SymbolServerInstance</span>, environment_path<span class="pl-k">::</span><span class="pl-c1">AbstractString</span>)</pre></div>
<p>Loads the symbols for the environment in <code>environment_path</code>. Returns a tuple, where the first element is a return status and the second element a payload. The status can be <code>:success</code> (in which case the second element is the new store), <code>:canceled</code> if another call to <code>getstore</code> was initiated before a previous one finished (with <code>nothing</code> as the payload), or <code>:failure</code> with the payload being the content of the error stream of the client process.</p>
<p>This function is long running and should typically be called in an <code>@async</code> block.</p>

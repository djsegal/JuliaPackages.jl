<p>One of the things I find annoying about MATLAB is dealing with binary data. There's a lot of boilerplate, a lot of <code>fread()</code>, and some weird thing involving a <code>=&gt;</code>.</p>
<p>Enter StrPack. StrPack decodes binary streams to Julia composite types, handling stream endianness and padding bytes for the source ABI along the way. StrPack also encodes instances of Julia composite types to binary streams, setting endianness and adding padding if required to meet an ABI. Along with Julia's <code>IOBuffer</code> type, StrPack can also be used to convert between Julia composite types and buffers suitable for some C function arguments.</p>
<p>StrPack is a serializer/deserializer only in a very limited sense. StrPack only handles Julia's <a href="http://docs.julialang.org/en/latest/manual/types/#id1" rel="nofollow">bits types</a> or user types for which <code>read(io, ::UserType)</code> and <code>write(io, data::UserType)</code> have been defined. However, you could use StrPack to build those things.</p>

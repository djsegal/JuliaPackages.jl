<p><strong>Turing.jl</strong> is a Julia library for general-purpose <a href="https://en.wikipedia.org/wiki/Probabilistic_programming_language" rel="nofollow">probabilistic programming</a>. Turing allows the user to write models using standard Julia syntax, and provides a wide range of sampling-based inference methods for solving problems across probabilistic machine learning, Bayesian statistics, and data science. Compared to other probabilistic programming languages, Turing has a special focus on modularity, and decouples the modelling language (i.e. the compiler) and inference methods. This modular design, together with the use of a high-level numerical language Julia, makes Turing particularly extensible: new model families and inference methods can be easily added.</p>
<p>Current features include:</p>
<ul>
<li><a href="https://turing.ml/dev/tutorials/6-infinitemixturemodel/" rel="nofollow">General-purpose</a> probabilistic programming with an <a href="https://turing.ml/dev/tutorials/0-introduction/" rel="nofollow">intuitive modelling interface</a></li>
<li>Robust, efficient <a href="https://github.com/TuringLang/AdvancedHMC.jl">Hamiltonian Monte Carlo (HMC)</a> sampling for differentiable posterior distributions</li>
<li><a href="https://github.com/TuringLang/AdvancedPS.jl">Particle MCMC</a> sampling for complex posterior distributions involving discrete variables and stochastic control flows</li>
<li>Compositional inference via Gibbs sampling that combines particle MCMC, HMC, <a href="https://github.com/TuringLang/AdvancedMH.jl">Random-Walk MH (RWMH)</a> and <a href="https://github.com/TuringLang/Turing.jl/blob/master/src/inference/ess.jl">Elliptical Slice Sampling</a></li>
<li>Advanced variational inference based on <a href="https://github.com/TuringLang/AdvancedVI.jl">ADVI</a> and <a href="https://github.com/TuringLang/Bijectors.jl">Normalising Flows</a></li>
</ul>

<p>Turing was originally created and is now managed by Hong Ge. Current and past Turing team members include <a href="http://mlg.eng.cam.ac.uk/hong/" rel="nofollow">Hong Ge</a>, <a href="http://mlg.eng.cam.ac.uk/?portfolio=kai-xu" rel="nofollow">Kai Xu</a>, <a href="http://martint.blog" rel="nofollow">Martin Trapp</a>, <a href="https://github.com/mohamed82008">Mohamed Tarek</a>, <a href="https://business.uoregon.edu/faculty/cameron-pfiffer" rel="nofollow">Cameron Pfiffer</a>, <a href="http://retiredparkingguard.com/about.html" rel="nofollow">Tor Fjelde</a>.
You can see the full list of on Github: <a href="https://github.com/TuringLang/Turing.jl/graphs/contributors">https://github.com/TuringLang/Turing.jl/graphs/contributors</a>.</p>
<p>Turing is an open source project so if you feel you have some relevant skills and are interested in contributing then please do get in touch. See the <a href="http://turing.ml/docs/contributing/" rel="nofollow">Contributing</a> page for details on the process. You can contribute by opening issues on Github or implementing things yourself and making a pull request. We would also appreciate example models written using Turing.</p>

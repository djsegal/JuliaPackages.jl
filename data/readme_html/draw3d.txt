<h2><a id="user-content-note---not-stable-not-usable" class="anchor" aria-hidden="true" href="#note---not-stable-not-usable"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Note - NOT STABLE, NOT USABLE</h2>
<p>This is pre-alpha and nothing about it is stable, or sometimes even
functioning.</p>
<p>Draw3D is a package for 3D graphics with a nice Julian API.</p>
<p>It is built on top of the OpenGL.jl and GLFW.jl packages and takes advantage of
the fantastic work they have done wrapping the OpenGL API. Those packages
provide an API that is a direct mapping of the OpenGL C API however, which
isn't particularly user-friendly if you are not already an OpenGL developer.</p>
<p>The main datatypes are Mesh, Light, Material, Transform</p>
<h2><a id="user-content-examples" class="anchor" aria-hidden="true" href="#examples"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Examples</h2>
<p>The simplest possible thing is to render a unit (1x1x1) cube in the default
position, which is with its local origin at the global origin.</p>
<pre><code>c = Cube()
display(c)
</code></pre>
<p>To make a cube that is twice as large and centered at the origin:</p>
<pre><code>c = Cube()
c = scale(c, 2)
c = translate(c, -1, -1, -1)
display(c)
</code></pre>
<p>As you can see this can get a little verbose. All the transform functions are
also defined for partial application, so you can also use Julia's built-in
composition operator like so:</p>
<pre><code>Cube() |&gt; scale(2) |&gt; translate(-1, -1, -1) |&gt; display
</code></pre>

<p>Transparent objects are assumed to be convex, and are rendered by first
rendering the faces pointing away from the camera and then those pointing
towards. Handling this properly for non-convex meshes requires z-sorting
the faces, which we don't do right now.</p>

<h1 dir="auto"><a id="user-content-replferencejl" class="anchor" aria-hidden="true" href="#replferencejl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>REPLference.jl</h1>
<p dir="auto"><code>REPLference.jl</code> is a package that addresses two challenges:</p>
<p dir="auto">First, it provides materials for beginners to learn basic programming in Julia within
Julia's versatile REPL. Instead of searching online or asking basic Julia questions
elsewhere, beginners can use the rich-feature REPL to learn.</p>
<p dir="auto">Second, it aims to solve the problem of accessing methods in Julia using the "dot" syntax,
which can be frustrating for new users. In many <a href="https://en.wikipedia.org/wiki/Object-oriented_programming" rel="nofollow">OOP languages</a>,
users can easily see and learn new methods to call on an object by using the "dot" syntax in
their editor, as shown in this example below:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="https://github.com/udohjeremiah/REPLference.jl/blob/master/assets/python_example.png"><img src="https://github.com/udohjeremiah/REPLference.jl/raw/master/assets/python_example.png" alt="Python example" style="max-width: 100%;"></a></p>
<p dir="auto">However, in Julia, this isn't possible because <code>a.x</code> refers to the property <code>x</code> bundled with
<code>a</code>, not the method <code>x</code> bundled with <code>a</code> (methods are not bundled with objects in Julia). So
when users type <code>a.&lt;tab&gt;</code> in the REPL, it will return the properties defined in <code>a</code>, as
shown in this example below:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="https://github.com/udohjeremiah/REPLference.jl/blob/master/assets/julia_example.png"><img src="https://github.com/udohjeremiah/REPLference.jl/raw/master/assets/julia_example.png" alt="Julia example" style="max-width: 100%;"></a></p>
<p dir="auto">This limitation often leads to "reinventing the wheel" for those who can write the methods
they're looking for and a "break in workflow" for those who cannot, as they have to go
online to ask, "Is there a method X to do Y?" There is an
<a href="https://discourse.julialang.org/t/allowing-the-object-method-args-syntax-as-an-alias-for-method-object-args/62051" rel="nofollow">ongoing discussion in Julia's discourse forum about this</a>.</p>
<blockquote>
<p dir="auto">NOTE: Julia provides functions like <code>methods</code>, <code>methodswith</code>, and <code>which</code> to help users
with this challenge, but new users may still struggle because these functions don't return
an exhaustive list of methods.</p>
</blockquote>
<p dir="auto">Finally, for convenience, this package uses <a href="https://github.com/JuliaCollections/AbstractTrees.jl">AbstractTrees.jl</a>
as a dependency to print the sub- and super-types of a type in a tree-like format.</p>

file (if the file does not exist, create it):</p>

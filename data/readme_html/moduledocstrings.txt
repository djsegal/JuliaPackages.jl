<p dir="auto">A package to create simple "module docstrings" for Julia packages. These are targeted at summarizing the main components of your package, essentially as a prompt or reminder to users.  For example:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using ModuleDocstrings

help?&gt; ModuleDocstrings
search: ModuleDocstrings

    •  ModuleDocstrings.generate: Create an API summary docstring for a
       module.

    •  ModuleDocstrings.write: add an API summary docstring to a package."><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> ModuleDocstrings

help?<span class="pl-k">&gt;</span> ModuleDocstrings
search<span class="pl-k">:</span> ModuleDocstrings

    •  ModuleDocstrings<span class="pl-k">.</span>generate<span class="pl-k">:</span> Create an API summary docstring <span class="pl-k">for</span> a
       <span class="pl-k">module</span>.

    •  ModuleDocstrings<span class="pl-k">.</span>write<span class="pl-k">:</span> add an API summary docstring to a package.</pre></div>
<p dir="auto">This reminds users that the two main functions are <code>ModuleDocstrings.generate</code> and <code>ModuleDocstrings.write</code>.</p>
<p dir="auto">These summaries are brief; to learn more about a particular function, read its help in full:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="help?&gt; ModuleDocstrings.generate
  ModuleDocstrings.generate(mod::Module)

  Return an API summary string for mod.

  The summary is assembled from all docstrings in the package, picking the first sentence of each docstring. When added to the

    •  exclude docstrings that shouldn't appear in the API summary

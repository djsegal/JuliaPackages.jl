<p dir="auto">Julia package for reading plane-wave DFT code <a href="http://www.castep.org" rel="nofollow">CASTEP</a> input/output files.</p>
<p dir="auto">Implemented readers:</p>
<ul dir="auto">
<li><code>ome_bin</code></li>
<li><code>cst_ome</code></li>
<li><code>bands</code></li>
<li><code>cell</code></li>
<li><code>param</code></li>
<li><code>pdos_bin</code></li>
</ul>
<p dir="auto">Working but still being improved:</p>
<ul dir="auto">
<li><code>castep_bin</code>/<code>check</code></li>
</ul>

<ul dir="auto">
<li>
<p dir="auto">Units are not handled for binary data from check/castep_bin file - these are the internal atomic units</p>
<ul dir="auto">
<li>Energies are likely to be in Hartree</li>
<li>Length units are likey to be in Bohr</li>
<li>Lattice vectors are stored as matrices of row vectors rather than the column vectors</li>
</ul>
</li>
<li>
<p dir="auto">The unit system may need to be reworked - returning data with units may not always be desirable (for sake type dispatch)</p>
</li>
<li>
<p dir="auto">More quantities from the <code>castep_bin/check</code> file can be read if the scehma is known.</p>
</li>
<li>
<p dir="auto">Units of the wavefunction/charge density needs to be determined</p>
</li>
</ul>

<p>Main features:</p>
<h3><a id="user-content-locale" class="anchor" aria-hidden="true" href="#locale"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Locale</h3>
<ul>
<li>create Locale from string formed according to standards Unicode Locale Identifier (BCP47 (tags for Identifying languages), RFC5646, RFC4647)</li>
<li>set/get default locale for different purposes</li>
<li>startup-locale derived form environment settings (LANG, LC_MESSAGES, ..., LC_ALL)</li>
<li>Locale patterns imply a canonical partial ordering by set inclusion</li>
</ul>
<h3><a id="user-content-resourcebundle" class="anchor" aria-hidden="true" href="#resourcebundle"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>ResourceBundle</h3>
<ul>
<li>Database of objects (e.g. text strings), identified by locale and individual text key</li>
<li>select most specific available object according to canonical ordering of locales</li>
<li>detect ambiguities for individual keys</li>
<li>storage in package module directory</li>
<li>database uses files witten in Julia source code</li>
<li>database files containing translated texts using gettext-PO format is supported</li>
</ul>
<h4><a id="user-content-message-text-localization-lc_messages" class="anchor" aria-hidden="true" href="#message-text-localization-lc_messages"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Message text localization (LC_MESSAGES)</h4>
<ul>
<li>a string macro providing translation of standard language according to default locale</li>
<li>support string interpolation and control ordering of interpolated substrings</li>
<li>includes mechanism for multiple plural forms for translations</li>
<li>fall back to text provided as key within the source text</li>
<li>Define global default Resource bundle for each module</li>
<li>Support features of Posix <code>gettext</code></li>
</ul>

<pre><code>using ResourceBundles

# The following code is to make the test resources of ResourceBundles itself available
# to the Main module. Typically the resources are accessible within their module.
rdir = abspath(pathof(ResourceBundles), "..", "..", "resources")
;ln -s $rdir .

# show current locale (inherited from env LC_ALL, LC_MESSAGES, LANG)
locale()

# change locale for messages programmatically:
set_locale!(LocaleId("de"), LC.MESSAGES)

# use string translation feature
println(tr"original text")
for n = (2,3)
    println(tr"$n dogs have $(4n) legs")
end
for lines in [1,2,3]
    println(tr"$lines lines of code")
end

# access the posix locale definitions
ResourceBundles.CLocales.nl_langinfo(ResourceBundles.CLocales.CURRENCY_SYMBOL)

</code></pre>

<h4><a id="user-content-locale-identifiers-and-locales" class="anchor" aria-hidden="true" href="#locale-identifiers-and-locales"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Locale Identifiers and Locales</h4>
<p>Locale Identifiers are converted from Strings, which are formatted according to Unicode Locale Identifier.
Examples: "en", "en_Latn", "en_US", "en_Latn_GB_london", "en_US_x_private".</p>
<p>Additionally the syntax for Posix environment variables <code>LANG</code> and <code>LC_...</code> are
supported.
Examples: "C", "en_US", "en_us.utf8", "en_US@posext".
All those formats are converted to a canonical form and stored in objects of type <code>LocaleID</code>: <code>LocaleID("en_US")</code>.
The <code>_</code> may be replaced by <code>-</code> in input.</p>
<p><code>LocaleId</code> implements the <code>equals</code> and the <code>issubset</code> (<code>⊆</code>) relations.
Here <code>LocaleId("en_US") ⊆ LocaleId("en") ⊆ LocaleId("C") == LocaleId("")</code>.</p>
<p>The <code>Locale</code> is a set of locale-properties for several categories. The categories are taken
from the GNU implementation. Each locale-property is identified by a <code>LocaleId</code>.</p>
<p>Each category corresponds to one of the <code>LC_</code> environment variables and there is a related
constant in module 'LC'. Here is the complete list:</p>
<pre><code>category          | remark
-------------------------------
CTYPE             | character classifications (letter, digit, ...)
NUMERIC           | number formating (decimal separator, ...)
TIME              | time and date formats
COLLATE           | text comparison
MONETARY          | currency symbol
MESSAGES          | translation of message texts
ALL               | not a category - used to override all other settings
PAPER             | paper formats
NAME              | person names
ADDRESS           | address formating
TELEPHONE         | formating of phone numbers
MEASUREMENT       | measurement system (1 for metric, 2 for imperial)
IDENTIFICATION    | name of the locale identifier
</code></pre>
<p>For all of those terms there is an environment variable <code>"LC_category"</code> and a Julia constant <code>LC.category</code>. The categories up to <code>ALL</code> are defined in POSIX(), which the others are GNU extensions.
The special variable <code>"LC_ALL"</code> overrides all other variables <code>"LC_category"</code> if set.
The additional environment variable <code>"LANG"</code> is used as a fallback, if neither <code>"LC_category"</code> nor <code>"LC_ALL"</code> is set.</p>
<p>Each task owns a task specific current locale. It is obtained by <code>locale()</code>.
For each category the valid locale identifier is accessed by <code>locale_id(LC.category)</code>.
For example the locale <code>locale_id(LC.MESSAGES)</code> is used as the default locale for message
text look-up in the current task.
The locale-ids of the current locale may be changed by <code>set_locale!(localeid, category)</code>.
For example <code>set_locale!(LocaleID("de_DE"), LC.MESSAGES)</code> modifies the current locale to
use German message translations.</p>
<p>All locale categories except for <code>LC.MESSAGES</code> are implemented by the GNU installation, which contains the shared library glibc, a set of predefined locale properties, and a tool <code>locales</code>, which delivers a list of all locale identifiers installed on the system.
In Julia, the values of all locale dependent variable of those categories may be obtained
like <code>ResourceBundles.CLocales.nl_langinfo.(ResourceBundles.CLocales.DAY.(1:7))</code>
or <code>ResourceBundles.CLocales.nl_langinfo(ResourceBundles.CLocales.IDENTIFICATION_TITLE)</code>.</p>
<p>These are only available on GNU based systems (including Linux and OSX).</p>
<h4><a id="user-content-resource-bundles" class="anchor" aria-hidden="true" href="#resource-bundles"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Resource Bundles</h4>
<p>A resource bundle is an association of string values with arbitrary objects, where the
actual mapping depends on a locale.
Conceptually it behaves like a <code>Dict{Tuple{String,Locale},Any}</code>.
Each resource bundle is bound to a package. The corresponding data are stored in the
subdirectory <code>resources</code> of the package directory.</p>
<p><code>bundle = @resource_bundle("pictures")</code> creates a resource bundle, which is bound
to the current module. The resource is populated by the content found in the resource files, which are associated with the current module.
The resources are looked for the resources subdirectory of a package module or, if the
base module is <code>Main</code>, in the current working directory.</p>
<p>The object stored in reousource bundles are not restricted to strings, but may have any Julia type.
For example, it could make sense to store locale-dependent pictures (containing natural language texts) in resource bundles. Also locale dependent information or algorithms are possible.</p>
<p>The actual data of all resource bundles of a package stored in the package directory in an extra subdirectory named <code>resources</code> (<code>resource-dir&gt;</code>. An individual resource bundle with name <code>&lt;name&gt;</code> consists of a collection of files with path names</p>
<p><code>&lt;resource-dir&gt;/&lt;name&gt;&lt;intermediate&gt;.[jl|po|mo]</code>.</p>
<p>Here <code>&lt;intermediate&gt;</code>may be empty or a canonicalized locale tag, whith separator characters replaced by <code>'_'</code> or <code>'/'</code>. That means, all files have the standard <code>Julia</code>extension (and they actually contain <code>Julia</code>code) or a <code>po</code>
extension indicating message resources in PO format. The files may be spread in a subdirectory hierarchy according to the structure of the languange tags.</p>
<p>Fallback strategy following structure of language tags.</p>
<h4><a id="user-content-string-translations" class="anchor" aria-hidden="true" href="#string-translations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>String Translations</h4>
<p>String translations make use of a current locale <code>(locale_id(LC.MESSAGES))</code> and a standard resource bundle <code>(@__MODULE__).RB_messages</code>, which is created on demand.</p>
<p>The macro <code>tr_str</code> allows the user to write texts in a standard locale and it is translated
at runtime to a corresponding text in the current locale.
It has the following features.</p>
<ul>
<li>translate plain text</li>
<li>support string interpolation, allowing re-ordering of variables</li>
<li>support multiple plural forms for exactly one interpolation variable</li>
<li>support context specifiers</li>
</ul>
<p>The interpolation expressions are embedded in the text of the string-macro <code>tr</code> in the usual
form, e.g. <code>tr" ...  $var ... $(expr) ..."</code>.
If the programmer wants the text to be translated into one of several grammatical plural forms,
he has to formulate the text in the program in the plural form of the standard language and
embed at least one interpolation expression. One of those expressions has to be flagged by
the unary negation operator ( <code>tr" ... $(!expr) ... "</code>). The flagged expression must provide an
integer value. The negation operation is not executed, but indicates to the implementation,
which of several plural options has to be selected.
For this purpose the translation text database defines a "plural-formula", which maps the
integer expression <code>n</code> to a zero-based index. This index peeks the proper translation from a
vector of strings.
Some typical formulas in the PO-file format:</p>
<pre><code>msgid ""
msgstr ""
"Plural-Forms: nplurals=1; plural=0;\n"                 # Chinese
"Plural-Forms: nplurals=1; plural=n &gt; 1 ? 1 : 0;\n"     # English (default)
"Plural-Forms: nplurals=1; plural=n == 1 ? 0 : 1;\n"    # French
"Plural-Forms: nplurals=1; plural=n%10==1&amp;&amp;n%100!=11 ? 0 : n%10&gt;=2&amp;&amp;n%10&lt;=4&amp;&amp;(n%100&lt;10||n%100&gt;=20) ? 1 : 2;\n"
  # Russian
</code></pre>
<p>For details see: <a href="https://www.gnu.org/software/gettext/manual/gettext.html#PO-Files" rel="nofollow">PO-Files</a>.</p>
<p>Message contexts are included in the tr string like <code>tr"§mood§blue"</code> or <code>tr"§color§blue"</code>. In the PO file it is defined for example like:</p>
<pre><code>msgctx "mood"
msgid "blue"
msgstr "melancholisch"

msgctx "color"
msgid "blue"
msgstr "blau"
</code></pre>
<p>The <code>tr_str</code> macro includes the functionality of the gnu library calls <code>gettext</code>and <code>ngettext</code>.
The database supports the file formats and infrastructure defined by <a href="https://www.gnu.org/software/gettext/manual" rel="nofollow">gettext</a>. Also binary files with extension <code>.mo</code> are be processed, which can be compiled from <code>.po</code> files by the gettext-utility <a href="https://www.gnu.org/software/gettext/manual/gettext.html#msgfmt-Invocation" rel="nofollow"><code>msgfmt</code></a>.</p>

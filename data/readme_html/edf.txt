<p dir="auto">Read and write <a href="https://www.edfplus.info/" rel="nofollow">European Data Format (EDF/EDF+)</a> and <a href="https://www.biosemi.com/faq/file_format.htm" rel="nofollow">BioSemi Data Format (BDF)</a> files in Julia.</p>
<p dir="auto">Compared to all features implied by the EDF/EDF+ specifications, this package is currently missing:</p>
<ul dir="auto">
<li>Out-of-core data record streaming; this package (and its type representations, i.e. <code>EDF.Signal</code>) generally assumes the user is loading all of a file's sample data into memory at once.</li>
<li>Specialization for discontinuous EDF+ files ("EDF+D" files).</li>
<li>Validation/specialization w.r.t. <a href="https://www.edfplus.info/specs/edftexts.html" rel="nofollow">"canonical/standard EDF texts"</a></li>
<li>Validation-on-write of manually constructed <code>EDF.File</code>s</li>
<li>Support for <a href="https://www.edfplus.info/specs/edffloat.html" rel="nofollow">the EDF+ <code>longinteger</code>/<code>float</code> extension</a></li>
</ul>
<p dir="auto">Where practical, this package chooses field names that are as close to EDF/EDF+ specification terminology as possible.</p>

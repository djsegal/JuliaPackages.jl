<p>This package provides support for one-dimensional numerical integration in Julia using adaptive
Gauss-Kronrod quadrature.
The code was originally part of Base Julia.  It supports integration of arbitrary numeric types,
including arbitrary precision (<code>BigFloat</code>), and even integration of arbitrary normed vector spaces
(e.g. matrix-valued integrands).</p>
<p>The package provides three functions: <code>quadgk</code>, <code>gauss</code>, and <code>kronrod</code>.
<code>quadgk</code> performs the integration, <code>gauss</code> computes Gaussian quadrature points and weights for integrating
over the interval [a, b], and <code>kronrod</code> computes Kronrod points, weights, and embedded Gaussian quadrature
weights for integrating over [-1, 1].   Typical usage looks like:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using QuadGK
integral, err = quadgk(x -&gt; exp(-x^2), 0, 1, rtol=1e-8)
"><pre><span class="pl-k">using</span> QuadGK
integral, err <span class="pl-k">=</span> <span class="pl-c1">quadgk</span>(x <span class="pl-k">-&gt;</span> <span class="pl-c1">exp</span>(<span class="pl-k">-</span>x<span class="pl-k">^</span><span class="pl-c1">2</span>), <span class="pl-c1">0</span>, <span class="pl-c1">1</span>, rtol<span class="pl-k">=</span><span class="pl-c1">1e-8</span>)</pre></div>
<p>which computes the integral of exp(–x²) from x=0 to x=1 to a relative tolerance of 10⁻⁸, and returns the approximate <code>integral = 0.746824132812427</code> and error estimate <code>err = 7.887024366937112e-13</code> (which is actually smaller than the requested tolerance: convergence was very rapid because the integrand is smooth).</p>
<p>For more information, see the <a href="https://JuliaMath.github.io/QuadGK.jl/stable" rel="nofollow">documentation</a>.</p>
<h2><a id="user-content-in-place-operations-for-array-valued-integrands" class="anchor" aria-hidden="true" href="#in-place-operations-for-array-valued-integrands"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>In-place operations for array-valued integrands</h2>
<p>For integrands whose values are <em>small</em> arrays whose length is known at compile-time,
it is usually most efficient to modify your integrand to return
an <code>SVector</code> from the <a href="https://github.com/JuliaArrays/StaticArrays.jl">StaticArrays.jl package</a>.</p>
<p>However, for integrands that return large or variabley-length arrays, we also provide a function
<code>quadgk!(f!, result, a,b...)</code> in order to exploit in-place operations where possible.   The
<code>result</code> argument is used to store the estimated integral <code>I</code> in-place, and the integrand function
is now of the form <code>f!(r, x)</code> and should write <code>f(x)</code> in-place into the result array <code>r</code>.</p>
<h2><a id="user-content-gaussian-quadrature-and-arbitrary-weight-functions" class="anchor" aria-hidden="true" href="#gaussian-quadrature-and-arbitrary-weight-functions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Gaussian quadrature and arbitrary weight functions</h2>
<p>If you are computing many similar integrals of smooth functions, you may not need an adaptive
integration — with a little experimentation, you may be able to decide on an appropriate number
<code>N</code> of integration points in advance, and re-use this for all of your integrals.    In this case
you can use <code>x, w = gauss(N, a, b)</code> to find the quadrature points <code>x</code> and weights <code>w</code>, so that
<code>sum(f.(x) .* w)</code> is an <code>N</code>-point approximation to <code>∫f(x)dx</code> from <code>a</code> to <code>b</code>.</p>
<p>For computing many integrands of similar functions with <em>singularities</em>,
<code>x, w = gauss(W, N, a, b)</code> function allows you to pass a <em>weight function</em> <code>W(x)</code> as the first argument,
so that <code>sum(f.(x) .* w)</code> is an <code>N</code>-point approximation to <code>∫W(x)f(x)dx</code> from <code>a</code> to <code>b</code>.   In this way,
you can put all of the singularities etcetera into <code>W</code> and precompute an accurate quadrature rule as
long as the remaining <code>f(x)</code> terms are smooth.   For example,</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using QuadGK
x, w = gauss(x -&gt; exp(-x) / sqrt(x), 10, 0, -log(1e-10), rtol=1e-9)
"><pre><span class="pl-k">using</span> QuadGK
x, w <span class="pl-k">=</span> <span class="pl-c1">gauss</span>(x <span class="pl-k">-&gt;</span> <span class="pl-c1">exp</span>(<span class="pl-k">-</span>x) <span class="pl-k">/</span> <span class="pl-c1">sqrt</span>(x), <span class="pl-c1">10</span>, <span class="pl-c1">0</span>, <span class="pl-k">-</span><span class="pl-c1">log</span>(<span class="pl-c1">1e-10</span>), rtol<span class="pl-k">=</span><span class="pl-c1">1e-9</span>)</pre></div>
<p>computes the points and weights for performing <code>∫exp(-x)f(x)/√x dx</code> integrals from <code>0</code> to <code>-log(1e-10) ≈ 23</code>, so that there is a <code>1/√x</code> singularity in the integrand at <code>x=0</code> and a rapid decay for increasing <code>x</code>.  (The <code>gauss</code> function currently does not support infinite integration intervals, but for a rapidly decaying weight function you can approximate an infinite interval to any desired accuracy by a sufficiently broad interval, with a tradeoff in computational expense.)  For example, with <code>f(x) = sin(x)</code>, the exact answer is <code>0.570370556005742…</code>.  Using the points and weights above with <code>sum(sin.(x) .* w)</code>, we obtain <code>0.5703706212868831</code>, which is correct to 6–7 digits using only 10 <code>f(x)</code> evaluations.  Obtaining similar
accuracy for the same integral from <code>quadgk</code> requires nearly 300 function evaluations.   However, the
<code>gauss</code> function itself computes many (<code>2N</code>) numerical integrals of your weight function (multiplied
by polynomials), so this is only more efficient if your <code>f(x)</code> is very expensive or if you need
to compute a large number of integrals with the same <code>W</code>.</p>
<p>See the <a href="https://juliamath.github.io/QuadGK.jl/stable/#QuadGK.gauss" rel="nofollow"><code>gauss</code> documentation</a> for more information.  See also our example using a <a href="https://nbviewer.jupyter.org/urls/math.mit.edu/~stevenj/Solar-Quadrature.ipynb" rel="nofollow">weight function interpolated from tabulated data</a>.</p>

<p>The <a href="https://github.com/ajt60gaibb/FastGaussQuadrature.jl">FastGaussQuadrature.jl</a> package provides
non-adaptive Gaussian quadrature variety of built-in weight functions — it is a good choice you need to go to very high orders N, e.g. to integrate rapidly oscillating functions, or use weight functions that incorporate some standard singularity in your integrand.  QuadGK, on the other hand, keeps the order N of the quadrature rule fixed and improves accuracy by subdividing the integration domain, which can be better if fine resolution is required only in a part of your domain (e.g if your integrand has a sharp peak or singularity somewhere that is not known in advance).</p>
<p>For multidimensional integration, see the <a href="https://github.com/stevengj/HCubature.jl">HCubature.jl</a>, <a href="https://github.com/stevengj/Cubature.jl">Cubature.jl</a>, and
<a href="https://github.com/giordano/Cuba.jl">Cuba.jl</a> packages.</p>

<p dir="auto"><a href="https://github.com/JuliaMath/Xsum.jl/actions?query=workflow%3ACI"><img src="https://github.com/JuliaMath/Xsum.jl/workflows/CI/badge.svg" alt="CI" style="max-width: 100%;"></a></p>
<p dir="auto">The Xsum package is a Julia wrapper around Radford Neal's <a href="https://gitlab.com/radfordneal/xsum" rel="nofollow">xsum package</a>
for exactly rounded double-precision floating-point summation.  The <a href="https://arxiv.org/abs/1505.05571" rel="nofollow">xsum algorithm</a> takes <code>n</code> double precision (<code>Float64</code> or smaller) floating-point values as input and computes the "exactly rounded sum" â€” equivalent to summing the values in <em>infinite</em> precision and rounding the result to the nearest <code>Float64</code> value.</p>
<p dir="auto">By clever use of additional precision, xsum can compute the exactly rounded sum only a few times more slowly than the naive summation algorithm (or the pairwise summation used in the built-in <code>sum</code> function), much faster than using generic arbitrary precision (like <code>BigFloat</code> operations).</p>

<p dir="auto">The Xsum package provides a function <code>xsum</code> to perform the summation.  To use it, simply do:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Xsum
xsum(iterator)"><pre><span class="pl-k">using</span> Xsum
<span class="pl-c1">xsum</span>(iterator)</pre></div>
<p dir="auto">where you can pass any iterable collection (arrays, generators, tuples, etcetera).  Real or complex collections can be summed, but note that each element is converted to double precision (<code>Float64</code> or <code>ComplexF64</code>) before it is summed, and the result is always double precision.</p>
<p dir="auto">The variant <code>xsum(function, iterator)</code> is also supported, similar to <code>sum(function, iterator)</code>, which sums the result of the
given <code>function</code> applied to each element of the <code>iterator</code>.</p>
<p dir="auto">There is also a lower-level object <code>XAccumulator()</code> that you can use to perform more
flexible sums.  A <code>s::XAccumulator</code> object represents partial sum, whose exactly
rounded <code>Float64</code> result is given by <code>float(s)</code>.   <code>s = XAccumulator()</code> initializes
a zero sum, and <code>accumulate!(s, x)</code> adds <code>x</code> to <code>s</code> where <code>x</code> is a real number
(converted to <code>Float64</code>), an array of <code>Float64</code> values, or another <code>XAccumulator</code>.
You can also add and subtract accumulators with <code>+</code> and <code>-</code> (which operate out-of-place
so they are less efficient), or negate one in-place with <code>Xsum.negate!(s)</code>.</p>
<p dir="auto">For example, if you wanted to compute an exactly rounded sum of a large vector <code>x</code> in parallel, you could call <code>accumulate!(XAccumulator(), xslice)</code> on a sequence of <em>slices</em>
(portions) of <code>x</code> in parallel, and then combine the sub-accumulators to obtain the
final sum.</p>

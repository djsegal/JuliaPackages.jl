<p dir="auto">Construct Gauss points and weights for an <code>n</code>-dimensional simplex
domain with vertices specified by the <code>n*(n-1)</code> matrix <code>vert</code>, where
each row contains the coordinates <code>(x1,...,xn)</code> for a vertex. The
order of the quadrature scheme in each dimension must be the same in
this implementation.</p>

<p dir="auto"><code>X</code> will be a matrix for which the <code>j</code>th column are the grid points in
each coordinate <code>xj</code>.</p>
<p dir="auto">To integrate a function <code>f</code>, use e.g.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="sum(W[i] * f(X[i,:]) for i in 1:length(W))"><pre><span class="pl-c1">sum</span>(W[i] <span class="pl-k">*</span> <span class="pl-c1">f</span>(X[i,:]) <span class="pl-k">for</span> i <span class="pl-k">in</span> <span class="pl-c1">1</span><span class="pl-k">:</span><span class="pl-c1">length</span>(W))</pre></div>
<p dir="auto">I tested the package for up to <code>D=5</code> dimensions and order <code>N=10</code>, and
found the integration error for polynomials of order <code>Pâ‰¤N</code> (which
should have only floating-point round-off error) to be less than
<code>10eps</code>. This is tested by the test suite.</p>

<p>Gloria is a simple 2D game engine built on top of <a href="https://github.com/jonathanBieler/SimpleDirectMediaLayer.jl">SDL2</a> and written in pure Julia.</p>

<h1><a id="user-content-engine-design" class="anchor" aria-hidden="true" href="#engine-design"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Engine design</h1>
<p>Gloria internally runs multiple asynchronous tasks/threads to separate event handling (mouse and keyboard), update calls and rendering.  These tasks call <code>onevent!</code>, <code>update!</code> and <code>render!</code>, respectively, on objects put into the game world.  Using multiple dispatch, behaviour can be added to objects, just as in an object-oriented design.  The engine also comes with a (slightly experimental) rigid body collision system, allowing for game physics without external dependencies.</p>

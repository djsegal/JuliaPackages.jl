<h2 dir="auto"><a id="user-content-why-didnt-you-instead-create-bulletjl" class="anchor" aria-hidden="true" href="#why-didnt-you-instead-create-bulletjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why didn't you instead create Bullet.jl</h2>
<p dir="auto">The PyBullet package is essentially Python bindings for the <a href="https://github.com/bulletphysics/bullet3">Bullet C++ library</a>.</p>
<p dir="auto">It seems like the more astute thing to do (for performance concerns) is to create a Julia wrapper for the core Bullet C++ library instead of creating a Julia wrapper for a Python wrapper of a C++ library. However, the great <a href="https://github.com/erwincoumans">Erwin Coumans</a> (Creator of Bullet and PyBullet), has created some really cool features in PyBullet, especially for Reinforcement learning applications, that might not be easily accessible when using the raw Bullet C++ library. PyBullet also as arguably the best and most convenient API of any robotics simulator.</p>
<p dir="auto">And since it is waaay easier to create this Julia wrapper for the PyBullet Python module than it is to create a Julia wrapper for the Bullet C++ library,</p>
<p dir="auto">and since it is currently 2:47am and I'm sleep-deprived,</p>
<p dir="auto">and since really coool people like <a href="https://github.com/cjdoris">Christopher Rowley</a> have created really coool Julia packages like <a href="https://github.com/cjdoris/PythonCall.jl">PythonCall.jl</a> for effortlessly creating Julia wrappers for Python modules,</p>
<p dir="auto">I simply decided to create a Julia Wrapper for the super-cool PyBullet module.</p>

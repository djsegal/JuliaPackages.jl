<p dir="auto"><code>ClimateERA.jl</code> is a Julia package that aims to streamline the following processes:</p>
<ul dir="auto">
<li>downloading of ECMWF reanalysis data (ERA-Interim and ERA5)</li>
<li>basic analysis (mean, maximum, minimum, standard deviation, etc.) of downloaded data</li>
<li>extraction of data for a given <strong>GeoRegion</strong> (see <code>GeoRegions.jl</code> for more information)</li>
</ul>
<p dir="auto"><code>ClimateERA.jl</code> can be installed via</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="] add ClimateERA"><pre class="notranslate"><code>] add ClimateERA
</code></pre></div>

<p dir="auto"><code>ClimateERA.jl</code> utlizes <code>GeoRegions.jl</code> to specify domains from which data is to be extracted.  If the option is not specified, then <code>ClimateERA.jl</code> will assume that the user wishes to process <strong>global</strong> data (which may not be wise especially for GPM due to the large file sizes involved and memory required).</p>
<p dir="auto">For more information, please see the repository for <code>GeoRegions.jl</code> <a href="https://github.com/natgeo-wong/GeoRegions.jl">here</a>.</p>

<p dir="auto"><code>ClimateERA.jl</code> does not directly download reanalysis data from ECMWF/CDS.  Instead, it generates a <code>Python</code> script according to the user's choice of parameters that the user will run with Python to download the data required.</p>

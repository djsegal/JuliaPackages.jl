<p dir="auto">This is a Julia package for optimizing controlled stochastic dynamic system,
in discrete time. It offers three methods of resolution :</p>
<ul dir="auto">
<li><em>Stochastic Dual Dynamic Programming</em> (SDDP) algorithm.</li>
<li><em>Extensive formulation</em>.</li>
<li><em>Stochastic Dynamic Programming</em>.</li>
</ul>
<p dir="auto">It is built on top of <a href="https://github.com/JuliaOpt/JuMP.jl">JuMP</a>.</p>
<p dir="auto">StochDynamicProgramming asks the user to provide explicit the cost <code>c(t, x, u, w)</code> and
dynamics <code>f(t, x, u, w)</code> functions. Also, the package was developed back
in 2016, and some parts of its API are not idiomatic, in a Julia sense.
For other implementations of the SDDP algorithm in Julia, we advise to
have a look at these two packages:</p>
<ul dir="auto">
<li><a href="https://github.com/odow/SDDP.jl">SDDP.jl</a></li>
<li><a href="https://github.com/JuliaStochOpt/StructDualDynProg.jl">StructDualDynProg.jl</a></li>
</ul>

<p dir="auto">StochDynamicProgramming targets problems with</p>
<ul dir="auto">
<li>Stage-wise independent discrete noise</li>
<li>Linear dynamics</li>
<li>Linear or convex piecewise linear costs</li>
</ul>
<p dir="auto">Extension to non-linear formulation are under development.</p>
<h3 dir="auto"><a id="user-content-why-sddp" class="anchor" aria-hidden="true" href="#why-sddp"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why SDDP?</h3>
<p dir="auto">SDDP is a dynamic programming algorithm relying on cutting planes. The algorithm requires convexity
of the value function but does not discretize the state space. The complexity is linear in the
number of stage, and can accomodate higher dimension state spaces than standard dynamic programming.
The algorithm returns exact lower bound and estimated upper bound as well as approximate optimal
control strategies.</p>
<h3 dir="auto"><a id="user-content-why-extensive-formulation-" class="anchor" aria-hidden="true" href="#why-extensive-formulation-"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why Extensive formulation ?</h3>
<p dir="auto">An extensive formulation approach consists in representing the stochastic problem as a deterministic
one and then calling a standard deterministic solver. It is mainly usable in a linear
setting. Computational complexity is exponential in the number of stages.</p>
<h3 dir="auto"><a id="user-content-why-stochastic-dynamic-programming-" class="anchor" aria-hidden="true" href="#why-stochastic-dynamic-programming-"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why Stochastic Dynamic Programming ?</h3>
<p dir="auto">Dynamic Programming is a standard tool to solve stochastic optimal control problem with
independent noise. The method requires discretizing the state space, and its
complexity is exponential in the dimension of the state space.</p>

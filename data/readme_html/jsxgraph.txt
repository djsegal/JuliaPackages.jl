<p><a href="https://github.com/jsxgraph/jsxgraph">jsxgraph</a> is a nice project to have interactive plots in the browser, it's a bit like Plotly except more geared towards maths and interactivity.</p>
<p>This package aims to help generate JSXGraph code from Julia code that can then be plugged in a statically served website (for instance one you would generate via <a href="https://github.com/tlienart/Franklin.jl">Franklin.jl</a>).</p>

<p>Check out <a href="https://tlienart.github.io/JSXGraphWeb/" rel="nofollow">this page</a> for a gallery of examples and explanations.
This is still very much WIP so your help and suggestions are very welcome if you think this project might be useful to you in some form.</p>

<p>The wrapper is incomplete to say the least, there are two easy ways to go forward (both of which should be done), it just requires the time to do it and test it:</p>
<ol>
<li>wrap more of the jsxgraph objects like <a href="https://jsxgraph.org/docs/symbols/Chart.html" rel="nofollow"><code>chart</code></a>, <a href="https://jsxgraph.org/docs/symbols/Glider.html" rel="nofollow"><code>glider</code></a>, <a href="https://jsxgraph.org/docs/symbols/Group.html" rel="nofollow"><code>group</code></a>, <a href="https://jsxgraph.org/docs/symbols/Grid.html" rel="nofollow"><code>grid</code></a>, <a href="https://jsxgraph.org/docs/symbols/Integral.html" rel="nofollow"><code>integral</code></a>, etc...</li>
<li>add aliases for the common options like <code>strokecolor</code>, <code>strokewidth</code> etc. so that people who are used to <code>Plots.jl</code> options can just use those, for instance we should allow <code>col</code>, <code>color</code> or <code>linecolor</code> as alias for <code>strokecolor</code>.</li>
</ol>
<p>Then of course, generating many examples is always good, they have <a href="https://jsxgraph.org/wp/about/index.html" rel="nofollow">many examples</a> which could be ported over in some form or other.</p>

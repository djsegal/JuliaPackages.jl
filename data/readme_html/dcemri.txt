<h2><a id="user-content-why-julia" class="anchor" aria-hidden="true" href="#why-julia"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Why Julia?</h2>
<p>From the <a href="http://julialang.org/" rel="nofollow">Julia website</a>,</p>
<blockquote>
<p>Julia is a high-level, high-performance dynamic programming language for technical computing, with syntax that is familiar to users of other technical computing environments. It provides a sophisticated compiler, distributed parallel execution, numerical accuracy, and an extensive mathematical function library. The library, largely written in Julia itself, also integrates mature, best-of-breed C and Fortran libraries for linear algebra, random number generation, signal processing, and string processing.</p>
</blockquote>
<p>Put simply, it looks like Matlab, which is simple to learn and familiar to most MRI researchers, but it works better and faster and is completely free.  In particular, for the problem of DCE MRI, Julia's simple and flexible parallel computing model allows almost perfect parallelization of the nonlinear least squares fitting problem.  In my informal testing, the intrinsic speed of Julia coupled to my parallel implementation produced a factor of 20-40 speedup over comparable Matlab and Python.</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p>Installation is simple.  First you need Julia.
The simplest way to get Julia is to grab the current release version from <a href="http://julialang.org/downloads/" rel="nofollow">julialang.org</a>.</p>
<p>Next you need <strong>DCEMRI.jl</strong>.  Open Julia. You should see a terminal window with the <code>julia&gt;</code> prompt.  This is analogous to the command line in Matlab.
To install <strong>DCEMRI.jl</strong>, first type <code>]</code> at the prompt to get into the package shell then type:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="(@v1.5) pkg&gt; add DCEMRI 
"><pre><code>(@v1.5) pkg&gt; add DCEMRI 
</code></pre></div>
<p>This might take a minute, because the validation data must be downloaded, as well as a few supporting packages.</p>
<p>If you want to <strong>DCEMRI.jl</strong> to create plots for you, you also need Python with <a href="http://matplotlib.org/" rel="nofollow">Matplotlib</a> installed.
Most OS X and Linux machines probably already have a version of Python with Matplotlib installed.
If you don't have Python with Matplotlib, such as if you are on Windows, you can grab the excellent, free <a href="https://store.continuum.io/cshop/anaconda/" rel="nofollow">Anaconda python distribution</a>, which comes with Matplotlib pre-installed.</p>
<p>If you have installation problems, check the next section on platform specific issues.  If none of that helps, please file an issue at the <a href="http://github.com/davidssmith/DCEMRI.jl">github repository</a>, and we will try to help you.</p>

<p>If you get an error that <code>xcrun</code> is missing on Mac OS X, follow the instructions to install the Developer Tools Command Line Tools, and then switch back to your Julia window and run <code>Pkg.build("HDF5")</code>.  This should fix everything.  If you don't get this error, but installation fails, make sure you have Xcode installed, along with the optional Command Line Tools.</p>
<p>There have also been reports of Julia popping up behind other windows on OS X.  Look behind your windows if you think Julia isn't starting.</p>

<p>On Windows, you will almost certainly need to install <a href="https://store.continuum.io/cshop/anaconda/" rel="nofollow">Anaconda</a> to have a Python distribution. Some Windows versions claim to be 64-bit, but aren't quite.  If you are having trouble on Windows, and you have installed a 64-bit Anaconda and Julia, you can try uninstalling them and installing 32-bit versions of both.</p>

<p>After installing the Julia and the <strong>DCEMRI</strong> module, you should run the validations, to make sure the calculations work correctly on your machine.  The easiest way to do this is to start Julia and then run</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="julia&gt; using DCEMRI

julia&gt; validate()
"><pre><code>julia&gt; using DCEMRI

julia&gt; validate()
</code></pre></div>
<p>This will run both validations (4 and 6), which could take up to an hour, depending on the number of cores you started Julia with and how fast your hardware is. Examine the results to make sure that the parameters have been recovered accurately.  The text output of the scripts will also print quantitative measures of parameter accuracy. By default validation results will be written to subdirectories of the <strong>DCEMRI.jl</strong> module directory, which is queried with <code>Pkg.dir("DCEMRI")</code>.  You can customize this directory for the validation by passing the output path as the second argument.  This only works the individual validation functions, e.g. <code>validate(6, "/my/path")</code> or <code>validate(4, "/my/path")</code>.  Since both validations write figures with identical names, they can't go to the same output directory.</p>

<h2><a id="user-content-a-note-about-units" class="anchor" aria-hidden="true" href="#a-note-about-units"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>A Note about Units</h2>
<p>All units in the code are SI where possible.  Sometimes, due to numerical accuracy issues, they have been converted internally. But all data should be supplied to the code in SI units.  In particular, time should be in seconds, and relaxation rates in inverse seconds.  The two exceptions to this rule are that flip angles should be in degrees and Ktrans is output in min^-1.</p>

<p>In the simplest incarnation, if you already have a MAT file containing your data,  you can run the analysis from within Julia using</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="julia&gt; using DCEMRI

<p><strong>DCEMRI.jl</strong> will look for parameters in the input MAT file, and if they are found will use them.  Anything not found in the MAT file will be initialized from the defaults.  These defaults can be viewed with the <code>defaults()</code> command.  You may also override both the MAT file and the defaults by passing keyword arguments to <code>fitdata</code>.</p>

<p><strong>DCEMRI.jl</strong> has two basic modes of operation.  The first is command-line invocation, like an operating system command.  To run it as a shell command, first edit the first line of <code>bin/dcefit</code> to point to where you installed your Julia binary, as in</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="#!/path/to/julia/binary


"><pre><code>#!/path/to/julia/binary


</code></pre></div>
<p>Next, make sure <code>bin/dcefit</code> is executable.  It should already be, but it doesn't hurt to check. Next copy it to a directory that is in your shell's search path.  A good place on UNIX systems, such as Mac OS X, is <code>/usr/local/bin</code>.</p>

<p>To process a DCEMRI data set from the command line, the minimum invocation is
<code>dcefit /path/to/my/dce/data.mat</code>.</p>
<p>The input data MAT file must contain the following:</p>
<ul>
<li><code>Cp</code>: Arterial input function as a vector, resampled to the DCE time points.</li>
<li><code>DCEdata</code>: DCE data as a 3-D array (1 time by 2 space dimensions).</li>
<li><code>DCEflip</code> : flip angle in deg of DCE data</li>
<li><code>t</code>: time vector representing the dcedata samples.</li>
<li><code>TR</code>: repetition time of DCE scan</li>
<li>R1 information as either <code>R10</code> and <code>S0</code>, representing pre-calculated R1 relaxation maps, or as <code>T1data</code>, indicating that
a multi-flip scan was performed and must be analyzed.  If <code>T1data</code> is supplied, the code also needs <code>T1flip</code>, a vector of flip angles (in degrees) for the multi-flip data.</li>
</ul>
<p>The results will be saved in the current directory as <code>results.mat</code>.  You can override the output file name and location with the <code>--outfile</code> flag.</p>

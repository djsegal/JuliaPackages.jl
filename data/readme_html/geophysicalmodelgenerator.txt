<p dir="auto">Creating consistent 3D images of geophysical and geological datasets and turning that into an input model for geodynamic simulations is often challenging. The aim of this package is to help with this, by providing a number of routines to easily import data and create a consistent 3D visualisation from it in the VTK-toolkit format, which can for example be viewed with <a href="https://www.paraview.org" rel="nofollow">Paraview</a>. In addition, we provide a range of tools that helps to generate input models to perform geodynamic simulations and import the results of such simulations back into julia.</p>

<p dir="auto">Some of the key features are:</p>
<ul dir="auto">
<li>Create 3D volumes of seismic tomography models.</li>
<li>Handle 2D data (e.g., along a cross-section), including surfaces such as the Moho depth.</li>
<li>Plot data along lines (e.g., drillholes) or at points (e.g., earthquake locations, GPS velocities).</li>
<li>Handle both scalar and vector data sets.</li>
<li>Grab screenshots of cross-sections or maps in published papers and view them in 3D (together with other data).</li>
<li>Create a consistent overview that includes all available data of a certain region.</li>
<li>Create initial model setups for the 3D geodynamic code <a href="https://bitbucket.org/bkaus/lamem/src/master/" rel="nofollow">LaMEM</a>.</li>
<li>Import LaMEM timesteps.</li>
</ul>
<p dir="auto">All data is transformed into either a <code>GeoData</code> or a <code>UTMData</code>  structure which contains info about <code>longitude/latitude/depth</code>, <code>ew/ns/depth</code> coordinates along with an arbitrary number of scalar/vector datasets, respectively. All data can be exported to Paraview with the <code>Write_Paraview</code> routine, which transfers the data to a <code>ParaviewData</code> structure (that contains Cartesian Earth-Centered-Earth-Fixed (ECEF) <code>x/y/z</code> coordinates, used for plotting)</p>

<p dir="auto">We rely on a number of additional packages, which are all automatically installed.</p>
<ul dir="auto">
<li><a href="https://github.com/JuliaGeodynamics/GeoParams.jl">GeoParams.jl</a> Defines dimensional units, and makes it easy to convert for km/s to m/s, etc.</li>
<li><a href="https://github.com/jipolanco/WriteVTK.jl">WriteVTK.jl</a> writes VTK files (to be opened with Paraview).</li>
<li><a href="https://github.com/JuliaIO/ImageIO.jl">ImageIO.jl</a>, <a href="https://github.com/JuliaIO/FileIO.jl">FileIO.jl</a>, <a href="https://github.com/JuliaGraphics/Colors.jl">Colors.jl</a> to import screenshots from papers.</li>
<li><a href="https://github.com/JuliaMath/Interpolations.jl">Interpolations.jl</a> for interpolations (for example related to importing screenshots).</li>
</ul>

<p dir="auto">We have used this package to interpret various data sets of the Alps (mostly openly available, sometimes derived from published papers). You can download the resulting paraview files here (using the <code>*.vts</code> format), where we also included the julia scripts to do the work (some of which are also described in more detail in the tutorials). Just unzip the files and open the corresponding <code>*.vts</code> in Paraview.</p>

<p dir="auto">If you want your data be included here as well, give us an email (or even better: send the files with julia scripts).</p>

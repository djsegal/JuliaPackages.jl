<p>Simulation of the propagation of plane waves through arbitrary layered system of thin film stacks using optical matrices. Performs the calculation of reflectance, transmittance, absorptance, electromagnetic field and photonic band gap (for photonic crystals only) using the transfer matrix formalism. For further details see <a href="https://arxiv.org/abs/1809.07708" rel="nofollow">https://arxiv.org/abs/1809.07708</a>.</p>

<div class="highlight highlight-source-julia"><pre>results <span class="pl-k">=</span> <span class="pl-c1">thinfilmoptics</span>(beam, nseq, emfflag<span class="pl-k">=</span><span class="pl-c1">0</span>, h<span class="pl-k">=</span><span class="pl-c1">1</span>, pbgflag<span class="pl-k">=</span><span class="pl-c1">0</span>)</pre></div>

<h2><a id="user-content-input-arguments" class="anchor" aria-hidden="true" href="#input-arguments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Input arguments</h2>
<h3><a id="user-content-lightsource-type" class="anchor" aria-hidden="true" href="#lightsource-type"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>LightSource type</h3>
<p><code>beam::PlaneWave</code> is a subtype that defines the type of light source used in the simulation. At the moment the only subtype accepted is <code>PlaneWave &lt;: LightSource</code>, which is exported by <code>TMMOptics.jl</code>. To call this subtype we can write <code>beam = PlaneWave(λ, λ0, θ)</code>, where the parameters required are described below.</p>
<h4><a id="user-content-wavelength-range" class="anchor" aria-hidden="true" href="#wavelength-range"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Wavelength range</h4>
<p><code>λ</code> defines the wavelength range in nanometers, over which the calculations are performed. The recommended input are 1-d arrays, linear or step ranges (although not explicitly defined). For instance:</p>
<div class="highlight highlight-source-julia"><pre>λ <span class="pl-k">=</span> <span class="pl-c1">LinRange</span>(<span class="pl-c1">200</span>, <span class="pl-c1">1000</span>, <span class="pl-c1">801</span>) <span class="pl-c"><span class="pl-c">#</span> this ensures floating input</span>
λ <span class="pl-k">=</span> <span class="pl-c1">200</span><span class="pl-k">:</span><span class="pl-c1">1000</span>
λ <span class="pl-k">=</span> [<span class="pl-c1">245.</span> <span class="pl-c1">300.</span> <span class="pl-c1">450.</span> <span class="pl-c1">868.</span>]
λ <span class="pl-k">=</span> [<span class="pl-c1">632.</span>]</pre></div>
<h4><a id="user-content-reference-wavelength" class="anchor" aria-hidden="true" href="#reference-wavelength"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Reference wavelength</h4>
<p><code>λ0::Float64</code> sets the reference wavelength in nanometers, especially useful when computing mutlilayer stacks. It is used to compute the geometrical (physical) thickness of each layer and also to plot the profile of index of refraction allowing to visualize the optical structure processed. Accepts floating numbers. For instance:</p>
<div class="highlight highlight-source-julia"><pre>λ<span class="pl-c1">0</span> <span class="pl-k">=</span> <span class="pl-c1">632.</span></pre></div>
<h4><a id="user-content-angle-of-incidence" class="anchor" aria-hidden="true" href="#angle-of-incidence"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Angle of incidence</h4>
<p><code>θ</code> defines the angle of incidence (in degrees) of the wave into the first (incident) medium. It accepts the same type of input as <code>λ</code>. For instance:</p>
<div class="highlight highlight-source-julia"><pre>θ <span class="pl-k">=</span> <span class="pl-c1">LinRange</span>(<span class="pl-c1">0</span>, <span class="pl-c1">90</span>, <span class="pl-c1">901</span>) <span class="pl-c"><span class="pl-c">#</span> this ensures floating input</span>
θ <span class="pl-k">=</span> [<span class="pl-c1">0.</span>] <span class="pl-c"><span class="pl-c">#</span> zero degress --&gt; normal incidence</span>
θ <span class="pl-k">=</span> [<span class="pl-c1">45.</span>] <span class="pl-c"><span class="pl-c">#</span> 45. degrees</span>
θ <span class="pl-k">=</span> <span class="pl-c1">0</span><span class="pl-k">:</span><span class="pl-c1">1</span><span class="pl-k">:</span><span class="pl-c1">60</span></pre></div>
<p>By setting <code>θ</code> and <code>λ</code> as arrays with more than one element each, the quantities that depends on them will be output as 2d-array or 3d-array.</p>
<h4><a id="user-content-polarization-of-the-wave" class="anchor" aria-hidden="true" href="#polarization-of-the-wave"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Polarization of the wave</h4>
<p>The module computes both types, p and s waves, at every run.</p>
<h3><a id="user-content-materials-type" class="anchor" aria-hidden="true" href="#materials-type"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Materials type</h3>
<p>The <code>Material</code> supertype allows two subtypes exported by <code>TMMOptics.jl</code>. The first one is <code>Geometrical(n, d)</code>, which is used to indicate an input layer with its complex index of refraction <code>n</code>and geometrical (physical) thickness <code>d</code> in nanometers. The other subtype allowed is <code>Optical(n, f)</code>, which is used to construct a layer with refractive <code>n</code> and the fraction of optical thickness at <code>beam.λ0</code>. The last one is useful for multilayer stacks normally written in terms of quarter/half-wave layers. Both subtypes can be mixed when defining the structure.</p>
<p><code>nseq::Array{Materials}</code> is an array setting the sequence of the materials involved in the simulation, containing the information about the index of refraction and the thickness and type of thickness that compose the multilayer stack. The order set here will be used to alternate the layers in the multilayer.</p>
<p>The first element in <code>nseq</code> indicates the incident medium while the last one the substrate.</p>
<h4><a id="user-content-geometrical-subtype" class="anchor" aria-hidden="true" href="#geometrical-subtype"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Geometrical subtype</h4>
<p>For instance, in the case of simulation a single layer configuration:</p>
<div class="highlight highlight-source-julia"><pre>l0 <span class="pl-k">=</span> <span class="pl-c1">Geometrical</span>(<span class="pl-c1">air</span>(beam<span class="pl-k">.</span>λ), <span class="pl-c1">0.</span>) <span class="pl-c"><span class="pl-c">#</span> incident medium</span>
l1 <span class="pl-k">=</span> <span class="pl-c1">Geometrical</span>(<span class="pl-c1">sno2f</span>(beam<span class="pl-k">.</span>λ), <span class="pl-c1">150.</span>)
l2 <span class="pl-k">=</span> <span class="pl-c1">Geometrical</span>(<span class="pl-c1">silicon</span>(beam<span class="pl-k">.</span>λ), <span class="pl-c1">0.</span>) <span class="pl-c"><span class="pl-c">#</span> subtrate</span>
nseq <span class="pl-k">=</span> [l0 l1 l2]</pre></div>
<h4><a id="user-content-optical-subtype" class="anchor" aria-hidden="true" href="#optical-subtype"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Optical subtype</h4>
<p>For a multilayer stack (<a href="https://en.wikipedia.org/wiki/Distributed_Bragg_reflector" rel="nofollow">Distributed Bragg Reflector</a>) alternating two different index of refraction materials 1 and 2, wrapped inside an incident medium 1 and a substrate 4, we can write:</p>
<div class="highlight highlight-source-julia"><pre>l0 <span class="pl-k">=</span> <span class="pl-c1">Geometrical</span>(<span class="pl-c1">air</span>(beam<span class="pl-k">.</span>λ), <span class="pl-c1">0.</span>)
l1 <span class="pl-k">=</span> <span class="pl-c1">Optical</span>(<span class="pl-c1">looyengaspheres</span>(<span class="pl-c1">air</span>(beam<span class="pl-k">.</span>λ),<span class="pl-c1">silicon</span>(beam<span class="pl-k">.</span>λ),<span class="pl-c1">0.54</span>), <span class="pl-c1">1</span><span class="pl-k">/</span><span class="pl-c1">4.</span>)
l2 <span class="pl-k">=</span> <span class="pl-c1">Optical</span>(<span class="pl-c1">looyengaspheres</span>(<span class="pl-c1">air</span>(beam<span class="pl-k">.</span>λ),<span class="pl-c1">silicon</span>(beam<span class="pl-k">.</span>λ),<span class="pl-c1">0.86</span>), <span class="pl-c1">1</span><span class="pl-k">/</span><span class="pl-c1">4.</span>)
l3 <span class="pl-k">=</span> <span class="pl-c1">Geometrical</span>(<span class="pl-c1">silicon</span>(beam<span class="pl-k">.</span>λ), <span class="pl-c1">0.</span>)
nseq <span class="pl-k">=</span> [l0 l1 l2 l1 l2 l1 l2 l1 l2 l1 l2 l1 l2 l1 l2 l1 l2 l3]</pre></div>
<p>where <code>l1</code> and <code>l2</code> are quarter-wave optical thickness at the reference wavelength <code>λ0</code>.</p>
<p>For all cases, the thicknesses of the first and last layers are not taken into account as the calculations are performed for semi-infinite incident/substrate media, so we just put a zero regardless of the subtype.</p>
<h3><a id="user-content-electromagnetic-field-emf-flag" class="anchor" aria-hidden="true" href="#electromagnetic-field-emf-flag"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Electromagnetic field (EMF) flag</h3>
<p><code>emfflag::Int64</code> is a flag that indicates whether to compute the field distribution inside the multilayer or not. If <code>emfflag = 1</code>, the elecromagnetic field is computed, while if <code>emfflag = 0</code> is not. This is useful especially when you just want the reflectance or transmission spectra information but not the field distribution, because the computation might be time consuming. The default value is <code>0</code>.</p>
<h3><a id="user-content-number-of-layers-division-for-emf-calculation" class="anchor" aria-hidden="true" href="#number-of-layers-division-for-emf-calculation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Number of layers division for EMF calculation</h3>
<p><code>h::Int64</code> set the number of sub-layers the script will use to divide each layer defined in <code>nseq</code> to calculate the EMF. If you opted for <code>emfflag = 1</code> then you might want to set this to, say <code>10</code> to better visualize the field distribution. You can think as to have more numerical resolution when increasing <code>h</code>. The default value is <code>1</code>. If you choose <code>emfflag = 0</code>, then <code>h</code> is neglected.</p>
<h3><a id="user-content-photonic-band-gap-flag" class="anchor" aria-hidden="true" href="#photonic-band-gap-flag"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Photonic band gap flag</h3>
<p><code>pbgflag::Int64</code> indicates whether to calculate or not the photonic dispersion. It is only useful when computing DBR, i.e., when computing binary periodic systems that alternate two different refractive indexes. By default <code>pbgflag = 0</code>.</p>
<h2><a id="user-content-output" class="anchor" aria-hidden="true" href="#output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Output</h2>
<p><code>results::Results &lt;: Output</code> is a structure containing different subtypes of outputs depending on their relation (more or less). There are five subtypes included in the output:</p>
<h3><a id="user-content-spectra-subtype-of-output" class="anchor" aria-hidden="true" href="#spectra-subtype-of-output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Spectra subtype of output</h3>
<p><code>results.Spectra::Results &lt;: Output</code> contains information on the reflectance, transmittance and fresnell coefficients spectra. Inside this subtype there exist the following fields:</p>
<p><code>Rp::Array{Float64}(length(beam.λ), length(beam.θ))</code>: p-wave reflectance.</p>
<p><code>Rs::Array{Float64}(length(beam.λ), length(beam.θ))</code>: s-wave reflectance.</p>
<p><code>Tp::Array{Float64}(length(beam.λ), length(beam.θ))</code>: p-wave transmittance.</p>
<p><code>Ts::Array{Float64}(length(beam.λ), length(beam.θ))</code>: s-wave transmittance.</p>
<p><code>ρp::Array{ComplexF64}(length(beam.λ), length(beam.θ))</code>: p-wave complex reflection coefficient.</p>
<p><code>ρs::Array{ComplexF64}(length(beam.λ), length(beam.θ))</code>: s-wave complex reflection coefficient.</p>
<p><code>τp::Array{ComplexF64}(length(beam.λ), length(beam.θ))</code>: p-wave complex transmission coefficient.</p>
<p><code>τs::Array{ComplexF64}(length(beam.λ), length(beam.θ))</code>: s-wave complex transmission coefficient.</p>
<h3><a id="user-content-field-subtype-of-output" class="anchor" aria-hidden="true" href="#field-subtype-of-output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Field subtype of output</h3>
<p><code>results.Field::Results &lt;: Output</code> contains information on the electromagnetic field distribution calculated, with the following fields:</p>
<p><code>emfp::Array{Float64}(length(beam.λ), length(beam.θ), length(results.Misc.ℓ))</code>: p-wave electric field distribution.</p>
<p><code>emfs::Array{Float64}(length(beam.λ), length(beam.θ), length(results.Misc.ℓ))</code>: s-wave electric field distribution.</p>
<h3><a id="user-content-bloch-subtype-of-output" class="anchor" aria-hidden="true" href="#bloch-subtype-of-output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Bloch subtype of output</h3>
<p><code>results.Bloch::Results &lt;: Output</code> contains information on the Bloch wavevectors calculated for DBRs, with the following fields:</p>
<p><code>κp::Array{ComplexF64}(length(beam.λ), length(beam.θ))</code>: p-wave Bloch dispersion wavevectors.</p>
<p><code>κs::Array{ComplexF64}(length(beam.λ), length(beam.θ))</code>: s-wave Bloch dispersion wavevectors.</p>
<h3><a id="user-content-misc-subtype-of-output" class="anchor" aria-hidden="true" href="#misc-subtype-of-output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Misc subtype of output</h3>
<p><code>results.Misc::Results &lt;: Output</code> gives information about the geometrical (physical) thicknesses in nanometers (even if you input the optical thickness information, the output returns the physical one), the geometrical thickness taking into account the <code>h</code> number defined above, and the index of refraction of each layer defined in <code>nseq</code> at the wavelength of reference <code>λ0</code>. The fields are as follow:</p>
<p><code>d::Array{Float64}(size(nseq,2)-2)</code>: geometrical (physical) thickness of each layer as ordered by <code>n</code> [nm].</p>
<p><code>ℓ::Array{Float64}((size(nseq,2)-2)*h)</code>: Geometrical (physical) thickness of each layer, where each layer is divided into <code>h</code> sub-layers [nm].</p>
<p><code>nλ0::Array{Float64}(size(nseq,2))</code>: profile of index of refraction profile computed at the wavelength reference <code>λ0</code>.</p>
<h3><a id="user-content-admphase-subtype-of-output" class="anchor" aria-hidden="true" href="#admphase-subtype-of-output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>AdmPhase subtype of output</h3>
<p><code>results.AdmPhase::Results &lt;: Output</code> wraps the admittances for both polarizations and the phase shift angle (thickness) for the whole structure. The fields are as follow:</p>
<p><code>ηp::Array{ComplexF64}(length(beam.λ), length(beam.θ), length(size(nseq,2)))</code>: p-wave admittance of the structure.</p>
<p><code>ηs::Array{ComplexF64}(length(beam.λ), length(beam.θ), length(size(nseq,2)))</code>: s-wave admittance of the structure.</p>
<p><code>δ::Array{ComplexF64}(length(beam.λ), length(beam.θ), length(size(nseq,2))-2)</code>: phase shift (thickness) of the structure without the incident and substrate media.</p>
<h2><a id="user-content-examples-of-index-of-refraction-functions-used" class="anchor" aria-hidden="true" href="#examples-of-index-of-refraction-functions-used"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Examples of index of refraction functions used</h2>
<p>The next two modules are optional since include functions with index of refraction information compiled for the purpose of the docs. They can also be used as provided for problems involving the same materials though.</p>
<h4><a id="user-content-ridbjl" class="anchor" aria-hidden="true" href="#ridbjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>RIdb.jl</h4>
<p>Module containing a collection of functions with index of refration for the following materials: aluminum, air, bk7, chrome, dummy, glass, gold, silicon, silicontemperature, silver, sno2f, h2o, etoh. These functions accept as input arguments the wavelength range <code>λ</code>, and return the index of refraction as a complex floating number. Even for non-aborbent materials (where a list of zeros in the imaginary part is placed), the index works better with complex character. Users can use their own functions as well that output complex types though.</p>
<p>This module depends on <a href="https://github.com/JuliaMath/Interpolations.jl">Interpolations.jl</a> to return the index of refraction as a function of the input wavelength, which might differ from that of the experimental data. Also depends on <a href="https://github.com/JuliaIO/HDF5.jl">HDF5.jl</a> since the data is compiled into a h5 file for simplicity.</p>
<p>Use of this module as follow, e.g.:</p>

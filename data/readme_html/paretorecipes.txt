<h2 dir="auto"><a id="user-content-summary" class="anchor" aria-hidden="true" href="#summary"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Summary</h2>
<p dir="auto">ParetoRecipes generates Pareto plots from a set of points in a multiobjective optimization problem using <a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a>.  A Pareto plot consists of the set of points together with a <a href="https://en.wikipedia.org/wiki/Pareto_efficiency" rel="nofollow">Pareto front</a> (alternatively, Pareto frontier or Pareto set).  Colloquially, we will refer to such plots as <em>Paretos</em>.</p>
<p dir="auto">ParetoRecipes does <em>not</em> focus on the multiobjective optimization itself, or even on breadth of exact and approximate algorithms to trace out Pareto fronts for optimization problems.  You can find such features and more in the <a href="https://github.com/anriseth/MultiJuMP.jl">MultiJuMP</a> package.</p>
<p dir="auto">Currently, ParetoRecipes only supports two-dimensional (2D) Paretos with an extremely inefficient brute-force algorithm for finding the Pareto front.</p>

<p dir="auto">Thomas Breloff (<a href="https://github.com/tbreloff">@tbreloff</a>) created the lightweight <a href="https://github.com/JuliaPlots/RecipesBase.jl">RecipesBase</a> package that inspired the design of ParetoRecipes.</p>
<p dir="auto">ParetoRecipes offers a set of <em>recipes</em>.  Recipes abstract away the dependencies on plotting libraries like <a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a>.  Each recipe is a <code>@recipe</code> macro from <a href="https://github.com/JuliaPlots/RecipesBase.jl">RecipesBase</a>.  There are different kinds of recipes. The <code>@recipe</code> for a Pareto plot in ParetoRecipes is a <em>series recipe</em> (<code>@series</code>) that composes two plots: a scatter plot (<code>:scatter</code>) and a line plot (<code>:path</code>).</p>
<p dir="auto">The <a href="https://github.com/JuliaPlots/GraphRecipes.jl">GraphRecipes.jl</a> repo has many more examples of the power of series recipes.</p>

<p dir="auto">ParetoRecipes does not currently support other concepts named after the economist <a href="https://en.wikipedia.org/wiki/Vilfredo_Pareto" rel="nofollow">Vilfredo Pareto</a>:</p>
<ul dir="auto">
<li><a href="https://en.wikipedia.org/wiki/Pareto_chart" rel="nofollow">Pareto Charts</a>: a different kind of visualization whose series recipe would consist of a sorted histogram composed with a cumulative line plot</li>
<li><a href="https://en.wikipedia.org/wiki/Pareto_distribution" rel="nofollow">Pareto Distributions</a>: offered in <a href="https://github.com/JuliaStats/Distributions.jl">Distributions.jl</a></li>
</ul>

<p dir="auto">This package implements a variety of interpolation schemes for the
Julia language.  It has the goals of ease-of-use, broad algorithmic
support, and exceptional performance.</p>
<p dir="auto">Currently this package supports
<a href="https://en.wikipedia.org/wiki/B-spline" rel="nofollow">B-splines</a> and also
irregular grids.  The API has been designed with
intent to support more options. Initial support for Lanczos
interpolation was recently added. Pull-requests are more than welcome!
It should be noted that the API may continue to evolve over time.</p>
<p dir="auto">At the bottom of this page, you can find a "performance shootout"
among these methods (as well as SciPy's <code>RegularGridInterpolator</code>).</p>

<p dir="auto">Other interpolation packages for Julia include:</p>
<ul dir="auto">
<li><a href="https://github.com/floswald/ApproXD.jl">ApproXD.jl</a> implements B-spline and linear interpolation in Julia.</li>
<li><a href="https://github.com/dawbarton/BarycentricInterpolation.jl">BarycentricInterpolation.jl</a> implements the Barycentric formula for polynomial interpolation on equispaced points and Chebyshev points of the first and second kind.</li>
<li><a href="https://github.com/markmbaum/BasicInterpolators.jl">BasicInterpolators.jl</a> provides a collection of common interpolation recipes for basic applications.</li>
<li><a href="https://github.com/jipolanco/BSplineKit.jl">BSplineKit.jl</a> offers tools for B-spline based Galerkin and collocation methods, including for interpolation and approximation.</li>
<li><a href="https://github.com/lungben/Curves.jl">Curves.jl</a> supports log-interpolation via immutable <code>Curve</code> objects.</li>
<li><a href="https://github.com/PumasAI/DataInterpolations.jl">DataInterpolations.jl</a> is a library for performing interpolations of one-dimensional data.</li>
<li><a href="https://github.com/kbarbary/Dierckx.jl">Dierckx.jl</a> is a wrapper for the dierckx Fortran library, which also underlies <code>scipy.interpolate</code>.</li>
<li><a href="https://github.com/gher-ulg/DIVAnd.jl">DIVAnd.jl</a> for N-dimensional smoothing interpolation.</li>
<li><a href="https://github.com/stevengj/FastChebInterp.jl">FastChebInterp.jl</a> does fast multidimensional Chebyshev interpolation on a hypercube using separable grid of interpolation points.</li>
<li><a href="https://github.com/JuliaFEM/FEMBasis.jl">FEMBasis.jl</a> contains interpolation routines for standard finite element function spaces.</li>
<li><a href="https://github.com/emmt/FineShift.jl">FineShift.jl</a> does fast sub-sample shifting of multidimensional arrays.</li>
<li><a href="https://github.com/bionanoimaging/FourierTools.jl">FourierTools.jl</a> includes sinc interpolation for up and down sampling.</li>
<li><a href="https://github.com/sisl/GridInterpolations.jl">GridInterpolations.jl</a> performs multivariate interpolation on a rectilinear grid.</li>
<li><a href="https://github.com/emmt/InterpolationKernels.jl">InterpolationKernels.jl</a> provides a library of interpolation kernels.</li>
<li><a href="https://github.com/francescoalemanno/KissSmoothing.jl">KissSmoothing.jl</a> implements denoising and a Radial Basis Function estimation procedure.</li>
<li><a href="https://github.com/jw3126/LinearInterpolations.jl">LinearInterpolations.jl</a> allows for interpolation using weighted averages allowing probability distributions, rotations, and other Lie groups to be interpolated.</li>
<li><a href="https://github.com/emmt/LinearInterpolators.jl">LinearInterpolators.jl</a> provides linear interpolation methods for Julia based on InterpolationKernels.jl, above.</li>
<li><a href="https://github.com/sisl/LocalFunctionApproximation.jl">LocalFunctionApproximation.jl</a> provides local function approximators that interpolates a scalar-valued function across a vector space.</li>
<li><a href="https://github.com/gerlero/PCHIPInterpolation.jl">PCHIPInterpolation.jl</a> for monotonic interpolation.</li>
<li><a href="https://github.com/RJDennis/PiecewiseLinearApprox.jl">PiecewiseLinearApprox.jl</a> performs piecewise linear interpolation over an arbitrary number of dimensions.</li>
<li><a href="https://github.com/eljungsk/ScatteredInterpolation.jl">ScatteredInterpolation.jl</a> interpolates scattered data in arbitrary dimensions.</li>
</ul>
<p dir="auto">Some of these packages support methods that <code>Interpolations</code> does not,
so if you can't find what you need here, check one of them or submit a
pull request here.</p>
<p dir="auto">If you would like to list a registered package that is related to interpolation, please create a Github issue.</p>

<p dir="auto">In the <code>perf</code> directory, you can find a script that tests
interpolation with several different packages.  We consider
interpolation in 1, 2, 3, and 4 dimensions, with orders 0
(<code>Constant</code>), 1 (<code>Linear</code>), and 2 (<code>Quadratic</code>).  Methods include
Interpolations <code>BSpline</code> (<code>IBSpline</code>) and <code>Gridded</code> (<code>IGridded</code>),
methods from the <a href="https://github.com/timholy/Grid.jl">Grid.jl</a>
package, methods from the
<a href="https://github.com/kbarbary/Dierckx.jl">Dierckx.jl</a> package, methods
from the
<a href="https://github.com/sisl/GridInterpolations.jl">GridInterpolations.jl</a>
package (<code>GI</code>), methods from the
<a href="https://github.com/floswald/ApproXD.jl">ApproXD.jl</a> package, and
methods from SciPy's <code>RegularGridInterpolator</code> accessed via <code>PyCall</code>
(<code>Py</code>).  All methods
are tested using an <code>Array</code> with approximately <code>10^6</code> elements, and
the interpolation task is simply to visit each grid point.</p>
<p dir="auto">First, let's look at the two B-spline algorithms, <code>IBspline</code> and
<code>Grid</code>.  Here's a plot of the "construction time," the amount of time
it takes to initialize an interpolation object (smaller is better):</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="perf/constructionB.png"><img src="perf/constructionB.png" alt="construction" style="max-width: 100%;"></a></p>
<p dir="auto">The construction time is negligible until you get to second order
(quadratic); that's because quadratic is the lowest order requiring
the solution of tridiagonal systems upon construction.  The solvers
used by Interpolations are much faster than the approach taken in
Grid.</p>
<p dir="auto">Now let's examine the interpolation performance.  Here we'll measure
"throughput", the number of interpolations performed per second
(larger is better):</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="perf/rateB.png"><img src="perf/rateB.png" alt="throughput" style="max-width: 100%;"></a></p>
<p dir="auto">Once again, Interpolations wins on every test, by a factor that ranges
from 7 to 13.</p>
<p dir="auto">Now let's look at the "gridded" methods that allow irregular spacing
along each axis.  For some of these, we compare interpolation performance in
both "vectorized" form <code>itp[xvector, yvector]</code> and in "scalar" form
<code>for y in yvector, x in xvector; val = itp[x,y]; end</code>.</p>
<p dir="auto">First, construction time (smaller is better):</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="perf/constructionG.png"><img src="perf/constructionG.png" alt="construction" style="max-width: 100%;"></a></p>
<p dir="auto">Missing dots indicate cases that were not tested, or not supported by
the package.  (For construction, differences between "vec" and
"scalar" are just noise, since no interpolation is performed during
construction.)  The only package that takes appreciable construction
time is Dierckx.</p>
<p dir="auto">And here's "throughput" (larger is better). To ensure we can see the
wide range of scales, here we use "square-root" scaling of the y-axis:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="perf/rateG.png"><img src="perf/rateG.png" alt="throughput" style="max-width: 100%;"></a></p>
<p dir="auto">For 1d, the "Dierckx scalar" and "GI" tests were interrupted because
they ran more than 20 seconds (far longer than any other test).  Both
performed much better in 2d, interestingly.  You can see that
Interpolations wins in every case, sometimes by a very large margin.</p>

<p dir="auto">This package is being maintained but not actively developed. Maintenance is
focused on fixing bugs and issues with the current code base. New features are
welcome via pull requests and will be reviewed and released in a timely fashion.</p>
<p dir="auto">If you would like to become involved in maintenance or active development of
the package please feel free to get in touch via a Github issue.</p>
<p dir="auto">This package follows semantic version in that documented features should not
break without changing the minor version.</p>
<p dir="auto">See the <a href="NEWS.md">news</a> for details on how to update between breaking releases,

<p dir="auto">This package contains definitions for common functions that are useful for
symbolic expression manipulation. This package is a simplified fork of
<a href="https://github.com/JuliaSymbolics/TermInterface.jl">TermInterface.jl</a></p>

<p dir="auto">You should define the following methods for an abstract syntax tree to implement SyntaxInterface.jl</p>

<p dir="auto">Check if <code>x</code> represents an expression tree. If returns true, it will be assumed
that <code>operation(::T)</code> and <code>arguments(::T)</code> methods are defined.</p>
<h4 dir="auto"><a id="user-content-operationxt" class="anchor" aria-hidden="true" href="#operationxt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>operation(x::T)</code></h4>
<p dir="auto">Returns the operation, or head, performed by an expression tree node. If <code>x</code> represents a function call, for example, the head might represent
the function being called. Called
only if <code>istree(::T)</code> is true.</p>
<h4 dir="auto"><a id="user-content-argumentsxt" class="anchor" aria-hidden="true" href="#argumentsxt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>arguments(x::T)</code></h4>
<p dir="auto">Returns the arguments, or children, to an expression tree node. Called only if <code>istree(x)</code> is <code>true</code>.</p>
<p dir="auto">In addition, the methods for <code>Base.hash</code> and <code>Base.isequal</code> should also be implemented by the types for the purposes of substitution and equality matching respectively.</p>

<p dir="auto">Construct a new term with the operation <code>f</code> and arguments <code>args</code>, the term
should be similar to <code>x</code> in type. If not specified, the result is computed as
<code>f(args...)</code>. This method helps avoid splatting. To define this method for your
term type, define <code>similarterm(::Type{&lt;:T}, ...)</code>.</p>

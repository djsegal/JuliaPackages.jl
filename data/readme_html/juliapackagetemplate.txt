<h1><a id="user-content-juliapackagetemplatejl" class="anchor" aria-hidden="true" href="#juliapackagetemplatejl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>JuliaPackageTemplate.jl</h1>
<p>JuliaPackageTemplate provides an example Julia project template to quickly setup
continuous integration, test coverage reports, and automatic documentation deployment.</p>
<h2><a id="user-content-documentation" class="anchor" aria-hidden="true" href="#documentation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Documentation</h2>
<p>The documentation for the package can be found here: <a href="https://sisl.github.io/JuliaPackageTemplate.jl/latest" rel="nofollow">https://sisl.github.io/JuliaPackageTemplate.jl/latest</a></p>
<p>More example code and examples will be added as time permits.</p>
<h2><a id="user-content-configuring-package-name" class="anchor" aria-hidden="true" href="#configuring-package-name"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuring Package Name</h2>
<p>To start off with, look through the package and replace <code>JuliaPackageTemplate</code>
with your own project name.</p>
<p><code>deps</code> contains C/C++ file dependencies of the packages which are compiled when
the package is installed by using the BinDeps.jl package.</p>
<p><code>docs</code> contains</p>
<p><code>src</code> contains the Julia source code of the package.</p>
<p><code>test</code> contains unit tests which can be run locally</p>
<p>Also be sure to update the project UUID in <code>Project.toml</code>!</p>
<h2><a id="user-content-testing-locally" class="anchor" aria-hidden="true" href="#testing-locally"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Testing Locally</h2>
<p>It is possible to test the package and code locally before commiting the update
and triggering a CI build.</p>
<p>First, open a terminal window and navigate to the package root directory and
start Julia</p>
<div class="highlight highlight-source-shell"><pre>deddy@Andromeda:<span class="pl-k">~</span>$ <span class="pl-c1">cd</span> /Stanford/repos/JuliaPackageTemplate.jl
deddy@Andromeda:<span class="pl-k">~</span>/Stanford/repos/JuliaPackageTemplate.jl$ julia
julia<span class="pl-k">&gt;</span></pre></div>
<p>Next, activate the local package development environment</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> ]
(v1<span class="pl-c1">.0</span>) pkg<span class="pl-k">&gt;</span> activate .
(JuliaPackageTemplate) pkg<span class="pl-k">&gt;</span> </pre></div>
<p>From here we can test the package by simply typing in the <code>test</code> command:</p>
<div class="highlight highlight-source-julia"><pre>(JuliaPackageTemplate) pkg<span class="pl-k">&gt;</span> test</pre></div>
<p>If the package depends on C/C++ source files, these first must be compiled before
testing the package. In which case testing would involve two commands:</p>
<div class="highlight highlight-source-julia"><pre>(JuliaPackageTemplate) pkg<span class="pl-k">&gt;</span> build
(JuliaPackageTemplate) pkg<span class="pl-k">&gt;</span> test</pre></div>
<h2><a id="user-content-setting-up-continuous-integration" class="anchor" aria-hidden="true" href="#setting-up-continuous-integration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Setting Up Continuous Integration</h2>
<p>To setup continuous integration for the package we will use Travis-CI. Travis is
free to use for open source projects (Thank you Travis!), or for build of private
repositories a subscription can be purchased.</p>
<p>To setup continuous integration your repository must contain a <code>.travis.yml</code> file
AND continuous integration must be enabled for your repository on the TRAVIS CI
webpage. This can be done either for your presonal repositories here:</p>
<p><code>https://travis-ci.org/account/repositories</code></p>
<p>or for your organizations' repositories here:</p>
<p><a href="https://travis-ci.org/organizations/YOUR_GITHUB_ORGANIZATION_NAME/repositories" rel="nofollow">https://travis-ci.org/organizations/YOUR_GITHUB_ORGANIZATION_NAME/repositories</a></p>
<p>Note: If the project does not appear immediately, you may need to hit the "sync
repositories" button to have it appear.</p>
<h2><a id="user-content-setting-up-test-coverage" class="anchor" aria-hidden="true" href="#setting-up-test-coverage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Setting Up Test Coverage</h2>
<p>To set up test coverage, go to <a href="https://coveralls.io/repos/new" rel="nofollow">coveralls.io</a>,
login with your github account, and activate the project to add coverage reports.</p>
<p>Note: If the project does not appear immediately, you may need to hit the "sync
repositories" button to have it appear.</p>
<h2><a id="user-content-setting-up-documentation-deployment" class="anchor" aria-hidden="true" href="#setting-up-documentation-deployment"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Setting Up Documentation Deployment</h2>
<p>If you have not done so, set up github pages by adding a pages repository to your
github account by following the <a href="https://pages.github.com/">setup instructions</a> or just add an empty repository
named <code>username.github.io</code> or <code>orgname.github.io</code> to your personal or organization account. This repository can be private.</p>
<p>Documentation and documentation deployment is accomplished with the Julia packager
<code>Documenter.jl</code></p>
<p>To setup the automated deployment of documentation as part of the CI build process
follow the <a href="https://juliadocs.github.io/Documenter.jl/stable/man/hosting/" rel="nofollow">Deploy Instructions</a>.</p>
<p>This involves two steps:</p>
<ol>
<li>Adding a deploy key to the github deploy keys to allow travis CI to push to new pages to the repository.</li>
<li>Adding an environment variable to your Travis CI build settings with the deployment keys.</li>
</ol>

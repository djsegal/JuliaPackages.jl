<p dir="auto">A package for optimizing robustified Non-Linear Least Squares (NLLS) problems, with the following features:</p>
<ul dir="auto">
<li><strong>Simple interface</strong>: Large scale problems can be defined with relatively little code. Derivatives are computed automatically (though can be user-provided if desired).</li>
<li><strong>Robust</strong>: Residual blocks can be robustified easily.</li>
<li><strong>Non-Euclidean variables</strong>: Variables do not need to exist in a Euclidean space. For example, 3D rotations can be represented as a 9 parameter SO(3) matrix, yet retain a minimal 3DoF update parameterization.</li>
</ul>
<p dir="auto">Features not currently supported:</p>
<ul dir="auto">
<li><strong>Constraints</strong>: Bounds on variables are not explicitly supported. However, bounded variables can be implemented using special, non-Euclidean parameterizations.</li>
</ul>

<h3 dir="auto"><a id="user-content-problem-definition" class="anchor" aria-hidden="true" href="#problem-definition"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Problem definition</h3>
<p dir="auto">Each NLLS problem is defined using two types of data structure:</p>
<ul dir="auto">
<li><strong>Variable blocks</strong>, which contain the parameters to be optimized.</li>
<li><strong>Residual blocks</strong>, which contain the data that defines the NLLS function to be minimized w.r.t. the variables.</li>
</ul>
<p dir="auto">Each instance of these two types must implement a standard API, as follows.</p>
<h4 dir="auto"><a id="user-content-variable-blocks" class="anchor" aria-hidden="true" href="#variable-blocks"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Variable blocks</h4>
<ul dir="auto">
<li><strong><code>N::Int = nvars(::MyVar)</code></strong> returns the intrinsic dimensionality, N, of the variable block.</li>
<li><strong><code>newvar::MyVar = update(oldvar::MyVar, updatevec)</code></strong> updates a variable, given an update vector of length N.</li>
</ul>
<h4 dir="auto"><a id="user-content-residual-blocks" class="anchor" aria-hidden="true" href="#residual-blocks"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Residual blocks</h4>
<ul dir="auto">
<li><strong><code>N::Int = nvars(::MyRes)</code></strong> returns the number of variable blocks the residual block depends on.</li>
<li><strong><code>M::Int = nres(::MyRes)</code></strong> returns the number of scalar residuals in the block.</li>
<li><strong><code>varind::SVector{N, Int} = varindices(res::MyRes)</code></strong> returns the indices of the variable blocks (stored in problem) that this residual block depends on. These values are assumed to remain fixed for the duration of an optimization.</li>
<li><strong><code>resvars::Tuple = getvars(res::MyRes, allvars::Vector)</code></strong> returns a tuple containing the variables the residual block depends on.</li>
<li><strong><code>res::SVector{M, Float} = computeresidual(res::MyRes, resvars...)</code></strong> returns the computed residual block.</li>
<li><strong><code>res::SVector{M, Float}, jac::SMatrix{M, P, Float} = computeresjac(::Val{varflags}, res::MyRes, resvars...)</code></strong> (<em>optional</em>) returns the computed residual block and its Jacobian, for all the variables whose corresponding bit in <code>varflags</code> is set. If this function isn't provided, the Jacobian is computed using ForwardDiff auto-differentiation.</li>
<li><strong><code>robker = robustkernel(res::MyRes)</code></strong> (<em>optional</em>) returns the robust kernel data structure for the residual block. If this function isn't provided, the cost is the squared norm of the residual block.</li>
</ul>
<p dir="auto">A problem is then defined by creating an <code>NLLSProblem</code> object, and adding variables and residuals to it using the <code>addvariable!</code> and <code>addresidual!</code> methods respectively.</p>
<h3 dir="auto"><a id="user-content-optimization" class="anchor" aria-hidden="true" href="#optimization"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Optimization</h3>
<p dir="auto">Optimization is done as follows:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="    result::NLLSResult = optimize!(problem::NLLSProblem, options::NLLSOptions)"><pre class="notranslate"><code>    result::NLLSResult = optimize!(problem::NLLSProblem, options::NLLSOptions)
</code></pre></div>
<p dir="auto">Various optimizer <code>options</code> can be defined. During optimization, the optimizer updates variable blocks (stored in <code>problem</code>) in-place. Information about the optimization is retruned in <code>result</code>.</p>

<ul dir="auto">

<p><a target="_blank" rel="noopener noreferrer" href="https://github.com/AlgebraicJulia/AlgebraicRelations.jl/workflows/Tests/badge.svg"><img src="https://github.com/AlgebraicJulia/AlgebraicRelations.jl/workflows/Tests/badge.svg" alt="Tests" style="max-width:100%;"></a></p>
<p>AlgebraicRelations.jl is a Julia library built to provide an intuitive and
elegant method for generating and querying a scientific database. This
package provides tooling for defining database schemas,
generating query visualizations, and connecting directly up to a PostgreSQL
server. This package is built on top of
<a href="https://github.com/AlgebraicJulia/Catlab.jl">Catlab.jl</a> which is the
powerhouse behind its functions.</p>

<h3><a id="user-content-defining-a-schema" class="anchor" aria-hidden="true" href="#defining-a-schema"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Defining a Schema</h3>
<p>Within this library, we define database schemas based on the <em>presentation</em> of a
workflow (more generally, the presentation of a symmetric monoidal category).
The presentation of a workflow includes the <strong>data types</strong> of products in the
workflow (objects in an SMC) and the <strong>processes</strong> that transform these products
(homomorphisms in an SMC). We will give an example of defining the schema of a
traditional computer vision workflow. This involves extracting images from a
file, performing a test/train split on images, training a neural network on
images, and finally evaluating a network on images. This example is also
presented in <a href="examples/ml_workflow_demo/ml_demo.ipynb">this notebook</a>.</p>
<h4><a id="user-content-defining-types" class="anchor" aria-hidden="true" href="#defining-types"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Defining Types</h4>
<p>In order to define types for the presentation, we need to provide the name of
the type (e.g. <code>File</code> for compressed files of images) and then the Julia
datatype which can store this type (The filename can be stored uniquely as a
<code>String</code>). The definition of all types that we will need for our example is as
follows:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="# Initialize presentation object

                                          (:NeuralNet, String),
                                          (:Accuracy, Real),
                                          (:Metadata, String)]);

<h4><a id="user-content-defining-processes" class="anchor" aria-hidden="true" href="#defining-processes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Defining Processes</h4>
<p>To define processes that operate on these types, we need three pieces of
information. First, we need the name of the processes (<code>extract</code> for the
process that extracts images from files), the input types (<code>File</code> for the file
to extract) and the output types (<code>Images</code> for the images which were
extracted). The symbol <code>⊗</code> (monoidal product) joins two types, allowing for multiple types
in the inputs and outputs of processes. To the schema, this means nothing more than that,
for the process <code>train</code> there are two objects need for the input, the first of
type <code>NeuralNet</code> and the second of type <code>Images</code>.</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="# Add Processes to presentation
extract, split, train,
evaluate = add_processes!(present, [(:extract, File, Images),
                                    (:split, Images, Images⊗Images),
                                    (:train, NeuralNet⊗Images, NeuralNet⊗Metadata),
                                    (:evaluate, NeuralNet⊗Images, Accuracy⊗Metadata)]);
"><pre><span class="pl-c"><span class="pl-c">#</span> Add Processes to presentation</span>
extract, split, train,
evaluate <span class="pl-k">=</span> <span class="pl-c1">add_processes!</span>(present, [(<span class="pl-c1">:extract</span>, File, Images),
                                    (<span class="pl-c1">:split</span>, Images, Images⊗Images),
                                    (<span class="pl-c1">:train</span>, NeuralNet⊗Images, NeuralNet⊗Metadata),
                                    (<span class="pl-c1">:evaluate</span>, NeuralNet⊗Images, Accuracy⊗Metadata)]);</pre></div>

<h3><a id="user-content-creating-queries" class="anchor" aria-hidden="true" href="#creating-queries"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Creating Queries</h3>
<p>In order to create queries, we use the <code>@query</code> macro (based on the <code>@relation</code>
macro in Catlab). For this, we must specify a list of objects to get as results
of the query, list of all objects used in the query, and finally a list of
relationships between these objects (based on the primitives defined for the
workflow). In this case, the relationships between objects are the processes
from the presentation and the types of objects are the types defined in the
presentation. Following is an example workflow</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="q = @query TrainDB() (im_train, nn, im_test, acc, md2) where (im_train, im_test, nn,
                                                              nn_trained, acc, md,
                                                              md2, _base_acc, im) begin
    train(nn, im_train, nn_trained, md)
    evaluate(nn_trained, im_test, acc, md2)
    split(im, im_train, im_test)
    &gt;=(acc, _base_acc)
end
print(to_sql(q))
"><pre>q <span class="pl-k">=</span> <span class="pl-c1">@query</span> <span class="pl-c1">TrainDB</span>() (im_train, nn, im_test, acc, md2) <span class="pl-k">where</span> (im_train, im_test, nn,
                                                              nn_trained, acc, md,
                                                              md2, _base_acc, im) <span class="pl-k">begin</span>
    <span class="pl-c1">train</span>(nn, im_train, nn_trained, md)
    <span class="pl-c1">evaluate</span>(nn_trained, im_test, acc, md2)
    <span class="pl-c1">split</span>(im, im_train, im_test)
    <span class="pl-k">&gt;=</span>(acc, _base_acc)
<span class="pl-k">end</span>
<span class="pl-c1">print</span>(<span class="pl-c1">to_sql</span>(q))</pre></div>

<p>The connection to PostgreSQL is fairly straightforward. We first create a
connection using the <a href="https://invenia.github.io/LibPQ.jl/stable/" rel="nofollow">LibPQ.jl</a>
library:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="conn = Connection(&quot;dbname=test_db&quot;);
"><pre>conn <span class="pl-k">=</span> <span class="pl-c1">Connection</span>(<span class="pl-s"><span class="pl-pds">"</span>dbname=test_db<span class="pl-pds">"</span></span>);</pre></div>
<p>We then can prepare statements and run them with arguments like:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="statement = prepare(conn,q)

"><pre>statement <span class="pl-k">=</span> <span class="pl-c1">prepare</span>(conn,q)
<span class="pl-c1">execute</span>(statement, [<span class="pl-c1">0.6</span>])</pre></div>
<p>which will obtain all of the rows from the previous query which contain
an accuracy of greater than 0.6.</p>
<p>The <code>execute</code> function will return a <code>DataFrame</code> object (from the
<a href="http://juliadata.github.io/DataFrames.jl/stable/" rel="nofollow"><code>DataFrames.jl</code></a> library)</p>

<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="org-mode-jl.svg"><img src="org-mode-jl.svg" align="right" style="max-width: 100%;"></a></p>
<p dir="auto">A library for working with Org files. Specifically, this provides utilities for:</p>
<ul dir="auto">
  <li>Parsing Org text to an AST (Abstract Syntax Tree)</li>
  <li>Regenerating Org text from an AST</li>
  <li>Basic manipulation and analysis of an Org document AST</li>
  <li>Generating basic representations Org content in the terminal, and a few other forms.</li>
</ul>
<p dir="auto">With the exception of some of the <i>particularly fancy</i> capabilities provided by
  <code>org-mode</code> (like Babel and Calc-based spreadsheeting), this project aims to
  exactly match the interpretation of Org mode markup â€” specifically the AST
  generated by <code>org-element.el</code>. This goal is not yet achieved, however
  the bulk of the work is now complete.</p>
<p dir="auto"><b>Org.jl</b> implements the vast majority of the <a href="https://orgmode.org/worg/dev/org-syntax.html" rel="nofollow">org-syntax</a> document (see the
  Progress table). This can be checked by looking at <code>Org.compatability</code> in
  Julia.</p>

<pre lang="julia"># after installing with ~] add Org~ or ~Pkg.add("Org")~
using Org
text1 = org"Some *Org* markup, written with ease using the ~org\"\"~ macro."
parsetree(text1)  # show the generated parse tree

text2 = parse(OrgDoc, "Some *Org* markup, written with ease using the ~parse~ function.")
diff(text1, text2)  # show the components of the parse trees that differ

dochead = @doc Org.Heading  # the documentation for the Heading component (::OrgDoc)
org(dochead)  # generate Org text that produces the Org.Heading object
string(dochead)  # as above, but produces a String

parse(OrgDoc, string(dochead)) == dochead  # round-trip equality

# get the lang of each source block
[c.lang for c in dochead.components if c isa Org.SourceBlock]
</pre>

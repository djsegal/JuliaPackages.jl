<ol start="3" dir="auto">
<li>
<p dir="auto">The wizard will take you through a process of building your software package. Note that the wizard may need to download a new compiler shard for each platform targeted, and there are quite a few of these, so a fast internet connection can be helpful.  The output of this stage is a <code>build_tarballs.jl</code> file, which is most commonly deployed as a pull request to the community buildtree <a href="https://github.com/JuliaPackaging/Yggdrasil">Yggdrasil</a>.  For experienced users, it is often more convenient to directly copy/modify an existing <code>build_tarballs.jl</code> file within Yggdrasil, then simply open a pull request where CI will test building the binary artifacts for all platforms again.</p>
</li>
<li>
<p dir="auto">The output of a build is a JLL package (typically hosted within the <a href="https://github.com/JuliaBinaryWrappers/">JuliaBinaryWrappers</a> GitHub organization) which can be added to packages just like any other Julia package.  The JLL package will export bindings for all products defined within the build recipe.</p>
</li>
</ol>
<p dir="auto">For more information, see the documentation for this package, viewable either directly in markdown within the <a href="docs/src"><code>docs/src</code></a> folder within this repository, or <a href="https://docs.binarybuilder.org" rel="nofollow">online</a>.</p>
<h1 dir="auto"><a id="user-content-philosophy" class="anchor" aria-hidden="true" href="#philosophy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Philosophy</h1>
<p dir="auto">Building binary packages is a pain.  <code>BinaryBuilder</code> follows a philosophy that is similar to that of building <a href="https://julialang.org" rel="nofollow">Julia</a> itself; when you want something done right, you do it yourself.  To that end, <code>BinaryBuilder</code> is designed from the ground up to facilitate the building of packages within an easily reproducible and reliable Linux environment, ensuring that the built libraries and executables are deployable to every platform that Julia itself will run on.  Packages are cross-compiled using a sequence of shell commands, packaged up inside tarballs, and hosted online for all to enjoy.  Package installation is merely downloading, verifying package integrity and extracting that tarball on the user's computer.  No more compiling on user's machines.  No more struggling with system package managers.  No more needing <code>sudo</code> access to install that little mathematical optimization library.</p>
<p dir="auto">All packages are cross compiled. If a package does not support cross compilation, we patch the package or, in extreme cases, rebundle prebuilt executables.</p>
<p dir="auto">The cross-compilation environment that we use is a homegrown Linux environment with many different compilers built for it, including various versions of <code>gcc</code>, <code>clang</code>, <code>gfortran</code>, <code>rustc</code> and <code>go</code>.  You can read more about this in <a href="https://github.com/JuliaPackaging/Yggdrasil/blob/master/RootFS.md">the <code>RootFS.md</code> file</a> within the Yggdrasil repository.</p>

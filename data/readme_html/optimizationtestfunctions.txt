<p dir="auto">This package provides pure-Julia implementations of the test functions enumerated in the references. Functions which were originally intended for only two arguments have been generalized to arbitrary dimensions using a similar approach to that applied to, for example, the Rosenbrock function. Note, however, the generalized form of a what was initially a two dimensional problem may not have readily identifiable global minima.</p>
<p dir="auto">The implementations of the functions themselves are not naive translations, but are written so as to re-use as much computation as possible. This can lead to significant speed-ups, thus, if benchmarks differ between this library and your other favorite library, have a look at the code.</p>
<p dir="auto">Notably, by virtue of being written in pure Julia, these functions should work with most automatic differentiation frameworks which operation on Julia code. Specifically, the functions in this library have been tested against <code>ForwardDiff</code>, <code>ReverseDiff</code>, <code>Enzyme</code> and <code>Zygote</code>.</p>

<p dir="auto">There is an implicit assumption that the single argument passed to a given function conforms to the <code>AbstractArray</code> interface. Furthermore, the array should support linear indexing, and the native indexing style should be <code>IndexLinear</code> Though, some functions may tolerate <code>IndexCartesian</code>, no guarantees are provided as to correctness.</p>
<p dir="auto">It is also worthwhile to note the behavior of the functions with respect to element type: all computations are performed in the nearest element type which supports the elementary functions invoked within the test function body. For example, if the argument is an array with eltype <code>Float32</code>, all computations (including a potentially large sum) will occur in <code>Float32</code> -- i.e., no intermediate promotions will be made to <code>Float64</code> and the result cast back at the end.</p>

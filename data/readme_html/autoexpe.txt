<p dir="auto">Automates repetitive tasks in numerical experiments and the generation of result tables.</p>
<p dir="auto">This package can be useful if you have:</p>
<ul dir="auto">
<li>1 <em>problem</em> (e.g., a <a href="./docs/knapsack.md">multiple knapsack problem</a>);</li>
<li>at least 1 <em>instance</em> of this problem (e.g, <a href="./src/examples/knapsack/data/knapsack1.json">weights and values of the objects that can be added in the knapsacks</a>);</li>
<li>at least 1 <em>resolution method</em> to solve this problem (e.g., <a href="./docs/knapsack.md#heuristics">these two knapsack methods</a>);</li>
<li>(optional) <em>parameters</em> of the problem (e.g., the number of knapsacks <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=m"><img src="https://render.githubusercontent.com/render/math?math=m" style="max-width: 100%;"></a> and the size of the knapsacks <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=K"><img src="https://render.githubusercontent.com/render/math?math=K" style="max-width: 100%;"></a>).</li>
</ul>
<p dir="auto">and if you want to automate:</p>
<ul dir="auto">
<li>the resolution of each instance by each method for each combination of the values of the parameters;</li>
<li>the creation of latex result tables.</li>
</ul>
<p dir="auto">Here is an example of a table generated from the  <a href="./src/examples/knapsack">knapsack example</a>:</p>

</p>

<p dir="auto">To perform an experiment through this package, you must provide:</p>
<ul dir="auto">
<li>the resolution methods;</li>
<li>the path of the instances;</li>
<li>the different values of each parameters;</li>
<li>the format of each result latex table (optional).</li>
</ul>
<p dir="auto">The package does the rest which includes:</p>
<ul dir="auto">
<li>apply all resolution methods on all instances for all combination of values of all parameters;</li>
<li>manage the results: format, reading, writing;</li>
<li>display logs: progress, errors, warnings, ...;</li>
<li>interruption management: regular backups, resumption without unecessary recalculations;</li>
<li>generation of latex result tables during and after the experiment.</li>
</ul>

<p dir="auto">The user describes his experiment in a JSON file which only contains one dictionary. This dictionary must at least contain the two following keys:</p>
<ul dir="auto">
<li><code>"resolutionMethods"</code> which represents the resolution methods; and</li>
<li><code>"instancesPath"</code> which represents the instances.</li>
</ul>

<p dir="auto">The user describes a latex table in a JSON file which contains one array of dictionaries.</p>
<p dir="auto">Example of the structure of such a file:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="[{ TABLE_PARAMETERS },

 { ROW_PARAMETER1 },
 { ROW_PARAMETER2 },
 { RESULT_COLUMN1 },
 { GROUP_OF_COLUMNS }
]
</code></pre></div>
<p dir="auto">Each dictionary in this array either represents:</p>
<ol dir="auto">
<li>the <em>table parameters</em> (e.g., caption, vertical lines, ...); or</li>
<li>a <em>row parameter</em>: an entry saved in the result files which value will vary across the rows of the table (e.g. : <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=m"><img src="https://render.githubusercontent.com/render/math?math=m" style="max-width: 100%;"></a> and <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=K"><img src="https://render.githubusercontent.com/render/math?math=K" style="max-width: 100%;"></a> in the table above);</li>
<li>a <em>result column</em>: describes the content of a column of the table which includes results (e.g. : each of the last five columns in the previous table);</li>
<li>a <em>group of columns</em>: several columns in the table that will appear under a common label (e.g., the groups <code>Objective</code> and <code>Time</code> in the previous table).</li>
</ol>

<h3 dir="auto"><a id="user-content-resolution-methods-format" class="anchor" aria-hidden="true" href="#resolution-methods-format"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Resolution methods format</h3>
<p dir="auto">Currently  resolution methods  must be  implemented in  Julia but  the
package should eventually be able to execute any resolution method from any language. The julia resolution methods must:</p>
<ul dir="auto">
<li>take  a single argument  of type <code>Dictionary{String, Any}</code>.  This input
will both provide a value for each parameter and the path of the current instance (which enables the method to access the data related to the instance);</li>
<li>return a <code>Dictionary{String, Any}</code> which contains all the results to save.</li>
</ul>

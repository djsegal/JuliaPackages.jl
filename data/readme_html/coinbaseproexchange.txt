<p dir="auto">Public endpoints can be accessed without a Coinbase Pro account.</p>
<p dir="auto">In order to use the private endpoints, you will need to register and create an account on <a href="https://www.coinbase.com/" rel="nofollow">Coinbase</a>. Please note that Coinbase Pro account is separate from Coinbase, but they use the same login credentials. More information about the API can be found <a href="https://docs.pro.coinbase.com/#api" rel="nofollow">here.</a></p>
<h2 dir="auto"><a id="user-content-api-key-permissions" class="anchor" aria-hidden="true" href="#api-key-permissions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>API key permissions</h2>
<p dir="auto">You can restrict the functionality of API keys. Before creating the key, you must choose what permissions you would like the key to have. The permissions are:</p>
<ul dir="auto">
<li>View - Allows a key read permissions. This includes all GET endpoints.</li>
<li>Transfer - Allows a key to transfer currency on behalf of an account, including deposits and   withdraws. Enable with caution - API key transfers WILL BYPASS two-factor authentication.</li>
<li>Trade - Allows a key to enter orders, as well as retrieve trade data. This includes POST /orders and several GET endpoints.</li>
</ul>

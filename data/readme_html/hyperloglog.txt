<p>In this example, we'll create a HyperLogLog counter with quality value 8:</p>
<pre><code>using HyperLogLog

counter = HLL(8)

consume!(counter, 1:100)
estimate(counter)

consume!(counter, 101:1000)
estimate(counter)
</code></pre>
<p>Note that the quality value must be an integer between 4 and 16. When selecting a value, keep in mind that the memory costs for using value <code>b</code> scales like <code>2^b</code>.</p>
<p>In this example, we'll use a custom hash function:</p>
<pre><code>using HyperLogLog

counter = HLL(8, x -&gt; uint32(hash(x)))

consume!(counter, 1:100)
estimate(counter)

consume!(counter, 101:1000)
estimate(counter)
</code></pre>
<p>Note that the hash function must return a <code>Uint32</code>. This is done to be consistent with the theory presented in the original HyperLogLog paper. We may allow 64-bit hashes in the future.</p>

<p>A full asset allocation strategy could make use of the following
sources of information:</p>
<ul>
<li>stochastic representation of asset returns and their future
evolution</li>
<li>asset prices: converting fractional weights to integers</li>
<li>investment history
<ul>
<li>turnover minimization</li>
<li>historic portfolio performance and risks taken</li>
<li>taxable gains / losses for tax optimization</li>
</ul>
</li>
<li>cash-flow data
<ul>
<li>initial investment</li>
<li>deposit in / out</li>
</ul>
</li>
<li>client data: tax allowance, tax classes</li>
<li>partial execution
<ul>
<li>live trading with feedback loop</li>
</ul>
</li>
<li>unknown unknowns?</li>
</ul>

information. All strategies are subtypes of abstract type

<p>Optimizes portfolio with respect to single period. Turnover
problematic becomes irrelevant: only asset return model as additional
information required.</p>
<pre><code>optimizeWgts(strat::InitialStrategy, univ::UniverseModel)
</code></pre>
<h2></h2>
<p>Multi-period setting: portfolio rebalancing with turnover
maximization. Specifies initial investment strategy, rebalancing
strategy and turnover filter. Doesn't deal with weight discreteness,
cash-flows or taxes.</p>
<pre><code>optimizeWgts(strat::??, univ::UniverseModel, invHistory::Investments)
</code></pre>

<p>Estimate model for each date in order to get insights into variation
in asset return distributions. If possible, array of estimated models
is simplified to more concise output.</p>
<pre><code>applyMuSigmaModelEstimator
</code></pre>
<p>Output: array of mus, sigmas and correlations, with values only for
those days with enough data for model estimation.</p>

<p dir="auto">A <a href="http://julialang.org" rel="nofollow">Julia</a> client for the <a href="http://neo4j.org" rel="nofollow">Neo4j</a> graph
database.</p>
<p dir="auto">Really easy to use, have a look at <code>test/runtests.jl</code> for the available methods.</p>

<p dir="auto">You can also submit a transaction to the server without committing it. This
will return a result set but will keep the transaction open both on the client
and server:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="results = tx(&quot;MATCH (n) RETURN n&quot;; submit=true)"><pre>results <span class="pl-k">=</span> <span class="pl-c1">tx</span>(<span class="pl-s"><span class="pl-pds">"</span>MATCH (n) RETURN n<span class="pl-pds">"</span></span>; submit<span class="pl-k">=</span><span class="pl-c1">true</span>)</pre></div>
<p dir="auto">Rollbacks are also supported:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="rollback(tx)"><pre><span class="pl-c1">rollback</span>(tx)</pre></div>
<p dir="auto">If the goal is to simply run a MATCH query and get the result in the form of a
<code>DataFrames.DataFrame</code> object, the <code>cypherQuery</code> function can be used.
The <code>cypherQuery</code> implementation performs the query in a single transaction which
automatically opens and closes the transaction:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="results = cypherQuery(c, &quot;MATCH (n) RETURN n.property AS Property&quot;)"><pre>results <span class="pl-k">=</span> <span class="pl-c1">cypherQuery</span>(c, <span class="pl-s"><span class="pl-pds">"</span>MATCH (n) RETURN n.property AS Property<span class="pl-pds">"</span></span>)</pre></div>

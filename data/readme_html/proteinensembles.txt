<p>This <a href="http://julialang.org/" rel="nofollow">Julia</a> package implements the ExProSE algorithm that takes two protein structures and generates an ensemble of protein structures. The ensembles span conformational space and can be used to predict allosteric sites. The method is described in:</p>
<p>Greener JG, Filippis I and Sternberg MJE, Predicting protein dynamics and allostery using multi-protein atomic distance constraints, <em>Structure</em>, 2017, 25, 546-558. <a href="http://www.cell.com/structure/fulltext/S0969-2126(17)30008-4" rel="nofollow">Link to paper</a>.</p>
<p>The package also has some functions that may be of general use for structural bioinformatics including superimposition, RMSD, PCA, interaction finding in structures and distance geometry.</p>

<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p>Julia is required and can be downloaded <a href="http://julialang.org/downloads" rel="nofollow">here</a>. Install ProteinEnsembles.jl by running <code>add ProteinEnsembles</code> from the Julia package REPL, which is entered by pressing <code>]</code>. This will also automatically install a few other required Julia packages. If you want, the tests can be run using <code>test ProteinEnsembles</code>. If you wish to use the auto-parameterisation procedure (see below) you must also have <a href="https://zhanglab.ccmb.med.umich.edu/TM-score" rel="nofollow">TM-score</a> installed.</p>
<h2><a id="user-content-requirements" class="anchor" aria-hidden="true" href="#requirements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Requirements</h2>
<p>To use ProteinEnsembles.jl you will need the following:</p>
<ul>
<li>PDB files of the protein of interest. Two is best, but one may be used (see the paper). They must have polar hydrogens only added; this can be done using tools such as <a href="https://www.cgl.ucsf.edu/chimera" rel="nofollow">Chimera</a> or <a href="https://github.com/harmslab/pdbtools">pdbtools</a>. The chain labelling and residue numbering must be consistent between the files as this is used to find common atoms. Alternative atom locations are discarded. PDB files must also be a single model and not have any inserted residues. HETATM records are discarded by default.</li>
<li>DSSP files corresponding to the PDB files above. These can be obtained using <a href="http://swift.cmbi.ru.nl/gv/dssp" rel="nofollow">dssp</a>.</li>
</ul>

<p>These instructions are tailored towards Mac/Unix. However they could be modified to work on Windows.</p>
<p>Although organised as a Julia package, ProteinEnsembles.jl is primarily designed for use from the command line. The <code>exprose</code> script in the <code>bin</code> directory implements this. For example, to see the command line options run</p>
<div class="highlight highlight-source-shell position-relative" data-snippet-clipboard-copy-content="~/.julia/packages/ProteinEnsembles/xxxxx/bin/exprose -h
"><pre><span class="pl-k">~</span>/.julia/packages/ProteinEnsembles/xxxxx/bin/exprose -h</pre></div>
<p>where <code>xxxxx</code> is the directory of the latest package version. For easy access to the <code>exprose</code> command you might like to add the following line to your profile:</p>
<div class="highlight highlight-source-shell position-relative" data-snippet-clipboard-copy-content="export PATH=$PATH:~/.julia/packages/ProteinEnsembles/xxxxx/bin
"><pre><span class="pl-k">export</span> PATH=<span class="pl-smi">$PATH</span>:<span class="pl-k">~</span>/.julia/packages/ProteinEnsembles/xxxxx/bin</pre></div>
<p>Then, if all input files are in your current directory, run the program as follows:</p>

<h3><a id="user-content-selecting-parameters" class="anchor" aria-hidden="true" href="#selecting-parameters"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Selecting parameters</h3>
<p>The auto-parameterisation procedure can select a more suitable tolerance weighting value (see the paper). <a href="https://zhanglab.ccmb.med.umich.edu/TM-score" rel="nofollow">TM-score</a> must be installed to do this. For example:</p>
<div class="highlight highlight-source-shell position-relative" data-snippet-clipboard-copy-content="exprose-param --i1 input_1.pdb --d1 input_1.dssp --i2 input_2.pdb \
    --d2 input_2.dssp -o exprose_param -t TMscore
"><pre>exprose-param --i1 input_1.pdb --d1 input_1.dssp --i2 input_2.pdb \
    --d2 input_2.dssp -o exprose_param -t TMscore</pre></div>
<p>runs the auto-parameterisation procedure with the <code>-t</code> option specifying the command to run TM-score. The last line of the output gives a suggested tolerance weighting. This value is also written out to <code>suggested.tsv</code>. Use this value in a normal <code>exprose</code> run as above.</p>
<h3><a id="user-content-allosteric-site-prediction" class="anchor" aria-hidden="true" href="#allosteric-site-prediction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Allosteric site prediction</h3>
<p>To predict allosteric sites you should run <a href="https://bmcstructbiol.biomedcentral.com/articles/10.1186/1472-6807-6-19" rel="nofollow">LIGSITEcs</a> on the <em>second</em> input structure (the one you give as <code>--i2</code>). You then need to run the <code>cluster-ligsite</code> script in <code>bin</code> to assign the points to pockets:</p>
<div class="highlight highlight-source-shell position-relative" data-snippet-clipboard-copy-content="cluster-ligsite pocket_r.pdb pocket_all.pdb pocket_points.pdb
"><pre>cluster-ligsite pocket_r.pdb pocket_all.pdb pocket_points.pdb</pre></div>
<p>where <code>pocket_r.pdb</code> and <code>pocket_all.pdb</code> are in the LIGSITEcs output. Then carry out an <code>exprose</code> run with the <code>pocket_points.pdb</code> file (<code>-l</code>) and the number of pockets (e.g. top 4) to perturb at (<code>-m</code>) as parameters:</p>
<div class="highlight highlight-source-shell position-relative" data-snippet-clipboard-copy-content="exprose --i1 input_1.pdb --d1 input_1.dssp --i2 input_2.pdb \
    --d2 input_2.dssp -n 50 -o exprose_out -l pocket_points.pdb -m 4
"><pre>exprose --i1 input_1.pdb --d1 input_1.dssp --i2 input_2.pdb \
    --d2 input_2.dssp -n 50 -o exprose_out -l pocket_points.pdb -m 4</pre></div>
<p>A tolerance weighting from an auto-parameterisation run can also be used here. View the <code>predictions.tsv</code> output file to get the order of allosteric pocket predictions. Note that other pocket prediction software can be used provided you can get the output into the same format as <code>pocket_points.pdb</code>, i.e. pocket cavity points with the pocket number in the residue number column.</p>
<h3><a id="user-content-output" class="anchor" aria-hidden="true" href="#output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Output</h3>
<p>The output directory contains the following:</p>
<ul>
<li><code>input_1.pdb</code> and <code>input_2.pdb</code>: atoms used from the input structures are written back out and superimposed.</li>
<li><code>pdbs</code>: generated structures in PDB format. Superimposed to <code>input_1.pdb</code> and <code>input_2.pdb</code>.</li>
<li><code>pcs</code>: projections onto the principal components (PCs) from the principal component analysis of the generated structures. Contains files for generated (<code>pcs.tsv</code>) and input structures (<code>pcs_input_1.tsv</code> and <code>pcs_input_2.tsv</code>) - line n corresponds to structure n and column c corresponds to PC c. Has graphs of these for the first few PCs (<code>pc_x_y.png</code>). Also includes a list of PCs ordered by decreasing distance between the input structures (<code>pcs_input_dist.tsv</code>) and the percentage variation explained by each PC (<code>evals_spread.tsv</code>).</li>
<li><code>pymol</code>: <a href="https://www.pymol.org/" rel="nofollow">PyMol</a> scripts to view PCs on <code>input_1.pdb</code>, e.g. run <code>pymol input_1.pdb pymol/view_pc_1.pml</code>.</li>
<li><code>rmsds_input_1.tsv</code> and <code>rmsds_input_2.tsv</code>: RMSDs of generated structures to the input structures. Line n corresponds to structure n.</li>
<li><code>rmsfs.tsv</code> and <code>rmsfs.png</code>: RMSFs of each residue over the ensemble of generated structures, and a plot of this. Line n corresponds to residue index n.</li>
<li><code>spe_scores.tsv</code>: SPE error scores of generated structures (see paper). Line n corresponds to structure n.</li>
</ul>
<p>For allosteric site prediction there will be <code>pdbs_mod_n</code> and <code>mod_n</code> containing similar information for each perturbed ensemble, as well as the ratio of RMSF values to the unperturbed ensemble (<code>rmsfs_ratio.tsv</code>). There will also be the order of allosteric predictions (<code>predictions.tsv</code>) and the size of the perturbation on modulating each site (<code>perturbations.tsv</code>), which is the RMSD between the centroid structure of the perturbed and unperturbed ensembles.</p>
<p>The default plot colours are blue for generated structures, red for input structure 1, green for input structure 2 and orange for perturbed ensemble structures.</p>

<p>The results from the paper can be generated using the instructions in <code>paper_results</code>. Information on the ensemble and allosteric datasets is in <code>datasets</code>.</p>

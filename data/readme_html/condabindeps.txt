<p dir="auto">This package, which builds on the <a href="https://github.com/JuliaPy/Conda.jl">Conda.jl package</a> allows one to use <a href="http://conda.pydata.org/" rel="nofollow">conda</a> as a <a href="https://github.com/JuliaPackaging/BinDeps.jl">BinDeps</a> binary
provider for Julia. While other binary providers like
<a href="https://github.com/JuliaLang/Homebrew.jl">Homebrew.jl</a>,
<a href="https://en.wikipedia.org/wiki/Advanced_Packaging_Tool#apt-get" rel="nofollow">AptGet</a> or
<a href="https://github.com/JuliaLang/WinRPM.jl">WinRPM.jl</a> are platform-specific,
CondaBinDeps.jl is a cross-platform alternative. It can also be used without
administrator rights, in contrast to the current Linux-based providers.</p>
<p dir="auto">As such, <code>Conda.jl</code> primary audience is Julia packages developers who have a dependency on  some native library.</p>
<p dir="auto"><code>conda</code> is a package manager which started as the binary package manager for the
Anaconda Python distribution, but it also provides arbitrary packages. Instead
of the full Anaconda distribution, <code>Conda.jl</code> uses the miniconda Python
environment, which only includes <code>conda</code> and its dependencies.</p>
<p dir="auto"><code>CondaBinDeps.jl</code> is <strong>NOT</strong> an alternative Julia package manager, nor a way to manage
Python installations. It will not use any pre-existing Anaconda or Python
installation on  your machine.</p>

<p dir="auto">You can install this package by running <code>Pkg.add("CondaBinDeps")</code> at the Julia prompt.  See the <a href="https://github.com/JuliaPy/Conda.jl">Conda.jl package</a> for information on setting
up <code>conda</code> environments, etcetera.</p>

<p dir="auto">CondaBinDeps.jl can be used as a <code>Provider</code> for
<a href="https://github.com/JuliaPackaging/BinDeps.jl">BinDeps</a> with the <code>Conda.Manager</code>
type. You first need to write a <a href="http://conda.pydata.org/docs/building/recipe.html" rel="nofollow">conda
recipe</a>, and upload the
corresponding build to <a href="https://binstar.org/" rel="nofollow">binstar</a>. Then, add CondaBinDeps in your
<code>REQUIRE</code> file, and add the following to your <code>deps/build.jl</code> file:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using BinDeps
@BinDeps.setup
netcdf = library_dependency(&quot;netcdf&quot;, aliases = [&quot;libnetcdf&quot; &quot;libnetcdf4&quot;])

...

using CondaBinDeps
provides(CondaBinDeps.Manager, &quot;libnetcdf&quot;, netcdf)"><pre><span class="pl-k">using</span> BinDeps
<span class="pl-c1">@BinDeps</span><span class="pl-k">.</span>setup

<span class="pl-k">...</span>

<p dir="auto">If your dependency is available in another channel than the default one, you
should register that channel.</p>

<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="provides(CondaBinDeps.Manager, &quot;libnetcdf&quot;, netcdf, os=:Linux)"><pre><span class="pl-c1">provides</span>(CondaBinDeps<span class="pl-k">.</span>Manager, <span class="pl-s"><span class="pl-pds">"</span>libnetcdf<span class="pl-pds">"</span></span>, netcdf, os<span class="pl-k">=</span><span class="pl-c1">:Linux</span>)</pre></div>

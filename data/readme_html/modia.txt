<p><a href="https://modiasim.github.io/" rel="nofollow">Modia</a> is a domain specific extension of <a href="http://julialang.org/" title="Julia" rel="nofollow">Julia</a> for <strong>modeling and simulation of physical systems</strong>.</p>
<p>Papers and presentations about Modia:</p>
<ul>
<li><a href="http://link.springer.com/chapter/10.1007%2F978-3-319-47169-3_15" rel="nofollow">Overview about Modia</a> (<a href="http://www.isola-conference.org/isola2016/" rel="nofollow">ISoLA conference Oct. 2016</a>; slides in <a href="https://modiasim.github.io/Modia.jl/slides/Systems-Modeling-and-Programming-Slides.pdf" rel="nofollow">pdf</a> format).</li>
<li><a href="https://www.modelica.org/events/modelica2017/proceedings/html/submissions/ecp17132693_ElmqvistHenningssonOtter.pdf" rel="nofollow">Overview and new features in Modia</a> (<a href="https://www.modelica.org/events/modelica2017/proceedings/html/index.html" rel="nofollow">12th International Modelica Conference, May 2017</a>; slides in <a href="https://modiasim.github.io/Modia.jl/slides/Innovations-for-Future-Modelica.pdf" rel="nofollow">pdf</a> format).</li>
<li><a href="https://www.modelica.org/events/modelica2017/proceedings/html/submissions/ecp17132565_OtterElmqvist.pdf" rel="nofollow">New algorithms in Modia</a> (<a href="https://www.modelica.org/events/modelica2017/proceedings/html/index.html" rel="nofollow">12th International Modelica Conference, May 2017</a>; slides in <a href="https://modiasim.github.io/Modia.jl/slides/Modelica2017-DAAE-Transformation.pptx" rel="nofollow">pptx</a> and <a href="https://modiasim.github.io/Modia.jl/slides/Modelica2017-DAAE-Transformation.pdf" rel="nofollow">pdf</a> format).</li>

<p>Modia is designed to model and simulate physical systems (electrical, mechanical, thermo-dynamical, etc.) described by differential and algebraic equations. A user defines a model on a high level with model components (like a mechanical body, an electrical resistance, or a pipe) that are physically connected together. A model component is constructed by "expression = expression" equations. The defined model is symbolically processed (for example, equations might be analytically differentiated), JIT compiled and simulated with <a href="http://computation.llnl.gov/projects/sundials/ida" rel="nofollow">Sundials IDA solver</a> with the KLU sparse matrix package. By this approach it's possible and convenient to build models with hundred thousands of equations describing the dynamics of a car, an airplane, a power plant, etc. and simulate them. The authors used previous experience from the design of the modeling language <a href="https://www.modelica.org/" rel="nofollow">Modelica</a>. Modia will also be used to design and evaluate features for future Modelica versions.</p>
<p>Component models are defined by <code>@model</code> macros. Such models contain definition of variables with various attributes such as start values, min, max, SI unit, etc. An <code>@equations</code> macro is used to define the equations of such a component. Coupling between components is expressed using a connect statement involving groups of variables. The semantics is either to constrain connected variables to be equal or to constrain a set of variables to sum to zero, for example to model Kirchhoff's current law.</p>

<p><strong>Modia</strong> is registered in METADATA.jl and can be installed with Pkg.add.
Version (0.2.3) is the last one with support for Julia &gt;= 0.6. Trunk and later versions support Julia &gt;=1.0.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c"><span class="pl-c">#</span> Julia &gt;= 0.6:</span>
julia<span class="pl-k">&gt;</span> Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>Modia<span class="pl-pds">"</span></span>)
julia<span class="pl-k">&gt;</span> Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>ModiaMath<span class="pl-pds">"</span></span>)

<span class="pl-c"><span class="pl-c">#</span> Julia &gt;= 0.7:</span>
julia<span class="pl-k">&gt;</span> ]add Modia ModiaMath</pre></div>
<p>Modia uses <a href="https://github.com/JuliaPy/PyPlot.jl">PyPlot</a> for plotting.
If <code>PyPlot</code> is not available in your current Julia environment
an information message is printed and all <code>plot(..)</code> calls are ignored.</p>
<p>In order that plot windows are displayed, you need to add <code>PyPlot</code> to your current environment
via <code>]add PyPlot</code>. Often this automatic installation fails and it is recommended to follow
the instructions
<a href="https://github.com/ModiaSim/ModiaMath.jl/wiki/Installing-PyPlot-in-a-robust-way">Installing PyPlot in a robust way</a>.</p>

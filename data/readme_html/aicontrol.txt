<p><a href="https://travis-ci.org/suinleelab/AIControl.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/48f46b66d99f6a12725231cf645e12adb98b9a4d/68747470733a2f2f7472617669732d63692e6f72672f7375696e6c65656c61622f4149436f6e74726f6c2e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/suinleelab/AIControl.jl.svg?branch=master" style="max-width:100%;"></a></p>
<p>AIControl makes ChIP-seq assays <strong>easier</strong>, <strong>cheaper</strong>, and <strong>more accurate</strong> by imputing background data from a massive amount of publicly available control data.</p>
<p>Here is an overview of the AIControl framework from our paper.
<a target="_blank" rel="noopener noreferrer" href="images/concept.png"><img src="images/concept.png" alt="alt text" style="max-width:100%;"></a></p>
<p><em>Figure 1: (a) Comparison of AIControl to other peak calling algorithms. (left) AIControl
learns appropriate combinations of publicly available control ChIP-seq datasets to impute background
noise distributions at a fine scale. (right) Other peak calling algorithms use only one
control dataset, so they must use a broader region (typically within 5,000-10,000 bps) to estimate
background distributions. (bottom) The learned fine scale Poisson (background) distributions are
then used to identify binding activities across the genome. (b) An overview of the AIControl
approach. A single control dataset may not capture all sources of background noise. AIControl
more rigorously removes background ChIP-seq noise by using a large number of publicly available
control ChIP-seq datasets</em></p>

<p>AIControl can be used on any <strong>Linux</strong> or <strong>macOS</strong> machine. While we tested and validated that AIControl works on <strong>Windows</strong> machines, we believe that it is easier for you to set up the AIControl pipeline on the Unix based systems.</p>
<p>AIControl expects a sorted <code>.bam</code> file as an input and outputs a <code>.narrowpeak</code> file. Typically, for a brand new ChIP-seq experiment, you start with a <code>.fastq</code> file, and you will need some external softwares to convert the <code>.fastq</code> file to a sorted <code>.bam</code> file. Thus, the whole AIControl pipeline needs the following sets of programs and packages installed on your local machine. We will explain how to install them in sections below.</p>
<ul>
<li><code>Julia</code> (v1.0.3): for running Julia scripts</li>
<li><code>bowtie2</code> (2.3.4.3): for aligning a <code>.fastq</code> file to the hg38 genome</li>
<li><code>samtools</code> (1.9): for sorting an aligned bam file</li>
<li><code>bedtools</code> (v2.27.1): for converting a bam file back to a fastq file (OPTIONAL for Step 3.1)</li>
</ul>

<p>AIControl uses a massive amount of public control data for ChIP-seq (roughly 450 chip-seq runs). We have done our best to compress them so that you only need to download about <strong>4.6GB</strong>. These files require approximately <strong>13GB</strong> of free disk space to decompress. The following terminal commands will download and decompress the compressed control data.</p>
<pre><code>wget https://dada.cs.washington.edu/aicontrol/forward.data100.nodup.tar.bz2
tar xvjf forward.data100.nodup.tar.bz2
wget https://dada.cs.washington.edu/aicontrol/reverse.data100.nodup.tar.bz2
tar xvjf reverse.data100.nodup.tar.bz2
</code></pre>
<p>You can also obtain the control files from <a href="https://dada.cs.washington.edu/aicontrol/" rel="nofollow">our data repository</a> or <a href="https://drive.google.com/open?id=1Xh6Fjah1LoRMmbaJA7_FzxYcbqmpNUPZ" rel="nofollow">Google Drive</a>.<br>
mac OS users can use <code>curl -O output_file_name</code> instead of <code>wget</code>.</p>

<h3><a id="user-content-step-1-map-your-fastq-file-from-chip-seq-to-the-hg38-assembly-from-the-ucsc-database" class="anchor" aria-hidden="true" href="#step-1-map-your-fastq-file-from-chip-seq-to-the-hg38-assembly-from-the-ucsc-database"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Step 1: Map your FASTQ file from ChIP-seq to the <code>hg38</code> assembly from the UCSC database.</h3>
<p>The following terminal commands will a) download and untar the reference database file for <code>bowtie2</code>, b) run <code>bowtie2</code> to map a <code>.fastq</code> file to the UCSC hg38 genome, which is available at <a href="http://hgdownload.soe.ucsc.edu/goldenPath/hg38/bigZips/hg38.fa.gz" rel="nofollow">the UCSC repository</a>, and c) sort the output.</p>
<pre><code>wget https://dada.cs.washington.edu/aicontrol/bowtie2ref.tar.bz2
tar xvjf bowtie2ref.tar.bz2
bowtie2 -x bowtie2ref/hg38 -q -p 10 -U example.fastq | samtools view -bS | samtools sort -o example.sorted.bam
</code></pre>
<p>If you are running this on a cluster machine or local laptop, you can remove the <code>-p 10</code> flag in order to let <code>bowtie2</code> use a single core rather than 10 cores.</p>
<p>Unlike other peak callers, the core idea of AIControl is to leverage all available control datasets. This requires all data (your target and public control datasets) to be mapped to the exact same reference genome. Our control datasets are currently mapped to the hg38 assembly from <a href="http://hgdownload.soe.ucsc.edu/goldenPath/hg38/bigZips/hg38.fa.gz" rel="nofollow">the UCSC repository</a>. <strong>So please make sure that your data is also mapped to the same assembly</strong>. Otherwise, our pipeline will report an error.</p>

<p>The terminal command below will run AIControl.</p>
<pre><code>julia aicontrolScript.jl example.sorted.bam --ctrlfolder=. --name=test
</code></pre>
<p>If you are running this on a cluster machine or local laptop, you can use the <code>--disableParallel</code> flag in order to prevent AIControl from using 2 cores. Do <code>julia aicontrolScript.jl --help</code> or <code>-h</code> for help.</p>
<p>The following files will be written to a <code>./test</code> folder.</p>
<ul>
<li><code>*.narrowPeak</code>: a resulting peak file in <a href="http://genome.ucsc.edu/FAQ/FAQformat.html#format12" rel="nofollow">.narrowPeak</a> format.</li>
<li><code>*.jld2</code>: a file that stores intermediate results.</li>
<li><code>*.fbin100</code> &amp; <code>*.rbin100</code>: binned forward/reverse signals of your target ChIP experiment.</li>
</ul>
<p>We support the following flags.</p>
<ul>
<li><code>--ctrlfolder=[path]</code>: path to a control folder [default:.]</li>
<li><code>--name=[string]</code>: prefix for output files [default:bamfile_prefix]</li>
<li><code>--p=[float]</code>: pvalue threshold [default: 0.03 (or -log10(0.03)=1.5)]</li>
<li><code>--disableParallel</code>: a flag to disable parallel processing [default:false]</li>
<li><code>--dup</code>: a flag to use duplicate reads [default:false].</li>
<li><code>--reduced</code>: a flag to use subsampled control datasets [default:false]</li>
<li><code>--fused</code>: a flag to fuse consecutive peaks [default:false]</li>
</ul>
<p>If you would like to use the <code>--dup</code> or <code>--reduced</code> options, please download appropriate versions of compressed control data indicated with <code>.dup</code> or <code>.reduced</code>.</p>

<p>You are likely here because the AIControl script raised an error that looks like the screenshot below.</p>
<p><a target="_blank" rel="noopener noreferrer" href="images/error3_1.png"><img src="images/error3_1.png" alt="alt text" width="500" style="max-width:100%;"></a></p>
<p>The error is most likely caused by a mismatch of genome assembly that your dataset and control datasets are mapped to. Our control datasets are mapped to the hg38 from <a href="http://hgdownload.soe.ucsc.edu/goldenPath/hg38/bigZips/hg38.fa.gz" rel="nofollow">the UCSC repository</a>. On the other hand, your bam file is probably mapped to a slightly different version of the hg38 assembly or different ordering of chromosomes (a.k.a. non-lexicographic). For instance, if you download a <code>.bam</code> file directly from the ENCODE website, it is mapped to a slightly different version of hg38. A recommended way of resolving this issue is to extract a <code>.fastq</code> file from your <code>.bam</code> file, go back to step 1, and remap it with <code>bowtie2</code> using the UCSC hg38 assembly. <code>bedtools</code> provides a way to generate a <code>.fastq</code> file from your <code>.bam</code> file.</p>
<pre><code>bedtools bamtofastq  -i example.bam -fq example.fastq
</code></pre>
<p>We will regularly update the control data when a new major version of the genome becomes available; however, covering for all versions with small changes to the existing version is not realistic.</p>

<p>If you have trouble running AICotrol, please make sure that:</p>
<ul>
<li>You are using Julia 1.0. and above.</li>

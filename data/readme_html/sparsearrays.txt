<p dir="auto">This package ships as part of the Julia stdlib.</p>
<p dir="auto">SparseArrays.jl provides functionality for working with sparse arrays in Julia.</p>
<h2 dir="auto"><a id="user-content-using-development-versions-of-this-package" class="anchor" aria-hidden="true" href="#using-development-versions-of-this-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Using development versions of this package</h2>
<p dir="auto">To use a newer version of this package, you need to build Julia from scratch. The build process is the same as any other build except that you need to change the commit used in <code>stdlib/SparseArrays.version</code>.</p>
<p dir="auto">It's also possible to load a development version of the package using <a href="https://github.com/JuliaLang/Pkg.jl#using-the-development-version-of-pkgjl">the trick used in the Section named "Using the development version of Pkg.jl" in the <code>Pkg.jl</code> repo</a>, but the capabilities are limited as all other packages will depend on the stdlib version of the package and will not work with the modified package.</p>
<p dir="auto">The main environment may become inconsistent so you might need to run <code>Pkg.instantiate()</code> and/or <code>Pkg.resolve()</code> in the main or project environments if Julia complains about missing <code>Serialization.jl</code> in this package's dependencies.</p>
<p dir="auto">For older (1.8 and before) <code>SuiteSparse.jl</code> needs to be bumped too.</p>
<h2 dir="auto"><a id="user-content-updating-suitesparse" class="anchor" aria-hidden="true" href="#updating-suitesparse"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Updating SuiteSparse</h2>
<p dir="auto">In order to upgrade SparseArrays.jl to use a new release of SuiteSparse, the following steps are necessary:</p>
<ol dir="auto">
<li>Update SuiteSparse in Yggdrasil</li>
<li>Update the SuiteSparse wrappers in SparseArrays.jl/gen and generate the new wrappers</li>
<li>Run BumpStdlibs to update the SparseArrays.jl version in julia master</li>
<li>Update the relevant stdlibs in Julia to pull in the new releases</li>
</ol>

<h2><a id="user-content-description" class="anchor" aria-hidden="true" href="#description"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Description</h2>
<p><code>CMDimCircuits.jl</code> provides an <em>environment</em> (a set of tools) to efficiently
post-process and analyze results from circuit measurement &amp; simulation:</p>
<ul>
<li><code>[MDDatasets.jl][MDDatasetsJL]</code>: Easily handle "parametric analysis" data using a simple, multi-dimensional container, and provide <code>struct DataF1</code> to represent <code>{x,y}</code> vectors as a continuous function of 1 argument: <code>y=f(x)</code>.</li>
<li><code>[CMDimData.jl][CMDimDataJL]/EasyPlot</code>: Easily plot "parametric analysis" data using a backend of your choosing.</li>
<li><code>[CMDimData.jl][CMDimDataJL]/EasyData</code>: Easily <em>save</em> "parametric analysis" data to a single .hdf5 file.</li>
<li>Various EDA file readers/writers.</li>
</ul>
<p>More specifically, the goal is to provide an analysis framework leading to
minimal code, written in a <em>natural</em>, and <em>readable</em> fashion.  The following
describes a typical workflow enabled by this package:</p>
<ol>
<li>Read in complex multi-dimensional input data from measurement/simulation results.</li>
<li>Treat transient datasets as continuous-time (automatically interpolate when time steps are mismatched).</li>
<li>Perform the same operation on all parametric sweeps (usually) without having to write explicit loops.</li>
<li>Quickly generate plots that shed light on the studied phenomenon.
<ol>
<li>Select from a multitude of plotting backends (possibly at a later date).</li>
</ol>
</li>
<li>Save both data &amp; plot configuration to a single .hdf5 file.
<ol>
<li>Reload plot &amp; data for further investigation at a later time.</li>
</ol>
</li>
</ol>
<p><a name="user-content-highlights"></a></p>
<h3><a id="user-content-featureshighlights" class="anchor" aria-hidden="true" href="#featureshighlights"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Features/Highlights</h3>
<ul>
<li>Support for popular EDA file formats: (.tr0, .psf, .sNp).</li>
<li>Plot the results of multi-dimensional parametric analyses with a single command.
<ul>
<li>Support for multiple plotting backends (<a href="https://github.com/ma-laforge/CMDimData.jl">see CMDimData.jl for more details</a>).</li>
<li>Generate eye diagrams (even for backends without native support).</li>
<li>Generate Smith Plots (<a href="https://github.com/ma-laforge/InspectDR.jl">InspectDR backend only</a>).</li>
<li>Read/write plots to .hdf5 files.</li>
</ul>
</li>
</ul>

<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p><code>CMDimCircuits.jl</code> is registered with Julia's <strong>General</strong> registry.
It can be installed using Julia's built-in package manager:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="julia&gt; ]
pkg&gt; add CMDimCircuits
pkg&gt; add CMDimData
pkg&gt; add MDDatasets
"><pre>julia<span class="pl-k">&gt;</span> ]
pkg<span class="pl-k">&gt;</span> add CMDimCircuits
pkg<span class="pl-k">&gt;</span> add CMDimData
pkg<span class="pl-k">&gt;</span> add MDDatasets</pre></div>
<p>Note that <code>CMDimData.jl</code> &amp; <code>MDDatasets.jl</code> will automatically be installed alongside <code>CMDimCircuits.jl</code>.  However, <code>add</code>-ing them explicitly gives code from the active project/environment direct access to their corresponding features.</p>
<p>Moreover, it is highly suggested to install <code>InspectDR.jl</code>. It is the most tested integration for plotting at the moment:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="julia&gt; ]
pkg&gt; add InspectDR
"><pre>julia<span class="pl-k">&gt;</span> ]
pkg<span class="pl-k">&gt;</span> add InspectDR</pre></div>

<h2><a id="user-content-activating-the-c-data-environment" class="anchor" aria-hidden="true" href="#activating-the-c-data-environment"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Activating the "C-Data" environment</h2>
<p>The "C-Data" environment provides a collection of tools to generate/process
parametric analysis data &amp; better visualize (see) mathematical relationships
buried within.  To activate C-Data, you must call the <code>@using_CData</code> macro:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using CMDimCircuits
CMDimCircuits.@using_CData
"><pre><span class="pl-k">using</span> CMDimCircuits
CMDimCircuits<span class="pl-k">.</span><span class="pl-c1">@using_CData</span></pre></div>
<p>Note that <code>CMDimCircuits.@using_CData</code> imports the following modules to your scope:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using MDDatasets
using CMDimData
using CMDimData.EasyPlot
using CMDimCircuits.CircuitAnalysis
using CMDimCircuits.NetwAnalysis
using CMDimCircuits.SignalProcessing
using CMDimCircuits.EDAData
"><pre><span class="pl-k">using</span> MDDatasets
<span class="pl-k">using</span> CMDimData
<span class="pl-k">using</span> CMDimData<span class="pl-k">.</span>EasyPlot
<span class="pl-k">using</span> CMDimCircuits<span class="pl-k">.</span>CircuitAnalysis
<span class="pl-k">using</span> CMDimCircuits<span class="pl-k">.</span>NetwAnalysis
<span class="pl-k">using</span> CMDimCircuits<span class="pl-k">.</span>SignalProcessing
<span class="pl-k">using</span> CMDimCircuits<span class="pl-k">.</span>EDAData</pre></div>
<p>Note that no plotting backend was imported. You must therefore "import" your
preferred plotting library using the procedure
<a href="https://github.com/ma-laforge/CMDimData.jl/blob/master/doc/EasyPlot_backends.md#ImportingBackends">described here</a>.</p>

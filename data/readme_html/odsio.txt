<p>Open Document Format Spreadsheet (ODS) I/O for Julia using the python ezodf module.</p>
<p>It allows to export (import) data from (to) Julia to (from) LibreOffice, OpenOffice and any other spreadsheet software that implements the OpenDocument specifications.</p>

<h4><a id="user-content-arguments" class="anchor" aria-hidden="true" href="#arguments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Arguments</h4>
<ul>
<li><code>sheetsNames=[]</code>: the list of sheet names from which to import data.</li>
<li><code>sheetsPos=[]</code>: the list of sheet positions (starting from 1) from which to import data.</li>
<li><code>ranges=[]</code>: a list of pair of touples defining the ranges in each sheet from which to import data, in the format ((tlr,tlc),(brr,brc))</li>
<li><code>innerType="Matrix"</code>: the type of the inner container returned. Either "Matrix", "Dict" or "DataFrame"</li>
</ul>
<h4><a id="user-content-notes" class="anchor" aria-hidden="true" href="#notes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Notes</h4>
<ul>
<li>sheetsNames and sheetsPos can not be given together</li>
<li>ranges is defined using integer positions for both rows and columns</li>
<li>individual dictionaries or dataframes are keyed by the values of the cells in the first row specified in the range, or first row if <code>range</code> is not given</li>
<li>innerType="Matrix", differently from innerType="Dict", preserves original column order, it is faster and require less memory</li>
<li>using innerType="DataFrame" also preserves original column order and try to auto-convert column types (working for Int64, Float64, String, in that order)</li>
</ul>

<h4><a id="user-content-arguments-2" class="anchor" aria-hidden="true" href="#arguments-2"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Arguments</h4>
<ul>
<li><code>filename</code>:    an existing ods file or the one to create.</li>
<li><code>data=Dict()</code>: a dictionary of locations in the files where to export the data =&gt; the actual data (see notes).</li>
</ul>
<h4><a id="user-content-notes-2" class="anchor" aria-hidden="true" href="#notes-2"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Notes:</h4>
<ul>
<li>The locations where to save the data (the keys in the dictionary) are a tuple of tree elements:
The first one is the sheet name or sheet position, the other two are the index of row and column of the top
left corner where to export the data.
If using sheet positions, these must be within current file sheets boundaries. If you want to create new sheets,
use names.</li>
<li>The actual data exported are either a Matrix (2D Array), a DataFrame or an OrderedDict. In case of DataFrame or
OrderedDict the headers ARE exported, so if you don't want them, first convert the DataFrame (or Dictionary)
to a Matrix. In case of OrderedDict, the inner data must all have the same length.</li>
<li>Some spreadsheet software may not automatically recalculate cells that depends on exported cells (e.g. we are exporting
some data o cell <code>A1</code> and cells <code>A2</code> depends on <code>A2</code>, the content of cell <code>A2</code> may not be updated after the export).
In such case most spreadsheet software have a command to force recalculation of cells (e.g. in LibreOffice/OpenOffice
use <code>CTRL+Shift+F9</code>)</li>
</ul>

<p>This package requires:</p>
<ul>
<li>the <a href="https://github.com/JuliaPy/PyCall.jl">PyCall</a> package to call Python</li>
<li>a working local installation of Python with the python <a href="https://github.com/T0ha/ezodf">ezodf</a> module already installed (if the <code>ezodf</code> module is not available and you have no access to the local python installation, you can use PyCall to try to install the <code>ezodf</code> using pip.. see <a href="https://gist.github.com/Luthaf/368a23981c8ec095c3eb">here</a>)</li>
<li>the <a href="https://github.com/JuliaStats/DataFrames.jl">DataFrames</a> package in order to return DataFrames.</li>
</ul>
<h2><a id="user-content-known-limitations" class="anchor" aria-hidden="true" href="#known-limitations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Known limitations</h2>
<ul>
<li>In reading, as the data is saved in a dictionary, the order of the columns is not maintained.</li>
<li>It is relatively slow with very large data.</li>
<li>If the data has many columns, the conversion from Dictionary to DataFrame made in the ods2dfs and ods2df functions may not work. In that case call the ods2dics or ods2dic functions and perform the conversion manually choosing the columns you need.</li>
</ul>

<p dir="auto">Related packages includes</p>
<ul dir="auto">
<li><a href="https://github.com/scheinerman/SimpleTropical.jl">SimpleTropical.jl</a></li>
<li><a href="https://github.com/saschatimme/TropicalSemiring.jl">TropicalSemiring.jl</a></li>
</ul>
<p dir="auto">Tropical numbers in these packages contains an extra field <code>isinf</code>. Which is not nessesary because we have <code>Inf</code> and <code>-Inf</code> for floating point numbers already, we can just use them directly. It is memory more efficient and computational cheap.</p>
<p dir="auto">Most importantly, we are going to release a BLAS package for tropical numbers, which is two orders faster than naive Julia loops. If a tropical number is defined as a composite data structure, it is hard to utilize SIMD.</p>

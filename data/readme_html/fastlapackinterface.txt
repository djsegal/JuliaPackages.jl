<p dir="auto"><code>FastLapackInterface</code> separates workspace allocation and actual
running for some Lapack algorithms:</p>
<ul dir="auto">
<li>LU factorization</li>
<li>QR factorization</li>
<li>Schur factorization</li>
<li>Eigen decomposition</li>
<li>Bunch-Kaufman decomposition</li>
<li>Cholesky decomposition</li>
<li>LSE decomposition</li>
</ul>
<p dir="auto">The API follows the Base julia <code>LAPACK</code> definitions and should thus be very transparent.
There is also a unified interface which returns the right <code>Workspace</code> to be used with a given
<code>LAPACK</code> function.
e.g.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="A = [1.2 2.3
     6.2 3.3]
ws = Workspace(LAPACK.getrf!, A)
decompose!(ws, A)"><pre>A <span class="pl-k">=</span> [<span class="pl-c1">1.2</span> <span class="pl-c1">2.3</span>
     <span class="pl-c1">6.2</span> <span class="pl-c1">3.3</span>]
ws <span class="pl-k">=</span> <span class="pl-c1">Workspace</span>(LAPACK<span class="pl-k">.</span>getrf!, A)
<span class="pl-c1">decompose!</span>(ws, A)</pre></div>
<p dir="auto">A similar API exists for the above decompositions. For more information and examples please see the <a href="https://dynarejulia.github.io/FastLapackInterface.jl/dev/" rel="nofollow">documentation</a>.</p>

<p>This is intended for experiments with graphical layout management in <a href="http://julialang.org" title="Julia" rel="nofollow">Julia</a>. It's not clear whether this has a long-term future as a standalone repository, or whether it will be integrated into other packages. For now, it's separate just to avoid breaking things in those other packages.</p>
<h2><a id="user-content-rough-sketch-of-vision" class="anchor" aria-hidden="true" href="#rough-sketch-of-vision"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Rough sketch of "vision"</h2>
<ol>
<li>Implement the "flexible core," probably based on a linear programming model such as the <a href="https://www.cs.auckland.ac.nz/courses/compsci705s1c/archive/2008/lectures/GW-HCI-02.pdf" rel="nofollow">Auckland Layout Model</a>. (<a href="https://www.cs.auckland.ac.nz/~lutteroth/publications/LutterothStrandhWeber2008-AucklandLayoutModel.pdf" rel="nofollow">Paper</a>) This will support figures as vector-graphics files (svg, pdf, etc), and impose very few built-in constraints on layout. We may also want to support other optimizers besides linear programming. If they can't run quickly, they may not be suitable for real-time window resizing, but they could be used to create figures for publication.</li>
<li>Find out whether the flexible core can be directly used for window resize events via callbacks (main targets might be Tk and HTML5/CSS). If not, then we probably need a raw representation that directly exposes the toolkit's manager, and then for SVG/PDF/etc we may need to mimic the toolkit's behavior in the context of the flexible core (yuck).</li>
<li>Figure out how to integrate this into <a href="https://github.com/dcjones/Compose.jl">Compose</a> and/or <a href="https://github.com/nolta/Winston.jl">Winston</a>.</li>
</ol>

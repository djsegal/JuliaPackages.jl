<p dir="auto"><a href="https://pkg.juliahub.com/ui/Packages/SciPy/g8Zlq/" rel="nofollow">SciPy.jl</a> is a Julia interface for <a href="https://scipy.org/" rel="nofollow">SciPy</a> using <a href="https://github.com/JuliaPy/PyCall.jl">PyCall.jl</a>.</p>
<p dir="auto">You can use many useful scientific functions of SciPy from Julia codes.</p>

<p dir="auto">Read an error message you get carefully. Since SciPy.jl depends on <a href="https://github.com/JuliaPy/PyCall.jl">PyCall.jl</a> to call Python functionalities from Julia, we also recommend to read <a href="https://github.com/JuliaPy/PyCall.jl#specifying-the-python-version">Specifying the Python version
section of PyCall.jl</a>.</p>
<p dir="auto">See also this <a href="https://github.com/AtsushiSakai/SciPy.jl/issues/48#issuecomment-1263517899" data-hovercard-type="issue" data-hovercard-url="/AtsushiSakai/SciPy.jl/issues/48/hovercard">instruction_1</a> and <a href="https://github.com/AtsushiSakai/SciPy.jl/issues/48#issuecomment-1263519351" data-hovercard-type="issue" data-hovercard-url="/AtsushiSakai/SciPy.jl/issues/48/hovercard">instruction_2</a> in <a href="https://github.com/AtsushiSakai/SciPy.jl/issues/48" data-hovercard-type="issue" data-hovercard-url="/AtsushiSakai/SciPy.jl/issues/48/hovercard">this issue</a></p>

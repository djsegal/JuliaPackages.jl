<p>This package implements several methods of generating or loading geometries associated with a <a href="https://github.com/tkoolen/RigidBodyDynamics.jl">RigidBodyDynamics.jl</a> <code>Mechanism</code> in Julia. It is currently used by <a href="https://github.com/JuliaRobotics/MeshCatMechanisms.jl">MeshCatMechanisms.jl</a> but can also be used independently.</p>
<h1><a id="user-content-interface" class="anchor" aria-hidden="true" href="#interface"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Interface</h1>
<p>This package exports one primary method:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">visual_elements</span>(mechanism<span class="pl-k">::</span><span class="pl-c1">Mechanism</span>, source<span class="pl-k">::</span><span class="pl-c1">AbstractGeometrySource</span>)</pre></div>
<p><code>visual_elements</code> returns a vector of <code>VisualElement</code> structs, each of which contains:</p>
<ul>
<li><code>frame</code>: A <code>CartesianFrame3D</code> indicating where the geometry is attached in the mechanism</li>
<li><code>geometry</code>: One of the <a href="https://github.com/JuliaGeometry/GeometryTypes.jl">GeometryTypes.jl</a> types</li>
<li><code>color</code>: an RGBA color from <a href="https://github.com/JuliaGraphics/ColorTypes.jl">ColorTypes.jl</a></li>
<li><code>transform</code>: a <code>Transformation</code> from <a href="https://github.com/FugroRoames/CoordinateTransformations.jl/">CoordinateTransformations.jl</a> indicating the pose of the geometry w.r.t its attached frame.</li>
</ul>

<p>These demonstrations use the Boston Dynamics Atlas robot from <a href="https://github.com/tkoolen/AtlasRobot.jl">AtlasRobot.jl</a>.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> AtlasRobot
<span class="pl-k">using</span> MechanismGeometries
mechanism <span class="pl-k">=</span> AtlasRobot<span class="pl-k">.</span><span class="pl-c1">mechanism</span>()</pre></div>

<div class="highlight highlight-source-julia"><pre>Skeleton <span class="pl-k">&lt;:</span> <span class="pl-c1">AbstractGeometrySource</span></pre></div>
<p>The <code>Skeleton</code> type uses only the joints and bodies in the mechanism itself to construct a visual representation of the robot's links. The sticks connect joints in the mechanism and the ellipsoids represent the mass and moment of inertia of each body:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">visual_elements</span>(mechanism, <span class="pl-c1">Skeleton</span>())</pre></div>
<p><a target="_blank" rel="noopener noreferrer" href="https://user-images.githubusercontent.com/591886/37125808-180c1eca-223c-11e8-8e60-af3f48ba52c2.png"><img src="https://user-images.githubusercontent.com/591886/37125808-180c1eca-223c-11e8-8e60-af3f48ba52c2.png" alt="skeleton_with_inertias" style="max-width:100%;"></a></p>
<p>The moment of inertia ellipsoids can also be turned off, leaving just the joint connections:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">visual_elements</span>(mechanism, <span class="pl-c1">Skeleton</span>(inertias<span class="pl-k">=</span><span class="pl-c1">false</span>))</pre></div>

<h2><a id="user-content-urdf-visuals" class="anchor" aria-hidden="true" href="#urdf-visuals"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>URDF Visuals</h2>
<div class="highlight highlight-source-julia"><pre>URDFVisuals <span class="pl-k">&lt;:</span> <span class="pl-c1">AbstractGeometrySource</span></pre></div>
<p>The <code>URDFVisuals</code> type loads the visual elements from a given URDF file (passed as either a filename or a parsed <code>XMLDocument</code> from LightXML.jl). One particularly useful argument is <code>package_path</code>, which accepts a list of strings to use as potential directories to search when encountering mesh files using the ROS <code>package://</code> syntax.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">visual_elements</span>(mechanism, 
                <span class="pl-c1">URDFVisuals</span>(AtlasRobot<span class="pl-k">.</span><span class="pl-c1">urdfpath</span>(), 
                            package_path<span class="pl-k">=</span>[AtlasRobot<span class="pl-k">.</span><span class="pl-c1">packagepath</span>()]))</pre></div>

<p>The following extensions to the URDF spec are parsed by MechanismGeometries.jl:</p>
<ul>
<li><code>&lt;plane normal="0 0 1"/&gt;</code>: Represents an infinite plane perpendicular to the <code>normal</code> given as an x y z unit vector. Returns a MechanismGeometries.HyperPlane</li>
</ul>

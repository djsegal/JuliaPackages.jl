<p dir="auto">This package implements several methods of generating or loading geometries associated with a <a href="https://github.com/tkoolen/RigidBodyDynamics.jl">RigidBodyDynamics.jl</a> <code>Mechanism</code> in Julia. It is currently used by <a href="https://github.com/JuliaRobotics/MeshCatMechanisms.jl">MeshCatMechanisms.jl</a> but can also be used independently.</p>
<h1 dir="auto"><a id="user-content-interface" class="anchor" aria-hidden="true" href="#interface"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Interface</h1>
<p dir="auto">This package exports one primary method:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="visual_elements(mechanism::Mechanism, source::AbstractGeometrySource)"><pre><span class="pl-c1">visual_elements</span>(mechanism<span class="pl-k">::</span><span class="pl-c1">Mechanism</span>, source<span class="pl-k">::</span><span class="pl-c1">AbstractGeometrySource</span>)</pre></div>
<p dir="auto"><code>visual_elements</code> returns a vector of <code>VisualElement</code> structs, each of which contains:</p>
<ul dir="auto">
<li><code>frame</code>: A <code>CartesianFrame3D</code> indicating where the geometry is attached in the mechanism</li>
<li><code>geometry</code>: One of the <a href="https://github.com/JuliaGeometry/GeometryBasics.jl">GeometryBasics.jl</a> types</li>
<li><code>color</code>: an RGBA color from <a href="https://github.com/JuliaGraphics/ColorTypes.jl">ColorTypes.jl</a></li>
<li><code>transform</code>: a <code>Transformation</code> from <a href="https://github.com/FugroRoames/CoordinateTransformations.jl/">CoordinateTransformations.jl</a> indicating the pose of the geometry w.r.t its attached frame.</li>
</ul>

<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="URDFVisuals &lt;: AbstractGeometrySource"><pre>URDFVisuals <span class="pl-k">&lt;:</span> <span class="pl-c1">AbstractGeometrySource</span></pre></div>
<p dir="auto">The <code>URDFVisuals</code> type loads the visual elements from a given URDF file (passed as either a filename or a parsed <code>XMLDocument</code> from LightXML.jl). One particularly useful argument is <code>package_path</code>, which accepts a list of strings to use as potential directories to search when encountering mesh files using the ROS <code>package://</code> syntax.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="visual_elements(mechanism,

<p dir="auto">Julia interface to crypto functions from OpenSSL.</p>
<p dir="auto">gen/generate.jl generates the wrapper using Clang.jl</p>
<p dir="auto">src/crpyto_* files are the generated files
src/Crypto.jl and OpenSSL.jl includes the generated files</p>
<h2 dir="auto"><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Usage</h2>
<ul dir="auto">
<li>
<p dir="auto">Currently functions in openssl/md5.h, openssl/hmac.h, openssl/sha.h and openssl/evp.h have been wrapped</p>
</li>
<li>
<p dir="auto">Please refer to openssl documentation on using them</p>
</li>
<li>
<p dir="auto">The EVP_* family of functions provide higer level functions for other lower level openssl functions.</p>
</li>
<li>
<p dir="auto"><code>man EVP_DigestInit</code> has information on using them.</p>
</li>
<li>
<p dir="auto">Additionally, the following utility functions have been provided.</p>
</li>
</ul>
<p dir="auto"><code>hmacsha256_digest(s::String, k::Union(String, Vector{Uint8})) -&gt; Vector{Uint8}</code> returns a 32 byte HMACSHA256 digest for the given data and key</p>
<p dir="auto"><code>hmacsha1_digest(s::String, k::Union(String, Vector{Uint8})) -&gt; Vector{Uint8}</code>  returns a 20 byte HMACSHA1 digest for given data and key</p>
<p dir="auto"><code>md5(s::String) -&gt; Vector{Uint8}</code> is a regular 16 byte MD5 digest of the string</p>
<p dir="auto"><code>md5(s::IO) -&gt; Vector{Uint8}</code> same as md5 above, except that it processes the IOStream  or IOBuffer in 64K chunks</p>

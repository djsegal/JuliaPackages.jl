<p dir="auto">Install a <a href="https://docs.github.com/en/free-pro-team@latest/actions">GitHub Action</a> <a href="https://docs.github.com/en/free-pro-team@latest/actions/reference/workflow-syntax-for-github-actions">workflow file</a> in one or more repositories.</p>
<p dir="auto">There are two key steps: (1) creating the <code>workflow</code> and (2) installing it, a.k.a,
submitting it as pull request(s) to one or more packages.</p>
<h2 dir="auto"><a id="user-content-creating-the-workflow" class="anchor" aria-hidden="true" href="#creating-the-workflow"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Creating the workflow</h2>

<p dir="auto">or you can create your own:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="workflow = MassInstallAction.Workflow(&quot;MyWorkflow&quot;, &quot;workflow_filename.yml&quot; =&gt; read(&quot;/home/me/template_workflow_file.yml&quot;, String))"><pre>workflow <span class="pl-k">=</span> MassInstallAction<span class="pl-k">.</span><span class="pl-c1">Workflow</span>(<span class="pl-s"><span class="pl-pds">"</span>MyWorkflow<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>workflow_filename.yml<span class="pl-pds">"</span></span> <span class="pl-k">=&gt;</span> <span class="pl-c1">read</span>(<span class="pl-s"><span class="pl-pds">"</span>/home/me/template_workflow_file.yml<span class="pl-pds">"</span></span>, String))</pre></div>
<p dir="auto">where you replace:</p>
<ul dir="auto">
<li><code>"/home/me/template_workflow_file.yml"</code> with the path to the local file you've prepared with the desired contents of the workflow;</li>
<li><code>"workflow_filename.yml"</code> with the name you want the file to have in the repositories' <code>.github/workflows</code> directory;</li>
<li><code>"MyWorkflow"</code> with the name used to identify this workflow when Actions runs.</li>
</ul>
<p dir="auto">You can add multiple workflow files simultaneously or even delete files, see <code>?MassInstallAction.Workflow</code>.</p>

<p dir="auto">This package uses and interacts with <a href="https://github.com/JuliaWeb/GitHub.jl">GitHub.jl</a>. In addition to the options above, you can <code>MassInstallAction.install(workflow, repo::GitHub.Repo)</code> directly. This may be the easiest approach if you want to filter repositories based on specific criteria. See <code>?MassInstallAction.install</code> for more information.</p>

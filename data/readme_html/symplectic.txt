<h3 dir="auto"><a id="user-content-symplectic-form" class="anchor" aria-hidden="true" href="#symplectic-form"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Symplectic form</h3>
<p dir="auto">Throughout the document, the skew-symmetric form defining the symplectic matrix is of the following form
$$\Omega = \oplus_{i=1}^n \begin{pmatrix}
0 &amp; 1 \
-1 &amp; 0
\end{pmatrix},$$
i.e., a 2n-by-2n block-diagonal matrix with n identical sub-blocks. This matrix is provided as a struct <code>SymplecticForm</code> which resembles the <code>UniformScaling</code> struct in <code>LinearAlgebra</code>. Similar as the symbol <code>I</code> for <code>UniformScaling</code>, the symbol <code>$\Omega$</code> is reserved for <code>SymplecticForm</code>. One can quickly create the skew-symmetric matrix as shown above by calling <code>$\Omega$(2n)</code> where <code>2n</code> represents the size of the matrix. Just as <code>I</code>, <code>$\Omega$</code> can be used without an argument. That is to say, basis matrix arithmetics for <code>$\Omega$</code> are supported so long as the size of the matrix can be inferred from the context.</p>
<p dir="auto"><a id="user-content-org7cd6e92"></a></p>
<h3 dir="auto"><a id="user-content-basis-of-the-symplectic-vector-space" class="anchor" aria-hidden="true" href="#basis-of-the-symplectic-vector-space"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Basis of the symplectic vector space</h3>
<p dir="auto">With the explicit form of <code>$\Omega$</code> as is specified above, the basis of the underlying symplectic vector space is accordingly fixed by default. We refer to this default choice of symplectic basis the <strong>QPQP</strong> basis. However, for certain use cases, it is more convenient to work with an alternative basis, referred to as the <strong>QQPP</strong> basis, in which the symplectic form is of the following form
$$
\Omega = \begin{pmatrix}
0_n &amp; I_n \
-I_n &amp; 0_n
\end{pmatrix}
$$
with <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$0_n$</math-renderer> represents an n-by-n zero matrix and <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$I_n$</math-renderer> an n-by-n identity matrix. The function <code>toQQPPBasis</code> (respectively, <code>toQPQPBasis</code>) is provided to convert a matrix represented in the QPQP (respectively, QQPP) basis to the QQPP (respectively, QPQP) basis.</p>
<p dir="auto"><a id="user-content-orge7c3d99"></a></p>
<h3 dir="auto">
<a id="user-content-symplectic-cayley-transform" class="anchor" aria-hidden="true" href="#symplectic-cayley-transform"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Symplectic cayley transform</h3>
<p dir="auto">Cayley transform is an important technique to parametrize orthogonal or unitary matrices. With appropriate modification, it can also be used to parametrize other groups including the group of symplectic matrices.</p>
<p dir="auto">The symplectic cayley transform is defined as follows
<math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$$S = (\Omega M - I/2)(\Omega M + I/2)^{-1}$$</math-renderer>
where <code>M</code> is a 2n-by-2n <strong>real</strong> matrix and <code>S</code> is a 2n-by-2n symplectic matrix; and its inverse is
<math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$$M = \Omega (S - I)^{-1}(S + I)/2$$</math-renderer>
i.e., the <code>M</code> thus obtained should be the same as that in the equation above.</p>
<p dir="auto">The symplectic cayley transform is implemented as a single-argument funciton <code>symplecticCayleyTransform</code> (with alias <code>cayley</code>), and its inverse as <code>inverseSymplecticCayleyTransform</code> (with alias <code>invcaylay</code>).</p>
<p dir="auto"><a id="user-content-org642c433"></a></p>
<h3 dir="auto">
<a id="user-content-pre-iwasawa-factorization" class="anchor" aria-hidden="true" href="#pre-iwasawa-factorization"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Pre-Iwasawa factorization</h3>
<p dir="auto">A symplectic matrix <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$S$</math-renderer> can always be written as the multiplication of three symplectic matrices of specific structures, which is known as the pre-Iwasawa factorization. The factorization can be specificied by the following equation (note that the matrices are represented in the <strong>QQPP</strong> basis)
\[
S = \begin{pmatrix}
I<sub>n</sub> &amp; 0<sub>n</sub><br>
P &amp; I<sub>n</sub>
\end{pmatrix}</p>
<p dir="auto">\begin{pmatrix}
L &amp; 0_n \
0_n &amp; (L^T)^{-1}
\end{pmatrix}</p>
<p dir="auto">\begin{pmatrix}
X &amp; Y<br>
-Y &amp; X
\end{pmatrix}.
\]
Here, <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$P$</math-renderer> is an n-by-n symmetric matrix; <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$L$</math-renderer> is an n-by-n non-singular matrix; <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$X$</math-renderer> and <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$Y$</math-renderer> are n-by-n matrices satisfying that <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$X+iY$</math-renderer> is a unitary matrix, i.e., the right-most of the three matrices on the right-hand-side of the above equation is an orthogonal matrixb. The pre-Iwasawa factorization is unique and well-defined for all symplectic matrices.</p>
<p dir="auto">The pre-Iwasawa factorization is implemented as the function <code>preIwasawaFactorization</code> (or <code>preiwa</code>), which yields a tuple of three symplectic matrices as are on the right-hand-side of the above equation.</p>
<p dir="auto"><a id="user-content-org97f64b2"></a></p>
<h3 dir="auto">
<a id="user-content-utilities" class="anchor" aria-hidden="true" href="#utilities"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Utilities</h3>
<p dir="auto">We provide <code>hs_norm</code> to facilliate the calculation of the Hibert-Schmidt norm of a matrix. That is, <code>hs_norm(M)</code> yields <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$\mbox{tr}(M^T*M)$</math-renderer>.</p>
<p dir="auto">To check whether a matrix is symplectic, the user can use the function <code>nonSymplecticity</code>. It takes a matrix <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$S$</math-renderer> as the input and yields the Hilber-Schmidt norm of <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="a7ef60f2583d0f8b55d89be4fd1e0856">$S^T \Omega S - \Omega$</math-renderer>.</p>
<p dir="auto">The function <code>dsum</code> (or the binary operator <code>$\oplus$</code>) is used to produce direct sum of two matrices. It can also take more than two matrices as input: <code>dsum(A1, A2, ...)</code></p>
<p dir="auto">The binary operator <code>\otimes</code> is assigned to <code>Base.kron</code> to allow tensor product.</p>

<p dir="auto">In physics, symplectic matrices significantly simplifies the analysis of unitary Gaussian processes transforming quadratures operators into their linear combinations. Representations of some commonly used physical componets are provided.</p>
<p dir="auto"><a id="user-content-orgd37b284"></a></p>
<h3 dir="auto">
<a id="user-content-phase-shifting" class="anchor" aria-hidden="true" href="#phase-shifting"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Phase shifting</h3>
<p dir="auto">The 2-by-2 symplectic matrix corresponding to a single-mode phase-shifting operation can be created by the <code>phaseShifting</code> with a single real argument. When multiple arguments are given, e.g. ~phaseShifting(x1, x2, x3), it yields the symplectic matrix correpsonding to the simultaneous application of multiple single-mode operations with angles specificed correspondingly by the
arguments.</p>

<h3 dir="auto">

<p dir="auto">The function <code>teleport(S::AbstractMatrix, inModes::Vector, outModes::Vector)</code> implements of the main result in Phys. Rev. Lett. 120, 020502<sup><a id="user-content-fnr.1" href="#fn.1">1</a></sup>. Here, <code>S</code> represents a 2n-by-2n symplectic matrix representing a unitary Gaussian operatio on an n-mode system. <code>inModes</code> and <code>outModes</code> are vectors of equal sizes representing the input and output modes. Let <code>ancModes</code> denote the vector of the modes that are not in <code>inModes</code>, and <code>idlModes</code> the vector of the modes that are not in <code>outModes</code>. Let <code>Usq</code> denote <code>2*ancModes.-1</code>, and <code>Hm</code> denote <code>2*idlModes.-1</code>, <code>In</code> denote the set of indices either in <code>2*inModes.-1</code> or <code>2*inModes</code>, and <code>Out</code> denote the set of indices either in <code>2*outModes.-1</code> or <code>2*outModes</code>. This function outputs <code>S[Out,In]-S[Out,Usq]*(S[Hm,Usq])^(-1)*S[Hm,In]</code> which is guaranteed to be a symplectic matrix so long as <code>S[Hm,Usq]</code> is non-singular.</p>
<p dir="auto">The function <code>feedforward(S::AbstractMatrix, inModes::Vector, outModes::Vector)</code> yields the product <code>-S[Out,USq]*(S[Hm,Usq])^(-1)</code> directly.</p>
<p dir="auto">The function <code>adaptiveMeasurement(F::AbstractMatrix, outModes::Vector, n::Integer)</code> yields the matrix defined in Eq.(4.3.11) of <sup><a id="user-content-fnr.2" href="#fn.2">2</a></sup> in the default QPQP basis. Here <code>F</code> is a linear map from <code>Hm</code> to <code>outModes</code>.</p>

<p dir="auto">The function <code>interferenceBasedSequence(S::AbstractMatrix; T=I(4))</code> yields an array of symplectic matrices consisting of multiple copies of <code>S</code> interspersed with symplectic matrices that are direct sum of single-mode diagonal blocks.

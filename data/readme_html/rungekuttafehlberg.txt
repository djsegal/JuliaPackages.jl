<p dir="auto">An implementation of the well-known <a href="https://dx.doi.org/10.1007/BF02241732" rel="nofollow">Runge-Kutta-Fehlberg time integration method</a> of 4th and 5th order (RKF45).
The algorithm integrates differential equations of the form:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="dx / dt = f[x](t)"><pre class="notranslate"><code>dx / dt = f[x](t)
</code></pre></div>
<p dir="auto">Notably, <code>f</code> can be either a function or a functional of <code>x</code>. This is useful for certain types of partial differential equations (e.g. the heat equation).</p>

<p dir="auto">You import the package as usual:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using RungeKuttaFehlberg"><pre class="notranslate"><code>using RungeKuttaFehlberg
</code></pre></div>
<p dir="auto">The package exports exactly one function</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="rkf45_step(f, x, t, tolerance, dt[, error, safety])"><pre class="notranslate"><code>rkf45_step(f, x, t, tolerance, dt[, error, safety])
</code></pre></div>
<p dir="auto">which returns <code>dx</code> and <code>dt</code> as a tuple.
Most arguments should be self-explanatory but more detailed documentation is included in the package.</p>
<h2 dir="auto"><a id="user-content-additional-comments" class="anchor" aria-hidden="true" href="#additional-comments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Additional comments</h2>
<p dir="auto">Most likely you will iterate over <code>rkf45_step()</code> and sum up <code>dx</code> and <code>dt</code>.
The algorithm will run most efficiently if you pass the last return value for <code>dt</code> back into <code>rkf45_step()</code> at the next iteration.</p>
<p dir="auto">The r.h.s. function <code>f()</code> must take exactly two arguments, <code>x</code> and <code>t</code>. Currently, there is no way to pass additional parameters to <code>f()</code>.
However, you can easily define an intermediate function which contains the values of each parameter and then pass it to <code>rkf45_step()</code>.</p>
<p dir="auto">RKF45 evaluates <code>f()</code> <em>at least</em> 6 times during each step, so optimizing <code>f()</code> can increase performance a lot.</p>
<p dir="auto">I am currently hosting this in a separate package, but I am open to suggestions w.r.t. inclusion in a package for time integration methods.</p>

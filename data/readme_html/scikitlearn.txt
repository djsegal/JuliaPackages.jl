<p dir="auto"><a href="https://github.com/cstjean/ScikitLearn.jl/actions"><img src="https://github.com/cstjean/ScikitLearn.jl/workflows/CI/badge.svg" alt="Build Status" style="max-width: 100%;"></a>
<a href="https://cstjean.github.io/ScikitLearn.jl/dev/" rel="nofollow"><img src="https://camo.githubusercontent.com/c97f0a5f2ae95755f64a27f1aa8d9a17462941fd3d6c907c7630abd5d3e60acf/68747470733a2f2f696d672e736869656c64732e696f2f62616467652f646f63732d737461626c652d626c75652e737667" alt="Stable" data-canonical-src="https://img.shields.io/badge/docs-stable-blue.svg" style="max-width: 100%;"></a></p>
<p dir="auto">ScikitLearn.jl implements the popular
<a href="http://scikit-learn.org/stable/" rel="nofollow">scikit-learn</a> interface and algorithms in
Julia. It supports both models from the Julia ecosystem and those of the
<a href="http://scikit-learn.org/stable/modules/classes.html" rel="nofollow">scikit-learn library</a>
(via PyCall.jl).</p>
<p dir="auto">Would you rather use a machine-learning framework specially-designed for Julia? Check out <a href="https://github.com/alan-turing-institute/MLJ.jl">MLJ.jl</a>, from the Alan Turing institute.</p>
<p dir="auto"><strong>Disclaimer</strong>: ScikitLearn.jl borrows code and documentation from
<a href="http://scikit-learn.org/stable/" rel="nofollow">scikit-learn</a>, but <em>it is not an official part
of that project</em>. It is licensed under <a href="LICENSE">BSD-3</a>.</p>
<p dir="auto">Main features:</p>
<ul dir="auto">
<li>Around 150 <a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/models/#Julia-models-1" rel="nofollow">Julia</a> and <a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/models/#Python-models-1" rel="nofollow">Python</a> models accessed through a uniform <a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/api/" rel="nofollow">interface</a></li>
<li><a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/pipelines/" rel="nofollow">Pipelines and FeatureUnions</a></li>
<li><a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/cross_validation/" rel="nofollow">Cross-validation</a></li>
<li><a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/model_selection/" rel="nofollow">Hyperparameter tuning</a></li>
<li><a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/dataframes/" rel="nofollow">DataFrames support</a></li>
</ul>
<p dir="auto">Check out the <a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/quickstart/" rel="nofollow">Quick-Start
Guide</a> for a
tour.</p>

<p dir="auto">To install ScikitLearn.jl, type <code>]add ScikitLearn</code> at the REPL.</p>
<p dir="auto">To import Python models (optional), ScikitLearn.jl requires <a href="https://cstjean.github.io/ScikitLearn.jl/dev/man/models/#Installation-and-importing-Python-models-1" rel="nofollow">the scikit-learn Python library</a>, which will be installed automatically when needed. Most of the examples use <a href="https://github.com/stevengj/PyPlot.jl">PyPlot.jl</a></p>

<p dir="auto">On Linux builds, importing python models via <code>@sk_import</code> is known to fail for Julia v&lt;0.8.4 when the <code>PYTHON</code> enviroment variable from <code>PyCall.jl</code> is set to <code>""</code> or <code>conda</code>. This is becuase the version libstdcxx loaded by Julia v&lt;0.8.4 isn't compatible with the version of scikit-learn installed via Conda.
The easiest and recommended way to resolve this is to upgrade to Julia v&gt;=1.8.4. If you must stick with your current julia version you can also resolve this issue by pre-appending your system's <code>LD_LIBRARY_PATH</code> enviroment variable as shown below</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="ROOT_ENV=`julia -e &quot;using Conda; print(Conda.ROOTENV)`

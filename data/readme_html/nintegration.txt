<h1><a id="user-content-nintegrationjl" class="anchor" aria-hidden="true" href="#nintegrationjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>NIntegration.jl</h1>
<p>This is library intended to provided multidimensional numerical integration
routines in pure <a href="http://julialang.org" rel="nofollow">Julia</a></p>
<h2><a id="user-content-status" class="anchor" aria-hidden="true" href="#status"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Status</h2>
<p>For the time being this library can only perform integrals in three dimensions.</p>

<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<p><code>NIntegration.jl</code> should work on Julia 0.5 and later versions and can be
installed from a Julia session by running</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> Pkg<span class="pl-k">.</span><span class="pl-c1">clone</span>(<span class="pl-s"><span class="pl-pds">"</span>https://github.com/pabloferz/NIntegration.jl.git<span class="pl-pds">"</span></span>)</pre></div>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<p>Once installed, run</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> NIntegration</pre></div>
<p>To integrate a function <code>f(x, y, z)</code> on the
<a href="https://en.wikipedia.org/wiki/Hyperrectangle" rel="nofollow">hyperrectangle</a> defined by
<code>xmin</code> and <code>xmax</code>, just call</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">nintegrate</span>(f<span class="pl-k">::</span><span class="pl-c1">Function</span>, xmin<span class="pl-k">::</span><span class="pl-c1">NTuple{N}</span>, xmax<span class="pl-k">::</span><span class="pl-c1">NTuple{N}</span>;
           reltol <span class="pl-k">=</span> <span class="pl-c1">1e-6</span>, abstol <span class="pl-k">=</span> <span class="pl-c1">eps</span>(), maxevals <span class="pl-k">=</span> <span class="pl-c1">1000000</span>)</pre></div>
<p>The above returns a tuple <code>(I, E, n, R)</code> of the calculated integral <code>I</code>, the
estimated error <code>E</code>, the number of integrand evaluations <code>n</code>, and a list <code>R</code> of
the subregions in which the integration domain was subdivided.</p>
<p>If you need to evaluate multiple functions <code>(f₁, f₂, ...)</code> on the same
integration domain, you can evaluate the function <code>f</code> with more "features" and
use its subregions list to estimate the integral for the rest of the functions
in the list, e.g.</p>
<div class="highlight highlight-source-julia"><pre>(I, E, n, R) <span class="pl-k">=</span> <span class="pl-c1">nintegrate</span>(f, xmin, xmin)
I₁ <span class="pl-k">=</span> <span class="pl-c1">nintegrate</span>(f₁, R)</pre></div>

<p>The integration algorithm is based on the one decribed in:</p>
<ul>
<li>J. Berntsen, T. O. Espelid, and A. Genz, "An Adaptive Algorithm for the
Approximate Calculation of Multiple Integrals," <em>ACM Trans. Math. Soft.</em>, 17
(4), 437-451 (1991).</li>
</ul>

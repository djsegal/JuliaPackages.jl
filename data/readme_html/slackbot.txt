<p dir="auto">A Julia REPL that interacts with Slack.com's webhook integration.</p>
<p dir="auto">WARNING: Use at your own risk. This is a very naive, insecure bot that is meant more as proof of concept than for production use.</p>
<h2 dir="auto"><a id="user-content-setup" class="anchor" aria-hidden="true" href="#setup"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Setup</h2>
<ol dir="auto">
<li>
<p dir="auto">Set up a Slack Incoming WebHook and a Slack Outgoing WebHook.</p>
</li>
<li>
<p dir="auto">Create a <code>config.jl</code> in this directory that contains the following entries:</p>
</li>
</ol>
<ul dir="auto">
<li><code>TOKEN = "..." #Replace this with your Slack token for your outgoing webhook</code></li>
<li><code>INHOOK = "https://hooks.slack.com/services/..." #Replace this with your Webhook URL for your incoming webhook</code></li>
<li>(Optional) <code>DEBUG = true #Print debugging information to console</code></li>
<li>(Optional) <code>DEFAULTPAYLOAD = Dict( #Add custom JSON entries to the returning payload to the incoming webhook "username"=&gt;"juliatan", "icon_emoji"=&gt;":juliatan:" ) </code></li>
</ul>
<ol start="3" dir="auto">
<li>
<p dir="auto">Make sure that <a href="https://github.com/JuliaWeb/Morsel.jl"><code>Morsel.jl</code></a> is installed in Julia.</p>
</li>
<li>
<p dir="auto">Run the <code>script/jl.sh</code></p>
</li>
</ol>

<h2 dir="auto"><a id="user-content-the-reboot-escape-hatch" class="anchor" aria-hidden="true" href="#the-reboot-escape-hatch"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>The reboot escape hatch</h2>
<p dir="auto">It's quite possible to bork the entire state of the current Julia session. When this happens, and if you set <code>DEBUG = true</code>, then sending a HTTP POST to <code>/reboot</code> causes the current Julia instance to terminate, in effect restarting Slackbot.jl.</p>

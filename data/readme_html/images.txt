<p>Images.jl is an open-source image processing library for <a href="http://julialang.org/" rel="nofollow">Julia</a>.</p>
<h2><a id="user-content-project-organization" class="anchor" aria-hidden="true" href="#project-organization"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Project organization</h2>
<p>Images.jl is increasingly becoming an "umbrella package" that exports a <a href="https://juliaimages.org/latest/pkgs/" rel="nofollow">set of packages</a> which are useful for common image processing tasks.
Most of these packages are hosted at
<a href="https://github.com/JuliaImages">JuliaImages</a>,
<a href="https://github.com/JuliaArrays">JuliaArrays</a>,
<a href="https://github.com/JuliaIO">JuliaIO</a>,
<a href="https://github.com/JuliaGraphics">JuliaGraphics</a>, and
<a href="https://github.com/JuliaMath">JuliaMath</a>.</p>

<p>You can join the Julia community by joining <a href="https://julialang.slack.com" rel="nofollow">slack</a> (get an invite from <a href="https://julialang.org/slack/" rel="nofollow">https://julialang.org/slack/</a>), <a href="https://julialang.zulipchat.com/" rel="nofollow">zulip</a>, and/or <a href="https://discourse.julialang.org/" rel="nofollow">discourse</a>.
For questions and discussions related to the JuliaImages ecosystem, you can <a href="https://github.com/JuliaImages/Images.jl/discussions">open an discussion</a>; issues are reserved to bug reports and feature tracking.
Any question about <a href="https://juliaimages.org/" rel="nofollow">the documentation</a> is considered an issue, hence if you have any questions please feel free to ask it in <a href="https://github.com/JuliaImages/juliaimages.github.io/issues">the documentation repo</a>.</p>

<p>Images v0.23.z is the last minor version that is compatible to Julia &lt; v1.3. It will still be under maintenance, but only with minimal efforts. No forward compatibility guarantee will be made, which means that you might see APIs and behaviors of Images v0.23 are quite different from that of the latest Images version. For new Julia users, we recommend you to use the latest stable Julia version even if it is not long-term-support(LTS) version.</p>

<p>Elements of this package descend from "image.jl"
that once lived in Julia's <code>extras/</code> directory.
That file had several authors, of which the primary were
Jeff Bezanson, Stefan Kroboth, Tim Holy, Mike Nolta, and Stefan Karpinski.
This repository has been quite heavily reworked;
please see the "contributors" tab above and on many of the other repositories at <a href="https://github.com/JuliaImages">JuliaImages</a> and elsewhere.</p>

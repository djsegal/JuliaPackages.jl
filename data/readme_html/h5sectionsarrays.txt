<p dir="auto"><a href="https://travis-ci.org/seung-lab/H5SectionsArrays.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/ed83d05071d795f4fe81ad39aa5027dfc4ff2c2ef6383a4033e5884f98a3940c/68747470733a2f2f7472617669732d63692e6f72672f7365756e672d6c61622f483553656374696f6e734172726179732e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/seung-lab/H5SectionsArrays.jl.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">cutout arbitrary chunks from a serials of 2D image sections in hdf5 format</p>
<ul dir="auto">
<li>only support cutout, no writting</li>
<li>used as normal Julia array</li>
<li>support negative coordinate</li>
</ul>
<p dir="auto">Note that we only support UInt8 data type for now, should be easy to extend if neccesary.</p>

<p dir="auto">the hdf5 file should contain several datasets:</p>
<ul dir="auto">
<li>img: the image array</li>
<li>offset: a vector containing x,y offsets</li>
<li>size: the size of the <code>img</code> dataset</li>
</ul>
<p dir="auto">For details please take a look of the test script <code>runtests.jl</code>, which construct a fake dataset.</p>

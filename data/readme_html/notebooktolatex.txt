<h2 dir="auto"><a id="user-content-why-this-package" class="anchor" aria-hidden="true" href="#why-this-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why this Package?</h2>
<p dir="auto">This package converts your notebook files (Pluto or Jupyter) to beautiful and
simple LaTeX files, that are easy to modify. Thus, making it ideal
to write reports, articles or books from notebooks.</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="./src/assets/notebooktolatexexample.png"><img src="./src/assets/notebooktolatexexample.png" alt="NotebookToLaTeX Example" style="max-width: 100%;"></a></p>
<p dir="auto">Although it's already possible to convert both Pluto and Jupyter notebooks
to PDFs, or even to LaTeX (via Pandoc), the PDFs are not very customizable
and the LaTeX files are usually very messy.
In contrast, NotebookToLaTeX.jl focuses less in generality, and
more on opinionated defaults.</p>
<p dir="auto">The package has it's own implementation to parse Markdown to LaTeX,
e.g. it turns <code># Example</code> to <code>\chapter{Example}</code>. Thus,
one can dive down into the actual Julia code and customize it
for his own preference. Or, submit an issue requesting
the feature. Hopefully, more and more customization will
be possible from the get go as the package evolves.</p>
<p dir="auto">Another very important point to note is that NotebookToLaTeX.jl uses
<a href="https://github.com/mossr/julia-mono-listings">julia-mono-listing</a>.
This enables it to produce beautiful Julia code inside the LaTeX pdf.
Note that <em>it requires using <code>lualatex</code> for compilation</em>.</p>

<p dir="auto">To convert the notebooks just use <code>notebooktolatex("mynotebook.ipynb", template=:book)</code>.
This will produce a directory <code>./build_latex/</code> where the LaTeX files
will be generated. Inside <code>build_latex/</code> you will have:</p>

<p dir="auto">Using <code>template=:book</code>, we get the LaTeX book format, thus, we have a <code>preface.tex</code>,
a <code>titlepage.tex</code> and a <code>copyright.tex</code> page. The notebook will be included
as a chapter. To get your final book pdf, just compile the <code>main.tex</code> using <code>lualatex</code>.</p>
<p dir="auto">In case you want a different project folder, you can run the command
with an extra argument providing the target directory for the LaTeX files, e.g.:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="`notebooktolatex(&quot;mynotebook.ipynb&quot;, &quot;./project/&quot;,template=:book)`."><pre><span class="pl-s"><span class="pl-pds">`</span>notebooktolatex("mynotebook.ipynb", "./project/",template=:book)<span class="pl-pds">`</span></span>.</pre></div>
<p dir="auto">This will create a <code>./project/</code> folder instead of the <code>./build_latex</code>.</p>
<p dir="auto">If instead you just want a simple report containing the Notebook,
you can use the <code>:article</code> template.</p>

<p dir="auto">If you want to convert Pluto notebooks, you have to import <code>Makie.jl</code>, <code>CairoMakie.jl</code> and
<code>Plots.jl</code>, which are the packages used in order to save your figures. The reason for this
is that NotebookToLaTeX.jl uses Requires.jl in order to avoid these packages as dependencies.
Hence, to convert Pluto, you have to run:</p>

<p dir="auto">At the moment, the available templates are:</p>
<ul dir="auto">
<li><code>:book</code> - The standard LaTeX book template;</li>
<li><code>:mathbook</code> - Very similar to <code>:book</code>, but with some extra packages for mathematics;</li>
<li><code>:article</code> - Simple template using the <code>article</code> document class;</li>
<li><code>:matharticle</code> - The <code>article</code> template with extra packages for mathematics.</li>
</ul>
<p dir="auto">Read the <a href="https://davibarreira.github.io/NotebookToLaTeX.jl/dev" rel="nofollow">documentation</a> for more information.</p>

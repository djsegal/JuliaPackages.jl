<p dir="auto"><a href="https://github.com/jblindsay/whitebox-tools">WhiteboxTools</a> is an open-source command-line program for geospatial analysis created by <a href="https://jblindsay.github.io/ghrg/index.html" rel="nofollow">Dr. John Lindsay</a>.</p>
<p dir="auto"><strong>Whitebox.jl</strong> installs <strong>WhiteboxTools</strong> (<strong>v2.1.0</strong>) and allows users to run any <strong>WhiteboxTools</strong> function from Julia.</p>

<p dir="auto">Load <strong>Whitebox.jl</strong> into your project with:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="import Whitebox as wbt"><pre><span class="pl-k">import</span> Whitebox <span class="pl-k">as</span> wbt</pre></div>
<p dir="auto">Loading the package automatically configures your <strong>WhiteboxTools</strong> session by:</p>
<ul dir="auto">
<li>Setting the working directory for <strong>WhiteboxTools</strong> to your current working directory (found with <code>pwd()</code>)</li>
<li>Setting the the path to the <strong>WhiteboxTools</strong> version that was installed with <strong>Whitebox.jl</strong></li>
<li>Setting up miscellaneous defaults such as the printing of results</li>
</ul>
<p dir="auto">In other words, just load the package and you should be good to go! Read more about this in <a href="@ref">How it works</a>.</p>
<h2 dir="auto"><a id="user-content-find-tools" class="anchor" aria-hidden="true" href="#find-tools"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Find tools</h2>
<p dir="auto"><strong>WhiteboxTools</strong> contains approximately <a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/index.html" rel="nofollow">485 tools</a> that are organized into the following thematic toolboxes:</p>
<ul dir="auto">
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/data_tools.html" rel="nofollow">Data Tools</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/geomorphometric_analysis.html" rel="nofollow">Geomorphometric Analysis</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/gis_analysis.html" rel="nofollow">GIS Analysis</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/hydrological_analysis.html" rel="nofollow">Hydrologic Analysis</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/image_processing_tools.html" rel="nofollow">Image Analysis</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/lidar_tools.html" rel="nofollow">LiDAR Analysis</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/mathand_stats_tools.html" rel="nofollow">Mathematical and Statistical Analysis</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/precision_agriculture.html" rel="nofollow">Precision Agriculture</a></li>
<li><a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/stream_network_analysis.html" rel="nofollow">Stream Network Analysis</a></li>
</ul>
<p dir="auto"><strong>Whitebox.jl</strong> provides a function to use each of the the tools provided by <strong>WhiteboxTools</strong>. Functions in <strong>Whitebox.jl</strong> use the snake_case convention for <strong>WhiteboxTools</strong> tool names.</p>
<p dir="auto">For example, to use the <a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/mathand_stats_tools.html?highlight=AbsoluteValue#absolutevalue" rel="nofollow"><code>AbsoluteValue</code> tool</a>, you would use <code>wbt.absolute_value()</code>.</p>
<p dir="auto">View the <strong>Whitebox.jl</strong> <a href="https://acgold.github.io/Whitebox.jl/dev/reference" rel="nofollow">Reference</a> section for a full list of available functions and documentation.</p>

<p dir="auto">We have a few different ways to find the required arguments or documentation for a function:</p>
<ul dir="auto">
<li>Visit the <a href="https://www.whiteboxgeo.com/manual/wbt_book/available_tools/index.html" rel="nofollow">WhiteboxTools user manual</a> and look at the Python example for a function (those use snake_case too)</li>
<li>Run <code>wbt.tool_parameters(tool_name::String)</code>, or</li>

<p dir="auto">Most of the functions in this package were generated using the included <code>whitebox_plugin_generator.py</code>. This file was adapted from the file of the same name in the <strong>WhiteboxTools</strong> <a href="https://github.com/jblindsay/whitebox-tools/blob/master/whitebox_plugin_generator.py">repo</a></p>
<p dir="auto">Using a python script was intentional so that future updates or added functions could be generated from the Python package.</p>
<p dir="auto">Some additional changes to the generated functions were required:</p>
<ul dir="auto">
<li><code>wbt.multiscale_topographic_position_image</code>: changed <code>local</code> input argument references to <code>localrast</code>. <code>local</code> is a reserved word for Julia.</li>
<li><code>wbt.conditional_evaluation</code>: <code>true</code> and <code>false</code> input arguments were changed to <code>True</code> and <code>False</code>.</li>
</ul>

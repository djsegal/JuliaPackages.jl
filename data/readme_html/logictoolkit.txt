<p dir="auto">Welcome to <strong>LogicToolkit</strong> package - logician's best friend!</p>
<p dir="auto">This package was created based on Bachelor Project by Jakub ≈Åucki and supervised by prof. dr. Rineke Verbrugge. Therefore, all important design choices can be found in the thesis itself under the following link: <a href="https://to-be-added" rel="nofollow">https://to-be-added</a>. Of course you are more than welcome to take a look if the topic of zero-one laws interests you.</p>
<p dir="auto">The package uses 2 main representations of formulas: Tree and String. You can switch from one to another by using parseFormula() and formula2String().</p>
<p dir="auto">Then formulas can be simplified using simplify().</p>
<p dir="auto">The tableau solver has function validate() which can take both strings and trees. To use it you just need to provide premise, consequent and contraints. Constraints can be following: "gl" for transitive and converse well-founded logic, "s4" for transitive and reflexive logic, "k4" for transitive logic.</p>
<p dir="auto">Then one can generate models and frames using dedicated functions. These structures can be either completely random or follow Kleitman and Rothschild's result. Then one can use checkModelValidity() or checkFrameValidity() to check validity in given structure.</p>
<p dir="auto">Ultimately, one can generate random formulas of arbitrary depth using generateFormula()</p>

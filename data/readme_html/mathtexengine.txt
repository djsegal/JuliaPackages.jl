<p dir="auto">This is a package aimed at providing a pure Julia engine for LaTeX math mode. It is composed of two main parts: a LaTeX parser and a LaTeX engine, both only for LaTeX math mode.</p>
<h1 dir="auto"><a id="user-content-main-features" class="anchor" aria-hidden="true" href="#main-features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Main features</h1>
<ul dir="auto">
<li>Parsing of (possibly nested) LaTeX expression and creation of a layout for them.</li>
<li>Equivalence between traditional LaTeX commands and their unicode input equivalent.</li>
<li>Pure julia.</li>
</ul>
<h1 dir="auto"><a id="user-content-engine" class="anchor" aria-hidden="true" href="#engine"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Engine</h1>
<p dir="auto">The main use of the package is through <code>generate_tex_elements</code> taking a LaTeX string as input and return a list of tuples <code>(TeXElement, position, scale)</code> where <code>TeXElement</code> is one of the following:</p>
<ul dir="auto">
<li><code>TeXChar(char, font)</code> a unicode character to be displayed in a specific font.</li>
<li><code>VLine(height, thickness)</code> a vertical line.</li>
<li><code>HLine(width, thickness)</code> an horizontal line.</li>
</ul>
<p dir="auto">This contains enough information to then draw everything with any plotting package that can draw arbitrary glyph in arbitrary font. The position is the origin of the character according to <code>FreeType</code>.</p>
<p dir="auto">Positions and scales are given for a font size of <code>1</code>. For bigger font size they simply need to be multiplied by the font size.</p>
<p dir="auto">The engine should support every construction the parser does (see below).</p>
<p dir="auto">Currently the only font set supported is Computer Modern.</p>

<p dir="auto">Parsing is done through the exported function <code>texparse</code> into nested <code>TeXExpr</code> objects forming a tree. The parser does not perform any operation to layout the elements, it only transforms them into a syntax tree.</p>
<p dir="auto">The readme file in the <code>src/parser</code> folder contains more details about the inner working of the parser.</p>

<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Introduction</h2>
<p>This package contains Julia versions of the mcmc models contained in the R package "rethinking" associated with the book <a href="https://xcelab.net/rm/statistical-rethinking/" rel="nofollow">Statistical Rethinking</a> by Richard McElreath. It is part of the <a href="https://github.com/StatisticalRethinkingJulia">StatisticalRethinkingJulia</a> Github organization of packages.</p>
<p>This package implements the models using <a href="https://github.com/TuringLang">TuringLang/Turing.jl</a>.</p>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Usage</h2>
<p>Most of the scripts and output can be inspected via the <a href="https://statisticalrethinkingjulia.github.io/TuringModels.jl/" rel="nofollow">website</a>.
If you want to run the scripts yourselves, then you can either</p>
<ol>
<li>copy the code from the webpages and the data from this repository, and run the scripts <strong>or</strong></li>
<li>clone this repository and run one of the files in <code>scripts</code>. For example, <code>julia --project -i scripts/basic-example.jl</code>.</li>
</ol>
<p>The scripts are written in <a href="https://github.com/fredrikekre/Literate.jl">Literate.jl</a> to allow them to be ran stand-alone, and as part of the website.
To generate the website locally, use <a href="https://github.com/tlienart/Franklin.jl">Franklin.jl</a>.
Specifically, clone this repository and go into the root directory of this repository.
Then, use</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="julia --project -ie 'using Franklin; Franklin.serve()'
"><pre><code>julia --project -ie 'using Franklin; Franklin.serve()'
</code></pre></div>
<p>This will activate the project environment (thanks to the <code>--project</code> flag) and will <em>interactively</em> execute <code>Franklin.serve()</code>.
Interactively means that if serve fails, then you will still be in an active REPL session which avoids having to completely restart Julia.
Building the site for the first time will take about 20 minutes.
After building the site, it will be available on <a href="http://localhost:8000/" rel="nofollow">http://localhost:8000/</a>.
Consecutive calls to serve will only take a few minutes because Franklin caches the output.</p>

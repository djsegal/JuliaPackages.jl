<h1 dir="auto"><a id="user-content-strategems" class="anchor" aria-hidden="true" href="#strategems"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Strategems</h1>
<p dir="auto"><strong>Strategems</strong> is a <a href="https://julialang.org/" rel="nofollow">Julia</a> package aimed at simplifying and streamlining the process of developing, testing, and optimizing algorithmic/systematic trading strategies. This package is inspired in large part by the quantstrat<sup><a href="http://past.rinfinance.com/agenda/2013/workshop/Humme+Peterson.pdf" rel="nofollow">1</a></sup><sup>,</sup><sup><a href="https://github.com/braverock/quantstrat">2</a></sup> package in <a href="https://www.r-project.org/" rel="nofollow">R</a>, adopting a similar general structure to the building blocks that make up a <em>strategy</em>.</p>
<p dir="auto">Given the highly iterative nature of event-driven trading strategy development, Julia's high-performance design (particularly in the context of loops) and straightforward syntax would seem to make it a natural fit as a language for systematic strategy research and development. While this package remains early in development, with time the hope is to be able to rapidly implement a trading idea, construct a historical backtest, analyze its results, optimize over a given parameter set, and visualize all of this with great detail.</p>

<p dir="auto">This package makes heavy use of the <a href="https://github.com/dysonance/Temporal.jl"><strong>Temporal</strong></a> package's <code>TS</code> time series type to facilitate the underlying computations involved in cleaning &amp; preprocessing the data used when testing a <code>Strategy</code>. Additionally, the <a href="https://github.com/dysonance/Indicators.jl/"><strong>Indicators</strong></a> package offers many technical analysis functions that have been written/designed with the goal of a highly generalized systematic trading strategy research engine in mind, and should thus should simplify the process of working with this data quite a bit.</p>

<p dir="auto">Below are the basic building blocks making up the general anatomy of a <em>Strategy</em> with respect to the <code>Strategems.jl</code> package design and the type definitions used to facilitate the research workflow.</p>
<ul dir="auto">
<li><code>Universe</code>: encapsulation of the assets/securities the strategy is to be allowed to trade</li>
<li><code>Indicator</code>: calculation done on each asset in the universe whose results we think have predictive potential for future price movement</li>
<li><code>ParameterSet</code>: inputs/arguments to the indicator calculations</li>
<li><code>Signal</code>: boolean flag sending messages to the trading logic/rules to be interpreted and acted upon</li>
<li><code>Rule</code>: applications of trading logic derived from interpretations of prior calculations &amp; signals at each time step</li>
<li><code>Strategy</code>: overarching object encapsulating and directing all of the above logic and data to power the backtesting engine</li>
</ul>

<p dir="auto">Below is a quick example demonstrating a simple use-case that one might use to get acquainted with how the package works. Note that the custom infix operators denoted by the uparrow and downarrow below are defined in this package as another way of expressing that one variable crosses over another. The intention of this infix operator definition is to hopefully make the definition of a strategy more syntactically expressive and intuitive.</p>
<p dir="auto">The key indicator used in this strategy is John Ehlers's MESA Adaptive Moving Average (or <em>MAMA</em> for short). This functionality is implemented in the <code>Indicators.jl</code> package described above, and outputs a <code>Matrix</code> (or <code>TS</code> object if one is passed as an input) of two columns, the first being the <em>MAMA</em> itself and the second being the <em>FAMA</em>, or following adaptive moving average.</p>
<p dir="auto">This strategy simply goes long when the <em>MAMA</em> crosses over the <em>FAMA</em>, and goes short when the <em>FAMA</em> crosses over the <em>MAMA</em>. Below is an implementation that shows how to set default arguments to the <code>Indicators.mama</code> function and run a simple backtest using those parameters, and also define specified ranges over which we might like to see how the strategy behaves under different parameter sets.</p>

    if isfile(savedata_path)
        return Temporal.tsread(savedata_path)
    else
        X = quandl(asset)
        if save_downloads
            if !isdir(dirname(savedata_path))
                mkdir(dirname(savedata_path))
            end
            Temporal.tswrite(X, savedata_path)
        end
        return X
    end
end

indicator     = Indicator(f, paramset)

sigshort = @signal MAMA ↓ FAMA

shortrule = @rule sigshort → short 100
exitrule  = @rule sigexit → liquidate 1.0
rules     = (longrule, shortrule, exitrule)

backtest!(strat)

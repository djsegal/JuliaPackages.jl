<p dir="auto">Encode the image into a <a href="https://en.wikipedia.org/wiki/Sixel" rel="nofollow">sixel</a> control sequence and vice versa. If your
terminal supports this format, then you can get a nice visualization of it.</p>
<p dir="auto">This package, although itself complete, is probably not the most convinient way to use for normal
users. Package authors that aims to support sixel for various image-like types (e.g., image, video,
gif, graphic, plot, latex) are the potential targeted users of this package.</p>
<p dir="auto">The core functionality of this package is supported by the C library <a href="https://github.com/libsixel/libsixel">libsixel</a>.</p>

<p dir="auto">This package exports two functions: <code>sixel_encode</code> and <code>sixel_decode</code>.</p>
<ul dir="auto">
<li><code>sixel_encode</code> converts the input array into sixel format sequence.</li>
<li><code>sixel_decode</code> converts the input sixel format sequence into colorant array.</li>
</ul>

<p dir="auto">One important thing about sixel is that not all terminals support sixel control sequence. The
following is an incomplete list of terminals that support sixel.</p>
<ul dir="auto">
<li>macOS: <a href="https://iterm2.com/" rel="nofollow">iTerm2</a>, <a href="https://github.com/christianparpart/contour">contour</a>, <a href="https://sourceforge.net/projects/mlterm/" rel="nofollow">mlterm</a> and <a href="https://wezfurlong.org/wezterm/" rel="nofollow">wezterm</a></li>
<li>Linux: <a href="https://sourceforge.net/projects/mlterm/" rel="nofollow">mlterm</a>, <a href="https://github.com/christianparpart/contour">contour</a>, <a href="https://invisible-island.net/xterm/" rel="nofollow">xterm</a>, <a href="https://konsole.kde.org/" rel="nofollow">konsole</a> and <a href="https://wezfurlong.org/wezterm/" rel="nofollow">wezterm</a></li>
<li>Windows: <a href="https://github.com/christianparpart/contour">contour</a>, <a href="https://github.com/mintty/mintty">mintty</a>, <a href="https://www.msys2.org/" rel="nofollow">msys2</a>, <a href="https://sourceforge.net/projects/mlterm/" rel="nofollow">mlterm</a> and <a href="https://wezfurlong.org/wezterm/" rel="nofollow">wezterm</a></li>
</ul>
<blockquote>
<p dir="auto">Above these I only manually test <a href="https://iterm2.com/" rel="nofollow">iTerm2</a>, <a href="https://sourceforge.net/projects/mlterm/" rel="nofollow">mlterm</a> and <a href="https://wezfurlong.org/wezterm/" rel="nofollow">wezterm</a>.</p>
</blockquote>
<blockquote>
<p dir="auto">To run <code>xterm</code> with sixel support, use <code>$ xterm -xrm "XTerm*decTerminalID: vt340" -xrm "XTerm*numColorRegisters: 256"</code>.</p>
</blockquote>
<p dir="auto">A demo in iTerm2:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer nofollow" href="https://user-images.githubusercontent.com/8684355/118017901-b98f4d80-b389-11eb-93cf-0f99c169121d.png"><img src="https://user-images.githubusercontent.com/8684355/118017901-b98f4d80-b389-11eb-93cf-0f99c169121d.png" alt="demo" width="1024" height="800" style="max-width: 100%;"></a></p>
<p dir="auto">It is the client terminal that renders the image. Hence if you're using any sixel-enabled terminal (e.g., iTerm2) and connect to other servers via ssh, you
still get sixel supported. Sixel is nothing special but a string sequence.</p>
<p dir="auto">Unfortunately, there are some famous widely used advanced terminal/emulator that do not support
sixel (yet):</p>
<ul dir="auto">
<li><a href="https://github.com/tmux/tmux/issues/1613#issuecomment-559940608" data-hovercard-type="issue" data-hovercard-url="/tmux/tmux/issues/1613/hovercard">tmux does not support sixel</a>.</li>
<li><a href="https://sw.kovidgoyal.net/kitty/" rel="nofollow">kitty</a> does not support sixel; it has its own image protocol.</li>
<li><a href="https://github.com/microsoft/terminal/issues/448" data-hovercard-type="issue" data-hovercard-url="/microsoft/terminal/issues/448/hovercard">Windows terminal does not support sixel</a></li>
</ul>
<p dir="auto">For more information, you can also read the <a href="https://github.com/libsixel/libsixel#terminal-requirements">Terminal requirements</a> section in the <a href="https://github.com/libsixel/libsixel">libsixel</a> repo.</p>
<p dir="auto">Sixel.jl provides a function to test if your terminal supports it: <code>Sixel.is_sixel_supported()</code>. If
your terminal actually supports sixel and it returns <code>false</code>, please open an issue for it.</p>

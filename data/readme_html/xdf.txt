<h1><a id="user-content-xdfjl" class="anchor" aria-hidden="true" href="#xdfjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>XDF.jl</h1>
<p>XDF.jl is an importer for <a href="https://github.com/sccn/xdf/wiki/Specifications">XDF files</a> written in Julia. This packages exposes the <code>read_xdf</code> function, which handles everything needed to load the contents of XDF files, including synchronizing time stamps of different streams.</p>

<p>Use the package manager to add XDF.jl by typing <code>] add XDF</code> in the Julia REPL.</p>

<h2><a id="user-content-current-status" class="anchor" aria-hidden="true" href="#current-status"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Current status</h2>
<p>This package is currently in an early stage, so here's an overview of what doesn't work (yet):</p>
<ul>
<li>Streams with <code>String</code> data types are not supported yet</li>
<li>Time synchronization (via linear regression) is always performed, this will be changed to an optional parameter</li>
<li>The complete XML headers and footers of streams are not yet available in the output</li>
<li>Dejittering of streams with regular sampling rates is not available yet</li>
<li>Selecting only specific streams to load does not work yet</li>
<li>XDF.jl is not available as a proper Julia package yet</li>
</ul>
<p>If you have a feature request or if you have found a bug, please open a new issue and let me know. I'd be especially interested in making the code more efficient, because this is basically my first Julia project and it might be neither idiomatic nor optimally efficient. Currently, I'm passing through the file twice: the first pass reads everything except sample chunks, whereas the second pass reads samples into preallocated arrays. I'm not sure if this is ideal, the code would be much simpler if it used just a simple pass (but then sample arrays will need to be concatenated).</p>

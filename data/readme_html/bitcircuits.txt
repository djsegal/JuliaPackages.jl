<p dir="auto"><a href="https://travis-ci.org/um-tech-evolution/BitCircuits.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/b0178f79df677e7ff907e6ec1e6e8e418ad01e618d9b98cf553c6122837e5a77/68747470733a2f2f7472617669732d63692e6f72672f756d2d746563682d65766f6c7574696f6e2f42697443697263756974732e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/um-tech-evolution/BitCircuits.jl.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">Boolean circuit tree implementation that uses bit strings to cache evaluation
results and bitwise operations to evaluate new circuits based on existing trees.</p>

<p dir="auto">The API is very simple. Trees consist of <code>Operation</code>s (interior nodes),
<code>Variable</code>s, and <code>Constant</code>s (both of which are leaf nodes). Right now all
operations are functions of two parameters.</p>
<h3 dir="auto"><a id="user-content-variable" class="anchor" aria-hidden="true" href="#variable"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Variable</h3>
<p dir="auto">The constructor takes a single parameter, an integer in <code>[0,5]</code> (there are six
possible variables.</p>

<p dir="auto">The constructor takes a function to apply, this must be a bitwise function that
can operate on integers, and two sub-trees, to which the function will be
applied.</p>

<p dir="auto">Evaluates a tree for a given set of variable values. The variable values are
specified as a bit string, where variable 5 is the high order bit and variable 0
is the low order bit.</p>

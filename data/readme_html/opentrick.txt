<p dir="auto">There are some <code>open</code> methods which only support the <code>open() do io ... end</code> conventions. This module provides a trick to enable keeping <code>io</code> for later usage. This is convenient for interactive programming.</p>

<p dir="auto"><a id="user-content-opentrick.opentrick" href="#OpenTrick.opentrick">#</a>
<strong><code>OpenTrick.opentrick</code></strong> — <em>Function</em>.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="opentrick(openfn[, args... [; &lt;keyword arguments&gt;]])"><pre class="notranslate"><code>opentrick(openfn[, args... [; &lt;keyword arguments&gt;]])
</code></pre></div>
<p dir="auto">Call <code>openfn</code> with <code>(handlefn, args... ,kwargs ...)</code> as arguments, return an <code>IOWrapper</code> instance. (NB:<code>handlefn</code> is provided by <code>opentrick</code>.)</p>
<p dir="auto"><strong>Arguments</strong></p>
<ul dir="auto">
<li><code>openfn::Function</code> function actually called to obtain a <code>IO</code> instance. <code>openfn</code> must take a <code>Function(::IO)</code> instance as its first argument</li>
<li><code>args</code> optional arguments that will be passed to <code>openfn</code></li>
<li><code>kwargs</code> optional keyword arguments that will be passed to <code>openfn</code></li>
</ul>

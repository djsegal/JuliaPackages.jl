<p dir="auto">This package provides basic functionality for parsing, viewing, and working with <a href="http://www.openstreetmap.org" rel="nofollow">OpenStreetMap</a> map data. The package is intended mainly for researchers who want to incorporate this rich, global data into their work, and has been designed with both speed and simplicity in mind, especially for those who might be new to Julia.</p>
<p dir="auto"><strong>Note:</strong> Our automated tests currently fail on Julia 0.4, but these problems appear to be contained to the Travis test system and are caused by a dependent package. OpenStreetMap.jl should run without issue on both Julia 0.3 and 0.4, and the tests pass on local machines for Julia 0.3.7 and 0.4.2 on OS X 10.11 and Ubuntu 14.04.</p>

<ul dir="auto">
<li>Parse an <a href="http://wiki.openstreetmap.org/wiki/OSM_XML" rel="nofollow">OpenStreetMap XML datafile</a> (OSM files)</li>
<li>Crop maps to specified boundaries</li>
<li><a href="https://en.wikipedia.org/wiki/Geographic_coordinate_conversion" rel="nofollow">Convert maps</a>
between LLA, ECEF, and ENU coordinates</li>
<li>Extract highways, buildings, and tagged features from OSM data</li>
<li>Filter data by various classes:
<ul dir="auto">
<li>Ways suitable for driving, walking, or cycling</li>
<li>Freeways, major city streets, residential streets, etc.</li>
<li>Accommodations, shops, industry, etc.</li>
</ul>
</li>
<li>Draw detailed maps using Julia's <a href="https://github.com/nolta/Winston.jl">Winston</a> graphics package
with a variety of options</li>
<li>Compute shortest or fastest driving, cycling, and walking routes using Julia's Graphs package</li>
</ul>

<p dir="auto">The following packages will automatically be added as "additional packages", if you do not already have them:</p>
<ul dir="auto">
<li>Geodesy.jl: Map coordinate transforms, formerly part of OpenStreetMap.jl</li>
<li>LightXML.jl: parsing OpenStreetMap datafiles</li>
<li>LibExpat.jl: streaming OpenStreetMap datafiles</li>
<li>Winston.jl: map plotting</li>
<li>Graphs.jl: map routing</li>
</ul>
<p dir="auto"><strong>Note:</strong> LightXML.jl relies on <em>libxml2</em>, which is shipped with Mac OS X and many Linux systems. So this package may work out of the box. If not, check whether you have <em>libxml2</em> installed and whether <em>libxml2.so</em> (for Linux) or <em>libxml2.dylib</em> (for Mac) is on your library search path. I have tested it to work out of the box in Mac OS X 10.9 and 10.10 (Mavericks and Yosemite), Ubuntu 14.04, and Windows 7. Winston.jl has a few additional dependencies, which it should resolve automatically. All other code is written in native Julia.</p>

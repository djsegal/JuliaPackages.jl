<p dir="auto">This package extends Julia's builtin <code>uppercase</code> and <code>lowercase</code> functions,
and adds <code>titlecase</code> and <code>foldcase</code>.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; uppercase(&quot;testingß&quot;)
&quot;TESTINGß&quot;

julia&gt; using UnicodeExtras

julia&gt; uppercase(&quot;testingß&quot;)
&quot;TESTINGSS&quot;

julia&gt; set_locale(&quot;tr&quot;)  # set locale to Turkish
&quot;tr&quot;

julia&gt; uppercase(&quot;testingß&quot;)
&quot;TESTİNGSS&quot;"><pre lang="jlcon" class="notranslate"><code>julia&gt; uppercase("testingß")
"TESTINGß"

julia&gt; using UnicodeExtras

julia&gt; uppercase("testingß")
"TESTINGSS"

julia&gt; set_locale("tr")  # set locale to Turkish
"tr"

julia&gt; uppercase("testingß")
"TESTİNGSS"
</code></pre></div>
<p dir="auto">Note that "ß" gets converted to "SS" after UnicodeExtras is loaded,
and "i" gets converted to "İ" (dotted capital I)
after the locale is set to Turkish.</p>

<p dir="auto">In julia, a string is conceptually an array of unicode code points.
While well defined, this occasionally causes confusion because a single
code point doesn't necessarily correspond to what people perceive as a single
"character".</p>
<p dir="auto">Take the following example:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; n1 = &quot;noe\u0308l&quot;
&quot;noël&quot;

julia&gt; length(s)
5"><pre lang="jlcon" class="notranslate"><code>julia&gt; n1 = "noe\u0308l"
"noël"

julia&gt; length(s)
5
</code></pre></div>
<p dir="auto">Here, the <code>ë</code> "character" here consists of two code points: 'e' &amp; '\u0308',
and so the length of the string is 5, not 4.</p>

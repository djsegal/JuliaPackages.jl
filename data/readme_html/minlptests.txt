<p dir="auto">This is a collection of JuMP models for testing nonlinear/polynomial solvers
with and without discrete variables in JuMP.</p>
<p dir="auto">This version is compatible with the current release of JuMP (using
MathOptInterface).</p>
<p dir="auto"><a href="https://github.com/jump-dev/MINLPTests.jl/actions?query=workflow%3ACI"><img src="https://github.com/jump-dev/MINLPTests.jl/workflows/CI/badge.svg?branch=master" alt="Build Status" style="max-width: 100%;"></a></p>
<h2 dir="auto"><a id="user-content-test-design-guidelines" class="anchor" aria-hidden="true" href="#test-design-guidelines"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Test Design Guidelines</h2>
<ul dir="auto">
<li>The tests are organized into broad categories based on the scope of typical
(e.g. continuous, convex functions, polynomial, ...)</li>
<li>Unit tests for JuMP integration testing should be "easy" models.  For example,
it should be possible for a non-global solver to find the global solution in
these tests</li>
<li>Mathematical property tests can be more difficult but should strive to be as
simple as possible</li>
</ul>

<p dir="auto">Directories:</p>
<ul dir="auto">
<li>nlp - nonconvex NLP models</li>
<li>nlp-cvx - convex NLP models</li>
<li>nlp-mi - nonconvex MINLP models</li>
<li>nlp-mi-cvx - convex MINLP models</li>
<li>poly - nonconvex polynomial models</li>
<li>poly-cvx - convex polynomial models</li>
<li>poly-mi - nonconvex polynomial models</li>
<li>poly-mi-cvx - convex polynomial models</li>
</ul>
<p dir="auto">File Names:</p>
<ul dir="auto">
<li>0xx_yyz - unit tests</li>
<li>1xx_yyz - 2D mathematical properties</li>
<li>2xx_yyz - 3D mathematical properties</li>
<li>5xx_yyz - nD mathematical properties</li>
<li>9xx_yyz - integration tests</li>
</ul>
<p dir="auto">z indicates a variant of a base problem</p>

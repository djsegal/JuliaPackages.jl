<p><em>Play with Julia code and share the fun!</em></p>
<p>This package is a collection of web-based widgets that you can use to:</p>
<ul>
<li>Do quick code experiments where you can tweak the inputs</li>
<li>Teach or share science</li>
<li>Make simple UIs for your projects, and <a href="#as-a-standalone-web-page">publish them over the Web</a></li>
<li>Make your own reusable widgets and publish them with your Julia library!</li>
</ul>
<p>It works in <a href="#ijulia--jupyter-notebooks">Jupyter notebooks</a>, <a href="#within-the-atom-text-editor">Atom IDE</a>, or as a <a href="#as-a-standalone-web-page">plain old web page</a>.</p>

<h2><a id="user-content-manipulate" class="anchor" aria-hidden="true" href="#manipulate"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>@manipulate</h2>
<p>The simplest way to use Interact is via the <code>@manipulate</code> macro.</p>
<p><a target="_blank" rel="noopener noreferrer" href="doc/imgs/psos.gif"><img alt="interacting with sliders and toggles" src="doc/imgs/psos.gif" width="600" height="auto" style="max-width:100%;"></a></p>
<p>The syntax of <code>@manipulate</code> is</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="@manipulate for var1=&lt;object&gt;, var2=&lt;object&gt;, etc...
    # do something with a, b, etc...
end
"><pre><span class="pl-c1">@manipulate</span> <span class="pl-k">for</span> var1<span class="pl-k">=</span><span class="pl-k">&lt;</span>object<span class="pl-k">&gt;</span>, var2<span class="pl-k">=</span><span class="pl-k">&lt;</span>object<span class="pl-k">&gt;</span>, etc<span class="pl-k">...</span>
    <span class="pl-c"><span class="pl-c">#</span> do something with a, b, etc...</span>
<span class="pl-k">end</span></pre></div>
<p>Here <code>&lt;object&gt;</code> can be one of the following:</p>
<ul>
<li><strong>A Range</strong>: (any <code>AbstractRange</code> of Real numbers): this will create a slider using which you can select one value for the variable from the range.</li>
<li><strong>An Array</strong>: This will create a series of toggle buttons which you can select one at a time.</li>
<li><strong>A Dictionary</strong>: Same as the above, but keys are used as the labels for the buttons, the value corresponding to the selected key will be the value of the variable.</li>
<li><strong>A String</strong>: This will create a text box with the given string as the default value, you can edit the string to update the value of the variable.</li>
<li><strong>A boolean</strong>: This will create a checkbox which is checked if the value is true, unchecked if it is false. You can toggle the checkbox to change the value of this variable.</li>
<li><strong>A number</strong>: This will create a spinbox where you can input, increment or decrement a number. Note that an Integer will only allow input of integers.</li>
<li><strong>A date or time</strong>: (from the Dates standard library). this will create a date picker with the date set to the given date.</li>
<li><strong>A color</strong>: (from the Colors package.) This will create a color picker!</li>
<li><strong>An Interact widget</strong>: (advanced) see below for how to create widgets à la carte. Check out the <a href="https://juliagizmos.github.io/Interact.jl/latest/widgets/" rel="nofollow">big list of widgets</a> here.</li>
<li><strong>An Observable</strong>: An <a href="https://juliagizmos.github.io/Observables.jl/stable/" rel="nofollow">Observable</a> can be used here to assign its current value to the variable. Any update to the variable will trigger an update to the <code>@manipulate</code>. Note that the result of an <code>@manipulate</code> is also an observable!</li>
</ul>
<h2><a id="user-content-widgets-à-la-carte" class="anchor" aria-hidden="true" href="#widgets-à-la-carte"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Widgets à la carte</h2>
<p>You can also create widgets by themselves using the same input scheme. Just pass the <code>&lt;object&gt;</code> to <code>widget()</code> which has the syntax:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="widget(&lt;object&gt;; label=&quot;&lt;my label&gt;&quot;)
"><pre><span class="pl-c1">widget</span>(<span class="pl-k">&lt;</span>object<span class="pl-k">&gt;</span>; label<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>&lt;my label&gt;<span class="pl-pds">"</span></span>)</pre></div>
<p>For example,</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="px=widget(0:0.01:.3, label=&quot;px&quot;)
py=widget(0:0.01:.3, label=&quot;py&quot;)
plane = widget(Dict(&quot;x=0&quot; =&gt; 1, &quot;y=0&quot; =&gt; 2), label=&quot;Section plane&quot;);
"><pre>px<span class="pl-k">=</span><span class="pl-c1">widget</span>(<span class="pl-c1">0</span><span class="pl-k">:</span><span class="pl-c1">0.01</span><span class="pl-k">:</span><span class="pl-c1">.3</span>, label<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>px<span class="pl-pds">"</span></span>)
py<span class="pl-k">=</span><span class="pl-c1">widget</span>(<span class="pl-c1">0</span><span class="pl-k">:</span><span class="pl-c1">0.01</span><span class="pl-k">:</span><span class="pl-c1">.3</span>, label<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>py<span class="pl-pds">"</span></span>)
plane <span class="pl-k">=</span> <span class="pl-c1">widget</span>(<span class="pl-c1">Dict</span>(<span class="pl-s"><span class="pl-pds">"</span>x=0<span class="pl-pds">"</span></span> <span class="pl-k">=&gt;</span> <span class="pl-c1">1</span>, <span class="pl-s"><span class="pl-pds">"</span>y=0<span class="pl-pds">"</span></span> <span class="pl-k">=&gt;</span> <span class="pl-c1">2</span>), label<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>Section plane<span class="pl-pds">"</span></span>);</pre></div>
<p>Then you can use <code>map</code> over the widgets to create new interactive output that depends on these widgets!</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="interactive_plot = map(plotsos, px, py, plane)
"><pre>interactive_plot <span class="pl-k">=</span> <span class="pl-c1">map</span>(plotsos, px, py, plane)</pre></div>
<p>And put them all together into a nice little layout:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="# stack vertically
vbox(
    hbox(px, py, plane), # stack horizontally
    interactive_plot)
"><pre><span class="pl-c"><span class="pl-c">#</span> stack vertically</span>
<span class="pl-c1">vbox</span>(
    <span class="pl-c1">hbox</span>(px, py, plane), <span class="pl-c"><span class="pl-c">#</span> stack horizontally</span>
    interactive_plot)</pre></div>

<p>Interact's output can be displayed either within Jupyter/Jupyterlab notebooks, the Atom text editor, or as a stand-alone web page. Below are the set up instructions for each of these front ends:</p>
<h2><a id="user-content-ijulia--jupyter-notebooks" class="anchor" aria-hidden="true" href="#ijulia--jupyter-notebooks"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>IJulia + Jupyter notebooks</h2>
<p>To set up Interact to work in Jupyter notebooks, first install Interact, IJulia and WebIO packages. Then install the required Jupyter extension by running:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using WebIO
WebIO.install_jupyter_nbextension()
"><pre><span class="pl-k">using</span> WebIO
WebIO<span class="pl-k">.</span><span class="pl-c1">install_jupyter_nbextension</span>()</pre></div>
<p><strong>Important:</strong> If you performed this step while the Jupyter notebook server is running, you will need to restart it for the extension to take effect.</p>
<blockquote>
<p><strong>If you have multiple Jupyter installations:</strong>
The <code>WebIO.install_jupyter_nbextension([jupyter])</code> function takes the path to the <code>jupyter</code> binary as an optional argument. If omitted, the installation first tries to find <code>jupyter</code> in your OS's default path. If it does not exist there, it tries to find a version that was installed using Conda.jl (which is the default when IJulia is installed without having an existing Jupyter set up, or by forcing it to install a copy via Conda.) If you have a <code>jupyter</code> binary in your OS path but also have a version installed via Conda and want to use the one installed via Conda, then you can set this keyword argument to be <code>WebIO.find_jupyter_cmd(condajl=true)</code>. For more information about the Jupyter integration, see <a href="https://juliagizmos.github.io/WebIO.jl/latest/providers/ijulia/" rel="nofollow">WebIO's documentation</a>. There is <a href="https://juliagizmos.github.io/WebIO.jl/latest/troubleshooting/not-detected/" rel="nofollow">more troubleshooting information here</a>.</p>
</blockquote>

<p>If you have set up the Julia integration provided by <a href="https://junolab.org/" rel="nofollow">Juno</a>, evaluating any expression which returns an Interact-renderable object (such as a widget or the output of <code>@manipulate</code>) will show up in a plot-pane within the editor. No extra setup steps are required.</p>

<p>Interact is built using <a href="https://github.com/JuliaGizmos/WebIO.jl">WebIO</a> which is a "hub" package that supports an ecosystem of composable web-based widgets that all share the same availability on the Julia front-ends.</p>
<p>Check out this <a href="https://gist.github.com/shashi/3c3bc877ab1b5096232586cfbba0ef8b">introduction notebook</a> about the WebIO ecosystem if you want to learn about how you can customize your UI using HTML, JavaScript and CSS. It will also get you started on writing your own widgets that others can use.</p>
<ul>
<li><a href="https://github.com/JuliaComputing/TableView.jl">TableView.jl</a>: show a spreadsheet view of any Table datatype</li>
<li><a href="https://github.com/JuliaGizmos/CSSUtil.jl">CSSUtil</a>: wraps CSS functionality in easy Julia functions. Also renders
markdown (and LaTeX). See the CSSUtil section in the introduction notebook linked above.</li>

<p><a href="https://github.com/JuliaMath/Quadmath.jl/actions"><img src="https://github.com/JuliaMath/Quadmath.jl/workflows/CI/badge.svg" alt="GitHub CI Build Status" style="max-width:100%;"></a></p>
<p>This is a Julia interface to libquadmath, providing a <code>Float128</code> type corresponding to the IEEE754 binary128 floating point format.</p>
<h2><a id="user-content-support" class="anchor" aria-hidden="true" href="#support"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Support</h2>
<p>Quadmath currently works on x86_64 Linux, macOS, and Windows.</p>
<ul>
<li>It may require a new-ish version of gcc which supports <code>__float128</code> type.</li>
<li>It has not been tested on 32 bit Linux.</li>
<li>It does not work on ARM due to the lack of libquadmath support for that platform.</li>
</ul>

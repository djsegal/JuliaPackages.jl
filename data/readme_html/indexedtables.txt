<p dir="auto"><strong>IndexedTables</strong> provides tabular data structures where some of the columns form a sorted index.
It provides the backend to <a href="https://github.com/JuliaData/JuliaDB.jl">JuliaDB</a>, but can
be used on its own for efficient in-memory data processing and analytics.</p>
<h2 dir="auto"><a id="user-content-data-structures" class="anchor" aria-hidden="true" href="#data-structures"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Data Structures</h2>
<p dir="auto">IndexedTables offers two data structures: <code>IndexedTable</code> and <code>NDSparse</code>.</p>
<ul dir="auto">
<li><strong>Both types store data <em>in columns</em></strong>.</li>
<li><strong><code>IndexedTable</code> and <code>NDSparse</code> differ mainly in how data is accessed.</strong></li>
<li><strong>Both types have equal performance for Table operations (<code>select</code>, <code>filter</code>, etc.).</strong></li>
</ul>

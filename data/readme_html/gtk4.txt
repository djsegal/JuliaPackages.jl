<p dir="auto">GUI building using the <a href="https://www.gtk.org" rel="nofollow">GTK</a> library, version 4. For a mature Julia package that supports GTK version 3, see <a href="https://github.com/JuliaGraphics/Gtk.jl">Gtk.jl</a>. <strong>Note that Gtk.jl and Gtk4.jl cannot be imported in the same Julia session.</strong></p>
<p dir="auto">This package builds on Gtk.jl but uses GObject introspection to support more of the functionality of the GTK library and its dependencies. GObject introspection for Julia is implemented using <a href="https://github.com/JuliaGtk/Gtk4.jl/tree/main/GI">GI.jl</a>, which is also hosted in this repository.</p>
<p dir="auto">Most of the code for GLib support (<code>GType</code>, <code>GValue</code>, <code>GObject</code>, etc.) was copied with minor changes from <a href="https://github.com/JuliaGraphics/Gtk.jl">Gtk.jl</a>. This includes integration of the GLib main loop with Julia's event loop.</p>
<p dir="auto">Documentation for this package can be found <a href="https://juliagtk.github.io/Gtk4.jl/dev/" rel="nofollow">here</a>. Complete GTK documentation is available at <a href="https://www.gtk.org/docs" rel="nofollow">https://www.gtk.org/docs</a>.</p>

<p dir="auto">Gtk4.jl is not recommended on 32 bit systems because of a deficiency with how GObject introspection is done here.</p>

<p dir="auto">On Wayland, a Cairo-based fallback backend will be used unless you tell <code>libglvnd_jll</code> where to find libEGL. This can be done by setting the environment variable __EGL_VENDOR_LIBRARY_DIRS. See <a href="https://gitlab.freedesktop.org/glvnd/libglvnd/-/blob/master/src/EGL/icd_enumeration.md" rel="nofollow">here</a> for details.</p>
<p dir="auto">For convenience, in Gtk4.jl this can be set as a preference for a particular Julia environment using Preferences.jl:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Gtk4
Gtk4.set_EGL_vendorlib_dirs(&quot;/usr/share/glvnd/egl_vendor.d&quot;)
[ Info: Setting will take effect after restarting Julia."><pre><span class="pl-k">using</span> Gtk4
Gtk4<span class="pl-k">.</span><span class="pl-c1">set_EGL_vendorlib_dirs</span>(<span class="pl-s"><span class="pl-pds">"</span>/usr/share/glvnd/egl_vendor.d<span class="pl-pds">"</span></span>)
[ Info<span class="pl-k">:</span> Setting will take effect after restarting Julia.</pre></div>
<p dir="auto">where "/usr/share/glvnd/egl_vendor.d" is a typical location for Mesa's libEGL (this should be modified if it's somewhere else on your distribution). Other vendor-provided libraries may be in other locations, and a colon-separated list of directories can be used for that situation. <strong>Note that this has only been tested for the Mesa-provided libEGL on Fedora and Ubuntu.</strong></p>

<p dir="auto">AlpacaMarkets.jl is in the General Registry so you can simply install locally using Pkg.</p>
<p dir="auto">You need to sign up to AlpacaMarkets to obtain your API keys.
Once you've obtained the key and secret you can authenticate manually with</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using AlpacaMarkets
AlpacaMarkets.auth(KEY, SECRET)"><pre><span class="pl-k">using</span> AlpacaMarkets
AlpacaMarkets<span class="pl-k">.</span><span class="pl-c1">auth</span>(KEY, SECRET)</pre></div>
<p dir="auto">Rather than setting it manually each time I recommend you add it to your <code>.julia/startup.jl</code> or '.julia/config/startup.jl'</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="ENV[&quot;ALPACA_KEY&quot;] = KEY
ENV[&quot;ALPACA_SECRET&quot;] = SECRET"><pre class="notranslate"><code>ENV["ALPACA_KEY"] = KEY
ENV["ALPACA_SECRET"] = SECRET
</code></pre></div>
<p dir="auto">This is automatically be picked up by AlpacaMarkets.jl each time you start Julia.</p>

<p dir="auto">All the historical market data API functions are available:</p>
<ul dir="auto">
<li><code>stock_bars</code>, <code>stock_quotes</code> and <code>stock_trades</code></li>
<li><code>stock_quotes_latest</code> and <code>stock_trades_latest</code></li>
<li><code>crypto_bars</code>, <code>crypto_quotes</code> and <code>crypto_trades</code></li>
</ul>
<p dir="auto">Plus the interface to their news API</p>
<ul dir="auto">
<li><code>news</code></li>
</ul>
<p dir="auto">Plus I've added helper functions to obtain data between two periods:</p>
<ul dir="auto">
<li><code>get_stock_quotes</code> and <code>get_stock_trades</code></li>
<li><code>get_crypto_trades</code></li>
</ul>
<p dir="auto">So you can pull some historical data as and when needed.</p>

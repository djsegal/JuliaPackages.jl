<p dir="auto">J-SPACE is a Julia package to simulate the spatial growth and the genomic evolution  of a cell population and the experiment of sequencing the genome of the sampled cells.
Firstly, the software simulates the spatial dynamics of the cells as a continuous-time multi-type birth-death stochastic process on a graph employing different rules of interaction and an optimised Gillespie algorithm.
After mimicking a spatial sampling of the tumour cells, J-SPACE  returns the phylogenetic tree of the sample and simulates molecular evolution of the genome under the infinite-site models or a set of different substitution models. Ther is also  the possibility of include indels. Finally, employing ART, J-SPACE generates the  synthetic single-end, paired-/mate-pair end reads of the next-generation sequencing platforms.</p>
<h3 dir="auto"><a id="user-content-spatial-clonal-dynamics" class="anchor" aria-hidden="true" href="#spatial-clonal-dynamics"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Spatial clonal dynamics</h3>
<p dir="auto">In J-SPACE the dynamics of the spatio-temporal evolution of a tumour is modelled by a stochastic multi-type Birth-Death process over an
arbitrary graph. J-SPACE could generate by itself a 2D or 3D regular lattice. In addition, it is possible to give as input any graph as an adjacency matrix ( an example of the format needed for this matrix is given in path "Example_adj_matrix", it must is symmetric and with only values(0,1) separeted from space between them).<br>
In this part, the user can tune the birth rate of wild type cells, the death rate of the cells, the rate of migration of cells (not tested), the rule of contact between cells (to simulate different mechanical interactions), the probability to develop a driver mutation per division, and the average birth rate advantage of a driver mutation.
Additionally, there is the possibility to performe an excision by specifying the timing and the ratio of cells that will die of the event associated.
If the user want a specific the clonal dynamics (i.e., <code>Tree_Driver_Configure = 1</code>), it is possible to indicate the edge list representing the mutational tree of drivers and the path where this file is supplied in txt (the parameter of the config file <code>edgelist_treedriver</code> ). In this case the user should also specify the birth rate of each subpopulation and the path where this file is supplied in txt (the parameter of the config file  <code>driver_birth_rates</code>  ). For example, a linear tree with tree drivers is described by the following:</p>
<p dir="auto">Driver_1 Driver_2</p>
<p dir="auto">Driver_2 Driver_3</p>
<p dir="auto">Driver_3 Driver_4</p>
<p dir="auto">The file with the birth rate must have the following format:</p>
<p dir="auto">Driver_1 0.2</p>
<p dir="auto">Driver_2 0.4</p>
<p dir="auto">Driver_3 0.5</p>
<p dir="auto">Driver_4 0.6</p>
<p dir="auto">In this case J-SPACE accept only the events that respect the mutational tree given.</p>
<p dir="auto">Note that every rate inserted in J-SPACE must have the same unit of time both for the spatial dynamics and the molecular evolution.</p>
<h3 dir="auto"><a id="user-content-molecular-evolution" class="anchor" aria-hidden="true" href="#molecular-evolution"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Molecular evolution</h3>
<p dir="auto">J-SPACE simulate the evolution of the sequence of the sample after the simulation of the clonal dynamics. The user can sample the whole population or a subset of it, and the J-SPACE evaluate the phylogenetic tree of the samples. This GT tree is returned as a Newick file in the folder specified by the variable "/path_to_save_files/" .</p>
<p dir="auto">The molecular evolution of an ancestral genome  (which can be given by the user as FASTA file or generated randomly) is simulated along the sampled tree via the Doob-Gillespie algorithm.
The user can use an infinite-site model to have fast simulations of situations where the genome is long, the mutational rate is very low (e.g.,&lt;10^-8 substitution for unit of time per site), and the total simulated time is long.</p>
<p dir="auto">In the case of finite-site models, J-SPACE takes as input the matrix of instantaneous rates for different substitution models: JC69, F81, K80, HKY85, TN93, and K81.
We suppose that the indels have a size distributed as  a Lavalette distribution.</p>
<p dir="auto">The user can also generate a custom time-dependent substitution model based on a linear combination of the Mutational signatures of the COSMIC database. In this case the user should provide the list of labels of the desired SBS signature in the COSMIC database (<a href="https://cancer.sanger.ac.uk/signatures/" rel="nofollow">https://cancer.sanger.ac.uk/signatures/</a>) (e.g.,  <code>used_sign = ["SBS1","SBS4","SBS16"]</code>
), the list of change points (e.g., one change-point at time  50 should be specified <code>vector_change_points = [0.0, 50.0]</code>), the values of the activations for each signature in each of the time span defined by the change-points (e.g., <code>vector_activities = [[0.7,0.2,0.1], [0.0,0.3,0.7]]</code>)
) and the ratio of mutation due to the background uniform process or due to the mutational signatures (e.g., <code>ratio_background_signature = 0.8</code>).
Note that using finite-site for long genomes come at the cost of computational performance.
After this computation, the sequences of the samples (i.e., the leafs of the phylogenetic tree) are returned as FASTA file in the folder  /"path_to_save_files"/Fasta output.</p>
<h3 dir="auto"><a id="user-content-sequencing-experiment" class="anchor" aria-hidden="true" href="#sequencing-experiment"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Sequencing experiment</h3>
<p dir="auto">To simulate the reads of a sequencing experiment J-SPACE calls ART (<a href="https://www.niehs.nih.gov/research/resources/software/biostatistics/art/index.cfm" rel="nofollow">https://www.niehs.nih.gov/research/resources/software/biostatistics/art/index.cfm</a>). The user can use a configuration file to specify the error model (for Illumina platforms), the number of reads, the length of the reads, and if the experiment uses single-end paired-end/mate-pair reads.
In addition, in the configuration file, there is the option to insert custom "calls" for ART  with the possibility to use it in any possible configuration.
If the user simulate the experiment, J-SPACE returns for each cell, the simulated reads as FASTQ file, the alignment map of the reads over the genome of the sampled cells in SAM and/or ALN format.
If the infinite-site model is used,  it is possible to obtain the VCF file directly without simulating the reads with ART.</p>

<p dir="auto">First move where is file <code>environment_j_space.yml</code></p>
<p dir="auto">Change field name and prefix into <code>environment_j_space.yml</code></p>
<p dir="auto">Open the conda environment</p>
<blockquote>
<p dir="auto">conda env create -f environment_j_space.yml --prefix "path to the enviroment directory"</p>
</blockquote>
<p dir="auto">Activate the conda environment</p>
<blockquote>
<p dir="auto">conda activate "path to the enviroment directory"</p>
</blockquote>
<p dir="auto">Then register to gatk (Not necessary in the same working folder)</p>
<blockquote>
<p dir="auto">gatk3-register "path to the gatk directory"</p>
</blockquote>
<p dir="auto">Move into working folder where you have <code>j_space_pipeline.sh</code>
Run pipeline</p>
<blockquote>
<p dir="auto">./j_space_pipeline.sh "path/to/reference/" "path/to/FastaQ" "path/working/directory"</p>
</blockquote>
<ul dir="auto">
<li>the first, "path/to/reference/", indicate where is the reference fasta, must specificated file.fasta</li>
<li>the second, "path/to/FastaQ", indicate where are the file FastaQ to analyzed, must specificated the folder</li>
<li>the third, "path/working/directory", indicate your working directory, where will save output files</li>
</ul>
<p dir="auto">NOTE: the paths are absolute paths</p>

<p dir="auto">J-SPACE provides the following outputs.</p>
<ul dir="auto">
<li>The state of the lattice at any time of the simulation (as plot).</li>
<li>The Ground Truth (GT) genotype of the sampled cells as FASTA files.</li>
<li>The GT phylogenetic  tree  of the sampled cells in Newick format.</li>
<li>The plot of the mutational tree of the driver mutations (if present).</li>
<li>The list of the driver mutation with their birth rate advantage as .csv (if present).</li>
<li>The simulated next-generation sequencing reads as FASTQ files in the folder "/"path_to_save_files"/Fasta output/sample_#", where sample_# is the #-th sample.</li>
<li>The alignment file, which maps the noisy reads on the sequences of the sampled cells both in formats SAM and ALN.</li>
<li>The alignment file, without noise in format SAM.</li>
<li>The list of the mutations for each sample as csv.</li>
</ul>

<p dir="auto">In the file "Config.toml" the user can manage the configuration of J-SPACE. This file is useful to choose the path where save the files, the desired plots and output files.
We provide an example in the main folder of J-SPACE.</p>
<p dir="auto">The following are the paramenters of such file:</p>
<ul dir="auto">
<li><code>seed</code>. Integer, the seed of the simulation.</li>
<li><code>generate_reference</code>. Integer, If 0 the user should inser the ancestral genome in fasta format in the folder "path_reference". If 1 J-SPACE generate a random sequence.</li>
<li><code>path_reference</code>. A string,  the path of the reference given by user.</li>
<li><code>path_to_save_files</code>.  A string, path of the folder where the output files should be saved.</li>
<li><code>path_to_save_plot</code>. A string,  path of the folder where the output plots should be saved.</li>
<li><code>Generate_graph</code>. Integer, if 0 the user should inser the graph of the dynamics as an adjacency matrix. If 1 J-SPACE generate a regular lattice, the paramenters of such lattice are specified in the file "Parameters.toml".</li>
<li><code>Path_to_Graph</code>.  Path to the adjacency matrix of the graph if imported.</li>
<li><code>Tree_Driver_Configure</code>. Integer number. if 1 J-SPACE takes as input a driver mutational tree. With 0 J-SPACE generates the tree randomly.</li>
<li><code>Tree_Newick</code>. Integer, if 1  the phylogenentic tree of the cells is saved as newick file.</li>
<li><code>Final_configuration</code>. Integer,  if 1 return the plot of the final configuration of the lattice.</li>
<li><code>Driver_list</code>. Integer,  if 1 returns the list of the driver mutations.</li>
<li><code>Driver_Tree</code>. Integer, if 1 returns the plot tree of the driver mutations.</li>
<li><code>Single_cell_fasta</code>. Integer, if 1 save the fasta of the GT sequences of the sampled cells.</li>
<li><code>Single_cell_noise</code>. Integer, if 0  the sequencing experiment is not performed, if 1 it returns the FASTQ  files of the reads and SAM files with noise  , if 2 it returns the FASTQ  files of the reads, the SAM files with noise and without noise.</li>
<li><code>Alignment</code>. Integer, if 1 it returns the GT aligment file in ALN format.</li>
<li><code>VAF_GT</code>. Integer, if 1 it returns the VAF of the sampled cells  (working only if <code>type_isa = 1</code>).</li>
<li><code>Bulk_noise</code>. Integer, if 1 it returns an approximate bulk experiment not using ART (working only if <code>type_isa = 1</code>).</li>
<li><code>Time_of_sampling</code>. Array of times in an incresing order (e.g., [10.0 , 20.3 , 50.2]). J-SPACE perform the plot of the state of the lattice in that times.</li>
<li><code>Graph_configuration</code>. Integer,  if 1  returns the plot of the state of the lattice.</li>
</ul>

if <code>sub_model= "JC69" </code>-&gt; <code>params = [{"alpha" = 0.5}]</code>.<br>
if <code>sub_model= "F81" </code> -&gt; <code>params = [{"alpha" = 0.5}]</code>.<br>
if <code>sub_models= "K80"  </code>-&gt; <code>params = [{"alpha" = 0.5, "beta" = 0.3}]</code>.<br>
if <code>sub_models= "HKY" </code> -&gt; <code>params = [{"alpha" = 0.5, "beta" = 0.3}]</code>.<br>
if <code>sub_models = "TrN93ef"</code>  -&gt; <code>params = [{"alpha" = 0.5, "alpha2"=0.1,"beta" = 0.3}]</code>.<br>
if <code>sub_models = "TrN" </code>-&gt; <code>params = [{"alpha" = 0.5, "alpha2"=0.1,"beta" = 0.3}]</code>.<br>
if <code>sub_models = "K81" </code> -&gt; <code>params = [{"alpha" = 0.5, "beta"=0.1,"beta2" = 0.3}]</code>.<br>
if <code>sub_models = "K81uf"</code>  -&gt; <code>params = [{"alpha" = 0.5, "beta"=0.1,"beta2" = 0.3}]</code>.</li>
</ul>

<ul dir="auto">
<li><code>mut_rate_avg</code>. A real number, the average mutational rate per trinucleotide and unit of time.</li>
<li><code>used_sign</code>. An array of strings. The list of the labels of the used signatures (e.g., <code>used_sign = ["SBS1","SBS4","SBS16"] </code>)</li>
<li><code>vector_change_points</code>. An array of real number . It contains the list of the time in the change points (e.g., one change-point at time  50 should be specified</li>
<li>-<code>vector_change_points = [0.0, 50.0]</code>). If no change point is desired (constant signature activity) one should use <code>vector_change_points = [0.0]</code></li>
<li><code>vector_activities</code>. A vector of vectors. For each elements of <code>vector_change_points</code> it is necessary insert an array with the values of the activities for each signature (e.g., if <code>vector_change_points</code> has two elements <code>vector_activities = [[0.7,0.2,0.1], [0.0,0.3,0.7]] </code>)</li>
<li><code>ratio_background_signature</code>. A real number between 1.0 and 0.0. If 1.0 all mutation will be due to mutational signatures, if 0 all ther mutations will be due to the background  process.</li>
</ul>

<p dir="auto">Write tabular data from Julia in LaTeX format.</p>
<p dir="auto">This is a <em>very thin wrapper</em>, basically for avoiding some loops and repeatedly used strings. It assumes that you know how the LaTeX <code>tabular</code> environment works, and you have formatted the cells to strings if you want anything fancy like rounding or alignment on the decimal dot.</p>
<p dir="auto">This is how it works:</p>

<p dir="auto">to <code>/tmp/table.tex</code>.</p>
<p dir="auto">Note that the position specifier <code>lcl</code> is not checked for valid syntax or consitency with the contents, just emitted as is, allowing the use of <a href="https://ctan.org/pkg/dcolumn" rel="nofollow">dcolumn</a> or similar, and the number of cells in each line is not checked for consistency. This means that the usual LaTeX rules apply: fewer cells than position specifiers gives you a ragged table, more cells and LaTeX will complain about having to change <code>&amp;</code> to <code>\\</code>.</p>
<p dir="auto">See <code>?latex_tabular</code> for the documentation of the syntax, and the unit tests for examples.</p>
<p dir="auto">Rule types in <a href="https://ctan.org/pkg/booktabs" rel="nofollow">booktabs</a> are supported. Vertical rules of any kind are <em>not explicitly supported</em> and it would be difficult to convince me to add them. The documentation of <a href="https://ctan.org/pkg/booktabs" rel="nofollow">booktabs</a> should explain why. That said, if you insist, you can use a cell like <code>\vline text</code>.</p>
<p dir="auto">The other tabular type currently implemented is <code>LongTable</code>. The code is generic, so <a href="https://en.wikibooks.org/wiki/LaTeX/Tables" rel="nofollow">other tabular-like types</a> can be easily added, just open an issue.</p>

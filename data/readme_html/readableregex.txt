<h2><a id="user-content-a-package-for-people-who-dont-want-to-learn-or-read-regexes" class="anchor" aria-hidden="true" href="#a-package-for-people-who-dont-want-to-learn-or-read-regexes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>A package for people who don't want to learn or read regexes</h2>
<p>ReadableRegex.jl gives you a syntax that is much easier to write and understand
than the rather cryptic standard Regex. The syntax is as close as possible to
a natural language description of the Regex.
Especially for nested and grouped expressions with special characters it is orders of magnitudes easier to understand than
a simple regex string. You can also compile the function expression before using it with the <code>@compile</code> macro for lower runtime costs.</p>

<p>Both of these match all kinds of floating point numbers like these:</p>
<p><code>"1 2.0 .3 -.4 -5 60 700 800.9 +9000"</code></p>
<p>But to understand the Regex you have to mentally parse it into the second version.
Why not avoid that effort?</p>
<p>ReadableRegex also escapes reserved regex characters like "." by default, saving you a lot of backslashes.</p>
<h2><a id="user-content-how-does-it-work" class="anchor" aria-hidden="true" href="#how-does-it-work"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>How does it work?</h2>
<p>ReadableRegex uses a simple intermediary type <code>RegexString</code> which holds a string
representation of a regex, the same thing you'd normally write yourself.</p>
<p>These <code>RegexString</code>s can be concatenated together with the <code>*</code> operator.
You can convert them into a <code>Regex</code> by calling <code>Regex(rs::RegexString)</code>,
or use them directly with <code>match</code> and <code>eachmatch</code>.</p>
<p>Let's say we wanted to extract all the possible integers from the list above. One way could be this:</p>

<p>If you need to, you can create a <code>RegexString</code> using its constructor with a properly escaped string.</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="r = RegexString(&quot;\\w&quot;)
"><pre>r <span class="pl-k">=</span> <span class="pl-c1">RegexString</span>(<span class="pl-s"><span class="pl-pds">"</span><span class="pl-cce">\\</span>w<span class="pl-pds">"</span></span>)</pre></div>
<p>Alternatively, there is the <code>@rs_str</code> macro which allows you to use normal regex syntax, without special escaping rules.</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="r = rs&quot;\w&quot;
"><pre>r <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds"><span class="pl-c1">rs</span>"</span><span class="pl-cce">\w</span><span class="pl-pds">"</span></span></pre></div>
<p>This can also be useful if you want to start a chain of concatenated parts where the multiplication operator needs one of the two first elements to be a <code>RegexString</code>.</p>

<p>Some constructs from Base Julia are useful to express building blocks of regular expression.
You can define <code>Base.convert(::Type{RegexString}, obj)</code> to use these directly in ReadableRegex expressions.
All building block functions call <code>convert</code> on their inputs, and so does the multiplication operator <code>*</code>.</p>
<p>Some predefined examples:</p>

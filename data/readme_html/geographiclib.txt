<p>GeographicLib allows you to accurately compute properties of
<a href="https://en.wikipedia.org/wiki/Geodesic" rel="nofollow">geodesics</a> on a flattened sphere (e.g., Earth).</p>

<h2><a id="user-content-testing" class="anchor" aria-hidden="true" href="#testing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Testing</h2>
<p>To perform some of its tests, the package relies on the Python port of GeographicLib.
Hence if you do <code>pkg&gt; test</code>, PyCall will attempt to import <code>geographiclib</code> and
install it if possible.</p>
<p>You may also run <code>julia script/test_GeodTest.jl</code> which compares the output of this
package and the routines in <code>libproj</code> to a set of test results computed with
high precision.  For this purpose, you need to have
<a href="https://proj.org/install.html" rel="nofollow">Proj installed</a> and will need to edit the script
to point to the location of the <code>libproj</code> library.</p>
<p>Neither of these tests are necessary to use GeographicLib.jl normally.</p>

<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview</h2>
<p><em>Docile</em> is a <a href="http://www.julialang.org" rel="nofollow">Julia</a> package documentation system.</p>
<p>It provides a macro, <code>@doc</code>, for documenting arbitrary Julia objects. This
functionality is equivalent to the new <code>@doc</code> macro available from <code>Base</code> in
Julia <code>0.4</code>.</p>
<p>Additionally, <em>Docile</em> has support for using plain strings for documentation
rather than using the <code>@doc</code> macro. See the package documentation for further
details.</p>

<p>Package documentation is available for the <a href="http://docilejl.readthedocs.org/en/stable/" rel="nofollow">stable</a> and
<a href="http://docilejl.readthedocs.org/en/latest/" rel="nofollow">development</a> versions.</p>
<p>To view documentation from the REPL and generate external documentation for a
package please install the package <a href="https://github.com/MichaelHatherly/Lexicon.jl"><em>Lexicon</em></a>.</p>

<p dir="auto"><a href="https://github.com/JuliaAI/MLJEnsembles.jl/actions"><img src="https://github.com/JuliaAI/MLJEnsembles.jl/workflows/CI/badge.svg" alt="Build status" style="max-width: 100%;"></a> <a href="http://codecov.io/github/JuliaAI/MLJEnsembles.jl?branch=master" rel="nofollow"><img src="https://camo.githubusercontent.com/70b44557a3d19e8ab69d61bf32cf687def74308ac5e76916a18f5ad0a866ff67/687474703a2f2f636f6465636f762e696f2f6769746875622f4a756c696141492f4d4c4a456e73656d626c65732e6a6c2f636f7665726167652e7376673f6272616e63683d6d6173746572" alt="codecov.io" data-canonical-src="http://codecov.io/github/JuliaAI/MLJEnsembles.jl/coverage.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">A package to create bagged homogeneous ensembles of
machine learning models using the
<a href="https://alan-turing-institute.github.io/MLJ.jl/dev/" rel="nofollow">MLJ</a> machine
learning framework.</p>
<p dir="auto">For combining models in more general ways, see the <a href="https://alan-turing-institute.github.io/MLJ.jl/dev/composing_models/#Composing-Models" rel="nofollow">Composing
Models</a>
section of the MLJ manual.</p>

<p dir="auto">No installation is necessary when using MLJ, which is installed like this:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Pkg
Pkg.add(&quot;MLJ&quot;)
using MLJ"><pre><span class="pl-k">using</span> Pkg
Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>MLJ<span class="pl-pds">"</span></span>)
<span class="pl-k">using</span> MLJ</pre></div>
<p dir="auto">Alternatively, for a "minimal" installation:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Pkg
Pkg.add(&quot;MLJBase&quot;)
Pkg.add(&quot;MLJEnsembles&quot;)
using MLJBase, MLJEnsembles"><pre><span class="pl-k">using</span> Pkg
Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>MLJBase<span class="pl-pds">"</span></span>)
Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>MLJEnsembles<span class="pl-pds">"</span></span>)
<span class="pl-k">using</span> MLJBase, MLJEnsembles</pre></div>
<p dir="auto">In this case you will also need to load code defining an atomic model
to ensemble. The easiest way to do this is run <code>Pkg.add("MLJModels"); using MLJModels</code> and use the <code>@load</code> macro.  See the <a href="https://alan-turing-institute.github.io/MLJ.jl/dev/loading_model_code/" rel="nofollow">Loading Model
Code</a>
of the MLJ manual for this and other possibilities.</p>

<h1 dir="auto"><a id="user-content-meshcorejl" class="anchor" aria-hidden="true" href="#meshcorejl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>MeshCore.jl</h1>
<p dir="auto">Small package for operating on the topology of meshes for the Finite Element
Methods (FEM). All essential topological incidence relations are provided: see
the guide in <a href="https://petrkryslucsd.github.io/MeshCore.jl/dev" rel="nofollow">documentation</a>.
The library provides efficient storage in static arrays for speed of access.</p>

<p dir="auto">This release is for Julia 1.8.</p>
<p dir="auto">The package is registered: doing</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="]add MeshCore
using MeshCore"><pre class="notranslate"><code>]add MeshCore
using MeshCore
</code></pre></div>
<p dir="auto">is enough.</p>
<p dir="auto">The user can either use/import individual functions from <code>MeshCore</code> like so:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using MeshCore: nrelations, nentities"><pre class="notranslate"><code>using MeshCore: nrelations, nentities
</code></pre></div>
<p dir="auto">or all exported symbols maybe made available in the user's context as</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using MeshCore.Exports"><pre class="notranslate"><code>using MeshCore.Exports
</code></pre></div>
<p dir="auto">Using the library by itself is certainly possible.
If you wish for a more comprehensive mesh-handling package, check out
<a href="https://github.com/PetrKryslUCSD/MeshSteward.jl">MeshSteward.jl</a>.</p>

<p dir="auto">A paper was accepted to the Journal Advances in Engineering Software in April 2021. The paper is available in final draft form in the <code>docs/src/concepts</code> folder.</p>
<p dir="auto">The package
<a href="https://github.com/PetrKryslUCSD/PaperMeshTopo.jl.git">PaperMeshTopo</a> gives an
example of the construction of a complex (full one-level) topological
representation of a tetrahedral mesh.</p>

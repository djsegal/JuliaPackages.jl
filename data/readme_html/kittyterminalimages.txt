<p dir="auto">KittyTerminalImages.jl can be installed from the package manager by typing the following to your REPL:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="] add KittyTerminalImages"><pre>] add KittyTerminalImages</pre></div>
<p dir="auto">In addition you will need install the Kitty terminal emulator. It works on macOS or Linux but (at least currently) not on Windows. You can get it here: <a href="https://github.com/kovidgoyal/kitty">https://github.com/kovidgoyal/kitty</a></p>
<h2 dir="auto"><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Usage</h2>
<p dir="auto">Simple load KittyTerminalImages in your REPL (assuming that you have opened Julia in kitty)</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using KittyTerminalImages"><pre><span class="pl-k">using</span> KittyTerminalImages</pre></div>
<p dir="auto">Sometimes when loading another package, such as Plots.jl or Gadfly.jl, they will put their own method for displaying images
in an external program on the display stack, thus overriding the behaviour of KittyTerminalImages. To put KittyTerminalImages back
on top of the stack, execute the following:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pushKittyDisplay!()"><pre><span class="pl-c1">pushKittyDisplay!</span>()</pre></div>
<p dir="auto">You can also override the general display behaviour so KittyTerminalImages is on top of the stack. This is a hack and can have
unwanted side effects. To do that, execute the following:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="forceKittyDisplay!()"><pre><span class="pl-c1">forceKittyDisplay!</span>()</pre></div>
<h3 dir="auto"><a id="user-content-makie" class="anchor" aria-hidden="true" href="#makie"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Makie</h3>
<p dir="auto">It is possible to use <a href="https://github.com/JuliaPlots/Makie.jl">Makie.jl</a> together with KittyTerminalImages although there is no interactivity nor any animations.
After loading a Makie backend it might be necessary to run <code>AbstractPlotting.inline!(true)</code> so that the plots are shown in the terminal instead
of an external window. In my experience that is necessary for the <a href="https://github.com/JuliaPlots/GLMakie.jl">GLMakie.jl</a> backend
but not for the <a href="https://github.com/JuliaPlots/CairoMakie.jl">CairoMakie.jl</a> backend.</p>
<h3 dir="auto"><a id="user-content-running-scripts" class="anchor" aria-hidden="true" href="#running-scripts"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Running scripts</h3>
<p dir="auto">Beware that KittyTerminalImages.jl relies on the <code>display</code> function to draw the plots, and this is <em>not</em> called automatically
when you are writing a script, unless the plotting command is the very last command in the script. If you want to force the
display of a plot, you can wrap the call with <code>display</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="# This is a script named &quot;test.jl&quot;

using Plots
using KittyTerminalImages

println(&quot;Here is the first plot:&quot;)
# We call &quot;display&quot; on plot's return value
plot(sin, 0:0.1:2π) |&gt; display
println() # Force a newline

# Another plot
println(&quot;Here is another plot:&quot;)
plot(exp, 0:0.1:1)
plot!(log, 0.1:0.1:1) |&gt; display  # Only call &quot;display&quot; on the last &quot;plot!&quot;
println() # Force a newline

println(&quot;End of the script&quot;)"><pre><span class="pl-c"><span class="pl-c">#</span> This is a script named "test.jl"</span>

<span class="pl-k">using</span> Plots
<span class="pl-k">using</span> KittyTerminalImages

<span class="pl-c1">println</span>(<span class="pl-s"><span class="pl-pds">"</span>Here is the first plot:<span class="pl-pds">"</span></span>)
<span class="pl-c"><span class="pl-c">#</span> We call "display" on plot's return value</span>
<span class="pl-c1">plot</span>(sin, <span class="pl-c1">0</span><span class="pl-k">:</span><span class="pl-c1">0.1</span><span class="pl-k">:</span><span class="pl-c1">2</span>π) <span class="pl-k">|&gt;</span> display
<span class="pl-c1">println</span>() <span class="pl-c"><span class="pl-c">#</span> Force a newline</span>

<span class="pl-c"><span class="pl-c">#</span> Another plot</span>
<span class="pl-c1">println</span>(<span class="pl-s"><span class="pl-pds">"</span>Here is another plot:<span class="pl-pds">"</span></span>)
<span class="pl-c1">plot</span>(exp, <span class="pl-c1">0</span><span class="pl-k">:</span><span class="pl-c1">0.1</span><span class="pl-k">:</span><span class="pl-c1">1</span>)
<span class="pl-c1">plot!</span>(log, <span class="pl-c1">0.1</span><span class="pl-k">:</span><span class="pl-c1">0.1</span><span class="pl-k">:</span><span class="pl-c1">1</span>) <span class="pl-k">|&gt;</span> display  <span class="pl-c"><span class="pl-c">#</span> Only call "display" on the last "plot!"</span>
<span class="pl-c1">println</span>() <span class="pl-c"><span class="pl-c">#</span> Force a newline</span>

<p dir="auto">The calls to <code>display</code> will make the plot appear when you run the script calling <code>julia test.jl</code> from the shell or writing
<code>include("test.jl")</code> in the REPL.</p>

<p dir="auto">If the images are too small or too large for your terminal,
you can specify a scale</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="set_kitty_config!(:scale, 2.0) # scale all images by a factor 2"><pre><span class="pl-c1">set_kitty_config!</span>(<span class="pl-c1">:scale</span>, <span class="pl-c1">2.0</span>) <span class="pl-c"><span class="pl-c">#</span> scale all images by a factor 2</span></pre></div>
<h3 dir="auto"><a id="user-content-setting-preference-for-png-or-svg" class="anchor" aria-hidden="true" href="#setting-preference-for-png-or-svg"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Setting preference for PNG or SVG</h3>
<p dir="auto">Certain Julia objects can be drawn as either a PNG or a SVG image. In case both formats are possible,
one can specify that PNG should be preferred by setting the <code>:prefer_png_to_svg</code> config value:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="set_kitty_config!(:prefer_png_to_svg, true)"><pre><span class="pl-c1">set_kitty_config!</span>(<span class="pl-c1">:prefer_png_to_svg</span>, <span class="pl-c1">true</span>)</pre></div>
<p dir="auto">At the moment this value is set to <code>true</code> by default as in some cases the svg renderer creates some incorrect images.
If the <code>:scale</code> config is also set, this has the disadvantage that scaled images may appear blurry.</p>

<p dir="auto">To transfer the image from Julia to kitty, one can select between two transfer modes:</p>
<ul dir="auto">
<li><code>:direct</code> (default) -- transfer the imagine with escape sequences</li>
<li><code>:temp_file</code> -- transfer the imagine by writing it to a temporary file and then transfer only the path of that image</li>
</ul>
<p dir="auto">Only <code>:direct</code> works if Julia is accessed remotely with SSH but if Julia is on the same machine as kitty then one might switch to <code>:temp_file</code> which might be slightly faster.
To switch the mode one can do</p>

<p dir="auto"><a href="https://travis-ci.org/JuliaParallel/Elly.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/e2b2affdce29da9519de49df320794ab4fcc400bee18ba5f374aba82834bd530/68747470733a2f2f7472617669732d63692e6f72672f4a756c6961506172616c6c656c2f456c6c792e6a6c2e706e67" alt="Build Status" data-canonical-src="https://travis-ci.org/JuliaParallel/Elly.jl.png" style="max-width: 100%;"></a></p>
<p dir="auto">Elly is a <a href="https://hadoop.apache.org/" rel="nofollow">Hadoop</a> HDFS and Yarn client. It provides:</p>
<ul dir="auto">
<li>A familiar Julia <code>ClusterManager</code> interface, making it possible to
use the familiar Julia parallel constructs on any Hadoop/Yarn
cluster: <code>addprocs</code>, <code>@parallel</code>, <code>spawn</code>, <code>pmap</code>, etc.</li>
<li>Lower level APIs to write native Yarn applications.</li>
<li>A familiar Julia IO API for accessing HDFS files. It is a pure Julia
implementation with no dependencies on <code>libhdfs</code>.</li>
</ul>

<h5 dir="auto"><a id="user-content-motivation" class="anchor" aria-hidden="true" href="#motivation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>motivation</h5>
<ul dir="auto">
<li>
<p dir="auto"><em>Every day should have all the times-of-day</em> (except where timezone changes apply)</p>
</li>
<li>
<p dir="auto">The <code>Time</code> type from Dates supports nanoseconds when dates are not used.</p>
</li>
<li>
<p dir="auto"><code>Time</code> works incompletely with<code>DateTime</code> which is limited to milliseconds.</p>
</li>
<li>
<p dir="auto">We need date-and-time together, fully resolved, accurate and precise.</p>
<ul dir="auto">
<li><code>NanoDate</code> is familiar and effective
<ul dir="auto">
<li>works well with Dates</li>
<li>uses <code>DateTime</code> methods</li>
</ul>
</li>
</ul>
</li>
<li>
<p dir="auto">This package is a redesign of <code>TimesDates</code></p>
<ul dir="auto">
<li><code>TimesDates</code> is widely used and well-liked
<ul dir="auto">
<li>offers nanoseond resolved dates       (as do we)</li>
<li>offers nanosecond accurate time zones (we do not)</li>
</ul>
</li>
</ul>
</li>
<li>
<p dir="auto">Contributors are welcome</p>
<ul dir="auto">
<li>clear code, robust performance, reliable interoperability</li>
<li>frendly docs, easily followed, well explained</li>
</ul>
</li>
</ul>

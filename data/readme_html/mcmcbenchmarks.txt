<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Introduction</h2>
<p>MCMCBenchmarks provides a lightweight yet flexible framework for benchmarking MCMC samplers in terms of runtime, memory usage, convergence metrics and effective sample size. Currently, MCMCBenchmarks provides out of the box support for benchmarking the <a href="http://jmlr.org/papers/volume15/hoffman14a/hoffman14a.pdf" rel="nofollow">No-U-Turn Sampler</a> (NUTS) algorithm as implemented in <a href="https://github.com/StanJulia/CmdStan.jl">CmdStan</a>, <a href="https://github.com/tpapp/DynamicHMC.jl">DynamicHMC</a> and <a href="https://github.com/TuringLang/AdvancedHMC.jl">AdvancedHMC</a> via <a href="https://github.com/TuringLang/Turing.jl">Turing</a>. However, methods can be extended to accommodate other samplers and test models.</p>

<ul>
<li>Benchmarking Parameters: vary factors such as sample size, data-generating parameters, prior distributions, and target acceptance rate. The use of optional keywords allows other benchmarking parameters to be varied.</li>
<li>Meta-data: saves relevant benchmarking information regarding package version, Julia version and system specs.</li>
<li>Plotting: generate and save plots comparing samplers in terms of run time, memory usage, convergence diagnostics and effective sample size.</li>
</ul>

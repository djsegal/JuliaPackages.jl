<p>This is the documentation for the Juno IDE, hosted at <a href="http://docs.junolab.org" rel="nofollow">http://docs.junolab.org</a>.

<h2><a id="user-content-contributing-to-the-documentation" class="anchor" aria-hidden="true" href="#contributing-to-the-documentation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Contributing to the Documentation</h2>
<p>In order to contribute to the documentation, simply modify the files in the <code>/docs/src</code> folder
and open a PR against the <code>master</code> branch.
Once a PR has been opened, the CI will automatically run Documenter.jl and generate a new documentation against the PR.
You can see the generate documentation on <code>http://docs.junolab.org/previews/PR##</code>, where <code>##</code> is the PR number.</p>

<p>Although it's not necessary, if you want to check how the new documentation pages would look like in advance of opening a PR,
you can build documentations locally and check the results generated in <code>docs/build</code> by the following step:</p>
<blockquote>
<p>from the root of this repository:</p>
</blockquote>
<div class="highlight highlight-source-shell"><pre>$ julia --project=docs/ -e <span class="pl-s"><span class="pl-pds">'</span>using Pkg; Pkg.develop(PackageSpec(path = pwd())); Pkg.instantiate()<span class="pl-pds">'</span></span>
$ julia --project=docs/ docs/make.jl</pre></div>

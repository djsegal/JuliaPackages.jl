<p dir="auto">All JuliaRun APIs require a context to operate.  A JuliaRun client context consists of:</p>
<ul dir="auto">
<li>URL of the JuliaRun remote server</li>
<li>an authentication token</li>
<li>namespace to operate in</li>
</ul>
<p dir="auto">It can be created simply as:</p>
<p dir="auto"><code>ctx = Context()</code></p>
<p dir="auto">Default values of all parameters are set to match those inside a JuliaRun cluster.</p>
<ul dir="auto">
<li>connects to a service endpoint at "juliarunremote-svc.juliarun"</li>
<li>reads the namespace from the default secret</li>
<li>presents the namespace service token (also read from the default secret) for authentication</li>
</ul>
<p dir="auto">Pass appropriate values for each when calling from outside the cluster.</p>
<p dir="auto">Apart from the context, some APIs also need a job to work with. A job in JuliaRunClient is an instance of one of the <a href="JobTypes.md">Job types</a> created by passing the job name. The other required and optional parameters of a job type are needed only need to be passed to the job creation API.</p>

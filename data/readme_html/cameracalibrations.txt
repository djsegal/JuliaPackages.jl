<p>First we build the calibration object based on the checker size, the extrinsic image, and intrinsic images:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using CameraCalibrations
c = buildcalibration(3, extrinsic, intrinsic)
"><pre><span class="pl-k">using</span> CameraCalibrations
c <span class="pl-k">=</span> <span class="pl-c1">buildcalibration</span>(<span class="pl-c1">3</span>, extrinsic, intrinsic)</pre></div>
<p>Then we can use that object to calibrate pixel coordinates and/or images:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="xy = calibrate(c, SVector(1.1, 2.2))

<h2><a id="user-content-use-this" class="anchor" aria-hidden="true" href="#use-this"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>use this</h2>
<ol>
<li>This is currently (26/10/2020) the only package for camera calibration in Julia.</li>
<li>It allows for calibration using only one (extrinsic) image as well as both extrinsic and intrinsic images.</li>
<li>The calibration objects are native Julia objects (an object from <code>CoordinateTransformations</code> or <code>Interpolations</code>), so you can save them.</li>
</ol>
<h2><a id="user-content-not-use-this" class="anchor" aria-hidden="true" href="#not-use-this"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>not use this</h2>
<ol>
<li>It's just a poor wrapper around Matlab's camera calibration toolbox (so without it you can't use this).</li>
<li>You can't supply it with in-memory images, just file names.</li>
<li>Error handling within Matlab is suboptimal.</li>
<li>It's not optimized for speed or anything really.</li>
</ol>

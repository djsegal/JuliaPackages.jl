<p>A module for the Julia language that implements several varieties of the dependent statistical bootstrap as well as the corresponding block-length selection procedures.</p>

<p>This package is compatible with julia v1.0+. If you are running v0.6, you will need to use <code>Pkg.pin("DependentBootstrap", v"0.1.1")</code> at the REPL, and if you are running v0.5, use <code>Pkg.pin("DependentBootstrap", v"0.0.1")</code>. Compability with versions before v0.5 is not available.</p>
<h2><a id="user-content-main-features" class="anchor" aria-hidden="true" href="#main-features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Main features</h2>
<p>This module allows Julia users to estimate the distribution of a test statistic using any of several varieties of the dependent bootstrap.</p>
<p>The following bootstrap methods are implemented:</p>
<ul>
<li>the <em>iid</em> bootstrap proposed in Efron (1979) "Bootstrap Methods: Another Look at the Jackknife",</li>
<li>the stationary bootstrap proposed in Politis, Romano (1994) "The Stationary Bootstrap"</li>
<li>the moving block bootstrap proposed in Kunsch (1989) "The jackknife and the bootstrap for general stationary observations" and (independently) Liu, Singh (1992) "Moving blocks jackknife and bootstrap capture weak dependence",</li>
<li>the circular block bootstrap proposed in Politis, Romano (1992) "A circular block resampling procedure for stationary data", and</li>
<li>the non-overlapping block bootstrap described in Lahiri (1999) <em>Resampling Methods for Dependent Data</em> (this method is not usually used and is included mainly as a curiosity).</li>
</ul>
<p>The module also implements the following block length selection procedures:</p>
<ul>
<li>the block length selection procedure proposed in Politis, White (2004) "Automatic Block Length Selection For The Dependent Bootstrap", including the correction provided in Patton, Politis, and White (2009)</li>
</ul>
<p>Bandwidth selection for the block length procedures is implemented using the method proposed in Politis (2003) "Adaptive Bandwidth Choice".</p>
<p>Some work has been done to implement the tapered block bootstrap of Paparoditis, Politis (2002) "The tapered block bootstrap for general statistics from stationary sequences", along with corresponding block-length selection procedures, but it is not yet complete.</p>
<p>The module is implemented entirely in Julia.</p>
<h2><a id="user-content-what-this-package-does-not-include" class="anchor" aria-hidden="true" href="#what-this-package-does-not-include"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>What this package does not include</h2>
<p>I have not included any procedures for bootstrapping confidence intervals in a linear regression framework, or other parametric models. This functionality is provided by <a href="https://github.com/juliangehring/Bootstrap.jl">Bootstrap.jl</a>, and work is currently under way to add bootstrap methods from this package to the <code>Bootstrap</code> API.</p>
<p>I also have not included support for the jackknife, wild bootstrap, or subsampling procedures. I would be quite open to pull requests that add these methods to the present package, but have not had time to implement them myself. Work is ongoing to include the tapered block bootstrap, and ideally, the package will also eventually include the extended tapered block bootstrap. If you are interested in working on any of these projects, please feel free to contact me.</p>
<h2><a id="user-content-how-to-use-this-package" class="anchor" aria-hidden="true" href="#how-to-use-this-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How to use this package</h2>
<h4><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h4>
<p>This package should be added using <code>using Pkg ; Pkg.add("DependentBootstrap")</code>, and can then be called with <code>using DependentBootstrap</code>. The package depends on <code>StatsBase</code> and <code>Distributions</code> for some functionality, and on <code>DataFrames</code> and <code>TimeSeries</code> so that <code>DataFrame</code> and <code>TimeArray</code> datasets can be supported by this packages methods.</p>
<h4><a id="user-content-terminology" class="anchor" aria-hidden="true" href="#terminology"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Terminology</h4>
<p>In what follows, I use the terminology from Lahiri (1999) <em>Resampling Methods for Dependent Data</em> and refer to the underlying test statistic of interest as a <em>level 1 statistic</em>, and the distribution parameter of the test statistic that is of interest as a <em>level 2 parameter</em>. For example, the user might have some dataset <code>x</code> of type <code>T_data</code>, and be interested in the variance of the sample mean of <code>x</code>. In this case, the level 1 statistic is the sample mean function <code>mean</code>, and the level 2 parameter is the sample variance function <code>var</code>.</p>
<p>I use <code>T_data</code> to refer to the type of the users dataset, <code>T_level1</code> to refer to the output type obtained by applying the level 1 statistic function to the dataset, and <code>T_level2</code> to refer to the output type obtained by applying the level 2 statistic to a <code>Vector{T_level1}</code> (i.e. a vector of resampled level 1 statistics).</p>
<h4><a id="user-content-exported-functions" class="anchor" aria-hidden="true" href="#exported-functions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Exported functions</h4>
<p>The package exports the following functions, all of which have docstrings that can be called interactively at the REPL:</p>
<ul>
<li><code>dbootinds(...)::Vector{Vector{Int}}</code> -&gt; Returns indices that can be used to index into the underlying data to obtain bootstrapped data. Note, each inner vector of the output corresponds to a single re-sample for the underlying data.</li>
<li><code>dbootdata(...)::Vector{T_data}</code> -&gt; Returns the bootstrapped data. Each element of the output vector corresponds to one re-sampled dataset, and the output vector will have length equal to <code>numresample</code> (a parameter discussed later).</li>
<li><code>dbootlevel1(...)::Vector{T_level1}</code> -&gt; Returns a vector of bootstrapped level 1 statistics, where the output vector will have length equal to <code>numresample</code>.</li>
<li><code>dbootlevel2(...)::T_level2</code> -&gt; Returns the bootstrapped distribution parameter of the level 1 statistic.</li>
<li><code>dboot(...)::T_level2</code> -&gt; Identical to dbootlevel2. Most users will want to use this function.</li>
<li><code>dbootvar(...)::Float64</code> -&gt; Identical to <code>dboot</code> but automatically sets <code>flevel2</code> to <code>var</code> (the sample variance function)</li>
<li><code>dbootconf(...)::Vector{Float64}</code> -&gt; Identical to <code>dboot</code> but automatically sets <code>flevel2</code> to the anonymous function <code>x -&gt; quantile(x, [0.025, 0.975])</code>, so the level 2 distribution parameter is a 95% confidence interval. In addition to the usual keywords, the keyword version of this function also accepts the keyword <code>alpha::Float64=0.05</code>, which controls the width of the confidence interval. Note, <code>0.05</code> corresponds to a 95% confidence interval, <code>0.1</code> to a 90% interval, and <code>0.01</code> to a 99% interval (and so on).</li>
<li><code>optblocklength(...)::Float64</code> -&gt; Returns the optimal block length.</li>
</ul>
<p>The function <code>bandwidth_politis_2003{T&lt;:Number}(x::AbstractVector{T})::Tuple{Int, Float64, Vector{Float64}}</code> is not exported, but the docstrings can be accessed using <code>?DependentBootstrap.bandwidth_politis_2003</code> at the REPL. This function implements the bandwidth selection procedure from Politis (2003) discussed above, and may be of independent interest to some users.</p>
<p>All of the above functions exhibit the following two core methods:</p>
<ul>
<li><code>f(data ; kwargs...)</code></li>
<li><code>f(data, bi::BootInput)</code></li>
</ul>
<p>where <code>data</code> is the users underlying dataset, <code>kwargs</code> is a collection of keyword arguments, and <code>bi::BootInput</code> is a core type exported by the module that will be discussed later (but can be safely ignored by most users). The following types for <code>data</code> are currently accepted:</p>
<ul>
<li><code>Vector{&lt;:Number}</code>,</li>
<li><code>Matrix{&lt;:Number}</code> where rows are observations and columns are variables,</li>
<li><code>Vector{Vector{&lt;:Number}}</code> where each inner vector is a variable,</li>
<li><code>DataFrame</code></li>
<li><code>TimeArray</code></li>
</ul>
<p>Of the two core methods, most users will want the <code>kwargs</code> method. A list of valid keyword arguments and their default values follows:</p>
<ul>
<li><code>blocklength</code>         &lt;- Block length for bootstrapping procedure. The default value is <code>0</code>. Set to &lt;= 0 to auto-estimate the optimal block length from the dataset. <code>Float64</code> inputs are allowed.</li>
<li><code>numresample</code>         &lt;- Number of times to resample the input dataset. The default value is the module constant <code>NUM_RESAMPLE</code>, currently set to <code>1000</code>.</li>
<li><code>bootmethod</code>          &lt;- Bootstrapping methodology to use. The default value is <code>:stationary</code> (for the stationary bootstrap).</li>
<li><code>blocklengthmethod</code>   &lt;- Block length selection procedure to use if user wishes to auto-estimate the block length. Default value is <code>:ppw2009</code> (use the method described in Patton, Politis, and White (2009)).</li>
<li><code>flevel1</code>             &lt;- A function that converts the input dataset to the estimator that the user wishes to bootstrap. The default value is <code>mean</code>.</li>
<li><code>flevel2</code>             &lt;- A function that converts a vector of estimators constructed by <code>flevel1</code> into a distributional parameter. The default value is <code>var</code>.</li>
<li><code>numobsperresample</code>   &lt;- Number of observations to be drawn (with replacement) per resample. The default value is the number of observations in the dataset (the vast majority of users will want this default value).</li>
<li><code>fblocklengthcombine</code> &lt;- A function for converting a <code>Vector{Float64}</code> of estimated blocklengths to a single <code>Float64</code> blocklength estimate, which is necessary when the input dataset is a multivariate type. The default value is <code>median</code>.</li>
</ul>
<p>A list of acceptable keyword arguments for <code>bootmethod</code> and <code>blocklengthmethod</code> follows. Note you can use either <code>String</code> or <code>Symbol</code> when specifying these arguments. For <code>bootmethod</code> we have:</p>
<ul>
<li><code>:iid</code> or <code>:efron</code>                     &lt;- IID bootstrap</li>
<li><code>:stationary</code>                          &lt;- Stationary bootstrap</li>
<li><code>:movingblock</code> or <code>:moving</code>            &lt;- Moving block bootstrap</li>
<li><code>:nonoverlappingblock</code> or <code>:nooverlap</code> &lt;- Nonoverlapping block bootstrap</li>
<li><code>:circularblock</code> or <code>circular</code>         &lt;- Circular block bootstrap</li>
</ul>
<p>For <code>blocklengthmethod</code> we have:</p>
<ul>
<li><code>:ppw2009</code> &lt;- Block length selection method of Patton, Politis, and White (2009)</li>
</ul>
<p>Acceptable arguments can also be examined interactively by examining the keys of the module dictionaries <code>BOOT_METHOD_DICT</code> and <code>BLOCKLENGTH_METHOD_DICT</code>.</p>
<p>In practice, the keyword argument method <code>f(data ; kwargs...)</code> actually just wraps a call to <code>f(data, BootInput(kwargs...))</code> under the hood. However, most users will not need to concern themselves with this level of detail.</p>
<p>For those who wish more fine-grained control, please use <code>?BootInput</code> at the REPL to get more information on this core module type.</p>
<h4><a id="user-content-examples" class="anchor" aria-hidden="true" href="#examples"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Examples</h4>
<p>Let <code>data::Vector{Float64}</code>.</p>
<p>The variance of the sample mean of <code>data</code> can be bootstrapped using a stationary bootstrap with optimally estimated block length using <code>dboot(data)</code> or <code>dbootvar(data)</code>.</p>
<p>A 90% confidence interval for the sample median using a circular block bootsrap with block length of 5 can be estimated using <code>dboot(data, blocklength=5, bootmethod=:circular, flevel1=median, flevel2=(x -&gt; quantile(x, [0.05, 0.95])))</code> or <code>dbootconf(data, blocklength=5, bootmethod=:circular, flevel1=median, alpha=0.1)</code>.</p>
<p>Moving block bootstrap indices for generating bootstrapped data with optimally estimated block length can be obtained using <code>dbootinds(data, bootmethod=:moving)</code>, or if the user wants the bootstrapped data not the indices, <code>dbootdata(data, bootmethod=:moving)</code>. If the user wants bootstrapped sample medians of <code>data</code>, then use <code>dbootlevel1(data, bootmethod=:moving, flevel1=median)</code>.</p>
<p>If the user wants the optimal block length using the method proposed in Patton, Politis, and White (2009), use <code>optblocklength(data, blmethod=:ppw2009)</code>.</p>
<p>Now let <code>data::Matrix{Float64}</code>.</p>
<p>If the user wants the median optimal block length from each column of <code>data</code>, use <code>optblocklength(data, blmethod=:ppw2009)</code>. If the user wants the average optimal block length use <code>optblocklength(data, blmethod=:ppw2009, fblocklengthcombine=mean)</code>.</p>
<p>If the user wants the median of the test statistic that is the maximum of the sample mean of each column, using a stationary bootstrap with optimal block length, then use <code>dboot(data, flevel1=(x -&gt; maximum(mean(x, dims=1))), flevel2=median)</code>. If <code>data::Vector{Vector{Float64}}</code> instead, and the user wanted the 95% confidence interval, use <code>dbootconf(data, flevel1=(x -&gt; maximum([ mean(x[k]) for k = 1:length(x) ])))</code>.</p>

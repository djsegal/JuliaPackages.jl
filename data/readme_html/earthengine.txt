<p dir="auto">Google Earth Engine in Julia!</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="docs/src/assets/logo-small.png"><img src="docs/src/assets/logo-small.png" alt="logo" style="max-width: 100%;"></a></p>
<p dir="auto"><code>EarthEngine.jl</code> is used to interface with the <em>amazing</em> cloud-based geospatial processing platform, <a href="https://earthengine.google.com" rel="nofollow">Google Earth Engine</a>, using the <a href="https://julialang.org/" rel="nofollow">Julia</a> programming language as a wrapper around the <a href="https://developers.google.com/earth-engine/guides/python_install" rel="nofollow">EE Python API</a>.</p>
<p dir="auto">You can use <code>EarthEngine.jl</code> in the following two ways.</p>
<ol dir="auto">
<li>Interface with the good ole' object-oriented Python API that everyone knows and loves through Julia (i.e. <code>imagecollection.filterDate(...).mean()</code>)</li>
<li>Interface with Earth Engine using Julia-like syntax that leverages types and multiple dispatch (i.e. <code>(mean(filterDate(imagecollection,...)))</code>)</li>
</ol>
<p dir="auto">See the <a href="https://kmarkert.github.io/EarthEngine.jl/dev/usage/" rel="nofollow">Usage page</a> of the documentation for more details on how to use.</p>
<h2 dir="auto"><a id="user-content-why-julia--earth-engine" class="anchor" aria-hidden="true" href="#why-julia--earth-engine"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why Julia + Earth Engine?</h2>
<p dir="auto">Earth Engine has become a vital geospatial tool for many Earth science researchers and organizations. The officially supported Earth Engine APIs are written in <a href="https://developers.google.com/earth-engine/guides/getstarted" rel="nofollow">JavaScript</a> and <a href="https://developers.google.com/earth-engine/guides/python_install" rel="nofollow">Python</a>. The Python API provides great flexability and many useful packages for downstream analysis, but what about other scientific programming languages? Community-developed APIs, like <a href="https://github.com/r-spatial/rgee/">rgee</a> are allowing developers to interface with EE using their favorite languages. This package provides the Earth Engine API for users who love programming in Julia!</p>
<p dir="auto">Julia is a modern programming language that has the feel of a scripting language with the performance of compiled languages (thanks to its JIT compilation). Julia is full of features e.g. types and multiple dispatch that make developing EE workflows more expressive.</p>

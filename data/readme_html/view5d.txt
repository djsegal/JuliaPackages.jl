<p dir="auto"><a href="https://github.com/RainerHeintzmann/View5D.jl">View5D.jl</a> is a Java-based viewer for up to 5-dimensional data (including complex images). It supports three mutually linked orthogonal slicing displays for XYZ coordinates, arbitrary numbers of colors (4th dimension) which can also be used to display spectral curves and a time slider for the 5th dimension.</p>

<p dir="auto">Display of <code>Complex</code>-valued data can be toggled between <code>magnitude</code>, <code>phase</code>, <code>real</code> and <code>imaginary</code> part.  A complex-valued array by default switches the viewer to a <code>gamma</code> of 0.3 easing the inspection of Fourier-transformed data. However, gamma is adjustable interactively as well as when invoking the viewer.</p>

<p dir="auto">See this YouTube Video for more details:
<a href="https://www.youtube.com/watch?v=fqa82MmJlAA&amp;list=PL3LueK3ij6Wm2VjaaibNdulxFvA6VhVRv" rel="nofollow"><img src="docs/src/assets/youtube.png" alt="IMAGE ALT TEXT" style="max-width: 100%;"></a></p>
<p dir="auto">The Java viewer <a href="https://nanoimaging.de/View5D" rel="nofollow">View5D</a> has been integrated into Julia with the help of <a href="https://github.com/JuliaInterop/JavaCall.jl">JavaCall.jl</a>.  Currently the viewer has its full Java functionality which includes displaying and interacting with 5D data. Generating up to three-dimensional histograms and interacting with them to select regions of interest in the 3D histogram but shown as a selection in the data. It allows selection of a gate <code>element</code> where thresholds can be applied to which have an effect on statistical evaluation (mean, max, min) in other <code>element</code>s if the <code>gate</code> is activated.
It further supports multiplicative overlay of colors. This feature is useful when processed data (e.g. local orientation information or polarization direction or ratios) needs to be presented along with brightness data. By choosing a gray-valued and a  constant brightness value-only (HSV) colormap for brightness and orientation data respectively, in multiplicative overlay mode a result is obtained that looks like the orientation information is staining the brightness. These results look often much nicer compared to gating-based display based on a brightness-gate, which is also supported.
Color display of floating-point or 16 or higher bit data supports adaptively updating colormaps.
Zooming in on a colormap,  by changing the lower and upper display threshold, for some time the colormap is simply changed to yield a smooth experience but occasionally the cached display data is recomputed to avoid loosing fine granularity on the color levels.</p>
<h2 dir="auto"><a id="user-content-interaction-with-julia" class="anchor" aria-hidden="true" href="#interaction-with-julia"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Interaction with Julia</h2>
<p dir="auto">The interaction to Julia supports calling the viewer via one main method <code>view5d</code> supporting various options. Since the viewer is most useful for quick visualization and debugging, a multitude of short-hand convenience functions are provided:</p>
<ul dir="auto">
<li><code>vv(data)</code>: starts a new viewer (window) and displays the data</li>
<li><code>vp(data)</code>: starts a new viewer (window) and displays the data as complex values with a <strong>color-phase</strong> (multiplicative) overlay as a second channel. This is particularly useful for displaying complex-valued data. By default the <code>gamma </code> correction for display is set to 0.3 enhancing the low-values. Naturally this can be prevented by the optional named argument <code>gamma</code> for all of the calls.</li>
<li><code>ve(data)</code>: concatenates the data to display to the currently "active" viewer as a new entry along the element (==color) direction.</li>
<li><code>vt(data)</code>: concatenates the data to display to the currently "active" viewer as a new entry along the time (slider on the right side) direction.</li>
<li><code>vtp(data)</code>, <code>vep(data)</code>: concatenates color-phase display (see <code>vp</code>) along respective data directions.</li>
<li><code>vr(data)</code>: replaces the currently displayed data in the currently "active" viewer by this new <code>data</code>.</li>
<li><code>vrp(data)</code>: replaces color-phase display data in the currently "active" viewer by this new color-phase display <code>data</code>.</li>
</ul>
<p dir="auto">All of these function have corresponding macros (<code>@vv</code>, <code>@vp</code> etc.) to be used in analogy to the <code>@show</code> method. This eases debugging. If the preceding expression is a viewer reference, display is directed to this specific viewer. Here an example to update a specific viewer display every second in a for loop:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="v= @vr rand(10,10,10,3); for n in 1:10  sleep(1); @vr v rand(10,10,10,3) end"><pre>v<span class="pl-k">=</span> <span class="pl-c1">@vr</span> <span class="pl-c1">rand</span>(<span class="pl-c1">10</span>,<span class="pl-c1">10</span>,<span class="pl-c1">10</span>,<span class="pl-c1">3</span>); <span class="pl-k">for</span> n <span class="pl-k">in</span> <span class="pl-c1">1</span><span class="pl-k">:</span><span class="pl-c1">10</span>  <span class="pl-c1">sleep</span>(<span class="pl-c1">1</span>); <span class="pl-c1">@vr</span> v <span class="pl-c1">rand</span>(<span class="pl-c1">10</span>,<span class="pl-c1">10</span>,<span class="pl-c1">10</span>,<span class="pl-c1">3</span>) <span class="pl-k">end</span></pre></div>
<p dir="auto">Importing and exporting hierarchical sets of markers into and from the viewer are supported (<code>export_marker_lists</code>, <code>import_marker_lists</code>).</p>
<p dir="auto">View5D features a wide range of data formats: <code>Float32</code>, <code>Float64</code>, <code>UInt8</code>, <code>Int8</code>, <code>UInt16</code>, <code>Int16</code>, <code>UInt32</code>, <code>Int32</code>, <code>Int</code>, <code>Complex32</code>, <code>RGB</code> and <code>Gray</code>, also the outputs of the <code>Images.jl</code> <code>BioformatsLoader.jl</code> package. If context information such as axes sizes is available, it will shown accordingly in the viewer.</p>
<p dir="auto">Display of <code>Complex</code>-valued data can be toggled between <code>magnitude</code>, <code>phase</code>, <code>real</code> and <code>imaginary</code> part.  A complex-valued array by default switches the viewer to a <code>gamma</code> of 0.3 easing the inspection of Fourier-transformed data. However, gamma is adjustable interactively as well as when invoking the viewer.</p>
<p dir="auto">Since the viewer is written in Java and launched via JavaCall its thread should be pretty independent from julia. This should make the user experience pretty smooth also with minimal implications to julia threading performance.</p>

<ul dir="auto">
<li><code>view5d()</code>: visualizes data. Via "mode" it can be selected whether a new viewer will be used (<code>mode="new"</code>) or the data is appended to the existing viewer via the element (<code>mode="add_time"</code>) or time direction (<code>mode="add_time"</code>). Data can also be replacing currently displayed data (<code>mode="replace"</code>), which is useful to display iterative updates.</li>
<li><code>process_keys()</code>: the easiest way to remote-control the viewer by sending it key-strokes. Be careful: almost all keys have a meaning assigned and the viewer has no undo function.</li>
<li><code>set_display_size()</code>: sets the size (in pixels) this viewer occupies on the screen.</li>
<li><code>set_title()</code>: sets a new title to the viewer window.</li>
<li><code>to_front()</code>: brings the viewer to the front</li>
<li><code>set_gamma()</code>: sets the gamma display-value for a particular color channel (element)</li>
<li><code>set_min_max_thresh()</code>: sets the minimum and maximum display value for a particular color channel (element).</li>
<li><code>export_marker_lists()</code>, <code>import_marker_lists()</code>: these functions allow you to display location information such as obtained from tracking algorithms.</li>
</ul>

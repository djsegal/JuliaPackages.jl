<p dir="auto">PiCraft is a Julia package which allows control over the Minecraft World using code. It is compatible with the Minecraft: Pi Edition and the Java Edition with the help of RaspberryJamMod or RaspberryJuice.</p>
<p dir="auto">Programming in the Minecraft world can be very beneficial in constructing large scale
projects which can be automated and this can also be a fun way to introduce programming
to children.</p>
<p dir="auto">Some sample projects available in the documentation are:</p>
<ul dir="auto">
<li>Write scripts to teleport and walk over Water and Lava.</li>
<li>Construct multi-storey skyscrapers within seconds.</li>
<li>Draw fractals and shapes using a 3-D turtle.</li>
<li>Import and export <code>.schematic</code> models without external software.</li>
</ul>

<p dir="auto">Load the <code>PiCraft</code> package on the Julia console:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using PiCraft"><pre><span class="pl-k">using</span> PiCraft</pre></div>
<p dir="auto">On load, the library will attempt to connect to a running Minecraft world on <code>localhost</code>. If Minecraft is not running, an error message will be printed. Subsequently, once Minecraft has been started, a connection can be forced by calling <code>connectToWorld()</code>.</p>
<p dir="auto">A 3-D coordinate system is used to describe the Minecraft world. Every position in the Minecraft World can be described with the help of 3 numbers. The X, Y and Z coordinates. These coordinates can be viewed by opening the Debug screen pressing the <code>F3</code> key.</p>

<p dir="auto">You can find and set your player's location:</p>
<ol dir="auto">
<li>Find Player position: <code>getPos()</code>
<ul dir="auto">
<li>Returns a <code>Tuple{Float64, Float64, Float64}</code> which contains the player's current coordinates.</li>
</ul>
</li>
<li>Set Player position: <code>setPos(pos::Tuple{Float64, Float64, Float64})</code>
<ul dir="auto">
<li>Teleports the player to the specified coordinates.</li>
</ul>
</li>
</ol>
<p dir="auto">For example : <code>setPos(getPos() .+ (0, 10, 0))</code> will teleport you 10m above you current position.</p>
<h3 dir="auto"><a id="user-content-blocks" class="anchor" aria-hidden="true" href="#blocks"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Blocks</h3>
<p dir="auto">Blocks are the heart of Minecraft. The package provides the ability to modify blocks at specified coordinates.
<code>Block</code> is an immutable datatype defined in <code>src/blocks.jl</code>.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="struct Block
    id::Int
    data::Int
end"><pre><span class="pl-k">struct</span> Block
    id<span class="pl-k">::</span><span class="pl-c1">Int</span>
    data<span class="pl-k">::</span><span class="pl-c1">Int</span>
<span class="pl-k">end</span></pre></div>
<p dir="auto">The <code>id</code> defines the type of block (like cobblestone, wool, wood,.etc) and the <code>data</code> attribute defines additional characteristics of the block, on default every block has it <code>data</code> set to <code>0</code>. For example, Wool's block id is 35, <code>Block(35,0)</code> refers to a block of white wool. Different wool colors can be accessed by changing the <code>data</code> attribute. Red Wool is <code>Block(35, 14)</code>, Pink Wool is <code>Block(35,6)</code> and so on. A complete reference can be found <a href="https://juliaberry.github.io/PiCraft.jl/blocks.html" rel="nofollow">here</a> in the documentation.</p>
<ol dir="auto">
<li>Place Block: <code>setBlock(Tuple{Int, Int, Int}, block::Block)</code>
<ul dir="auto">
<li>Place the specified Block at the specified coordinates.</li>
</ul>
</li>
<li>Place Blocks: <code>setBlocks(p1::Tuple{Real, Real, Real}, p2::Tuple{Real, Real, Real}, block::Block)</code>
<ul dir="auto">
<li>Set an entire region to the specified block type defined by corners.</li>
</ul>
</li>
<li>Get Block information: <code>getBlock(Tuple{Int,Int,Int})</code>
<ul dir="auto">
<li>Returns the block present at the specified coordinates.</li>
</ul>
</li>
</ol>
<p dir="auto">While the <code>getBlock</code> and <code>setBlock</code> functions will accept <code>Float64</code> as arguments these will be rounded and set to <code>Int</code> as a block cannot be placed on non-integer coordinates.</p>

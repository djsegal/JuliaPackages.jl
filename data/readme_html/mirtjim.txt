<p dir="auto">This Julia language repo
exports the <code>jim</code> method that provides a "jiffy image display"
for showing 2D grayscale images
(and 3D grayscale images as a mosaic).
It is basically a wrapper around <code>Plots.heatmap</code>
with natural defaults.
As of v0.9 it supports
axes, colorbar limit (<code>clim</code>) and images
with physical units.</p>

<p dir="auto">This repo also exports some small helper functions.</p>
<ul dir="auto">
<li>
<p dir="auto"><code>prompt()</code> first calls <code>Plots.gui()</code>
to display the current plot,
then waits for a user key press.</p>
<p dir="auto">Some keys have special effects:</p>
<ul dir="auto">
<li><code>[q]uit</code> throws an error</li>
<li><code>[d]raw</code> disables further prompting and the plots are just drawn</li>
<li><code>[n]odraw</code> avoids the <code>gui()</code> call (useful for non-interactive testing)</li>
</ul>
<p dir="auto">Calling <code>prompt(:prompt)</code> reverts the default key-press behavior.</p>
</li>
<li>
<p dir="auto"><code>caller_name()</code> uses <code>stacktrace</code> (in <code>Base</code>)
to return the file name and line number that called the current function.
It can be helpful for debugging and for giving warnings some context.</p>
</li>
<li>
<p dir="auto"><code>mid3()</code> extracts the middle three slices
(transaxial, coronal, sagittal) of a 3D array
and arranges them in a 2D mosaic for quick display.</p>
</li>
</ul>

<p dir="auto">This method is used in many of the image reconstruction examples in
<a href="https://github.com/JuliaImageRecon">https://github.com/JuliaImageRecon</a>.</p>
<p dir="auto">Isolating these functions in this repo,
separate from other repos like
the
<a href="https://github.com/JeffFessler/MIRT.jl">Michigan Image Reconstruction Toolbox (MIRT)</a>
keeps
those repos lighter
by avoiding a dependence on <code>Plots.jl</code> there.</p>

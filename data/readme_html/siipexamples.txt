<p dir="auto">This package contains examples and tutorials for <a href="https://www.nrel.gov/analysis/siip.html" rel="nofollow">Scalable Integrated Infrastructure Planning (SIIP) packages developed at the National Renewable Energy Laboratory (NREL)</a>. This repository uses a template established by <a href="https://github.com/JuliaOpt/JuMPTutorials.jl">JuMPTutorials.jl</a>.</p>
<p dir="auto">The examples included here are intended as practical examples of using <a href="https://github.com/nrel-siip">SIIP packages</a>. However,
in many cases users may find the documentation pages for each package more informative:</p>
<ul dir="auto">
<li><a href="https://nrel-siip.github.io/InfrastructureSystems.jl/stable/" rel="nofollow">InfrastructureSystems.jl docs</a></li>
<li><a href="https://nrel-siip.github.io/PowerSystems.jl/stable/" rel="nofollow">PowerSystems.jl docs</a></li>
<li><a href="https://nrel-siip.github.io/PowerSimulations.jl/stable/" rel="nofollow">PowerSimulations.jl docs</a></li>
<li><a href="https://nrel-siip.github.io/PowerGraphics.jl/stable/" rel="nofollow">PowerGraphics.jl docs</a></li>
<li><a href="https://nrel-siip.github.io/PowerSimulationsDynamics.jl/stable/" rel="nofollow">PowerSimulationsDynamics.jl docs</a></li>
</ul>

<p dir="auto">To try out any of the tutorials in the browser without downloading Julia, click on the launch binder button above. Note that this functionality only supports open-source solvers which do not have additional requirements (for e.g. BLAS or MATLAB). This is also very slow and can take several minutes to start as it has to first install Julia and all the dependencies. Thus, you should download and run the notebooks on your local machine for the best experience.</p>

<p dir="auto">To get started running the Jupyter notebooks included in this package, you can follow the process in <a href="https://youtu.be/gHoGV4RDz-U" rel="nofollow">this video</a> demonstrating the following steps:</p>

<p dir="auto">On occasion, you may have constructed a <code>System</code> using PowerSystemsCaseBuilder.jl that needs
to be reconstructed. In this case, you may receive an error such as:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="ERROR: UndefVarError: PowerSystems.ReserveUp not defined"><pre>ERROR<span class="pl-k">:</span> UndefVarError<span class="pl-k">:</span> PowerSystems<span class="pl-k">.</span>ReserveUp not defined</pre></div>
<p dir="auto">To resolve this issue, you can purge the serialized system data from your
PowerSystemsCaseBuilder.jl instance by running:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using PowerSystemCaseBuilder
PowerSystemCaseBuilder.clear_all_serialized_system()"><pre><span class="pl-k">using</span> PowerSystemCaseBuilder
PowerSystemCaseBuilder<span class="pl-k">.</span><span class="pl-c1">clear_all_serialized_system</span>()</pre></div>

<p dir="auto">This package explores a new approach to latency reduction in Julia. It determines the unique <a href="https://en.wikipedia.org/wiki/Basic_block" rel="nofollow">basic blocks</a> present across a swath of Julia code and rewrites functions as calls to "fragments" that represent these blocks.</p>
<p dir="auto"><strong>This is a work in progress, and most of the features described below have not yet been implemented.</strong></p>
<h2 dir="auto"><a id="user-content-analysis-pass" class="anchor" aria-hidden="true" href="#analysis-pass"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Analysis pass</h2>
<p dir="auto">Starting from a method, generate <em>relocatable fragments</em>:</p>
<ul dir="auto">
<li>split the lowered code into blocks. Trim gotos, which are external to fragments.</li>
<li>for each trimmed block, renumber <code>SlotNumber</code>s and internal <code>SSAValue</code>s by the order in which they appear</li>
<li>convert external <code>SSAValue</code>s into slots</li>
</ul>
<h2 dir="auto"><a id="user-content-generating-callable-fragments" class="anchor" aria-hidden="true" href="#generating-callable-fragments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Generating callable fragments</h2>
<p dir="auto">Starting from the original lowered code, block-internal SSAValues accessed from other blocks, as well as updates to slots, are assembled to create the fragment return values. Ideally this gets done "swath-wide" because the same fragment may be used in different ways (using or ignoring different possible return values) by other code.</p>
<p dir="auto">Then the fragments from the analysis pass get wrapped in methods.</p>

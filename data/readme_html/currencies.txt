<p>This is a core package for the JuliaFinance ecosytem.</p>
<p>It provides bare singleton types based on the standard ISO 4167 3-character alpha codes to be used primarily for dispatch in other JuliaFinance packages together with five methods:</p>
<ul>
<li><code>symbol</code>: The 3-character symbol of the currency.</li>
<li><code>currency</code>: The singleton type instance for a particular currency symbol</li>
<li><code>name</code>: The full name of the currency.</li>
<li><code>code</code>: The ISO 4217 code for the currency.</li>
<li><code>unit</code>: The minor unit, i.e. number of decimal places, for the currency.</li>
</ul>
<p>Within JuliaFinance, currencies are defined in two separate packages:</p>
<ul>
<li><a href="https://github.com/JuliaFinance/Currencies.jl">Currencies.jl</a></li>
<li><a href="https://github.com/JuliaFinance/Assets.jl">Assets.jl</a></li>
</ul>
<p>A brief explanation and motivation for each is presented below.</p>

<p>As mentioned, this package defines standard currencies as primordial singleton types that can be thought of as labels.</p>
<p>For example:</p>

<p>If all you need is a list of currencies with names, ISO 4167 codes and minor units, e.g. for building a dropdown menu in a user interface, then this lightweight package is what you want.</p>
<h2><a id="user-content-assetsjl" class="anchor" aria-hidden="true" href="#assetsjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="https://github.com/JuliaFinance/Assets.jl">Assets.jl</a></h2>
<p>When a currency is thought of as an asset (as opposed to a mere label), we choose to refer to it as "Cash" as it would appear in a balance sheet. <a href="https://github.com/JuliaFinance/Assets.jl">Assets.jl</a> provides a <code>Cash</code> asset together with <code>Position</code> that allows for basic algebraic manipulations of <code>Cash</code> and other financial instrument positions, e.g.</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">import</span> Assets<span class="pl-k">:</span> USD, JPY

julia<span class="pl-k">&gt;</span> <span class="pl-c1">10</span>USD
<span class="pl-c1">10.00</span>USD

julia<span class="pl-k">&gt;</span> <span class="pl-c1">10</span>JPY
<span class="pl-c1">10</span>JPY

julia<span class="pl-k">&gt;</span> <span class="pl-c1">10</span>USD<span class="pl-k">+</span><span class="pl-c1">20</span>USD
<span class="pl-c1">30.00</span>USD

julia<span class="pl-k">&gt;</span> <span class="pl-c1">10</span>USD<span class="pl-k">+</span><span class="pl-c1">10</span>JPY
ERROR<span class="pl-k">:</span> Can<span class="pl-k">'</span>t add Positions of different Instruments USD, JPY</pre></div>
<p>If you need currency as an asset with corresponding asset positions, you want <a href="https://github.com/JuliaFinance/Assets.jl">Assets.jl</a>.</p>

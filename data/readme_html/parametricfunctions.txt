<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Introduction</h2>
<p>This is a package for handling parametric function families of the form <code>f(x; θ)</code>.</p>
<p>A particular <code>f(⋅,⋅)</code> is a <em>function family</em>. It can be queried about its <code>domain</code>, degrees of freedom <code>degf</code> (the dimension of <code>θ ∈ ℝⁿ</code>), evaluated at <code>x</code> and <code>θ</code>.</p>
<p>This package is mainly useful for solving functional problems using projection methods, ie given some mapping <code>F</code>, solve for <code>F(f(x;θ)) = 0</code>. The library should be transparent to automatic differentiation tools such as <a href="https://github.com/JuliaDiff/ForwardDiff.jl">ForwardDiff.jl</a>.</p>
<p>The goal of this package is facilitating the management of function families for a mostly "manual" approach to constructing approximations to functions, and then minimizing then some kind of residual. The function families are built from a set of basic building blocks using univariate families (such as Chebyshev polynomials, splines), domain and value transformations (function families need not be linear), combined into tensored multivariate families, or more sophisticated variations such as complete polynomials or sparse approximations.</p>
<p>There is also limited support for calculating <code>θ</code> for a given <code>y = f(x; θ)</code>, where <code>x</code> are the collocation <code>points</code> of the function family, but this is mainly useful for initial guesses for <code>θ</code>.</p>

<p>Currently I am refining the API for univariate functions, using simple function families. Multivariate constructs are coming later. This is work in progress, the API may change without notice.</p>

<p dir="auto">Aqua.jl provides functions to run a few automatable checks for Julia packages:</p>
<ul dir="auto">
<li>There are no method ambiguities.</li>
<li>There are no undefined <code>export</code>s.</li>
<li>There are no unbound type parameters.</li>
<li>There are no stale dependencies listed in <code>Project.toml</code>.</li>
<li>Check that test target of the root project <code>Project.toml</code> and test project
(<code>test/Project.toml</code>) are consistent.</li>
<li>Check that all external packages listed in <code>deps</code> have corresponding
<code>compat</code> entry.</li>
<li><code>Project.toml</code> formatting is compatible with Pkg.jl output.</li>
<li>There are no "obvious" type piracies (<a href="#notes-on-aqua-06"><strong>new in 0.6</strong></a>)</li>
</ul>
<p dir="auto">See more in the <a href="https://juliatesting.github.io/Aqua.jl/dev" rel="nofollow">documentation</a>.</p>

<p dir="auto">Aqua 0.6 includes the type piracy detection, thanks to <a href="https://github.com/JuliaTesting/Aqua.jl/pull/88" data-hovercard-type="pull_request" data-hovercard-url="/JuliaTesting/Aqua.jl/pull/88/hovercard">the PR</a> by Jakob
Nybo Nissen (@jakobnissen) and <a href="https://discourse.julialang.org/t/pirate-hunter/20402" rel="nofollow">the original implementation</a> by
Frames Catherine White (@oxinabox).</p>
<p dir="auto">If this part of Aqua 0.6 causes a trouble, there are two ways to solve the issue:</p>
<ol dir="auto">
<li>Keep using Aqua 0.5.  It is still maintained.</li>
<li>Disable the piracy detection by the flag as in
<code>Aqua.test_all(YourPackage; piracy = false)</code>.</li>
</ol>

<p>This repository contains ROMEO 3D/4D unwrapping on arrays.<br>
For MR data in the NIfTI format, a compiled command line tool is available under <a href="https://github.com/korbinian90/ROMEO">ROMEO</a> (windows and linux binaries; does not require a Julia installation) and otherwise, for opening NIfTI files in Julia <a href="https://github.com/JuliaIO/NIfTI.jl">NIfTI.jl</a> or <a href="https://github.com/korbinian90/MriResearchTools.jl">MriResearchTools.jl</a> can be helpful.</p>

<h3><a id="user-content-multi-echo" class="anchor" aria-hidden="true" href="#multi-echo"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Multi-Echo</h3>
<p>If multi-echo data is available, supplying ROMEO with multi-echo information should improve the unwrapping accuracy. The same is true for magnitude information.</p>
<p>If the multi-echo data contains <strong>large phase offsets</strong> (phase at echo time zero), default template unwrapping might fail. Setting the <code>individual-unwrapping</code> flag is a solution, as it performs spatial unwrapping for each echo instead. The computed B0 map is in the current version not corrected for remaining phase offsets.</p>

<h2 dir="auto"><a id="user-content-purpose" class="anchor" aria-hidden="true" href="#purpose"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Purpose</h2>
<p dir="auto">Optimization-FEM consists in optimising a target function (here called the cost function) under the constraint that a FEM model be exactly verified. This in turn implies that the problem has more unknowns than the model has equation (or at least: more unknowns than linearly independent equations, as would be the case with “insufficient” boundary conditions).</p>
<p dir="auto">The unknowns that are dual to the FEM equations are noted X-dofs (the <em>response</em> of the model). The rest of the unknowns can be separated into U-dofs (varying with time, generaly <em>unknown loads</em>) and A-dofs (constant over time, generaly <em>unknown model parameters</em>). The conditions for such a constrained optimization problem to be well-posed are the Babushka-Brezzi conditions, which say, in essence “if you do not restrain, then at least measure”.</p>
<h2 dir="auto"><a id="user-content-applications-of-optimization-fem" class="anchor" aria-hidden="true" href="#applications-of-optimization-fem"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Applications of optimization-FEM</h2>
<p dir="auto">Besides solving well-posed FEM problems (obtaining the response of a system, given adequate boundary conditions and known loading terms), many applications can be, or should be, possible within <code>Muscade</code>.</p>
<p dir="auto"><strong>Reliability analysis</strong>: Finding a design point. What is the most probable combination of external loads <code>U</code> and strength of the structure <code>A</code> that may cause the response <code>X</code> to exceed, in one of many ways, the acceptable?</p>
<p dir="auto"><strong>Design optimization</strong>: What is the cheapest way to engineer a system (for example a structure) that will survive a set of loading conditions?</p>
<p dir="auto"><strong>Load identification and monitoring</strong>: Given incomplete and noisy measurements of the response of a system, what are the loads that are most likely to cause a response close to what has been measured?</p>
<p dir="auto"><strong>Optimal control</strong>: how to steer a system with many dofs into a wanted behaviour?</p>
<p dir="auto"><strong>Model identification</strong>: given enough measurements on the response of a system responding to at least partly unknown load, is it possible to adjust the model of the system (model calibration, damage detection)?</p>
<p dir="auto"><strong>Sensor array optimization</strong>: how best to place sensors in a system in order to support the above applications?</p>

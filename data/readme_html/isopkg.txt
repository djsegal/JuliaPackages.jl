<div id="readme" class="md" data-path="README.md"><article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-isopkg" class="anchor" aria-hidden="true" href="#isopkg"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>IsoPkg</h1>
<p>Environment-isolated package manager</p>
<p><strong>No more package version conflicts. Easily upgrade packages to the newest version. Scripts keep working after package upgrading</strong></p>
<p>Manage packages by creating their own isolated project environment for every directly installed package. The directly installed packages will never conflict with each other. If updating a package, it can always be upgraded to the newest version. Even the same package with different versions can be installed together. Scripts can easily specify dependent packages and never need to worry about the package upgrading.</p>
<h2><a id="user-content-showcase" class="anchor" aria-hidden="true" href="#showcase"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Showcase:</h2>
<p>If <code>]add UnicodePlots@1.2.0 StatsBase@0.33.0</code> in Julia v1.4, then it will return <code>ERROR: Unsatisfiable requirements detected for package StatsBase [2913bbd2]</code> because of the version restrictions of package dependencies (see <a href="https://www.juliabloggers.com/understanding-package-version-restrictions-in-julia/" rel="nofollow">1</a>, <a href="https://www.juliabloggers.com/my-practices-for-managing-project-dependencies-in-julia/" rel="nofollow">2</a>).</p>
<p>With <code>IsoPkg</code>, simplely <code>using IsoPkg; IsoPkg.add("UnicodePlots@1.2.0"); IsoPkg.add("StatsBase@0.33.0")</code>, then both packages can be used together: <code>@iso using UnicodePlots "1.2.0"; @iso using StatsBase "0.33.0"</code> (Notice: this will bypass the version compatibility check). Foremore, the script loading UnicodePlots or StatsBase as this form will always work without worrying about the package upgrading.</p>
<h2><a id="user-content-implementation-detail" class="anchor" aria-hidden="true" href="#implementation-detail"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Implementation detail:</h2>
<p>This package simply creates the <code>env_isolated_packages</code> folder in <code>~/.julia</code>. Then creates a folder (default is Julia version) for grouping the packages. Installing package will create an isolated environment in the activated group folder. Before operating or loading a package, the corresponding environment will be activated automatically. The whole process is quite lightweight. You can even manually manage the folders in <code>~/.julia/env_isolated_packages</code> to manage the installed packages.</p>

<h1><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Usage</h1>
<ul>
<li>install a package: <code>IsoPkg.add(pkg_name)</code></li>
<li>install a package and pin to the specified version: <code>IsoPkg.add(pkg_name@version)</code></li>
<li>remove a package: <code>IsoPkg.rm(pkg_name)</code></li>
<li>update a package: <code>IsoPkg.update(pkg_name)</code></li>
<li>update all installed packages: <code>IsoPkg.update()</code></li>
<li>show package status: <code>IsoPkg.status(pkg_name)</code></li>
<li>show status of all installed packages: <code>IsoPkg.status()</code></li>
<li>operate in a package environment: <code>@iso pkg_name statement</code></li>
<li>using/import package: <code>@iso using/import pkg_name</code></li>
<li>using/import package with the specified version: <code>@iso using/import pkg_name version</code></li>
<li>switch package group: <code>IsoPkg.swith(group_name)</code></li>
<li>pin the package version: <code>IsoPkg.pin(pkg_name)</code></li>
<li>free the package version: <code>IsoPkg.pin(pkg_name)</code></li>
</ul>

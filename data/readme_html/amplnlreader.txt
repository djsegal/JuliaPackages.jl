<p dir="auto">For an introduction to the AMPL modeling language, see</p>
<ul dir="auto">
<li>R. Fourer, D. M. Gay, and B. W. Kernighan, <a href="http://ampl.com/REFS/amplmod.pdf" rel="nofollow">AMPL: A Mathematical Programming Language</a>, Management Science 36, pp. 519-554, 1990.</li>
<li>R. Fourer, D. M. Gay, and B. W. Kernighan, <a href="http://ampl.com/BOOK/download.html" rel="nofollow">AMPL: A Modeling Language for Mathematical Programming</a>, Duxbury Press / Brooks/Cole Publishing Company, 2003.</li>
<li>D. Orban, <a href="https://gerad.ca/en/papers/G-2009-66" rel="nofollow">The Lightning AMPL Tutorial. A Guide for Nonlinear Optimization Users</a>, <a href="http://www.gerad.ca" rel="nofollow">GERAD</a> Technical Report G-2009-66, 2009.</li>
</ul>
<p dir="auto">Suppose you have an AMPL model represented by the model and data files <code>mymodel.mod</code> and <code>mymodel.dat</code>. Decode this model as a so-called <code>nl</code> file using</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="ampl -ogmymodel mymodel.mod mymodel.dat"><pre class="notranslate"><code>ampl -ogmymodel mymodel.mod mymodel.dat
</code></pre></div>

<p dir="auto"><code>AmplNLReader.jl</code> currently focuses on continuous problems conforming to <a href="https://github.com/JuliaSmoothOptimizers/NLPModels.jl"><code>NLPModels.jl</code></a>.</p>

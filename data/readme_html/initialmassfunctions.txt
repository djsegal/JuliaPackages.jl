<p dir="auto">Stellar initial mass functions describe the distribution of initial masses that stars are born with. This package aims to implement and provide interfaces for working with initial mass functions, including but not limited to evaluating and sampling from published distributions. See the linked documentation above for more details.</p>
<p dir="auto">Published IMFs we include are</p>

<p dir="auto">These all return subtypes of <a href="https://juliastats.org/Distributions.jl/latest/univariate/#univariates" rel="nofollow"><code>Distributions.ContinuousUnivariateDistribution</code></a> and have many of the typical methods from <a href="https://github.com/JuliaStats/Distributions.jl"><code>Distributions.jl</code></a> defined for them. These include</p>
<ul dir="auto">
<li>pdf, logpdf</li>
<li>cdf, ccdf</li>
<li><code>quantile(d, x::Real), quantile(d,x::AbstractArray), quantile!(y::AbstractArray, d, x::AbstractArray)</code></li>
<li>rand</li>
<li>minimum, maximum, extrema</li>
<li>mean, median, var, skewness, kurtosis (some of the higher moments don't work for <code>mmax=Inf</code> with instances of <code>BrokenPowerLaw</code>)</li>
</ul>
<p dir="auto">Note that <code>var</code>, <code>skewness</code>, and <code>kurtosis</code> are not currently defined for instances of <code>LogNormalBPL</code>, such as those returned by the <code>Chabrier2003</code> function.</p>
<p dir="auto">Many other functions that work on <a href="https://juliastats.org/Distributions.jl/latest/univariate/#univariates" rel="nofollow"><code>Distributions.ContinuousUnivariateDistribution</code></a> will also work transparently on these <code>AbstractIMF</code> instances.</p>
<p dir="auto">Continuous broken-power-law distributions (such as those used in <a href="https://ui.adsabs.harvard.edu/abs/2001ApJ...554.1274C/abstract" rel="nofollow">Chabrier 2001</a> and <a href="https://ui.adsabs.harvard.edu/abs/2001MNRAS.322..231K/abstract" rel="nofollow">Kroupa 2001</a>) are provided through a new <code>BrokenPowerLaw</code> type. The lognormal distribution for masses <code>m&lt;1.0</code> with a power law extension for higher masses as given in <a href="https://ui.adsabs.harvard.edu/abs/2003PASP..115..763C/abstract" rel="nofollow">Chabrier 2003</a> is provided by a new <code>LogNormalBPL</code> type. Simpler models (e.g., <code>Salpeter1955</code> and <code>Chabrier2001LogNormal</code>) are implemented using built-in distributions from <a href="https://github.com/JuliaStats/Distributions.jl"><code>Distributions.jl</code></a> in concert with their <a href="https://juliastats.org/Distributions.jl/stable/truncate/#Distributions.truncated" rel="nofollow"><code>truncated</code></a> function.</p>
<p dir="auto">Efficient samplers are implemented for the new types <code>BrokenPowerLaw</code> and <code>LogNormalBPL</code> such that batched calls (e.g., <code>rand(Chabrier2003(),1000)</code>) are more efficient than single calls (e.g., <code>d=Chabrier2003(); [rand(d) for i in 1:1000]</code>). These samplers can be created explicitly by calling <code>Distributions.sampler(d)</code>, with <code>d</code> being a <code>BrokenPowerLaw</code> or <code>LogNormalBPL</code> instance.</p>

<p dir="auto"><code>InitialMassFunctions.jl</code> follows Julia's <a href="https://pkgdocs.julialang.org/v1/compatibility/#compat-pre-1.0" rel="nofollow">recommended versioning strategy</a>, where breaking changes prior to version 1.0 will result in a bump of the minor version (e.g., 0.1.x |&gt; 0.2.0) whereas feature additions and bug patches will increment the patch version (0.1.0 |&gt; 0.1.1).</p>

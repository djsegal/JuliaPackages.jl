<p>Julia is a high-level, high-performance dynamic language for technical
computing.  The main homepage for Julia can be found at
<a href="https://julialang.org/" rel="nofollow">julialang.org</a>.  This is the GitHub
repository of Julia source code, including instructions for compiling
and installing Julia, below.</p>

<h2><a id="user-content-binary-installation" class="anchor" aria-hidden="true" href="#binary-installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Binary Installation</h2>
<p>If you would rather not compile the latest Julia from source,
platform-specific tarballs with pre-compiled binaries are also
<a href="https://julialang.org/downloads/" rel="nofollow">available for download</a>. The
downloads page also provides details on the
<a href="https://julialang.org/downloads/#support-tiers" rel="nofollow">different tiers of support</a>
for OS and platform combinations.</p>
<p>If everything works correctly, you will see a Julia banner and an
interactive prompt into which you can enter expressions for
evaluation.  You can read about <a href="https://docs.julialang.org/en/v1/manual/getting-started/" rel="nofollow">getting
started</a> in the manual.</p>
<p><strong>Note</strong>: Although some system package managers provide Julia, such
installations are neither maintained nor endorsed by the Julia
project. They may be outdated, broken and/or unmaintained. We
recommend you use the official Julia binaries instead.</p>
<h2><a id="user-content-building-julia" class="anchor" aria-hidden="true" href="#building-julia"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Building Julia</h2>
<p>First, make sure you have all the <a href="https://github.com/JuliaLang/julia/blob/master/doc/build/build.md#required-build-tools-and-external-libraries">required
dependencies</a> installed.
Then, acquire the source code by cloning the git repository:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="git clone https://github.com/JuliaLang/julia.git
"><pre><code>git clone https://github.com/JuliaLang/julia.git
</code></pre></div>
<p>By default you will be building the latest unstable version of
Julia. However, most users should use the most recent stable version
of Julia. You can get this version by changing to the Julia directory
and running:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="git checkout v1.6.1
"><pre><code>git checkout v1.6.1
</code></pre></div>
<p>Now run <code>make</code> to build the <code>julia</code> executable.</p>
<p>Building Julia requires 2GiB of disk space and approximately 4GiB of virtual memory.</p>
<p><strong>Note:</strong> The build process will fail badly if any of the build directory's parent directories have spaces or other shell meta-characters such as <code>$</code> or <code>:</code> in their names (this is due to a limitation in GNU make).</p>
<p>Once it is built, you can run the <code>julia</code> executable after you enter your julia directory and run</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="./julia
"><pre><code>./julia
</code></pre></div>
<p>Your first test of Julia determines whether your build is working
properly. From the UNIX/Windows command prompt inside the <code>julia</code>
source directory, type <code>make testall</code>. You should see output that
lists a series of running tests; if they complete without error, you
should be in good shape to start using Julia.</p>
<p>You can read about <a href="https://docs.julialang.org/en/v1/manual/getting-started/" rel="nofollow">getting
started</a>
in the manual.</p>
<p>In case this default build path did not work, detailed build instructions
are included in the <a href="https://github.com/JuliaLang/julia/blob/master/doc/build">build documentation</a>.</p>
<h3><a id="user-content-uninstalling-julia" class="anchor" aria-hidden="true" href="#uninstalling-julia"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Uninstalling Julia</h3>
<p>Julia does not install anything outside the directory it was cloned
into. Julia can be completely uninstalled by deleting this
directory. Julia packages are installed in <code>~/.julia</code> by default, and
can be uninstalled by deleting <code>~/.julia</code>.</p>

<p>The Julia REPL is quite powerful. See the section in the manual on
<a href="https://docs.julialang.org/en/v1/stdlib/REPL/" rel="nofollow">the Julia REPL</a>
for more details.</p>
<p>On Windows we highly recommend running Julia in a modern terminal,
such as <a href="https://aka.ms/terminal" rel="nofollow">Windows Terminal from the Microsoft Store</a>.</p>
<p>Support for editing Julia is available for many
<a href="https://github.com/JuliaEditorSupport">widely used editors</a>:
<a href="https://github.com/JuliaEditorSupport/julia-emacs">Emacs</a>,
<a href="https://github.com/JuliaEditorSupport/julia-vim">Vim</a>,
<a href="https://github.com/JuliaEditorSupport/Julia-sublime">Sublime Text</a>, and many
others.</p>
<p>Supported IDEs include: <a href="http://junolab.org/" rel="nofollow">Juno</a> (Atom plugin),
<a href="https://github.com/JuliaEditorSupport/julia-vscode">julia-vscode</a> (VS
Code plugin), and
<a href="https://github.com/JuliaEditorSupport/julia-intellij">julia-intellij</a>
(IntelliJ IDEA plugin). The popular <a href="https://jupyter.org/" rel="nofollow">Jupyter</a>
notebook interface is available through
<a href="https://github.com/JuliaLang/IJulia.jl">IJulia</a>.</p>

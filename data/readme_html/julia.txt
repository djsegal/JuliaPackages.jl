<p dir="auto">Julia is a high-level, high-performance dynamic language for technical
computing.  The main homepage for Julia can be found at
<a href="https://julialang.org/" rel="nofollow">julialang.org</a>.  This is the GitHub
repository of Julia source code, including instructions for compiling
and installing Julia, below.</p>

<h2 dir="auto"><a id="user-content-binary-installation" class="anchor" aria-hidden="true" href="#binary-installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Binary Installation</h2>
<p dir="auto">If you would rather not compile the latest Julia from source,
platform-specific tarballs with pre-compiled binaries are also
<a href="https://julialang.org/downloads/" rel="nofollow">available for download</a>. The
downloads page also provides details on the
<a href="https://julialang.org/downloads/#supported_platforms" rel="nofollow">different tiers of support</a>
for OS and platform combinations.</p>
<p dir="auto">If everything works correctly, you will see a Julia banner and an
interactive prompt into which you can enter expressions for
evaluation.  You can read about <a href="https://docs.julialang.org/en/v1/manual/getting-started/" rel="nofollow">getting
started</a> in the manual.</p>
<p dir="auto"><strong>Note</strong>: Although some system package managers provide Julia, such
installations are neither maintained nor endorsed by the Julia
project. They may be outdated, broken and/or unmaintained. We
recommend you use the official Julia binaries instead.</p>
<h2 dir="auto"><a id="user-content-building-julia" class="anchor" aria-hidden="true" href="#building-julia"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Building Julia</h2>
<p dir="auto">First, make sure you have all the <a href="https://github.com/JuliaLang/julia/blob/master/doc/src/devdocs/build/build.md#required-build-tools-and-external-libraries">required
dependencies</a> installed.
Then, acquire the source code by cloning the git repository:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="git clone https://github.com/JuliaLang/julia.git"><pre class="notranslate"><code>git clone https://github.com/JuliaLang/julia.git
</code></pre></div>
<p dir="auto">and then use the command prompt to change into the resulting julia directory. By default you will be building the latest unstable version of
Julia. However, most users should use the <a href="https://github.com/JuliaLang/julia/releases">most recent stable version</a>
of Julia. You can get this version by running:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="git checkout v1.9.0"><pre class="notranslate"><code>git checkout v1.9.0
</code></pre></div>
<p dir="auto">To build the <code>julia</code> executable, run <code>make</code> from within the julia directory.</p>
<p dir="auto">Building Julia requires 2GiB of disk space and approximately 4GiB of virtual memory.</p>
<p dir="auto"><strong>Note:</strong> The build process will fail badly if any of the build directory's parent directories have spaces or other shell meta-characters such as <code>$</code> or <code>:</code> in their names (this is due to a limitation in GNU make).</p>
<p dir="auto">Once it is built, you can run the <code>julia</code> executable. From within the julia directory, run</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="./julia"><pre class="notranslate"><code>./julia
</code></pre></div>
<p dir="auto">Your first test of Julia determines whether your build is working
properly. From the julia
directory, type <code>make testall</code>. You should see output that
lists a series of running tests; if they complete without error, you
should be in good shape to start using Julia.</p>
<p dir="auto">You can read about <a href="https://docs.julialang.org/en/v1/manual/getting-started/" rel="nofollow">getting
started</a>
in the manual.</p>
<p dir="auto">Detailed build instructions, should they be necessary,
are included in the <a href="https://github.com/JuliaLang/julia/blob/master/doc/src/devdocs/build/">build documentation</a>.</p>

<p dir="auto">By default, Julia does not install anything outside the directory it was cloned
into and <code>~/.julia</code>. Julia and the vast majority of Julia packages can be
completely uninstalled by deleting these two directories.</p>

<p dir="auto">The Julia REPL is quite powerful. See the section in the manual on
<a href="https://docs.julialang.org/en/v1/stdlib/REPL/" rel="nofollow">the Julia REPL</a>
for more details.</p>
<p dir="auto">On Windows we highly recommend running Julia in a modern terminal,
such as <a href="https://aka.ms/terminal" rel="nofollow">Windows Terminal from the Microsoft Store</a>.</p>
<p dir="auto">Support for editing Julia is available for many
<a href="https://github.com/JuliaEditorSupport">widely used editors</a>:
<a href="https://github.com/JuliaEditorSupport/julia-emacs">Emacs</a>,
<a href="https://github.com/JuliaEditorSupport/julia-vim">Vim</a>,
<a href="https://github.com/JuliaEditorSupport/Julia-sublime">Sublime Text</a>, and many
others.</p>
<p dir="auto">For users who prefer IDEs, we recommend using VS Code with the
<a href="https://www.julia-vscode.org/" rel="nofollow">julia-vscode</a> plugin.
For notebook users, <a href="https://jupyter.org/" rel="nofollow">Jupyter</a> notebook support is available through the
<a href="https://github.com/JuliaLang/IJulia.jl">IJulia</a> package, and
the <a href="https://github.com/fonsp/Pluto.jl">Pluto.jl</a> package provides Pluto notebooks.</p>

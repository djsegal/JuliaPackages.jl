<p dir="auto">For more info on Stan, please go to <a href="http://mc-stan.org" rel="nofollow">http://mc-stan.org</a>.</p>
<p dir="auto">The Julia package CmdStan.jl is based on Stan's command line interface, 'cmdstan'.</p>
<p dir="auto">The 'cmdstan' interface needs to be installed separatedly. Please see <a href="https://github.com/StanJulia/CmdStan.jl/blob/master/docs/src/INSTALLATION.md">cmdstan installation</a> for further details.</p>
<p dir="auto">The location of the cmdstan executable and related programs is now obtained from the environment variable JULIA_CMDSTAN_HOME. This used to be CMDSTAN_HOME.</p>
<p dir="auto">Right now <code>versioninfo()</code> will show its setting (if defined).</p>

<ol dir="auto">
<li>Fixed an issue related to naming of created files. Thanks to mkshirazi.</li>
<li>Several bumps to deal with package versions.</li>
<li>Re-enabling Coverage.</li>
</ol>
<p dir="auto">Release 6.0.2</p>
<ol dir="auto">
<li>Init files were not properly included in cmd. Thanks to ueliwechsler and andrstef.</li>
</ol>
<p dir="auto">Release 6.0.1</p>
<ol dir="auto">
<li>Removed dependency on Documenter.</li>
</ol>
<p dir="auto">Release 6.0.0 contains:</p>
<ol dir="auto">
<li>Revert back to output an array by default.</li>
<li>Switch to Requires.jl to delay/prevent loading of MCMCChains if not needed (thanks to suggestions by @Byrth and Stijn de Waele).</li>
<li>Updates to documentation.</li>
</ol>
<p dir="auto">Release 6.0.0 is a breaking release.</p>
<p dir="auto">To revert back to v5.x behavior a script needs to include <code>using MCMCChains</code> (which thus must be installed) and specify <code>output_format=:mcmcchains</code> in the call to <code>stanmodel()</code>. This option is not tested on Travis. A sub-directory examples_mcmcchains has been added which demonstrate this usage pattern.</p>
<p dir="auto">CmdStan.jl tested on cmdstan v2.21.0.</p>

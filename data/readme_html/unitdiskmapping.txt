<p dir="auto">
<code>UnitDiskMapping</code> is a  
    <a href="https://julialang.org" rel="nofollow">
        <img src="https://raw.githubusercontent.com/JuliaLang/julia-logo-graphics/master/images/julia.ico" width="16em" style="max-width: 100%;">
        Julia Language
    </a>
      package for reducing a <a href="https://en.wikipedia.org/wiki/Independent_set_(graph_theory)" rel="nofollow">generic maximum (weighted) independent set problem</a>, <a href="https://en.wikipedia.org/wiki/Quadratic_unconstrained_binary_optimization" rel="nofollow"> quadratic unconstrained binary optimization (QUBO) problem</a> or <a href="https://en.wikipedia.org/wiki/Integer_factorization" rel="nofollow">integer factorization problem</a> to a maximum independent set problem on a unit disk grid graph (or hardcore lattice gas in physics), which can then be naturally encoded in neutral-atom quantum computers. To install <code>UnitDiskMapping</code>,
    please <a href="https://docs.julialang.org/en/v1/manual/getting-started/" rel="nofollow">open
    Julia's interactive session (known as REPL)</a> and press the <kbd>]</kbd> key in the REPL to use the package mode, and then type the commands below.
</p>
<p dir="auto">For installing the current master branch, please type:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pkg&gt; add https://github.com/QuEraComputing/UnitDiskMapping.jl.git"><pre>pkg<span class="pl-k">&gt;</span> add https<span class="pl-k">:</span><span class="pl-k">//</span>github<span class="pl-k">.</span>com<span class="pl-k">/</span>QuEraComputing<span class="pl-k">/</span>UnitDiskMapping<span class="pl-k">.</span>jl<span class="pl-k">.</span>git</pre></div>
<p dir="auto">For stable release (not yet ready):</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pkg&gt; add UnitDiskMapping"><pre>pkg<span class="pl-k">&gt;</span> add UnitDiskMapping</pre></div>

<p dir="auto">Please check this <a href="https://queracomputing.github.io/UnitDiskMapping.jl/notebooks/tutorial.html" rel="nofollow">notebook</a>, which contains the following examples:</p>
<ul dir="auto">
<li>Reduction from a generic weighted or unweighted maximum independent set (MIS) problem to that on a diagonal coupled unit-disk grid graph (DUGG).</li>
<li>Reduction from a generic or square-lattice QUBO problem to an MIS problem on a unit-disk grid graph.</li>
<li>Reduction from an integer factorization problem to an MIS problem on a unit-disk grid graph.</li>
</ul>
<p dir="auto"><a target="_blank" rel="noopener noreferrer nofollow" href="https://user-images.githubusercontent.com/6257240/198861111-4499c17d-9938-406b-8253-943b01f4633c.png"><img src="https://user-images.githubusercontent.com/6257240/198861111-4499c17d-9938-406b-8253-943b01f4633c.png" alt="" style="max-width: 100%;"></a></p>
<p dir="auto">To run the notebook locally, you will need the <a href="https://github.com/fonsp/Pluto.jl">Pluto</a> and <a href="https://github.com/QuEraComputing/GenericTensorNetworks.jl">GenericTensorNetworks</a> Julia packages installed. You can run the following after entering the Package mode:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="pkg&gt; add Pluto
pkg&gt; add GenericTensorNetworks"><pre class="notranslate"><code>pkg&gt; add Pluto
pkg&gt; add GenericTensorNetworks
</code></pre></div>
<p dir="auto">and returning to the Julia REPL (you can do this by hitting Backspace in the Package mode) to run:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; import Pluto; Pluto.run()"><pre class="notranslate"><code>julia&gt; import Pluto; Pluto.run()
</code></pre></div>
<p dir="auto">in the <code>notebooks</code> directory of this project. At this point, your browser should automatically launch and display a list of available notebooks you can run. You should just see <code>tutorial.jl</code> listed.</p>

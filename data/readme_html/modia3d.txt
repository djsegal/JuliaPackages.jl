<h2><a id="user-content-modia-platform" class="anchor" aria-hidden="true" href="#modia-platform"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Modia Platform</h2>
<p>The <em>Modia platform</em> is a prototype system for the next modeling and simulation generation of physical systems described by differential and algebraic equations. It consists currently of the following Julia packages that are all under development (not all are yet publicly available):</p>
<ul>
<li><em><a href="https://github.com/ModiaSim/Modia.jl">Modia</a></em> - Equation based modeling</li>
<li><em>Modiator</em> - 2D/3D web-app model editor</li>
<li><em><a href="https://github.com/ModiaSim/ModiaMath.jl">ModiaMath</a></em> - Simulation enviroment</li>
<li><em><a href="https://github.com/ModiaSim/Modia3D.jl">Modia3D</a></em> - 3D geometry and 3D mechanics</li>
<li><em><a href="https://github.com/ModiaSim/ModiaMedia.jl">ModiaMedia</a></em> - Thermodynamic property models</li>
<li><em>Modelia</em> - <a href="https://www.modelica.org/modelicalanguage" rel="nofollow">Modelica model</a> importer</li>
</ul>
<h2><a id="user-content-modia3d-1" class="anchor" aria-hidden="true" href="#modia3d-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Modia3D</h2>
<p>Modia3D provides 3D geometry to physical systems so that geometrical objects can be
directly accessed and utilized in a model. Functions are provided, for example, to compute the volume,
mass, and inertia of a geometrical object or the distance between two objects.
Furthermore, Modia3D models 3D mechanical systems and shall be expanded into other domains in the future
(for example to utilize the 3D geometry to model heat flow in buildings or satellites).
In the future it will be possible, for example, to model the 3D mechanical part of a robot with Modia3D and the electrical motors and gearboxes that are driving the joints with Modia.</p>
<p>Modia3D uses ideas from modern computer game engines to achieve a highly flexible setup of mechanical systems including collision handling. Other features are utilized from multi-body programs, such as support for closed kinematic loops, and elastic response calculation. The underlying mathematical formulation are hybrid Differential Algebraic Equations (DAEs) that are solved with the variable-step solver IDA via the <a href="https://github.com/JuliaDiffEq/Sundials.jl">Sundials.jl</a> Julia package.</p>
<p>Collision handling with elastic response calculation is performed for geometrical objects that are defined with a contact material and have a convex geometry or are approximated by the convex hull of a concave geometry.
Penetration depths and Euclidean distances are computed with the improved Minkowski Portal Refinement (MPR) algorithm [1].
The details of the contact law are provided in [5] and several examples are discussed in [4]. The user's view of Modia3D is introduced in [2] showing the very flexible definition of 3D systems. Some key algorithms are discussed in [3].
A more detailed overview of the available features is also given in the <a href="https://ModiaSim.github.io/Modia3D.jl/stable" rel="nofollow">Modia3D documentation</a>.</p>

<h3><a id="user-content-3d-renderer-and-work-in-progress-web-app" class="anchor" aria-hidden="true" href="#3d-renderer-and-work-in-progress-web-app"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>3D Renderer and Work-In-Progress Web App</h3>
<p>Modia3D provides a generic interface to visualize simulation results with different 3D renderers. The <a href="https://visualization.ltx.de/" rel="nofollow">free community edition</a> as well as the professional edition of the
<a href="http://www.systemcontrolinnovationlab.de/the-dlr-visualization-library/" rel="nofollow">DLR Visualization library</a> are supported. Currently, another team is developing the free 2D/3D web-based authoring tool <em>Modiator</em> that includes result visualization, but it is not yet publicly available.</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p><strong>Modia3D</strong> is registered in METADATA.jl and can be installed in the following way (Julia &gt;= 1.0 is required):</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> ]add Modia3D
        add ModiaMath  <span class="pl-c"><span class="pl-c">#</span> in order to use simulate!(..) and plot(..)</span>
        add PyPlot     <span class="pl-c"><span class="pl-c">#</span> in order that plots are shown</span></pre></div>
<p>Modia3D uses <a href="https://github.com/JuliaPy/PyPlot.jl">PyPlot</a> for plotting. If <code>PyPlot</code> is not available in your current Julia environment an information message is printed and all <code>plot(..)</code> calls are ignored.</p>
<p>In order that plot windows are displayed, you need to add <code>PyPlot</code> to your current environment via <code>]add PyPlot</code>. Often this automatic installation fails and it is recommended to follow the instructions <a href="https://github.com/ModiaSim/ModiaMath.jl/wiki/Installing-PyPlot-in-a-robust-way">Installing PyPlot in a robust way</a>.</p>
<p>Modia3D visualizes the movement of 3D objects with a renderer. Currently, the (free) community or the (commercial) professional version of the <a href="http://www.systemcontrolinnovationlab.de/the-dlr-visualization-library/" rel="nofollow">DLR Visualization</a> library are supported. To install the free version for <em>Windows</em> or for <em>Linux</em> perform the following steps:</p>
<ol>
<li>
<p>Go to <a href="https://visualization.ltx.de/" rel="nofollow">https://visualization.ltx.de/</a>, provide your contact information and click on <em>Request download</em> for <em>Community Edition</em>. Afterwards, you get a link to download the library and you need to unzip the file.</p>
</li>
<li>
<p>In your Julia <strong>startup file</strong> (<code>HOME/.julia/config/startup.jl</code>) include the environment variable <code>ENV["DLR_VISUALIZATION"] = "&lt;path-to-library&gt;/Visualization/Extras/SimVis"</code>. Make sure that the SimVis executable under this directory has execution rights. For example in Linux with command:<code>chmod +x &lt;path-to-library&gt;/Visualization/Extras/SimVis</code></p>
</li>
<li>
<p>Start Julia and run one of the examples, for example   <code>include("$(Modia3D.path)/examples/dynamics/Simulate_DoublePendulumWithDampers.jl")</code></p>
</li>
</ol>
<p>If Modia3D cannot use one of the renderers above, it will continue with renderer <strong>NoRenderer</strong> where result animation is switched off.</p>
<h2><a id="user-content-example" class="anchor" aria-hidden="true" href="#example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Example</h2>
<h3><a id="user-content-define-a-simple-pendulum-model" class="anchor" aria-hidden="true" href="#define-a-simple-pendulum-model"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Define a simple pendulum model</h3>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">import</span> ModiaMath  <span class="pl-c"><span class="pl-c">#</span> Simulation environment</span>
<span class="pl-k">using</span>  Modia3D

visuMaterial1 <span class="pl-k">=</span> Modia3D<span class="pl-k">.</span><span class="pl-c1">Material</span>(color<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>LightBlue<span class="pl-pds">"</span></span>, transparency<span class="pl-k">=</span><span class="pl-c1">0.3</span>);
visuMaterial2 <span class="pl-k">=</span> Modia3D<span class="pl-k">.</span><span class="pl-c1">Material</span>(color<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>Red<span class="pl-pds">"</span></span>);

<span class="pl-c1">@assembly</span> <span class="pl-c1">Pendulum</span>(;Lx <span class="pl-k">=</span> <span class="pl-c1">1.0</span>, Ly<span class="pl-k">=</span><span class="pl-c1">0.2</span><span class="pl-k">*</span>Lx, Lz<span class="pl-k">=</span><span class="pl-c1">0.2</span><span class="pl-k">*</span>Lx) <span class="pl-k">begin</span>
   world    <span class="pl-k">=</span> <span class="pl-c1">Object3D</span>(Modia3D<span class="pl-k">.</span><span class="pl-c1">CoordinateSystem</span>(<span class="pl-c1">0.5</span><span class="pl-k">*</span>Lx))
   beam     <span class="pl-k">=</span> <span class="pl-c1">Object3D</span>(Modia3D<span class="pl-k">.</span><span class="pl-c1">Solid</span>(Modia3D<span class="pl-k">.</span><span class="pl-c1">SolidBeam</span>(Lx,Ly,Lz), <span class="pl-s"><span class="pl-pds">"</span>Aluminium<span class="pl-pds">"</span></span>, visuMaterial1))
   bearing  <span class="pl-k">=</span> <span class="pl-c1">Object3D</span>(beam; r<span class="pl-k">=</span>[<span class="pl-k">-</span>Lx<span class="pl-k">/</span><span class="pl-c1">2</span>, <span class="pl-c1">0.0</span>, <span class="pl-c1">0.0</span>])
   cylinder <span class="pl-k">=</span> <span class="pl-c1">Object3D</span>(bearing,Modia3D<span class="pl-k">.</span><span class="pl-c1">Cylinder</span>(Ly<span class="pl-k">/</span><span class="pl-c1">2</span>,<span class="pl-c1">1.2</span><span class="pl-k">*</span>Ly; material<span class="pl-k">=</span>visuMaterial2))
   revolute <span class="pl-k">=</span> Modia3D<span class="pl-k">.</span><span class="pl-c1">Revolute</span>(world, bearing)
<span class="pl-k">end</span>;
simulationModel <span class="pl-k">=</span> Modia3D<span class="pl-k">.</span><span class="pl-c1">SimulationModel</span>(<span class="pl-c1">Pendulum</span>(Lx<span class="pl-k">=</span><span class="pl-c1">0.8</span>),stopTime<span class="pl-k">=</span><span class="pl-c1">5.0</span>);</pre></div>
<p>Two visualization materials (<code>visuMaterial1, visuMaterial2</code>) are defined, and optional parameters (<code>color, transparency</code>) are set. There are further optional parameters, but they are not used here.</p>
<p>The hierarchical structure of the pendulum is defined with the <code>@assembly</code> macro. A <code>SimulationModel</code> defines the top assembly and other properties needed for simulation (e.g. a default value for the <code>stopTime</code>).</p>
<p>Constructor <code>Object3D</code> generates a coordinate system in 3D with optional associated data, defining
for example a geometry and/or mass properties.
If the first argument of the constructor is another <code>Object3D</code>, then the generated <code>Object3D</code>
is defined relatively to the coordinate system of its first argument.
Function <code>Revolute</code> generates a revolute joint that constrains the movement of two <code>Object3Ds</code>.</p>

<h2><a id="user-content-status" class="anchor" aria-hidden="true" href="#status"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Status</h2>
<p>The package has been tested with Julia on Windows 7, via the TravisCL on Linux (x86_64-pc-linux-gnu) and macOS (x86_64-apple-darwin14.5.0) and via the Appveyor CL on Windows.</p>
<p>Note, the collision handling is currently reasonably working for contacts between spheres as well as spheres and boxes. Still improvements are needed. Furthermore, kinematic loops are currently only supported for 2D loops and if they are driven kinematically. The technique to handle kinematic loops for dynamic simulations is demonstrated in the example <code>include("$(ModiaMath.path)/examples/withoutMacros_withoutVariables/Simulate_PendulumDAE.jl")</code> and is described in the paper <em><a href="http://www.ep.liu.se/ecp/132/064/ecp17132565.pdf" rel="nofollow">Transformation of Differential Algebraic Array Equations to Index One Form</a></em>.</p>
<p>Up to now, Modia3D is implemented for functionality and not tuned for efficiency. Therefore, there are no benchmarks yet and in particular no comparison with Modelica models.</p>

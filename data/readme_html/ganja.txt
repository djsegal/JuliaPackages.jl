<h2><a id="user-content-ganjajs---geometric-algebra-for-javascript" class="anchor" aria-hidden="true" href="#ganjajs---geometric-algebra-for-javascript"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>ganja.js - Geometric Algebra for javascript.</h2>
<p><strong>G</strong>eometric <strong>A</strong>lgebra - <strong>N</strong>ot <strong>J</strong>ust <strong>A</strong>lgebra</p>
<p>Ganja.js is a Geometric Algebra code generator for javascript. It generates
Clifford algebras and sub-algebras of any signature and implements operator
overloading and algebraic constants.</p>
<p>(<strong>Mathematically</strong>, an algebra generated by ganja.js is a graded exterior (Grassmann) algebra
(or one of its subalgebras) with a non-metric outer product, extended (Clifford) with geometric and contraction inner products, a Poincare duality operator and the main
involutions and morphisms.)</p>
<p>(<strong>Technically</strong>, ganja.js is a code generator producing classes that reificate algebraic literals
and expressions by using reflection, a built-in tokenizer and a simple AST translator to
rewrite functions containing algebraic constructs to their procedural counterparts.)</p>
<p>(<strong>Practically</strong>, ganja.js enables real math syntax inside javascript, with element, vector and matrix
operations over <strong>reals</strong>, <strong>complex numbers</strong>, <strong>dual numbers</strong>, <strong>hyperbolic numbers</strong>, <strong>vectors</strong>, <strong>spacetime events</strong>, <strong>quaternions</strong>, <strong>dual quaternions</strong>, <strong>biquaternions</strong> or <strong>any other Clifford Algebra</strong>.)</p>
<p>(<strong>Seriously</strong>, look at the <a href="https://enkimute.github.io/ganja.js/examples/coffeeshop.html" rel="nofollow">examples</a>,
run some quick numbers using the <a href="https://enkimute.github.io/ganja.js/examples/galculator.html" rel="nofollow">GAlculator</a>
or play <a href="https://enkimute.github.io/ganja.js/examples/example_game_wedge.html" rel="nofollow">the wedge game</a> first.)</p>
<h2><a id="user-content-grassmannjl---geometric-algebra-for-julia-language" class="anchor" aria-hidden="true" href="#grassmannjl---geometric-algebra-for-julia-language"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Grassmann.jl - Geometric Algebra for Julia language</h2>
<p>The <a href="https://github.com/chakravala/Grassmann.jl">Grassmann.jl</a> package provides tools for doing computations based on multi-linear algebra, differential geometry, and spin groups using the extended tensor algebra known as Leibniz-Grassmann-Clifford-Hestenes geometric algebra.
Combinatorial products include <code>∧, ∨, ⋅, *, ⋆, ', ~, ∂</code> (which are the exterior, regressive, inner, and geometric products; along with the Hodge star, adjoint, reversal, and boundary operators).
The kernelized operations are built up from composite sparse tensor products and Hodge duality, with high dimensional support for up to 62 indices using staged caching and precompilation. Code generation enables concise yet highly extensible definitions.
The <a href="https://github.com/chakravala/DirectSum.jl">DirectSum.jl</a> multivector parametric type polymorphism is based on tangent bundle vector spaces and conformal projective geometry to make the dispatch highly extensible for many applications.
Additionally, the universal interoperability between different sub-algebras is enabled by <a href="https://github.com/chakravala/AbstractTensors.jl">AbstractTensors.jl</a>, on which the type system is built.</p>

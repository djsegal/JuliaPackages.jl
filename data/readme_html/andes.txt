<p dir="auto"><code>Andes.jl</code> is the Julia API for ANDES, a power system simulation tool for symbolic modeling and numerical simulation.</p>
<p dir="auto"><code>Andes.jl</code> provides APIs through <code>PyCall</code> and <code>Conda</code> for calling ANDES from Julia.</p>

<p dir="auto">If you have installed andes in an existing Python environment and do not want to reinstall it from conda, you can configure it in Pycall.</p>
<p dir="auto">You can manually set the Python path with the following commands in Julia:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="ENV[&quot;PYTHON&quot;] = &quot;... path of the python executable ...&quot;

# ENV[&quot;PYTHON&quot;] = &quot;/usr/bin/python3.7&quot;                  # example for *nix
# ENV[&quot;PYTHON&quot;] = &quot;/home/name/miniconda3/envs/andes&quot;    # example for conda
Pkg.build(&quot;PyCall&quot;)"><pre class="notranslate"><code>ENV["PYTHON"] = "... path of the python executable ..."
# ENV["PYTHON"] = "C:\\Python37-x64\\python.exe"        # example for Windows

</code></pre></div>
<p dir="auto">Check out the <a href="https://github.com/JuliaPy/PyCall.jl#specifying-the-python-version">documentation</a> of <code>PyCall.jl</code> for more details.</p>

<p dir="auto">A Julia package for probability distributions and associated functions. Particularly, <em>Distributions</em> implements:</p>
<ul dir="auto">
<li>Moments (e.g mean, variance, skewness, and kurtosis), entropy, and other properties</li>
<li>Probability density/mass functions (pdf) and their logarithm (logpdf)</li>
<li>Moment generating functions and characteristic functions</li>
<li>Sampling from a population or from a distribution</li>
<li>Maximum likelihood estimation</li>
</ul>
<p dir="auto"><strong>Note:</strong> The functionalities related to conjugate priors have been moved to the <a href="https://github.com/JuliaStats/ConjugatePriors.jl">ConjugatePriors package</a>.</p>

<ul dir="auto">
<li>If you need help or an explanation of how to use <em>Distributions</em> ask in the forum (<a href="https://discourse.julialang.org" rel="nofollow">https://discourse.julialang.org</a>) or, for informal questions, visit the chat (<a href="https://julialang.slack.com" rel="nofollow">https://julialang.slack.com</a>).</li>
</ul>
<p dir="auto">If you have a bug linked with <em>Distributions</em>, check that it has
not been reported yet on the issues of the repository.
If not, you can file a new issue, add your version of the package
which you can get with this command in the Julia REPL:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; ]status Distributions"><pre>julia<span class="pl-k">&gt;</span> ]status Distributions</pre></div>
<p dir="auto">Be exhaustive in your report, summarize the bug, and provide:
a Minimal Working Example (MWE), what happens, and what you
expected to happen.</p>

<p dir="auto">To contribute to the package, fork the repository on GitHub,
clone it and make modifications on a new branch,
<strong>do not commit modifications on master</strong>.
Once your changes are made, push them on your fork and create the
Pull Request on the main repository.</p>

<p dir="auto">Distributions is a central package which many rely on,
the following are required for contributions to be accepted:</p>
<ol dir="auto">
<li>
<p dir="auto">Docstrings must be added to all interface and non-trivial functions.</p>
</li>
<li>
<p dir="auto">Tests validating the modified behavior in the <code>test</code> folder. If new test files are added, do not forget to add them in <code>test/runtests.jl</code>. Cover possible edge cases. Run the tests locally before submitting the PR.</p>
</li>
<li>
<p dir="auto">At the end of the tests, <code>Test.detect_ambiguities(Distributions)</code> is run to check method ambiguities. Verify that your modified code did not yield method ambiguities.</p>
</li>
<li>
<p dir="auto">Make corresponding modifications to the <code>docs</code> folder, build the documentation locally and verify that your modifications display correctly and did not yield warnings. To build the documentation locally, you first need to instantiate the <code>docs/</code> project:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia --project=docs/
pkg&gt; instantiate
pkg&gt; dev ."><pre class="notranslate"><code>julia --project=docs/
pkg&gt; instantiate
pkg&gt; dev .
</code></pre></div>
<p dir="auto">Then use <code>julia --project=docs/ docs/make.jl</code> to build the documentation.</p>
</li>
</ol>

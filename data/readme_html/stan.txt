<h2 dir="auto"><a id="user-content-purpose" class="anchor" aria-hidden="true" href="#purpose"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Purpose</h2>
<p dir="auto">A collection of example Stan Language programs demonstrating all methods available in Stan's cmdstan executable (as an external program) from Julia.</p>
<p dir="auto">For most applications one of the "single method" packages, e.g. StanSample.jl, StanDiagnose.jl, etc., is a better choice for day to day use.</p>
<p dir="auto">To execute the most important method in Stan ("sample"), use StanSample.jl.</p>
<p dir="auto">Some Pluto notebook examples can be found <a href="https://github.com/StanJulia/StanExampleNotebooks.jl">here</a>.</p>
<h2 dir="auto"><a id="user-content-background-info" class="anchor" aria-hidden="true" href="#background-info"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Background info</h2>
<p dir="auto">Stan.jl has always taken the approach as the recently released <a href="https://mc-stan.org/cmdstanr/" rel="nofollow">CmdStanR</a> and <a href="https://github.com/stan-dev/cmdstanpy">CmdStanPy</a> options to use Stan's <a href="https://mc-stan.org/users/interfaces/cmdstan.html" rel="nofollow">cmdstan executable</a>.</p>
<p dir="auto">In Stan.jl v7 all of cmdstan's methods were moved to separate packages, i.e. StanSample.jl, StanOptimize.jl, StanVariational.jl and StanDiagnose.jl, including an option to run <code>generate_quantities</code> as part of StanSample.jl.</p>
<p dir="auto">Stan.jl v10 uses StanSample.jl v7, StanOptimize.jl v4, StanQuap.jl v4, StanDiagnose.jl v4 and StanVariational v4 and supports multithreading on C++ level. Stan.jl v10 also uses JSON.jl to generate data and init input files for cmdstan.</p>
<p dir="auto">Stan.jl v10 supports InferenceObjects.jl and PosteriorDB.jl. See the notebook examples <a href="https://github.com/StanJulia/StanExampleNotebooks.jl">here</a>.</p>
<p dir="auto">To use the :dimarray or :dimarrays option in <code>read_samples())</code>, see the example notebook <code>dimarray.jl</code>.</p>
<p dir="auto">The StanJulia ecosystem includes 2 additional packages, StanQuap.jl (to compute <a href="https://en.wikipedia.org/wiki/Maximum_a_posteriori_estimation" rel="nofollow">MAP</a> estimates) and DiffEqBayesStan.jl.</p>
<h2 dir="auto"><a id="user-content-requirements" class="anchor" aria-hidden="true" href="#requirements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Requirements</h2>
<p dir="auto"><strong>Stan's cmdstan executable needs to be installed separatedly.</strong> Please see <a href="https://stanjulia.github.io/Stan.jl/latest/INSTALLATION/" rel="nofollow">cmdstan installation</a>. If you plan to use C++ level threads, please read the <code>make/local-example</code> instructions and below section and <a href="https://github.com/StanJulia/StanSample.jl/blob/master/INSTALLING_CMDSTAN.md">this file</a>.</p>
<h2 dir="auto"><a id="user-content-options-for-multi-threading-and-multi-chaining" class="anchor" aria-hidden="true" href="#options-for-multi-threading-and-multi-chaining"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Options for multi-threading and multi-chaining</h2>
<p dir="auto">Stan.jl v10 is intended to use Stan's <code>cmdstan</code> v2.28.2+ and StanSample.jl v6.</p>
<p dir="auto">StanSample.jl v6+ enables the use of c++ multithreading in the <code>cmdstan</code> binary. To activate multithreading in <code>cmdstan</code> this needs to be specified during the build process of <code>cmdstan</code>. I typically create a <code>path_to_cmdstan_directory/make/local</code> file (before running <code>make -j9 build</code>) containing <code>STAN_THREADS=true</code>. For an example, see the <code>.github/CI.yml</code> script</p>
<p dir="auto">This means StanSample supports 2 mechanisms for in parallel drawing samples for chains, i.e. on C++ level (using C++ threads) and on Julia level (by spawing a Julia process for each chain).</p>
<p dir="auto">The <code>use_cpp_chains</code> keyword argument for <code>stan_sample()</code> determines if chains are executed on C++ level or on Julia level. By default, <code>use_cpp_chains=false</code>.</p>
<p dir="auto">By default in ether case <code>num_chains=4</code>. See <code>??stan_sample</code>. Internally, <code>num_chains</code> will be copied to either <code>num_cpp_chains</code> or <code>num_julia_chains'.</code></p>
<p dir="auto"><strong>Note:</strong> Currently I do not suggest to use both C++ level chains and Julia level chains. Based on  <code>use_cpp_chains</code> the <code>stan_sample()</code> method will set either <code>num_cpp_chains=num_chains; num_julia_chains=1</code> or <code>num_julia_chains=num_chains;num_cpp_chain=1</code> (the default of <code>use_cpp_chains</code> is false).</p>
<p dir="auto">Set the <code>check_num_chains</code> keyword argument in the call to <code>stan_sample()</code> to <code>false</code> to prevent above default behavior. See the example in the <code>Examples/RedCardsStudy</code> directory for more details and an example.</p>
<p dir="auto">Threads on C++ level can be used in multiple ways, e.g. to run separate chains and to speed up certain Stan Language operations.</p>
<p dir="auto">StanSample.jl's SampleModel sets the C++ <code>num_threads</code> to 4 but for compatibility with previous versions of StanJulia this is by default (<code>use_cpp_chains=false</code>) not included in the generated command line, e.g. see <code>sm.cmds</code> where <code>sm</code> is your SampleModel.</p>
<p dir="auto">An example of the possible performance trade-offs between <code>use_cpp_threads</code>, <code>num_cpp_chains</code> and <code>num_julia_chains</code> can be found in the <a href="https://github.com/StanJulia/Stan.jl/tree/master/Examples/RedCardsStudy/graphs">this</a> directory.</p>
<h3 dir="auto"><a id="user-content-conda-based-installation-walkthrough-for-running-stan-from-julia-on-windows" class="anchor" aria-hidden="true" href="#conda-based-installation-walkthrough-for-running-stan-from-julia-on-windows"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Conda based installation walkthrough for running Stan from Julia on Windows</h3>
<p dir="auto"><strong>Note 1:</strong> The conda way of installing also works on other platforms. See <a href="https://mc-stan.org/docs/2_28/cmdstan-guide/index.html" rel="nofollow">also</a>.</p>
<p dir="auto"><strong>Note 2:</strong> I believe if you have used CmdstanR (or CmdstanPy) to install cmdstan you can use that cmdstan version in Julia.</p>
<p dir="auto">Make sure you have conda installed on your system and available from the command line (you can use the conda version that comes with Conda.jl or install your own).</p>
<p dir="auto">Activate the conda environment into which you want to install cmdstan (e.g. run <code>conda activate stan-env</code> from the command line) or create a new environment (<code>conda create --name stan-env</code>) and then activate it.</p>
<p dir="auto">Install cmdstan into the active conda environment by running <code>conda install -c conda-forge cmdstan</code>.</p>
<p dir="auto">You can check that cmdstan, g++, and mingw32-make are installed properly by running <code>conda list cmdstan, g++ --version</code> and <code>mingw32-make --version</code>, respectively, from the activated conda environment.</p>
<p dir="auto">Start a Julia session from the conda environment in which cmdstan has been installed (this is necessary for the cmdstan installation and the tools to be found).</p>
<p dir="auto">Add the StanSample.jl package by running ] add StanSample from the REPL.</p>
<p dir="auto">Set the CMDSTAN environment variable so that Julia can find the cmdstan installation, e.g. from the Julia REPL do: ENV["CMDSTAN"] = "C:/Users/Jakob/.julia/conda/3/envs/stan-env/Library/bin/cmdstan" This needs to be set before you load the StanSample package by e.g. using it. You can add this line to your startup.jl file so that you don't have to run it again in every fresh Julia session.</p>

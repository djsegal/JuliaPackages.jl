<p dir="auto">A simple package to add <a href="https://github.com/domluna/JuliaFormatter.jl">JuliaFormatter</a> autoformatting support to Jupyter lab and notebooks running <a href="https://github.com/JuliaLang/IJulia.jl">IJulia.</a>  Inspired by my favorite Jupyter extension, <a href="https://github.com/dnanhkhoa/nb_black">nb_black.</a></p>

<p dir="auto">In your notebook,</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using JupyterFormatter
enable_autoformat()"><pre><span class="pl-k">using</span> JupyterFormatter
<span class="pl-c1">enable_autoformat</span>()</pre></div>
<p dir="auto">Afterwards, whenever you run a cell, it will automatically be formatted.  If you want
to turn off auto-formatting, run</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="disable_autoformat()"><pre class="notranslate"><code>disable_autoformat()
</code></pre></div>

<p dir="auto">While this package works with both legacy Jupyter Notebooks and Jupyter Lab, it does have some issues in Jupyter Notebooks.  <strong>On legacy notebooks only</strong>, when a cell is formatted its output will be suppressed and it will lose its <code>In[]</code> and <code>Out[]</code>.  This will only happen if the cell is not already formatted properly.</p>

<p dir="auto">However, running this unformatted cell will not print properly; running this cell</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="assets/unformatted_will_suppress.png"><img src="assets/unformatted_will_suppress.png" alt="Running With Formatted Cell" style="max-width: 100%;"></a></p>

<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="assets/unformatted_suppresses_output.png"><img src="assets/unformatted_suppresses_output.png" alt="Running With Formatted Cell" style="max-width: 100%;"></a></p>

<p>Exported dictionaries include: <code>NordColours</code>, <code>PolarNight</code>, <code>SnowStorm</code>, <code>Frost</code>, <code>Aurora</code>.
These match those described on the original nord project.</p>
<h3><a id="user-content-convenience-types" class="anchor" aria-hidden="true" href="#convenience-types"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Convenience types</h3>
<p>Within the <code>Nord</code> module, you can access the following convenience variables:</p>
<ul>
<li>all of the nord colours, e.g. <code>nord0</code>, <code>nord8</code>, etc.</li>
<li>colours: <code>black</code>, <code>grey</code>, <code>white</code>, <code>blue</code>, <code>red</code>, <code>orange</code>, <code>yellow</code>, <code>green</code></li>
<li>debugging: <code>info</code>, <code>debug</code>, <code>pass</code>, <code>warn</code>, <code>fail</code>, <code>error</code></li>
</ul>

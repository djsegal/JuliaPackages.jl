<h2 dir="auto"><a id="user-content-kml-objects--julia-structs" class="anchor" aria-hidden="true" href="#kml-objects--julia-structs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>KML Objects ←→ Julia structs</h2>
<p dir="auto">This package is designed to be used intuitively alongside <a href="https://developers.google.com/kml/documentation/kmlreference" rel="nofollow">Google's KML Reference Page</a>.  Thus, there are rules that guide the mapping between KML (XML) Objects and Julia structs.</p>
<ol dir="auto">
<li>In Julia, each <code>Object</code> is constructed with keyword arguments only.</li>
<li>Keywords are the associated attributes as well as child elements of the <code>Object</code>
<ul dir="auto">
<li>E.g. <code>pt = Point(id="mypoint", coordinates=(0,1))</code> sets the <code>id</code> attribute and <code>coordinates</code> child element.</li>
</ul>
</li>
<li>Every keyword has a default value (most often <code>nothing</code>).  They can be set after construction.
<ul dir="auto">
<li>E.g. <code>pt.coordinates = (2, 3)</code></li>
</ul>
</li>
<li>If a child element is itself an <code>Object</code>, the keyword matches the type name.
<ul dir="auto">
<li>E.g. <code>pl = Placemark(); pl.Geometry = Point()</code>.  Here, a <code>Placemark</code> can hold any <code>Geometry</code>, which is an abstract type.  A <code>Point</code> is a subtype of <code>Geometry</code>.</li>
</ul>
</li>
<li>Some <code>Object</code>s can hold several children of the same type.  Fields with plural names expect a <code>Vector</code>.
<ul dir="auto">
<li>E.g. <code>mg = MultiGeometry(); mg.Geometries = [Point(), Polygon()]</code></li>
</ul>
</li>
<li>Enum types are in the <code>KML.Enums</code> module.  However, you shouldn't need to create them directly as conversion is handled for you/helpful error messages are provided.</li>
</ol>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; pt.altitudeMode = &quot;clamptoground&quot;

ERROR<span class="pl-k">:</span> altitudeMode <span class="pl-k">∉</span> clampToGround, relativeToGround, absolute</pre></div>
<ol start="7" dir="auto">
<li>Google extensions (things with <code>gx:</code> in the name) replace <code>:</code> with <code>_</code>.
<ul dir="auto">
<li>E.g. <code>gx:altitudeMode</code> → <code>gx_altitudeMode</code></li>
</ul>
</li>
</ol>

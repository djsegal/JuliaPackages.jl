<h1 dir="auto"><a id="user-content-sarsop" class="anchor" aria-hidden="true" href="#sarsop"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>SARSOP</h1>
<p dir="auto">This Julia package wraps the <a href="http://bigbird.comp.nus.edu.sg/pmwiki/farm/appl/" rel="nofollow">SARSOP</a> software for offline POMDP planning.
It works with the <a href="https://github.com/JuliaPOMDP/POMDPs.jl">POMDPS.jl</a> interface.
A module for writing POMDPX files is provided through the <a href="https://github.com/JuliaPOMDP/POMDPXFile.jl">POMDPXFile.jl</a> package, and is a dependency for SARSOP.jl.</p>
<p dir="auto">For a pure Julia implementation that avoids the pomdpx file bottleneck, see <a href="https://github.com/JuliaPOMDP/NativeSARSOP.jl">NativeSARSOP</a>.</p>

To update and build new binaries:</p>
<ul dir="auto">
<li>Modify the JuliaPOMDP fork of <a href="https://github.com/JuliaPOMDP/sarsop">SARSOP</a> (<a href="https://github.com/JuliaPOMDP/sarsop">https://github.com/JuliaPOMDP/sarsop</a>)</li>
<li>Fork and update the Yggdrasil <code>SARSOP_jll</code> <a href="https://github.com/JuliaPackaging/Yggdrasil/blob/a1b3930059b5ef818154fc7553c220829ff31066/S/SARSOP/build_tarballs.jl"><code>build_tarballs.jl</code></a> script</li>
<li>Create a pull request on <a href="https://github.com/JuliaPackaging/Yggdrasil">JuliaPackaging/Yggdrasil</a></li>
<li>Update this README with the correct links</li>

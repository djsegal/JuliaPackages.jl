<h2 dir="auto"><a id="user-content-details-of-the-framework" class="anchor" aria-hidden="true" href="#details-of-the-framework"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Details of the framework</h2>
<p dir="auto">Article containing details about 2.0+ release is available in pre-print: <a href="https://arxiv.org/abs/2112.00075" rel="nofollow">A Multi-purposed Unsupervised Framework for Comparing Embeddings of Undirected and Directed Graphs</a></p>
<p dir="auto">Additional experiments for undirected framework version based on <a href="https://github.com/bkamins/ABCDGraphGenerator.jl">ABCD</a> graphs are available in: <a href="https://doi.org/10.1093/comnet/cnac030" rel="nofollow">Evaluating Node Embeddings of Complex Networks</a></p>
<p dir="auto">There is also paper <a href="https://math.ryerson.ca/~pralat/papers/2020_WAW-Scalable_Embeddings.pdf" rel="nofollow">A Scalable Unsupervised Framework for
Comparing Graph Embeddings</a> presented at <a href="https://math.ryerson.ca/waw2020/" rel="nofollow">WAW2020</a> with publication in <a href="https://www.springer.com/gp/book/9783030484774" rel="nofollow">Springer LNCS</a>.</p>
<p dir="auto">Framework version without landmarks (written in C) is available under: <a href="https://github.com/ftheberge/Comparing_Graph_Embeddings">https://github.com/ftheberge/Comparing_Graph_Embeddings</a></p>

<p dir="auto">The current version uses Julia 1.6. Install <code>CGE.jl</code> by running Julia REPL, switching to package manager by pressing <code>]</code> and running:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="add https://github.com/KrainskiL/CGE.jl"><pre class="notranslate"><code>add https://github.com/KrainskiL/CGE.jl
</code></pre></div>
<p dir="auto">In order to use the CLI for the package you should locate CGE_CLI.jl file after the installation.</p>
<p dir="auto">The directory with CGE_CLI.jl file and example files can be found by running the following command in Julia:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using CGE; cd(pwd, joinpath(dirname(pathof(CGE)), &quot;..&quot;, &quot;example&quot;))"><pre><span class="pl-k">using</span> CGE; <span class="pl-c1">cd</span>(pwd, <span class="pl-c1">joinpath</span>(<span class="pl-c1">dirname</span>(<span class="pl-c1">pathof</span>(CGE)), <span class="pl-s"><span class="pl-pds">"</span>..<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>example<span class="pl-pds">"</span></span>))</pre></div>
<p dir="auto">Make sure to copy the CLI file from this location (as it is read only).</p>
<p dir="auto">Alternatively you can just download CGE_CLI.jl from GitHub repository - it is located in <code>example/</code> folder.</p>
<p dir="auto">Finally you might also download the whole repository and extract the CGE_CLI.jl file from it.</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="git clone https://github.com/KrainskiL/CGE.jl

<p dir="auto">The repository contains <code>Dockerfile</code> to build the Docker image locally. The remote image is also available on <a href="https://hub.docker.com/repository/docker/lukaszkrainski/julia_cge" rel="nofollow">Docker Hub</a> and is used in a <code>docker-compose.yaml</code> file. Running <code>docker-compose run --rm app bash</code> will initiate interactive session with <code>input</code> and <code>output</code> folders mounted in the working directory.</p>

<p dir="auto">Code computes the global and local score for specified graph, embedding and graph's clustering. The lower score value is the better.</p>
<p dir="auto">Format:</p>

<h1 dir="auto"><a id="user-content-file-formats" class="anchor" aria-hidden="true" href="#file-formats"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>File Formats</h1>
<p dir="auto">For a graph with <code>n</code> nodes, the nodes can be represented with numbers 1 to n or 0 to n-1.</p>
<p dir="auto">Two input files are required to run the algorithm:</p>
<ol dir="auto">
<li>the graph (undirected or directed), represented by a sequence of edges, 1 per line and with optional weights in third column</li>
<li>the node embedding in on of the supported formats (see below)</li>
</ol>
<p dir="auto">Additional file with the node's cluster number (1 per line) may be provided. If it's missing communities are calculated automatically with Louvain algorithm.</p>

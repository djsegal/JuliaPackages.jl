<p dir="auto">GRIBDatasets.jl uses <a href="https://weech.github.io/GRIB.jl" rel="nofollow">GRIB.jl</a> to provide a higher level interface for reading GRIB files. This package implements the <a href="https://github.com/JuliaGeo/CommonDataModel.jl">CommonDataModel.jl</a> interface, which mean that the datasets can be accessed in the same way as netCDF files opened with <a href="https://github.com/Alexander-Barth/NCDatasets.jl">NCDatasets.jl</a>.</p>

<p dir="auto">This package is similar to <a href="https://github.com/ecmwf/cfgrib.jl">CfGRIB.jl</a>, but the code has been adapted to be more Julian and to follow the <code>CommonDataModel</code> interface.</p>

<ul dir="auto">
<li>Only reading of GRIB format is currently possible with this package. But it should normally be straightforward to write a <code>GRIBDataset</code> to netCDF with <code>NCDatasets</code>.</li>
<li>GRIB format files may have a (very) large amount of different shapes. <code>GRIBDatasets</code> might not work for your specific edge case. If this happens, do not hesitate to open an issue.</li>
</ul>

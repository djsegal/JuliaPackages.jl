<p dir="auto">A simple tool, built on top of <a href="https://qojulia.org/" rel="nofollow">QuantumOptics.jl</a>, for simulating the dynamics of a configuration of
trapped ions interacting with laser light.</p>
<p dir="auto"><strong>IonSim.jl</strong> primarily performs two jobs:</p>
<ol dir="auto">
<li>Keeps track of the physical parameters necessary for describing the system.</li>
<li>Using these parameters, constructs a function that quickly computes the system's Hamiltonian as a function of time.</li>
</ol>
<p dir="auto">The functional form of the Hamiltonian can then be used as input to any of the solvers implemented in
<a href="https://qojulia.org/documentation/timeevolution/timeevolution/" rel="nofollow"><code>QuantumOptics.timeevolution</code></a>. For more information see:</p>
<ul dir="auto">
<li>Main code: <a href="https://github.com/HaeffnerLab/IonSim.jl/tree/master/src">https://github.com/HaeffnerLab/IonSim.jl/tree/master/src</a></li>
<li>Documentation: <a href="https://docs.ionsim.org" rel="nofollow">https://docs.ionsim.org</a></li>
<li>Examples: <a href="https://examples.ionsim.org" rel="nofollow">https://examples.ionsim.org</a></li>
<li>Benchmarks: (coming soon)</li>
</ul>
<p dir="auto">If you have an idea for how to improve IonSim, need some help getting things working or have any other IonSim-related questions feel free to open a GitHub issue.</p>

<ol dir="auto">
<li>Install Julia (<a href="https://julialang.org/downloads/" rel="nofollow">instructions here</a>).</li>
<li>Open up a Terminal session and fire up the <a href="https://docs.julialang.org/en/v1/stdlib/REPL/#The-Julia-REPL-1" rel="nofollow">Julia REPL</a> with</li>
</ol>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="$ julia"><pre>$ julia</pre></div>
<p dir="auto">(<em>If using Windows it's easier to start a REPL with the julia executable available after installation.</em>)</p>
<ol start="3" dir="auto">
<li>Now run</li>
</ol>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Pkg

<p dir="auto">If you want to run IonSim locally:</p>
<ul dir="auto">
<li>Open up the Julia REPL</li>
</ul>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="$ julia"><pre>$ julia</pre></div>
<ul dir="auto">
<li>Open the package manager by pressing <code>]</code></li>
<li>Run the following:</li>
</ul>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pkg&gt; dev IonSim"><pre>pkg<span class="pl-k">&gt;</span> dev IonSim</pre></div>
<p dir="auto">This will clone a version of this repo in <code>~/.julia/dev/IonSim/</code>. Then when you make changes to that repo, it will be immediately reflected when using Julia.</p>
<p dir="auto">You can run tests with</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pkg&gt; test IonSim"><pre>pkg<span class="pl-k">&gt;</span> test IonSim</pre></div>
<p dir="auto">Don't forget to format the code! <code>./scripts/format.sh</code></p>
<p dir="auto">To go back to the version in the registry, use</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="pkg&gt; free IonSim"><pre>pkg<span class="pl-k">&gt;</span> free IonSim</pre></div>
<p dir="auto">In order to update the IonSim.jl version that lives in the Julia general registry, change the version number in the Project.toml file, commit the changes and then add a comment to the commit that reads:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="@JuliaRegistrator register"><pre class="notranslate"><code>@JuliaRegistrator register
</code></pre></div>
<p dir="auto">If you have any questions, please make a GitHub issue.</p>

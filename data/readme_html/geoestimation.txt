<h3><a id="user-content-idw" class="anchor" aria-hidden="true" href="#idw"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>IDW</h3>
<p>This solver provides a high-performance implementation of the inverse distance weighting scheme
introduced in the very early days of geostatistics (see <a href="https://dl.acm.org/citation.cfm?id=810616" rel="nofollow">Shepard 1968</a>).
It is perhaps the simplest first attempt in the literature to perform estimation based on the
notion of proximity to data locations.</p>
<p>This implementation makes use of k-d trees from the <a href="https://github.com/KristofferC/NearestNeighbors.jl">NearestNeighbors.jl</a>
package, which leads to a fast estimation method for large or high-resolution spatial domains.
Although this method is recommended for fast assessment of a new field, it has poor statistical
properties (lacks covariance model) and should mainly be used for qualitative purposes.</p>
<h3><a id="user-content-lwr" class="anchor" aria-hidden="true" href="#lwr"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>LWR</h3>
<p>This solver provides an implementation of locally weighted regression (a.k.a. LOESS) introduced by
<a href="http://www.stat.washington.edu/courses/stat527/s13/readings/Cleveland_JASA_1979.pdf" rel="nofollow">Cleveland 1979</a>.
It is the most natural generalization of inverse distance weighting in which one is allowed to use a
custom weight function instead of distance-based weights.</p>
<p>Like in the inverse distance weighting solver, this solver makes use of k-d trees from the
<a href="https://github.com/KristofferC/NearestNeighbors.jl">NearestNeighbors.jl</a> package for fast data
lookup. Locally weighted regression (LWR or LOESS) is a popular non-parametric method, however
it still has poor statistical properties compared to other estimation methods such as Kriging
that explicitly model spatial correlation.</p>
<p>In the current implementation, the estimation variance is computed assuming Gaussian residuals.</p>
<h3><a id="user-content-kriging" class="anchor" aria-hidden="true" href="#kriging"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Kriging</h3>
<p>This polyalgorithm solver provides an implementation of various forms of Kriging introduced by
<a href="https://books.google.com.br/books/about/The_Theory_of_Regionalized_Variables_and.html" rel="nofollow">Matheron 1971</a>.
Kriging is a popular method in various industries due to its good statistical properties and flexibility.
Unlike the previous solvers, Kriging relies on the specification of a variogram model, which can be
fit to geospatial data.</p>

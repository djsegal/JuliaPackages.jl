<p dir="auto"><a href="https://github.com/jump-dev/SDPT3.jl">SDPT3.jl</a> is wrapper for the
<a href="https://blog.nus.edu.sg/mattohkc/softwares/sdpt3" rel="nofollow">SDPT3</a> solver.</p>
<p dir="auto">The wrapper has two components:</p>
<ul dir="auto">
<li>an exported <code>sdpt3</code> function that is a thin wrapper on top of the
<code>sdpt3</code> MATLAB function</li>
<li>an interface to <a href="https://github.com/jump-dev/MathOptInterface.jl">MathOptInterface</a></li>
</ul>

<p dir="auto">The error means that we could not find the <code>sdpt3</code> function with one output
argument using the MATLAB C API. This most likely means that you did not add
SDPT3 to the MATLAB's path (that is, the <code>toolbox/local/pathdef.m</code> file).</p>
<p dir="auto">If modifying <code>toolbox/local/pathdef.m</code> does not work, the following should work,
where <code>/path/to/sdpt3/</code> is the directory where the <code>sdpt3</code> folder is located:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using MATLAB

<p dir="auto">If you get the error:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="Brace indexing is not supported for variables of this type.

Error in validate

Error in sdpt3 (line 171)
   [blk,At,C,b,blkdim,numblk,parbarrier] = validate(blk,At,C,b,par,parbarrier);

Error using save
Variable 'jx_sdpt3_arg_out_1' not found.
</code></pre></div>
<p dir="auto">It might mean that you have added <a href="https://github.com/jump-dev/SDPNAL.jl">SDPNAL</a>
in addition to SDPT3 in the MATLAB's path (that is, the <code>toolbox/local/pathdef.m</code>
file). Because SDPNAL also defines a <code>validate</code> function, this can make <code>sdpt3</code>
call SDPNAL's <code>validate</code> function instead of SDPT3's <code>validate</code> function, which
causes the issue.</p>
<p dir="auto">One way to fix this from the Julia REPL is to reset the search path to the
factory-installed state using <code>restoredefaultpath</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using MATLAB

julia&gt; MATLAB.restoredefaultpath()

julia&gt; MATLAB.mat&quot;savepath&quot;"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> MATLAB

julia<span class="pl-k">&gt;</span> MATLAB<span class="pl-k">.</span><span class="pl-c1">restoredefaultpath</span>()

julia<span class="pl-k">&gt;</span> MATLAB<span class="pl-k">.</span><span class="pl-s"><span class="pl-pds"><span class="pl-c1">mat</span>"</span>savepath<span class="pl-pds">"</span></span></pre></div>

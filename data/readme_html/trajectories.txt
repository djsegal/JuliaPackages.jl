<p dir="auto">A trajectory in the sense of this package is a vector of time points <code>t</code> and a corresponding
vector of spatial points <code>x</code>, which are though as locations <code>x[i]</code> of an object at times
<code>t[i]</code>.</p>
<p dir="auto">A key decision which has to be made for a time series object,
is whether iteration is used to iterate values, pairs or is leveraged for destruction. See issue #1. At the moment,
all iteration and destructuring is explicit.</p>
<p dir="auto">To iterate values <code>xᵢ</code>, pairs <code>(tᵢ, xᵢ)</code> or components <code>(t, x)</code>, use <code>values</code>, <code>pairs</code> or <code>Pair</code></p>

<p dir="auto">A second key decision is what constitutes indexing. Also here this package is <em>agnostic</em>: Only key look-up
with <code>get</code> is implemented so far.</p>
<p dir="auto">Trajectories support <code>Tables.jl</code> with <code>columns</code> being a named tuple <code>(t = X.t, x = X.x)</code>.</p>

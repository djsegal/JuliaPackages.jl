<p dir="auto">This repository contains relevant implementation details for the
ForSyDe IO subproject in ForSyDe. This includes:</p>
<ol dir="auto">
<li>The type meta-model for proper syntatic consistency in the graph models,</li>
<li>Code that generates and packages each of the supporting libraries implemented,</li>
<li>Examples for the model specification showing how a model is saved on disk.</li>
</ol>
<p dir="auto">For further info, please refer to the <a href="https://forsyde.github.io/forsyde-io" rel="nofollow">documentation website</a>! This readme won't contain
much as to no repeat information where not necessary.</p>

<p dir="auto">New support of the model on other languages are always welcome! The goal is that the most famous and well-known repository
for packages in the new language contains your <code>forsyde-io</code> libraries. The canonical way to do this would be to add a new subfolder
in this source repo which can upload all code that is both generated and written by hand.</p>
<p dir="auto">As an example, the Python supporting package generates the <code>types</code> classes from the graph.json <code>file</code> along the NetworkX-backed core classes before
uploading all the sources to PyPI.</p>

<p dir="auto">Alternative to <a href="https://github.com/JuliaStats/RDatasets.jl/">RDatasets.jl</a>
which simply grabs datasets directly from CRAN.</p>
<p dir="auto">The basic usage is similar to <code>RDatasets.jl</code>. You can install it as follows:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="Pkg.add(url=&quot;https://github.com/frankier/RDataGet.jl.git&quot;)"><pre class="notranslate"><code>Pkg.add(url="https://github.com/frankier/RDataGet.jl.git")
</code></pre></div>
<p dir="auto">After installing the RDataGet package, you can then load data sets using the
<code>dataset()</code> function, which takes the name of a package and a data set as
arguments:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="using RDataGet
harman_political = dataset(&quot;psych&quot;, &quot;Harman.political&quot;)
neuro = dataset(&quot;boot&quot;, &quot;neuro&quot;)"><pre class="notranslate"><code>using RDataGet
harman_political = dataset("psych", "Harman.political")
neuro = dataset("boot", "neuro")
</code></pre></div>

<p dir="auto">This package currently just downloads source packages from CRAN and loads its
dataset into memory in Julia. It does not depend on R itself.</p>
<p dir="auto">The package has a few limitation, some of which are caused by this design, while
others could be addressed in future:</p>
<ul dir="auto">
<li>Does not support built-in R datasets, including the <code>datasets</code> package, only
ones which can be downloaded from CRAN</li>
<li>Can only load rda/RData/csv.gz files in the data directory
<ul dir="auto">
<li>As such it does not support packages which generate their data using a
build script</li>
</ul>
</li>
<li>Cannot get any descriptions or further documentation related to the datasets
from Julia (maybe TODO but needs .Rd parsing)</li>
<li>Only supports getting the latest version of each package (TODO)</li>
<li>Fixed, very-limited caching strategy
<ul dir="auto">
<li>The package index is re-downloaded every time we need to download any
package (so as to find the latest version number) (TODO: should be by-default
cached per session + longer caching allowed)</li>
<li>Packages are downloaded exactly once per session, after which the same data
is reused until Julia is restarted (TODO: should be
customisable for longer caching)</li>
</ul>
</li>
</ul>

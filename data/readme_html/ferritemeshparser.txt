<p dir="auto">Import mesh from Abaqus input (<code>.inp</code>) files into <code>Ferrite.jl</code>'s <code>Grid</code> type. User-defined cell types are supported.</p>
<p dir="auto">Pull requests that add julia code to parse other mesh formats are very welcome! Note that for <a href="https://gmsh.info/" rel="nofollow"><code>Gmsh</code></a>, the package <a href="https://github.com/Ferrite-FEM/FerriteGmsh.jl"><code>FerriteGmsh.jl</code></a> already provides a versitile interface.</p>

<h3 dir="auto"><a id="user-content-custom-cells-elements" class="anchor" aria-hidden="true" href="#custom-cells-elements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Custom cells (elements)</h3>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="grid = get_ferrite_grid(filename; user_elements::Dict{String,DataType}=Dict{String,DataType}())"><pre>grid <span class="pl-k">=</span> <span class="pl-c1">get_ferrite_grid</span>(filename; user_elements<span class="pl-k">::</span><span class="pl-c1">Dict{String,DataType}</span><span class="pl-k">=</span><span class="pl-c1">Dict</span><span class="pl-c1">{String,DataType}</span>())</pre></div>
<p dir="auto">Supply a dictionary with keys being element codes in the mesh file <code>filename</code> and the value being the corresponding concrete subtype of <code>Ferrite.AbstractCell</code>.</p>
<p dir="auto">Additionally, you may have to overload the function</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="FerriteMeshParser.create_cell(::Type{CellType}, node_numbers, format::FerriteMeshParser.AbaqusMeshFormat) where{CellType&lt;:Ferrite.AbstractCell}"><pre>FerriteMeshParser<span class="pl-k">.</span><span class="pl-c1">create_cell</span>(<span class="pl-k">::</span><span class="pl-c1">Type{CellType}</span>, node_numbers, format<span class="pl-k">::</span><span class="pl-c1">FerriteMeshParser.AbaqusMeshFormat</span>) <span class="pl-k">where</span>{CellType<span class="pl-k">&lt;:</span><span class="pl-c1">Ferrite.AbstractCell</span>}</pre></div>
<p dir="auto">Overloading is required if the constructor of your <code>CellType</code> needs different constructor input arguments than a tuple of node indices in the order given in the input file.
<code>create_cell</code> should return an instance of your subtype of <code>Ferrite.AbstractCell</code> with the given node_numbers.</p>

<ul dir="auto">
<li>Only one part and one instance can exist in the input file</li>
<li>Node and element numbers must start at 1 and not have any gaps (e.g if there are 10 nodes, no node number higher than 10 can be given)</li>
</ul>

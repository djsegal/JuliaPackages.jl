<p dir="auto">We provide a <code>Julia</code> interface (a wrapper) to the <code>Compressed Sparse Blocks (CSB)</code> library. The library is written in <code>C/C++</code>, available at
<a href="https://people.eecs.berkeley.edu/~aydin/csb/html/index.html" rel="nofollow">https://people.eecs.berkeley.edu/~aydin/csb/html/index.html</a>. The
library supports fast computation of large sparse matrix-vector
products, <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$Ax$</math-renderer> and <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$A^{\rm T}x$</math-renderer> operations specifically, with sparse
matrix <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$A$</math-renderer> in the format of compressed sparse blocks (CSB), on
shared-memory computer systems. The CSB format was introduced by A.
Buluç, J. Fineman, M. Frigo, J. Gilbert, and C. Leiserson [1].</p>
<p dir="auto">The CSB format and library have the following advantages: (1) The
matrix-vector operations in CSB format often outpace the conventional
general-purposes sparse formats, namely, CSC and CSR [2]. (2) The
multiplication with the transposed matrix <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$A^{\rm T}$</math-renderer> does not suffer from
longer latency than that with <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$A$</math-renderer>. The symmetric performance eliminates
the need for an additional copy in a different layout for <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$A^{\rm T}$</math-renderer>
(as with CSR or CSC) for reducing the speed gap at the cost of double
memory consumption. (3) The library is integrated with <code>Cilk</code> [3]. The
latter offers optimal run-time scheduling (in theory and in practice) on
parallel computers with shared memory.</p>
<p dir="auto">This Julia interface is intended to extend and ease the use of <code>CSB</code> to
broader applications. This interface supports up to <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$32$</math-renderer> multiple
vectors to be applied with the same matrix <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="baf47f4f0fe7690c4fdf6b1a0e7d9353">$A$</math-renderer>.</p>

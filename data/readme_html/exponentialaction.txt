<p>ExponentialAction is a lightweight package that implements the action of the <a href="https://en.wikipedia.org/wiki/Matrix_exponential" rel="nofollow">Matrix exponential</a> using the algorithm of Al-Mohy and Higham<sup><a href="#references">1</a></sup>.</p>
<p>The API of this package is a single function <code>expv(t, A, B)</code> that computes <code>exp(t * A) * B</code> for a scalar <code>t</code>, matrix <code>A</code>, and matrix or vector <code>B</code>, without computing <code>exp(t * A)</code>.
For large matrices <code>A</code>, this is significantly less expensive than calling <code>exp(t * A) * B</code> directly.</p>
<p>While <code>exp(X)</code> is only defined in LinearAlgebra for <code>Diagonal</code>, <code>Symmetric{&lt;:Real}</code>/<code>Hermitian</code>, and <code>StridedMatrix</code>, <code>expv</code> can take an arbitrary matrix type.
It also can be differentiated using ForwardDiff, ReverseDiff, and Zygote.
Note that currently this just means we avoid patterns such as type constraints or mutation that are problematic for these automatic differentiation engines; no custom rules are defined.
This may change in the future.</p>
<p>For description of keyword arguments, see the docstring of <code>expv</code>.</p>

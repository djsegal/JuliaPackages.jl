<h2><a id="user-content-showln" class="anchor" aria-hidden="true" href="#showln"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>@showln</h2>
<p><code>@showln</code> shows variable values and the line number at which the
statement was executed. This can be useful when variables change value
in the course of a single function. For example:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> DebuggingUtilities

<span class="pl-k">function</span> <span class="pl-en">foo</span>()
    x <span class="pl-k">=</span> <span class="pl-c1">5</span>
    <span class="pl-c1">@showln</span> x
    x <span class="pl-k">=</span> <span class="pl-c1">7</span>
    <span class="pl-c1">@showln</span> x
    <span class="pl-c1">nothing</span>
<span class="pl-k">end</span></pre></div>
<p>might, when called (<code>foo()</code>), produce output like</p>
<pre><code>            x = 5
            (in foo at ./error.jl:26 at /tmp/showln_test.jl:5)
            x = 7
            (in foo at ./error.jl:26 at /tmp/showln_test.jl:7)
</code></pre>
<h2><a id="user-content-test_showline" class="anchor" aria-hidden="true" href="#test_showline"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>test_showline</h2>
<p>This is similar to <code>include</code>, except it displays progress. This can be
useful in debugging long scripts that cause, e.g., segfaults.</p>
<h2><a id="user-content-time_showline" class="anchor" aria-hidden="true" href="#time_showline"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>time_showline</h2>
<p>Also similar to <code>include</code>, but it also measures the execution time of
each expression, and prints them in order of increasing duration.</p>

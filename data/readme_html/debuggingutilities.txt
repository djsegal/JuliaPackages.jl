<p>Install with</p>
<div class="highlight highlight-source-julia"><pre>pkg<span class="pl-k">&gt;</span> dev https<span class="pl-k">:</span><span class="pl-k">//</span>github<span class="pl-k">.</span>com<span class="pl-k">/</span>timholy<span class="pl-k">/</span>DebuggingUtilities<span class="pl-k">.</span>jl<span class="pl-k">.</span>git</pre></div>

<h2><a id="user-content-showln" class="anchor" aria-hidden="true" href="#showln"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>@showln</h2>
<p><code>@showln</code> shows variable values and the line number at which the
statement was executed. This can be useful when variables change value
in the course of a single function. For example:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> DebuggingUtilities

<span class="pl-k">function</span> <span class="pl-en">foo</span>()
    x <span class="pl-k">=</span> <span class="pl-c1">5</span>
    <span class="pl-c1">@showln</span> x
    x <span class="pl-k">=</span> <span class="pl-c1">7</span>
    <span class="pl-c1">@showln</span> x
    <span class="pl-c1">nothing</span>
<span class="pl-k">end</span></pre></div>
<p>might, when called (<code>foo()</code>), produce output like</p>
<pre><code>            x = 5
            (in foo at ./error.jl:26 at /tmp/showln_test.jl:5)
            x = 7
            (in foo at ./error.jl:26 at /tmp/showln_test.jl:7)
</code></pre>
<h2><a id="user-content-test_showline" class="anchor" aria-hidden="true" href="#test_showline"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>test_showline</h2>
<p>This is similar to <code>include</code>, except it displays progress. This can be
useful in debugging long scripts that cause, e.g., segfaults.</p>
<h2><a id="user-content-time_showline" class="anchor" aria-hidden="true" href="#time_showline"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>time_showline</h2>
<p>Also similar to <code>include</code>, but it also measures the execution time of
each expression, and prints them in order of increasing duration.</p>

<p dir="auto">This package provides Julia language bindings for the C++ library <a href="https://github.com/precice/precice">preCICE</a>. It is a Julia package that wraps the API of preCICE.</p>
<p dir="auto">Note that the first two digits of the version number of the bindings indicate the preCICE major and minor version that the bindings support. The last digit represents the version of the bindings. Example: <code>v2.5.1</code> and <code>v2.5.2</code> of the bindings represent versions <code>1</code> and <code>2</code> of the bindings that are compatible with preCICE <code>v2.5.0</code>.</p>
<p dir="auto">Technical information about the initial design of these bindings can be found in the <a href="http://dx.doi.org/10.18419/opus-11836" rel="nofollow">Bachelor thesis of Pavel Kharitenko</a>.</p>

<p dir="auto">The Julia bindings for preCICE can be used by either by adding them as a package in a Julia environment or also directly including the package in a Julia script. For both type of usages preCICE needs to be installed on the system. For preCICE installation, have a look at the <a href="https://precice.org/installation-overview.html" rel="nofollow">installation documentation</a>.</p>

<p dir="auto">If preCICE is installed at a custom path, errors of the form <code>ERROR: could not load library "/..."</code> can occur after adding the Julia bindings package. <p dir="auto">If you get an error message like:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="ERROR: ERROR: LoadError: LoadError: could not load library &quot;libprecice&quot;
/usr/bin/../lib/libstdc++.so.6: version `GLIBCXX_3.4.30' not found (required by /usr/local/lib/libprecice.so)"><pre lang="julia-repl" class="notranslate"><code>ERROR: ERROR: LoadError: LoadError: could not load library "libprecice"
/usr/bin/../lib/libstdc++.so.6: version `GLIBCXX_3.4.30' not found (required by /usr/local/lib/libprecice.so)
</code></pre></div>
<p dir="auto">It is caused by an incompatible version of <code>libstdc++</code> as described <a href="https://github.com/precice/PreCICE.jl/issues/44#issuecomment-1259655654" data-hovercard-type="issue" data-hovercard-url="/precice/PreCICE.jl/issues/44/hovercard">in the issue</a>. Julia ships its own libraries, which are older than the ones shipped by the system or the ones used to compile preCICE.</p>
<p dir="auto">As a first solution, consider updating your julia version. The versions used in Julia are fairly up to date and should work with older systems. The problem is also known to the developers of Julia and they are <a href="https://github.com/JuliaGL/GLFW.jl/issues/198" data-hovercard-type="issue" data-hovercard-url="/JuliaGL/GLFW.jl/issues/198/hovercard">working on a solution</a>. From Julia <code>v1.8.3</code> on, the problem <a href="https://github.com/JuliaLang/julia/pull/46976" data-hovercard-type="pull_request" data-hovercard-url="/JuliaLang/julia/pull/46976/hovercard">should be fixed</a>.</p>
<p dir="auto">If you cannot update your julia version, or the problem persists, you can try preloading the system <code>libstdc++</code>:</p>
<details>

<p dir="auto">Preload the system <code>libstdc++</code> with</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libstdc++.so.6 julia"><pre>LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libstdc++.so.6 julia</pre></div>
<p dir="auto">You may have to <a href="https://precice.org/installation-source-preparation.html" rel="nofollow">compile preCICE from source</a> for this to work.</p>

<p dir="auto">This checks if the preCICE bindings can be found and accessed correctly.
You can also test the full functionality of PreCICE.jl. If not set up, the output of the previous test shows an info on what command you need to execute. It will be along the lines of:</p>

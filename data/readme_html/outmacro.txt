<p dir="auto">OutMacro.jl packages the <code>@out</code> macro which mimics <code>@show</code> but breaking the output in multiple lines for a prettier display (similarly to <code>display(x)</code>).</p>
<p dir="auto">It is specially useful for debugging variables in the REPL: arrays get their variable name displayed together with the size and a pretty output for their values. Still, the printed output is not "copy-paste"-ready in the REPL, which is the main reason this was not finally included in <code>@show</code>.</p>

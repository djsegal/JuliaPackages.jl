<h2 dir="auto"><a id="user-content-summary" class="anchor" aria-hidden="true" href="#summary"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Summary</h2>
<p dir="auto">The purpose of <code>PreprocessMD.jl</code> is to provide a suite of functions for preprocesing biomedical data.
The scope of this package is <em><strong>medical data preprocessing</strong></em>, so
we develop functions that are specific to biomedical research but general enough for widespread use.
These tools are developed for the OMOP Common Data Model<sup><a href="#user-content-fn-OMOP-c87541c2827c0ab4ed9d6f75ef6425ff" id="user-content-fnref-omop-c87541c2827c0ab4ed9d6f75ef6425ff" data-footnote-ref="" aria-describedby="footnote-label">1</a></sup>,
especially the MIMIC-IV demo set<sup><a href="#user-content-fn-MIMIC-c87541c2827c0ab4ed9d6f75ef6425ff" id="user-content-fnref-mimic-c87541c2827c0ab4ed9d6f75ef6425ff" data-footnote-ref="" aria-describedby="footnote-label">2</a></sup>.</p>
<p dir="auto">Following the definitions of Hu et al.<sup><a href="#user-content-fn-Wu-c87541c2827c0ab4ed9d6f75ef6425ff" id="user-content-fnref-wu-c87541c2827c0ab4ed9d6f75ef6425ff" data-footnote-ref="" aria-describedby="footnote-label">3</a></sup>, we consider <em><strong>data preprocessing</strong></em> to include project-level data manipulations,
as opposed to the upstream <em><strong>data cleaning</strong></em> (e.g., error-corrections and standardizations) that is typically performed over an entire database,
and the downstream <em><strong>data preparing</strong></em> (e.g., labelling and classification), which might vary across any number of analyses within a project.</p>

<p><a href="SMO.md">Spider Monkey Optimization</a> is a numerical optimization algorithm which is based on the foraging begavior of spider monkeys. It is one of the swarm intelligence approaches which can be broadly classified as an algorithm inspired by intelligent foraging behavior of fission–fusion social structure based animals. The animals which follow fission–fusion social systems (like Spider Monkeys), split themselves from large to smaller groups and vice-versa based on the scarcity or availability of food.<br>
<a href="https://link.springer.com/article/10.1007/s12293-013-0128-0" rel="nofollow">Here</a> is the research paper for SMO algorithm and <a href="https://www.sciencedirect.com/science/article/abs/pii/S2210650216000122" rel="nofollow">here</a> is my Aegist Spider Monkey algorithm research paper which proposed a few tweaks in the original algorithm to make it more efficient (will be implemented soon as a part of this package).</p>

<p>The function can be defined like the ones in <a href="https://github.com/jonathanBieler/BlackBoxOptimizationBenchmarking.jl">BlackBoxOptimizationBenchmarking</a> or you can use one of those implemented bechmarking functions.</p>
<p>For the SMO optimization algorithm we need the function <code>f</code> to be optimized, the dimension of the search space <code>D</code> or the function and population of the spider monkeys <code>P</code> for the heuristic search. We also need to define the upper <code>ub</code> and lower bounds <code>lb</code> of the search space coordinates for the boundaries which are <code>DX1</code> arrays.</p>

<p>There are lots of parameters that can be tweaked. The algorithm is generally fairly robust meaning that slight changes in parameters should not result in drastically different performance.</p>
<p>I suggest reading on Spider Monkey Optimization algorithm <a href="SMO.md">here</a> before starting. However if you don't feel like it, the parameters worth playing around with might be:</p>
<ul>
<li><code>ɳ</code>: perturbation rate</li>
<li><code>λ</code>: local leader limit, max count for which the position of the local leader remains unchanged</li>
<li><code>α</code>: global leader limit, max count for which the position of the global leader remains unchanged</li>
<li><code>β</code>: maximum number of groups allowed</li>
<li><code>τ</code>: maximum number of iterations</li>
</ul>

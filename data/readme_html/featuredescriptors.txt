<p>FeatureDescriptors.jl is an interface package for describing features used in <a href="https://invenia.github.io/Models.jl/stable/" rel="nofollow">models</a>, <a href="https://invenia.github.io/FeatureTransforms.jl/stable/" rel="nofollow">feature engineering</a>, and other data-science workflows.</p>
<h2><a id="user-content-getting-started" class="anchor" aria-hidden="true" href="#getting-started"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Getting Started</h2>
<p><code>Descriptor</code>s provide a way to define at a high level the properties of - and relationships between - a collection of features and the corresponding data.
By associating a type with a given dataset, it allows users to write methods that can dispatch on the given feature and compose pipelines that are agnostic to the characteristics of any particular feature.
Subtypes of <code>Descriptor</code>s inherit the properties of the supertype, but these can be overloaded as required.</p>
<p>For example, say that some weather data is contained in a <code>weather.csv</code> file where each column describes a different feature we are interested in using, such as <code>:temperature</code> and <code>:humidity</code>.
We can define a general <code>Weather</code> <code>Descriptor</code> with subtypes <code>Temperature</code> and <code>Humidity</code> that are loaded from the same table but use the appropriate columns:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using FeatureDescriptors

<p>A more specific instance of a feature can also be defined, such as a <code>MeanTemperature</code>, perhaps if that feature requires some <a href="https://invenia.github.io/FeatureTransforms.jl/stable/" rel="nofollow">feature engineering</a> before it can be used.</p>

function FeatureTransforms.transform(D::Type{&lt;:MeanTemperature}, df)

end

For example, if we had rainfall data that was saved in another file that we also wanted to use.</p>

<h2><a id="user-content-the-descriptors-api" class="anchor" aria-hidden="true" href="#the-descriptors-api"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The Descriptors API</h2>
<p>All <code>Descriptor</code>s are required to implement the following:</p>
<ol>
<li>A <code>sources</code> method that specifies where to retrieve the data. A <code>Descriptor</code> may be associated with multiple sources, because it may be necessary to perform <a href="https://invenia.github.io/FeatureTransforms.jl/stable/" rel="nofollow">feature engineering</a> to create the derived feature.</li>
<li>A <code>quantity_key</code> method that denotes the name of the quantitative variable for the feature, such as <code>:temperature</code> or <code>:price</code>. For the sake of transparency and simplicity, only one <code>quantity_key</code> may be associated with a given <code>Descriptor</code>.</li>
<li>A <code>categorical_keys</code> method that denotes the names of the categorical variables for the feature, such as <code>:colour</code> or <code>:type</code>. If no categorical variables are needed, this returns an empty vector.</li>
</ol>
<p>You can ensure your <code>Descriptor</code> is implemented correctly by calling the <a href="https://invenia.github.io/FeatureDescriptors.jl/stable/index/#Testutils" rel="nofollow"><code>TestUtils.test_interface</code></a> function.</p>

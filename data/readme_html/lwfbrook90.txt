<p align="center" dir="auto"><a target="_blank" rel="noopener noreferrer" href="https://github.com/fabern/LWFBrook90.jl/blob/develop/docs/src/assets/logo2.png?raw=true"><img src="https://github.com/fabern/LWFBrook90.jl/raw/develop/docs/src/assets/logo2.png?raw=true" width="150" style="max-width: 100%;"></a></p><p dir="auto">
The model LWF-BROOK90 is a 1D Soil-Vegetation-Atmosphere Transfer (SVAT) model, calculating the soil water balance in forest soil on the scale of forest stands. Modelled processes include vertical soil water movement, soil and plant evapotranspiration and temporary storages in snowpack or interception layer.
LWFBrook90.jl is an implementation of the existing LWF-BROOK90 rewritten entirely in the Julia programming language.
</p><p dir="auto">Processes and state variables of the model in LWF-BROOK90 are summarised visually in Figure 1 below.
LWFBrook90.jl is developed with the following objectives in mind:</p>
<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" id="" disabled="" class="task-list-item-checkbox" checked=""> support for stable isotopes (δ¹⁸O and δ²H) by including transport equation <del>and fractionation processes</del></li>
<li class="task-list-item"><input type="checkbox" id="" disabled="" class="task-list-item-checkbox"> efficient parameter estimation (optimizing computational costs)</li>
<li class="task-list-item"><input type="checkbox" id="" disabled="" class="task-list-item-checkbox" checked=""> model flexibility for alternative processes parametrizations <del>(possibly resulting in a <a href="https://presentations.copernicus.org/EGU2020/EGU2020-17975_presentation.pdf" rel="nofollow">flexible model framework</a>)</del></li>
</ul>

<p dir="auto">To get started with Julia: see the section <a href="https://fabern.github.io/LWFBrook90.jl/stable/user-guide/#Installation" rel="nofollow">Installation</a> in the documentation.</p>
<p dir="auto">An example data set <code>BEA2016*</code> was generated using the R package LWFBrook90R and is located in the folder <code>examples/</code>. See outputs in the next section.</p>

Another possibility is to follow the script <code>main.jl</code>, which sets up a simulation and shows some simple plotting commands. See documentation section <a href="https://fabern.github.io/LWFBrook90.jl/stable/example/" rel="nofollow">Example</a>.</p>
<p dir="auto">Further documentation for LWFBrook90.jl is available <a href="https://fabern.github.io/LWFBrook90.jl/stable" rel="nofollow">here</a> and for BROOK90 <a href="http://www.ecoshift.net/brook/b90doc.html" rel="nofollow">here</a>.</p>

<ul dir="auto">
<li>Preprocessing steps to define a convenient format of input data sets are not yet designed. Currently, LWFBrook90.jl makes use of an input data set that is best prepared using the preprocessing steps provded in the the R package LWFBrook90R.</li>
<li>Newest developments of BROOK90 are not included. LWFBrook90R is based on LWF-BROOK90, which was itself forked from an older vresion of BROOK90 (v3.1F). Developments in BROOK90 up to v4.8 (such as <code>KKMEAN</code> the intercell averages of hydraulic conductivity) are partially included, but not activated by default in LWFBrook90.jl.</li>
<li>Currently a part of LWFBrook90R activated when <code>Reset==1</code>, is not implemented in LWFBrook90.jl. (A test implementation of this is available in the branch <code>005-simplify-time-step-control</code>.)</li>
</ul>
<h3 dir="auto"><a id="user-content-improve-agreement-with-lwfbrook90r" class="anchor" aria-hidden="true" href="#improve-agreement-with-lwfbrook90r"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Improve agreement with LWFBrook90R</h3>
<p dir="auto">Note that some features of LWFBrook90R are not implemented in the main version of LWFBrook90.jl. The time step adaptivity and <code>Reset==1</code> of LWFBrook90R would require code refactoring, which goes slightly against the intended use of the library for ODEs DifferentialEquations.jl. Because of this, implementation of these features is left away from the main version. However, an attempt at their implementation resides currently in a feature branch <code>005</code>.</p>

<p dir="auto">Julia has <code>@edit</code>, <code>@less</code>, etc. which are very handy for reading the implementation of functions.  However, you need to specify a "good enough" set of (type) parameters for them to find the location of the code.</p>
<p dir="auto">Instead, <code>InteractiveCodeSearch</code> provides a few macros to interactively choose the code you want to read.</p>
<h2 dir="auto"><a id="user-content-features" class="anchor" aria-hidden="true" href="#features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Features</h2>
<ul dir="auto">
<li>Interactively choose a method signature before opening the code location in your editor.</li>
<li>Various ways to search methods, such as: by function name <code>@search show</code>, function call expression <code>@search show(stdout, "hello")</code>, function call signature <code>@search show(::IO, ::String)</code>, module name <code>@search Base</code>, argument value <code>@searchmethods 1</code>, and argument type <code>@searchmethods ::Int</code>.</li>
<li>Interactively search history.  It works in IJulia as well.</li>
</ul>

<h3 dir="auto"><a id="user-content-search" class="anchor" aria-hidden="true" href="#search"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>@search</code></h3>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="@search x [:shallow | :s | :recursive | :r]"><pre class="notranslate"><code>@search x [:shallow | :s | :recursive | :r]
</code></pre></div>
<p dir="auto">List file locations at which <code>x</code> are defined in an interactive matcher and then open the chosen location in the editor.</p>
<p dir="auto">When <code>x</code> is a module, only the top-level definitions are searched.  To search all definitions in the submodule, pass <code>:recursive</code> or <code>:r</code> flag.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="@search"><pre class="notranslate"><code>@search
</code></pre></div>
<p dir="auto">If no expression is provided, search for the method returned by the previous execution; i.e., <code>x</code> defaults to <code>ans</code>.</p>
<p dir="auto"><strong>Examples</strong></p>

<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="@searchhistory"><pre class="notranslate"><code>@searchhistory
</code></pre></div>
<p dir="auto">Search history interactively.  Interactively narrows down the code you looking for from the REPL history.</p>
<p dir="auto"><em>Limitation/feature in IJulia</em>: In IJulia, <code>@searchhistory</code> searches history of terminal REPL, not the history of the current IJulia session.</p>

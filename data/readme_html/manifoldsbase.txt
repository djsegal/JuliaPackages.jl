<p>Basic interface for manifolds in Julia.</p>
<p>The project <a href="https://github.com/JuliaManifolds/Manifolds.jl"><code>Manifolds.jl</code></a>
is based on this interface and provides a variety of manifolds.</p>
<h2><a id="user-content-number-system" class="anchor" aria-hidden="true" href="#number-system"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Number system</h2>
<p>A number system represents the field a manifold is based upon.
Most prominently, these are real-valued (<code>ℝ</code>) and complex valued (<code>ℂ</code>) fields that
parametrize certain manifolds.
A further type to represent the field of quaternions (<code>ℍ</code>) can also be used.</p>
<h2><a id="user-content-bases" class="anchor" aria-hidden="true" href="#bases"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Bases</h2>
<p>Several different types of bases for a tangent space at <code>p</code> on a <a href="https://juliamanifolds.github.io/Manifolds.jl/stable/interface.html#ManifoldsBase.Manifold" rel="nofollow"><code>Manifold</code></a> are provided.
Methods are provided to obtain such a basis, to represent a tangent vector in a basis and to reconstruct a tangent vector from coefficients with respect to a basis.
The last two can be performed without computing the complete basis.
Further a basis can be cached and hence be reused, see <a href="https://juliamanifolds.github.io/Manifolds.jl/stable/interface.html#ManifoldsBase.CachedBasis" rel="nofollow"><code>CachedBasis</code></a>.</p>
<h2><a id="user-content-decoratormanifold" class="anchor" aria-hidden="true" href="#decoratormanifold"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>DecoratorManifold</code></h2>
<p>The decorator manifold enhances a manifold by certain, in most cases implicitly
assumed to have a standard case, properties, see for example the <code>EmbeddedManifold</code>.
The decorator acts semi transparently, i.e. <code>:transparent</code> for all functions not affected by that
decorator and <code>:intransparent</code> otherwise. Another possibility is, that the decorator just
passes to <code>:parent</code> in order to fill default values.</p>
<h2><a id="user-content-defaultmanifold" class="anchor" aria-hidden="true" href="#defaultmanifold"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>DefaultManifold</code></h2>
<p>This interface includes a simple <code>DefaultManifold</code>, which is a reduced version
of the <a href="https://juliamanifolds.github.io/Manifolds.jl/stable/manifolds/euclidean.html" rel="nofollow"><code>Euclidean</code></a>
manifold from <a href="https://github.com/JuliaManifolds/Manifolds.jl"><code>Manifolds.jl</code></a>,
such that the interface functions can be tested.</p>
<h2><a id="user-content-embeddedmanifold" class="anchor" aria-hidden="true" href="#embeddedmanifold"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>EmbeddedManifold</code></h2>
<p>The embedded manifold models the embedding of a manifold into another manifold.
This way a manifold can benefit from existing implementations.
One example is the <code>TransparentIsometricEmbeddingType</code> where a manifold uses the metric,
<code>inner</code>, from its embedding.</p>
<h2><a id="user-content-validationmanifold" class="anchor" aria-hidden="true" href="#validationmanifold"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>ValidationManifold</code></h2>
<p>The <code>ValidationManifold</code> further illustrates how one can also used types to
represent points on a manifold, tangent vectors, and cotangent vectors,
where values are encapsulated in a certain type.</p>
<p>In general, <code>ValidationManifold</code> might be used for manifolds where these three types are represented
by more complicated data structures or when it is necessary to distinguish these
by type.</p>
<p>This adds a semantic layer to the interface, and the default implementation of
<code>ValidationManifold</code> adds checks to all inputs and outputs of typed data.</p>

<ul dir="auto">
<li>Detection is based on optimising a cost function over segments of the data.</li>
<li>Implementations of the most efficient search algorithms (PELT , Binary Segmentation).</li>
<li>A wide choice of parametric cost functions already implemented such as a change in mean/variance/mean and variance for Normal errors.</li>
<li>Changepoint algorithms have an interface which allows users to input their own cost functions</li>
<li>Implementations of testing-based segmentation algorithms (Wild/Seeded Binary Segmentation, MOSUM) for the univariate mean change problem</li>
</ul>

<p dir="auto">Change point detection aims to model time series data as piecewise stationary between change points <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=k_j"><img src="https://render.githubusercontent.com/render/math?math=k_j" style="max-width: 100%;"></a>, such that</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=X_t = f_t, \quad f_t \sim \mathbb{P}_j, j=1, \dots, q; k_j &lt; t \leq k_{j+1} "><img src="https://render.githubusercontent.com/render/math?math=X_t = f_t, \quad f_t \sim \mathbb{P}_j, j=1, \dots, q; k_j &lt; t \leq k_{j+1} " style="max-width: 100%;"></a></p>
<p dir="auto">for distributions <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=\mathbb{P}_j"><img src="https://render.githubusercontent.com/render/math?math=\mathbb{P}_j" style="max-width: 100%;"></a>,
with the convention that <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=k_0=1"><img src="https://render.githubusercontent.com/render/math?math=k_0=1" style="max-width: 100%;"></a>  and <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=k_{q%2B1}=n"><img src="https://render.githubusercontent.com/render/math?math=k_{q%2B1}=n" style="max-width: 100%;"></a> denote the start and end of the data.</p>
<p dir="auto">The simplest such model is the piecewise-constant mean setting, where <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=f_t = \mu_j + \epsilon_t, \mu_j \neq  \mu_{j%2B1}, E(\epsilon_t) = 0, Var(\epsilon_t) = \sigma^2"><img src="https://render.githubusercontent.com/render/math?math=f_t = \mu_j + \epsilon_t, \mu_j \neq  \mu_{j%2B1}, E(\epsilon_t) = 0, Var(\epsilon_t) = \sigma^2" style="max-width: 100%;"></a>.</p>
<p dir="auto">The methods in this package aim to estimate the number and location of changes in a given model. Penalty-based approaches aim to minimise the quantity <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=\sum_{j=1}^{q%2B1}[\mathcal{C}(x_{k_{j-1}:k_j)}%20]%20%2B%20\beta%20f(q)"><img src="https://render.githubusercontent.com/render/math?math=\sum_{j=1}^{q%2B1}[\mathcal{C}(x_{k_{j-1}:k_j)}%20]%20%2B%20\beta%20f(q)" style="max-width: 100%;"></a>
where <a target="_blank" rel="noopener noreferrer nofollow" href="https://render.githubusercontent.com/render/math?math=\mathcal{C}, \beta f(q)"><img src="https://render.githubusercontent.com/render/math?math=\mathcal{C}, \beta f(q)" style="max-width: 100%;"></a> are the cost function and penalty respectively. Segmentation methods form statistics comparing the sample either side of a candidate change point, and use the maximum statistic to evaluate a hypothesis test.</p>
<p dir="auto">For a general overview of the multiple changepoint problem and mathematical details see <a href="http://arxiv.org/pdf/1101.1438.pdf" rel="nofollow">PELT</a>. For an overview of segmentation algorithms, see <a href="https://arxiv.org/pdf/2012.12814" rel="nofollow">Data segmentation algorithms: Univariate mean change and beyond</a>.</p>

<p dir="auto">To segment a time series using PELT we need a cost function for segments of our data, and optionally a penalty for each changepoint.
Twice the negative log-likelihood is a commonly used cost function in changepoint detection, and this package provides a variety of these for different parametric models.</p>
<p dir="auto">The following code constructs a log-likelihood based cost function for segments of the data generated above which are assumed to follow a Normal distribution with unknown mean and a known fixed variance (1 in this case):</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="σ = 1.0
seg_cost = NormalMeanSegment(data, σ)  # Create segment cost function
"><pre class="notranslate"><code>σ = 1.0
seg_cost = NormalMeanSegment(data, σ)  # Create segment cost function

</code></pre></div>
<p dir="auto">We can now run PELT for this cost function with the <code>PELT</code> function which requires a cost function and the length of our sequence of data:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="pelt_cps, pelt_cost = PELT(seg_cost, length(data))   # Run PELT"><pre class="notranslate"><code>pelt_cps, pelt_cost = PELT(seg_cost, length(data))   # Run PELT
</code></pre></div>
<p dir="auto">The <code>PELT</code> function returns an integer array containing the indices of the changepoints, and the total cost of the segmentation.
By default, the <code>PELT</code> function uses a penalty of <code>log(n)</code> where <code>n</code> is the length of the sequence of data, but this can also be specified by the user as an optional third argument.</p>
<p dir="auto">For convenience, we also provide a macro for running PELT, <code>@PELT</code>,  which allows one to construct a cost function and run PELT in a single line:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="pelt_cps, cost = @PELT data Normal(:?, 1.0)"><pre class="notranslate"><code>pelt_cps, cost = @PELT data Normal(:?, 1.0)
</code></pre></div>
<p dir="auto">This takes as arguments the data to be segmented and a model to construct a cost function, and returns the same output as the <code>PELT</code> function.
Again, an optional third argument can be used to specify a changepoint penalty.
The model specified in the second argument is a distribution (using the same distribution names as in the <code>Distributions</code> package<code>) with the symbol </code>:?` replacing any parameters whose values are assumed to change at changepoints.
Some other examples of expressions which can be used with PELT in this way are:</p>
<ul dir="auto">
<li><code>Normal(μ, :?)</code>: Normally distributed data with known mean (μ) and changing variance</li>
<li><code>Normal(:?, :?)</code>: Normally distributed data with changing mean and variance</li>
<li><code>Exponential(:?)</code>: data distributed as Exponential distribution with changing mean</li>
<li><code>Gamma(:?, β)</code>: data distributed as Gamma distribution with changing shape parameter and known rate parameter <code>β</code></li>
</ul>
<p dir="auto">See documentation for <code>@segment_cost</code> for a full list of available cost functions for penalty-based changepoint methods.</p>

<p dir="auto">The methods implemented view the problem as one of optimising a penalised cost function where the penalty comes in whenever a new changepoint is added.
Assuming we have specified the correct model/cost function then the only area of possible misspecification is in the value of the penalty.
There is no "correct" choice of penalty however, but it can be very instructive to look at the segmentations and especially the number of changepoints for a range of penalties.
The Changepoints for a Range Of Penalties (CROPS) method allows us to do this efficiently using PELT, by exploiting the relationship between the penalised and constrained versions of the same optimisation problem.
For more information see <a href="http://arxiv.org/abs/1412.3617" rel="nofollow">CROPS</a>.</p>
<p dir="auto">To run the PELT algorithm for a range of penalties say pen1 to pen2 where pen1 &lt; pen2 then we can use the <code>CROPS</code> function
which takes as input a segment cost function, the length of the data set and the two penalties:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="seg_cost = NormalMeanSegment(data, σ)
pen1, pen2 = 4.0, 100.0 # Penalty range
crops_output = CROPS(seg_cost, n, pen1, pen2)"><pre class="notranslate"><code>seg_cost = NormalMeanSegment(data, σ)
pen1, pen2 = 4.0, 100.0 # Penalty range
crops_output = CROPS(seg_cost, n, pen1, pen2)
</code></pre></div>
<p dir="auto">The <code>CROPS</code> function returns a dictionary containing outputs such as the penalties for which PELT was run, and the corresponding changepoints.
See the function documentation for more details.
For convenience, CROPS can also be run using the <code>@PELT</code> macro by simply specifying a second penalty:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="crops_output = @PELT data Normal(:?, 1.0) pen1 pen2"><pre class="notranslate"><code>crops_output = @PELT data Normal(:?, 1.0) pen1 pen2
</code></pre></div>

Again, if a plotting package has been loaded, we can create a so called "elbow" plot from these results.</p>

<p dir="auto">By instead using segmentation algorithms, we can avoid specifying a cost function or penalty.
These algorithms use local information to form test statistics, which are compared to a threshold for detection, and maximising locations are used as changepoint estimates. Those implemented in this package are for the change in mean setting.</p>
<p dir="auto">The MOSUM procedure requires specifying a bandwidth <code>G</code>, which should be at most half of the true minimum segment length (see <a href="https://projecteuclid.org/euclid.bj/1501142454" rel="nofollow">MOSUM</a>). Optionally, <code>var_est_method</code> specifies the variance estimator to normalise by; this can be the average <code>mosum</code> (default) or minimum <code>mosum.min</code> across windows. <code>alpha</code> determines the signicance level (default 0.1). <code>criterion</code> determines whether to use the <code>eta</code> (default) or <code>epsilon</code> location procedure (see references). <code>eta</code> and <code>epsilon</code> are tuning parameters for the mentioned procedures (default 0.4 and 0.2).
This returns a dictionary with outputs including change point locations and the detector statistic.</p>
<p dir="auto">To run the procedure we use the following code:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="G = 35 # pick bandwidth
MOSUM_output = MOSUM(data, G) # run MOSUM procedure"><pre class="notranslate"><code>G = 35 # pick bandwidth
MOSUM_output = MOSUM(data, G) # run MOSUM procedure
</code></pre></div>
<p dir="auto">We can plot the detector statistic, located changes, and threshold with</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="mosum_plot(MOSUM_output)"><pre class="notranslate"><code>mosum_plot(MOSUM_output)
</code></pre></div>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="/docs/Plots_mosum_plot.png?raw=true"><img src="/docs/Plots_mosum_plot.png?raw=true" alt="MOSUM plot" title="MOSUM plot" style="max-width: 100%;"></a></p>
<p dir="auto">We can perform the MOSUM procedure with a series of increasing bandwiths to detect smaller or awkwardly-arranged signals.
We have implemented the multi-scale merging procedure of <a href="https://arxiv.org/pdf/1303.3594.pdf" rel="nofollow">Messer et. al. 2014</a>, which runs the procedure for bandwidths in increasing order, adding as a change point only those located which are not too close to any points already located. This returns a vector of estimated change points.
To run this, we enter:</p>

Specifying <code>M=1</code> will call the CUSUM-based BS procedure. We are returned an array of tuples containing change point information, in decreasing detection order; see <code>?WBS</code> for details.</p>

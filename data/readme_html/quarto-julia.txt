<p>Julia interface to the <a href="https://github.com/quarto-dev/quarto-cli/wiki">Quarto</a> CLI.</p>
<p>Quarto is an academic, scientific, and technical publishing system built on <a href="https://pandoc.org" rel="nofollow">Pandoc</a>.</p>
<p>In addition to the core capabilities of Pandoc, Quarto includes:</p>
<ol>
<li>Support for embedding output from Python, R, and Julia via integration with <a href="https://jupyter.org/" rel="nofollow">Jupyter</a> and <a href="https://yihui.org/knitr/" rel="nofollow">knitr</a> .</li>
<li>A project system for rendering groups of documents at once.</li>
<li>Flexible ways to specify rendering options, including project-wide options and per-format options.</li>
<li>Cross references for figures, tables, equations, sections, listings, proofs, and more.</li>
<li>Sophisticated layout for panels of figures, tables, and other content.</li>
<li>HTML output based on <a href="https://getbootstrap.com/" rel="nofollow">Bootstrap</a> (including support for <a href="https://bootswatch.com/" rel="nofollow">Bootswatch</a> themes).</li>
<li>Automatic installation of required LaTeX packages when rendering PDF output.</li>
</ol>
<p>The overall design of Quarto is influenced heavily by <a href="https://rmarkdown.rstudio.com/" rel="nofollow">R Markdown</a>, however unlike R Markdown the architecture is language agnostic. In it's current iteration, Quarto can render plain markdown, Rmd documents, and Jupyter notebooks.</p>
<p>For additional documentation, see the <a href="https://github.com/quarto-dev/quarto-cli/wiki">Quarto Wiki</a>.</p>
<p>To get started with using the Julia Jupyter kernel with Quarto, see the documentation on <a href="https://github.com/quarto-dev/quarto-cli/wiki/Jupyter-Markdown">Jupyter Markdown</a>.</p>

<p>While a few great signal processing packages (e.g. <a href="https://github.com/JuliaDSP/DSP.jl"><code>DSP.jl</code></a>, <a href="https://github.com/haberdashPI/SignalOperators.jl"><code>SignalOperators.jl</code></a>) are available, they have limited time-frequency analysis, sonar analysis, and baseband analysis capabilities. This <code>SignalAnalysis.jl</code> package aims to fill that gap. The package has grown out of my research needs, but I hope to expand it over time to provide a wide variety of time-frequency analysis, and baseband signal processing tools.</p>
<p>The <code>SignalAnalysis.jl</code> works closely with, and complements other package like <a href="https://github.com/JuliaDSP/DSP.jl"><code>DSP.jl</code></a>, and <a href="https://github.com/JuliaPlots/Plots.jl"><code>Plots.jl</code></a>, to provide its functionality. While some overlap is inevitable, we mostly leverage functionality already available in those packages, and avoid duplicating it. In some cases, simpler wrappers are provided, or some symbols are re-exported for convenience. The package also works closely with <a href="https://github.com/PainterQubits/Unitful.jl"><code>Unitful.jl</code></a> for common real-world units.</p>
<p>While this package works with most array-like data types, it uses the <a href="https://github.com/haberdashPI/SignalBase.jl"><code>SignalBase.jl</code></a> API to represent multichannel 1D signals (time, frequency or spatial domain). While the package adopts the <code>SampleBuf</code> data type from the <a href="https://github.com/JuliaAudio/SampledSignals.jl"><code>SampledSignals.jl</code></a> package to carry sampling rate information with the sampled signal, the API design allows sampling rate to be provided as a keyword argument in most cases, enabling the user to pass in any array-like data.</p>

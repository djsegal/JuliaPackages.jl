<p dir="auto">Imagine is an acquisition program for light sheet microscopy written
by Zhongsheng Guo in Tim Holy's lab. This package implements a loader
for the file format for the Julia programming language. Each Imagine
"file" consists of two parts (as two separate files): a <code>*.imagine</code>
file which contains the (ASCII) header, and a <code>*.cam</code> file which
contains the camera data.  The <code>*.cam</code> file is a raw byte dump, and is
compatible with the NRRD "raw" file.</p>

<p dir="auto">You can write an NRRD header (<code>*.nhdr</code>) from an Imagine header as follows:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="h = ImagineFormat.parse_header(filename)  # the .imagine file name

<p dir="auto">where <code>datafilename</code> is the name of the <code>*.cam</code> file. It is required by the <code>*.nhdr</code> file to point to the actual data.</p>

<p dir="auto">You can use the non-exported function <code>ImagineFormat.save_header</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="save_header(destname, srcname, img, [T])"><pre><span class="pl-c1">save_header</span>(destname, srcname, img, [T])</pre></div>
<p dir="auto"><code>destname</code> is the output <code>*.imagine</code> file; <code>srcname</code> is the name of
the "template" file.  Certain header values (e.g., size information)
are updated by reference to <code>img</code>.  The optional argument <code>T</code> allows
you to specify a different element type than possessed by <code>img</code>.</p>

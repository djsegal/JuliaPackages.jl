<p dir="auto"><em>Because</em> <code>NaN</code> <em>is just</em> <code>missing</code> <em>with hardware support!</em></p>
<p dir="auto">Fast (often <a href="https://github.com/JuliaSIMD/LoopVectorization.jl">LoopVectorization.jl</a>-based) summary statistics, histograms, and binning — all ignoring <code>NaN</code>s, as if <code>NaN</code> represented missing data.</p>
<p dir="auto">See also <a href="https://github.com/JuliaSIMD/VectorizedStatistics.jl">JuliaSIMD/VectorizedStatistics.jl</a> for similar vectorized implementations that don't ignore <code>NaN</code>s.</p>

<h5 dir="auto"><a id="user-content-measures-of-central-tendency" class="anchor" aria-hidden="true" href="#measures-of-central-tendency"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Measures of central tendency</h5>
<ul dir="auto">
<li><code>nanmean</code>   arithmetic mean, ignoring <code>NaN</code>s</li>
<li><code>nanmedian</code>   median, ignoring <code>NaN</code>s</li>

<h5 dir="auto"><a id="user-content-measures-of-dispersion" class="anchor" aria-hidden="true" href="#measures-of-dispersion"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Measures of dispersion</h5>
<ul dir="auto">
<li><code>nanvar</code>   variance</li>
<li><code>nanstd</code>   standard deviation</li>
<li><code>nancov</code>   covariance</li>
<li><code>nancor</code>   Pearson's product-moment correlation</li>
<li><code>nanaad</code>   mean (average) absolute deviation from the mean</li>
<li><code>nanmad</code>   median absolute deviation from the median</li>
<li><code>nanmad!</code>   as <code>nanmad</code> but quicksorts in-place for efficiency</li>
<li><code>nanrange</code>   range between nanmaximum and nanminimum</li>
<li><code>nanpctile</code>   percentile</li>
<li><code>nanpctile!</code>   as <code>nanpctile</code> but quicksorts in-place for efficiency</li>
</ul>
<p dir="auto">Note that, regardless of implementation, functions involving medians or percentiles are generally significantly slower than other summary statistics, since calculating a median or percentile requires a quicksort or quickselect of the input array; if not done in-place as in <code>nanmedian!</code> and <code>nanpctile!</code> then a copy of the entire array must also be made.</p>
<p dir="auto">These functions will generally support the same <code>dims</code> keyword argument as their normal Julia counterparts (though are most efficient when operating on an entire collection).
As an alternative to <code>dims</code>, the <code>dim</code> keyword is also supported, which behaves identially to <code>dims</code> except that it also (as is the norm in some other languages) drops any singleton dimensions that have been reduced over.</p>

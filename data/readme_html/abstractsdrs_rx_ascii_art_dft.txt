<blockquote>
<p dir="auto">Spectrum signature from a Software Defined Radio, directly in the REPL !</p>
</blockquote>
<p dir="auto">This small package intends to render in the terminal the output spectrum from a Software Defined Radio.
It is very similar to the GnuRadio tool <code>rx_ascii_art_dft</code> and has a similar API</p>

No functions are exported and the spectrum can be obtained with the following command</p>

<p dir="auto">The following parameters are supported</p>
<h3 dir="auto"><a id="user-content-input-parameters" class="anchor" aria-hidden="true" href="#input-parameters"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Input parameters</h3>
<ul dir="auto">
<li><code>sdr</code> : SDR type supported by AbstractSDRs (for instance :uhd). A symbol is expected</li>
<li><code>carrierFreq</code> : Carrier Freq in Hz (e.g 2400e6)</li>
<li><code>samplingRate</code> : Radio sampling rate in Hz (e.g 2e6)</li>
<li><code>gain</code>          : gain in normalized scale (e.g 20)</li>
</ul>

<ul dir="auto">
<li><code>nFFT</code> : Size of the FFT/DFT as an <code>Int</code> (e.g 1024)</li>
<li><code>avg</code> : averaging factor for PSD computation  as an <code>Int</code> (e.g 12)</li>
<li><code>canvas</code> : Canvas used to display. Can be AsciiCanvas, DotCanvas or BlockCanvas (see UnicodePlots.jl documentation)</li>
<li><code>width</code> and <code>heigth</code> to control the size of the plot. <g-emoji class="g-emoji" alias="exclamation" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/2757.png">‚ùóÔ∏è</g-emoji>These parameters can help to have a proper visualisation but be sure that the terminal you are working on is larger than setup size (otherwise graphic issues will occur)</li>
<li><code>kwargs...</code> : All arguments supported by AbstractSDRs to open configure and monitor radio devices</li>
</ul>

<p dir="auto">It is possible to change the magnitude axis during acquisition to zoom in/out and to shift the center magnitude. By default the plot is centered around 0dB and have 120dB dynamic.
By pressing different keys we can:</p>
<ul dir="auto">
<li><code>UP arrow</code> to shit the center limit up</li>
<li><code>DOWN arrow</code> to shift the center  down</li>
<li><code>LEFT arrow</code> to zoom out (increase range)</li>
<li><code>RIGHT arrow</code> to decrease range of ylims (zoom in)</li>
</ul>
<p dir="auto">Note that it is not possible to change radio parameters during runtime such as carrier frequency of sampling Rate.</p>

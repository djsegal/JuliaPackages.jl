<p dir="auto">SimGBS is a versatile method of simulating  <a href="https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0019379" rel="nofollow">Genotyping-by-Sequencing (GBS)</a> data. It can be implemented with any genome of choice. Users can modify different parameters to customise GBS setting, such as the choice of <a href="https://en.wikipedia.org/wiki/Restriction_enzyme#Examples" rel="nofollow">restriction enzyme</a> and <a href="https://www.nature.com/articles/nrg3642" rel="nofollow">sequencing depth</a>. By taking the <a href="https://academic.oup.com/g3journal/article/5/7/1415/6025367" rel="nofollow">gene-drop</a> approach, users can also specify the demographic history and define population structure (by supplying a pedigree file). Like real sequencers, SimGBS will output data into <a href="https://en.wikipedia.org/wiki/FASTQ_format" rel="nofollow">FASTQ</a> format.</p>

<p dir="auto">The following tools are recommended for downstream analyses of GBS data,</p>
<ul dir="auto">
<li>
<p dir="auto"><a href="https://github.com/AgResearch/snpGBS"><strong>snpGBS</strong></a>: a simple bioinformatics workflow to identify single nucleotide polymorphism (SNP) from Genotyping-by-Sequencing (GBS) data.</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/AgResearch/KGD"><strong>KGD</strong></a>: R code for the analysis of genotyping-by-sequencing (GBS) data, primarily to construct a genomic relationship matrix for the genotyped individuals.</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/AgResearch/GUS-LD"><strong>GUSLD</strong></a>: An R package for estimating linkage disequilibrium using low and/or high coverage sequencing data without requiring filtering with respect to read depth.</p>
</li>
<li>
<p dir="auto"><a href="https://gitlab.com/truttink/smap" rel="nofollow"><strong>SMAP</strong></a> a software package that analyzes read mapping distributions and performs haplotype calling to create multi-allelic molecular markers.</p>
</li>
</ul>

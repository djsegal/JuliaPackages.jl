<p>From the japanese word <a href="https://en.wikipedia.org/wiki/Maki-e" rel="nofollow"><em>Maki-e</em></a>, which is a technique to sprinkle lacquer with gold and silver powder.
Data is basically the gold and silver of our age, so let's spread it out beautifully on the screen!</p>

On linux you also need to add the following to get GLFW to build (if you don't have those already):</p>

<h1><a id="user-content-ecosystem" class="anchor" aria-hidden="true" href="#ecosystem"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Ecosystem</h1>
<p><code>Makie.jl</code> is the metapackage for a rich ecosystem, which consists of <a href="https://github.com/JuliaPlots/GLMakie.jl"><code>GLMakie.jl</code></a>, <a href="https://github.com/JuliaPlots/CairoMakie.jl"><code>CairoMakie.jl</code></a> and <a href="https://github.com/JuliaPlots/WGLMakie.jl"><code>WGLMakie.jl</code></a> (the backends); <a href="https://github.com/JuliaPlots/AbstractPlotting.jl"><code>AbstractPlotting.jl</code></a> (the bulk of the package); and <a href="https://github.com/JuliaPlots/StatsMakie.jl"><code>StatsMakie.jl</code></a> (statistical plotting support, as in <a href="https://github.com/JuliaPlots/StatsPlots.jl"><code>StatsPlots.jl</code></a>).</p>
<p>Examples, and test infrastructure, are hosted at <a href="https://github.com/JuliaPlots/MakieGallery.jl"><code>MakieGallery.jl</code></a></p>
<h2><a id="user-content-using-juno-with-makie" class="anchor" aria-hidden="true" href="#using-juno-with-makie"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Using Juno with Makie</h2>
<p>The default OpenGL backend for Makie is not interactive in the Juno plotpane - it just shows a PNG instead.  To get full interactivity, you can run `AbstractPlotting.inline!(false).</p>
<p>If that fails, you can disable the plotpane in Atom's settings by going to <code>Juno</code> - <code>Settings</code> - <code>UI Options</code> - Then, make sure <code>Enable Plot Plane</code> is <strong>not</strong> checked.</p>

<p>You can compile a binary for Makie and add it to your system image for fast plotting times with no JIT overhead.
To do that, you need to check out the additional packages for precompilation.
Then you can build a system image like this:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Pkg
<span class="pl-c"><span class="pl-c">#</span> add PackageCompiler and other dependencies</span>
<span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg</span>"</span>add PackageCompiler#master<span class="pl-pds">"</span></span> <span class="pl-c"><span class="pl-c">#</span> since there hasn't been a release of the new one yet</span>

<span class="pl-k">using</span> PackageCompiler

<span class="pl-c"><span class="pl-c">#</span> This will create a system image in the current directory, which you can </span>
<span class="pl-c"><span class="pl-c">#</span> use by launching Julia with `julia -J ./MakieSys.so`.</span>
PackageCompiler<span class="pl-k">.</span><span class="pl-c1">create_sysimage</span>(
    <span class="pl-c1">:Makie</span>; 
    sysimage_path<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>MakieSys.so<span class="pl-pds">"</span></span>, 
    precompile_execution_file<span class="pl-k">=</span><span class="pl-c1">joinpath</span>(<span class="pl-c1">pkgdir</span>(Makie), <span class="pl-s"><span class="pl-pds">"</span>test<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>test_for_precompile.jl<span class="pl-pds">"</span></span>)
)</pre></div>
<p>Should the display not work after compilation, call <code>AbstractPlotting.__init__()</code> immediately after <code>using Makie</code>.</p>

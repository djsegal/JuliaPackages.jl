<p>Makie is the metapackage for a rich ecosystem, which consists of <a href="https://github.com/JuliaPlots/GLMakie.jl">GLMakie.jl</a>, <a href="https://github.com/JuliaPlots/CairoMakie.jl">CairoMakie.jl</a> and <a href="https://github.com/JuliaPlots/WGLMakie.jl">WGLMakie.jl</a> (the backends); <a href="https://github.com/JuliaPlots/AbstractPlotting.jl">AbstractPlotting.jl</a> (the bulk of the package); and <a href="https://github.com/JuliaPlots/StatsMakie.jl">StatsMakie.jl</a> (statistical plotting support, as in <a href="https://github.com/JuliaPlots/StatsPlots.jl">StatsPlots.jl</a>).</p>
<p><a href="https://github.com/jkrumbiegel/MakieLayout.jl">MakieLayout.jl</a> is a layout manager for Makie.  It brings grid layouts, a new 2D axis, and widgets like sliders and buttons that behave well in a layout to Makie.</p>
<p>There is experimental support for using Plots.jl and RecipesBase.jl recipes in <a href="https://github.com/JuliaPlots/MakieRecipes.jl">MakieRecipes.jl</a>, and a prototype TeX plotting implementation with vector support in <a href="https://github.com/JuliaPlots/MakieTeX.jl">MakieTeX.jl</a>.</p>
<p>Examples, and test infrastructure, are in <a href="https://github.com/JuliaPlots/MakieGallery.jl">MakieGallery.jl</a>.</p>

<h2><a id="user-content-using-juno-with-makie" class="anchor" aria-hidden="true" href="#using-juno-with-makie"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Using Juno with Makie</h2>
<p>The default OpenGL backend for Makie is not interactive in the Juno plotpane - it just shows a PNG instead.  To get full interactivity, you can run <code>AbstractPlotting.inline!(false)</code>.</p>
<p>If that fails, you can disable the plotpane in Atom's settings by going to <code>Juno</code> - <code>Settings</code> - <code>UI Options</code> - Then, make sure <code>Enable Plot Plane</code> is <strong>not</strong> checked.</p>

<p>Currently, only non-interactive plots are supported. (See the issues <a href="https://github.com/JuliaPlots/Makie.jl/issues/15">#15</a> and <a href="https://github.com/JuliaPlots/Makie.jl/issues/266">#266</a>.)</p>

<h2><a id="user-content-precompilation" class="anchor" aria-hidden="true" href="#precompilation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Precompilation</h2>
<p>You can compile a binary for Makie and add it to your system image for fast plotting times with no JIT overhead.
To do that, you need to check out the additional packages for precompilation.
Then you can build a system image like this:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Pkg
<span class="pl-c"><span class="pl-c">#</span> add PackageCompiler and other dependencies</span>
<span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg</span>"</span>add PackageCompiler<span class="pl-pds">"</span></span>
<span class="pl-c"><span class="pl-c">#</span> Make sure you have v1.0 or higher of PackageCompiler!</span>

<span class="pl-k">using</span> PackageCompiler

<span class="pl-c"><span class="pl-c">#</span> This will create a system image in the current directory, which you can</span>
<span class="pl-c"><span class="pl-c">#</span> use by launching Julia with `julia -J ./MakieSys.so`.</span>
PackageCompiler<span class="pl-k">.</span><span class="pl-c1">create_sysimage</span>(
    <span class="pl-c1">:Makie</span>;
    sysimage_path<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>MakieSys.so<span class="pl-pds">"</span></span>,
    precompile_execution_file<span class="pl-k">=</span><span class="pl-c1">joinpath</span>(<span class="pl-c1">pkgdir</span>(Makie), <span class="pl-s"><span class="pl-pds">"</span>test<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>test_for_precompile.jl<span class="pl-pds">"</span></span>)
)</pre></div>
<p>Should the display not work after compilation, call <code>AbstractPlotting.__init__()</code> immediately after <code>using Makie</code>.</p>

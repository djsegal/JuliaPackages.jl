<p dir="auto">RegressionAndOtherStories.jl contains supporting (Julia) functions and the data files used in <a href="https://avehtari.github.io/ROS-Examples/" rel="nofollow">"Regression and Other Stories"</a> by Andrew Gelham, Jennifer Hill and Aki Vehtari.</p>
<p dir="auto">The package is also used in project SR2StanPluto.jl v9+, a revised inplementation of the <a href="https://github.com/StatisticalRethinkingJulia">Statistical Rethinking</a> support functions using Makie.jl, CausalInference.jl and GraphViz.jl.</p>
<h2 dir="auto"><a id="user-content-contents" class="anchor" aria-hidden="true" href="#contents"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Contents</h2>
<p dir="auto">The <strong>supporting functions</strong> are intended to be used in (currently) 3 Julia projects (also under development), <a href="https://github.com/RegressionAndOtherStoriesJulia/ROSStanPluto.jl">ROSStanPluto.jl</a>, <a href="https://github.com/RegressionAndOtherStoriesJulia/ROSTuringPluto.jl">ROSTuringPluto.jl</a> and <a href="https://xcelab.net/rm/statistical-rethinking/" rel="nofollow">SR2StanPluto</a>.</p>
<p dir="auto">All <strong>data files</strong> are in <code>.csv</code> format and located in the <code>data</code> directory.</p>
<p dir="auto">If RegressionAndOtherStories.jl is loaded, the files can be read in as a DataFrame using:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="hibbs = CSV.read(ros_datadir(&quot;ElectionsEconomy&quot;, &quot;hibbs.csv&quot;), DataFrame)"><pre class="notranslate"><code>hibbs = CSV.read(ros_datadir("ElectionsEconomy", "hibbs.csv"), DataFrame)
</code></pre></div>
<p dir="auto">For that purpose <code>ros_datadir()</code> is exported.</p>
<p dir="auto">If needed, Stata files (<code>.dat</code>) have been converted to <code>.csv</code> files using the scripts in the <code>scripts</code> directory, e.g. see <code>scripts\hdi.jl</code>. To access the Stata files in the R package <a href="https://github.com/avehtari/ROS-Examples">ROS-Examples</a> RegressionAndOtherStories.jl expects the environment variable <code>JULIA_ROS_HOME</code> to be defined, e.g.:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="ENV[&quot;JULIA_ROS_HOME&quot;] = expanduser(&quot;~/Projects/R/ROS-Examples&quot;)"><pre class="notranslate"><code>ENV["JULIA_ROS_HOME"] = expanduser("~/Projects/R/ROS-Examples")
</code></pre></div>
<p dir="auto">R itself does not necessarily need to be installed for this to work.</p>
<p dir="auto">If so desired, direct use of the Stata files is also possible as the Stata to .csv file conversion scripts mentioned above show.</p>

<p dir="auto">RegressionAndOtherStories.jl v9+ is using Julia's package extension option. In particular Turing, Stan, Makie, GraphViz and CausalInference, if needed, are included as extensions.</p>
<p dir="auto">Over time I might minimize the use of AlgebraOfGraphics.jl. It is a nice package but also a bit more difficult to tailor (compared to Makie/GLMakie).</p>
<p dir="auto">In working on this I will move over (and likely update) several important functions from StatisticalRethinking.jl as well, e.g. <code>link()</code>.</p>
<p dir="auto">I expect I can use ParetoSmoothedImportanceSampling.jl as is but will take another look at PSIS.jl and ParetoSmooth.jl when revising the relevant chapters.</p>
<h2 dir="auto"><a id="user-content-project-maintenance-for-pluto-notebooks" class="anchor" aria-hidden="true" href="#project-maintenance-for-pluto-notebooks"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Project maintenance for Pluto notebooks</h2>
<p dir="auto">In the subdirectory <code>src/Maintenance/reset_notebooks.jl</code> is a function I use in the Pluto notebook projects (SR2StanPluto, ROSStanPluto, etc.). The function potentially makes two changes to selected notebooks:</p>
<ol dir="auto">
<li>If it finds a line starting with <code>Pkg.activate</code> it disables that line if <code>reset_activate = true</code>.</li>
<li>If it finds a line starting with <code>#Pkg.activate</code> it enables that line if <code>set_activate = true</code>.</li>
<li>It removes the Project and Manifest sections of all notebooks selected for reset. See the maintenance notebooks in projects such as SR2StanPluto and ROSStanPluto.</li>
</ol>
<p dir="auto">Using <code>Pkg.activate(...)</code> is useful if your workflow uses many different notebooks.</p>

<p dir="auto"><strong>Installation</strong></p>
<p dir="auto">To install the package:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; using Pkg
julia&gt; Pkg.add(&quot;DICOM&quot;)"><pre class="notranslate"><code>julia&gt; using Pkg
julia&gt; Pkg.add("DICOM")
</code></pre></div>
<p dir="auto">Load the package by</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; using DICOM"><pre class="notranslate"><code>julia&gt; using DICOM
</code></pre></div>
<p dir="auto"><strong>Reading Data</strong></p>
<p dir="auto">Read a DICOM file by</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; dcm_data = dcm_parse(&quot;path/to/dicom/file&quot;)"><pre class="notranslate"><code>julia&gt; dcm_data = dcm_parse("path/to/dicom/file")
</code></pre></div>
<p dir="auto">The data in <code>dcm_data</code> is structured as a dictionary, and individual DICOM elements can be accessed by their hex tag.
For example, the hex tag of "Pixel Data" is <code>7FE0,0010</code>, and it can be accessed in Julia by <code>dcm_data[(0x7FE0,0x0010)]</code> or by <code>dcm_data[tag"PixelData"]</code>.</p>
<p dir="auto">Multiple DICOM files in a folder can be read by</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; dcm_data_array = dcmdir_parse(&quot;path/to/dicom/folder&quot;)"><pre class="notranslate"><code>julia&gt; dcm_data_array = dcmdir_parse("path/to/dicom/folder")
</code></pre></div>
<p dir="auto"><strong>Writing Data</strong></p>
<p dir="auto">Data can be written to a DICOM file by</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; dcm_write(&quot;path/to/output/file&quot;, dcm_data)"><pre class="notranslate"><code>julia&gt; dcm_write("path/to/output/file", dcm_data)
</code></pre></div>
<p dir="auto"><strong>Additional Notes</strong></p>
<p dir="auto">DICOM files should begin with a 128-bytes (which are ignored) followed by the string <code>DICM</code>.
If this preamble is missing, then the file can be parsed by <code>dcm_parse(path/to/file, preamble = false)</code>.</p>
<p dir="auto">DICOM files use either explicit or implicit value representation (VR). For implicit files, <code>DICOM.jl</code> will use a lookup table to guess the VR from the DICOM element's hex tag. For explicit files, <code>DICOM.jl</code> will read the VRs from the file.</p>
<ul dir="auto">
<li>
<p dir="auto">An auxiliary user-defined dictionary can be supplied to override the default lookup table
For example, the "Instance Number" - tag <code>(0x0020,0x0013)</code> - is an integer (default VR = "IS"). We can read this as a float by setting the VR to "DS" by:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="my_vrs = Dict( (0x0020,0x0013) =&gt; &quot;DS&quot; )
dcm_data = dcm_parse(&quot;path/to/dicom/file&quot;, aux_vr = my_vrs)"><pre class="notranslate"><code>my_vrs = Dict( (0x0020,0x0013) =&gt; "DS" )
dcm_data = dcm_parse("path/to/dicom/file", aux_vr = my_vrs)
</code></pre></div>
<p dir="auto">Now <code>dcm_data[(0x0020,0x0013)]</code> will return a float instead of an integer.</p>
<ul dir="auto">
<li>The parsed VRs are stored in <code>dcm_data.vr</code></li>
</ul>
</li>
<li>
<p dir="auto">It is possible to skip an element by setting its VR to <code>""</code>.
For example, we can skip reading the Instance Number by</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="my_vrs = Dict( (0x0020,0x0013) =&gt; &quot;&quot; )
dcm_data = dcm_parse(&quot;path/to/dicom/file&quot;, aux_vr = my_vr)"><pre class="notranslate"><code>my_vrs = Dict( (0x0020,0x0013) =&gt; "" )
dcm_data = dcm_parse("path/to/dicom/file", aux_vr = my_vr)
</code></pre></div>
<p dir="auto">and now <code>dcm_data[(0x0020,0x0013)]</code> will return an error because the key <code>(0x0020,0x0013)</code> doesn't exist - it was skipped during reading.</p>
</li>
<li>
<p dir="auto">The user-supplied VR can contain a master VR with the tag <code>(0x0000,0x0000)</code> which will be used whenever <code>DICOM.jl</code> is unable to guess the VR on its own. This is convenient for reading older dicom files and skipping retired elements - i.e. where the VR lookup fails - by:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="my_vrs = Dict( (0x0000,0x0000) =&gt; &quot;&quot; )
dcm_data = dcm_parse(&quot;path/to/dicom/file&quot;, aux_vr = my_vrs)"><pre class="notranslate"><code>my_vrs = Dict( (0x0000,0x0000) =&gt; "" )
dcm_data = dcm_parse("path/to/dicom/file", aux_vr = my_vrs)
</code></pre></div>
</li>
<li>
<p dir="auto">A user-supplied VR can also be supplied during writing, e.g.:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; dcm_write(&quot;path/to/output/file&quot;, dcm_data, aux_vr = user_defined_vr)"><pre class="notranslate"><code>julia&gt; dcm_write("path/to/output/file", dcm_data, aux_vr = user_defined_vr)
</code></pre></div>
<p dir="auto">where <code>user_defined_vr</code> is a dictionary which maps the hex tag to the VR.</p>
</li>
</ul>

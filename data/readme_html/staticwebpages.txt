<h1 dir="auto"><a id="user-content-staticwebpagesjl" class="anchor" aria-hidden="true" href="#staticwebpagesjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>StaticWebPages.jl</h1>
<p dir="auto">A user-friendly static website generator written in Julia. No programming nor webdev skills required!!! Entirely Open-Source (GPLv2 licence)!</p>
<p dir="auto">Although this generator originally targets academicians, it works out well for personal webpage and any static usage (front-end only) A more advanced static framework in Julia is available (cf <a href="https://github.com/tlienart/Franklin.jl">Franklin.jl</a>). For a full-stack web framework, please look at the great <a href="https://www.genieframework.com/" rel="nofollow">Genie.jl</a>.</p>
<p dir="auto">This generator is for you if</p>
<ul dir="auto">
<li>you want something simple to update and manage</li>
<li>responsive design (looks good on most devices)</li>
<li>want light load on the server and/or low bandwidth use</li>
<li>have no time/interest in making your own website by "hand"</li>
<li>new item/gadget access with a simple update in the julia command line</li>
</ul>
<p dir="auto">The beta only uses text files and command lines, but the first stable release will also integrate a graphical user interface (GUI).</p>
<p dir="auto">I deeply apologize for those that tested the versions <code>v0.1.x</code>, as, from <code>v0.2.0</code>, the syntax has changed a lot. The reason is to add options to most components. It should not change drastically from now on.</p>

<p dir="auto">The user provides the content (text, images, files) and select which <code>Items</code> will display it. As simple as that. A full working example is provided here (or in the example folder). The result is available as <a href="https://baffier.ovh" rel="nofollow">https://baffier.ovh</a></p>

<p dir="auto">The only requirement is to install Julia and this package (preferably through the package interface of Julia). Please install the last stable release of <a href="https://julialang.org/downloads/" rel="nofollow">Julia</a> on the official download page.</p>
<p dir="auto">In the Julia REPL (that you can launch as a standalone or call within a console), please enter Pkg REPL. To quote the package manager documentation:</p>
<blockquote>
<p dir="auto">Pkg comes with a REPL. Enter the Pkg REPL by pressing ] from the Julia REPL. To get back to the Julia REPL, press backspace or ^C.</p>
</blockquote>
<p dir="auto">The following code snippet update the general registry of Julia's packages, then install the <code>StaticWebPages.jl</code> package.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="(@v1.5) pkg&gt; up
(@v1.5) pkg&gt; add StaticWebPages"><pre class="notranslate"><code>(@v1.5) pkg&gt; up
(@v1.5) pkg&gt; add StaticWebPages
</code></pre></div>
<p dir="auto">You can check that the installation is complete and trigger a precompilation of the package (usually take a few minutes) by using the following command.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="import StaticWebPages"><pre><span class="pl-k">import</span> StaticWebPages</pre></div>
<p dir="auto">Please note that precompilation occurs at first use after installation and updates.</p>
<p dir="auto">The package can be use from the REPL, but we recommend it to be used through a script file as the <code>run.jl</code> presented below. Running the script should be as simple as writing the following command line.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia run.jl"><pre>julia run<span class="pl-k">.</span>jl</pre></div>
<p dir="auto">or in the REPL</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="include(&quot;run.jl&quot;)"><pre><span class="pl-c1">include</span>(<span class="pl-s"><span class="pl-pds">"</span>run.jl<span class="pl-pds">"</span></span>)</pre></div>

<p dir="auto">A compressed template can be found in the <code>example/</code> folder as either <a href="https://raw.githubusercontent.com/Humans-of-Julia/StaticWebPages.jl/master/example/template.tar" rel="nofollow">template.tar</a> or <a href="https://raw.githubusercontent.com/Humans-of-Julia/StaticWebPages.jl/master/example/template.zip" rel="nofollow">template.zip</a>.</p>
<p dir="auto">The two first following files are the only requirement. However, most users will prefer to provide some images/pictures and additional files such as a cv, a bibliography, and some downloadable content.</p>

<h3 dir="auto"><a id="user-content-writing-tests" class="anchor" aria-hidden="true" href="#writing-tests"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Writing tests</h3>
<p dir="auto">In the framework of this package, you write your tests inside <code>@testitem</code> macros. Each test item is the smallest unit of code that can be individually run and must be entirely self contained. In fact, when a <code>@testitem</code> is executed, all the code in the test item will be placed into a new Julia <code>module</code> and then this new module will be run. A typical <code>@testitem</code> might look like this:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="@testitem &quot;First tests&quot; begin
    x = foo(&quot;bar&quot;)

    @test length(x)==3
    @test x == &quot;bar&quot;
end"><pre><span class="pl-c1">@testitem</span> <span class="pl-s"><span class="pl-pds">"</span>First tests<span class="pl-pds">"</span></span> <span class="pl-k">begin</span>
    x <span class="pl-k">=</span> <span class="pl-c1">foo</span>(<span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>)

    <span class="pl-c1">@test</span> <span class="pl-c1">length</span>(x)<span class="pl-k">==</span><span class="pl-c1">3</span>
    <span class="pl-c1">@test</span> x <span class="pl-k">==</span> <span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>
<span class="pl-k">end</span></pre></div>
<p dir="auto">Note how the first argument to the <code>@testitem</code> macro is a name for the test item, followed by a <code>begin end</code> block that contains the actual test code. Note that you do not need to load the <code>Test</code> package nor the package you are testing, both of these are automatically loaded into the <code>@testitem</code>. Inside the test code, we first  run some code from our package and finally use the standard base library <code>@test</code> macros to test whether our function returns the correct results.</p>
<p dir="auto">You can put <code>@testitem</code> macros into any *.jl Julia file in your package, even next to the functions that you are testing, or more traditionally into Julia files in your <code>test</code> folder.</p>
<h4 dir="auto"><a id="user-content-writing-test-items-inside-the-regular-package-code" class="anchor" aria-hidden="true" href="#writing-test-items-inside-the-regular-package-code"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Writing test items inside the regular package code</h4>
<p dir="auto">If you want to put your test code inside the code files of your actual package, you should add the <a href="https://github.com/julia-vscode/TestItems.jl">TestItems.jl</a> package as a regular dependency of your package. <code>TestItems</code> exports the <code>@testitem</code> macro (and nothing else), so your main package file might look like this:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="module MyPackage

using TestItems

function foo(x)
    return x*x
end

@testitem &quot;Test for foo&quot; begin
    x = foo(&quot;bar&quot;)

    @test x == &quot;barbar&quot;
end

end"><pre><span class="pl-k">module</span> MyPackage

<span class="pl-k">using</span> TestItems

<span class="pl-k">function</span> <span class="pl-en">foo</span>(x)
    <span class="pl-k">return</span> x<span class="pl-k">*</span>x
<span class="pl-k">end</span>

<span class="pl-c1">@testitem</span> <span class="pl-s"><span class="pl-pds">"</span>Test for foo<span class="pl-pds">"</span></span> <span class="pl-k">begin</span>
    x <span class="pl-k">=</span> <span class="pl-c1">foo</span>(<span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>)

    <span class="pl-c1">@test</span> x <span class="pl-k">==</span> <span class="pl-s"><span class="pl-pds">"</span>barbar<span class="pl-pds">"</span></span>
<span class="pl-k">end</span>

<span class="pl-k">end</span></pre></div>
<p dir="auto">Adding tests like this to your regular package should have a minimal runtime overhead for your package: The <code>TestItems</code> package only defines the <code>@testitem</code> macro, and the <code>@testitem</code> macro always returns <code>nothing</code>, so that all test code is essentially removed from your package when a user loads it.</p>
<h4 dir="auto"><a id="user-content-writing-test-items-in-test-files" class="anchor" aria-hidden="true" href="#writing-test-items-in-test-files"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Writing test items in test files</h4>
<p dir="auto">You can also place <code>@testitem</code>s in test files inside your <code>test</code> folder (or any folder in your package). These test files do <em>not</em> need to be included in the <code>test/runtests.jl</code> file, they can just be standalone files. A typical test file <code>test/test_foo.jl</code> might look like this:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="@testitem &quot;Another test for foo&quot; begin
    x = foo(&quot;bar&quot;)

    @test x != &quot;bar&quot;
end"><pre><span class="pl-c1">@testitem</span> <span class="pl-s"><span class="pl-pds">"</span>Another test for foo<span class="pl-pds">"</span></span> <span class="pl-k">begin</span>
    x <span class="pl-k">=</span> <span class="pl-c1">foo</span>(<span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>)

    <span class="pl-c1">@test</span> x <span class="pl-k">!=</span> <span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>
<span class="pl-k">end</span></pre></div>
<p dir="auto">Note that in this case you don't even have to use the <code>TestItems</code> package in this file. Because this new file <code>test/test_foo.jl</code> is never going to be run as an entire file, we can skip the <code>using TestItems</code> part.</p>
<h4 dir="auto"><a id="user-content-test-item-options" class="anchor" aria-hidden="true" href="#test-item-options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Test item options</h4>
<p dir="auto">You can set specific options for individual <code>@testitem</code>s. At the moment the framework supports two, namely a tag system and an option to control default imports.</p>
<h5 dir="auto"><a id="user-content-tags" class="anchor" aria-hidden="true" href="#tags"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Tags</h5>
<p dir="auto">You can assign arbitrary tags to a <code>@testitem</code>. A tag must be a <code>Symbol</code>, and you can assign multiple tags to each <code>@testitem</code>. Tags can be used to more easily select subsets of <code>@testitem</code>s for execution. Here is an example that assigns the <code>:skipci</code> tag to a <code>@testitem</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="@testitem &quot;Another test for foo&quot; tags=[:skipci] begin
    x = foo(&quot;bar&quot;)

    @test x != &quot;bar&quot;
end"><pre><span class="pl-c1">@testitem</span> <span class="pl-s"><span class="pl-pds">"</span>Another test for foo<span class="pl-pds">"</span></span> tags<span class="pl-k">=</span>[<span class="pl-c1">:skipci</span>] <span class="pl-k">begin</span>
    x <span class="pl-k">=</span> <span class="pl-c1">foo</span>(<span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>)

    <span class="pl-c1">@test</span> x <span class="pl-k">!=</span> <span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>
<span class="pl-k">end</span></pre></div>
<h5 dir="auto"><a id="user-content-default-imports" class="anchor" aria-hidden="true" href="#default-imports"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Default imports</h5>
<p dir="auto">When you write a <code>@testitem</code>, by default the package being tested and the <code>Test</code> package are imported via an invisible <code>using</code> statement. In some cases this might not be desirable, so one can control this behavior on a per <code>@testitem</code> level via the <code>default_imports</code> option, which accepts a <code>Bool</code> value. To disable these default imports you, you would write:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="@testitem &quot;Another test for foo&quot; default_imports=false begin
    using MyPackage, Test

    x = foo(&quot;bar&quot;)

    @test x != &quot;bar&quot;
end"><pre><span class="pl-c1">@testitem</span> <span class="pl-s"><span class="pl-pds">"</span>Another test for foo<span class="pl-pds">"</span></span> default_imports<span class="pl-k">=</span><span class="pl-c1">false</span> <span class="pl-k">begin</span>
    <span class="pl-k">using</span> MyPackage, Test

    x <span class="pl-k">=</span> <span class="pl-c1">foo</span>(<span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>)

    <span class="pl-c1">@test</span> x <span class="pl-k">!=</span> <span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>
<span class="pl-k">end</span></pre></div>
<p dir="auto">Note how we now need to add the line <code>using MyPackage, Test</code> manually to our <code>@testitem</code> so that we have access to the <code>foo</code> function and <code>@test</code> macro.</p>
<h3 dir="auto"><a id="user-content-running-tests" class="anchor" aria-hidden="true" href="#running-tests"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Running tests</h3>
<p dir="auto">At the moment there are two ways to run <code>@testitem</code>s: with the integrated test UI in the Julia VS Code extension, or with this package TestItemRunner.jl. In both cases test item detection is based on syntactic analysis, i.e. no code from your package is run to detect test items. Both execution engines will instead look at all *.jl files in your package folder, identify all <code>@testitem</code> calls and then provide ways to run them.</p>

<p dir="auto">If you want your tests to run when a user calls the regular base test functionality, or have you your tests run during CI runs, you can simply add this package TestItemRunner.jl as a test dependency to your package, and then add this content as your <code>test/runtests.jl</code> file:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using TestItemRunner

@run_package_tests"><pre><span class="pl-k">using</span> TestItemRunner

<span class="pl-c1">@run_package_tests</span></pre></div>
<p dir="auto">Sometimes it is convenient to not run all detected <code>@testitem</code>s but only a subset. You can specify a custom filter function to achieve this. Your filter function will be called for each detected <code>@testitem</code>, and must return a <code>Bool</code> value indicating whether this particular <code>@testitem</code> should be executed or not. For example, here is an example that will only run <code>@testitem</code>s that don't have the <code>:skipci</code> tag assigned:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using TestItemRunner

@run_package_tests filter=ti-&gt;!(:skipci in ti.tags)"><pre><span class="pl-k">using</span> TestItemRunner

<span class="pl-c1">@run_package_tests</span> filter<span class="pl-k">=</span>ti<span class="pl-k">-&gt;</span><span class="pl-k">!</span>(<span class="pl-c1">:skipci</span> <span class="pl-k">in</span> ti<span class="pl-k">.</span>tags)</pre></div>
<p dir="auto">The value that is passed to your custom filter function has three fields that you can use to extract information about a test item:</p>
<ul dir="auto">
<li><code>name</code>: The full name of the <code>@testitem</code>, as a <code>String</code>.</li>
<li><code>filename</code>: The full absolute filename of the file in which the <code>@testitem</code> is defined.</li>
<li><code>tags</code>: A <code>Vector{Symbol}</code> with all the tags that you defined for this particular <code>@testitem</code>.</li>
</ul>
<p dir="auto">If you want to print a summary of test results which shows all the <code>@testitems</code>, then you can pass <code>verbose=true</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using TestItemRunner

@run_package_tests verbose=true"><pre><span class="pl-k">using</span> TestItemRunner

<span class="pl-c1">@run_package_tests</span> verbose<span class="pl-k">=</span><span class="pl-c1">true</span></pre></div>

<p>CartesianGeneticProgramming.jl can be installed through the Julia package manager</p>
<div class="highlight highlight-source-julia"><pre>pkg<span class="pl-k">&gt;</span> add https<span class="pl-k">:</span><span class="pl-k">//</span>github<span class="pl-k">.</span>com<span class="pl-k">/</span>d9w<span class="pl-k">/</span>CartesianGeneticProgramming<span class="pl-k">.</span>jl</pre></div>
<p>This package depends on <a href="https://github.com/d9w/Cambrian.jl">Cambrian.jl</a> which
is not yet in the General repository and therefore must also be installed from
github.</p>
<h2><a id="user-content-examples" class="anchor" aria-hidden="true" href="#examples"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Examples</h2>
<p>Examples of usage are provided in <code>scripts</code>. To run any of these examples, do</p>
<div class="highlight highlight-source-shell"><pre>$ julia scripts/gym.jl</pre></div>
<p>This will create and populate <code>gens</code> and <code>logs</code> folders, which store CGP
individuals and evolutionary logs, respectively.</p>
<p>Note that these examples have only been tested on Linux and have additional
dependencies which must be installed, like <code>PyCall</code> or
<code>ArcadeLearningEnvironment</code>.</p>

<p><code>CGP.jl</code> comes with tests which can offer examples of detailed usage of the
different genetic operators and CGP extensions. To run all tests, use</p>
<pre lang="julilang"><code>pkg&gt; test CartesianGeneticProgramming
</code></pre>

<p>Configuration is handled by <code>src/config.jl</code>. Configurable options are read in
through YAML files (an example being <code>cfg/test.yaml</code>). All possible node
functions are defined in <code>src/functions.jl</code> and are populated in the
configuration files.</p>

<h2><a id="user-content-examples" class="anchor" aria-hidden="true" href="#examples"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Examples</h2>
<p>Examples of usage are provided in <code>scripts</code>. To run any of these examples, do</p>
<div class="highlight highlight-source-shell position-relative" data-snippet-clipboard-copy-content="$ julia scripts/gym.jl
"><pre>$ julia scripts/gym.jl</pre></div>
<p>This will create and populate <code>gens</code> and <code>logs</code> folders, which store CGP
individuals and evolutionary logs, respectively.</p>
<p>Note that these examples have only been tested on Linux and have additional
dependencies which must be installed, like <code>PyCall</code> or
<code>ArcadeLearningEnvironment</code>.</p>

<p>Configuration is handled by <code>src/config.jl</code>. Configurable options are read in
through YAML files (an example being <code>cfg/test.yaml</code>). All possible node
functions are defined in <code>src/functions.jl</code> and are populated in the
configuration files.</p>
<h2><a id="user-content-mixed-type-cgp" class="anchor" aria-hidden="true" href="#mixed-type-cgp"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Mixed-Type CGP</h2>
<p>This repository is now intended only for floating point inputs and therefore no
longer implements Mixed-Type CGP. The commit reference for the results in [1] is
<a href="https://github.com/d9w/CartesianGeneticProgramming.jl/commit/096e84316c0c265e17ea75de7b5fcaa6427fa5dc">096e843</a>,
however this requires Julia 0.6 and may no longer function. MTCGP is currently
being rewritten at <a href="https://github.com/d9w/MTCGP.jl">https://github.com/d9w/MTCGP.jl</a>. This is done to separate
heavy image processing package dependencies which are not necessary for base
CGP.</p>
<p>[1] Wilson, Dennis G., et al. "Evolving simple programs for playing Atari
games." Proceedings of the Genetic and Evolutionary Computation Conference. ACM,
2018.</p>

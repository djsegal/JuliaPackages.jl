<p dir="auto">The InspectDR programming interface is not particularly refined. Code used to generate more complex plots can therefore be a bit difficult to read/maintain.</p>
<p dir="auto">It will most likely be easier to generate <code>InspectDR</code> plots by leveraging higher-level APIs (unless you have stringent requirements on time-to-first-plot or number of dependencies).  Alternative APIs supporting <code>InspectDR</code> include:</p>
<ul dir="auto">
<li><a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a>: Succinct plotting interface ideal for interactive exploration (supports multiple backends).</li>
<li><a href="https://github.com/ma-laforge/CMDimData.jl">CMDimData.jl</a>: Facilitates parametric analysis with continous <var>f(x)</var> interpolation &amp; multi-dimensional plots. Built using <a href="https://github.com/ma-laforge/MDDatasets.jl">MDDatasets.jl</a> module.</li>
<li><a href="https://github.com/ma-laforge/CMDimCircuits.jl">CMDimCircuits.jl</a>: Extends <a href="https://github.com/ma-laforge/CMDimData.jl">CMDimData.jl</a> with circuit-specific functionnality (ex: signal processing, network analysis, ...).</li>
</ul>
<p dir="auto">Note: In instances where higher-level APIs have limited control over the plot, it is typically possible to tweak the final apearance of the "rendered" <code>::InspectDR.Plot</code> object using the <code>InspectDR</code> API.</p>

<h2 dir="auto"><a id="user-content-description" class="anchor" aria-hidden="true" href="#description"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Description</h2>
<p dir="auto">InspectDR is a fast plotting tool with a responsive GUI, targeting quick navigation of simulation results.  In design applications, InspectDR allows for efficient, interactive data exploration, thus shortening each iteration of the design cycle.</p>
<p dir="auto"><strong>Motivation:</strong> Despite their great quality, most of Julia's current plotting options were found to be either too slow, and/or provide inadequate interactivity for the author's needs.</p>
<p dir="auto">The InspectDR library is implemented using <strong>3 distinct plot layers</strong>:</p>
<ul dir="auto">
<li><strong>Plot image layer:</strong> Implemented with the <a href="https://cairographics.org" rel="nofollow">Cairo library</a>, the plot image layer allows the user to render (multi-) plots as simple images.</li>
<li><strong>Plot widget layer:</strong> Library users can also integrate plots to their own <a href="https://www.gtk.org" rel="nofollow">GTK+</a> application by instantiating a single InspectDR widget.</li>
<li><strong>Plot application layer:</strong> Most end users will likely display/interact with plots/data using the built-in Julia/<a href="https://www.gtk.org" rel="nofollow">GTK+</a> multi-plot application.</li>
</ul>
<p dir="auto">Users are encouraged to open an issue if it is unclear how to utilize a particular layer.  Documentation is a bit limited at the moment.</p>

<p dir="auto">The following highlights a few interesting features of InspectDR:</p>
<ul dir="auto">
<li>Publication-quality output.</li>
<li>Included as a "backend" of <a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a>.</li>
<li>Relatively short load times / time to first plot.</li>
<li>Designed with larger datasets in mind:
<ul dir="auto">
<li>Responsive even with moderate (&gt;200k points) datasets.</li>
<li>Confirmed to handle 2GB datsets with reasonable speed on older desktop running Windows 7 (drag+pan of data area highly discouraged).</li>
</ul>
</li>
<li>Support for stacked, multi-<em><strong>strip</strong></em> plots with common x-axis values.</li>
<li>Support for Smith charts (admittance &amp; impedance - see <a href="doc/api.md#PlotGenerators">Plot generators</a>).</li>
<li>Support for various types of annotation:
<ul dir="auto">
<li>User-programmable text, polyline, vertical &amp; horizontal bars.</li>
<li>Drag &amp; drop Œî-markers (Measures/displays Œîx, Œîy &amp; slope).</li>
</ul>
</li>
<li>Interactive <a href="doc/input_bindings.md">mouse/keybindings</a>.
<ul dir="auto">
<li>Fast &amp; simple way to pan/zoom into data.</li>
<li>In line with other similar tools.</li>
<li>Create drag &amp; drop Œî-markers.</li>
</ul>
</li>
<li><a href="doc/api.md#Layout_Stylesheets">Layout &amp; stylesheets</a>.
<ul dir="auto">
<li>See <a href="sample/demo12.jl">demo targeting IEEE publications @300 dpi</a></li>
<li>Add custom stylesheets.</li>
</ul>
</li>
</ul>

<h3 dir="auto"><a id="user-content-f1-acceleration" class="anchor" aria-hidden="true" href="#f1-acceleration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>"F1" acceleration</h3>
<p dir="auto">InspectDR.jl includes specialized algorithms to accellerate plotting of large "F1" datasets (functions of 1 argument) in order to maintain a good "real-time" (interactive) user experience.</p>
<p dir="auto">A dataset is defined as a function of 1 argument ("F1") if it satisfies:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="y = f(x), where x: sorted, real vector"><pre class="notranslate"><code>y = f(x), where x: sorted, real vector
</code></pre></div>
<p dir="auto">Examples of "F1" datasets include <strong>time domain</strong> (<code>y(x=time)</code>) and <strong>frequncy domain</strong> (<code>X(w)</code>) data.</p>
<p dir="auto">"F1" acceleration is obtained by dropping points in order to speed up the rendering process.</p>
<p dir="auto"><em><strong>IMPORTANT:</strong></em> "F1" acceleration tends to generate erroneous-looking plots whenever glyphs are displayed.  This is because the dropped points may become very noticeable.  Consequently, InspectDR will, by default, only apply "F1" acceleration on datasets drawn without glyphs (lines only).</p>
<p dir="auto">To change when InspectDR applies "F1" acceleration to drop points, look for the <code>:droppoints</code> entry in the <a href="doc/config.md">Configuration/Defaults</a> section.</p>

<p dir="auto">InspectDR.jl also supports generic 2D plotting.  More specifically, the tool is capable of plotting arbitrary 2D datasets that satisfy:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="(x,y) = (u[i], v[i]), for i in [1...N]"><pre class="notranslate"><code>(x,y) = (u[i], v[i]), for i in [1...N]

<p dir="auto">Examples of of such plots (where x-values are not guaranteed to be sorted) include:</p>
<ul dir="auto">
<li>Nyquist plots</li>
<li>Lissajous plots</li>
<li>Smith/polar (S-Parameter) charts</li>
</ul>

<h2><a id="user-content-description" class="anchor" aria-hidden="true" href="#description"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Description</h2>
<p>InspectDR is a fast plotting tool with a responsive GUI, targeting quick navigation of simulation results.  In design applications, InspectDR allows for efficient, interactive data exploration, thus shortening each iteration of the design cycle.</p>
<p><strong>Motivation:</strong> Despite their great quality, most of Julia's current plotting options were found to be either too slow, and/or provide inadequate interactivity for the author's needs.</p>
<p>The InspectDR library is implemented using <strong>3 distinct plot layers</strong>:</p>
<ul>
<li><strong>Plot image layer:</strong> Implemented with the <a href="https://cairographics.org/" rel="nofollow">Cairo library</a>, the plot image layer allows the user to render (multi-) plots as simple images.</li>
<li><strong>Plot widget layer:</strong> Library users can also integrate plots to their own <a href="https://www.gtk.org/" rel="nofollow">GTK+</a> application by instantiating a single InspectDR widget.</li>
<li><strong>Plot application layer:</strong> Most end users will likely display/interact with plots/data using the built-in Julia/<a href="https://www.gtk.org/" rel="nofollow">GTK+</a> multi-plot application.</li>
</ul>
<p>Users are encouraged to open an issue if it is unclear how to utilize a particular layer.  Documentation is a bit limited at the moment.</p>

<p>The following highlights a few interesting features of InspectDR:</p>
<ul>
<li>Publication-quality output.</li>
<li>Included as a "backend" of <a href="https://github.com/JuliaPlots/Plots.jl">JuliaPlots/Plots.jl</a>.</li>
<li>Relatively short load times / time to first plot.</li>
<li>Designed with larger datasets in mind:
<ul>
<li>Responsive even with moderate (&gt;200k points) datasets.</li>
<li>Confirmed to handle 2GB datsets with reasonable speed on older desktop running Windows 7 (drag+pan of data area highly discouraged).</li>
</ul>
</li>
<li>Support for Smith charts (admittance &amp; impedance - see <a href="#Templates_Scales">Plot Templates</a>).</li>
<li>Support for various types of annotation:
<ul>
<li>User-programmable text, polyline, vertical &amp; horizontal bars.</li>
<li>Drag &amp; drop Δ-markers (Measures/displays Δx, Δy &amp; slope).</li>
</ul>
</li>
<li>Interactive <a href="#Bindings">mouse/keybindings</a>.
<ul>
<li>Fast &amp; simple way to pan/zoom into data.</li>
<li>In line with other similar tools.</li>
<li>Create drag &amp; drop Δ-markers.</li>
</ul>
</li>
<li><a href="#Layout_Stylesheets">Layout &amp; Stylesheets</a>.
<ul>
<li>See <a href="sample/demo12.jl">demo targeting IEEE publications @300 dpi</a></li>
<li>Add custom stylesheets.</li>
</ul>
</li>
</ul>
<p>See following subsections for more information.</p>

<h4><a id="user-content-f1-acceleration" class="anchor" aria-hidden="true" href="#f1-acceleration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>"F1" Acceleration</h4>
<p>InspectDR.jl includes specialized algorithms to accellerate plotting of large "F1" datasets (functions of 1 argument) in order to maintain a good "real-time" (interactive) user experience.</p>
<p>A dataset is defined as a function of 1 argument ("F1") if it satisfies:</p>
<pre><code>y = f(x), where x: sorted, real vector
</code></pre>
<p>Examples of "F1" datasets include <strong>time domain</strong> (<code>y(x=time)</code>) and <strong>frequncy domain</strong> (<code>X(w)</code>) data.</p>
<p>"F1" acceleration is obtained by dropping points in order to speed up the rendering process.</p>
<p><em><strong>IMPORTANT:</strong></em> "F1" acceleration tends to generate erroneous-looking plots whenever glyphs are displayed.  This is because the dropped points may become very noticeable.  Consequently, InspectDR will, by default, only apply "F1" acceleration on datasets drawn without glyphs (lines only).</p>
<p>To change when InspectDR applies "F1" acceleration to drop points, look for the <code>:droppoints</code> entry in the <a href="#Config_Defaults">Configuration/Defaults</a> section.</p>
<h4><a id="user-content-2d-plot-support" class="anchor" aria-hidden="true" href="#2d-plot-support"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>2D Plot Support</h4>
<p>InspectDR.jl also supports generic 2D plotting.  More specifically, the tool is capable of plotting arbitrary 2D datasets that satisfy:</p>
<pre><code>(x,y) = (u[i], v[i]), for i in [1...N]
</code></pre>
<p>Examples of of such plots (where x-values are not guaranteed to be sorted) include:</p>
<ul>
<li>Nyquist plots</li>
<li>Lissajous plots</li>
<li>Smith/polar (S-Parameter) charts</li>
</ul>

<h2><a id="user-content-select-documentation" class="anchor" aria-hidden="true" href="#select-documentation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Select Documentation</h2>
<p>Note that many types &amp; functions are not exported from InspectDR in order to avoid namespace pollution.  That being said, the module name will often be omitted below in an attempt to improve readability.</p>
<h3><a id="user-content-plot-objects" class="anchor" aria-hidden="true" href="#plot-objects"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Plot Objects</h3>
<p>Principal objects:</p>
<ul>
<li><strong><code>InspectDR.Plot</code></strong>: An abstract plot object.</li>
<li><strong><code>InspectDR.Plot2D &lt;: Plot</code></strong>:  A 2D plot object.  Construct empty 2D plot using <code>InspectDR.Plot2D(title="Plot Title")</code>.</li>
<li><strong><code>InspectDR.Multiplot</code></strong>:  A multi-plot object.  Construct empty multi-plot using: <code>InspectDR.Multiplot(title="Multiplot Title")</code>.</li>
</ul>
<p>Subplots (<code>T&lt;:Plot</code>) are added to a multi-plot object using the <code>add()</code> method:</p>
<pre><code>mplot = InspectDR.Multiplot()
plot1 = InspectDR.Plot2D()
plot2 = InspectDR.Plot2D()

add(mplot, plot1)
add(mplot, plot2)
</code></pre>
<p>Similarly, the <code>add()</code> method adds waveforms to plots/subplots:</p>
<pre><code>wfrm = add(plot1, x, y, id="Waveform label")
</code></pre>
<p><strong>WARNING:</strong> Only <code>Vector</code> data can be added (<code>AbstractVector</code>/<code>Range</code> not currently supported).</p>
<h4><a id="user-content-displaying-plots" class="anchor" aria-hidden="true" href="#displaying-plots"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Displaying Plots</h4>
<p>InspectDR provides the <code>GtkDisplay</code> object derived from <code>Base.Multimedia.Display</code>.  <code>GtkDisplay</code> is used in combination with <code>Base.display()</code>, to spawn a new instance of a GTK-based GUI.</p>
<p>To display a single <code>plot::Plot</code> object, one simply calls:</p>
<pre><code>gplot = display(InspectDR.GtkDisplay(), plot)
</code></pre>
<p>Similarly, to display <code>mplot::Multiplot</code> object, one calls:</p>
<pre><code>gplot = display(InspectDR.GtkDisplay(), mplot)
</code></pre>
<p>To programmatically close a Gtk plot window, the <code>close()</code> function can be used:</p>
<pre><code>close(gplot)
</code></pre>
<p><a name="user-content-templates_scales"></a></p>
<h3><a id="user-content-plot-templatesaxis-scales" class="anchor" aria-hidden="true" href="#plot-templatesaxis-scales"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Plot Templates/Axis Scales</h3>
<p>In order to support stacked graphs with independent y-axes (tied to the same x-axis), specifying axis scales is a bit involved:</p>
<ul>
<li><code>InspectDR.Plot2D.xscale</code> controls the x-axis scale.</li>
<li><code>InspectDR.Plot2D.strips[STRIP_INDEX].yscale</code> controls the y-axis scale.</li>
</ul>
<p>To streamline control over plot axes/grids/labels/..., it is highly recommended to use the following <strong>plot templates</strong>:</p>
<ol>
<li>
<p><code>Plot2D(xscale, yscalelist; kwargs...)</code>: Generic 2D plot template.</p>
<ul>
<li><code>Plot2D(:lin, :log, title="title", xlabel="X", ylabels=["log(Y)"])</code>: Construct plot with a linear X-axis &amp; log10 Y-axis.</li>
<li><code>Plot2D(:log10, [:dB20, :lin, :lin], title="title", xlabel="X", ylabels=["Y1 (dB)", "Y2", "Y3"])</code>: Construct plot with a log10 X-axis, and 2 Y-strips: the top-most with a dB20 Y-scale, and the next two with linear Y-scale.</li>
</ul>
</li>
<li>
<p><code>bodeplot(; kwargs...)</code>: Template for generating Bode plots.</p>
<ul>
<li><code>bodeplot()</code>: Default <code>kwargs</code> already set: <code>xlabel="Frequency (Hz)"</code>, ylabels=["Magnitude (dB)", "Phase (°)]"`.</li>
</ul>
</li>
<li>
<p><code>transientplot(yscalelist; kwargs...)</code>: Template for plotting transient data.</p>
<ul>
<li><code>transientplot([:lin, :lin, :lin], title="title", ylabels=["Voltage", "Voltage", "Current"])</code>: <code>xlabel</code> already set to <code>"Time (s)"</code>, by default.</li>
</ul>
</li>
<li>
<p><code>smithchart(TYPE; ref, kwargs...)</code>: Template for plotting onto a Smith Chart.</p>
<ul>
<li><code>smithchart(:Z, ref=50)</code>: Impedance (<code>Z</code>) Smith Chart with a 50Ω reference. Default <code>kwargs</code> already set: <code>xlabel="Real(Γ)"</code>, <code>ylabels=["Imaginary(Γ)"]"</code>.</li>
<li><code>smithchart(:Y, ref=75)</code>: Admittance (<code>Y</code>) Smith Chart with a 75Ω reference.</li>
</ul>
</li>
</ol>
<p>NOTE: X/Y-axis scales are specified using one of the following <code>::Symbols</code>:</p>
<ul>
<li><code>:lin</code></li>
<li><code>:log10</code>, <code>:log</code> (= <code>:log10</code>)</li>
<li><code>:ln</code>, <code>:log2</code>: Grid lines might need improvement here.</li>
<li><code>:dB20</code>, <code>:dB10</code></li>
</ul>
<p><a name="user-content-layout_stylesheets"></a></p>
<h3><a id="user-content-layout--stylesheets" class="anchor" aria-hidden="true" href="#layout--stylesheets"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Layout &amp; Stylesheets</h3>
<p>The appearance of InspectDR plots is configured through the <code>.layout</code> properties of <code>::Plot2D</code> &amp; <code>::Multiplot</code> objects.  Until better documentation is available, one is encouraged to look at the fields of the <code>PlotLayout</code> &amp; <code>MultiplotLayout</code> for more information:</p>
<pre><code>#To control apperance of Plot2D elements:
?InspectDR.PlotLayout

#To control apperance of Multiplot elements:
?InspectDR.MultiplotLayout
</code></pre>
<p>The <code>.layout</code> properties should be accessed using the <code>[]</code> operators, using the field names names of <code>PlotLayout</code> / <code>MultiplotLayout</code> as arguments:</p>
<pre><code>plot.layout[:valloc_top] = 20 #Modify space above data area
</code></pre>
<h4><a id="user-content-pre-defined-stylesheets" class="anchor" aria-hidden="true" href="#pre-defined-stylesheets"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Pre-defined Stylesheets</h4>
<p>InspectDR uses "Stylesheets" to control the default values of plot elements.  To apply a different stylesheet to a given plot, use the <code>setstyle!</code> methods:</p>
<pre><code>InspectDR.setstyle!(::Plot2D, stylesheet::Symbol; kwargs...)
InspectDR.setstyle!(::Multiplot, stylesheet::Symbol; kwargs...)
</code></pre>
<p>Currently supported values for <code>stylesheet</code> include:</p>
<ul>
<li><code>:screen</code></li>
<li><code>:IEEE</code></li>
</ul>
<p>Custom stylesheets are added by extending <code>InspectDR.getstyle()</code>, as done in <a href="src/stylesheets.jl">stylesheets.jl</a> (Search for: <code>StyleID{:screen}</code> &amp; <code>StyleID{:IEEE}</code>).</p>
<h4><a id="user-content-legends" class="anchor" aria-hidden="true" href="#legends"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Legends</h4>
<p>At this point in time, legends have limited configurability.  When displayed, legends will consume fixed horizontal real-estate.  The idea is to display a large number of labels without hiding the data area.</p>
<p>In order to display the legend of a <code>plot::Plot2D</code> object, one would set:</p>
<pre><code>plot.layout[:enable_legend] = true #Enables legend
plot.layout[:halloc_legend] = 150 #Control size of legend
</code></pre>
<p><a name="user-content-config_defaults"></a></p>
<h3><a id="user-content-configurationdefaults" class="anchor" aria-hidden="true" href="#configurationdefaults"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuration/Defaults</h3>
<p>Default InspectDR.jl settings can be overwritten once the module is loaded by editing the <code>InspectDR.defaults</code> structure:</p>
<pre><code>#Dissalow SVG MIME output for performance reasons:
InspectDR.defaults.rendersvg = false

#Change when plots drop points to enable "F1"-acceleration:
#(One of: {:always, :never, :noglyph, :hasline})
InspectDR.defaults.pointdropmatrix = InspectDR.PDM_DEFAULTS[:always]

#Enable time stamp &amp; legend:
InspectDR.defaults.plotlayout[:enable_timestamp] = true
InspectDR.defaults.plotlayout[:enable_legend] = true

#Set data-area dimensions (saving single plot):
InspectDR.defaults.plotlayout[:halloc_data] = 500
InspectDR.defaults.plotlayout[:valloc_data] = 350

#Set plot dimensions (saving multi-plot):
InspectDR.defaults.mplotlayout[:halloc_plot] = 500
InspectDR.defaults.mplotlayout[:valloc_plot] = 350

#Configure # of columns in multi-plot outputs:
InspectDR.defaults.mplotlayout[:ncolumns] = 2
</code></pre>
<p>Until better documentation is available, one is encouraged to look at the fields of the <code>PlotLayout</code> for more information:</p>
<pre><code>?InspectDR.PlotLayout
</code></pre>
<p>Defaults can also be specified <em>before</em> importing InspectDR.jl with the help of <code>Main.DEFAULTS_INSPECTDR::Dict</code>.  Simply create the variable in your <code>~/.juliarc.jl</code> file, using the following pattern:</p>
<pre><code>DEFAULTS_INSPECTDR = Dict(
	:rendersvg =&gt; false,

	#Special options available @ initialization:
	:droppoints =&gt; :always, #One of: {:always, :never, :noglyph, :hasline}
	:notation_x =&gt; :SI,   #Change x-axis notation
	:notation_y =&gt; :SI,   #Change y-axis notation
	:fontname =&gt; "Sans",  #Change default font family
	:fontscale =&gt; 1.2,    #Scale up/down font default sizes

	#Basic plot options:
	:enable_timestamp =&gt; true,
	:enable_legend =&gt; true,
	:halloc_legend =&gt; 150,

	#Supported multiplot options:
	:ncolumns =&gt; 2,
	:halloc_plot =&gt; 500,
	:valloc_plot =&gt; 350,
)
</code></pre>

<h2><a id="user-content-known-limitations" class="anchor" aria-hidden="true" href="#known-limitations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Known Limitations</h2>
<ul>
<li>Stability of <a href="notebook/">IJulia (Jupyter) notebooks</a> is not very good at the moment.  Also: many examples are not yet ported to Julia 1.0.</li>
<li>Documentation is a bit limited at the moment.  See <a href="#SampleUsage">Sample Usage</a> to learn from examples.</li>
<li>API is still a bit rough.  User often has to manipulate data structures directly.
<ul>
<li>Workaround: Use <a href="https://github.com/JuliaPlots/Plots.jl">JuliaPlots/Plots.jl</a> as a "frontend" (increases plot times).</li>
</ul>
</li>
<li>Font control is not ideal.  The default font might not be available on all platforms - and the fallback font might not have Unicode characters to display exponent values (ex: <code>10⁻¹⁵</code>).  Some Greek characters might also be missing.
<ul>
<li>Workaround: Overwrite default font, as described in <a href="#Config_Defaults">Configuration/Defaults</a>.</li>
</ul>
</li>
<li>Legends not very configurable (currently optimized to display many labels @ cost of horizontal real-estate).</li>
<li>Does not yet render plot data in separate thread (will improve interactive experience with large datasets).</li>
<li>Mouse events currently function even outside data area (a bit odd).</li>
<li>Significant slowdowns observed when zooming <strong>deep</strong> into non-F1 data... Can likely be solved by discarding data outside plot extents.
<ul>
<li>Workaround: make sure x-values are sorted (F1-acceleration discards data &amp; is less prone to slowdowns).</li>
</ul>
</li>
<li>By default, "F1"-acceleration is only applied to datasets drawn without glyphs (lines only).
<ul>
<li>Look for the <code>:droppoints</code> entry in the <a href="#Config_Defaults">Configuration/Defaults</a> section to change this behaviour.</li>
</ul>
</li>
</ul>

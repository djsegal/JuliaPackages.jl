<p dir="auto">CfGRIB.jl is a julia interface to map GRIB files to the <a href="https://www.unidata.ucar.edu/software/thredds/current/netcdf-java/CDM/" rel="nofollow">Unidata's Common Data
Model v4</a>
following the <a href="http://cfconventions.org" rel="nofollow">CF Conventions</a>.</p>
<p dir="auto">This package is based on the python implementation in
<a href="https://github.com/ecmwf/cfgrib">cfgrib.py</a> and closely follows the approaches
of that package, but in Julia instead. Parity tests are automatically performed
between the two to ensure that the data returned by the Julia version is
equivalent to that from Python.</p>
<p dir="auto">In Python <a href="http://xarray.pydata.org" rel="nofollow">xarray</a> has come out to be a standard
way to implement named arrays, however as Julia is a much younger language no
stable array interface has been adopted by the community yet, so the approach
here is more flexible and allows for multiple array backends to be used.</p>
<p dir="auto">Currently two array backends are supported:</p>
<ul dir="auto">
<li><a href="https://github.com/JuliaArrays/AxisArrays.jl"><code>AxisArrays</code></a></li>
<li><a href="https://github.com/rafaqz/DimensionalData.jl"><code>DimensionalData</code></a></li>
</ul>
<p dir="auto">If a backend is found to be installed then its functionality will automatically
be enabled, otherwise only the built-in bare data types will be returned.</p>
<p dir="auto">Low level access and decoding is performed by calling
<a href="https://github.com/weech/GRIB.jl">GRIB.jl</a> which itself calls the
<a href="https://software.ecmwf.int/wiki/display/ECC/" rel="nofollow">ECMWF ecCodes library</a>.</p>

<p dir="auto">The package is currently under heavy development so it has not been added to the
Julia package registry yet. To install the package first clone this repository:</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="git clone https://github.com/ecmwf/cfgrib.jl/
cd CfGRIB.jl"><pre>git clone https://github.com/ecmwf/cfgrib.jl/
<span class="pl-c1">cd</span> CfGRIB.jl</pre></div>
<p dir="auto">Then start Julia, enter the
<a href="https://docs.julialang.org/en/v1/stdlib/Pkg/" rel="nofollow">pkg mode</a>, activate the
<a href="https://julialang.github.io/Pkg.jl/stable/environments/" rel="nofollow">project</a> (projects are
similar to python <code>venv</code>'s), install the GRIB.jl package (as it is also not on
the registry), and then finally you can instantiate CfGRIB.jl to get the rest of
the dependencies:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="#  Activate the current directory as a project
activate .
#  To enable backend support
add AxisArrays
add DimensionalData
instantiate"><pre><span class="pl-c"><span class="pl-c">#</span>  Activate the current directory as a project</span>
activate .
<span class="pl-c"><span class="pl-c">#</span>  To enable backend support</span>
add AxisArrays
add DimensionalData
instantiate</pre></div>
<p dir="auto">Finally exit pkg mode by pressing backspace, and use the package as usual:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using CfGRIB"><pre><span class="pl-k">using</span> CfGRIB</pre></div>

<p dir="auto">To install the package for development you can run:</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="git clone https://github.com/ecmwf/cfgrib.jl/
cd CfGRIB.jl"><pre>git clone https://github.com/ecmwf/cfgrib.jl/
<span class="pl-c1">cd</span> CfGRIB.jl</pre></div>
<p dir="auto">Then in Julia:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="] activate .
] develop ."><pre>] activate .
] develop .</pre></div>
<p dir="auto">Will install the package as a development package. When you run <code>] test</code> the
tests will run locally. If you want to run the tests within a container similar
to the ones used when the CI runs via GitHub, then install
<a href="https://github.com/nektos/act">nektos/act</a> and run the command
<code>act -P ubuntu-latest=nektos/act-environments-ubuntu:18.04 -j tests</code>, this will
set up a docker container and run the full test suite within it.</p>

<p dir="auto">This implementation of Glob is based on the IEEE Std 1003.1, 2004 Edition (Open Group Base Specifications Issue 6) for fnmatch and glob. The specification of which can be found online: <a href="http://pubs.opengroup.org/onlinepubs/009696899/functions/fnmatch.html" rel="nofollow">fnmatch</a> and <a href="http://pubs.opengroup.org/onlinepubs/009696899/functions/glob.html" rel="nofollow">glob</a>.</p>
<blockquote>
<p dir="auto">Note, because this is based on the POSIX specification, the path separator in a glob pattern is always <code>/</code> and the escape character is always <code>\</code>. However, the returned path string will always contain the system path separator character <code>Base.path_separator</code>. Therefore, it may be true that a path returned by <code>glob</code> will fail to match a <code>Glob.FilenameMatch</code> constructed from the same pattern.</p>
</blockquote>

<p dir="auto">Glob is implemented to have both a functional form and an object-oriented form. There is no "correct" choice; you are encouraged to pick whichever is better suited to your application.</p>
<ul dir="auto">
<li>
<p dir="auto"><code>glob(pattern, [directory::AbstractString])</code> ::</p>
<ul dir="auto">
<li>Returns a list of all files matching <code>pattern</code> in <code>directory</code>.</li>
<li>If directory is not specified, it defaults to the current working directory.</li>
<li>Pattern can be any of:
<ol dir="auto">
<li>
<p dir="auto">A <code>Glob.GlobMatch</code> object:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content=" glob&quot;a/?/c&quot;"><pre class="notranslate"><code> glob"a/?/c"
</code></pre></div>
</li>
<li>
<p dir="auto">A string, which will be converted into a GlobMatch expression:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content=" &quot;a/?/c&quot; # equivalent to 1, above"><pre class="notranslate"><code> "a/?/c" # equivalent to 1, above
</code></pre></div>
</li>
<li>
<p dir="auto">A vector of strings and/or objects which implement <code>occursin</code>, including <code>Regex</code> and <code>Glob.FilenameMatch</code> objects</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content=" [&quot;a&quot;, r&quot;.&quot;, fn&quot;c&quot;] # again, equivalent to 1, above"><pre class="notranslate"><code> ["a", r".", fn"c"] # again, equivalent to 1, above
</code></pre></div>
<ul dir="auto">
<li>Each element of the vector will be used to match another level in the file hierarchy</li>
<li>no conversion of strings to <code>Glob.FilenameMatch</code> objects or directory splitting on <code>/</code> will occur.</li>
</ul>
</li>
<li>
<p dir="auto">A trailing <code>/</code> (or equivalently, a trailing empty string in the vector) will cause glob to only match directories</p>
</li>
<li>
<p dir="auto">Attempting to creat a GlobMatch object from a string with a leading <code>/</code> or the empty string is an error</p>
</li>
</ol>
</li>
</ul>
</li>
<li>
<p dir="auto"><code>readdir(pattern::GlobMatch, [directory::AbstractString])</code> ::</p>
<ul dir="auto">
<li>alias for <code>glob()</code></li>
</ul>
</li>
<li>
<p dir="auto"><code>glob"pattern"</code> ::</p>
<ul dir="auto">
<li>Returns a <code>Glob.GlobMatch</code> object, which can be used with <code>glob()</code> or <code>readdir()</code>. See above descriptions.</li>
</ul>
</li>
<li>
<p dir="auto"><code>fn"pattern"ipedx</code> ::</p>
<ul dir="auto">
<li>Returns a <code>Glob.FilenameMatch</code> object, which can be used with <code>ismatch()</code> or <code>occursin()</code>. Available flags are:
<ul dir="auto">
<li><code>i</code> = <code>CASELESS</code> : Performs case-insensitive matching</li>
<li><code>p</code> = <code>PERIOD</code> : A leading period (<code>.</code>) character must be exactly matched by a period (<code>.</code>) character (not a <code>?</code>, <code>*</code>, or <code>[]</code>). A leading period is a period at the beginning of a string, or a period after a slash if PATHNAME is true.</li>
<li><code>e</code> = <code>NOESCAPE</code> : Do not treat backslash (<code>\</code>) as a special character (in extended mode, this only outside of <code>[]</code>)</li>
<li><code>d</code> = <code>PATHNAME</code> : A slash (<code>/</code>) character must be exactly matched by a slash (<code>/</code>) character (not a <code>?</code>, <code>*</code>, or <code>[]</code>)</li>
<li><code>x</code> = <code>EXTENDED</code> : Additional features borrowed from newer shells, such as <code>bash</code> and <code>tcsh</code>
<ul dir="auto">
<li>Backslash (<code>\</code>) characters in <code>[]</code> groups escape the next character</li>
</ul>
</li>
</ul>
</li>
</ul>
</li>
</ul>

<ul dir="auto">
<li><code>[.</code> collating symbols only accept single characters (the Unicode locale has no collating symbols defined)</li>
<li><code>[=</code> equivalence classes only match the exact character specified (the Unicode locale has no equivalence classes defined)</li>
<li>Advanced extended features (beyond the POSIX spec) such as <code>{}</code> groups, have not yet been implemented</li>
</ul>

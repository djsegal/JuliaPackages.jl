<p dir="auto">This package defines a type, <code>FrankenTuple</code>, which is like a cross between a <code>Tuple</code> and a
<code>NamedTuple</code>; it contains both positional and named elements.</p>
<blockquote>
<p dir="auto"><em>Accursed creator! Why did you form a monster so hideous that even you turned from me in disgust?</em></p>
</blockquote>
<p dir="auto">A function call has the form <code>f(args...; kwargs...)</code>.
Take away the function, and you get <code>(args...; kwargs...)</code>, a tuple with both positional
and named elements.
No one Base type currently models this, so <code>FrankenTuple</code> was created as an experiment to
see if and when this precise structure could be useful.</p>

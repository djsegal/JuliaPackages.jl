<p dir="auto">This is a prototype reactive web app for bitemporal data management based on <a href="https://github.com/actuarial-sciences-for-africa-asa/BitemporalPostgres.jl">a Julia bitemporal data management API</a> and a UI based on <a href="https://github.com/GenieFramework/StippleUI.jl">Stipple</a> and <a href="https://quasar.dev/" rel="nofollow">QUASAR (where Stipple did not yet provide a solution, or I didn't find one)</a>.</p>
<h3 dir="auto"><a id="user-content-architecture-is-mvvm" class="anchor" aria-hidden="true" href="#architecture-is-mvvm"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Architecture is <a href="https://012.vuejs.org/guide/" rel="nofollow">MVVM</a></h3>
<ul dir="auto">
<li>BitemporalReactive connects the business model to the view model and provides event handling.</li>
<li>ContractSectionView defines the view model, the UI elements, and their data bindings.</li>
<li><a href="https://github.com/Actuarial-Sciences-for-Africa-ASA/LifeInsuranceDataModel.jl">LifeInsuranceDataModel</a> - an imported package - provides the business model and logic based on a framework
for <a href="https://github.com/Actuarial-Sciences-for-Africa-ASA/BitemporalPostgres.jl">bitemporal data management</a></li>
<li><a href="https://github.com/Actuarial-Sciences-for-Africa-ASA/LifeInsuranceProduct.jl">LifeInsuranceProduct</a> - an imported package - provides actuarial computations (mostly ony a placeholder by now)</li>
</ul>

<p dir="auto">The app creates or mutates bitemporal entities as of a point in reference time.
A bitemporal transaction - here also called workflow - is started in two ways:</p>
<h3 dir="auto"><a id="user-content-creation-of-entities-initially-produces" class="anchor" aria-hidden="true" href="#creation-of-entities-initially-produces"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>creation of entities initially produces</h3>
<ul dir="auto">
<li>an instance of history - the root of bitemporal entities -</li>
<li>a version</li>
<li>a validity_interval</li>
</ul>
<h3 dir="auto"><a id="user-content-update-of-entities-initially-creates" class="anchor" aria-hidden="true" href="#update-of-entities-initially-creates"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>update of entities initially creates</h3>
<ul dir="auto">
<li>a version</li>
<li>a valididity_interval
and references</li>
<li>a given history</li>
</ul>
<p dir="auto">While these data comprise the scaffolding needed for bitemporal tracking of mutations, components and revisions comprise the business payload, as well as subcomponents and their revisions. Thus</p>
<h3 dir="auto"><a id="user-content-mutation-of-business-data-consists-of" class="anchor" aria-hidden="true" href="#mutation-of-business-data-consists-of"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>mutation of Business data consists of</h3>
<p dir="auto">-creation or invalidation of components/subcomponents together with their revisions and of</p>
<ul dir="auto">
<li>updates of attributes of revisions</li>
</ul>

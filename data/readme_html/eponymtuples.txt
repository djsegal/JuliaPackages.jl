<p dir="auto">Julia package for deconstructing dispatch on <code>NamedTuple</code>s.</p>
<p dir="auto">Uses the variable names <em>both</em> for the <code>NamedTuple</code> and deconstruction.</p>
<p dir="auto">Allows replacing</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="f((a, b)::NamedTuple{(:a, :b), &lt;: Tuple{Any, Int}}) = ...

(a = a, b = b, c = 3)"><pre><span class="pl-en">f</span>((a, b)<span class="pl-k">::</span><span class="pl-c1">NamedTuple{(:a, :b), &lt;: Tuple{Any, Int}}</span>) <span class="pl-k">=</span> <span class="pl-k">...</span>

(a <span class="pl-k">=</span> a, b <span class="pl-k">=</span> b, c <span class="pl-k">=</span> <span class="pl-c1">3</span>)</pre></div>
<p dir="auto">with</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="f(@eponymargs(a, b::Int)) = ...

@eponymtuple(a, b, c = 3)"><pre><span class="pl-en">f</span>(<span class="pl-c1">@eponymargs</span>(a, b<span class="pl-k">::</span><span class="pl-c1">Int</span>)) <span class="pl-k">=</span> <span class="pl-k">...</span>

<span class="pl-c1">@eponymtuple</span>(a, b, c <span class="pl-k">=</span> <span class="pl-c1">3</span>)</pre></div>
<p dir="auto">It is pretty lightweight: <code>@eponymargs</code> and <code>@eponymtuple</code> are the only symbols exported; and the package has no dependencies.</p>

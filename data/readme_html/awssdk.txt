<div id="readme" class="md" data-path="README.md"><article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-awssdkjl" class="anchor" aria-hidden="true" href="#awssdkjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>AWSSDK.jl</h1>
<p>Julia interface for <a href="https://aws.amazon.com" rel="nofollow">Amazon Web Services</a>.</p>
<p>Based on <a href="https://github.com/JuliaCloud/AWSCore.jl">JuliaCloud/AWSCore.jl</a>.</p>
<p>This package provides automatically generated low-level API wrappers and
documentation strings for each operation in each Amazon Web Service.</p>
<p>The following high-level packages are also available:
<a href="http://github.com/samoconnor/AWSS3.jl">AWS S3</a>,
<a href="http://github.com/samoconnor/AWSSQS.jl">AWS SQS</a>,
<a href="http://github.com/samoconnor/AWSSNS.jl">AWS SNS</a>,
<a href="http://github.com/samoconnor/AWSIAM.jl">AWS IAM</a>,
<a href="http://github.com/samoconnor/AWSEC2.jl">AWS EC2</a>,
<a href="http://github.com/samoconnor/AWSLambda.jl">AWS Lambda</a>,
<a href="http://github.com/samoconnor/AWSSES.jl">AWS SES</a> and
<a href="http://github.com/samoconnor/AWSSDB.jl">AWS SDB</a>.
These packages include operation specific result structure parsing, error
handling, type convenience functions, iterators, etc.</p>
<p>Full documentation <a href="https://juliacloud.github.io/AWSCore.jl/build/index.html" rel="nofollow">is available here</a>,
or see below for some examples of how to get started.</p>
<p><em>This package is generated by <a href="https://github.com/JuliaCloud/AWSCore.jl/blob/master/src/AWSAPI.jl">AWSCore.jl/src/AWSAPI.jl</a>.</em></p>
<h2><a id="user-content-configuration" class="anchor" aria-hidden="true" href="#configuration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuration</h2>
<p>Option 1: <a href="http://docs.aws.amazon.com/cli/latest/userguide/cli-environment.html" rel="nofollow">environment variables</a>:
<code>AWS_ACCESS_KEY_ID</code>, <code>AWS_SECRET_ACCESS_KEY</code> (and <code>AWS_DEFAULT_REGION</code>),</p>
<p>Option 2: <a href="http://docs.aws.amazon.com/cli/latest/userguide/cli-config-files.html" rel="nofollow"><code>~/.aws/credentials</code></a> file:</p>
<pre><code>[default]
aws_access_key_id = AKIAXXXXXXXXXXXXXXXX
aws_secret_access_key = XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
</code></pre>
<p>Option 3: run the <a href="http://docs.aws.amazon.com/cli/latest/userguide/cli-chap-getting-started.html#cli-quick-configuration" rel="nofollow">AWS CLI</a>
configure command: <code>aws configure</code>.</p>

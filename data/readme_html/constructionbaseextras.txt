<p dir="auto">ConstructionBaseExtras.jl provides additional extensions to ConstructionBase.jl,
that allows modifying immutable objects from other packages in Setfield.jl, Accessors.jl, BangBang.jl,
and Flatten.jl. It's a stop-gap measure until <a href="https://github.com/JuliaLang/Pkg.jl/issues/1285" data-hovercard-type="issue" data-hovercard-url="/JuliaLang/Pkg.jl/issues/1285/hovercard">first-class conditional modules</a>
are available, or packages decide to implement the necessary methods themselves.</p>
<p dir="auto">Note: use of Requires.jl means that dependency compat bounds cannot be set.
However, these types are not likely to change frequently. But manually fixing
package versions in a Manifest.toml file is recommended for reproducibility.</p>
<p dir="auto">This initially includes StaticArrays.jl, but will be expanded as more types
are found to fail with <code>constructorof</code>, <code>setproperties</code> or <code>getproperties</code>.</p>
<p dir="auto">Please make an issue for any objects in other packages that cause errors in
these methods.</p>

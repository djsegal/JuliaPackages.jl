<p dir="auto">BrainFlow is a library intended to obtain, parse and analyze EEG, EMG, ECG, and other kinds of data from biosensors.</p>
<p dir="auto">It provides a uniform SDK to work with biosensors with a primary focus on neurointerfaces, all features available for free and distributed under MIT license.</p>
<h4 dir="auto"><a id="user-content-advantages-of-brainflow" class="anchor" aria-hidden="true" href="#advantages-of-brainflow"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Advantages of BrainFlow:</h4>
<ul dir="auto">
<li>powerful API with many features to simplify development
<ul dir="auto">
<li>Straightforward API for data acquisition</li>
<li>Powerful API for signal filtering, denoising, downsampling...</li>
<li>Development tools like Synthetic board, Streaming board, logging API</li>
</ul>
</li>
<li>easy to use
<ul dir="auto">
<li>BrainFlow has many bindings, you can choose programming language you like</li>
<li>All programming languages provide the same API, so it's simple to switch</li>
<li>API is uniform for all boards, it makes applications on top of BrainFlow almost board agnostic</li>
</ul>
</li>
<li>easy to support and extend
<ul dir="auto">
<li>Code to read data and to perform signal processing is implemented only once in C/C++, bindings just call C/C++ methods</li>
<li>Powerful CI/CD system which runs integrations tests for each commit automatically using BrainFlow's Emulator</li>
<li>Simplified process to add new boards and methods</li>
</ul>
</li>
</ul>

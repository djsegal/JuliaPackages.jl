### Piano

<p dir="auto">Since MusicXML's types are not exported from the package to avoid conflicts with the similarly named types from other libraries (such as <code>Dates.Time</code>, <code>MIDI.Note</code>). There are 3 different solutions:</p>
<ol dir="auto">
<li><code>import</code> the types yourself if you are sure that there is no conflict. Use <code>@importMX</code> to <code>import</code> all the types.</li>
<li>or use <code>MX.</code> before the type names (which <code>MX</code> is an alias for <code>MusicXML</code>)</li>
<li>or use <code>@MX</code> macro, which adds <code>MX.</code> automatically to the type names. See <code>@MX</code> docstring for examples.</li>
</ol>

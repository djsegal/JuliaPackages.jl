<p dir="auto">Julia package for students in Decision Making under Uncertainty. This package contains code needed for homeworks and submission to gradescope.</p>
<p dir="auto">The code will be updated over the course of the semester, so <strong>it is extemely important to <a href="#Updating">update the code</a> before starting a homework assignment</strong> to ensure that you have the latest version.</p>
<p dir="auto">Examining and using any of the code in this package is <strong>encouraged</strong>, except for obfuscated code. Obfuscated code is hidden by default and will look like files full of numbers; <em>it will not be easy to accidentally see</em>. However a <em>deliberate</em> attempt to de-obfuscate this code or look inside it using another tool will be considered a violation of the Honor Code.</p>

<p dir="auto">Each homework module has an <code>evaluate</code> function that will be used for the challenge problems. The first argument is the submission required by the homework, and the second argument is the email that you used to register for gradescope. For example, on homework 1, you might call:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="f(x, y) = x+y
HW1.evaluate(f, &quot;your.email@colorado.edu&quot;)"><pre><span class="pl-en">f</span>(x, y) <span class="pl-k">=</span> x<span class="pl-k">+</span>y
HW1<span class="pl-k">.</span><span class="pl-c1">evaluate</span>(f, <span class="pl-s"><span class="pl-pds">"</span>your.email@colorado.edu<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">This will produce a <a href="https://en.wikipedia.org/wiki/JSON" rel="nofollow">json file</a> (<code>results.json</code> by default) that can be submitted to gradescope.</p>
<p dir="auto">For some homeworks, the <code>evaluate</code> function may have keyword arguments. These can be accessed through julia's built in help, i.e. with <code>?HW1.evaluate</code>.</p>

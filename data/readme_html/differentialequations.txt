<p dir="auto">This is a suite for numerically solving differential equations written in Julia
and available for use in Julia, Python, and R. The
purpose of this package is to supply efficient Julia implementations of solvers
for various differential equations. Equations within the realm of this package
include:</p>
<ul dir="auto">
<li>Discrete equations (function maps, discrete stochastic (Gillespie/Markov)
simulations)</li>
<li>Ordinary differential equations (ODEs)</li>
<li>Split and Partitioned ODEs (Symplectic integrators, IMEX Methods)</li>
<li>Stochastic ordinary differential equations (SODEs or SDEs)</li>
<li>Stochastic differential-algebraic equations (SDAEs)</li>
<li>Random differential equations (RODEs or RDEs)</li>
<li>Differential algebraic equations (DAEs)</li>
<li>Delay differential equations (DDEs)</li>
<li>Neutral, retarded, and algebraic delay differential equations (NDDEs, RDDEs, and DDAEs)</li>
<li>Stochastic delay differential equations (SDDEs)</li>
<li>Experimental support for stochastic neutral, retarded, and algebraic delay differential equations (SNDDEs, SRDDEs, and SDDAEs)</li>
<li>Mixed discrete and continuous equations (Hybrid Equations, Jump Diffusions)</li>
<li>(Stochastic) partial differential equations ((S)PDEs) (with both finite
difference and finite element methods)</li>
</ul>
<p dir="auto">The well-optimized DifferentialEquations solvers benchmark as some of the fastest
implementations of classic algorithms. It also includes algorithms from recent
research which routinely outperform the "standard" C/Fortran methods, and algorithms
optimized for high-precision and HPC applications. Simultaneously, it wraps
the classic C/Fortran methods, making it easy to switch over to them whenever
necessary. Solving differential equations with different methods from
different languages and packages can be done by changing one line of code,
allowing for easy benchmarking to ensure you are using the fastest method possible.</p>
<p dir="auto">DifferentialEquations.jl integrates with the Julia package sphere with:</p>
<ul dir="auto">
<li>GPU acceleration through <a href="https://cuda.juliagpu.org/stable/" rel="nofollow">CUDA.jl</a> and <a href="https://docs.sciml.ai/DiffEqGPU/dev/" rel="nofollow">DiffEqGPU.jl</a></li>
<li>Automated sparsity detection with <a href="https://github.com/JuliaSymbolics/Symbolics.jl">Symbolics.jl</a></li>
<li>Automatic Jacobian coloring with <a href="https://docs.sciml.ai/SparseDiffTools/stable/" rel="nofollow">SparseDiffTools.jl</a>, allowing for fast solutions
to problems with sparse or structured (Tridiagonal, Banded, BlockBanded, etc.) Jacobians</li>
<li>Allowing the specification of linear solvers for maximal efficiency with <a href="https://docs.sciml.ai/LinearSolve/stable/" rel="nofollow">LinearSolve.jl</a></li>
<li>Progress meter integration with the Visual Studio Code IDE for estimated time to solution</li>
<li>Automatic plotting of time series and phase plots</li>
<li>Built-in interpolations</li>
<li>Wraps for common C/Fortran methods like Sundials and Hairer's radau</li>
<li>Arbitrary precision with BigFloats and Arbfloats</li>
<li>Arbitrary array types, allowing the definition of differential equations on
matrices and distributed arrays</li>
<li>Unit checked arithmetic with Unitful</li>
</ul>
<p dir="auto">Additionally, DifferentialEquations.jl comes with built-in analysis features, including:</p>
<ul dir="auto">
<li><a href="https://docs.sciml.ai/SciMLSensitivity/stable/" rel="nofollow">Forward and Adjoint Sensitivity Analysis (Automatic Differentiation)</a> for fast gradient computations</li>
<li><a href="https://docs.sciml.ai/Overview/stable/highlevels/inverse_problems/" rel="nofollow">Parameter Estimation and Bayesian Analysis</a></li>
<li>Neural differential equations with <a href="https://docs.sciml.ai/DiffEqFlux/stable/" rel="nofollow">DiffEqFlux.jl</a>
for efficient scientific machine learning (scientific ML) and scientific AI.</li>
<li>Automatic distributed, multithreaded, and GPU <a href="https://diffeq.sciml.ai/dev/features/ensemble/" rel="nofollow">Parallel Ensemble Simulations</a></li>
<li><a href="https://docs.sciml.ai/GlobalSensitivity/stable/" rel="nofollow">Global Sensitivity Analysis</a></li>
<li><a href="https://docs.sciml.ai/Overview/stable/highlevels/uncertainty_quantification/" rel="nofollow">Uncertainty Quantification</a></li>
</ul>
<p dir="auto">This gives a powerful mixture of speed and productivity features to help you
solve and analyze your differential equations faster.</p>
<p dir="auto">For information on using the package,
<a href="https://diffeq.sciml.ai/stable/" rel="nofollow">see the stable documentation</a>. Use the
<a href="https://diffeq.sciml.ai/dev/" rel="nofollow">in-development documentation</a> for the version of
the documentation which contains the unreleased features.</p>
<p dir="auto">All of the algorithms are thoroughly tested to ensure accuracy via convergence
tests. The algorithms are continuously tested to show correctness.
IJulia tutorial notebooks
<a href="https://github.com/SciML/SciMLTutorials.jl">can be found at DiffEqTutorials.jl</a>.
Benchmarks
<a href="https://github.com/SciML/SciMLBenchmarks.jl">can be found at DiffEqBenchmarks.jl</a>.
If you find any equation where there seems to be an error, please open an issue.</p>

please feel free to chat in the <a href="https://gitter.im/JuliaDiffEq/Lobby?utm_source=badge&amp;utm_medium=badge&amp;utm_campaign=pr-badge&amp;utm_content=badge" rel="nofollow">Gitter channel</a>.
For bug reports, feature requests, etc., please submit an issue. If you're
interested in contributing, please see the

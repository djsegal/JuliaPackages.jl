<p dir="auto">Treat Jupyter notebooks as visual scripts which can be executed from the command line or from a script.
JupyterParameters creates and executes a new copy of the notebook with the parameters that have been passed and preserves the original.</p>
<p dir="auto">My main use case for JupyterParameters is for batch processes that I also want to generate inline sophiscated plots.
This essentially creates log files of my data analysis along with plots.
Running Jupyter notebooks from the command line is already possible using</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="jupyter nbconvert --to notebook --execute mynotebook.ipynb"><pre class="notranslate"><code>jupyter nbconvert --to notebook --execute mynotebook.ipynb
</code></pre></div>
<p dir="auto">The issue with using <code>nbconvert</code> in this fashion, is you <strong><em>can not pass arguments to the notebook</em></strong>.</p>
<p dir="auto">Using <code>jjnbparam</code> provided by JupyterParameters you are able to pass variables to a notebook.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using JupyterParameters
jjnbparam([&quot;notebook_orig.ipynb&quot;,&quot;notebook_new.ipynb&quot;,&quot;--varname1&quot;,&quot;varvalue1&quot;,&quot;--varname2&quot;,&quot;varvalue2&quot;,...]"><pre><span class="pl-k">using</span> JupyterParameters
<span class="pl-c1">jjnbparam</span>([<span class="pl-s"><span class="pl-pds">"</span>notebook_orig.ipynb<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>notebook_new.ipynb<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>--varname1<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>varvalue1<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>--varname2<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>varvalue2<span class="pl-pds">"</span></span>,<span class="pl-k">...</span>]</pre></div>
<h2 dir="auto"><a id="user-content-how-to-call-jjnbparam-from-the-shell" class="anchor" aria-hidden="true" href="#how-to-call-jjnbparam-from-the-shell"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>How to call jjnbparam from the shell</h2>
<p dir="auto">We can create an alias in <code>.bashrc</code> as</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="alias jjnbparam='julia -e &quot;using JupyterParameters; jjnbparam()&quot;'"><pre class="notranslate"><code>alias jjnbparam='julia -e "using JupyterParameters; jjnbparam()"'
</code></pre></div>
<p dir="auto">or add the following executable script (named <code>jjnbparam</code>) to your <code>PATH</code>.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia --color=yes -e '
try
    using JupyterParameters
catch
    import Pkg; Pkg.add(&quot;JupyterParameters&quot;)
    using JupyterParameters
end
jjnbparam()' &quot;$@&quot;"><pre class="notranslate"><code>julia --color=yes -e '
try
    using JupyterParameters
catch
    import Pkg; Pkg.add("JupyterParameters")
    using JupyterParameters
end
jjnbparam()' "$@"
</code></pre></div>
<p dir="auto">The command (from the shell) becomes</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="jjnbparam notebook_orig.ipynb notebook_new.ipynb --varname1 varvalue1 --varname2 varvalue2 ..."><pre class="notranslate"><code>jjnbparam notebook_orig.ipynb notebook_new.ipynb --varname1 varvalue1 --varname2 varvalue2 ...
</code></pre></div>
<p dir="auto">The command above <strong>creates and executes a new copy of the notebook with the parameters that have been passed and preserves the original</strong>.
If one wants to overwrite the original then</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="jjnbparam notebook.ipynb notebook.ipynb --varname1 varvalue1 --varname2 varvalue2 ..."><pre class="notranslate"><code>jjnbparam notebook.ipynb notebook.ipynb --varname1 varvalue1 --varname2 varvalue2 ...
</code></pre></div>
<p dir="auto">The target notebook needs to include a <code>parameters</code> cell (this does not have to be the first cell):
<a target="_blank" rel="noopener noreferrer" href="https://github.com/m-wells/jjnbparam/blob/master/parameters_cell_tagging.png"><img src="https://github.com/m-wells/jjnbparam/raw/master/parameters_cell_tagging.png" alt="Example of a tagged parameters cell" style="max-width: 100%;"></a></p>
<p dir="auto">To create a parameters cell simply edit the cell's metadata to include the following:</p>
<div class="highlight highlight-source-json notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="{
    &quot;tags&quot;: [
        &quot;parameters&quot;
    ]
}"><pre>{
    <span class="pl-ent">"tags"</span>: [
        <span class="pl-s"><span class="pl-pds">"</span>parameters<span class="pl-pds">"</span></span>
    ]
}</pre></div>
<p dir="auto">It is also helpful (for the user) to have a comment inside of the cell like so</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="# this is the parameters cell
foo = 10
bar = &quot;hi&quot;"><pre><span class="pl-c"><span class="pl-c">#</span> this is the parameters cell</span>
foo <span class="pl-k">=</span> <span class="pl-c1">10</span>
bar <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>hi<span class="pl-pds">"</span></span></pre></div>
<p dir="auto">In the cell above <code>foo</code> and <code>bar</code> are defined with what can be thought of as default values which will be used if the user does not replace them.</p>

<h2 dir="auto"><a id="user-content-customizing-notebook-execution" class="anchor" aria-hidden="true" href="#customizing-notebook-execution"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Customizing Notebook Execution</h2>
<p dir="auto">The execution of the notebook can be customized with</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="jjnbparam refnote.ipynb outnote.ipynb \
    --kernel_name julia-nodeps-1.1 \
    --timeout -1 \
    --var1 1234 \
    --var2 &quot;abcd&quot;"><pre>jjnbparam refnote.ipynb outnote.ipynb \
    --kernel_name julia-nodeps-1.1 \
    --timeout -1 \
    --var1 1234 \
    --var2 <span class="pl-s"><span class="pl-pds">"</span>abcd<span class="pl-pds">"</span></span></pre></div>
<p dir="auto">where <code>kernel_name</code> specifies the <a href="https://github.com/JuliaLang/IJulia.jl">IJulia</a> kernel and timeout defines the maximum time (in seconds) each notebook cell is allowed to run.
These values are passed under-the-hood to <code>jupyter nbconvert</code> as <a href="https://nbconvert.readthedocs.io/en/latest/execute_api.html#execution-arguments-traitlets" rel="nofollow">traitlets</a>.
If not passed the default values for <code>jupyter nbconvert</code> are used (again, see <a href="https://nbconvert.readthedocs.io/en/latest/execute_api.html#execution-arguments-traitlets" rel="nofollow">traitlets</a>).</p>

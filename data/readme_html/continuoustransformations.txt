<p>Continuous transformations (or more precisely, homeomorphisms) from ℝ (and two-point compactified version) and ℝⁿ to various open (or closed) sets used in statistics and numerical methods, such as intervals, simplexes, ordered vectors.</p>
<p><strong>Work in progress, API may change without notice.</strong></p>
<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Overview</h2>
<p>This package was born because I was tired of coding the same transformations over and over, with occasional bugs, and wanted something well-tested.</p>
<p>Transformations defined by the package can be</p>
<ol>
<li>called as functions,</li>
<li>provide the <code>logjac(transformation, x)</code> method for the log Jacobian determinant,</li>
<li>prodide the <code>inverse(transformation, x)</code> method for the inverse.</li>
</ol>
<p>Log jacobian determinants and their log are useful for domain transformations in MCMC, among other things.</p>
<p>In addition, the package includes types to represent intervals, and some basic methods of working with them. The concept of intervals is slightly different from <a href="https://github.com/JuliaMath/IntervalSets.jl">IntervalSet.jl</a> and <a href="https://github.com/dpsanders/ValidatedNumerics.jl">ValidatedNumerics.jl</a>, and as a result not compatible with either.</p>
<p>The convenience function <code>bridge(dom, img)</code> figures out the right transformation from <code>dom</code> to <code>img</code>. Currently implemented for intervals.</p>
<p>Examples:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> ContinuousTransformations
t <span class="pl-k">=</span> <span class="pl-c1">bridge</span>(ℝ, <span class="pl-c1">Segment</span>(<span class="pl-c1">0.0</span>, <span class="pl-c1">3.0</span>)) <span class="pl-c"><span class="pl-c">#</span> will use a real-circle transformation, stretched</span>
<span class="pl-c1">t</span>(<span class="pl-c1">0.0</span>)             <span class="pl-c"><span class="pl-c">#</span> 1.5</span>
<span class="pl-c1">inverse</span>(t, <span class="pl-c1">1.5</span>)    <span class="pl-c"><span class="pl-c">#</span> ≈ 0.0</span>
<span class="pl-c1">logjac</span>(t, <span class="pl-c1">0</span>)       <span class="pl-c"><span class="pl-c">#</span> ≈ 0.405</span>
<span class="pl-c1">image</span>(t)           <span class="pl-c"><span class="pl-c">#</span> Segment(0.0, 3.0)</span></pre></div>
<p><code>ArrayTransformation(transformation, dimensions...)</code> transforms a vector of numbers to an array elementwise using <code>transformation</code>.</p>
<p><code>TransformationTuple(transformations)</code> can be used for heterogeneous collections of transformations.</p>
<p><code>TransformLogLikelihood</code> wraps a log likelihood function, and <code>TransformDistribution</code> transforms a distribution. Both of them take care of the log Jacobian determinant adjustment.</p>
<p>Special transformations, useful for Bayesian methods, are also available (WIP). Feature and pull requests are appreciated.</p>

<p dir="auto"><code>SmoothLivePlot.jl</code> is a Julia package for creating live-style plots during calculations.</p>
<h1 dir="auto"><a id="user-content-motivation" class="anchor" aria-hidden="true" href="#motivation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Motivation</h1>
<p dir="auto">Updating the Juno plot plane during calculations creates new plots on top of the old ones. This produces a flickering effect e.g.:</p>
<ul dir="auto">
<li><a href="https://discourse.julialang.org/t/current-state-of-live-plots-in-atom-juno/30379" rel="nofollow">Can you update a plot in Julia?</a></li>
<li><a href="https://discourse.julialang.org/t/current-state-of-live-plots-in-atom-juno/30379" rel="nofollow">Current State of Live Plots in Atom/Juno?</a></li>
<li><a href="https://discourse.julialang.org/t/suppress-plot-window-when-output-to-animation/30724" rel="nofollow">Suppress Plot Window when output to animation</a></li>
</ul>
<p dir="auto">To smoothly update of plots, I generalised a <a href="https://discourse.julialang.org/t/current-state-of-live-plots-in-atom-juno/30379/7" rel="nofollow">solution found by user ckneale</a>. It uses <a href="https://github.com/JuliaGizmos/Observables.jl">Observables.jl</a> and <a href="https://github.com/JuliaGizmos/WebIO.jl">WebIO.jl</a> so that the plot can listen to changes in its elements.</p>
<p dir="auto">Currently, I have tested the following capabilities:</p>
<ul dir="auto">
<li>Modifying values in X and/or Y array(s) in scatter and plot</li>
<li>Modifying colours in scatter and plot</li>
<li>Modifying text elements (e.g. titles, xlabels, etc...)</li>
<li>Modifying matricies in contour plots</li>
<li>Adding new elements to X,Y arrays in 2d line and scatter plots</li>
<li>Adding new elements to X,Y,Z in 3d line and scatter plots</li>
</ul>
<p dir="auto">Note: this package is designed to work with the <strong>plot plane in Juno</strong>. If you force it to plot in a gui it will look really weird.</p>
<h1 dir="auto"><a id="user-content-using-the-package" class="anchor" aria-hidden="true" href="#using-the-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Using the package</h1>
<ol dir="auto">
<li>Import the module using <code>using SmoothLivePlot</code>.</li>
<li>Create a live plot with macro <code>outPlotObject = @makeLivePlot myPlotFunction(argument1, argument2, ...)</code>.
<ul dir="auto">
<li>Function <code>myPlotFunction(argument1, argument2, ...)</code> is a user defined plotting function.</li>
<li>Output <code>outPlotObject</code> is a mutable output array of plot features. Its elements are the input aruments of <code>myPlotFunction()</code>.</li>
</ul>
</li>
<li>Modify plot elements with function <code>modifyPlotObject!(outPlotObject, arg2 = newArg2, arg1 = newArg1, ...)</code>.
<ul dir="auto">
<li>The first argment of <code>modifyPlotObject!()</code> must be the mutable output array.</li>
<li>The following argments are optional and named. The name/value pair must be <code>arg&lt;x&gt; = newArg1</code>, where <code>&lt;x&gt;</code> in the name is an integer that indicates the position of the argument in the original plotting function <code>myPlotFunction()</code>.</li>
<li>E.g. to modify <code>argument2</code> to <code>newArgument2</code>, use <code>modifyPlotObject!(outPlotObject, arg2 = newArgument2)</code>.</li>
<li>The modified arguments do not have to be in any specific order, and are updated at the same time.</li>
</ul>
</li>
</ol>

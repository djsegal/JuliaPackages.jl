<h2 dir="auto"><a id="user-content-purpose-of-this-package" class="anchor" aria-hidden="true" href="#purpose-of-this-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Purpose of this package</h2>
<p dir="auto">The StatisticalRethinking.jl <code>package</code> contains functions comparable to the functions in the R package "rethinking" associated with the book <a href="https://xcelab.net/rm/statistical-rethinking/" rel="nofollow">Statistical Rethinking</a> by Richard McElreath.</p>
<p dir="auto">These functions are used in Jupyter and  Pluto notebook <code>projects</code> specifically intended for hands-on use while studying the book or taking the course.</p>
<p dir="auto">Currently there are 3 of these notebook projects:</p>
<ol dir="auto">
<li>
<p dir="auto">Max Lapan's <a href="https://github.com/Shmuma/rethinking-2ed-julia">rethinking-2ed-julia</a> which uses Turing.jl and Jupyter notebooks.</p>
</li>
<li>
<p dir="auto">The <a href="https://github.com/StatisticalRethinkingJulia/SR2TuringPluto.jl">SR2TuringPluto.jl</a> project, also Turing.jl based but using Pluto.jl instead of Jupyter. It is based on Max Lapan's work above.</p>
</li>
<li>
<p dir="auto">The <a href="https://github.com/StatisticalRethinkingJulia/SR2StanPluto.jl">SR2StanPluto.jl</a> project, which uses Stan as implemented in StanSample.jl and StanQuap.jl. See <a href="https://github.com/StanJulia">StanJulia</a>.</p>
</li>
</ol>
<p dir="auto">There is a fourth option to study the Turing.jl versions of the models in the Statistical Rethinking book which is in the form of a package and Franklin web pages: <a href="https://github.com/StatisticalRethinkingJulia/TuringModels.jl">TuringModels.jl</a>.</p>
<h2 dir="auto"><a id="user-content-why-a-statisticalrethinking-v4" class="anchor" aria-hidden="true" href="#why-a-statisticalrethinking-v4"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why a StatisticalRethinking v4?</h2>
<p dir="auto">Over time more options become available to express the material covered in Statistical Rethinking, e.g. the use of KeyedArrays (provided by <a href="https://github.com/JuliaArrays/AxisArrays.jl">AxisKeys.jl</a>) for the representation of mcmc chains.</p>
<p dir="auto">Other examples are the recently developed <a href="https://github.com/TuringLang/ParetoSmooth.jl">ParetoSmooth.jl</a> which could be used in the PSIS related examples as a replacement for ParetoSmoothedImportanceSampling.jl and the preliminary work by <a href="https://github.com/Shmuma/Dagitty.jl">SHMUMA</a> on Dagitty.jl (a potential replacement for StructuralCausalModels.jl).</p>
<p dir="auto">While StatisticalRethinking v3 focused on making StatisticalRethinking.jl mcmc package independent, StatisticalRethinking v4 aims at de-coupling it from a specific graphical package and thus enables new choices for graphics, e.g. using Makie.jl and AlgebraOfGraphics.jl.</p>
<p dir="auto">StatisticalRethinking.jl v4 also fits better with the new setup of Pluto notebooks which keep track of used package versions in the notebooks themselves (<a href="https://github.com/fonsp/Pluto.jl/wiki/%F0%9F%8E%81-Package-management">see here</a>).</p>
<h2 dir="auto"><a id="user-content-workflow-of-statisticalrethinkingjulia-v4" class="anchor" aria-hidden="true" href="#workflow-of-statisticalrethinkingjulia-v4"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Workflow of StatisticalRethinkingJulia (v4):</h2>
<ol dir="auto">
<li>
<p dir="auto">Data preparation, typically using CSV.jl, DataFrames.jl and some statistical methods from StatsBase.jl and Statistics.jl. In some cases simulations are used which need Distributions.jl and a few special methods (available in StatisticalRethinking.jl).</p>
</li>
<li>
<p dir="auto">Define the mcmc model, e.g. using StanSample.jl or Turing.jl, and obtain draws from the model.</p>
</li>
<li>
<p dir="auto">Capture the draws for further processing. In Turing that is usually done using MCMCChains.jl, in StanSample.jl v4 it's mostly in the form of a DataFrame, a StanTable, a KeyedArray chains (obtained from AxisKeys.jl).</p>
</li>
<li>
<p dir="auto">For further processing, the projects nearly always convert chains to a DataFrame.</p>
</li>
<li>
<p dir="auto">Inspect the chains using statistical and visual methods. In many cases this will need one or more statistical packages and one of the graphical options.</p>
</li>
</ol>
<p dir="auto">Currently visual options are StatsPlots/Plots based, e.g. in MCMCChains.jl and StatisticalRethinkingPlots.jl.</p>
<ol start="5" dir="auto">
<li>The above 5 steps could all be done by just using StanSample.jl or Turing.jl.</li>
</ol>
<p dir="auto"><strong>The book Statistical Rethinking has a different objective and studies how models compare, how models can help (or mislead) and why multilevel modeling might help in some cases.</strong></p>
<ol start="6" dir="auto">
<li>For this, additional packages are available, explained and demonstrated, e.g. StructuralCausalModels.jl, ParetoSmoothedImportanceSampling.jl and quite a few more.</li>
</ol>
<h2 dir="auto"><a id="user-content-using-statisticalrethinking-v4" class="anchor" aria-hidden="true" href="#using-statisticalrethinking-v4"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Using StatisticalRethinking v4</h2>
<p dir="auto">To work through the StatisticalRethinking book using Julia and Turing or Stan, download either one of the above mentioned <code>projects</code> and start Pluto (or Jupyter).</p>
<p dir="auto">An early, experimental version of <a href="https://github.com/StatisticalRethinkingJulia/StructuralCausalModels.jl">StructuralCausalModels.jl</a> is also included as a dependency in the StatisticalRethinking.jl package.</p>
<p dir="auto">In the meantime I will definitely keep my eyes on <a href="https://github.com/Shmuma/Dagitty.jl">Dagitty.jl</a>, <a href="https://github.com/zenna/Omega.jl">Omega.jl</a> and <a href="https://github.com/mschauer/CausalInference.jl">CausalInference.jl</a>. In particular Dagitty.jl has very similar objectives as StructuralCausalModels.jl and over time might replace it in the StatisticalRethinkingJulia ecosystem. For now, StructuralCausalModels does provide ways to convert DAGs to Dagitty and ggm formats.</p>
<p dir="auto">Similarly, a dependency <a href="https://github.com/StatisticalRethinkingJulia/ParetoSmoothedImportanceSampling.jl">ParetoSmoothedImportanceSampling.jl</a> is used which provides PSIS and WAIC statistics for model comparison.</p>

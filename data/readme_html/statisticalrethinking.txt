<h2><a id="user-content-purpose-of-this-package" class="anchor" aria-hidden="true" href="#purpose-of-this-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Purpose of this package</h2>
<p>This package contains Julia versions of selected <code>code snippets</code> and <code>mcmc models</code> contained in the R package "rethinking" associated with the book <a href="https://xcelab.net/rm/statistical-rethinking/" rel="nofollow">Statistical Rethinking</a> by Richard McElreath.</p>
<p>As stated many times by the author in his <a href="https://www.youtube.com/watch?v=ENxTrFf9a7c&amp;list=PLDcUM9US4XdNM4Edgs7weiyIguLSToZRI" rel="nofollow">online lectures</a>, this package is not intended to take away the hands-on component of the course. The clips are just meant to get you going but learning means experimenting, in this case using Julia and Stan.</p>

<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Introduction</h2>
<p>In the book and associated R package <code>rethinking</code>, statistical models are defined as illustrated below:</p>
<pre><code>flist &lt;- alist(
  height ~ dnorm( mu , sigma ) ,
  mu &lt;- a + b*weight ,
  a ~ dnorm( 156 , 100 ) ,
  b ~ dnorm( 0 , 10 ) ,
  sigma ~ dunif( 0 , 50 )
)
</code></pre>
<p>Posterior values can be approximated by</p>
<pre><code># Simulate quadratic approximation (for simpler models)
m4.31 &lt;- quad(flist, data=d2)
</code></pre>
<p>or generated using Stan by:</p>
<pre><code># Generate a Stan model and run a simulation
m4.32 &lt;- ulam(flist, data=d2)
</code></pre>
<p>The author of the book states: "<em>If that (the statistical model) doesn't make much sense, good. ... you're holding the right textbook, since this book teaches you how to read and write these mathematical descriptions</em>" (page 77).</p>
<p><a href="https://github.com/StatisticalRethinkingJulia/StatisticalRethinking.jl">StatisticalRethinking.jl</a> is intended to allow experimenting with this learning process using <a href="https://github.com/StanJulia">StanJulia</a>.</p>
<h2><a id="user-content-rethinking-rethinking" class="anchor" aria-hidden="true" href="#rethinking-rethinking"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Rethinking <code>rethinking</code></h2>
<p>There are a few important differences between <code>rethinking</code> and <code>StatisticalRethinking.jl</code>:</p>
<ol>
<li>In v2.x of StatisticalRethinking.jl, ulam() has been replaced by StanSample.jl.</li>
</ol>
<p>This means that much earlier on than in the book, StatisticalRethinking.jl introduces the reader to the Stan language.</p>
<p>To help out with this, in the subdirectory <code>scripts/03/intro-stan</code> the Stan language is introduced and the execution of Stan language programs illustrated. Chapter 9 of the book contains a nice introduction to translating the <code>alist</code> R models to the Stan language (just before section 9.5).</p>
<p>To check the chains produced MCMCChains.jl can be used, e.g. see the scripts in chapter 5.</p>
<ol start="2">
<li>The equivalent of the R function <code>quap()</code> in StatisticalRethinking.jl v2.0 uses the MAP density of the Stan samples as the mean of the Normal distribution and reports the approximation as a NamedTuple. e.g. from <code>scripts/04-part-1/clip-31.jl</code>:</li>
</ol>
<pre><code>if success(rc)
  df = read_samples(sm; output_format=:dataframe)
  q = quap(df)
  q |&gt; display
end
</code></pre>
<p>returns:</p>
<pre><code>(mu = 178.0 ± 0.1, sigma = 24.5 ± 0.94)
</code></pre>
<p>The above call to read_samples(...) appends all chains in a single dataframe. To retrieve the chains in separate dataframes ( <code>Vector{DataFrames}</code> ) use:</p>
<pre><code>df = read_samples(sm; output-Format=:dataframes)
</code></pre>
<p>To obtain the mu quap:</p>
<pre><code>q.mu
</code></pre>
<p>To obtain the samples:</p>
<pre><code>q.mu.particles
</code></pre>
<p>Examples and comparisons of different ways of computing a quap approximation can be found in <code>scripts/03/intro-stan/intro-part-4.jl</code>.</p>
<ol start="3">
<li>
<p>In <code>scripts/04-part-1</code> an additional section has been added, <code>intro-logpdf</code> which introduces an alternative way to compute the MAP (quap) using Optim.jl. This kind of builds on the logpdf formulation introduced in <code>scripts/03/intro-stan/intro-part-4.jl</code></p>
</li>
<li>
<p>In <code>scripts/09</code> an additional intro section has been included, <code>scripts/09/intro-dhmc</code>. It is envisage that a future version of StatisticalRethinking.jl will be based on DynamicHMC.jl. No time line has been set for this work.</p>
</li>
</ol>
<h2><a id="user-content-layout-of-the-package" class="anchor" aria-hidden="true" href="#layout-of-the-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Layout of the package</h2>
<p>Instead of having all snippets in a single file, the snippets are organized by chapter and grouped in clips by related snippets. E.g. chapter 0 of the R package has snippets 0.1 to 0.5. Those have been combined into 2 clips:</p>
<ol>
<li><code>clip-01-03.jl</code> - contains snippets 0.1 through 0.3</li>
<li><code>clip-04-05.jl</code> - contains snippets 0.4 and 0.5.</li>
</ol>
<p>A single snippet clip will be referred to as <code>03/clip-02.jl</code>.</p>
<p>As mentioned above, a few chapters contain additional scripts intended as introductions for specific topics.</p>

<p>Developing <code>rethinking</code> must have been an on-going process over several years, <code>StatisticalRethinking.jl</code> will likely follow a similar path.</p>
<ol>
<li>
<p>The first version (v1.x) of <code>StatisticalRethinking</code> attempts to capture the models and to show ways of setting up those models, execute the models and post-process the results using Julia.</p>
</li>
<li>
<p>Many R functions such as precis(), shade(), etc. are either not in v1 or replaced by Julia equivalents, e.g. the Particles approach is used instead of precis(). Expect significant refactoring of those in future versions of StatisticalRethinking.jl.</p>
</li>
<li>
<p>Several other interesting approaches to mcmc modeling are being explored in Julia, e.g. Soss.jl and Omega.jl. These are tracked as candidates for use in a future version of StatisticalRethinking.jl.</p>
</li>
</ol>

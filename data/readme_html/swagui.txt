<p dir="auto"><a href="https://codecov.io/gh/jiachengzhang1/SwaggerUI" rel="nofollow"><img src="https://camo.githubusercontent.com/f3901ea9775d6f7bb386e69e23c5ab688e0747b4265971f95087e152a86cae6b/68747470733a2f2f636f6465636f762e696f2f67682f6a69616368656e677a68616e67312f5377616767657255492f6272616e63682f6d61737465722f67726170682f62616467652e7376673f746f6b656e3d483838544b3447314e59" alt="codecov" data-canonical-src="https://codecov.io/gh/jiachengzhang1/SwaggerUI/branch/master/graph/badge.svg?token=H88TK4G1NY" style="max-width: 100%;"></a></p>
<p dir="auto">Want to use <a href="https://swagger.io/tools/swagger-ui/" rel="nofollow">Swagger UI</a> in Julia? This package has your back!</p>
<p dir="auto">Inspired by <a href="https://github.com/scottie1984/swagger-ui-express">swagger-ui-express</a>, the package auto-generates Swagger UI based on <code>swagger.json</code>. The generated API documentation can be served as an endpoint using package such as <a href="https://github.com/GenieFramework/Genie.jl">Genie.jl</a>.</p>
<p dir="auto">Pre-built <a href="https://github.com/swagger-api/swagger-ui/tree/master/dist">swagger-ui</a> is used. Because swagger-ui is implemented in Node.js, all pre-built files and assets are included in the <a href="dist">dist</a> folder for serving. (Open an issue if there are better ways to do this.)</p>

<p dir="auto"><a href="https://github.com/jiachengzhang1/SwaggerMarkdown">SwaggerMarkdown</a> builds the swagger document from markdown comments in the code. It returns a <code>swagger_document::Dict{String, Any}</code> that can be used by <code>SwaggerUI.render_swagger</code> to render the API documentation as Swagger's fashion.</p>

<p dir="auto"><strong>Swagger Explorer</strong></p>
<p dir="auto">The explorer can be turned off through setting the <code>Options.show_explorer</code> to <code>false</code>. It's turned on, i.e. <code>Options.show_explorer = true</code> by default.</p>

<p dir="auto"><strong>Custom swagger options</strong></p>
<p dir="auto">Swagger UI can be configured by setting <code>Options.swagger_options::Dict{String, Any}</code>. The key is the name of the configuration while the value is the setting. More details about configuration can be found through the <a href="https://github.com/swagger-api/swagger-ui/blob/master/docs/usage/configuration.md">Official Swagger UI Configuration page</a>. The example below sets the URL pointing to API definition to <code>https://petstore.swagger.io/v2/swagger.json</code>.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Genie, Genie.Router

    # swagger_document is not needed
    render_swagger(nothing, options=options)

<p dir="auto">A dummy package for the <a href="https://syl1.gitbook.io/julia-language-a-concise-tutorial/language-core/11-developing-julia-packages" rel="nofollow">Julia Concise Tutorial</a></p>
<p dir="auto">To create a new package (let's say <code>MyPackage</code>):</p>
<ul dir="auto">
<li>fork this package in your own or your organisation account (let's say <code>MyUser</code>)</li>
<li>from the github interface:
<ul dir="auto">
<li>edit the <code>README.md</code> file to update the badges</li>
<li>edit the <code>Project.toml</code> file to change package name, author, dependences and UUID (you can get a new UUID from a Julia prompt with <code>using UUIDs; uuid1()</code>)</li>
<li>edit the file <code>.github/workflows/ci.yml</code> to update the package name</li>
<li>rename the file <code>src/MyAwesomePackage.jl</code> and edit its content (docstring and module name) to reflect the new name</li>
<li>in <code>docs/make.jl</code> rename all the occurrences of <code>MyAwesomePackage</code> to <code>MyPackage</code> (including the account/organisation name where to deploy the documentation)</li>
<li>update <code>docs/src/index.md</code> and <code>docs/src/anotherPage.md</code></li>
</ul>
</li>
<li>get it in julia. From a Julia prompt type <code>] [ENTER]</code> to enter the <code>pkg</code> prompt and then <code>add git@github.com:MyUser/MyPackage.jl.git</code></li>
<li>declare you want to develop it. Still from the <code>pkg</code> prompt, type; <code>dev MyPackage</code>. Your new package will be in <code>[JULIA_ROOT]/dev/MyPackage</code>.</li>
</ul>

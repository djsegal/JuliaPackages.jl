<p dir="auto">Due to the way Julia functions, it will recompile a function if input/output changes. A basic example would be running a function with a "Float32" and then a "Float64" input. If the input type is hardcoded Julia will under the hood recompile a type specific version for each input type. Therefore, as an user, if you experience slow initial performance, when trying to load a new variable, try in the first run only to read one file, ie. <code>vel_array = readBi4Array("Part_0000.bi4",Cat(3))</code> and then afterwards read all files, by using only "Part".</p>

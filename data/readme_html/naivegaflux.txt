<p dir="auto">Neural architecture search for <a href="https://github.com/FluxML/Flux.jl">Flux</a> models using genetic algorithms.</p>
<p dir="auto">Tired of tuning hyperparameters? Once you've felt the rush from reasoning about hyper-hyperparameters there is no going back!</p>

<p dir="auto">The basic idea is to create not just one model, but a population of several candidate models with different hyperparameters. The whole population is then evolved while the models are being trained.</p>

<p dir="auto">More concretely, this means train each model for a number of iterations, evaluate the fitness of each model, select the ones with highest fitness, apply random mutations (e.g. add/remove neurons/layers) to some of them and repeat until a model with the desired fitness has been produced.</p>
<p dir="auto">By controlling the number of training iterations before evolving the population, it is possible tune the compromise between fully training each model at the cost of longer time to evolve versus the risk of discarding a model just because it trains slower than the other members.</p>
<p dir="auto">Like any self-respecting AutoML-type library, NaiveGAflux provides an application with a deceivingly simple API:</p>

Head over to the <a href="https://DrChainsaw.github.io/NaiveGAflux.jl/stable" rel="nofollow">documentation</a> to get started.</p>

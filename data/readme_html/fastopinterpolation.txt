<p dir="auto">Fast multidimensional interpolation based on orthogonal polynomials. Supports domains build from tensor products, e.g. a prism as product of a line and a triangle.</p>
<p dir="auto">Currently implemented combinations of Domains &amp; Polynomials are</p>

<td>1d</td>

<td>2d</td>

<p dir="auto">You can specify the set of nodes yourself or use the predefined ones, their maximum order is also stated in the table above.</p>
<p dir="auto">A complete working example can be found <a href="examples/">the example folder</a>.</p>
<p dir="auto">For a given set of orthogonal polynomials (with their recurrence relation) it is easy to extend the code to other geometries. Please do not hesitate to get in contact with us (via mail or github issue) to request further geometries or features. You are of course also always invited to implement it yourself and create a pull request.</p>
<p dir="auto">Current limitations include:</p>
<ul dir="auto">
<li>no shared-memory parallelization (due to buffers)</li>
<li>autodiff (same reason as shared-memory parallelization)</li>
<li>only scalar functions (matching type of coordinates and return type of the function to be interpolated)</li>
</ul>
<p dir="auto">All three limitations are partially related to the heavy use internal buffers to maximize performance for repeated evaluations.
We are working on a simulation to circument these limitations.</p>

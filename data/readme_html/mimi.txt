<h1><a id="user-content-mimi---integrated-assessment-modeling-framework" class="anchor" aria-hidden="true" href="#mimi---integrated-assessment-modeling-framework"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Mimi - Integrated Assessment Modeling Framework</h1>
<p>Mimi is a <a href="http://julialang.org" rel="nofollow">Julia</a> package that provides a component model for <a href="https://en.wikipedia.org/wiki/Integrated_assessment_modelling" rel="nofollow">integrated assessment models</a>, as described in detail on the <a href="https://www.mimiframework.org" rel="nofollow">Mimi Framework Homepage</a>. The <a href="https://www.mimiframework.org/Mimi.jl/stable/" rel="nofollow">Documentation</a> includes information on the installation and use of this package, step-by-step tutorials, how-to guides, and techincal reference. The development team closely monitors the <a href="https://forum.mimiframework.org" rel="nofollow">Mimi Framework forum</a> and we suggest this as a starting place for any questions users may have.</p>
<h2><a id="user-content-news" class="anchor" aria-hidden="true" href="#news"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>NEWS</h2>
<p>On 7/15/2020 we officially tagged and released Mimi v1.0.0, which has some new features, documentation, and quite a bit of internals work as well.  Since this is a major version change, there are some breaking changes that may require you to update your code.  We have done the updates for the existing models in the Mimi registry (FUND, DICE, etc.), and will release new major versions of those today as well, so if you are using the latest version of Mimi and the latest version of the packages, all should run smoothly.</p>
<p><strong>Please view the how to guide here: <a href="https://www.mimiframework.org/Mimi.jl/stable/howto/howto_6/" rel="nofollow">https://www.mimiframework.org/Mimi.jl/stable/howto/howto_6/</a> for a run-down of how you should update your own code.</strong></p>
<p>In addition please do not hesitate to ask any questions on the forum, we are working hard to keep this transition smooth.</p>

<h1><a id="user-content-mimi---integrated-assessment-modeling-framework" class="anchor" aria-hidden="true" href="#mimi---integrated-assessment-modeling-framework"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Mimi - Integrated Assessment Modeling Framework</h1>
<p>Mimi is a <a href="http://julialang.org" rel="nofollow">Julia</a> package that provides a component model for <a href="https://en.wikipedia.org/wiki/Integrated_assessment_modelling" rel="nofollow">integrated assessment models</a>, as described in detail on the <a href="https://www.mimiframework.org" rel="nofollow">Mimi Framework Homepage</a>. The <a href="https://www.mimiframework.org/Mimi.jl/stable/" rel="nofollow">Documentation</a> includes information on the installation and use of this package, as well as details on the Mimi registry and step-by-step tutorials which users of different experience levels, starting with first-time users. The development team closely monitors the <a href="https://forum.mimiframework.org" rel="nofollow">Mimi Framework forum</a> and we suggest this as a starting place for any questoins users may have.</p>

<p>You can interact with the Mimi development team via issues and pull requests here on github, and in the <a href="https://forum.mimiframework.org" rel="nofollow">Mimi Framework forum</a>. Contributions to Mimi are also most welcome, and

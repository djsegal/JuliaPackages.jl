<p dir="auto">Mimi is a <a href="http://julialang.org" rel="nofollow">Julia</a> package that provides a component model for <a href="https://en.wikipedia.org/wiki/Integrated_assessment_modelling" rel="nofollow">integrated assessment models</a>, as described in detail on the <a href="https://www.mimiframework.org" rel="nofollow">Mimi Framework Homepage</a>. The <a href="https://www.mimiframework.org/Mimi.jl/stable/" rel="nofollow">Documentation</a> includes information on the installation and use of this package, step-by-step tutorials, how-to guides, and technical reference. The development team closely monitors the <a href="https://forum.mimiframework.org" rel="nofollow">Mimi Framework forum</a> and we suggest this as a starting place for any questions users may have.</p>
<p dir="auto">Also, note that if at any point julia-related issues with workflows, environments, and/or package versioning become frustrating, please do not hesitate to reach out via the <a href="https://forum.mimiframework.org" rel="nofollow">forum</a>.  This can be a hurdle to learn at first, but taking a moment to get it right early will save you <strong>a lot</strong> of time down the road.  We are more than happy to help you, and are getting together some standardized resources in the meantime.</p>

<p dir="auto">11/27/2022</p>
<p dir="auto">The newest model on our platform, <a href="https://github.com/rffscghg/MimiGIVE.jl/tree/main/src">MimiGIVE</a> is a fully open source model featured in the recent Nature publication <a href="https://www.nature.com/articles/s41586-022-05224-9" rel="nofollow">Comprehensive evidence implies a higher social cost of CO2</a> and recent <a href="https://www.epa.gov/environmental-economics/scghg" rel="nofollow">EPA work</a>.</p>
<p dir="auto">11/1/2021</p>
<p dir="auto">Check out the new <a href="https://github.com/anthofflab/MimiSSPs.jl">MimiSSPs.jl</a> component, pulling in various SSP and RCP data sources into a component to streamline running Mimi models as forced by the SSPs and RCPs.</p>
<p dir="auto">6/14/2021</p>
<p dir="auto">We recently tagged and released a feature release revamping the API surrounding parameters, please see <a href="https://www.mimiframework.org/Mimi.jl/dev/howto/howto_5/" rel="nofollow">https://www.mimiframework.org/Mimi.jl/dev/howto/howto_5/</a> and <a href="https://www.mimiframework.org/Mimi.jl/dev/howto/howto_9/" rel="nofollow">https://www.mimiframework.org/Mimi.jl/dev/howto/howto_9/</a>.</p>
<p dir="auto">5/3/2021</p>
<p dir="auto">We have written up a repository and accompanying notebook <a href="https://github.com/anthofflab/MimiFUND-MimiFAIR-Flat.jl/blob/main/MimiFUND-MimiFAIR-Flat.ipynb">here</a> to demonstrate the steps to couple together two models, specifically in this case replacing (most of) the <a href="https://github.com/fund-model/MimiFUND.jl">FUND model</a>'s climate module with the <a href="https://github.com/anthofflab/MimiFAIR.jl">FAIR model</a>.</p>
<p dir="auto">7/15/2020</p>
<p dir="auto">We officially tagged and released Mimi v1.0.0, which has some new features, documentation, and quite a bit of internals work as well.  Since this is a major version change, there are some breaking changes that may require you to update your code.  We have done the updates for the existing models in the Mimi registry (FUND, DICE, etc.), and will release new major versions of those today as well, so if you are using the latest version of Mimi and the latest version of the packages, all should run smoothly.</p>
<p dir="auto"><strong>Please view the how to guide here: <a href="https://www.mimiframework.org/Mimi.jl/stable/howto/howto_7/" rel="nofollow">https://www.mimiframework.org/Mimi.jl/stable/howto/howto_7/</a> for a run-down of how you should update your own code.</strong></p>
<p dir="auto">In addition please do not hesitate to ask any questions on the forum, we are working hard to keep this transition smooth.</p>

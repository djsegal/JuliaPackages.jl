<ul dir="auto">
<li>
<p dir="auto"><strong>Minimal macros</strong>: <a href="https://docs.julialang.org/en/stable/manual/metaprogramming/" rel="nofollow">Macros</a> are neat, but can be hard to understand. The <code>Base.Test</code> macros do many things in the name of generality, but as a result generate a lot of code and stress the compiler. Pukeko aims for minimal complexity in its test macros, which essentially just call functions. Pukeko has familiar <code>@test</code> and <code>@test_throws</code> macros to make moving from <code>Base.Test</code> as easy as possible. As little work as possible is done for the usual case (tests passing).</p>
</li>
<li>
<p dir="auto"><strong>Functions</strong>: <code>Base.Test</code> has testsets that are defined by <code>@testset begin end</code> blocks. These blocks do not introduce proper scopes and tend to lead to very large functions with a high level of nesting. Pukeko uses plain-old-functions as a testset and uses modules to collect all the tests to be executed together. Minimal magic, maximum clarity, less danger of reusing variables accidentally, and less compiler strain (large functions are hard on the Julia compiler).</p>
</li>
<li>
<p dir="auto"><strong>Parallel testing</strong>: Larger projects inevitably end up with a large number of tests. There are typically many tests per file, spread across many files. Normally this involves having one central <code>test/runtests.jl</code> file that includes other <code>test/*.jl</code> files. This is good for automated CI services like Travis, but often developer testing machines can run more than one test at a time. Pukeko's module-and-functions pattern naturally makes each of these <code>test/*.jl</code> files runnable individually or as part of a larger test run: <code>ls test/ | xargs -I % julia --project=. %</code></p>
</li>
<li>
<p dir="auto"><strong>Use command line for...</strong>: Pukeko makes use of commandline flags for customization. Highlights include selectively running tests, printing out run times for tests to identify slow tests, and changing behaviour on test failures.</p>
</li>
</ul>
<p dir="auto">Pukeko follows the <a href="http://www.juliaopt.org/JuMP.jl/latest/style.html" rel="nofollow">JuMP Style Guide</a>.</p>

module MyTests
    using Pukeko  # @test, @test_throws

    function test_arithmetic()
        @test 2 + 2 == 4
        @test 2 * 3 == 6
    end

    function _test_parametric(value, value_exp)
        @test value * value == value_exp
    end
    Pukeko.@parametric _test_parametric [(1, 1), (2, 4), (3, 9)]
end

import Pukeko
Pukeko.run_tests(MyTests)

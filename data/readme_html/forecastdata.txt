<p dir="auto">This package handles access to small to medium size time series datasets (in the Megabytes) to be used in Julia.</p>
<p dir="auto">Since the number of datasets could be potentially large and relatively big in size they are not downloaded when the package is installed, but only when they are called to then be stored locally as a Julia artifact.</p>
<p dir="auto">The format chosen to return the time series is via a <a href="https://dataframes.juliadata.org/stable/" rel="nofollow">DataFrame</a> with its first column using <a href="https://docs.julialang.org/en/v1/stdlib/Dates/" rel="nofollow">Dates</a> when appropiate.</p>
<p dir="auto">In general time series available in ForecastData will be "unique" in one of these two ways:</p>
<ul dir="auto">
<li>They are a combination of publicly available datasets (e.g. <code>london</code> dataset)</li>
<li>They are publicly available datasets with some preparation to facilitate their use (e.g. <code>quakes</code> dataset)</li>
</ul>
<p dir="auto">Simple publicly available datasets will not be generally considered, for instance <a href="https://cran.r-project.org" rel="nofollow">R</a> datasets like  <code>AirPassengers</code> will not be stored in ForecastData since they can be easily accessed via <a href="https://github.com/JuliaInterop/RCall.jl">RCall</a>.</p>

<p dir="auto">if you would like to contribute new datasets to the package please consider a Pull Request with the following actions:</p>
<ul dir="auto">
<li>Create a data/[new_dataset].tar.gz file with a [new_dataset].md description file.</li>
<li>Create a src/[new_dataset].jl file</li>
<li>Create a test/[new_dataset].jl file</li>
<li>Update the README.md and Forecast.jl files with the new dataset information</li>
<li>Update Artifacts.toml with the new dataset</li>
</ul>

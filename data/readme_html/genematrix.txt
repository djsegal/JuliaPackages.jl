<p dir="auto">This package is iniatially contended for the constructions various
matrices used in animal breeding and genetics.  It also provides a set
of file I/O functions to store and retrieve the matrices to and from a
storage, e.g, HDD.  The reason for the latter is that saving the
matrix might occupy too much disk space, e.g., the 3-column,
row-column-value storage.  It also prevents precision loss while store
float number in text format.</p>

<p dir="auto">The storage format is called <code>XY</code> format.  Each contains a header, as
defined by <code>struct xyheader</code>.  The starting 3 characters of a header
are "XY ".  The 4th character is any one of "FLUS", that is <code>F</code>ull
matrix, <code>L</code>ower triangle, <code>U</code>pper triangle, or lower triangle of a
<code>S</code>ymmetric matrix.  the 5th character specify if the genotypes matrix
is of <code>nLoci $\times$ nID</code> (<code>N</code>), or <code>T</code>ransposed if applies.  The 6th
character specify element types of the matrix.  The currently
supported types are <code>Bool, Int8, Int16, Int32, Int64, Int128, UInt8, UInt16, UInt32, UInt64, UInt128, Float16, Float32, Float64</code>, though
some of them may never be used.  If a new type is to be added, it must
appended to the end of above.</p>

<p dir="auto">The function for reading is <code>readxy</code>.  It only take one argument of
file name.  The one for writing is <code>writexy(file, mat; mattp = 'F', trans = 'N')</code>.  It write matrix <code>mat</code> into <code>file</code>.  If <code>mat</code> is
symmetric, then <code>mattp</code> is overruled as <code>S</code>.</p>
<p dir="auto">The package also provides two other I/O functions to input and output
plink bed files.</p>

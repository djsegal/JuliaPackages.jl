<p dir="auto">This package is a thin, efficient wrapper around <a href="http://arblib.org" rel="nofollow">Arb</a> - a C library for arbitrary-precision ball arithmetic.</p>
<p dir="auto">The package is currently in early development. More features and
documentation will be added. While we try to avoid it there might be
breaking changes.</p>

<p dir="auto">From the Arb documentation:</p>
<blockquote>
<p dir="auto">Arb is a C library for rigorous real and complex arithmetic with arbitrary precision. Arb tracks numerical errors automatically using ball arithmetic, a form of interval arithmetic based on a midpoint-radius representation. On top of this, Arb provides a wide range of mathematical functionality, including polynomials, power series, matrices, integration, root-finding, and many transcendental functions. Arb is designed with efficiency as a primary goal, and is usually competitive with or faster than other arbitrary-precision packages.</p>
</blockquote>

<p dir="auto">Indexing of an <code>ArbMatrix</code> returns an <code>Arb</code> whereas indexing <code>ArbRefMatrix</code> returns an <code>ArbRef</code>.
An <code>ArbMatrix</code> <code>A</code> can also be index using the <code>ref</code> function , e.g, <code>ref(A, i, j)</code> to obtain
an <code>ArbRef</code>.</p>
<p dir="auto">Additionally, there are multiple union types defined to capture a <code>Ref</code> and non-<code>Ref</code> version.
For example <code>Arb</code> and <code>ArbRef</code> are subtypes of <code>ArbLike</code>. Similarly, we provide
<code>MagLike</code>, <code>ArfLike</code>, <code>ArbLike</code>, <code>AcbLike</code>, <code>ArbVectorLike</code>, <code>AcbVectorLike</code>, <code>ArbMatrixLike</code>,
<code>AcbMatrixLike</code>.</p>
<p dir="auto">Both <code>ArbPoly</code> and <code>ArbSeries</code> wrap the <code>arb_poly_t</code> type. The
difference is that <code>ArbSeries</code> has a fixed length and is therefore
suitable for use when Taylor series are computed using the <code>_series</code>
functions in Arb. Similar for <code>AcbPoly</code> and <code>AcbSeries</code>.</p>

<h2 dir="auto"><a id="user-content-naming-convention" class="anchor" aria-hidden="true" href="#naming-convention"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Naming convention</h2>
<p dir="auto">Arb functions are wrapped by parsing the Arb documentation and applying the following set of rules to "Juliafy" the function names:</p>
<ol dir="auto">
<li>The parts of a function name which only refer to the type of input are removed since Julia has multiple dispatch to deal with this problem.</li>
<li>Functions which modify the first argument get an <code>!</code> appened.</li>
<li>For functions which take a precision argument this arguments becomes a <code>prec</code> keyword argument
which is by default set to the precision of the first argument (if applicable).</li>
<li>For functions which take a rounding mode argument this arguments becomes a <code>rnd</code> keyword argument
which is by default set to <code>RoundNearest</code>.</li>
</ol>

The function</p>

<h2 dir="auto"><a id="user-content-constructors-and-setters" class="anchor" aria-hidden="true" href="#constructors-and-setters"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Constructors and setters</h2>
<p dir="auto">Arb defines a number of functions for setting something to a specific
value, for example <code>void arb_set_si(arb_t y, slong x)</code>. All of these
are renamed to <code>set!</code> and rely on multiple dispatch to choose the
correct one. In addition to the ones defined in Arb there is a number
of methods of <code>set!</code> added in Arblib to make it more convenient to
work with. For example there are setters for <code>Rational</code> and all
irrationals defined in <code>Base.MathConstants</code>. For <code>Arb</code> there is also a
setter which takes a tuple <code>(a, b)</code> representing an interval and
returns a ball containing this interval.</p>
<p dir="auto">Almost all of the constructors are simple wrappers around these
setters. This means that it's usually more informative to look at the
methods for <code>set!</code> than for say <code>Arb</code> to figure out what constructors
exists. Both <code>Arb</code> and <code>Acb</code> are constructed in such a way that the
result will always enclose the input.</p>

<p dir="auto">Arb is made for rigorous numerics and any functions which do not
produce rigorous results are clearly marked as such. This is not the
case with Julia in general and you therefore have to be careful when
interacting with the ecosystem if you want your results to be
completely rigorous. Below are three things which you have to be extra
careful with.</p>

<p dir="auto">In some cases the implementation in Julia implicitly makes certain
assumptions to improve performance and this can lead to issues. For
example, prior to Julia version 1.8 the <code>minimum</code> and <code>maximum</code>
methods in Julia checked for <code>NaN</code> results (on which is short fuses)
using <code>x == x</code>, which works for most numerical types but not for <code>Arb</code>
(<code>x == x</code> is only true if the radius is zero). See
<a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="638413066" data-permission-text="Title is private" data-url="https://github.com/JuliaLang/julia/issues/36287" data-hovercard-type="issue" data-hovercard-url="/JuliaLang/julia/issues/36287/hovercard" href="https://github.com/JuliaLang/julia/issues/36287">JuliaLang/julia#36287</a> and in particular
<a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="1293936437" data-permission-text="Title is private" data-url="https://github.com/JuliaLang/julia/issues/45932" data-hovercard-type="issue" data-hovercard-url="/JuliaLang/julia/issues/45932/hovercard" href="https://github.com/JuliaLang/julia/issues/45932">JuliaLang/julia#45932</a> for more details.
Since Julia version 1.8 the <code>minimum</code> and <code>maximum</code> methods work
correctly for <code>Arb</code>, for earlier versions of Julia it only works
correctly in some cases.</p>
<p dir="auto">These types of problems are the hardest to find since they are not
clear from the documentation but you have to read the implementation,
<code>@which</code> and <code>@less</code> are your friends in these cases.</p>

<p dir="auto">Arblib extends the methods from
<a href="https://github.com/JuliaMath/SpecialFunctions.jl">SpecialFunctions.jl</a>
with versions from Arb. In some cases the Arb version is more general
than the version in SpecialFunctions, for example <code>ellipk</code> is not
implemented for complex arguments in SpecialFunctions but it is in
Arb. We refer to the Arb documentation for details about the
Arb-versions.</p>
<p dir="auto">Some methods from SpecialFunctions are however not implemented in Arb
and does are not extended, these are mostly scaled version of methods.
Arb does however implement many special functions that are not in
SpecialFunction and at the moment there is no user friendly interface
for most of them.</p>

<p dir="auto">Enabling a threaded version of flint can be done by setting the
environment variable <code>NEMO_THREADED=1</code>. Note that this should be
set before <code>Arblib.jl</code> is loaded. To set the actual number of threads,
use <code>Arblib.flint_set_num_threads($numberofthreads)</code>.</p>

<h1><a id="user-content-softpositjl" class="anchor" aria-hidden="true" href="#softpositjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>SoftPosit.jl</h1>
<p><a href="https://julialang.org/" rel="nofollow">Julia</a> types for the C-based <a href="https://gitlab.com/cerlane/SoftPosit" rel="nofollow">SoftPosit</a> library - a posit arithmetic emulator.</p>
<p>Posit numbers are an alternative to floating-point numbers. Posits extend floats by introducing regime bits, that allow for a higher precision around one, yet a wide dynamic range of representable numbers. For further information see <a href="https://posithub.org" rel="nofollow">https://posithub.org</a></p>
<p>If this library doesn't support a desired functionality or for anything else, please raise an issue.</p>
<h1><a id="user-content-8-16-and-32bit-posit-formats" class="anchor" aria-hidden="true" href="#8-16-and-32bit-posit-formats"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>8, 16 and 32bit posit formats</h1>
<p>SoftPosit.jl emulates the following Posit number formats <code>Posit(n,es)</code>, with <code>n</code> number of bits including <code>es</code> exponent bits: Posit(8,0), Posit(16,1), Posit(32,2) as primitive types called</p>
<pre><code>Posit8, Posit16, Posit32
</code></pre>
<p>following the <a href="https://posithub.org/docs/posit_standard.pdf" rel="nofollow">draft standard</a>. Additionally, the following off-standard formats are defined as primitive types, which are internally stored as 32bit (the remaining bits are kept as zeros): Posit(8,1), Posit(8,2), Posit(16,1), Posit(16,2), Posit(24,1), and Posit(24,2) called <code>Posit8_1</code>, <code>Posit8_2</code>, <code>Posit16_1</code>, <code>Posit16_2</code>, <code>Posit24_1</code>, and <code>Posit24_2</code>.</p>
<p>For all the types <code>Posit8, Posit16, Posit32, Posit8_2, Posit16_2, Posit24_2</code> conversions between Integers and Floats and basic arithmetic operations <code>+</code>, <code>-</code>, <code>*</code>, <code>/</code> and <code>sqrt</code> (among others) are defined. Unfortunately, <code>Posit8_1, Posit16_1, Posit24_1</code> are not yet fully supported by the underlying C library.</p>
<p>To support quires, <code>Quire8</code>, <code>Quire16</code> and <code>Quire32</code> are implemented as 32 / 128 / 512bit types for fused multiply-add and fused multiply-subtract. Additional math functions like <code>exp</code>,<code>log</code>,<code>sin</code>,<code>cos</code>,<code>tan</code> are defined via conversion to <code>Float64</code> (no support yet of the C library) and therefore do not have error-free rounding.</p>

<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> SoftPosit

<span class="pl-c1">Posit16</span>(<span class="pl-c1">0x6000</span>)

<p dir="auto">The primary purpose of this package is to enable saving <code>PlotlyBase.Plot</code> and <code>PlotlyJS.SyncPlot</code> figures to files like svg, png, pdf, etc.</p>
<p dir="auto">This functionality is now built in to PlotlyBase.jl itself -- and thus available to PlotlyJS -- making ORCA.jl obsolete and not needed.</p>
<p dir="auto">Please use the <code>savefig</code> routines built directly in to PlotlyBase instead of using this package</p>

<p dir="auto">Julia interface to plotly's <a href="https://github.com/plotly/orca">ORCA</a> tool for
generating static files based on plotly graph specs.</p>

<p dir="auto">There is a single exported function <code>savefig</code>. After loading PlotlyBase.jl or
PlotlyJS.jl and creating a plot named <code>p</code> you can call
<code>savefig(p::Union{Plot,SyncPlot}, filename::String; options...)</code> to save the
plot in <code>p</code> to a file named <code>filename</code>. See docstring for a description of
<code>options</code>.</p>

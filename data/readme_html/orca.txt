<p><a href="https://travis-ci.org/sglyon/ORCA.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/edf76b669372c28753131eae37ee280ba131af03/68747470733a2f2f7472617669732d63692e6f72672f73676c796f6e2f4f5243412e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/sglyon/ORCA.jl.svg?branch=master" style="max-width:100%;"></a></p>
<p>Julia interface to plotly's <a href="https://github.com/plotly/orca">ORCA</a> tool for
generating static files based on plotly graph specs.</p>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Usage</h2>
<p>There is a single exported function <code>savefig</code>. After loading PlotlyBase.jl or
PlotlyJS.jl and creating a plot named <code>p</code> you can call
<code>savefig(p::Union{Plot,SyncPlot}, filename::String; options...)</code> to save the
plot in <code>p</code> to a file named <code>filename</code>. See docstring for a description of
<code>options</code>.</p>

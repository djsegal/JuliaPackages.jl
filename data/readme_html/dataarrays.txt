<p>THIS PACKAGE IS DEPRECATED with Julia versions above 0.7. Use <code>Array{Union{T, Missing}}</code> instead: see <a href="https://julialang.org/blog/2018/06/missing" rel="nofollow">this blog post</a>.</p>
<p>The DataArrays package provides the <code>DataArray</code> type for working efficiently with <a href="https://en.wikipedia.org/wiki/Missing_data" rel="nofollow">missing data</a>
in Julia, based on the <code>missing</code> value from the <a href="https://github.com/JuliaData/Missings.jl">Missings.jl</a> package.</p>
<p>Most Julian arrays cannot contain <code>missing</code> values: only <code>Array{Union{T, Missing}}</code> and more generally <code>Array{&gt;:Missing}</code> can contain <code>missing</code> values.</p>
<p>The generic use of heterogeneous <code>Array</code> is discouraged in Julia versions below 0.7 because it is inefficient: accessing any value requires dereferencing a pointer. The <code>DataArray</code> type allows one to work around this inefficiency by providing tightly-typed arrays that can contain values of exactly one type, but can also contain <code>missing</code> values.</p>
<p>For example, a <code>DataArray{Int}</code> can contain integers and <code>missing</code> values. We can construct one as follows:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="da = @data([1, 2, missing, 4])
"><pre><code>da = @data([1, 2, missing, 4])
</code></pre></div>
<p>This package used to provide the <code>PooledDataArray</code> type, a variant of <code>DataArray{T}</code> optimized for representing arrays that contain many repetitions of a small number of unique values. <code>PooledDataArray</code> has been deprecated in favor of <a href="https://github.com/JuliaData/CategoricalArrays.jl"><code>CategoricalArray</code></a> or <a href="https://github.com/JuliaComputing/PooledArrays.jl"><code>PooledArray</code></a>.</p>

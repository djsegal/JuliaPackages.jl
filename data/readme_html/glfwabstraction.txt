<p dir="auto"><em>GLFWAbstraction</em> is designed to provide a more Julia-native and partly functional experience to working with the <a href="https://www.glfw.org/" rel="nofollow">GLFW</a> library.</p>

<h2 dir="auto"><a id="user-content-monitors" class="anchor" aria-hidden="true" href="#monitors"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Monitors</h2>
<p dir="auto">Monitors revolve around the <code>Monitor</code> struct retrieved from the <code>monitor</code> function.</p>
<p dir="auto">Use <code>monitor(window::Window)</code> to retrieve the monitor associated with a fullscreen window. If none (i.e. the window is not in fullscreen mode), returns <code>nothing</code>. Use <code>monitor(n::Integer)</code> to retrieve the <code>n</code>-th connected monitor. Usually, you will simply call <code>monitor(1)</code> to assign a window to the primary monitor. A convenient <code>monitor(::Nothing) = nothing</code> exists if needed.</p>
<p dir="auto">Another abstraction exists through the <code>Monitors</code> meta type. Note the plural <em>s</em>. This meta type allows querying information on the collective of connected monitors.</p>
<p dir="auto">Use <code>length(Monitors)</code> to retrieve the total number of connected monitors. You may also use <code>Monitors[index]</code> in place of <code>monitor(index)</code>. You may also <code>collect(Monitors)</code> to retrieve a vector of all currently connected monitors, or <code>iterate(Monitors)</code> which allows usage with a regular <code>for</code> loop:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="for monitor in Monitors

end"><pre><span class="pl-k">for</span> monitor <span class="pl-k">in</span> Monitors

<span class="pl-k">end</span></pre></div>

<p dir="auto">Monitors are associated with one or more <code>VideoMode</code>s. The current video mode of the monitor - either the default desktop video mode or the video mode of its current fullscreen application - can be retrieved through <code>videomode(monitor)</code>. All of its supported video modes can be queried through <code>videomodes(monitor)</code>, respectively.</p>
<p dir="auto">While the <code>VideoMode</code> contains the same information as <code>GLFW.VidMode</code>, it arranges it slightly differently:</p>

<p dir="auto">The centerpiece of GLFW is arguably the windowing system. <em>GLFWAbstraction</em> tries to simplify it as much as possible. Various GLFW constants were wrapped in their own enums. Updating &amp; retrieving window attributes is accomplished through virtual properties. Events are delegated through Julia's native multiple dispatch.</p>

<p dir="auto"><code>Window{ID}</code>s are created through the <code>window</code> factory. Every <code>Window</code> is decorated with an arbitrary <code>ID</code>entifier. These are used to hook into the multiple dispatch based event system. A window can be created as such:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="window(id::Symbol, title::AbstractString, width::Integer, height::Integer, [monitor::Monitor], [share::Window])"><pre><span class="pl-c1">window</span>(id<span class="pl-k">::</span><span class="pl-c1">Symbol</span>, title<span class="pl-k">::</span><span class="pl-c1">AbstractString</span>, width<span class="pl-k">::</span><span class="pl-c1">Integer</span>, height<span class="pl-k">::</span><span class="pl-c1">Integer</span>, [monitor<span class="pl-k">::</span><span class="pl-c1">Monitor</span>], [share<span class="pl-k">::</span><span class="pl-c1">Window</span>])</pre></div>
<p dir="auto">If <code>monitor === nothing</code>, the window will be created in windowed mode. Otherwise, it will be created in fullscreen on the specified window. <code>share</code> may be provided if multiple windows need to share the same OpenGL context. One such use case is spanning multiple monitors in fullscreen with two distinct windows.</p>
<p dir="auto">The <code>title</code> will be displayed in the window's title bar - given it uses border decorations. <code>width</code> and <code>height</code> describe the desired window's drawing size - although the size of the window need not necessarily match the framebuffer's size.</p>
<p dir="auto"><code>id</code> is passed to the <code>Window{id}</code> such that it may be used to uniquely identify your window in the event system described below.</p>

<p dir="auto"><em>Borderless Fullscreen Mode</em> is a special variant of fullscreen mode where the video mode of the window in fullscreen mode matches that of the monitor's video mode in desktop mode. Note that, if another application is already in fullscreen on the queried monitor, its video mode will be retrieved. Currently, there is no way to retrieve the desktop video mode directly from the OS through GLFW.</p>

<p dir="auto">Both keyboard &amp; mouse events can come with <code>ModifierKey</code>s that were active at the moment the key or mouse button was pressed or released. <ul dir="auto">
<li><code>NoMod</code> - no modifier key was held.</li>
<li><code>ShiftMod</code></li>
<li><code>ControlMod</code></li>
<li><code>AltMod</code></li>
<li><code>SuperMod</code></li>
<li><code>CapsLockMod</code></li>
<li><code>NumLockMod</code></li>
</ul>
<p dir="auto">Note that caps lock and num lock may not be usable yet as <em>GLFW.jl</em> is not up-to-date.</p>

<p dir="auto">Keyboard input has been simplified through a more humanized <code>Key</code> wrapper and dynamic dispatch-style events. The <code>Key</code> wrapper provides both more human &amp; programmatic access to all <code>GLFW.KEY_*</code> constants. Its signatures like as thus:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="Key(digit::Integer, numpad::Bool = false)

<p dir="auto">The <code>digit</code> overload retrieves Keys <code>GLFW.KEY_0</code> through <code>GLFW.KEY_9</code>. It throws an <code>ArgumentError</code> if <code>digit âˆ‰ 0:9</code>. If <code>numpad</code> is true, returns the corresponding <code>GLFW.KEY_KP_*</code> instead.</p>
<p dir="auto">The <code>char</code> overload retrieves any key on a standard <em>US QWERTY</em> keyboard which produces a character. As with the <code>digit</code> overload, it throws an <code>ArgumentError</code> if the character is invalid. <strong>Note</strong> that, unfortunately, other languages are not supported by the underlying GLFW library itself.</p>
<p dir="auto">The <code>special</code> overload retrieves any key on a standard <em>US QWERTY</em> keyboard which <em>does not</em> produce a character, such as the <em>escape</em>, <em>print screen</em>, arrow keys, or <em>enter</em> key. Following is a full list of supported special characters:</p>
<ul dir="auto">
<li>Arrow keys (<code>:up</code>, <code>:down</code>, <code>:left</code>, <code>:right</code>)</li>
<li>F keys (<code>:f1</code>, <code>:f2</code>, ..., <code>:f25</code>)</li>
<li>Left/Right <code>shift</code>, <code>control</code>, <code>alt</code>, <code>super</code>, prefixed with <code>:left_</code>/<code>:l</code>/<code>:right_</code>/<code>:r</code> respectively</li>
<li>Numpad keys: <code>:add</code>, <code>:decimal</code>, <code>:divide</code>, <code>:keypad_enter</code>, <code>:keypad_equal</code>, <code>:multiply</code>, <code>:subtract</code></li>
<li><code>:capslock</code>, <code>:end</code>, <code>:enter</code>, <code>:escape</code>/<code>:esc</code>, <code>:home</code>, <code>:menu</code>, <code>:numlock</code>, <code>:pagedown</code>, <code>:pageup</code>, <code>:pause</code>, <code>:print</code>/<code>:printscreen</code>, <code>:scrolllock</code>, <code>:space</code>, <code>:world1</code>, <code>:world2</code></li>
</ul>

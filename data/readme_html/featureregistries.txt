<p dir="auto"><a href="http://lorenzoh.github.io/FeatureRegistries.jl/dev/i" rel="nofollow">Documentation</a></p>
<p dir="auto">A Julia package to create registries of your own package's features, letting your users discover and use them more easily.</p>
<p dir="auto">FeatureRegistries.jl lets you create tables that list package features so that</p>
<ul dir="auto">
<li>users can explore and discover features,</li>
<li>users can search features through a consistent interface; and</li>
<li>developers of third-party packages can transparently add functionality</li>
</ul>
<p dir="auto">But what are "features"? Features can be any group of package functionality with some attributes in common that you want users to be able to discover and use. For example, a registry could store</p>
<ul dir="auto">
<li>datasets with metadata and documentation that a user can download,</li>
<li>machine learning architectures that a user can instantiate; or</li>
<li>a set of remotely stored images</li>
</ul>
<p dir="auto">Read on in the <a href="http://lorenzoh.github.io/FeatureRegistries.jl/dev/i" rel="nofollow">documentation</a> for more information on <a href="docs/using.md">how to use registries</a> and <a href="docs/examples.md">examples of such registries</a>.</p>

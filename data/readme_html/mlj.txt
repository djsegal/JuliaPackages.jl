<p><strong>New to MLJ? Start <a href="https://alan-turing-institute.github.io/MLJ.jl/stable/" rel="nofollow">here</a></strong>.</p>
<p>MLJ (Machine Learning in Julia) is a toolbox written in Julia
providing a common interface and meta-algorithms for selecting,
tuning, evaluating, composing and comparing machine learning models written in Julia and other languages.  MLJ is released
under the MIT licensed and sponsored by the <a href="https://www.turing.ac.uk/" rel="nofollow">Alan Turing
Institute</a>.</p>

<p>MLJ provides access to to a wide variety of machine learning models.
We are always looking for <a href="CONTRIBUTING.md">help</a> adding new models or
test existing ones.  Currently available models are listed below; for
the most up-to-date list, run <code>using MLJ; models()</code>.</p>
<ul>
<li><em>experimental</em>: indicates the package is fairly new and/or is under active development; you can help by testing these packages and making them more robust,</li>
<li><em>medium</em>: indicates the package is fairly mature but may benefit from optimisations and/or extra features; you can help by suggesting either,</li>
<li><em>high</em>: indicates the package is very mature and functionalities are expected to have been fairly optimised and tested.</li>
</ul>

<p>For users of Mac OS using Julia 1.3 or higher, using ScikitLearn
models can lead to unexpected MKL errors due to an issue not related
to MLJ. See
<a href="https://discourse.julialang.org/t/julia-1-3-1-4-on-macos-and-intel-mkl-error/36469/2" rel="nofollow">this Julia Discourse discussion</a>
and
<a href="https://github.com/JuliaPackaging/BinaryBuilder.jl/issues/700">this issue</a>
for context.</p>
<p>A temporary workaround for this issue is to force the installation of
an older version of the <code>OpenSpecFun_jll</code> library. To install an
appropriate version, activate your MLJ environment and run <code>using Pkg; Pkg.develop(PackageSpec(url="https://github.com/tlienart/OpenSpecFun_jll.jl"))</code>.</p>

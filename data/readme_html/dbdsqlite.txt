<p dir="auto">To function, this package needs to have access to a custom build of SQLite3
that has the <code>SQLITE_ENABLE_COLUMN_METADATA</code> compile-time option enabled. The
steps below describe one possible way to enable this option:</p>
<p dir="auto"><strong>Step 1</strong>: Download a copy of the autoconf-ready source code for SQLite3
<a href="https://sqlite.org/download.html" rel="nofollow">here</a>. This package has been tested
against <code>sqlite-autoconf-3080300.tar.gz</code>.</p>
<p dir="auto"><strong>Step 2</strong>: Modify the first line of the <code>sqlite3.c</code> file to include the
following code:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="// For use with DBI.jl, we always build SQLite3 with column metadata enabled
#define SQLITE_ENABLE_COLUMN_METADATA"><pre class="notranslate"><code>// For use with DBI.jl, we always build SQLite3 with column metadata enabled
#define SQLITE_ENABLE_COLUMN_METADATA
</code></pre></div>
<p dir="auto"><strong>Step 3</strong>: Compile and install the modified SQLite3 library:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="./configure
make
make install"><pre class="notranslate"><code>./configure
make
make install
</code></pre></div>
<p dir="auto">On most systems that provide SQLite3, the <code>SQLITE_ENABLE_COLUMN_METADATA</code>
option is not enabled. Under these circumstances, most of the functionality of
this library will work, except for the <code>tableinfo</code> and <code>columninfo</code> functions.</p>

<p>This package manager questions for the <a href="https://moodle.org/" rel="nofollow">Moodle educational
tool</a>.</p>
<p>This package was created by my own usage, so the functionality is initially
reduced. Due to the covid-19, the classrooms are getting more virtual at my
University, and the moodle is getting more usage.</p>
<p>Create Questions in Moodle is a bit tedious, so I have created a import function
from a text file.</p>

<h2><a id="user-content-input-text-file-format" class="anchor" aria-hidden="true" href="#input-text-file-format"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Input text file format</h2>
<p>This package is able to read a text file. The format has been designed to be as
simple and readable as possible.</p>
<pre lang="text"><code>* Category 1

Text of question

- Option 1
+ Option 2
- Option 3
</code></pre>
<p>The sentences starting with <em>*</em> is a new category, with the name.</p>
<p>The sentences without <em>*</em>, <em>+</em>, or <em>-</em> are the text of the question. It is
expected to be from only one line.</p>
<p>The sentences starting with <em>-</em> or <em>+</em> and the different answers for the
previous question. The <em>-</em> means that the answer is false, and the <em>+</em> means
that the sentence is the right answer.</p>
<p>The answers in the question are shuffle, except when one of the word of <em>A</em>,
<em>B</em>, ... is used.</p>
<h2><a id="user-content-export-functionality" class="anchor" aria-hidden="true" href="#export-functionality"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Export functionality</h2>
<p>It is able to export to the MoodleXML format.</p>
<p>This functionality is done by function</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">save_to_moodle</span>(quiz<span class="pl-k">::</span><span class="pl-c1">Quiz</span>, template<span class="pl-k">::</span><span class="pl-c1">AbstractString</span>)</pre></div>
<p>When template is the output filename (with .xml extension).</p>
<p>Actually, due to problem importing in moodle, it creates a XML file for each
category. Thus, if template is "output.xml" and the Quiz has categories "Cat1"
and "Cat2", the output will be "output_Cat1.xml" with the questions of category
<em>Cat1</em> and "output_Cat2.xml" with the questions in category <em>Cat2</em>*.</p>

<p>This package can be used to create a main program to create questions from a
text file. The function could be similar tool</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> MoodleQuestions

<span class="pl-k">function</span> <span class="pl-en">main</span>(ARGS)
    <span class="pl-k">if</span> <span class="pl-c1">length</span>(ARGS)<span class="pl-k">!=</span><span class="pl-c1">2</span>
        <span class="pl-c1">println</span>(stderr, <span class="pl-s"><span class="pl-pds">"</span>usage: textfile outputfile<span class="pl-pds">"</span></span>)

    <span class="pl-k">end</span>

    fname <span class="pl-k">=</span> ARGS[<span class="pl-c1">1</span>]
    foutput <span class="pl-k">=</span> ARGS[<span class="pl-c1">2</span>]

    <span class="pl-k">if</span> <span class="pl-k">!</span><span class="pl-c1">isfile</span>(fname)
        <span class="pl-c1">println</span>(<span class="pl-s"><span class="pl-pds">"</span>Error, the file '<span class="pl-v">$fname</span>' does not exist<span class="pl-pds">"</span></span>)
        <span class="pl-k">return</span>
    <span class="pl-k">end</span>

<p dir="auto">If you are using Linux and Julia version <code>&gt;=1.8.3</code>, the <code>libstdcxx</code> version is not compatible with <code>scikit-learn&gt;=1.2</code>. To get around this issue, you have to swap out the <code>libstdcxx</code> version that is loaded in with Julia. There is two methods to do this. The first is to build an environment with <code>Conda.jl</code> and use that as your <code>LD_LIBRARY_PATH</code>.</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="ROOT_ENV=`julia -e &quot;using Conda; print(Conda.ROOTENV)`
export LD_LIBRARY_PATH=$ROOT_ENV&quot;/lib&quot;:$LD_LIBRARY_PATH"><pre>ROOT_ENV=<span class="pl-s"><span class="pl-pds">`</span>julia -e <span class="pl-s"><span class="pl-pds">"</span>using Conda; print(Conda.ROOTENV)<span class="pl-s"><span class="pl-pds">`</span></span></span></span>
<span class="pl-s"><span class="pl-s"><span class="pl-s"><span class="pl-k">export</span> LD_LIBRARY_PATH=<span class="pl-smi">$ROOT_ENV</span><span class="pl-s"><span class="pl-pds">"</span>/lib<span class="pl-pds">"</span></span>:<span class="pl-smi">$LD_LIBRARY_PATH</span></span></span></span></pre></div>
<p dir="auto">Another method is to link your OS's version of <code>libstdcxx</code> instead of the one that comes with Julia. More details can be found <a href="https://github.com/hhaensel/ReplaceLibstdcxx.jl">here</a>.</p>
<p dir="auto">In both cases, it is recommended to use a Julia version <code>&gt;=1.8.4</code>, but these fixes can be used as a last resort if you have to stay on an older Julia version.</p>

<p><strong>This package was made obsolete by the new <code>Pkg</code> framework of Julia v0.7, which simplifies development for local packages.</strong> It is retired, and unlikely to receive any updates.</p>
<h2><a id="user-content-description" class="anchor" aria-hidden="true" href="#description"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Description</h2>
<p>This package defines some methods for functions in <code>Base.Pkg</code> that should make testing, benchmarking, and locating files in packages outside the default directory (which is returned by <code>Pkg.dir()</code>, eg <code>~/.julia/v0.6</code>) easier.</p>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Usage</h2>
<p>First, load the module:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using RoguePkg
"><pre><span class="pl-k">using</span> RoguePkg</pre></div>
<p>Then you can refer to packages by the module they contain:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="pkg_for&quot;MyFancyModule&quot;
"><pre><span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg_for</span>"</span>MyFancyModule<span class="pl-pds">"</span></span></pre></div>
<p>A path:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="pkg_at&quot;~/this_is/where_I/keep/FancyPkg&quot;
"><pre><span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg_at</span>"</span>~/this_is/where_I/keep/FancyPkg<span class="pl-pds">"</span></span></pre></div>
<p>Or, if you set <code>ENV[JULIA_LOCAL_PACKAGES]</code>, by their subdirectory:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="pkg_local&quot;FancyPkg&quot;
"><pre><span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg_local</span>"</span>FancyPkg<span class="pl-pds">"</span></span></pre></div>
<p>Then you can use the resulting objects with some functions, including <code>Pkg.test</code>, <code>Pkg.dir</code>. The methods for the latter even make <code>PkgBenchmark.benchmarkpkg</code> work. For example,</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="Pkg.test(pkg_for&quot;FancyModule&quot;)
"><pre>Pkg<span class="pl-k">.</span><span class="pl-c1">test</span>(<span class="pl-s"><span class="pl-pds"><span class="pl-c1">pkg_for</span>"</span>FancyModule<span class="pl-pds">"</span></span>)</pre></div>

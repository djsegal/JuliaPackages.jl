<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview</h2>
<p><em>Lexicon</em> is a <a href="http://www.julialang.org" rel="nofollow">Julia</a> package documentation generator
and viewer.</p>
<p>It provides access to the documentation created by the <code>@doc</code> macro from
<a href="https://github.com/MichaelHatherly/Docile.jl"><em>Docile</em></a>. <em>Lexicon</em> allows querying of package documentation from
the Julia REPL and building standalone documentation that can be hosted on GitHub
Pages or <a href="https://readthedocs.org/" rel="nofollow">Read the Docs</a>.</p>
<p><em>Lexicon</em> separates the non-essential parts from <em>Docile</em> so that
package load times are not impacted when documenting packages using
<code>@doc</code>. See <a href="https://github.com/MichaelHatherly/Docile.jl/issues/27">this issue</a> for details regarding the split.</p>

<p dir="auto">As stated many times by the author in his <a href="https://www.youtube.com/watch?v=ENxTrFf9a7c&amp;list=PLDcUM9US4XdNM4Edgs7weiyIguLSToZRI" rel="nofollow">online lectures</a>, StatisticalRethinking is a hands-on course. This project is intended to assist with the hands-on aspect of learning the key ideas in StatisticalRethinking.</p>
<p dir="auto">SR2StanPluto is a Julia project that uses Pluto notebooks for this purpose. Each notebook demonstrates Julia versions of <code>code snippets</code> and <code>mcmc models</code> contained in the R package "rethinking" associated with the book <a href="https://xcelab.net/rm/statistical-rethinking/" rel="nofollow">Statistical Rethinking</a> by Richard McElreath.</p>
<p dir="auto">This Julia project uses Stan (the <code>cmdstan</code> executable) as the underlying mcmc implementation. Please see Stan.jl and/or StanSample.jl for details.</p>

<p dir="auto">From v5 onwards the basis will no longer be StatisticalRethinking.jl but RegressionAndOtherStories.jl. Both packages have very similar content, but StatisticalRethinking.jl uses Plots.jl while RegressionAndOtherStories.jl is using (Cairo)Makie.jl.</p>

<h2 dir="auto"><a id="user-content-usage-details" class="anchor" aria-hidden="true" href="#usage-details"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Usage details</h2>
<p dir="auto">All "rethinking" data files are stored and maintained in StatisticalRethinking.jl and can be accessed via <code>sr_datadir(...)</code>. See <code>notebooks/00-Preface.jl</code> for an example.</p>
<p dir="auto">In scripts, for naming models and results of simulations I tend to use:</p>
<p dir="auto">Models and results:</p>
<ol start="0" dir="auto">
<li>stan5_1           : Stan language program</li>
<li>m5_1s             : The sampled StanSample model</li>
<li>q5_1s             : Stan quap model (NamedTuple similar to Turing)</li>
</ol>
<p dir="auto">Draws:</p>
<ol start="3" dir="auto">
<li>chns5_1s          : MCMCChains object (4000 samples from 4 chains)</li>
<li>part5_1s          : Stan samples (Particles notation)</li>
<li>quap5_1s          : Quap samples (Particles notation)</li>
<li>nt5_1s            : NamedTuple with samples values</li>
<li>ka5_1s            : KeyedArray object (see AxisArrays.jl)</li>
<li>da5_1s            : DimArray object (see DimensionalData.jl)</li>
<li>st5_1s            : StanTable 0bject (see Tables.jl)</li>
<li>i5_1s            : InferenceObjects (see InferenceObjects.jl)</li>
</ol>
<p dir="auto">The default for <code>read_samples(m1_1s)</code> is a StanTable chains object.</p>
<p dir="auto">Results as a DataFrame:</p>
<ol start="10" dir="auto">
<li>prior5_1s_df      : Prior samples (DataFrame)</li>
<li>post5_1s_df       : Posterior samples (DataFrame)</li>
<li>quap5_1s_df       : Quap approximation to posterior samples (DataFrame)</li>
<li>pred5_1s_df       : Posterior predictions (DataFrame)</li>
</ol>
<p dir="auto">The <code>s</code> at the end indicates Stan.</p>
<p dir="auto">By default <code>read_samples(m5_1s)</code> returns a StanTable with the results. In general
it is safer to specify the desired format, i.e. <code>read_samples(m5_1s, :table)</code> as
the Julia eco-sytem is still evolving rapidly with new options.</p>
<p dir="auto">Using <code>read_samples(m5_1s, :...)</code> makes it easy to convert samples to other formats.</p>
<p dir="auto">In version 5 I expect to mainly use the output_formats :dataframe and :namedtuple.</p>
<p dir="auto">For InferenceObjects.jl there is a separate function <code>inferencedata(m1_1s)</code>.
See the Notebook_Examples in Stan.jl for an example Pluto notebook.</p>
<p dir="auto">See the project maintenance section in the Readme of RegressionAndOtherStories.jl for a note on using projects environments instead of Pluto's package management features.</p>

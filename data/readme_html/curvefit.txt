<p dir="auto">A package that implements a few curve fitting functions.</p>
<h2 dir="auto"><a id="user-content-linear-least-squares" class="anchor" aria-hidden="true" href="#linear-least-squares"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Linear Least squares</h2>
<p dir="auto">Linear least square is commonly used technique to find approximation to a discrete
set of data. Given the sets of points <code>x[i]</code> and <code>y[i]</code> and a list of functions <code>f_i(x)</code>
the least squares method finds coefficients <code>a[i]</code> such that</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="a[1]*f_1(x) + a[2]*f_2(x) + ... + a[n]*f_n(x)"><pre class="notranslate"><code>a[1]*f_1(x) + a[2]*f_2(x) + ... + a[n]*f_n(x)
</code></pre></div>
<p dir="auto">minimizes the squares of the errors in relation to <code>y[i]</code>.</p>
<p dir="auto">The basic function is implemented using QR decomposition: <code>A \ y</code>:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="coefs = A \ y"><pre class="notranslate"><code>coefs = A \ y
</code></pre></div>
<p dir="auto">where <code>A[:,i] = f_i(x)</code>. While usually <code>x</code> is a single variable, in general, if several
independent variables are required, the same procedure can be used, something along the line of:
<code>A[:,i] = f_i(x1, x2, ..., xn)</code>.</p>
<p dir="auto">Several typical cases are possible:</p>
<ul dir="auto">
<li><code>linear_fit(x, y)</code> finds coefficients <code>a</code> and <code>b</code> for <code>y[i] = a + b*x[i]</code></li>
<li><code>power_fit(x, y)</code> finds coefficients <code>a</code> and <code>b</code> for <code>y[i] = a *x[i]^b</code></li>
<li><code>log_fit(x, y)</code> finds coefficients <code>a</code> and <code>b</code> for <code>y[i] = a + b*log(x[i])</code></li>
<li><code>exp_fit(x, y)</code> finds coefficients <code>a</code> and <code>b</code> for <code>y[i] = a*exp(b*x[i])</code></li>
<li><code>expsum_fit(x, y, 2, withconst = true)</code> finds coefficients <code>k</code>, <code>p</code>, and <code>λ</code> for <code>y[i] = k + p[1]*exp(λ[1]*x[i]) + p[2]*exp(λ[2]*x[i])</code></li>
<li><code>poly_fit(x, y, n)</code> finds coefficients <code>a[k]</code>  for
<code>y[i] = a[1] + a[2]*x[i] + a[3]*x[i]^2 + a[n+1]*x[i]^n</code></li>
<li><code>linear_king_fit(E, U)</code>, find coefficients <code>a</code> and <code>b</code> for <code>E[i]^2 = a + b*U^0.5</code></li>
<li><code>linear_rational_fit(x, y, p, q)</code> finds the coefficients for rational polynomials: <code>y[i] = (a[1] + a[2]*x[i] + ... + a[p+1]*x[i]^p) / (1 + a[p+1+1]*x[i] + ... + a[p+1+q]*x[i]^q)</code></li>
</ul>
<h2 dir="auto"><a id="user-content-nonlinear-least-squares" class="anchor" aria-hidden="true" href="#nonlinear-least-squares"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Nonlinear least squares</h2>
<p dir="auto">Sometimes the fitting function is non linear with respect to the  fitting coefficients. In this case, given
an approximation of the coefficients, the fitting function is linearized around this
approximation and linear least squares is used to calculate a correction to the approximate coefficients. This iteration is repeated until convergence is
reached. The fitting function has the following form:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="f(x_1, x_2, x_3, ..., x_n, a_1, a_2, ...,  a_p) = 0"><pre class="notranslate"><code>f(x_1, x_2, x_3, ..., x_n, a_1, a_2, ...,  a_p) = 0
</code></pre></div>
<p dir="auto">where <code>xi</code> are the known data points and <code>ai</code> are the coefficients that
should be fitted.</p>
<p dir="auto">When the model formula is not linear on the fitting coefficients, a nonlinear algorithm is necessary. This library implements a Newton-type algorithm that doesn't explicitly need derivatives. This is implemented in the function:</p>
<p dir="auto"><code>coefs, converged, iter = nonlinear_fit(x, fun, a0, eps=1e-7, maxiter=200)</code></p>
<p dir="auto">In this function, <code>x</code> is an array where each column represents a different variable of the data set,
<code>fun</code> is a callable that returns the fitting error and should be callable with the following signature:</p>
<p dir="auto"><code>residual = fun(x, a)</code></p>
<p dir="auto">where <code>x</code> is a vector representing a row of the argument array <code>x</code> and <code>a</code> is an estimate of the
fitting coefficients which should all be different from zero (to provide a scale). <code>eps</code> and <code>maxiter</code>
are convergence parameters.</p>
<p dir="auto">The <code>nonlinear_fit</code> function is used to implement the following fitting functions.</p>
<ul dir="auto">
<li><code>king_fit(E, U)</code> find coefficients <code>a</code>, <code>b</code> and <code>n</code> for <code>E[i]^2 = a + b*U^n</code></li>
<li><code>rational_fit</code> Just like <code>linear_rational_fit</code> but tries to improve the results using nonlinear least squares (<code>nonlinear_fit</code>)</li>
</ul>
<h2 dir="auto"><a id="user-content-generic-interface" class="anchor" aria-hidden="true" href="#generic-interface"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Generic interface</h2>
<p dir="auto">A generic interface was developed to have a common interface for all curve fitting possibilities and to make it easy to use the results:</p>
<p dir="auto"><code>fit = curve_fit(::Type{T}, x, y...)</code></p>
<p dir="auto">where <code>T</code> is the curve fitting type.</p>
<p dir="auto">The following cases are implemented:</p>
<ul dir="auto">
<li><code>curve_fit(LinearFit, x, y)</code></li>
<li><code>curve_fit(LogFit, x, y)</code></li>
<li><code>curve_fit(PowerFit, x, y)</code></li>
<li><code>curve_fit(ExpFit, x, y)</code></li>
<li><code>curve_fit(Polynomial, x, y, n=1)</code></li>
<li><code>curve_fit(LinearKingFit, E, U)</code></li>
<li><code>curve_fit(KingFit, E, U)</code></li>
<li><code>curve_fit(RationalPoly, x, y, p, q)</code></li>
</ul>
<p dir="auto">The <code>curve_fit</code> generic function returns an object that can be use to compute estimates of the model with <code>apply_fit</code>. <code>call</code> is overloaded so that the object can be used as a function.</p>

<h2 dir="auto"><a id="user-content-kings-law" class="anchor" aria-hidden="true" href="#kings-law"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>King's law</h2>
<p dir="auto">In hotwire anemometry, a simple expression for the calibration curve of the probe
is known as King's law, expressed as:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="E^2 = A + B*sqrt(U)"><pre class="notranslate"><code>E^2 = A + B*sqrt(U)
</code></pre></div>
<p dir="auto">where <code>E</code> is voltage on the anemometer bridge, <code>U</code> is the flow velocity.
The coefficients A and B are obtained from a calibration. The function
<code>linear_king_fit</code> estimates coefficients <code>A</code> and <code>B</code>.</p>
<p dir="auto">A better approximation for the calibration curve is known as modified
King's law:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="E^2 = A + B*U^n"><pre class="notranslate"><code>E^2 = A + B*U^n
</code></pre></div>
<p dir="auto">Now, this is a nonlinear curve fit. The linear fit (<code>linear_king_fit</code>) is usually
a very good first guess for the coefficients (where <code>n=0.5</code>). This curve fit is
implemented in function <code>king_fit</code>.</p>

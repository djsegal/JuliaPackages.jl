<h1><a id="user-content-signeddistancefields" class="anchor" aria-hidden="true" href="#signeddistancefields"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>SignedDistanceFields</h1>
<p>This package implements an algorithm for quickly computing signed distance fields in 2D.</p>
<p><code>result = sdf(img)</code>, where <code>img</code> is a 2-dimensional boolean array with <code>true</code> indicating the foreground and <code>false</code> indicating the background.</p>
<p>If you want a downsampled distance field, you can call <code>sdf(img, xsize[, ysize=xsize])</code>, and pass it your desired output dimensions. Right now we only support scaling by integer amounts (e.g. turning a 1024x1024 image into a 64x64 SDF or a 500x500 image into a 125x125 SDF).</p>
<p>There is also an <code>edf</code> function with identical usage that calculates the Euclidean distance transformâ€”the distance from every background pixel to the closest pixel in the foreground.</p>
<p>Signed distance fields are a useful representation for rendering glyphs and other shapes with crisp edges using OpenGL. By allowing the graphics hardware to interpolate between distance samples and testing for the shape boundary in the vertex shader, you can get surprisingly high-resolution output from smaller SDFs.</p>
<p>Here's a <a href="http://www.valvesoftware.com/publications/2007/SIGGRAPH2007_AlphaTestedMagnification.pdf" rel="nofollow">paper from Valve</a> introducing the idea and here's a <a href="https://www.mapbox.com/blog/text-signed-distance-fields/" rel="nofollow">good blog post</a> about using signed distance fields for rendering text.</p>

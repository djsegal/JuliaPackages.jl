<p dir="auto">Currently, this package provides only one function: <code>question</code>. However, there are plans to
support two more important ones, as indicated <a href="https://github.com/udohjeremiah/ProjectEuler.jl/blob/3272257a9046cdf153593690457793233e9ef2c6/src/problemutils.jl#L54C1-L58">here</a>.</p>
<h2 dir="auto"><a id="user-content-question" class="anchor" aria-hidden="true" href="#question"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>question</h2>
<p dir="auto">The <code>question</code> function takes only one argument, an <code>Integer</code> value, and then it returns
a struct of type <code>Problem</code>. The <code>content</code> field of the returned <code>Problem</code> struct is edited to
render it as identical as possible to its appearance on <a href="https://projecteuler.net" rel="nofollow">https://projecteuler.net</a>. An example
is shown below:</p>
<div align="center" dir="auto">
    <a target="_blank" rel="noopener noreferrer" href="assets/problem_example.png"><img src="assets/problem_example.png" style="max-width: 100%;"></a><br><br>
</div>
<p dir="auto">Some problems have not been edited yet because they require an image or a complex
mathematical equation (which, unfortunately, Julia's REPL cannot render at present).
Therefore, passing certain values to <code>question</code> will result in an error.</p>

<p dir="auto">To fully experience all the questions, you may need to increase the width of your terminal
screen to a value equal to or greater than 115. You can do this by dragging either the left
or right end of your terminal.</p>
<p dir="auto">You can use the function <code>displaysize</code> to view the lines (height) and columns (width) of
your terminal screen. The returned tuple represents the lines and columns of your terminal
screen, respectively:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; displaysize(stdout)
(37, 115)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">displaysize</span>(<span class="pl-c1">stdout</span>)
(<span class="pl-c1">37</span>, <span class="pl-c1">115</span>)</pre></div>

<p dir="auto">We welcome contributions to this package! Users are encouraged to contribute by implementing
new features and identifying/fixing bugs through pull requests (PRs) on this GitHub
repository.</p>
<p dir="auto">Currently, there are many unedited problems in the <a href="https://github.com/udohjeremiah/ProjectEuler.jl/blob/main/src/unedited_questions.txt">unedited_questions.txt</a>
file. We invite users with artistic skills in the REPL to take a look at these problems and
see how they can render them in the Julia REPL to look exactly as they do on
<a href="https://projecteuler.net" rel="nofollow">https://projecteuler.net</a>.</p>
<p dir="auto">When contributing, please ensure that your code adheres to the workflow and coding standards
used in this package and that it is thoroughly tested. We also ask that you provide clear
and concise documentation, including examples of how to use the new features.</p>
<p dir="auto">Thank you for considering contributing to this package, and we look forward to your
contributions!</p>

<p dir="auto"><a href="https://travis-ci.org/njwilson23/Iceberg.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/54556ec0d62dae81f511d482abc7ba0a867be3e50bd898701f01b4aca4040f88/68747470733a2f2f7472617669732d63692e6f72672f6e6a77696c736f6e32332f496365626572672e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/njwilson23/Iceberg.jl.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">Performs ice-seawater interface calculations using level set methods. The level
set scheme is similar to that described by Chen, et al. (1997). Model state is
passed around using a subtype of the <code>ModelState</code> abstract type. Physical
constants and parameters are contained in a <code>PhysicalParameters</code> type. See
<code>initialize1d()</code> for an example.</p>
<p dir="auto"><strong>Iceberg is new and under development. It requires a recent build of Julia
(i.e. 0.3).</strong></p>

<p dir="auto">Models in one, two, and three dimensions will be supported. Dimension is
indicated based on the concrete type of the <code>ModelState</code> type passed to
functions. Currently, one-dimensional models are implemented.</p>

<p dir="auto">Methods to solve the heat equation are contained in <code>heat.jl</code>.</p>
<p dir="auto">Interface fluxes can be calculated using the <code>front_velocity()</code> function. Level
set reinitialization uses a global method described by Chen et al. (1997) and
Osher and Fedkiw (2003). In the future, there will be functions to handle level
set function evolution with a hyperbolic equation solver.</p>

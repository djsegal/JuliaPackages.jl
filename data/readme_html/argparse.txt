<p>ArgParse.jl is a package for parsing command-line arguments to
<a href="http://julialang.org" rel="nofollow">Julia</a> programs.</p>
<h3><a id="user-content-installation-and-usage" class="anchor" aria-hidden="true" href="#installation-and-usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation and usage</h3>
<p>To install the module, use Julia's package manager: start pkg mode by pressing <code>]</code> and then enter:</p>
<pre><code>(v1.3) pkg&gt; add ArgParse
</code></pre>
<p>Dependencies will be installed automatically.
The module can then be loaded like any other Julia module:</p>
<pre><code>julia&gt; using ArgParse
</code></pre>

<h3><a id="user-content-breaking-changes" class="anchor" aria-hidden="true" href="#breaking-changes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Breaking changes</h3>
<p>Upgrading from versions 0.2.X to 0.3.X, the following API changes were made,
which may break existing code:</p>
<ul>
<li>Option arguments are no longer evaluated by default. This is for security
reasons. Evaluation can be forced on a per-option basis with the
<code>eval_arg=true</code> setting (although this is discuraged).</li>
<li>The syntax of the <code>add_arg_table</code> function has changed, it now takes a <code>Dict</code>
object instead of an <code>@options</code> opbject, since the dependency on the
Options.jl module was removed. (The <code>@add_arg_table</code> macro is unchanged
though.)</li>

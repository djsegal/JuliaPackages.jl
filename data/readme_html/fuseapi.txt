<p dir="auto">The FUSE Api is a C-library (libfuse3) on Linux and similar systems, which support the FUSE kernel feature.
This Filesystem in User space option allows to implement the internals of a file system in a process, which
runs in user space (not in a kernel module).</p>
<p dir="auto">This package is designed to support the low level operations in Julia.</p>

<p dir="auto">There is only one setup function <code>fuse_main_loop(args, module[, user_data])</code>, which delivers commandline arguments
of the mount process, the name of the implementing module, which defines the callbacks, and an optional user data
object.</p>

<p dir="auto">For each of the (44) callback functions defined by the library, an implementation may be provided as an exported
function of a separate module (named <code>MyFileSystem</code> in this description).</p>
<p dir="auto">The names and signatures are fixed and can be derived from file <a href="https://github.com/KlausC/FuseApi.jl/blob/main/src/fusebridge.jl">fuse-bridge.jl</a>.
The callback functions shall return 0 after calling one of the fuse_reply functions, or a error number, as accessible by <code>Base.UV_E...</code>.
These error numbers are typically directly passed to the system functions which evoked the callback.</p>
<p dir="auto">Each callback function requires a call to a specific reply function in the success case. Which one is documented here:
In the error case, the framework calls a <code>fuse_reply_err(req, errno)</code> from the returned error code.</p>
<p dir="auto">The user data object can be accessed with <code>fuser_user_data(req)</code> from the callback functions, which have the <code>req::FuseReq</code> argument
and from special arguments in <code>init</code> and <code>destroy</code>.</p>

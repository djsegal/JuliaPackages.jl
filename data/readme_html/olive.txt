<h3 dir="auto"><a id="user-content-basic-olive" class="anchor" aria-hidden="true" href="#basic-olive"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>basic olive</h3>
<p dir="auto">When first starting Olive, you will arrive at the <code>setup</code> route. This screen will ask you to select a home directory. After picking your directory, press <code>confirm</code> and you will be greeted with the second portion of setup which will ask you for your name and if you would like to add OliveDefaults. Respond -- you'll definitely want to pick your username! OliveDefaults is an entirely optional extension that will get added -- not loaded -- to your Olive Pkg environment. After confirming, your <code>olive</code> home directory will be setup. This home directory is important because it is used in order to extend Olive. After the setup completes, you will automatically be redirected to your new Olive homepage, also known as <code>explorer</code> at route <code>/</code>. This page requires a key to enter. The directories here will be your olive home and working directories. The home directory will contain a <code>Project.toml</code> file with your Olive data, as well as a module source file for a module called <code>olive</code>.</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="https://github.com/ChifiSource/image_dump/blob/main/olive/olsc/rtjrjrtjrhjgjhjytuj.png"><img src="https://github.com/ChifiSource/image_dump/raw/main/olive/olsc/rtjrjrtjrhjgjhjytuj.png" style="max-width: 100%;"></a></p>
<p dir="auto">This is where extensions and changes can be written on top of olive from the outside in. Double clicking a file in the directory will yield a loading of the <code>/session</code> route. This route contains the actual editor, as well as the project explorer. The project explorer can be opened by clicking the top left. Client settings can be altered by clicking the top right settings icon. These settings are saved whenever the settings menu is closed.</p>
<h5 dir="auto"><a id="user-content-cells" class="anchor" aria-hidden="true" href="#cells"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>cells</h5>
<p dir="auto">There are several types of cells that come with Olive, and adding cells is as easy as using modules. The main cell that one will interact with in this regard is the <code>code</code> cell. This is your standard Julia input and output. Pressing <code>;</code> in a <code>code</code> cell will yield a <code>shell</code> cell, pressing <code>]</code> will yield a <code>pkgrepl</code> cell, and <code>?</code> will yield a <code>helprepl</code> cell. It should also be noted that <code>#=TODO</code> and <code>#=NOTE</code> also create their own respective comment cells. The default cell bindings are as follows</p>
<ul dir="auto">
<li><code>ctrl</code> + <code>shift</code> + <code>Enter</code> runs current cell.</li>
<li><code>ctrl</code> + <code>shift</code> + <code>Q</code> Adds new creator cell.</li>
<li><code>ctrl</code> + <code>shift</code> + <code>Delete</code> Deletes the current cell</li>
<li><code>ctrl</code> + <code>shift</code> + <code>ArrowUp</code> Moves the current cell up</li>
<li><code>ctrl</code> + <code>shift</code> + <code>ArrowDown</code> moves cells down.</li>
</ul>
<p dir="auto">These bindings are editable inside of the settings menu. <code>creator</code> cells are used to create new cells with creator keys. These bindings may also be changed inside of the settings menu, and allow you to create many different cell types with different key-bindings for the <code>creator</code> cell.</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="https://github.com/ChifiSource/image_dump/blob/main/olive/olsc/eththethehhethh.png?raw=true"><img src="https://github.com/ChifiSource/image_dump/raw/main/olive/olsc/eththethehhethh.png?raw=true" style="max-width: 100%;"></a></p>
<p dir="auto">The repl cells include <code>pkgrepl</code>, <code>helprepl</code>, and <code>shell</code>. These three cells are all accessible from a <code>code</code> cell, or can be created on their own from a creator cell. The <code>pkgrepl</code> cell takes simple Pkg commands, and is meant to work pretty similarly to its Julia equivalent. The same can be said for the <code>shell</code> REPL cell. The <code>helprepl</code> will take a name of something you want documentation for. There are more plans for these cells in the future, and many bugs with them that are going to need to be ironed out. The final cell type that olive includes is the <code>tomlvalues</code> cell. This is a cell which can evaluate TOML into a dictionary, and can also be written to Julia or TOML.</p>

<p dir="auto">Projects are what holds your olive session together. Whenever a project is built, it will present itself in your session as a window with a tab. Clicking the tab will yield some controls, these are... (from left to right)</p>
<ul dir="auto">
<li>collapse</li>
<li>save</li>
<li>save as</li>
<li>new</li>
<li>resource</li>
<li>run all</li>
<li>close</li>
</ul>
<p dir="auto">This is the portion of Olive that is the most under development, so there is not much to say -- but these are definitely something to be aware of.</p>

<p dir="auto">Directories are the final piece of the puzzle. When arriving at the main explorer screen, you will be greeted with two different directories: these are your <code>olive</code> home directory and project, and then your working directory. These directories contain file cells inside of them, as well as some controls to make new files or folders inside of the tab for the cells. Double clicking a file will open that file inside of session.</p>

<p dir="auto">If there is one thing that Olive has built its methodology on, this would be extensibility. The primary extension focus for <code>0.0.8</code> is on cells, but there will be even more extension capabilities coming to Olive in the near future. Projects and Directories are both able to be extended, there just is not much convenience to doing so at the moment -- so this is currently not recommended. If you would like to attempt to create your own extensions with this current iteration of Olive, it is sensible to look into the documentation for</p>
<ul dir="auto">
<li><code>build</code></li>
<li><code>evaluate</code></li>
<li><code>build_base_cell</code></li>
<li><code>cell_bind!</code></li>
<li><code>olive_save</code></li>
<li><code>cell_highlight!</code></li>
</ul>

<p dir="auto">Olive has been built with deployability in mind, but we should keep in mind that <code>0.0.8</code> is still an early version of Olive. It is not recommended to deploy this version as there is insufficient memory management and permission management on the back-end. This cuurrent form of Olive is certainly recommended to be used personally, and locally, but this is set to change by <code>0.1.0</code> :)</p>

<p dir="auto"><strong>toolips packages</strong></p>
<ul dir="auto">
<li><a href="https://github.com/ChifiSource/Toolips.jl">Toolips</a> - Base web-development framework.</li>

<li><a href="https://github.com/ChifiSource/ToolipsMarkdown.jl">ToolipsMarkdown</a> - Markdown interpolation, syntax highlighting.</li>
<li><a href="https://github.com/ChifiSource/ToolipsDefaults.jl">ToolipsDefaults</a> - Default Components.</li>
<li><a href="https://github.com/ChifiSource/ToolipsBase64.jl">ToolipsBase64</a> - Image types into Components -- for Olive display.</li>
</ul>

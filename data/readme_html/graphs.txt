<p dir="auto">The goal of <em>Graphs.jl</em> is to offer a performant platform for network and graph analysis in Julia, following the example of libraries such as <a href="http://networkx.github.io" rel="nofollow">NetworkX</a> in Python. To this end, <em>Graphs.jl</em> offers:</p>
<ul dir="auto">
<li>a set of simple, concrete graph implementations -- <code>SimpleGraph</code> (for undirected graphs) and <code>SimpleDiGraph</code> (for directed graphs)</li>
<li>an API for the development of more sophisticated graph implementations under the <code>AbstractGraph</code> type</li>
<li>a large collection of graph algorithms with the same requirements as this API.</li>
</ul>

<p dir="auto"><em>Graphs.jl</em> includes numerous convenience functions for generating graphs, such as <code>path_graph</code>, which builds a simple undirected <a href="https://en.wikipedia.org/wiki/Path_graph" rel="nofollow">path graph</a> of a given length. Once created, these graphs can be easily interrogated and modified.</p>

<p dir="auto">The <em>Graphs.jl</em> project is a reboot of the <em>LightGraphs.jl</em> package (archived in October 2021), which remains available on GitHub at <a href="https://github.com/sbromberger/LightGraphs.jl">sbromberger/LightGraphs.jl</a>. If you don't need any new features developed since the fork, you can continue to use older versions of <em>LightGraphs.jl</em> indefinitely. New versions will be released here using the name <em>Graphs.jl</em> instead of <em>LightGraphs.jl</em>. There was an older package also called <em>Graphs.jl</em>. The source history and versions are still available in this repository, but the current code base is unrelated to the old <em>Graphs.jl</em> code and is derived purely from <em>LightGraphs.jl</em>. To access the history of the old <em>Graphs.jl</em> code, you can start from <a href="https://github.com/JuliaGraphs/Graphs.jl/commit/9a2501948053f60c630caf9d4fb257e689629041">commit 9a25019</a>.</p>
<h3 dir="auto"><a id="user-content-transition-from-lightgraphs-to-graphs" class="anchor" aria-hidden="true" href="#transition-from-lightgraphs-to-graphs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Transition from LightGraphs to Graphs</h3>
<p dir="auto"><em>LightGraphs.jl</em> and <em>Graphs.jl</em> are functionally identical, still there are some steps involved making the change:</p>
<ul dir="auto">
<li>Change <code>LightGraphs = "093fc24a-ae57-5d10-9952-331d41423f4d"</code> to <code>Graphs = "86223c79-3864-5bf0-83f7-82e725a168b6"</code> in your Project.toml.</li>
<li>Update your <code>using</code> and <code>import</code> statements.</li>
<li>Update your type constraints and other references to <code>LightGraphs</code> to <code>Graphs</code>.</li>
<li>Increment your version number. Following semantic versioning, we suggest a patch release when no graphs or other <code>Graphs.jl</code>-objects can be passed through the API of your package by those depending on it, otherwise consider it a breaking release. "Passed through" entails created outside and consumed inside your package and vice versa.</li>
<li>Tag a release.</li>
</ul>

<ul dir="auto">
<li>The master branch of <em>Graphs.jl</em> is generally designed to work with versions of Julia starting from the <a href="https://julialang.org/downloads/#long_term_support_release" rel="nofollow">LTS release</a> all the way to the <a href="https://julialang.org/downloads/#current_stable_release" rel="nofollow">current stable release</a>, except during Julia version increments as we transition to the new version.</li>
<li>Later versions: Some functionality might not work with prerelease / unstable / nightly versions of Julia. If you run into a problem, please file an issue.</li>
<li>The project was previously developed under the name <em>LightGraphs.jl</em> and older versions of <em>LightGraphs.jl</em> (≤ v1.3.5) must still be used with that name.</li>
<li>There was also an older package also called <em>Graphs.jl</em> (git tags <code>v0.2.5</code> through <code>v0.10.3</code>), but the current code base here is a fork of <em>LightGraphs.jl</em> v1.3.5.</li>
<li>All older <em>LightGraphs.jl</em> versions are tagged using the naming scheme <code>lg-vX.Y.Z</code> rather than plain <code>vX.Y.Z</code>, which is used for old <em>Graphs.jl</em> versions (≤ v0.10) and newer versions derived from <em>LightGraphs.jl</em> but released with the <em>Graphs.jl</em> name (≥ v1.4).</li>
<li>If you are using a version of Julia prior to 1.x, then you should use <em>LightGraphs.jl</em> at <code>lg-v.12.*</code> or <em>Graphs.jl</em> at <code>v0.10.3</code></li>
</ul>

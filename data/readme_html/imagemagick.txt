<p>This package provides a wrapper around
<a href="http://www.imagemagick.org/" rel="nofollow">ImageMagick</a> version 6.  It was split off from
<a href="https://github.com/timholy/Images.jl">Images.jl</a> to make image I/O more
modular.</p>

<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Usage</h2>
<p>ImageMagick will be used as needed if you've said</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> FileIO</pre></div>
<p>in your session or module. You should <strong>not</strong> generally say <code>using ImageMagick</code>.  See <a href="https://github.com/JuliaIO/FileIO.jl">FileIO</a> for
further details.</p>
<p>It's worth pointing out that packages such as <a href="https://github.com/JuliaImages/Images.jl">Images.jl</a> load FileIO for you.</p>
<p>Loading an image is then as simple as</p>
<div class="highlight highlight-source-julia"><pre>img <span class="pl-k">=</span> <span class="pl-c1">load</span>(filename[; view<span class="pl-k">=</span><span class="pl-c1">false</span>])</pre></div>
<p>Set <code>view=true</code> to reduce memory consumption when loading large files, possibly
at some slight cost in terms of performance of future operations.</p>

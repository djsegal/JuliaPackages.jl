<p dir="auto">Implementation in <a href="https://julialang.org/" rel="nofollow">Julia</a> of the
<a href="https://github.com/astropy/reproject"><code>reproject</code></a> package by Thomas
Robitaille, part of the Astropy project.</p>
<p dir="auto">This package can be used to reproject Astronomical Images from one world coordinate to another. By reproject we mean re-gridding of images using interpolation (i.e changing the pixel resolution, orientation, coordinate system).</p>

<p dir="auto"><code>Reproject.jl</code> is avilable for Julia 1.0 and later versions and can be installed with <a href="https://docs.julialang.org/en/v1/stdlib/Pkg/" rel="nofollow">Julia built-in package manager</a>.

<h2 dir="auto"><a id="user-content-reprojecting-images" class="anchor" aria-hidden="true" href="#reprojecting-images"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Reprojecting Images</h2>
<p dir="auto">To reproject Astronomical Images, primary requirements are Image data (2D Matrix), world cordinate frame of input Image and required output frame in which it needs to be reprojected.</p>
<p dir="auto">The Image data and input frame is given together as an <a href="http://juliaastro.github.io/FITSIO.jl/latest/index.html" rel="nofollow">ImageHDU</a> or <a href="https://github.com/JuliaAstro/FITSIO.jl">FITS</a> file or name of the FITS file in <code>input_data</code>. A keyword argument <code>hdu_in</code> can be given while using FITS or FITS file name to specify specific HDU in FITS file.</p>
<p dir="auto">The <code>output_projection</code> is the output world coordinate frame and needs to be a a <a href="https://github.com/JuliaAstro/WCS.jl">WCSTransform</a> or an <a href="http://juliaastro.github.io/FITSIO.jl/latest/index.html" rel="nofollow">ImageHDU</a> or <a href="https://github.com/JuliaAstro/FITSIO.jl">FITS</a> file or name of the FITS file. A keyword argument <code>hdu_out</code> can be given while using FITS or FITS file name to specify specific HDU in FITS file.
WCS information is extracted from header when ImageHDU or FITS file is given as <code>output_projection</code>.</p>
<p dir="auto">Order of Interpolation can be specified by keyword <code>order</code> (i.e 0, 1(default), 2).
The dimensions of output Image can be given by keyword <code>shape_out</code>. This can be used to change resolution.</p>

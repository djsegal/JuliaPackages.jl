<p dir="auto">This package provides lightweight methods for mesh I/O.
The input interface is based on the Python project <a href="https://github.com/nschloe/meshio">meshio</a>.
The supported formats include:</p>
<blockquote>
<p dir="auto"><a href="http://abaqus.software.polimi.it/v6.14/index.html" rel="nofollow">Abaqus</a> (<code>.inp</code>),
ANSYS msh (<code>.msh</code>),
<a href="https://lanl.github.io/LaGriT/pages/docs/read_avs.html" rel="nofollow">AVS-UCD</a> (<code>.avs</code>),
<a href="https://cgns.github.io/" rel="nofollow">CGNS</a> (<code>.cgns</code>),
<a href="https://manpages.ubuntu.com/manpages/disco/man1/dolfin-convert.1.html" rel="nofollow">DOLFIN XML</a> (<code>.xml</code>),
<a href="https://nschloe.github.io/meshio/exodus.pdf" rel="nofollow">Exodus</a> (<code>.e</code>, <code>.exo</code>),
<a href="https://www.itascacg.com/software/flac3d" rel="nofollow">FLAC3D</a> (<code>.f3grid</code>),
<a href="https://www.mcs.anl.gov/~fathom/moab-docs/h5mmain.html" rel="nofollow">H5M</a> (<code>.h5m</code>),
<a href="https://github.com/KratosMultiphysics/Kratos/wiki/Input-data">Kratos/MDPA</a> (<code>.mdpa</code>),

<p dir="auto">FiniteMesh.jl is a registered package in the official Julia package registry.
We recommend installing it with the built-in Julia package manager.
From the Julia REPL, you can get in the package manager (by pressing ]) and add the package.
This will automatically install the package and all its dependencies.</p>

<p dir="auto">To read a mesh, simply do</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using FiniteMesh
cells, points = read_mesh(&quot;path-of-mesh-file&quot;)"><pre><span class="pl-k">using</span> FiniteMesh
cells, points <span class="pl-k">=</span> <span class="pl-c1">read_mesh</span>(<span class="pl-s"><span class="pl-pds">"</span>path-of-mesh-file<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">The resulted <code>points</code> are the coordinates of nodes, and <code>cells</code> provides the affiliation information of these points to cell IDs.</p>
<p dir="auto">The connectivity calculator is provided with native Julia.
The following information can be inferred from <code>cells</code> and <code>points</code>:</p>
<ul dir="auto">
<li>Node IDs of a face</li>
<li>Adjacent cell IDs of a face</li>
<li>Neighbor cell IDs of a cell</li>
<li>Cell type (inner / boundary)</li>
<li>Cell volume</li>
<li>Face area</li>
<li>Cell center location</li>
<li>Face center location</li>
<li>Unit normal vector of a face</li>
</ul>
<p dir="auto">For convenience, a <code>Mesh</code> struct is defined to contain all the above information as its fields.
It can be constructed simply via</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="mesh = Mesh(&quot;path-of-mesh-file&quot;)"><pre>mesh <span class="pl-k">=</span> <span class="pl-c1">Mesh</span>(<span class="pl-s"><span class="pl-pds">"</span>path-of-mesh-file<span class="pl-pds">"</span></span>)</pre></div>

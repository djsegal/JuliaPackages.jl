<br>
<p dir="auto">Packages in the Julia standard library are by default faded-out.
To remove them entirely from the graph, set the keyword argument
<code>stdlib = false</code>.</p>
<p dir="auto">Similarly, you can filter out binary dependencies (<a href="https://docs.binarybuilder.org/stable/jll" rel="nofollow">JLL packages</a>)
by setting <code>jll = false</code>.</p>

<p dir="auto">PkgGraph is available in the Julia general registry and can be installed <a href="https://pkgdocs.julialang.org/v1/getting-started" rel="nofollow">as usual</a> with</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="pkg&gt; add PkgGraph"><pre class="notranslate"><code>pkg&gt; add PkgGraph
</code></pre></div>
<p dir="auto">It supports Julia versions down to 1.6</p>
<h3 dir="auto"><a id="user-content-global-install" class="anchor" aria-hidden="true" href="#global-install"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Global Install</h3>
<p dir="auto">You might want to install <code>PkgGraph</code> in your base environment (e.g. <code>v1.9</code>).<br>
You can then use it in any project, without having to install it in that project
or having to switch projects.</p>
<details>
<p dir="auto">You can activate your base environment using <code>] activate</code> (i.e. activate 'nothing'),
and then <code>add PkgGraph</code> there.</p>
<p dir="auto">Another way to obtain a global install is to run – from within <em>any</em> environment:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; using PkgGraph"><pre class="notranslate"><code>julia&gt; using PkgGraph
</code></pre></div>
<p dir="auto">If the package is not found, Julia will offer to install it.<br>
Type '<code>o</code>' to choose to install it in your base environment.</p>
</details>

<p dir="auto">Check out the code for development using</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="pkg&gt; dev PkgGraph"><pre class="notranslate"><code>pkg&gt; dev PkgGraph
</code></pre></div>
<p dir="auto">One fun development method is to open one of the <a href="src"><code>src/</code></a> files
in, e.g, VS Code, and treat is as a script, breaking open functions
and executing individual lines in the <a href="https://www.julia-vscode.org/docs/stable/userguide/runningcode/#The-Julia-REPL" rel="nofollow">integrated Julia REPL</a>.
(Dummy input data can be copy-pasted in from the corresponding file
in <a href="test"><code>test/</code></a>).</p>
<p dir="auto">See the readmes <a href="test/ReadMe.md">in <code>test/</code></a> and <a href="docs/ReadMe.md">in <code>docs/</code></a>
for how to locally run the tests and build the documentation.</p>
<p dir="auto">See the <a href="https://tfiers.github.io/PkgGraph.jl/dev/devguide" rel="nofollow">Developer Guide</a> in the documentation for more.</p>

<p dir="auto">Sometimes, we use numeric quantities for pure representational purposes without any
true numeric significance. For example, the page numbering is carried out using
simple Arabic numerals, roman numerals, alphabets. These numbers have additive
properties but may not have any multiplicative significance. Such numbers can have
prefix notations as well. In some cases, the representation can be in upper case
or lower case as well. This package implements such numerals. The interface has a
similar interface as the RomanNumerals package.</p>

<p dir="auto">LabelNumerals introduces the following new types:</p>
<ol dir="auto">
<li>LetterNumeral - A, B, ..., Z, AA, BB, ..., ZZ, AAA... (group of 26 characters each)</li>
<li>ArabicNumeral - 1, 2, 3, ...</li>
<li>LookupNumeral - Mapped strings to integers like English words "One", "Two" etc.</li>
<li>AlphaNumNumeral - Alphabets representing numbers like BA, BB, BC etc.</li>
</ol>
<p dir="auto">It also supports <code>RomanNumeral</code> from
<a href="https://github.com/anthonyclays/RomanNumerals.jl"><code>RomanNumerals</code></a> package.</p>
<p dir="auto">They support ability to provide a string prefix. The prefix does not get incremented as
numbers are incremented.</p>

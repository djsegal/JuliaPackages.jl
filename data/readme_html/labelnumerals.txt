<p>Numeric quantities are used sometimes for pure representational purposes without any true a
numeric significance. For example, the page numbering is carried out using simple arabic
numerals, roman numerals, alphabets. These numbers have additive properties but may not
have any multiplicative significance. Such numbers can also have prefix notations as well.
In some cases, the representation can be in upper case or lower case as well. This package
implements such a numerals. Such numeric schemes are used as page numbers in PDF file
specification. However, the need may be felt else where as well, which prompted the author
to implement it as an independent package. The interface has been also influenced
significantly by the <a href="https://github.com/anthonyclays/RomanNumerals.jl">RomanNumerals</a> package.</p>

<p>LabelNumerals introduces the following new types:</p>
<ol>
<li>LetterNumeral - A, B, ..., Z, AA, BB, ..., ZZ, AAA... (group of 26 characters each)</li>
<li>ArabicNumeral - 1, 2, 3, ...</li>
<li>LookupNumeral - Mapped strings to integers like English words "One", "Two" etc.</li>
<li>AlphaNumNumeral - Alphabets representing numbers like BA, BB, BC etc.</li>
</ol>
<p>It also supports <code>RomanNumeral</code> from
<a href="https://github.com/anthonyclays/RomanNumerals.jl"><code>RomanNumerals</code></a> package.</p>
<p>They support ability to provide a string prefix. The prefix does not get incremented as
numbers are incremented.</p>

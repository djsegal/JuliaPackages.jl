<h2><a id="user-content-what-are-portraits" class="anchor" aria-hidden="true" href="#what-are-portraits"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>What are portraits?</h2>
<p>To visualize a function f: ℂ → ℂ these steps are done:</p>
<ol>
<li>Coloring of the codomain/target set: every w ∊ ℂ gets a color, lets call it C(w).</li>
<li>To get the color at z ∊ ℂ (where f is defined) one calculates w = f(z) and</li>
<li>the color at z is computed using the map C∘f, i.e. the color at the point z is given by the C(f(z)).</li>
</ol>
<p>Here is an example for f(z) = z².</p>
<p align="center">
<a target="_blank" rel="noopener noreferrer" href="https://github.com/luchr/ComplexPortraitsImages/blob/master/images/docintro.png"><img alt="z^2 example" src="https://github.com/luchr/ComplexPortraitsImages/raw/master/images/docintro.png" style="max-width:100%;"></a> 
</p>
<p>For more flexibility in this package the color at z may also depend on z itself: color at z = CS(z, f(z)). Such a function is called <em>color scheme</em>.</p>
<p>This process is called <a href="https://en.wikipedia.org/wiki/Domain_coloring" rel="nofollow">domain coloring</a>.
There are many different ways to color the w-plane. Often the color is primarily determined by the angle/phase of w. In such a situation this is also called a (complex) phase portrait. A lot of properties of f can be directly seen in or deduced from such a phase portrait.</p>
<p>For a lot more details, see the book <a href="http://www.mathe.tu-freiberg.de/ana/mitarbeiter/elias-wegert/forschung/publikationen/visual-complex-functions" rel="nofollow">"Visual Complex Functions" by Elias Wegert, Birkhäuser, 2012</a>.</p>
<h2><a id="user-content-phase-colors-and-colormaps" class="anchor" aria-hidden="true" href="#phase-colors-and-colormaps"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Phase colors and colormaps</h2>
<p>Some of the ideas of Elias Wegert's <a href="https://www.mathworks.com/matlabcentral/fileexchange/45464-complex-function-explorer" rel="nofollow">PhasePlot package/Complex Function Explorer for Matlab</a> are implemented here for Julia. Especially the following one-lettter-colorschemes are also available in ComplexPortraits: c, d, e, j, m, p, q. Hence a copy of the BSD-licensed Complex Funktion Explorer source code can be found in the <a href="./based_on/">based_on</a> directory.</p>
<p>Additionally some other color schemes are implemented. Please see the list of examples below.</p>
<p>The basis for many color schemes are the colormaps for the angles/phases. There exist two typical choices (color wheels):</p>
<p>HSV wheel: <a target="_blank" rel="noopener noreferrer" href="https://github.com/luchr/ComplexPortraitsImages/blob/master/images/wheel01.png"><img alt="color wheel" align="center" width="200px" src="https://github.com/luchr/ComplexPortraitsImages/raw/master/images/wheel01.png" style="max-width:100%;"></a> and
<a href="https://dlmf.nist.gov/help/vrml/aboutcolor" rel="nofollow">NIST wheel</a>: <a target="_blank" rel="noopener noreferrer" href="https://github.com/luchr/ComplexPortraitsImages/blob/master/images/wheel02.png"><img alt="NIST color wheel" align="center" width="200px" src="https://github.com/luchr/ComplexPortraitsImages/raw/master/images/wheel02.png" style="max-width:100%;"></a></p>
<p>Of course you can define and use your own colormap.</p>

<h2><a id="user-content-creating-viewing-and-saving-portraits" class="anchor" aria-hidden="true" href="#creating-viewing-and-saving-portraits"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Creating, viewing and saving portraits</h2>
<p>The main function is <code>portrait</code>:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="function portrait(z_upperleft, z_lowerright, f;
                  no_pixels=(600,600), point_color=cs_j())
"><pre><span class="pl-k">function</span> <span class="pl-en">portrait</span>(z_upperleft, z_lowerright, f;
                  no_pixels<span class="pl-k">=</span>(<span class="pl-c1">600</span>,<span class="pl-c1">600</span>), point_color<span class="pl-k">=</span><span class="pl-c1">cs_j</span>())</pre></div>
<p>Two complex numbers are needed: the upper left and lower right corner of the rectangular domain which should be colored. <code>f</code> is the complex function.</p>
<p>The other arguments are optional: With <code>no_pixels</code> the size of the output image (number of pixels) can be given. <code>point_color</code> is a color scheme function. A color scheme function has the form</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="mycolorscheme(z, fz) =  ... # computation of color depending on z and f(z)
"><pre><span class="pl-en">mycolorscheme</span>(z, fz) <span class="pl-k">=</span>  <span class="pl-k">...</span> <span class="pl-c"><span class="pl-c">#</span> computation of color depending on z and f(z)</span></pre></div>
<p>It takes two scalar complex numbers (typically of type <code>Complex{Float64}</code>) and returns a color, see <a href="https://github.com/JuliaGraphics/ColorTypes.jl">ColorTypes</a>.</p>
<p>Please see below for a list of predefined parameterized color schemes. All <code>cs_foobar(baz)</code>
functions in the package are function generators, i.e. they use the given parameters
to generate and return a color scheme function. For example:
<code>cs_e(phaseres=10)</code> uses the <code>phaseres</code> argument to create and return a color scheme
function with the given phase-resolution.</p>
<p>The output of <code>portrait</code> is a matrix of color entries (a.k.a. image).</p>
<p>There are many ways to save such an image, e.g. using <a href="https://github.com/JuliaIO/FileIO.jl">FileIO</a> and <a href="https://github.com/JuliaIO/ImageMagick.jl">ImageMagick</a>:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using ComplexPortraits

"><pre><span class="pl-k">using</span> ComplexPortraits

<p>To directly view such an image in Jupyter, Atom, etc. (or in an extra window)
and/or process the image in other ways, please
see and use the <a href="https://github.com/JuliaImages">JuliaImages ecosystem</a>.</p>

<p>For all the examples the function <code>z -&gt; (z-1)/(z*z + z + 1)</code> is used. It
has two simple poles, one root and one saddle point.</p>
<h3><a id="user-content-cs_grid" class="anchor" aria-hidden="true" href="#cs_grid"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>cs_grid...</h3>
<p>For the color schemes in this section the 1-periodic "spike function"
<code>x -&gt; 1 - exp(a*abs(mod(x,1)-0.5)^b)</code> with two parameters a and b is

<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="cs_stripes(directions, colors)
"><pre><span class="pl-c1">cs_stripes</span>(directions, colors)</pre></div>
<p>Directions is a vector of complex numbers.
For each such number d in directions the stripes are perpendicular
to (real(d), imag(d)). And abs(d) is the number of stripes per unit
length.
This devides the complex plane in two sets Sd and ℂ\Sd.</p>
<p>Each z ∊ ℂ has n=length(directions) flags (z ∊ S1, z ∊ S1, ..., z ∊ Sn).
This make 2^n possible flag-combinations. For every combination there
need to be a color in colors.</p>
<p><code>cs_stripes([5.0 + 0.0im], [HSV(0.0, 0.0, 1.0), HSV(0.0, 0.0, 0.0)]))</code></p>

<ol dir="auto">
<li>
<p dir="auto">If you need to store huge datasets that cannot fit uncompressed in memory consider using <a href="https://github.com/JuliaIO/HDF5.jl">https://github.com/JuliaIO/HDF5.jl</a> or <a href="https://github.com/JuliaIO/Zarr.jl">https://github.com/JuliaIO/Zarr.jl</a></p>
</li>
<li>
<p dir="auto">If you just want to serialize arbitrary Julia data consider using <a href="https://github.com/JuliaIO/JLD2.jl">https://github.com/JuliaIO/JLD2.jl</a> or <a href="https://github.com/invenia/JLSO.jl">https://github.com/invenia/JLSO.jl</a></p>
</li>
<li>
<p dir="auto">Numpy datetime64 (“M”) and timedelta64 (“m”) data types are read as Int64.</p>
</li>
<li>
<p dir="auto">Zarr filters are not supported.</p>
</li>
</ol>

<ol dir="auto">
<li><code>ZGroup</code> represents a tree with arrays as leaves.</li>
<li><code>ZGroup</code> leaf arrays are uncompressed but store metadata about how they should be compressed when saved to disk.</li>
<li><code>ZGroup</code> can have JSON3 serializable attributes attached to any node or leaf.</li>
<li>Data can be quickly accessed and modified in <code>ZGroup</code>.</li>
<li>No file open close semantics. Use the Julia garbage collector to clean memory up.</li>
<li>Save and load <code>ZGroup</code> in a directory or zip file in <a href="https://zarr.readthedocs.io/en/stable/spec/v2.html#" rel="nofollow">Zarr v2 format</a>.</li>
<li><code>ZGroup</code> saved to a directory or zip file can be read in other languages.</li>
</ol>

<p dir="auto">C data structures are different from Julia structures in that they do not keep type information.
While most primitive types and bitstype structures of Julia have an identical memory layout as the corresponding
C data, complexities arise when pointers and strings are embedded.</p>
<p dir="auto">This package handles the situation, when pointered C-data generated in C have to be accessed in Julia code.
It is also possible in Julia to construct a byte array, which can be processed by a C program given the
layout of the C-structures.</p>
<p dir="auto">This package was fundamental to implementing the <a href="https://github.com/KlausC/FuseApi.jl">FuseApi</a> package.</p>

<h2 dir="auto"><a id="user-content-layout-elements" class="anchor" aria-hidden="true" href="#layout-elements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Layout Elements</h2>
<p dir="auto">A data layout is described by a Julia bitstype struct, a fixed or variable vector descriptor, a <code>Cstring</code>, or a
reference descriptor.</p>
<h3 dir="auto"><a id="user-content-bits-types" class="anchor" aria-hidden="true" href="#bits-types"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Bits Types</h3>
<p dir="auto">All primitve types defined in Julia, which have an identical C-representation can be used as layout descriptions.
That includes immutable structs of such types, which have the <code>isbitstype</code> attribute.</p>
<h3 dir="auto"><a id="user-content-string-type" class="anchor" aria-hidden="true" href="#string-type"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>String Type</h3>
<p dir="auto">The special type <code>Cstring</code> is used to represent a <code>char*</code> pointer. It occupies the space of any <code>Ptr</code>.</p>
<h3 dir="auto"><a id="user-content-reference-types" class="anchor" aria-hidden="true" href="#reference-types"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Reference Types</h3>
<p dir="auto">To describe <code>C</code>-pointers to primitive objects or C-structures, the <code>Ptr{T}</code> notion is used.
Here <code>T</code> is a Julia type name. It needs to be defined in the code before the usage.</p>
<p dir="auto">To support referencing types, which will be defined later, as is possible in <code>C</code>, the
special construct <code>LForwardReference{:S}</code> was introduced which uses the symbolized name <code>:S</code> of
the referenced type, which can be defined later. This feature will become obsolete, as soon as <code>Julia</code>
will support forward references <a href="https://github.com/JuliaLang/julia/pull/32658" data-hovercard-type="pull_request" data-hovercard-url="/JuliaLang/julia/pull/32658/hovercard">PR#32658</a>.</p>
<p dir="auto">Element type <code>T</code> must be a reference type.</p>
<h3 dir="auto"><a id="user-content-vector-types" class="anchor" aria-hidden="true" href="#vector-types"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Vector Types</h3>
<p dir="auto">A fixed length vector of <code>N</code> elements of type <code>T</code> is denoted <code>LFixedVector{T,N}</code>. It has the size of
<code>NTuple{N,T}</code>, where <code>T</code> is any of the supported types. A pointer to a vector is <code>Ptr{LFixedVector{T,N}}</code>
or <code>LForwardReference{LFixedVector{T,N}}</code>.</p>
<p dir="auto">A variable length vector is denoted <code>LVarVector{T,F}</code> with the same restrictions to <code>T</code> as for fixed vectors.
It can be embedded as the last element of a layout structure or the element type of a reference.
The actual length can be calculated by <code>F(x)</code> with <code>x</code> the current structure, where the vector is referenced.
Typically that uses to be <code>F = (x) -&gt; x.fieldname</code>, the vector length stored in an integer field.</p>

<p dir="auto">An bitstype struct, which is a subtype of <code>Layout</code> is composed of fields, which have an identical memory layout as
a corresponding C-struct.
The fields may be any of the mentioned layout types, but not directly self-referential
(only via <code>Ptr</code> or <code>LForwardReference</code>).</p>

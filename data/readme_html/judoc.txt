</h2>
<p dir="auto">Franklin is a simple <strong>static site generator</strong> (SSG) oriented towards technical blogging (code, maths, ...), flexibility and extensibility.
The base syntax is plain markdown with a few extensions such as the ability to define and use LaTeX-like commands in or outside of maths environments and the possibility to evaluate code blocks on the fly.</p>
<p dir="auto">Franklin has a channel <strong>#franklin</strong> on the Julia slack, this is the best place to ask usage questions.</p>
<p dir="auto"><strong>Note</strong>: I'm looking for people with web-dev chops who would be keen to help improve and enrich the base themes available at <a href="https://github.com/tlienart/FranklinTemplates.jl">FranklinTemplates.jl</a>, even if you're not super confident in Julia. More generally, if you would like to work on something or fix something in either Franklin or FranklinTemplates, please reach out on Slack, I will gladly help you get started.</p>

<ul dir="auto">
<li>if, upon deployment, your website doesn't seem to apply CSS, you likely forgot a step, please <a href="https://franklinjl.org/workflow/deploy/#creating_a_repo_on_github" rel="nofollow">see here</a>.</li>
<li>it is currently recommended to <strong>switch off</strong> pre-rendering and minification (you can do this by passing <code>prerender=false, minify=false</code> to the <code>optimize(...)</code> call in your deploy script), the corresponding logic will be removed in future version of Franklin as it's tricky to maintain and caused too many issues.</li>
<li>if you're getting warning messages about some JS library not being found on your computer, you can safely ignore this.</li>
</ul>
<h2 dir="auto"><a id="user-content-key-features" class="anchor" aria-hidden="true" href="#key-features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Key features</h2>
<ul dir="auto">
<li>Use standard markdown with the possibility to use LaTeX-style commands and generating functions written in Julia</li>
<li>Simple way to introduce div blocks allowing easy styling on a page (e.g. "Theorem" boxes etc.),</li>
<li>Can execute and show the output of Julia code blocks,</li>
<li>Easy HTML templating to define or adapt a given layout,</li>
<li>Custom RSS generation,</li>
<li>Simple website optimisation step:
<ul dir="auto">
<li>compression of HTML and CSS of the generated pages,</li>
<li>optional pre-rendering of KaTeX and highlighted code blocks to remove javascript dependency,</li>
</ul>
</li>
<li>Extremely flexible and extensible.</li>
</ul>
<p dir="auto">See <a href="https://franklinjl.org" rel="nofollow">the docs</a> for more information and examples.</p>

<h3 dir="auto"><a id="user-content-heads-up" class="anchor" aria-hidden="true" href="#heads-up"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Heads up!</h3>
<p dir="auto">While Franklin broadly supports standard Markdown there are a few things that may trip you which are either due to Franklin or due to Julia's Markdown library, here are key ones you should keep in mind:</p>
<ul dir="auto">
<li>when writing a list, the content of the list item <strong>must</strong> be on a single line (no line break)</li>
<li>you can write comments with <code>&lt;!-- comments --&gt;</code> the comment markers <code>&lt;!--</code> and <code>--&gt;</code> <strong>must</strong> be separated by a character that is not a <code>-</code> to work properly so <code>&lt;!--A--&gt;</code> is ok but <code>&lt;!---A---&gt;</code> is not, best is to just systematically use whitespace: <code>&lt;!-- A --&gt;</code>.</li>
<li>be careful writing double braces, <code>{{...}}</code> has a <em>meaning</em> (html functions) this can cause issues in latex commands, if you have double braces in a latex command, <strong>make sure to add whitespaces</strong> for instance write <code>\dfrac{1}{ {101}_{2} }</code> instead of <code>\dfrac{1}{{101}_{2}}</code>. In general use whitespaces liberally to help the parser in math and latex commands.</li>
<li>(as of <code>v0.7</code>) code blocks should be delimited with backticks <code>`</code> you <em>can</em> also use indented blocks to delimit code blocks but you now have to <strong>opt in</strong> explicitly on pages that would use them by using <code>@def indented_code = true</code>, if you want to use that everywhere, write that in the <code>config.md</code>. Note that indented blocks are <strong>ambiguous</strong> with some of the other things that Franklin provides (div blocks, latex commands) and so if you use them, you are responsible for avoiding ambiguities (effectively that means <em>not using indentation for anything else than code</em>)</li>
</ul>

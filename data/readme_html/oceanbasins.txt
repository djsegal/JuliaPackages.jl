<p dir="auto">OceanBasins.jl essentially provides functions to algorithmically determine if a (lat,lon) coordinate lies in a specific ocean or sea.
(It was developed to mask regions of interest for my personal research endeavours, i.e., global marine biogeochemical modelling with <a href="https://github.com/briochemc/AIBECS.jl">AIBECS.jl</a>.)</p>

<p dir="auto">To load the ocean/sea polygons, start with</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="OCEANS = oceanpolygons()"><pre>OCEANS <span class="pl-k">=</span> <span class="pl-c1">oceanpolygons</span>()</pre></div>
<p dir="auto">Note that the first time you call <code>oceanpolygons()</code>, it will download the <a href="https://figshare.com/articles/Limits_of_oceans_and_seas_in_digitized_machine_readable_form/10860656" rel="nofollow"><em>Limits of oceans and seas in digitized, machine readable</em> dataset</a> and store it in a safe place using <a href="https://github.com/oxinabox/DataDeps.jl">DataDeps.jl</a>.</p>
<p dir="auto">You can then test if a given <code>lat,lon</code> coordinate is, e.g., in the Pacific, via</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="ispacific(lat, lon, OCEANS)"><pre><span class="pl-c1">ispacific</span>(lat, lon, OCEANS)</pre></div>

<p dir="auto"><strong>This package does <em>not</em> determine if a coordinate is on land.</strong>
This is because the polygons from the <a href="https://figshare.com/articles/Limits_of_oceans_and_seas_in_digitized_machine_readable_form/10860656" rel="nofollow"><em>Limits of oceans and seas in digitized, machine readable</em> dataset</a> overlap with land.
For example, this</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer nofollow" href="https://user-images.githubusercontent.com/4486578/79623984-40133480-8162-11ea-84b8-3b654a09abaf.png"><img src="https://user-images.githubusercontent.com/4486578/79623984-40133480-8162-11ea-84b8-3b654a09abaf.png" width="50%" style="max-width: 100%;"></a></p>

<p dir="auto"><em>Mikrubi: a model for species distributions using region-based records</em></p>
<p dir="auto">Many species occurrence records from specimens and publications are based on regions such as administrative units (thus sometimes called <code>counties</code> in the codes). These region-based records are accessible and dependable, and sometimes they are the only available data source; however, few species distribution models accept such data as direct input. In <a href="https://onlinelibrary.wiley.com/doi/full/10.1111/ecog.06283" rel="nofollow">Yang et al. (2023)</a>, we present a method named Mikrubi for robust prediction of species distributions from region-based occurrence data. This is the Julia package implementing the algorithms.</p>

<p dir="auto">To estimate the fine-scale distribution of a species using its presence or absence in each region, the package generally requires three types of input data:</p>
<ul dir="auto">
<li>
<p dir="auto">A map describing the shapes of all regions as polygons. For many countries or regions, such an administrative partition map can be found from <a href="https://gadm.org/" rel="nofollow">Database of Global Administrative Areas</a> (accessible via <a href="https://github.com/JuliaGeo/GADM.jl">GADM.jl</a>, see examples/prinsepia/jui.jl). Specifically for China, the correct county-level shapefile is available from <a href="https://www.tianditu.gov.cn/" rel="nofollow">National Platform of Common Geospatial Information Services</a> and <a href="https://lbs.amap.com/" rel="nofollow">Gaode Map Open Platform</a>.</p>
</li>
<li>
<p dir="auto">Raster layers of climatic factors of the same size, shape, and resolution. A commonly used dataset is <a href="https://worldclim.org/data/index.html" rel="nofollow">WorldClim</a> (accessible via <a href="https://github.com/EcoJulia/RasterDataSources.jl">RasterDataSources.jl</a>).</p>
</li>
<li>
<p dir="auto">A list of regions occupied by the species.</p>
</li>
</ul>

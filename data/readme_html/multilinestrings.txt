<p>Tooling for manipulating multiline strings.</p>
<h2><a id="user-content-features" class="anchor" aria-hidden="true" href="#features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Features</h2>
<p>The package features support for:</p>
<ul>
<li>Multiline string literals (<code>@m_str</code>, <code>multiline</code>)</li>
<li>An indent function which only indents non-blank lines (<code>indent</code>)</li>
</ul>
<h3><a id="user-content-multiline-string-literal" class="anchor" aria-hidden="true" href="#multiline-string-literal"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Multiline String Literal</h3>
<p>The multiline string literal (<code>@m_str</code>), inspired from <a href="https://yaml-multiline.info/" rel="nofollow">YAML's block scalars</a>, which provide options for manipulating multiline string literals via a style and chomp indicator:</p>
<ul>
<li>Style indicator:
<ul>
<li><code>f</code> replace newlines with spaces (folded)</li>
<li><code>l</code> keep newlines (literal)</li>
</ul>
</li>
<li>Chomp indicator:
<ul>
<li><code>s</code> no newlines at the end (strip)</li>
<li><code>c</code> single newline at the end (clip)</li>
<li><code>k</code> keep all newlines from the end (keep)</li>
</ul>
</li>
</ul>
<p>The indicators are provided after the ending quote of the string (e.g. <code>m"hello\nworld!"fc</code>).
If no indicators are provided the default behaviour is folded/strip.</p>
<h4><a id="user-content-example" class="anchor" aria-hidden="true" href="#example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Example</h4>
<p>When writing a long string literal you may want to break the string up over multiple lines in the code, to make it easier to read, but have the string be printed as a single line.
Specifically, when writing an long error message you may want to break up the string over multiple lines:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="&quot;An unexpected error has occurred which really shouldn't have happened but somehow did. "><pre><span class="pl-s"><span class="pl-pds">"</span>An unexpected error has occurred which really shouldn't have happened but somehow did. Please check that the inputs to this function doesn't contain magic that may interfere with with the non-magical computation occurring on this device.<span class="pl-pds">"</span></span></pre></div>
<p>Alternatively written over multiple lines:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="&quot;An unexpected error has occurred which really shouldn't &quot; *
&quot;have happened but somehow did. Please check that the inputs &quot; *

<span class="pl-s"><span class="pl-pds">"</span>have happened but somehow did. Please check that the inputs <span class="pl-pds">"</span></span> <span class="pl-k">*</span>
<span class="pl-s"><span class="pl-pds">"</span>to this function doesn't contain magic that may interfere with <span class="pl-pds">"</span></span> <span class="pl-k">*</span>
<span class="pl-s"><span class="pl-pds">"</span>with the non-magical computation occurring on this device.<span class="pl-pds">"</span></span></pre></div>
<p>Writing strings this way can be cumbersome as you need to remember to add spaces between each line.
The MultilineStrings package provides an alternative way of writing this using the multiline string macro:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="m&quot;&quot;&quot;

<span class="pl-s">with the non-magical computation occurring on this device.</span>
<span class="pl-s"><span class="pl-pds">"""</span></span></pre></div>
<p>Take note that a Julia <a href="https://docs.julialang.org/en/v1/manual/strings/#Triple-Quoted-String-Literals" rel="nofollow">triple-quoted string literal</a> will leave most newlines in place.</p>
<h3><a id="user-content-indent" class="anchor" aria-hidden="true" href="#indent"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Indent</h3>

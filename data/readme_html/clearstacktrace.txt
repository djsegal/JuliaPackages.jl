<p><code>ClearStacktrace.jl</code> is an experimental package that replaces <code>Base.show_backtrace</code> with a clearer version that uses alignment and colors to reduce visual clutter and indicate module boundaries, and expands base paths so they are clickable.</p>

<p>You can change these values to modify stack trace printing behavior:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c"><span class="pl-c">#</span> module name color cycler</span>
ClearStacktrace<span class="pl-k">.</span>MODULECOLORS[] <span class="pl-k">=</span> [<span class="pl-c1">:light_blue</span>, <span class="pl-c1">:light_yellow</span>, <span class="pl-c1">:light_red</span>, <span class="pl-c1">:light_green</span>, <span class="pl-c1">:light_magenta</span>, <span class="pl-c1">:light_cyan</span>, <span class="pl-c1">:blue</span>, <span class="pl-c1">:yellow</span>, <span class="pl-c1">:red</span>, <span class="pl-c1">:green</span>, <span class="pl-c1">:magenta</span>, <span class="pl-c1">:cyan</span>]

<span class="pl-c"><span class="pl-c">#</span> prints the full path for base files, making them much longer but clickable in VSCode, Atom, etc.</span>
ClearStacktrace<span class="pl-k">.</span>EXPAND_BASE_PATHS[] <span class="pl-k">=</span> <span class="pl-c1">true</span>

<span class="pl-c"><span class="pl-c">#</span> prints ~ instead of the user directory</span>
ClearStacktrace<span class="pl-k">.</span>CONTRACT_USER_DIR[] <span class="pl-k">=</span> <span class="pl-c1">true</span>

<span class="pl-c"><span class="pl-c">#</span> print line breaks between stack frames</span>
ClearStacktrace<span class="pl-k">.</span>LINEBREAKS[] <span class="pl-k">=</span> <span class="pl-c1">true</span></pre></div>

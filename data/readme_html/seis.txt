<p>Seis.jl is an open, fast and flexible framework for analysing seismic
data in <a href="https://julialang.org" rel="nofollow">Julia</a>.</p>
<h2><a id="user-content-current-state-of-seisjl" class="anchor" aria-hidden="true" href="#current-state-of-seisjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Current state of Seis.jl</h2>
<p>Right now we are asking the passive seismic community what they want in a
package; see the <a href="https://github.com/anowacki/Seis.jl/blob/master/Seis.jl.md">draft specification document</a> for
more detail.</p>
<p>Whilst this is happening, this package is being developed and is a
trial implementation of the core
functionality which we think is needed.  Although the package is extensively
tested, please note that in this phase the public interface is subject to
change at any time.</p>
<p>Because things may change, documentation is also extremely limited.  All
public functions have extensive docstrings, however, and PRs to clarify
any problems with these are always welcome.</p>
<p>One way to explore the package until  documentation is ready is via
the REPL.  After <code>using Seis</code>, type <code>Seis.</code> and hit tab a couple of
times to see what Seis defines.</p>

<p>At present, Seis is unregistered and it and its dependencies must be
installed manually.  This will change once Seis is registered in
the General registry.</p>

<h2><a id="user-content-basic-introduction" class="anchor" aria-hidden="true" href="#basic-introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Basic introduction</h2>
<p><code>Seis</code> is based around single traces of evenly-sampled continuous data where
time is the independent variable.  (In future we may support unevenly-sampled
data and traces with gaps, and we plan to soon support frequency-domain traces.)
Each trace holds its sampling interval (the property <code>.delta</code>) and a start time
(<code>.b</code>).  Traces have the type <code>Trace</code>.</p>
<p><code>Trace</code>s also optionally hold information about the event (<code>.evt</code>) and station
(<code>.sta</code>) associated with this recording.  <code>Event</code>s and <code>Station</code>s are the
corresponding types, which also contain useful properties.</p>
<p>For more information, see the docstrings for <code>Trace</code>, <code>Event</code> and <code>Station</code>.
This is easily done in the Julia REPL like so:</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Seis

julia<span class="pl-k">&gt;</span> ? <span class="pl-c"><span class="pl-c">#</span> As soon as you press '?', the prompt changes to 'help?&gt;'</span>

help?<span class="pl-k">&gt;</span> Trace
search<span class="pl-k">:</span> Trace trace backtrace AbstractSet AbstractVector AbstractVecOrMat

  Trace

  Evenly<span class="pl-k">-</span>sampled time series recorded at a single seismic station. The start
  time of the trace, <span class="pl-k">in</span> s, is <span class="pl-k">in</span> the b field, whilst the sampling interval, <span class="pl-k">in</span>
  s, is delta. The trace itself is accessed <span class="pl-k">using</span> the trace method, like
  <span class="pl-c1">trace</span>(t).

  [<span class="pl-k">...</span>]</pre></div>
<p>All three types above also hold a <code>.meta</code> property, which contains any extra
metadata you want to associate with the trace, event or station.</p>

<h3><a id="user-content-f-vs-f" class="anchor" aria-hidden="true" href="#f-vs-f"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>f</code> vs <code>f!</code></h3>
<p><code>Trace</code>s are <code>mutable struct</code>s, and therefore for all functions which
potentially modify a trace, there are two versions.  Firstly, an in-place
function (e.g., <code>bandpass!</code>) which as per Julia convention ends with an
exclamation mark, and modifies the trace.  Secondly, for convenience, there
is always a copying version without the exclamation mark (e.g., <code>bandpass</code>)
which returns a modified copy of the input trace.</p>

<h2><a id="user-content-io" class="anchor" aria-hidden="true" href="#io"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>IO</h2>
<p>Currently, only SAC data are read or written, but may be either bigendian
(SAC/BRIS convention) or little-endian (usual IRIS SAC convention).</p>
<p>Use the <code>read_sac</code> and <code>write_sac</code> functions for SAC-formatted IO.</p>
<p>Future work will add support for reading many more formats and format
auto-detection.</p>
<h2><a id="user-content-plotting" class="anchor" aria-hidden="true" href="#plotting"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Plotting</h2>
<p>At present, some basic plotting is available when <code>using Plots</code>.
It makes use of <a href="https://github.com/JuliaPlots/RecipesBase.jl">RecipesBase.jl</a>
so that if you do not need plotting, the package does not introduce this
dependency.  If you do want plots, simply install <a href="https://github.com/JuliaPlots/Plots.jl">Plots.jl</a>
by <code>import Pkg; Pkg.add("Plots")</code>.  You then need to do <code>using Plots</code> when you
want to start using Seis.jl's plotting routines.</p>
<p>By default,
plotting functionality is not exported, so you must also do <code>using Seis.Plot</code>
before the routines <code>plot</code> (plots traces) and <code>section</code> (plot record sections)
are available.</p>
<p>As an example:</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Seis, Seis<span class="pl-k">.</span>Plot, Plots

<h2><a id="user-content-basic-processing" class="anchor" aria-hidden="true" href="#basic-processing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Basic processing</h2>
<p>Basic time series processing of traces is possible using functions such as
<code>integrate</code>, <code>bandpass</code>, <code>remove_trend</code>.  Until proper documentation is

<p dir="auto">MethodOfLines.jl is a package for automated finite difference discretization
of symbolically-defined PDEs in N dimensions.</p>
<p dir="auto">It uses symbolic expressions for systems of partial differential equations as defined with <code>ModelingToolkit.jl</code>, and <code>Interval</code> from <code>DomainSets.jl</code> to define the space(time) over which the simulation runs.</p>
<p dir="auto">This project is under active development, therefore the interface is subject to change. The <a href="https://docs.sciml.ai/MethodOfLines/dev/" rel="nofollow">docs</a> will be updated to reflect any changes, please check back for current usage information.</p>
<p dir="auto">Note that this package does not currently scale well to high resolution (high point count) problems, though there are changes in the works to remedy this.</p>
<p dir="auto">Allowable terms in the system include, but are not limited to</p>
<ul dir="auto">
<li>Advection</li>
<li>Diffusion</li>
<li>Reaction</li>
<li>Nonlinear Diffusion</li>
<li>Spherical Laplacian</li>
<li>Any Julia function of the symbolic parameters/dependant variables and other parameters in the environment that's defined on the whole domain.</li>
</ul>
<p dir="auto">Boundary conditions include, but are not limited to:</p>
<ul dir="auto">
<li>Dirichlet</li>
<li>Neumann (can also include time derivative)</li>
<li>Robin (can also include time derivative)</li>
<li>Periodic</li>
<li>Any equation, which can include arbitrary Julia functions defined on that boundary, with the only symbolic parameters being those appearing in the referenced boundary.</li>
</ul>

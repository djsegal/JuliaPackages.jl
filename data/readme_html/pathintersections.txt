<p dir="auto">This package locates intersections between parameterized curves (paths) and Cartesian meshes in arbitrary dimension. The package locates intersections by walking along the curve using the user provided step size, <code>ds</code>, and sensing for boundary crossings. When a boundary is crossed, a hybrid bracketed secant-bisection method is used to determine the location of the intersection within the user-specified tolerances <code>arc_tol</code> and <code>corner_tol</code>.</p>
<p dir="auto">While the use of a bracketed method is robust in many situations, it does not allow all non-simple intersections to be found; cases that are currently unsupported or may result in unsatisfactory accuracy are noted below.</p>

<h3 dir="auto"><a id="user-content-arguments" class="anchor" aria-hidden="true" href="#arguments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Arguments:</h3>
<ul dir="auto">
<li>
<p dir="auto"><code>coords</code>: A Cartesian mesh, which is to be provided in the form of an array of arrays with indexing <code>coords[dimension][edge index]</code>. Note the mesh does not need to be uniform.</p>
</li>
<li>
<p dir="auto"><code>curve</code>/<code>curves</code> : The curve/curves to be check against the mesh. The package expects curves to be callable using <code>pt = curve(s)</code> where <code>pt</code> is indexed <code>pt[dimension]</code>. For an array/tuple of curves the expected indexing/signature is <code>pt = curves[curve index](s)</code>.</p>
</li>
<li>
<p dir="auto"><code>ds</code> : optional argument, the/an array of step size(s) use for walking along the curve. Arrays are to be indexed <code>ds[curve index]</code>. Defaults to the minimum of 1/1000 (note <code>s</code> is limited to [0,1]) or a value such that <code>norm(curve(s+ds) - curve(s))</code> is less than the minimum mesh spacing.</p>
</li>
<li>
<p dir="auto"><code>arc_tol</code> : optional argument, the/an array of arc length tolerance(s) used as the stopping criteria for finding intersections. Arrays are to be indexed <code>arc_tol[curve index]</code>. Default is <code>100*eps(eltype(coords))</code>.</p>
</li>
<li>
<p dir="auto"><code>corner_tol</code>: optional argument, the/an array of single-dimension tolerance(s) used for identifying if other dimensions participated in an intersecton (i.e. whether the intersection occurred at a corner). Arrays are to be indexed <code>corner_tol[curve index]</code>. Default is <code>100*eps(eltype(coords))</code>.</p>
</li>
</ul>
<h3 dir="auto"><a id="user-content-assumptions" class="anchor" aria-hidden="true" href="#assumptions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Assumptions</h3>
<ol dir="auto">
<li>Each array in the array of mesh coordinates, <code>coords</code>, must:
<ol dir="auto">
<li>be sorted in increasing order (i.e. <code>coords[dim][i] &lt; coords[dim][i+1]</code>),</li>
<li>not contain duplicates, and</li>
<li>contain the start and end points of the domain in that dimension.</li>
</ol>
</li>
<li>All curves start at <code>s = 0</code> and end at <code>s = 1</code> and are defined at every point in between. The end points will be checked regardless of the step size provided.</li>
<li>The provided step size is sufficiently small in comparison to the mesh for all intersections to be sensed.</li>
<li>Differentiability of the curve: When the step size, <code>ds</code>, is defaulted, the derivative of the curve will be evaluated in order to find a suitable step size using automatic differentiation as implemented in <code>ForwardDiff.jl</code>. If the curve or its derivative is not defined at certain points the resulting step size may be undesirable.</li>
</ol>

<ol dir="auto">
<li>This package is (currently) only concerned with mesh-curve intersections, not curve-curve intersections.</li>
<li>Intersections are returned sorted in increasing order according to their <code>s</code> values.</li>
<li>Curves may start and arbitrarily enter and leave regions outside the domain defined by the mesh.</li>
</ol>

<h3 dir="auto"><a id="user-content-tolerance-definitions" class="anchor" aria-hidden="true" href="#tolerance-definitions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Tolerance Definitions</h3>
<ul dir="auto">
<li><code>arc_tol</code>
The main tolerance used to find intersections. This tolerance is applied to the Euclidean distance between the upper and lower bounds of the secant-bisection method at each iteration. The method stops when the Euclidean distance between these points is less than or equal to <code>arc_tol</code>.</li>
</ul>
<p align="center" dir="auto">
  <a target="_blank" rel="noopener noreferrer" href="/figures/tolerance_arcTol.png"><img src="/figures/tolerance_arcTol.png" width="800" style="max-width: 100%;"></a>
</p>
<ul dir="auto">
<li><code>corner_tol</code>
The secondary tolerance for corner intersections. This tolerance is used to determine if another dimension participated in previously found intersection (i.e. to determine if the intersection went through a corner). It is applied to the distance between the previously estimated intersection point and the nearest mesh edge in the dimension being checked. It is, with the exception of folded intersections, a weaker condition than <code>arc_tol</code>.</li>
</ul>

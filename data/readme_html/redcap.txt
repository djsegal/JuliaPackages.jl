<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview</h2>
<p>A Julia frontend for the REDCap API. REDCap.jl supports both importing and exporting records, as well as deletion from the REDCap Database. It also includes functions for surveys and report generation.</p>
<p>Available under the MIT license.</p>

<p>Julia can be found using the standard package manager, and installed like any other package.</p>
<div class="highlight highlight-source-shell"><pre>add REDCap</pre></div>
<h2><a id="user-content-tests" class="anchor" aria-hidden="true" href="#tests"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Tests</h2>
<p>Tests can be run using the Julia package manager</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c1">test</span> REDCap</pre></div>
<p>Due to the nature of the API, the API-Key and URL must be set in the users <code>.juliarc</code> file:</p>
<div class="highlight highlight-source-shell"><pre>ENV[<span class="pl-s"><span class="pl-pds">"</span>REDCAP_SUPER_API<span class="pl-pds">"</span></span>] = <span class="pl-s"><span class="pl-pds">"</span>&lt;super-key&gt;<span class="pl-pds">"</span></span>
ENV[<span class="pl-s"><span class="pl-pds">"</span>REDCAP_API<span class="pl-pds">"</span></span>] = <span class="pl-s"><span class="pl-pds">"</span>&lt;key&gt;<span class="pl-pds">"</span></span>
ENV[<span class="pl-s"><span class="pl-pds">"</span>REDCAP_URL<span class="pl-pds">"</span></span>] = <span class="pl-s"><span class="pl-pds">"</span>&lt;url&gt;<span class="pl-pds">"</span></span></pre></div>
<p>If a super-key is not provided, project creation will not be tested. Otherwise, a project will be created, and verified.</p>
<h3><a id="user-content-note" class="anchor" aria-hidden="true" href="#note"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>NOTE:</h3>
<p>Testing must be performed on the users own REDCap environment. If you are unable to access your own REDCap environment, testing will also fail.
Projects are tested according to a rough template. Several tests may fail if your project deviates from this template such as record contents and users.</p>

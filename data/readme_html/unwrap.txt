<ul>
<li>(2D) M. A. Herr√°ez, D. R. Burton, M. J. Lalor, and M. A. <h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<p>The interface consists of a single function, unwrap:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Unwrap
unwrapped_array <span class="pl-k">=</span> <span class="pl-c1">unwrap</span>(wrapped_array)</pre></div>
<p>where <code>wrapped_array</code> is an <code>AbstractArray</code>. There is also an
in-place version, <code>unwrap!(A)</code> which mutates <code>A</code>. The returned array will have
the same precision as <code>A</code> (i.e. <code>BigFloat</code>, <code>Float32</code>, etc. work). Type
<code>?unwrap</code> for more.</p>
<p>1D, 2D, and 3D unwrapping are all supported, as well as arbitrary dimensions, for whatever reason.</p>

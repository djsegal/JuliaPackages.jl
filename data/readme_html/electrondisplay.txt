<p><a target="_blank" rel="noopener noreferrer" href="screenshot.png"><img src="screenshot.png" alt="ElectronDisplay.jl screenshot" width="50%" align="right" style="max-width:100%;"></a></p>
<p>This package provides a display for figures, plots and tables. When you load the package, it will push a new display onto the julia display stack and from then on it will display any value that can be rendered as png, svg, vega, vega-lite or plotly in an electron based window. This is especially handy when one works on the REPL and wants plots or tables to show up in a nice window.</p>
<br clear="all">
<h2><a id="user-content-getting-started" class="anchor" aria-hidden="true" href="#getting-started"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Getting Started</h2>
<p>You can install the package via the normal julia package manger:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="Pkg.add(&quot;ElectronDisplay&quot;)
"><pre>Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>ElectronDisplay<span class="pl-pds">"</span></span>)</pre></div>
<p>As soon as you load the package with <code>using ElectronDisplay</code>, it will start to show plots that have the correct <code>show</code> methods in an electron window.</p>
<p><code>ElectronDisplay</code> also exports a function <code>electrondisplay</code>.  You can use <code>electrondisplay(x)</code> to show <code>x</code> explicitly in <code>ElectronDisplay</code> (e.g., when another display has higher precedence).  You can also use <code>electrondisplay(mime, x)</code> to specify a MIME to be used.  For example, to read the docstring of <code>reduce</code> in <code>ElectronDisplay</code>, you can use <code>electrondisplay(@doc reduce)</code>.</p>
<p>See the original <a href="https://www.queryverse.org/2019/02/13/electrondisplay/" rel="nofollow">announcement</a> for a quick start guide.</p>
<h2><a id="user-content-configuration" class="anchor" aria-hidden="true" href="#configuration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Configuration</h2>
<p>You can use the following configuration option to reuse existing window for displaying a new content.  The default behavior is to create a new window for each display.</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="using ElectronDisplay
ElectronDisplay.CONFIG.single_window = true
"><pre><span class="pl-k">using</span> ElectronDisplay
ElectronDisplay<span class="pl-k">.</span>CONFIG<span class="pl-k">.</span>single_window <span class="pl-k">=</span> <span class="pl-c1">true</span></pre></div>
<p>To control objects to be handled by <code>ElectronDisplay</code>, you can set <code>ElectronDisplay.CONFIG.showable</code>.  By default, <code>ElectronDisplay</code> does not show markdown, HTML, and <code>application/vnd.dataresource+json</code> output.  To show everything in <code>ElectronDisplay</code> whenever it's supported, you can use:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="ElectronDisplay.CONFIG.showable = showable
"><pre>ElectronDisplay<span class="pl-k">.</span>CONFIG<span class="pl-k">.</span>showable <span class="pl-k">=</span> showable</pre></div>
<p>To override the default configuration temporary, use the keyword arguments to <code>electrondisplay</code>:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="electrondisplay(@doc reduce; single_window=true, focus=false)
"><pre><span class="pl-c1">electrondisplay</span>(<span class="pl-c1">@doc</span> reduce; single_window<span class="pl-k">=</span><span class="pl-c1">true</span>, focus<span class="pl-k">=</span><span class="pl-c1">false</span>)</pre></div>

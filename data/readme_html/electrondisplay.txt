<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Overview</h2>
<p><a target="_blank" rel="noopener noreferrer" href="screenshot.png"><img src="screenshot.png" alt="ElectronDisplay.jl screenshot" width="50%" align="right" style="max-width:100%;"></a></p>
<p>This package provides a display for figures and plots. When you load the package, it will push a new display onto the julia display stack and from then on it will display any value that can be rendered as png, svg, vega, vega-lite or plotly in an electron based window. This is especially handy when one works on the REPL and wants plots to show up in a nice window.</p>
<br clear="all">
<h2><a id="user-content-getting-started" class="anchor" aria-hidden="true" href="#getting-started"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Getting Started</h2>
<p>You can install the package via the normal julia package manger:</p>
<div class="highlight highlight-source-julia"><pre>Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>ElectronDisplay<span class="pl-pds">"</span></span>)</pre></div>
<p>As soon as you load the package with <code>using ElectronDisplay</code>, it will start to show plots that have the correct <code>show</code> methods in an electron window.</p>
<p><code>ElectronDisplay</code> also exports a function <code>electrondisplay</code>.  You can use <code>electrondisplay(x)</code> to show <code>x</code> explicitly in <code>ElectronDisplay</code> (e.g., when another display has higher precedence).  You can also use <code>electrondisplay(mime, x)</code> to specify a MIME to be used.  For example, to read the docstring of <code>reduce</code> in <code>ElectronDisplay</code>, you can use <code>electrondisplay(@doc reduce)</code>.</p>
<h2><a id="user-content-configuration" class="anchor" aria-hidden="true" href="#configuration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuration</h2>
<p>You can use the following configuration option to reuse existing window for displaying a new content.  The default behavior is to create a new window for each display.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> ElectronDisplay
ElectronDisplay<span class="pl-k">.</span>CONFIG<span class="pl-k">.</span>single_window <span class="pl-k">=</span> <span class="pl-c1">true</span></pre></div>
<p>To control objects to be handled by <code>ElectronDisplay</code>, you can set <code>ElectronDisplay.CONFIG.showable</code>.  By default, <code>ElectronDisplay</code> does not show markdown, HTML, and <code>application/vnd.dataresource+json</code> output.  To show everything in <code>ElectronDisplay</code> whenever it's supported, you can use:</p>
<div class="highlight highlight-source-julia"><pre>ElectronDisplay<span class="pl-k">.</span>CONFIG<span class="pl-k">.</span>showable <span class="pl-k">=</span> showable</pre></div>
<p>To override the default configuration temporary, use the keyword arguments to <code>electrondisplay</code>:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">electrondisplay</span>(<span class="pl-c1">@doc</span> reduce; single_window<span class="pl-k">=</span><span class="pl-c1">true</span>, focus<span class="pl-k">=</span><span class="pl-c1">false</span>)</pre></div>

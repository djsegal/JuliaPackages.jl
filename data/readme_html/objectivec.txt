<p dir="auto">ObjectiveC.jl is a registered package, so you can install it using the package manager:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="Pkg.add(&quot;ObjectiveC&quot;)"><pre>Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>ObjectiveC<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">The library allows you to call Objective-C methods using almost-native syntax:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using ObjectiveC

julia&gt; @objc [NSString new]::id{Object}

<span class="pl-c1">id</span><span class="pl-c1">{Object}</span>(<span class="pl-c1">0x00006000008a4760</span>)</pre></div>
<p dir="auto">For performance reasons, ObjectiveC.jl requires you to specify the type of the call and
any arguments using Julia type-assertion syntax (<code>::id{Object}</code> in the example above).</p>
<p dir="auto">Although it is possible to build Julia APIs around this functionality, manually keeping
track of <code>id</code> pointers, it is possible to have ObjectiveC.jl do this for you:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; @objcwrapper NSValue

julia&gt; obj_ptr = @objc [NSValue valueWithPointer:C_NULL::Ptr{Cvoid}]::id{NSValue}
id{NSValue}(0x00006000023cfca0)

julia&gt; obj = NSValue(obj_ptr)
NSValueInstance (object of type NSConcreteValue)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">@objcwrapper</span> NSValue

julia<span class="pl-k">&gt;</span> obj_ptr <span class="pl-k">=</span> <span class="pl-c1">@objc</span> [NSValue valueWithPointer<span class="pl-k">:</span><span class="pl-c1">C_NULL</span><span class="pl-k">::</span><span class="pl-c1">Ptr{Cvoid}</span>]<span class="pl-k">::</span><span class="pl-c1">id{NSValue}</span>
<span class="pl-c1">id</span><span class="pl-c1">{NSValue}</span>(<span class="pl-c1">0x00006000023cfca0</span>)

julia<span class="pl-k">&gt;</span> obj <span class="pl-k">=</span> <span class="pl-c1">NSValue</span>(obj_ptr)
NSValueInstance (object of type NSConcreteValue)</pre></div>
<p dir="auto">The generated <code>NSValue</code> class is an abstract type that implements the type hierarchy, while
the <code>NSValueInstance</code> object is a concrete structure that houses the <code>id</code> pointer. This
split makes it possible to implement multi-level inheritance and attach functionality at
each level of the hierarchy, and should be entirely transparent to the user (i.e., you
should never need to use the <code>*Instance</code> types in code or signatures).</p>
<p dir="auto">The <code>@objcwrapper</code> macro also generates conversion routines and accessors that makes it
possible to use these objects directly with <code>@objc</code> calls that require <code>id</code> pointers:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; get_pointer(val::NSValue) = @objc [val::id{NSValue} pointerValue]::Ptr{Cvoid}

julia&gt; get_pointer(obj)

<p dir="auto">A common pattern in Objective-C is to use properties to acces instance variables. Although
it is possible to access these directly using <code>@objc</code>, ObjectiveC.jl provides a macro to
automatically generate the appropriate <code>getproperty</code>, <code>setproperty!</code> and <code>propertynames</code>
definitions:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; @objcproperties NSValue begin

       end

julia&gt; obj.pointerValue

<p dir="auto">The behavior of <code>@objcproperties</code> can be customized by passing keyword arguments to the
property macros:</p>

  # also generate a setter
  @autoproperty someProperty::DstTyp setter=setSomeProperty

  end

    # do something with obj and val

<p dir="auto">Julia callables can be converted to Objective-C blocks using the <code>@objcblock</code> macro:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; function hello(x)
         println(&quot;Hello, $x!&quot;)
         x+1
       end
julia&gt; block = @objcblock(hello, Cint, (Cint,))"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">function</span> <span class="pl-en">hello</span>(x)
         <span class="pl-c1">println</span>(<span class="pl-s"><span class="pl-pds">"</span>Hello, <span class="pl-v">$x!</span><span class="pl-pds">"</span></span>)
         x<span class="pl-k">+</span><span class="pl-c1">1</span>
       <span class="pl-k">end</span>
julia<span class="pl-k">&gt;</span> block <span class="pl-k">=</span> <span class="pl-c1">@objcblock</span>(hello, Cint, (Cint,))</pre></div>
<p dir="auto">This object can now be passed to Objective-C methods that take blocks as arguments. Note
that before Julia 1.9, blocks should only ever be called from Julia-managed threads, or else
your application will crash.</p>
<p dir="auto">If you need to use blocks that may be called from unrelated threads on Julia 1.8 or earlier,
you can use the <code>@objasyncblock</code> macro instead. This variant takes an <code>AsyncCondition</code> that
will be executed on the libuv event loop after the block has been called. Note that there
may be some time between the block being called and the condition being executed, and libuv
may decide to coalesce multiple conditions into a single execution, so it is preferred to
use <code>@objcblock</code> whenever possible. It is also not possible to pass any arguments to the
condition, but you can use a closure to capture any state you need:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; counter = 0
julia&gt; cond = Base.AsyncCondition() do async_cond
          counter += 1
        end
julia&gt; block = @objcasyncblock(cond)"><pre>julia<span class="pl-k">&gt;</span> counter <span class="pl-k">=</span> <span class="pl-c1">0</span>
julia<span class="pl-k">&gt;</span> cond <span class="pl-k">=</span> Base<span class="pl-k">.</span><span class="pl-c1">AsyncCondition</span>() <span class="pl-k">do</span> async_cond
          counter <span class="pl-k">+=</span> <span class="pl-c1">1</span>
        <span class="pl-k">end</span>

<p dir="auto">ObjectiveC.jl has recently been revamped, and is still under heavy development. Do not
assume its APIs are stable until version 1.0 is released. That said, it is being used
as the main FFI for <a href="https://github.com/JuliaGPU/Metal.jl">Metal.jl</a>, so you can expect
the existing functionality to be fairly solid.</p>
<p dir="auto">In the process of revamping the package, some functionality was lost, including the ability
to define Objective-C classes using native-like syntax. If you are interested, please take a
look at the repository <a href="https://github.com/JuliaInterop/ObjectiveC.jl/tree/22118319da1fb7601d2a3ecefb671ffbb5e57012">before the
revamp</a>
and consider contributing a PR to bring it back.</p>

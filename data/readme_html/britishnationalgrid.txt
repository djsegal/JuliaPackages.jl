<h2><a id="user-content-convert-between-wgs84-coordinates-and-british-national-grid-references" class="anchor" aria-hidden="true" href="#convert-between-wgs84-coordinates-and-british-national-grid-references"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Convert between WGS84 coordinates and British National Grid references</h2>
<p><code>BritishNationalGrid</code> provides the type <code>BNGPoint</code> to convert between
longitude-latitude and grid references in the <a href="https://en.wikipedia.org/wiki/Ordnance_Survey_National_Grid" rel="nofollow">British National Grid system</a>.
It assumes your points are geodetic longitude and latitude in decimal
degrees using the WGS84 ellipsoid.</p>
<p>This package is reliable to within a metre or so.  Advanced users needing
greater accuracy will probably already know how to convert between different
systems, but any additions to the package that remain easy to use will
be welcome.</p>

<ul>
<li>Use the Ordnance Survey's <a href="https://www.ordnancesurvey.co.uk/gps/transformation/" rel="nofollow">online converter</a>.  This also
includes links to the OS's Pascal programs to do coordinate transforms.</li>
<li>Use the British Geological Survey's <a href="http://www.bgs.ac.uk/data/webservices/convertform.cfm" rel="nofollow">online converter</a>, which also
assumes WGS84 longitude and latitude.</li>
</ul>

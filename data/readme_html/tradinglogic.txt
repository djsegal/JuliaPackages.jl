<h1><a id="user-content-tradinglogicjl" class="anchor" aria-hidden="true" href="#tradinglogicjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>TradingLogic.jl</h1>
<p>An attempt at a unified framework in Julia language for both event-driven backtesting and live trading. This package utilizes reactive programming elements implemented in <a href="https://github.com/JuliaLang/Reactive.jl">Reactive.jl</a>.</p>
<p>The main objective is to be able to backtest and place real-time orders using the same trading strategy functions.</p>
<p>NOTE: The current <code>master</code>-branch code requires <code>Julia &gt;=0.4.0</code>. See the <code>julia03</code> branch for the last version tested with Julia 0.3 before <a href="https://github.com/JuliaLang/julia/releases">0.4.0 final release</a>.</p>

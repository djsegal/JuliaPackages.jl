<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Introduction</h2>
<p>This software is the start of a computer algebra system specialized to
discrete calculations in the area of integer numbers <code>ℤ</code>, modular arithmetic <code>ℤ/m</code>
fractional <code>ℚ</code>, polynomials <code>ℤ[x]</code>. Also multivariate polynomials <code>ℤ[x,y,...]</code> and Galois fields <code>GF(p^r)</code> are supported.</p>
<p>This is not seen as a replacement of <code>Nemo.jl</code> or <code>AbstractAlgebra.jl</code>, which should be used for serious work.
It is understood more like a sandbox to try out a simpler API.</p>
<p>It is important, that rings may be freely combined, for example <code>(ℤ/p)[x]</code> (polynomials over the quotient ring for a prime number <code>p</code>),
<code>Frac(ℤ[x])</code>, the rational functions with integer coefficients, or <code>GF(64)[:x]</code>, polynomials over the Galois field.
The quotient rings include ideals, which are of major importance with multivariate polynomials.</p>
<p>The mentioned examples are elementary examples for ring structures. The can be
liberately combined to fractional fields, quotient rings, and polynomials of previously defined structures.</p>
<p>So it is possible to work with rational functions, which are fractions of polynomials, where the polynomial coefficients are in ℤ/p, for example.
The the current standard library we have modules <code>Rational</code> and <code>Polynomial</code> besides the numeric subtypes of <code>Number</code> and some support for modular calculations with integers.</p>
<p>The original motivation for writing this piece of sofware, when I tried to handle polynomials over a quotient ring. There was no obvious way of embedding my ring elements
into the <code>Julia</code> language and for example exploit polynomial calculations from the <code>Polynomial</code> package for that. There seems to be a correspondence between <code>Julia</code> types and structures and the algebraic stuctures I want to work with. So the idea was born to define
abstract and concrete types in <code>Julia</code>, the objects of those types representing the ring
elements to operate on. As types are first class objects in <code>Julia</code> it was also possible to
define combinations in a language affine way. Also <code>ring homomorphisms</code>, i.e. strucure-respecting mappings between rings (of differnt kind) find a natural representation as one-argument-functions or methods with corresponding domains. The typical canonical homomorphisms, can be conveniently implemented as constructors.</p>
<p>The exploitation of the julia structures is in contrast to the alternative package <a href="https://github.com/Nemocas/AbstractAlgebra.jl">AbstractAlgebra</a>, which defines separate types for ring elements and the ring classes themselves, the elements keeping an explicit link to the owner structure.</p>
<p>To distinct variants of rings, we use type parameters, for example the <code>m</code> in <code>ℤ/m</code> or the <code>x</code> in <code>ℤ[:x]</code>. Other type parameters may be used to specify implementation restictions, for
example typically the integer types used for the representation of the objects.</p>

<h3><a id="user-content-class-construction" class="anchor" aria-hidden="true" href="#class-construction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>class construction</h3>
<p>Each complete <code>Julia</code> type (with all type parameters specified) defines a singlton algebraic class. Sometimes it is necessary to use distinguishing symbols as a first type parameter if the parameter value cannot be used directly.
For that purpose, there is a special function <code>new_class</code>:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="    m = big&quot;.....&quot;
    Zm = new_class(ZZmod{:p,BigInt}, m)

 # as opposed to

    p = Int128(2)^127 - 1
    Zp = ZZmod{p,Int128}
"><pre>    m <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds"><span class="pl-c1">big</span>"</span>.....<span class="pl-pds">"</span></span>
    Zm <span class="pl-k">=</span> <span class="pl-c1">new_class</span>(ZZmod{<span class="pl-c1">:p</span>,BigInt}, m)

 <span class="pl-c"><span class="pl-c">#</span> as opposed to</span>

    p <span class="pl-k">=</span> <span class="pl-c1">Int128</span>(<span class="pl-c1">2</span>)<span class="pl-k">^</span><span class="pl-c1">127</span> <span class="pl-k">-</span> <span class="pl-c1">1</span>
    Zp <span class="pl-k">=</span> ZZmod{p,Int128}</pre></div>
<p>For general quotient classes and for polynomials there are convenient constructors, which
look like the typical mathematical notation <code>R[:x,:y,...]</code> and <code>R/I</code>.
Here the symbols <code>:x, :y</code> define the name of the variables of a uni-
or multivariate polynomial ring over <code>R</code>. <code>I</code> is an ideal of R or an element of
<code>R</code>, which represents the corresponding principal ideal.</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="    S = ZZ{Int}
    P = S[:x]
    x = monom(P, 1) # same as P([0,1])
    Q = P/(x^2 + 1)
"><pre>    S <span class="pl-k">=</span> ZZ{Int}
    P <span class="pl-k">=</span> S[<span class="pl-c1">:x</span>]
    x <span class="pl-k">=</span> <span class="pl-c1">monom</span>(P, <span class="pl-c1">1</span>) <span class="pl-c"><span class="pl-c">#</span> same as P([0,1])</span>
    Q <span class="pl-k">=</span> P<span class="pl-k">/</span>(x<span class="pl-k">^</span><span class="pl-c1">2</span> <span class="pl-k">+</span> <span class="pl-c1">1</span>)</pre></div>
<p>The <code>/</code> notion is also implemented for <code>Julia</code> integer types,
so this works:</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="    Z31 = Int8/31    # equivalent to ZZmod{31,Int8}
    Zbig = BigInt/(big&quot;2&quot;^521-1) # equivalent to new_class(ZZmod{gensym(),BigInt}, m)
"><pre>    Z31 <span class="pl-k">=</span> Int8<span class="pl-k">/</span><span class="pl-c1">31</span>    <span class="pl-c"><span class="pl-c">#</span> equivalent to ZZmod{31,Int8}</span>
    Zbig <span class="pl-k">=</span> BigInt<span class="pl-k">/</span>(<span class="pl-s"><span class="pl-pds"><span class="pl-c1">big</span>"</span>2<span class="pl-pds">"</span></span><span class="pl-k">^</span><span class="pl-c1">521</span><span class="pl-k">-</span><span class="pl-c1">1</span>) <span class="pl-c"><span class="pl-c">#</span> equivalent to new_class(ZZmod{gensym(),BigInt}, m)</span></pre></div>
<h3><a id="user-content-ideals" class="anchor" aria-hidden="true" href="#ideals"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Ideals</h3>
<p>Ideals are can be denoted as <code>Ideal([a, ...])</code> where a, ... are elements of <code>R</code> or
<code>Ideal(a)</code>. For convenience, principal ideals support also <code>a * R == Ideal(a)</code> and
<code>p*R == Ideal(R(p))</code>, where p is an integer.</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="    Z = ZZ{Int8}
    Z1 = Z/31
    Z2 = Z/Z(31)
    Z3 = Z/31Z
    Z4 = Z/Ideal(Z(31))

    Z1 == Z2 == Z3 == Z4
"><pre>    Z <span class="pl-k">=</span> ZZ{Int8}
    Z1 <span class="pl-k">=</span> Z<span class="pl-k">/</span><span class="pl-c1">31</span>
    Z2 <span class="pl-k">=</span> Z<span class="pl-k">/</span><span class="pl-c1">Z</span>(<span class="pl-c1">31</span>)
    Z3 <span class="pl-k">=</span> Z<span class="pl-k">/</span><span class="pl-c1">31</span>Z
    Z4 <span class="pl-k">=</span> Z<span class="pl-k">/</span><span class="pl-c1">Ideal</span>(<span class="pl-c1">Z</span>(<span class="pl-c1">31</span>))

    Z1 <span class="pl-k">==</span> Z2 <span class="pl-k">==</span> Z3 <span class="pl-k">==</span> Z4</pre></div>
<p>It may be noted, that <code>0R</code> is the zero ideal (containing only the zero element of <code>R</code>) and <code>u*R == R</code> for all unit elements <code>u</code> of <code>R</code>.
We also have <code>R/0R == R</code> and <code>R/1R</code> == <code>0R</code>.</p>
<p>If there is one generating element of an ideal, <code>a*R</code>, internally a
unit multiple of <code>a</code> is stored to achieve a standard form, for
example a monic univariate polynomial.</p>
<p>In the case of multiple generating elements, <code>a, b...</code>, an attempt
is made to standardize and reduce the stored base. For example if
<code>R</code> is an integral domain, then <code>gcd(a, b...)</code> is stored.</p>
<p>Ideals are most useful with multivariate polynomials, when they
are best represented by a minimal base - see below.</p>
<h3><a id="user-content-constructors-for-elements" class="anchor" aria-hidden="true" href="#constructors-for-elements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Constructors for elements</h3>
<p>The class names of all concrete types serve also as constructor names.
That means, if <code>R</code> is a class name, then <code>R(a)</code> is an element of <code>R</code> for all
<code>a</code>, which are integers or elements of (other) rings, which can be natuarally
embedded into <code>R</code>.</p>
<p>For polynomial rings <code>P</code>, the method call <code>monom(P, i)</code> constructs the monic
monomials <code>x^i</code> for non-negative integers <code>i</code>. That is extended to multivariate
cases <code>monom(P, i, j, ...)</code>.</p>

<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="    abstract type Ring{&lt;:RingClass} end

<p>The subtypes of <code>RingClass</code> are currently only containers for constant type variables. It may be necessary to hold values, which are specific for the algebraic structure, and cannot be stored in as type paramters. That is for example the case, when the modulus <code>m</code> in the example above is a <code>BigInt</code>.
In other cases, the classes are unused. The user needs not deal with those types.
Access to the type variable is used within the implementation by method <code>owner(::Type{&lt;:Ring}}</code> which provides the <code>RingClass</code> object, when the complete type is known.
Preferred operation mode is to take the type parameters directly.</p>
<h2><a id="user-content-univariate-polynomials" class="anchor" aria-hidden="true" href="#univariate-polynomials"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Univariate Polynomials</h2>
<p>For each ring type <code>R</code> the class of polynomials over <code>R</code> is created like <code>P = R[:x]</code> where the symbol <code>:x</code> defines the variable name.</p>
<p>Polynomials of this class are obtained by the constructor
<code>g = UnivariatePolynomial{R,:x}([1, 2, 3])</code>
or more conveniently by</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="x = monom(P)
g = 3x^2 + 2x + 1  
"><pre>x <span class="pl-k">=</span> <span class="pl-c1">monom</span>(P)
g <span class="pl-k">=</span> <span class="pl-c1">3</span>x<span class="pl-k">^</span><span class="pl-c1">2</span> <span class="pl-k">+</span> <span class="pl-c1">2</span>x <span class="pl-k">+</span> <span class="pl-c1">1</span>  </pre></div>
<p>If <code>R</code> is a finite Field (that means <code>ZZ/p</code> or GaloisField - see below) the following options are available:</p>
<p>Univariate polynomials may be checked by <code>isirreducible(p)</code> for their irreducibility
and <code>factor(g)</code> delivers the list of irreducible factors of <code>g</code>.
The factorization is also implemented for univariate polynomials over the integers (for example of type <code>UnivariatePolynomials{ZZ{BigInt}[:x]}</code>)</p>
<p>The function <code>irreducible(P, r)</code> delivers the first irreducible polynomial with degree <code>r</code>.
All irreducible polynomials of <code>P</code> with degree <code>r</code> are obtained by <code>irreducibles(P, r)</code>
which is an iterator. That allows to apply  <code>Iterators.filter</code> or <code>find</code> on this list.</p>
<p>While the number of polynomials of degree <code>r</code> is <code>order(R)^r</code>, the subset of
irreducibles has order <code>num_irreducibles(P, r)</code>.</p>
<h2><a id="user-content-galois-fields" class="anchor" aria-hidden="true" href="#galois-fields"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Galois Fields</h2>
<p>All finite field have order <code>p^r</code> where <code>p</code> is a prime number and <code>r &gt;= 1</code> an integer.
It can be represented as a quotient ring of univariate polynomials over <code>ZZ/p</code> by an irreducible monic polynomial <code>g</code> of degree <code>r</code>.
In short, if <code>g</code> is known, we have <code>(ZZ/p)[:x] / g</code> as a working implementation of a Galois field.
For <code>r == 1</code> this can be identified with <code>ZZ/p</code> (using <code>g(x) = x</code>).</p>
<p><code>g</code> can be any monic polynomial of degree <code>r</code>. When constructing the class <code>GF(p^r) = GaloisField{p,r}</code> a brute force search for such polynomials is
performed using an efficient method to detect irreducibility. For <code>r &gt; 1</code> the monom <code>x ∈ (ZZ/p)[:x] / g</code> with <code>0</code> and <code>1</code> generates the field by
applying addition and multiplication operations. Calculated in <code>GF</code>, we have always <code>g(x) = 0</code>.
We restrict the selection of <code>g</code> in order to <code>x</code> let generate the multiplicative subgroup of <code>GF</code> by multiplication. That is possible for all <code>p, r</code>.</p>
<p>Time efficiency of algebraic operations is improved by avoiding the expensive multiplicative calculations in the quotient ring and the use of
logarithmic tables in the size of <code>p^r</code>. Each element is represented by an integer in <code>0:p^r-1</code>, which corresponds to a polynomial of degree <code>&lt; r</code>
in a canonical manner (for example the number <code>2p^3 + p + 1</code> maps uniquley to <code>2x^3 + x + 1</code>).</p>

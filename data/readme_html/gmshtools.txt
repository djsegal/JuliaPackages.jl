<ul>
<li>
<p>Users could set the environment variable <code>GMSH_LIB_PATH</code> to your source-compiled Gmsh library
directory and rebuild this package. For windows user, you need to create a link from <code>gmsh-*.*.dll</code> to <code>libgmsh.dll</code> since something has changed with <code>find_library</code> after Julia <em>v1.4</em>.</p>
</li>
<li>
<p>If your Julia is compiled with <a href="https://github.com/JuliaComputing/MKL.jl">Intel MKL</a>, then it must be compiled with 32 integer interface (otherwise Julia will crash due to BLAS interface incompatibility), i.e. linking to <strong>lp64</strong> instead of <strong>ipl64</strong>. To do so, in <code>Make.inc</code>, change to</p>
<div class="highlight highlight-source-makefile"><pre><span class="pl-k">export</span> <span class="pl-smi">MKL_INTERFACE_LAYER</span> := LP64
<span class="pl-smi">MKLLIB</span> := <span class="pl-s">$(<span class="pl-smi">MKLROOT</span>)</span>/lib/intel64</pre></div>
</li>
<li>
<p>Since <em>v0.4.0</em>, you will need to manually install <a href="https://github.com/shipengcheng1230/Gmsh_SDK_jll.jl">Gmsh_SDK_jll.jl</a>, which, due to some building issues, cannot fit into <a href="https://github.com/JuliaBinaryWrappers">JuliaBinaryWrappers</a> for now. The building procedure is hosted at <a href="https://github.com/shipengcheng1230/GmshBuilder.jl">GmshBuilder.jl</a>.</p>
</li>
</ul>

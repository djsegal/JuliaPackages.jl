<h3><a id="user-content-collecting-packages-you-have-commit-access-to" class="anchor" aria-hidden="true" href="#collecting-packages-you-have-commit-access-to"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Collecting packages you have commit access to:</h3>
<p>It can be useful to filter the output above to only those packages you have
commit access to. <code>find_julia_packages_github()</code> returns a list of such packages
by using the GitHub API.  This requires you to have given a github API token
into the environment variable <code>GITHUB_AUTH</code>.  Note that due to what looks like a
bug in the GitHub API, this does currently not return the repos you have access
to via membership in an organization, making it not a lot less useful than
otherwise.</p>

<p dir="auto">This package implements Julia's <code>--bug-report</code> flag, simplyfing bug reporting by enabling
users to easily generate and upload reports to help developers fix bugs.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="    julia --bug-report=REPORT_TYPE[,REPORT_FLAG,...]"><pre class="notranslate"><code>    julia --bug-report=REPORT_TYPE[,REPORT_FLAG,...]
</code></pre></div>
<p dir="auto">Currently, only the <a href="https://github.com/rr-debugger/rr">rr</a> tool is supported to generate
bug reports, but in the future other types of reports may be supported as well.</p>

<p dir="auto">Run <code>julia</code> inside <code>rr record</code> but do not upload the recorded trace. Useful for local debugging.</p>

<p dir="auto">Generate a bug report, but limit the execution time of the debugged process to <code>SECONDS</code> seconds.
This is useful for generating reports for hangs.</p>

<p dir="auto">Generate an rr trace, while enabling so-called chaos mode. This is useful for flushing
out threading-related issues (refer to the rr documentation for more details).</p>

<p dir="auto">You can use this package also for debugging your own Julia code locally. Use <code>--bug-report=rr-local</code>
to record a trace, and <code>replay()</code> to replay the latest trace.</p>
<p dir="auto">For example, if you have a script in a project that you'd like to trace, run <code>julia --bug-report=rr -- --project=foo run.jl</code>.</p>

<p dir="auto"><code>StateMachine</code> objects are set up using the <code>state_machine</code> function, which takes a "model" describing your state machine's layout as input.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; using FiniteStateMachine

<p dir="auto">Unless other specified, a special "startup" event fires automatically when the state machine is created.</p>
<p dir="auto">The "initial" field can be either a string, or a dict specifying: <code>state</code> (the name of the state in which to start), <code>event</code> (the startup event), and/or <code>defer</code> (set to <code>true</code> if the startup event should <em>not</em> be fired automatically when the state machine is created).</p>
<p dir="auto">Supports multiple source states, for example, the following code allows "hop" to take place from either the "first" or "third" states:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="&quot;events&quot; =&gt; [{

    &quot;to&quot; =&gt; &quot;second&quot;,
},"><pre class="notranslate"><code>"events" =&gt; [{
    "name" =&gt; "hop",

</code></pre></div>

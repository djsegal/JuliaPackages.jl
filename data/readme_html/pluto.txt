<p><em>Writing a notebook is not just about writing the final document â€” Pluto empowers the experiments and discoveries that are essential to getting there.</em></p>
<p><strong>Explore models and share results</strong> in a notebook that is</p>
<ul>
<li><strong><em>reactive</em></strong> - when changing a function or variable, Pluto automatically updates all affected cells.</li>
<li><strong><em>lightweight</em></strong> - Pluto is written in pure Julia and is easy to install.</li>
<li><strong><em>simple</em></strong> - no hidden workspace state; friendly UI.</li>
</ul>

<h3><a id="user-content-input" class="anchor" aria-hidden="true" href="#input"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Input</h3>
<p>A Pluto notebook is made up of small blocks of Julia code (<em>cells</em>) and together they form a <a href="https://medium.com/@mbostock/a-better-way-to-code-2b1d2876a3a0" rel="nofollow"><strong><em>reactive</em></strong> notebook</a>.
When you change a variable, Pluto automatically re-runs the cells that refer to it. Cells can even be placed in arbitrary order - intelligent syntax analysis figures out the dependencies between them and takes care of execution.</p>
<p>Cells can contain <em>arbitrary</em> Julia code, and you can use external libraries. There are no code rewrites or wrappers, Pluto just looks at your code once before evaluation.</p>
<h3><a id="user-content-output" class="anchor" aria-hidden="true" href="#output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Output</h3>
<p>Your notebooks are <strong>saved as pure Julia files</strong> (<a href="https://github.com/fonsp/Pluto.jl/blob/main/sample/Basic.jl">sample</a>), which you can then import as if you had been programming in a regular editor all along. You can also export your notebook with cell outputs as attractive HTML and PDF documents. By reordering cells and hiding code, you have full control over how you tell your story.</p>
<br>
<h2><a id="user-content-dynamic-environment" class="anchor" aria-hidden="true" href="#dynamic-environment"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Dynamic environment</h2>
<p>Pluto offers an environment where changed code takes effect instantly and where deleted code leaves no trace.
Unlike Jupyter or Matlab, there is <strong>no mutable workspace</strong>, but rather, an important guarantee:</p>
<blockquote align="center"><em><b>At any instant</b>, the program state is <b>completely described</b> by the code you see.</em></blockquote>
No hidden state, no hidden bugs.
<h3><a id="user-content-interactivity" class="anchor" aria-hidden="true" href="#interactivity"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Interactivity</h3>
<p>Your programming environment becomes interactive by splitting your code into multiple cells! Changing one cell <strong>instantly shows effects</strong> on all other cells, giving you a fast and fun way to experiment with your model.</p>
<p>In the example below, changing the parameter <code>A</code> and running the first cell will directly re-evaluate the second cell and display the new plot.</p>

<h3><a id="user-content-html-interaction" class="anchor" aria-hidden="true" href="#html-interaction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>HTML interaction</h3>
<p>Lastly, here's <em><strong>one more feature</strong></em>: Pluto notebooks have a <code>@bind</code> macro to create a <strong>live bond between an HTML object and a Julia variable</strong>. Combined with reactivity, this is a very powerful tool!</p>
<p><a target="_blank" rel="noopener noreferrer" href="https://user-images.githubusercontent.com/6933510/80617037-e2c09280-8a41-11ea-9fb3-18bb2921dd9e.gif"><img alt="@bind macro screencap" src="https://user-images.githubusercontent.com/6933510/80617037-e2c09280-8a41-11ea-9fb3-18bb2921dd9e.gif" width="70%" style="max-width:100%;"></a></p>
<p><em>notebook from <a href="https://github.com/vdplasthijs/julia_sir">vdplasthijs/julia_sir</a></em></p>
<br>
<p>You don't need to know HTML to use it! The <a href="https://github.com/fonsp/PlutoUI.jl">PlutoUI package</a> contains basic inputs like sliders and buttons.</p>
<p>But for those who want to dive deeper - you can use HTML, JavaScript and CSS to write your own widgets! Custom update events can be fired by dispatching a <code>new CustomEvent("input")</code>, making it compatible with the <a href="https://observablehq.com/@observablehq/a-brief-introduction-to-viewof" rel="nofollow"><code>viewof</code> operator of observablehq</a>. Have a look at the sample notebooks inside Pluto to learn more!</p>

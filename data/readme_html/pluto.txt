<p dir="auto"><em>Writing a notebook is not just about writing the final document ‚Äî Pluto empowers the experiments and discoveries that are essential to getting there.</em></p>
<p dir="auto"><strong>Explore models and share results</strong> in a notebook that is</p>
<ul dir="auto">
<li><strong><em>reactive</em></strong> - when changing a function or variable, Pluto automatically updates all affected cells.</li>
<li><strong><em>lightweight</em></strong> - Pluto is written in pure Julia and is easy to install.</li>
<li><strong><em>simple</em></strong> - no hidden workspace state; friendly UI.</li>
</ul>

<p dir="auto">A Pluto notebook is made up of small blocks of Julia code (<em>cells</em>) and together they form a <a href="https://medium.com/@mbostock/a-better-way-to-code-2b1d2876a3a0" rel="nofollow"><strong><em>reactive</em></strong> notebook</a>.
When you change a variable, Pluto automatically re-runs the cells that refer to it. Cells can even be placed in arbitrary order - intelligent syntax analysis figures out the dependencies between them and takes care of execution.</p>
<p dir="auto">Cells can contain <em>arbitrary</em> Julia code, and you can use external libraries. There are no code rewrites or wrappers, Pluto just looks at your code once before evaluation.</p>
<h3 dir="auto"><a id="user-content-output" class="anchor" aria-hidden="true" href="#output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Output</h3>
<p dir="auto">Your notebooks are <strong>saved as pure Julia files</strong> (<a href="https://github.com/fonsp/Pluto.jl/blob/main/sample/Basic.jl">sample</a>), which you can then import as if you had been programming in a regular editor all along. You can also export your notebook with cell outputs as attractive HTML and PDF documents. By reordering cells and hiding code, you have full control over how you tell your story.</p>
<br>
<h2 dir="auto"><a id="user-content-dynamic-environment" class="anchor" aria-hidden="true" href="#dynamic-environment"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Dynamic environment</h2>
<p dir="auto">Pluto offers an environment where changed code takes effect instantly and where deleted code leaves no trace.
Unlike Jupyter or Matlab, there is <strong>no mutable workspace</strong>, but rather, an important guarantee:</p>
<blockquote align="center"><em><b>At any instant</b>, the program state is <b>completely described</b> by the code you see.</em></blockquote>
No hidden state, no hidden bugs.

<p dir="auto">Your programming environment becomes interactive by splitting your code into multiple cells! Changing one cell <strong>instantly shows effects</strong> on all other cells, giving you a fast and fun way to experiment with your model.</p>
<p dir="auto">In the example below, changing the parameter <code>A</code> and running the first cell will directly re-evaluate the second cell and display the new plot.</p>

<p dir="auto">Pluto uses syntax analysis to understand which packages are being used in a notebook, and it <strong>automatically manages a package environment</strong> for your notebook. You no longer need to install packages, you can directly import any registered package like <code>Plots</code> or <code>DataFrames</code> and use it.</p>
<p dir="auto">To ensure reproducibility, the information to exactly reproduce the package environment is <strong>stored in your notebook file</strong>. When someone else opens your notebook with Pluto, the exact same package environment will be used, and packages will work on their computer, automatically! <em><a href="https://github.com/fonsp/Pluto.jl/wiki/%F0%9F%8E%81-Package-management">more info</a></em></p>

<p dir="auto">Lastly, here's <em><strong>one more feature</strong></em>: Pluto notebooks have a <code>@bind</code> macro to create a <strong>live bond between an HTML object and a Julia variable</strong>. Combined with reactivity, this is a very powerful tool!</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer nofollow" href="https://user-images.githubusercontent.com/6933510/134825003-bd72ef08-677b-42fa-a655-e842868b10f6.gif"><img alt="@bind macro screencap" src="https://user-images.githubusercontent.com/6933510/134825003-bd72ef08-677b-42fa-a655-e842868b10f6.gif" width="50%" data-animated-image="" style="max-width: 100%;"></a></p>
<br>
<p dir="auto">You don't need to know HTML to use it! The <a href="https://github.com/fonsp/PlutoUI.jl">PlutoUI package</a> contains basic inputs like sliders and buttons. Pluto's interactivity is very easy to use, you will learn more from the featured notebooks inside Pluto!</p>
<p dir="auto">But for those who want to dive deeper - you can use HTML, JavaScript and CSS to write your own widgets! Custom update events can be fired by dispatching a <code>new CustomEvent("input")</code>, making it compatible with the <a href="https://observablehq.com/@observablehq/a-brief-introduction-to-viewof" rel="nofollow"><code>viewof</code> operator of observablehq</a>. Have a look at the JavaScript featured notebook inside Pluto!</p>

<p>Lightweight <em><strong>reactive</strong></em> notebooks for Julia <g-emoji class="g-emoji" alias="zap" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/26a1.png">âš¡</g-emoji> - welcome to Pluto!</p>
<p><em><strong>Explore models and share results</strong> in a notebook that is</em></p>
<ul>
<li><strong><em>reactive</em></strong> - when changing a function or variable, Pluto automatically updates all affected cells.</li>
<li><strong><em>lightweight</em></strong> - Pluto is written in pure Julia and is an installable package.</li>
<li><strong><em>modern</em></strong> - responsive, intuitive user experience; beautiful exported documents; custom themes.</li>
</ul>

<h2><a id="user-content-input" class="anchor" aria-hidden="true" href="#input"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Input</h2>
<p>The central idea is that Pluto notebooks are <em><strong>reactive</strong></em>, just like <a href="https://observablehq.com/@observablehq/observables-not-javascript" rel="nofollow">Observable notebooks</a>, but using Julia instead of JavaScript. Cells can be placed in any order - our intelligent AST parser figures out the dependencies between them and takes care of execution. When you change a variable, Pluto automatically re-runs the cells that refer to it.</p>
<p>Unlike Jupyter or Matlab, there is <strong>no mutable workspace</strong>, but rather a <em>one-to-one corresponce</em> between variables and code. In a Pluto notebook, the value of a variable <em>always</em> corresponds to the code that defines it.</p>
<h2><a id="user-content-output" class="anchor" aria-hidden="true" href="#output"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Output</h2>
<p>Cell output is simple: one cell outputs one variable, which is displayed using the richest available formatter. We believe that this limitation actually <a href="https://medium.com/@mbostock/a-better-way-to-code-2b1d2876a3a0" rel="nofollow"><em>makes programming easier</em></a>!</p>
<p>Your notebooks are saved as pure Julia files, which you can then import as if you had been programming in a regular editor all along. You can also export your notebook with output as attractive HTML and PDF documents. By reordering cells and hiding code, you have full control over how you tell your story.</p>

<p><em>(To developers: follow <a href="https://github.com/fonsp/Pluto.jl/blob/master/dev_instructions.md">these instructions</a> to start working on the package.)</em></p>
<p>After <a href="https://julialang.org/" rel="nofollow">installing julia</a>, add the package:</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> ]
(v1<span class="pl-c1">.0</span>) pkg<span class="pl-k">&gt;</span> add Pluto</pre></div>
<p>To run the notebook server:</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Pluto
julia<span class="pl-k">&gt;</span> Pluto<span class="pl-k">.</span><span class="pl-c1">run</span>(<span class="pl-c1">1234</span>)</pre></div>
<p>Then go to <a href="http://localhost:1234/" rel="nofollow"><code>http://localhost:1234/</code></a> to start coding!</p>

<p dir="auto">A Julia package which can help porting Python code into Julia. It does
a syntactical transformation of a Python file at the level of tokens.</p>
<p dir="auto">What this package is <em>not</em>:</p>
<ul dir="auto">
<li>
<p dir="auto">It is not a transpiler. Most of the time the output will be an
unrunnable mix of Python and Julia.</p>
</li>
<li>
<p dir="auto">It is not something which allows you to port code without knowing
both languages.</p>
</li>
<li>
<p dir="auto">It does not help you with the design changes you may need when
porting the code.</p>
</li>
</ul>
<p dir="auto">What it does do:</p>
<ul dir="auto">
<li>
<p dir="auto">It reduces the amount of typing you have to do for a literal port.
In particular it eliminates the most repetitive parts of the typing.</p>
</li>
<li>
<p dir="auto">It automates the error prone step of converting indentation-marked
blocks into <code>end</code>-delimited blocks.</p>
</li>
<li>
<p dir="auto">It retains comments, docstrings, and formatting.</p>
</li>
<li>
<p dir="auto">It supports adding custom translation rules which are appropriate
for your code base.</p>
</li>
</ul>

<p dir="auto">A custom translation rule is a function which receives a sequence of
tokens and returns a possibly modified sequence of tokens. To inspect
what tokens you currently have, use the <code>verbose</code> keyword argument to
<code>translate</code>. For examples of translation rules, see the implementation
of the standard translations in the source code. Enter your custom
translations as additional positional arguments of <code>translate</code>, or as
a vector of functions.</p>

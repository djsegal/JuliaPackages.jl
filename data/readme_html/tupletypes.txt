<p dir="auto">This package explores an interface to work with <code>Tuple{}</code>s.  See
<a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="84231098" data-permission-text="Title is private" data-url="https://github.com/JuliaLang/julia/issues/11547" data-hovercard-type="pull_request" data-hovercard-url="/JuliaLang/julia/pull/11547/hovercard" href="https://github.com/JuliaLang/julia/pull/11547">JuliaLang/julia#11547</a></p>
<p dir="auto">It provides the following methods to work with Tuples as though they are containers of their parameters.  It does not export these methods as they would clash with the Base methods of the same name.  As such, they must be called fully-qualified with the module name.</p>
<ul dir="auto">
<li><code>Tuples.length(T)</code>: The number of component types in a Tuple T.</li>
<li><code>Tuples.collect(T)</code>: Returns the component types inside the Tuple T as an iterable container</li>
<li><code>Tuples.getindex(T, i::Integer)</code>: Returns the i-th component of the Tuple T</li>
<li><code>Tuples.getindex(T, Is)</code>: Returns an iterable container with each element set to the i-th component, for each i in Is.</li>
</ul>
<p dir="auto">Additionally, it provides a convenience method for constructing Tuple types by calling <code>NTuple</code>, akin to the base method <code>ntuple</code>:</p>
<ul dir="auto">
<li><code>NTuple(f, n::Integer)</code>: Construct a Tuple{} such that the i-th component is the result of f(i).</li>
</ul>

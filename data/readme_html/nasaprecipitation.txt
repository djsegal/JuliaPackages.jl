<p dir="auto"><code>NASAPrecipitation.jl</code> is a Julia package that aims to streamline the following processes:</p>
<ul dir="auto">
<li>downloads of NASA Precipitation Measurement Mission datasets</li>
<li>basic analysis of said datasets</li>
<li>perform all the above operations innately over a given <a href="https://github.com/JuliaClimate/GeoRegions.jl"><code>GeoRegion</code></a></li>
</ul>
<p dir="auto"><code>NASAPrecipitation.jl</code> has been registered.  Install it via:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="] add NASAPrecipitation"><pre class="notranslate"><code>] add NASAPrecipitation
</code></pre></div>
<p dir="auto">To get the most recent release on the <code>#main</code> branch, do</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="] add NASAPrecipitation#main"><pre class="notranslate"><code>] add NASAPrecipitation#main
</code></pre></div>

<p dir="auto">In order to access NASA's EOSDIS OPeNDAP servers, the following steps are required for <a href="https://github.com/Alexander-Barth/NCDatasets.jl">NCDatasets.jl</a> to access the data:</p>
<ul dir="auto">
<li>You need to register an account with Earthdata</li>
<li>Create a <code>.netrc</code> file, and paste the following:
<ul dir="auto">
<li><code>machine urs.earthdata.nasa.gov login &lt;your login&gt; password &lt;your password&gt;</code></li>
</ul>
</li>
<li>Create a <code>.dodsrc</code> file, and paste the following:
<ul dir="auto">
<li><code>HTTP.COOKIEJAR=/&lt;home directory&gt;/.urs_cookies</code></li>
<li><code>HTTP.NETRC=/&lt;home directory&gt;/.netrc</code></li>
</ul>
</li>
</ul>

<p dir="auto">Please refer to the <a href="https://natgeo-wong.github.io/NASAPrecipitation.jl/dev/" rel="nofollow">documentation</a> for instructions and examples.  A working knowledge of the <code>GeoRegion</code> and <code>RegionInfo</code> supertypes used in <a href="https://github.com/JuliaClimate/GeoRegions.jl"><code>GeoRegions.jl</code></a> v2 is also needed.</p>
<p dir="auto"><em><strong>Tip:</strong> Both the <code>GeoRegions.jl</code> and <code>Dates</code> dependencies are reexported by <code>NASAPrecipitation.jl</code>, and therefore there is no need to call either <code>GeoRegions.jl</code> or <code>Dates</code> separately when using the <code>NASAPrecipitation.jl</code> package.</em></p>
<h2 dir="auto"><a id="user-content-supported-datasets" class="anchor" aria-hidden="true" href="#supported-datasets"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><strong>Supported Datasets</strong></h2>
<p dir="auto">The following datasets are supported:</p>
<ul dir="auto">
<li><strong>GPM IMERGv6</strong>, 0.1º resolution, 60ºS to 60ºN
<ul dir="auto">
<li>Final runs for Half-Hourly, Daily and Monthly Data</li>
<li>Early and Late runs of Half-Hourly and Daily Data</li>
</ul>
</li>
<li><strong>TRMM TMPAv7</strong>, 0.25º resolution, 50ºS to 50ºN
<ul dir="auto">
<li>Final runs for 3-Hourly, Daily and Monthly Data</li>
<li>Near Real-Time runs of 3-Hourly and Daily Data</li>
</ul>
</li>
</ul>
<p dir="auto">Only the calibrated precipitation data is downloaded, with units of rate in mm/s.</p>
<p dir="auto">If there is demand, I can easily add other datasets available on the <code>gpm1</code> and <code>disc2</code> NASA OPeNDAP servers to the mix as well. Please open an issue if you want me to do so.</p>

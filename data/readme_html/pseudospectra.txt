<p dir="auto">Pseudospectra is a Julia package for computing pseudospectra of
non-symmetric matrices, and plotting them along with eigenvalues
("spectral portraits"). Some related computations and plots are
also provided.</p>
<h2 dir="auto"><a id="user-content-mathematical-background" class="anchor" aria-hidden="true" href="#mathematical-background"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Mathematical background</h2>
<p dir="auto">Whereas the spectrum of a matrix is the set of its eigenvalues,
a pseudospectrum is the set of complex numbers "close" to the spectrum
in some practical sense.</p>
<p dir="auto">More precisely, the ϵ-pseudospectrum of a matrix <code>A</code>, <code>σ_ϵ(A)</code>, is the set of
complex numbers <code>λ</code> such that</p>
<ul dir="auto">
<li><code>λ</code> is an eigenvalue of some matrix <code>A+E</code>, where the norm of the perturbation <code>‖E‖ &lt; ϵ</code>, or</li>
<li>the resolvent norm <code>‖(A-λI)^(-1)‖ &gt; 1/ϵ</code>,</li>
</ul>
<p dir="auto">(the definitions are equivalent). This sense of "closeness" is trivial for
Hermitian matrices, but interesting for others.
Specifically, this package is currently limited to the unweighted 2-norm.</p>
<p dir="auto">Among other things, pseudospectra:</p>
<ul dir="auto">
<li>elucidate transient behavior hidden to eigen-analysis, and</li>
<li>indicate the utility of eigenvalues extracted via iterative methods like <code>eigs</code> (from the Arpack package).</li>
</ul>
<p dir="auto">See <a href="http://www.cs.ox.ac.uk/pseudospectra/intro.html" rel="nofollow">the Pseudospectra gateway</a>
for details, references, and more.</p>

<p dir="auto">Pseudospectra.jl (along with a QML-based GUI, in the forthcoming PseudospectraView
package) is essentially a translation of the acclaimed MATLAB-based EigTool
(<a href="http://www.comlab.ox.ac.uk/pseudospectra/eigtool" rel="nofollow">homepage here</a>),
code now hosted <a href="https://github.com/eigtool/eigtool">on GitHub</a>.</p>
<p dir="auto">No endorsement or promotion of Pseudospectra.jl by the authors of EigTool
is implied.</p>
<p dir="auto">Specific documentation for Pseudospectra is a work in progress; a draft is
available <a href="https://RalphAS.github.io/Pseudospectra.jl/dev" rel="nofollow">here</a>. See the
examples and tests for more.</p>

<p dir="auto">The plotting interface is somewhat schizophrenic. Drivers are included
for Plots.jl and/or PyPlot.jl (i.e., PyPlot is a useful back end for
Plots as used here; other Plots backends have been partially tested).
Experimental support for Makie.jl is also available.</p>
<p dir="auto">Although this package is designed with an eye to plotting results,
the computational routines are usable without a plotting package,
To avoid forcing potential users to install a particular one, none are
specified in the formal package requirements.  The <code>setgs</code> function
can import one conditionally.</p>
<p dir="auto">Some functions used for examples require other packages. They should
give a useful complaint if invoked without that support.</p>

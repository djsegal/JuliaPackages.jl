<h4><a id="user-content-available-polynomials" class="anchor" aria-hidden="true" href="#available-polynomials"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Available Polynomials</h4>
<pre><code>* `Polynomial` - Standard basis polynomials, `a_0 + a_1⋅x + a_2⋅x^2 + ⋯ + a_n⋅xⁿ`,  `n ∈ ℕ`
</code></pre>
<ul>
<li><code>ImmutablePolynomial</code> - Standard basis polynomials backed by a tuple for faster evaluation of values</li>
<li><code>SparsePolynomial</code> - Standard basis polynomial backed by a dictionary to hold  sparse high-degree  polynomials</li>
<li><code>LaurentPolynomial</code> - Laurent polynomials, <code>a_m⋅x^m + ⋯ a_n⋅x^n</code> <code>m ≤ n</code>, <code>m,n ∈ ℤ</code> backed by an offset array</li>
<li><code>ChebyshevT</code> - Chebyshev polynomials of the first kind</li>
</ul>

<p>Polynomial objects also have other methods:</p>
<ul>
<li>
<p>0-based indexing is used to extract the coefficients of <code>[a0, a1, a2, ...]</code>, coefficients may be changed using indexing
notation.</p>
</li>
<li>
<p><code>coeffs</code>: returns the entire coefficient vector</p>
</li>
<li>
<p><code>degree</code>: returns the polynomial degree, <code>length</code> is number of stored coefficients</p>
</li>
<li>
<p><code>variable</code>: returns the polynomial symbol as polynomial in the underlying type</p>
</li>
<li>
<p><code>norm</code>: find the <code>p</code>-norm of a polynomial</p>
</li>
<li>
<p><code>conj</code>: finds the conjugate of a polynomial over a complex field</p>
</li>
<li>
<p><code>truncate</code>: set to 0 all small terms in a polynomial;</p>
</li>
<li>
<p><code>chop</code> chops off any small leading values that may arise due to floating point operations.</p>
</li>
<li>
<p><code>gcd</code>: greatest common divisor of two polynomials.</p>
</li>
<li>
<p><code>Pade</code>: Return the
<a href="https://en.wikipedia.org/wiki/Pad%C3%A9_approximant" rel="nofollow">Pade approximant</a> of order <code>m/n</code> for a polynomial as a <code>Pade</code> object.</p>
</li>
</ul>

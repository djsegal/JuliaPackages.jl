<p dir="auto">This package provides a model, <code>MoistAirModel</code> to compute thermodynamic properties of moist air, using the <a href="https://github.com/longemen3000/ThermoState.jl">ThermoState</a> interface. The model uses real gas correlations as recommended by ASHRAE (see reference [5]).</p>
<p dir="auto">This package also provides the underlying models used for moist air, namely, dry air (<code>ASHRAEDryAir</code>) and water vapor (<code>ASHRAEWaterVapor</code>). those models have the same interface as the moist air model.</p>
<p dir="auto">This is a fork of <a href="https://github.com/pjabardo/Psychro.jl/">Psychro.jl</a> working with julia 1.5+.</p>

<h2 dir="auto"><a id="user-content-user-interface---thermodynamic-properties-of-moist-air-dry-air-and-saturated-water-vapor" class="anchor" aria-hidden="true" href="#user-interface---thermodynamic-properties-of-moist-air-dry-air-and-saturated-water-vapor"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>User interface - Thermodynamic properties of moist air, dry air and saturated water vapor.</h2>
<p dir="auto">The models defined in this package accept the following <code>ThermoState</code> thermodynamic states:</p>
<ul dir="auto">
<li><code>ASHRAEDryAir</code>: pressure,temperature</li>
<li><code>ASHRAEWaterVapor</code>: pressure,temperature</li>
<li><code>MoistAirModel</code> :
<ul dir="auto">
<li>pressure, temperature, amounts (any of <code>mol_fraction</code>,<code>mass_fraction</code>,<code>mol_number</code>,<code>mass_number</code>)</li>
<li>pressure,temperature, humidity spec*</li>
</ul>
</li>
</ul>
<p dir="auto">The models defined in this package support the following <code>ThermoState</code> functions:</p>
<ul dir="auto">
<li><code>mol_enthalpy</code>, <code>mass_enthalpy</code>, <code>total_enthalpy</code></li>
<li><code>mol_entropy</code>, <code>mass_entropy</code>, <code>total_entropy</code></li>
<li><code>mol_volume</code>, <code>mass_volume</code>, <code>total_volume</code></li>
<li><code>mol_enthalpy</code>, <code>mass_enthalpy</code></li>
<li><code>compressibility_factor</code></li>
</ul>
<p dir="auto">The package also provides its own functions, with <code>ThermoState</code> syntax, specific for moist air:</p>
<ul dir="auto">
<li>
<p dir="auto">Dry properties: properties per 1 kg of dry air:</p>
<ul dir="auto">
<li><code>dry_volume</code>,<code>dry_enthalpy</code>,<code>dry_entropy</code></li>
</ul>
</li>
<li>
<p dir="auto"><code>rel_hum</code> : Relative humidity, defined as the fraction between the amount of water in the air and the amount of water of saturated air at the same temperature and pressure (between 0 and 1)</p>
</li>
<li>
<p dir="auto"><code>hum_massfrac</code> : kg of water / kg of moist air</p>
</li>
<li>
<p dir="auto"><code>hum_molfrac</code> : mol of water / mol of moist air</p>
</li>
<li>
<p dir="auto"><code>hum_wetbulb</code> : wet-bulb temperature or adiabatic saturation temperature is the temperature a volume of air would have if cooled adiabatically to saturation by evaporation of water into it, always lower than the specified temperature (also known as dry-bulb temperature)</p>
</li>
<li>
<p dir="auto"><code>hum_ratio</code> : Humidity ratio (kg of vapor / kg of dry air)</p>
</li>
<li>
<p dir="auto"><code>hum_dewpoint</code>  temperature to which moist air must be cooled to become saturated,always lower than the specified temperature.</p>
</li>
</ul>

<p dir="auto">As mentioned before, the MoistAirModel accepts thermodynamic states of the form pressure-temperature-humidity spec. you can use the same function names as keywords in the <code>ThermoState.state</code> function.</p>

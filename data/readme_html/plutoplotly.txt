<p dir="auto">This package provides a wrapper type <code>PlutoPlot</code> around the <code>Plot</code> type from <a href="https://github.com/sglyon/PlotlyBase.jl">PlotlyBase.jl</a> that exposes the <a href="https://plotly.com/julia/" rel="nofollow">plotly library</a>.</p>
<p dir="auto">The wrapper mostly defines a custom <code>show</code> method specifically optimized for displaying inside of <a href="https://github.com/fonsp/Pluto.jl/">Pluto.jl</a> and adds the options of providing custom javascript functions to attach to the <a href="https://plotly.com/javascript/plotlyjs-events/" rel="nofollow">plolty JS events</a></p>
<p dir="auto">Basic use of this package is to load this inside Pluto instead of PlotlyBase or PlotlyJS, and then simply wrap the intended <code>Plot</code> objects from PlotlyBase inside a PlutoPlot as</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="p = Plot(args...)
PlutoPlot(p)"><pre>p <span class="pl-k">=</span> <span class="pl-c1">Plot</span>(args<span class="pl-k">...</span>)
<span class="pl-c1">PlutoPlot</span>(p)</pre></div>
<p dir="auto">One could also used the convenience exported function <code>plot</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="function plot(args...;kwargs...) 
	@nospecialize
	PlutoPlot(Plot(args...;kwargs...))
end"><pre><span class="pl-k">function</span> <span class="pl-en">plot</span>(args<span class="pl-k">...</span>;kwargs<span class="pl-k">...</span>) 
	<span class="pl-c1">@nospecialize</span>
	<span class="pl-c1">PlutoPlot</span>(<span class="pl-c1">Plot</span>(args<span class="pl-k">...</span>;kwargs<span class="pl-k">...</span>))
<span class="pl-k">end</span></pre></div>

<h2 dir="auto"><a id="user-content-persistent-layout" class="anchor" aria-hidden="true" href="#persistent-layout"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Persistent Layout</h2>
<p dir="auto">The custom show method relies on the <a href="https://plotly.com/javascript/plotlyjs-function-reference/#plotlyreact" rel="nofollow"><code>Plotly.react</code></a> function, which is optimized for updating or re-plotting data on an existing graph object, making it ideal in combination with Pluto's reactivity. The data transfer between Pluto and the browser also exploits Pluto's own <code>publish_to_js</code> function which is usually faster than standard JSON serialization, especially for big datasets.</p>
<p dir="auto">One advantage of using <code>react</code> as opposed to <code>newPlot</code> from the Plotly library is that one can have the layout (e.g. zoom level, camera view angle, etc...) persist across reactive re-runs by exploiting the <a href="https://plotly.com/javascript/uirevision/" rel="nofollow"><code>uirevision</code></a> attribute of Plotly layouts:</p>

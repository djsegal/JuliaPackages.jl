<p dir="auto"><strong>Braket.jl is not an officially supported AWS product.</strong></p>
<p dir="auto">This package is a Julia implementation of the <a href="https://github.com/aws/amazon-braket-sdk-python">Amazon Braket SDK</a> allowing customers to access <a href="https://aws.amazon.com/braket/quantum-computers/" rel="nofollow">Quantum Hardware and Simulators</a>.</p>
<p dir="auto">This is <em>experimental</em> software, and support may be discontinued in the future. For a fully supported SDK, please use
the <a href="https://github.com/aws/amazon-braket-sdk-python">Python SDK</a>. We may change, remove, or deprecate parts of the API when making new releases.
Please review the <a href="CHANGELOG.md">CHANGELOG</a> for information about changes in each release.</p>

<p dir="auto">You do <em>not</em> need a Python installation or the Python Amazon Braket SDK installed to use this package.
However, to use the Amazon Braket <a href="https://docs.aws.amazon.com/braket/latest/developerguide/braket-send-to-local-simulator.html" rel="nofollow">Local Simulators</a>
you'll need to install the sub-package <code>PyBraket.jl</code>,
included in this repository. See its <a href="PyBraket/README.md"><code>README</code></a> for more information.</p>
<p dir="auto">All necessary Julia packages will be installed for you when you run <code>Pkg.add("Braket")</code>
or <code>] instantiate</code> (if you're doing a <code>dev</code> install).</p>
<p dir="auto">If you want to run tasks on Amazon Braket's <a href="https://docs.aws.amazon.com/braket/latest/developerguide/braket-devices.html" rel="nofollow">managed simulators or QPUs</a> or run
<a href="https://docs.aws.amazon.com/braket/latest/developerguide/braket-jobs-works.html" rel="nofollow">managed jobs</a>,
you will need an AWS account and to have <a href="https://docs.aws.amazon.com/braket/latest/developerguide/braket-enable-overview.html" rel="nofollow">onboarded</a> with the Amazon Braket service.
<code>Braket.jl</code> can load your AWS credentials from your environment variables or your <code>~/.aws/config</code> file thanks to the <a href="https://github.com/JuliaCloud/AWS.jl"><code>AWS.jl</code></a> package.</p>

<p dir="auto">Keep in mind that the first qubit has index <code>0</code>, <strong>not</strong> index <code>1</code>.
Some Amazon Braket Hybrid Jobs return results as pickled objects, which are currently not decompressible using <code>Braket.jl</code>.
In such cases you may need to download and extract the results using <code>PyBraket.jl</code> and the Amazon Braket SDK.</p>

<p dir="auto">As Amazon Braket expands its featureset, we welcome contributions and development from the Julia community.</p>
<p dir="auto">What's currently implemented in <em>pure</em> Julia:</p>
<ul dir="auto">
<li>All of the <a href="https://github.com/aws/amazon-braket-schemas-python"><code>Amazon Braket schemas</code></a>.</li>
<li>Submitting <a href="https://docs.aws.amazon.com/braket/latest/developerguide/braket-jobs.html" rel="nofollow"><code>Amazon Braket Hybrid Jobs</code></a></li>
<li>Local jobs</li>
<li>Building and submitting circuits to managed simulators and QPUs</li>
<li>Reading results from managed simulators and QPUs</li>
<li>Cancelling tasks and jobs</li>
<li>Fetching quantum task and device information from AWS</li>
<li>Searching device information and looking up availability windows</li>
<li>Noise models</li>
<li>Convenience <code>Circuit</code> features</li>
<li>Pretty printing of <code>Circuit</code>s</li>
<li>Cost tracking</li>
<li>All gates and noise operations</li>
<li>Analog Hamiltonian Simulation</li>
</ul>

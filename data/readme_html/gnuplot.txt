<p><strong>Gnuplot.jl</strong> is a simple package able to send both data and commands from Julia to an underlying <a href="http://gnuplot.sourceforge.net/" rel="nofollow">gnuplot</a> process.  Its main purpose it to provide a fast and powerful data visualization framework, using an extremely concise Julia syntax.</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p>Install with:</p>
<div class="highlight highlight-source-julia"><pre>]add Gnuplot</pre></div>
<p>A working <a href="http://gnuplot.sourceforge.net/" rel="nofollow">gnuplot</a> package must be installed on your platform.</p>
<p>You may check the installed <strong>Gnuplot.jl</strong> version with:</p>
<div class="highlight highlight-source-julia"><pre>]st Gnuplot</pre></div>
<p>If the displayed version is not v1.3.0 you are probably having a dependency conflict. In this case try forcing installation of the latest version with:</p>
<div class="highlight highlight-source-julia"><pre>]add Gnuplot@<span class="pl-c1">1.3.0</span></pre></div>
<p>and check which package is causing the conflict.</p>
<p>Test package:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Gnuplot
<span class="pl-c1">println</span>(Gnuplot<span class="pl-k">.</span><span class="pl-c1">gpversion</span>())
<span class="pl-c1">test_terminal</span>()</pre></div>

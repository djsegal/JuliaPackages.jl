<p dir="auto"><a href="https://codecov.io/gh/jiachengzhang1/SwaggerMarkdown" rel="nofollow"><img src="https://camo.githubusercontent.com/85660e9e327ffe26c94d25f14a95ce0017783a5b15b50e85765767e087f346b8/68747470733a2f2f636f6465636f762e696f2f67682f6a69616368656e677a68616e67312f537761676765724d61726b646f776e2f6272616e63682f6d61737465722f67726170682f62616467652e7376673f746f6b656e3d474636355055414e4a32" alt="codecov" data-canonical-src="https://codecov.io/gh/jiachengzhang1/SwaggerMarkdown/branch/master/graph/badge.svg?token=GF65PUANJ2" style="max-width: 100%;"></a></p>
<p dir="auto">Swagger Markdown allows you to generate <code>swagger.json</code> for API documentation from the julia source code. The package uses marco to process the markdown that contains an API endpoint's documentation. The markdowon needs to follow the <code>paths</code> described by the OpenAPI Specification (<a href="https://swagger.io/specification/#paths-object" rel="nofollow">v3</a>, <a href="https://swagger.io/docs/specification/2-0/paths-and-operations/" rel="nofollow">v2</a>), and in YAML format.</p>
<p dir="auto">For more information about OpenAPI Specification, <a href="https://swagger.io/docs/specification/basic-structure/" rel="nofollow">version 3</a>, <a href="https://swagger.io/docs/specification/2-0/basic-structure/" rel="nofollow">version 2</a>.</p>
<p dir="auto">The package is inspired by, and it uses the similar style as <a href="https://github.com/Surnet/swagger-jsdoc">swagger-jsdoc</a>.</p>

<p dir="auto">Write markdown for the documentation of the API endpoints. The markdowon needs to follow the <code>paths</code> described by the OpenAPI Specification (<a href="https://swagger.io/specification/#paths-object" rel="nofollow">version 3</a>, <a href="https://swagger.io/docs/specification/2-0/paths-and-operations/" rel="nofollow">version 2</a>), and in YAML format.</p>

<h2 dir="auto"><a id="user-content-functionality" class="anchor" aria-hidden="true" href="#functionality"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Functionality</h2>
<ul dir="auto">
<li>Skyler identifies the coarsest abstract graph structure underlying a point cloud, and modells it. Currently, we are restricted to graphs with linear edges which satisfy conditions detailed below.</li>
<li>You can read the article <a href="https://www.aimsciences.org/article/doi/10.3934/fods.2021026" rel="nofollow">Reconstructing linearly embedded graphs: A first step to stratified space learning</a> which introduces the algorithm used in Skyler.</li>
</ul>
 
<h3 dir="auto"><a id="user-content-obtaining-abstract-structure" class="anchor" aria-hidden="true" href="#obtaining-abstract-structure"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Obtaining Abstract Structure</h3>
<p dir="auto">As input Skyler accepts a point cloud, as an <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$d \times n$</math-renderer> array, a parameter <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$\varepsilon$</math-renderer>, an inner radius, an outer radius, and an angle condition. The parameter <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$\varepsilon$</math-renderer> is the same paramter such that the point cloud is an <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$\varepsilon$</math-renderer>-sample of an embedded graph <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$|G|$</math-renderer>. The radii and angle condition are realted to assumptions on the embedding of <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$|G|$</math-renderer>, and their derivations can be found in <a href="https://www.aimsciences.org/article/doi/10.3934/fods.2021026" rel="nofollow">Reconstructing linearly embedded graphs: A first step to stratified space learning</a>. To obtain the abstract graph <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$G$</math-renderer>, run</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="ppc, model_verts, avg_verts = skyler(points, sample_epsilon, radius, EM_it=3, sigma=sample_epsilon/2)"><pre>ppc, model_verts, avg_verts <span class="pl-k">=</span> <span class="pl-c1">skyler</span>(points, sample_epsilon, radius, EM_it<span class="pl-k">=</span><span class="pl-c1">3</span>, sigma<span class="pl-k">=</span>sample_epsilon<span class="pl-k">/</span><span class="pl-c1">2</span>)</pre></div>
<p dir="auto">which returns a PartitionedPointCloud  <code>ppc</code>, an array <code>model_verts</code> containing the optimized vertex locations, and an array <code>avg_verts</code> containing the average point of each vertex cluster..</p>
<p dir="auto">A <code>PartitionedPointCloud</code> has the following fields:
- <code>Points</code> which is an array of all the points,
- <code>Strata</code> a <code>Dict{Int, Set}</code> which collates which points have been assigned to which stratum
- <code>Dimensions</code> a <code>Dict{Int, Set}</code> listing which strata are of each dimesnions,
- <code>Boundaries</code> an array which represents the boundary operator.</p>
<h3 dir="auto">
<a id="user-content-modeling-the-embedding" class="anchor" aria-hidden="true" href="#modeling-the-embedding"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Modeling the Embedding</h3>
<p dir="auto">To model the underlying structure, use</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="vertex_locations = structure= skyler(points, sample_epsilon, angle_condition, inner_radius, outer_radius, vertex_threshold, edge_threshold)"><pre>vertex_locations <span class="pl-k">=</span> structure<span class="pl-k">=</span> <span class="pl-c1">skyler</span>(points, sample_epsilon, angle_condition, inner_radius, outer_radius, vertex_threshold, edge_threshold)</pre></div>
<p dir="auto">which returns an <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$d \times n_v$</math-renderer> array, where <math-renderer class="js-inline-math" style="display: inline" data-static-url="https://github.githubassets.com/static" data-run-id="efc3343923df013eb2b9d57e6b17ea2a">$n_v$</math-renderer> is the number of vertices detected, with each column a modelled vertex location.</p>

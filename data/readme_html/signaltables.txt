<p dir="auto">Package <a href="https://github.com/ModiaSim/SignalTables.jl">SignalTables</a> (see <a href="https://modiasim.github.io/SignalTables.jl/stable/index.html" rel="nofollow">docu</a>)
provides abstract and concrete types and functions for <em>signal tables</em>.
A <em>signal table</em> is basically a table where the table columns can be multi-dimensional arrays with attributes.
Typically, simulation results, reference signals, table-based input signals, measurement data,
look-up tables can be represented by a signal table.</p>
<p dir="auto">A <em>signal table</em> is an <em>ordered dictionary</em> of <em>signals</em> with string keys. A <em>signal</em> can be defined in the following forms:</p>
<ul dir="auto">
<li>As <a href="https://modiasim.github.io/SignalTables.jl/stable/Functions/Signals.html#SignalTables.Var" rel="nofollow">Var</a> <em>dictionary</em> that has a required <em>values</em> key
(or an <em>alias</em> key) representing a <em>signal array</em> of any element type as function of the independent signal(s) (or is the k-th independent signal). A <em>signal array</em> is a <em>multi-dimensional array</em> with indices <code>[i1,i2,...,j1,j2,...]</code> to hold variable elements <code>[j1,j2,...]</code> at the <code>[i1,i2,...]</code> independent signal(s). If an element of a signal array is <em>not defined</em>, it has a value of <em>missing</em>.</li>
<li>As <a href="https://modiasim.github.io/SignalTables.jl/stable/Functions/Signals.html#SignalTables.Par" rel="nofollow">Par</a> <em>dictionary</em> that has a required <em>value</em> key (or and <em>alias</em> key) representing a constant of any type.</li>
<li>As <a href="https://modiasim.github.io/SignalTables.jl/stable/Functions/Signals.html#SignalTables.Map" rel="nofollow">Map</a> <em>dictionary</em> that has no required keys and collects attributes/meta-data that are associated with a Var, Par, Map, or signal table dictionary.</li>
</ul>
<p dir="auto">In all these dictionaries, additional attributes can be stored, for example <em>unit</em>, <em>info</em>, <em>variability</em> (continuous, clocked, ...), <em>interpolation</em>,
<em>extrapolation</em>, and user-defined attributes.</p>
<p dir="auto">This logical view is directly mapped to Julia data structures, but can be also mapped to data structures in other
programming languages. It is then possible to use existing textual or binary persistent serialization formats
(JSON, HDF5, BSON, MessagePack, ...) to store a signal table in these formats on file. Furthermore, a subset of a signal table
can also be stored in traditional tables (Excel, CSV, pandas, DataFrames.jl, ...) by <em>flattening</em> the multi-dimensional arrays and
not storing constants and attributes.</p>

<p dir="auto"><em>Concrete implementations</em> of the <a href="https://modiasim.github.io/SignalTables.jl/stable/Internal/AbstractSignalTableInterface.html" rel="nofollow">Abstract Signal Table Interface</a> are provided for:</p>
<ul dir="auto">
<li>
<p dir="auto"><a href="https://modiasim.github.io/SignalTables.jl/stable/Functions/SignalTables.html#SignalTables.SignalTable" rel="nofollow"><code>SignalTable</code></a> (included in SignalTables.jl).</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/ModiaSim/Modia.jl">Modia.jl</a> (a modeling and simulation environment; version &gt;= 0.9.0)</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/JuliaData/DataFrames.jl">DataFrames.jl</a>
(tabular data; first column is independent variable; <em>only scalar variables</em>))</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/JuliaData/Tables.jl">Tables.jl</a>
(abstract tables, e.g. <a href="https://github.com/JuliaData/CSV.jl">CSV</a> tables;
first column is independent variable; <em>only scalar variables</em>).</p>
</li>
</ul>
<p dir="auto"><em>Concrete implementations</em> of the <a href="https://modiasim.github.io/SignalTables.jl/stable/Internal/AbstractPlotInterface.html" rel="nofollow">Abstract Plot Interface</a> are provided for:</p>
<ul dir="auto">
<li>
<p dir="auto"><a href="https://github.com/JuliaPy/PyPlot.jl">PyPlot</a> (plots with <a href="https://matplotlib.org/stable/" rel="nofollow">Matplotlib</a> from Python;
via <a href="https://github.com/ModiaSim/SignalTablesInterface_PyPlot.jl">SignalTablesInterface_PyPlot.jl</a>),</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/JuliaPlots/GLMakie.jl">GLMakie</a> (interactive plots in an OpenGL window;
via <a href="https://github.com/ModiaSim/SignalTablesInterface_GLMakie.jl">SignalTablesInterface_GLMakie.jl</a>),</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/JuliaPlots/WGLMakie.jl">WGLMakie</a> (interactive plots in a browser window;
via <a href="https://github.com/ModiaSim/SignalTablesInterface_WGLMakie.jl">SignalTablesInterface_WGLMakie.jl</a>),</p>
</li>
<li>
<p dir="auto"><a href="https://github.com/JuliaPlots/CairoMakie.jl">CairoMakie</a> (static plots on file with publication quality;
via <a href="https://github.com/ModiaSim/SignalTablesInterface_CairoMakie.jl">SignalTablesInterface_CairoMakie.jl</a>).</p>
</li>
</ul>
<p dir="auto">Furthermore, there is a dummy implementation included in SignalTables that is useful when performing tests with runtests.jl,
in order that no plot package needs to be loaded during the tests:</p>
<ul dir="auto">
<li>SilentNoPlot (= all plot calls are silently ignored).</li>
</ul>

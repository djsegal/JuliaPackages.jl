<p dir="auto"><em>A tool for creating 3D stratigraphy from 2D geostatistical processes.</em></p>
<p dir="auto"><a href="https://github.com/JuliaEarth/StratiGraphics.jl/actions"><img src="https://camo.githubusercontent.com/60d842ef6b11bb43d014869c1f6afd61dddc593428fe33d614699a6e510d3672/68747470733a2f2f696d672e736869656c64732e696f2f6769746875622f616374696f6e732f776f726b666c6f772f7374617475732f4a756c696145617274682f53747261746947726170686963732e6a6c2f43492e796d6c3f6272616e63683d6d6173746572267374796c653d666c61742d737175617265" alt="" data-canonical-src="https://img.shields.io/github/actions/workflow/status/JuliaEarth/StratiGraphics.jl/CI.yml?branch=master&amp;style=flat-square" style="max-width: 100%;"></a> <a href="https://codecov.io/gh/JuliaEarth/StratiGraphics.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/a09c3d9714ad3239a7dae5fe55c7a2ac3af9b4b8576e7c93ee5f8881020f4907/68747470733a2f2f696d672e736869656c64732e696f2f636f6465636f762f632f6769746875622f4a756c696145617274682f53747261746947726170686963732e6a6c3f7374796c653d666c61742d737175617265" alt="" data-canonical-src="https://img.shields.io/codecov/c/github/JuliaEarth/StratiGraphics.jl?style=flat-square" style="max-width: 100%;"></a></p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="docs/stratigraphics.gif"><img src="docs/stratigraphics.gif" alt="StratiGraphics Animation" data-animated-image="" style="max-width: 100%;"></a></p>
<p dir="auto">This package provides an implementation of Markov-Poisson sampling as described
in <a href="https://searchworks.stanford.edu/view/12746435" rel="nofollow">Hoffimann 2018</a>. In this method,
geostatistical algorithms from the <a href="https://github.com/JuliaEarth/GeoStats.jl">GeoStats.jl</a>
framework are used to quickly generate surfaces of a 3D stratigraphic model.</p>

<p dir="auto">To illustrate the usage of the package, consider a set of satellite images containing
spatial patterns that we would like to reproduce in a 3D stratigraphic model:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="docs/flowimages.png"><img src="docs/flowimages.png" alt="Flow Images" style="max-width: 100%;"></a></p>
<p dir="auto">Each image can serve as a training image for a multiple-point geostatistical process
such as <a href="https://github.com/JuliaEarth/ImageQuilting.jl">ImageQuilting.jl</a>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using ImageQuilting

solver1 = IQ(:land =&gt; (TI=TI1, tilesize=(30,30)))
solver2 = IQ(:land =&gt; (TI=TI2, tilesize=(30,30)))
solver3 = IQ(:land =&gt; (TI=TI3, tilesize=(30,30)))

procs = [GeoStatsProcess(solver) for solver in [solver1,solver2,solver3]]"><pre><span class="pl-k">using</span> ImageQuilting

solver1 <span class="pl-k">=</span> <span class="pl-c1">IQ</span>(<span class="pl-c1">:land</span> <span class="pl-k">=&gt;</span> (TI<span class="pl-k">=</span>TI1, tilesize<span class="pl-k">=</span>(<span class="pl-c1">30</span>,<span class="pl-c1">30</span>)))
solver2 <span class="pl-k">=</span> <span class="pl-c1">IQ</span>(<span class="pl-c1">:land</span> <span class="pl-k">=&gt;</span> (TI<span class="pl-k">=</span>TI2, tilesize<span class="pl-k">=</span>(<span class="pl-c1">30</span>,<span class="pl-c1">30</span>)))
solver3 <span class="pl-k">=</span> <span class="pl-c1">IQ</span>(<span class="pl-c1">:land</span> <span class="pl-k">=&gt;</span> (TI<span class="pl-k">=</span>TI3, tilesize<span class="pl-k">=</span>(<span class="pl-c1">30</span>,<span class="pl-c1">30</span>)))

procs <span class="pl-k">=</span> [<span class="pl-c1">GeoStatsProcess</span>(solver) <span class="pl-k">for</span> solver <span class="pl-k">in</span> [solver1,solver2,solver3]]</pre></div>
<p dir="auto">We define a geological environment as a set of geological processes, a set of transition
probabilities between the processes, and an event duration process:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using StratiGraphics

# transition probabilities
P = rand(3,3)
P = P ./ sum(P, dims=2)

# event duration process
ΔT = ExponentialDuration(1.0)

env = Environment(procs, P, ΔT)"><pre><span class="pl-k">using</span> StratiGraphics

<span class="pl-c"><span class="pl-c">#</span> transition probabilities</span>
P <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">3</span>,<span class="pl-c1">3</span>)
P <span class="pl-k">=</span> P <span class="pl-k">./</span> <span class="pl-c1">sum</span>(P, dims<span class="pl-k">=</span><span class="pl-c1">2</span>)

<span class="pl-c"><span class="pl-c">#</span> event duration process</span>
ΔT <span class="pl-k">=</span> <span class="pl-c1">ExponentialDuration</span>(<span class="pl-c1">1.0</span>)

env <span class="pl-k">=</span> <span class="pl-c1">Environment</span>(procs, P, ΔT)</pre></div>
<p dir="auto">We can simulate the envinroment from an initial state (e.g. flat land) and for a number of
epochs to produce a geological record:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="nepochs = 10

init = LandState(zeros(100,100))

record = simulate(env, init, nepochs)"><pre>nepochs <span class="pl-k">=</span> <span class="pl-c1">10</span>

init <span class="pl-k">=</span> <span class="pl-c1">LandState</span>(<span class="pl-c1">zeros</span>(<span class="pl-c1">100</span>,<span class="pl-c1">100</span>))

record <span class="pl-k">=</span> <span class="pl-c1">simulate</span>(env, init, nepochs)</pre></div>
<p dir="auto">From the record, we can produce the strata in the form of surfaces:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="strata = Strata(record)"><pre>strata <span class="pl-k">=</span> <span class="pl-c1">Strata</span>(record)</pre></div>
<p dir="auto">We can choose between an <code>:erosional</code> (default) versus a <code>:depositional</code> stacking:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="strata = Strata(record, :depositional)"><pre>strata <span class="pl-k">=</span> <span class="pl-c1">Strata</span>(record, <span class="pl-c1">:depositional</span>)</pre></div>
<p dir="auto">We can then convert the surfaces into a 3D voxel model by specifying the vertical resolution:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="voxelize(strata, 50) # produce a 100x100x50 voxel model"><pre><span class="pl-c1">voxelize</span>(strata, <span class="pl-c1">50</span>) <span class="pl-c"><span class="pl-c">#</span> produce a 100x100x50 voxel model</span></pre></div>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="docs/voxelmodel.png"><img src="docs/voxelmodel.png" alt="Voxelized Models" style="max-width: 100%;"></a></p>
<p dir="auto">For a reproducible example, please check the <a href="https://github.com/JuliaEarth/GeoStatsTutorials">GeoStatsTutorials</a>.</p>

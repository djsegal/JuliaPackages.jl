<p>This package provides support for Stata '.dta' files from Stata verison 8 and above.</p>
<h1><a id="user-content-quickstart" class="anchor" aria-hidden="true" href="#quickstart"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Quickstart</h1>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Pkg
Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>DTALib.jl<span class="pl-pds">"</span></span>)

<span class="pl-k">using</span> DTALib
data <span class="pl-k">=</span> DTALib<span class="pl-k">.</span><span class="pl-c1">read</span>(<span class="pl-s"><span class="pl-pds">"</span>&lt;your_stata_dataset&gt;.dta<span class="pl-pds">"</span></span>)</pre></div>
<p>The read function returns a DTA structure which includes two fields: 1. the Header, and 2. the Data.</p>
<p>Continuing with the previous example. The Header can be separately accessed from the 'data' variable by the dot syntax, i.e.</p>
<div class="highlight highlight-source-julia"><pre>data<span class="pl-k">.</span>Header</pre></div>
<p>Similarly the Data can be accessed by</p>
<div class="highlight highlight-source-julia"><pre>data<span class="pl-k">.</span>Data</pre></div>

<h2><a id="user-content-currently-supports" class="anchor" aria-hidden="true" href="#currently-supports"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Currently supports:</h2>
<ul>
<li>Reading .dta files. Specifically it supports dta versions:
<ul>
<li>113: Stata 8</li>
<li>114: Stata 10</li>
<li>115: Stata 12</li>
<li>117: Stata 13</li>
<li>118: Stata 14, 15, and 16</li>
<li>119: Stata 15 and 16 (when dataset has more than 32,767 variables)</li>
</ul>
</li>
<li>Reading Header information including:
<ul>
<li>the Dataset Label</li>
<li>the Dataset Timestamp</li>
</ul>
</li>
<li>Missing values</li>
</ul>
<h2><a id="user-content-there-are-plans-to-support" class="anchor" aria-hidden="true" href="#there-are-plans-to-support"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>There are plans to support:</h2>
<ul>
<li>Writing to .dta files</li>
<li>Very long strings, specifically, strings longer than 2,045 bytes, aka strL's in Stata's terms</li>
<li>Variable labels: i.e. no labelling for binary (0,1) or categorical encoded variables (0, 1, 2, ...)</li>
</ul>

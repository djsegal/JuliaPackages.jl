<p><em>Julia package for the construction of quantum lattice systems.</em></p>
<p>Welcome to <a href="https://github.com/Quantum-Many-Body/QuantumLattices.jl">QuantumLattices</a>. Here we provide a general framework to construct the <strong>second quantized operator formed Hamiltonian of any quantum lattice system</strong>, with the inputs as simple as its description by natural languages. Combined with <a href="https://github.com/JuliaPy/SymPy.jl">SymPy</a>, this operator formed Hamiltonian supports <strong>complete symbolic computations</strong>, making it a convenient prerequisite of quantum many-body algorithms, such as <strong>TBA</strong>(tight-bind approximation), <strong>SCMF</strong>(self-consistent mean field theory), <strong>ED</strong>(exact diagonalization), <strong>CPT/VCA</strong>(cluster perturbation theory and variational cluster approach ), <strong>DMRG</strong>(density matrix renormalization group), etc. Generic interfaces are defined to give a unified access to these algorithms although their real implementations come in separate packages. Only minor modifications need be made when users alter from one algorithm to another.</p>

<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Introduction</h2>
<p>The core of the package is the construction of the <strong>operator representations of lattice Hamiltonians</strong>. This is based on the following mathematical observations that the operators in a lattice Hamiltonian:</p>
<ul>
<li><strong>act on local Hilbert spaces</strong>, and</li>
<li><strong>form algebras over the complex field</strong>.</li>
</ul>
<p>The first observation is the starting point of our <a href="https://quantum-many-body.github.io/QuantumLattices.jl/dev/tutorials/UnitcellDescription/Introduction/" rel="nofollow"><strong>unitcell description framework</strong></a> and the second is the mathematical foundation of our <strong>symbolic computing system</strong>.</p>
<p>It is noted that our implementation of the symbolic computation only involves</p>
<ul>
<li>the mathematical operations between a scalar and an operator, and</li>
<li>the mathematical operations between two operators.</li>
</ul>
<p>The symbolic operations between two scalars are <strong>not</strong> implemented because:</p>
<ul>
<li>in condensed matter physics, for many cases, only the numerical values of operators are important because the analytical expressions can be too complicated to analyze or they may even not exist;</li>
<li>our construction process of the operators and their mathematical operations are <strong>completely compatible with the <a href="https://github.com/JuliaPy/SymPy.jl">SymPy</a> package</strong>, therefore, a fully symbolic computation can be achieved by a simple combination of both.</li>
</ul>
<p>Another major aim of this package is to provide unified interfaces to access all quantum-many algorithms. Much of the job can be done by the construction of the operator-formed Hamiltonians, which serves as a common input for different algorithms. The remaining stuff concerns mainly with project management, such as result recording, data caching, parameter updating, code logging, dependency managing, etc. Utilities are provided to handle these tasks.</p>
<h2><a id="user-content-package-features" class="anchor" aria-hidden="true" href="#package-features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Package Features</h2>
<ul>
<li><strong>Unitcell Description Framework</strong>: by telling the information of the quantum lattice system within a unitcell, the construction of the symbolic representation of the Hamiltonian is just as simple as describing the system in a usual research paper.</li>
<li><strong>Complete symbolic computation</strong>: with only this package, symbolic computation between operators is realized whereas the coefficient of any operator remains numeric; by integrating it with <a href="https://github.com/JuliaPy/SymPy.jl">SymPy</a>, complete symbolic computation can be achieved and no modifications need be made on the methods in this package.</li>
<li><strong>Generic Engine-App Interfaces</strong>: by regarding the relation between algorithms and tasks as that between engines and apps, automatic project management is realized, including that of result recording, data caching, parameter updating, code logging, dependency managing, etc., moreover, all algorithms are initialized in quite similar ways with only minor modifications needed.</li>
</ul>
<h2><a id="user-content-supported-systems" class="anchor" aria-hidden="true" href="#supported-systems"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Supported Systems</h2>
<p>Three common kinds of systems in condensed matter physics are perfectly supported:</p>
<ul>
<li><strong>canonical fermionic systems</strong></li>
<li><strong>canonical/hard-core bosonic systems</strong></li>
<li><strong>SU(2) spin systems</strong></li>
</ul>
<p>Furthermore, other systems can be supported easily by extending the generic "protocols" provided in this package.</p>
<h2><a id="user-content-supported-algorithms" class="anchor" aria-hidden="true" href="#supported-algorithms"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Supported Algorithms</h2>
<p>Concrete algorithms are implemented in separate packages (still in progress):</p>
<ul>
<li><strong>TBA</strong>: tight-binding approximation for fermionic/bosonic systems;</li>
<li><strong>SCMF</strong>: self-consistent mean field theory for fermionic systems;</li>
<li><strong><a href="https://github.com/Quantum-Many-Body/ExactDiagonalization.jl">ED</a></strong>: exact diagonalization for fermionic/hard-core-bosonic/spin systems;</li>
<li><strong>CPT/VCA</strong>: cluster perturbation theory and variational cluster approach for fermionic systems;</li>
<li><strong>DMRG</strong>: density matrix renormalization group for fermionic/hard-core-bosonic/spin systems;</li>
<li><strong>LSWT</strong>: linear spin wave theory for local spin systems;</li>
<li><strong>FBFMSW</strong>: spin wave theory for flatband ferromagnets.</li>
</ul>

<p dir="auto">When working with currency amounts and exchange rates, it is very easy to do mistakes, e.g. summing up amounts in different currencies or applying an exchange rate for currency conversion in the wrong direction (what does 1.2 EURUSD mean again???).</p>
<p dir="auto">This little package provides currency-aware data types for amounts and exchange rates, together with mathematical operations between them. These operations are intrinsically safe, operations which are mathematically invalid or produce non-currency or currency-pair results are not defined.</p>
<p dir="auto">The currency (pair) information is encoded in the type system, therefore it has zero run-time cost (if type stable code is written).</p>

<p dir="auto">The following package provides similar functionality for currencies and currency amounts:</p>
<p dir="auto"><a href="https://github.com/JuliaFinance/Currencies.jl">https://github.com/JuliaFinance/Currencies.jl</a></p>
<p dir="auto">The main differences are:</p>
<ul dir="auto">
<li>Currencies.jl is more general, it aims supporting general assets. This package focuses on currency amounts and exchange rates.</li>
<li>Currencies.jl does not define exchange rates and mathematical operations between currency amounts and / or exchange rates (yet?).</li>
</ul>

<p dir="auto">Toolips Session is a Server Extension for toolips that enables full-stack web-development capabilities. This extension is loaded by default whenever the <code>Toolips.new_webapp</code> from the base toolips package is used. Ideally, your project would be setup that way instead of by adding this directly; but of course, you can still add this directly.</p>

</ul>

<h6 dir="auto"><a id="user-content-new-in-030" class="anchor" aria-hidden="true" href="#new-in-030"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>new in 0.3.0</h6>
<ul dir="auto">
<li>new <code>bind</code> method replaces old keys, can now use event and hotkeys quite easily with this method.</li>
<li><code>KeyMap</code>s and <code>InputMap</code>s</li>
<li>simplified <code>on</code>.</li>
<li>multi-client remote procedure <code>ComponentModifier</code> sessions. (<code>rpc!</code>, <code>open_rpc!</code>, <code>join_rpc!</code>, <code>disconnect_rpc!</code>, <code>close_rpc!</code>, <code>is_host</code>, <code>is_client</code>, <code>is_dead</code>)</li>
<li>new <code>script!</code> and <code>script</code> interface for creating client functions and observable functions in a consistent way.</li>
<li>Additional abstraction to Modifiers.</li>
<li>Client Modifiers</li>
<li><code>Modifier</code> Abstract type moved to <a href="https://github.com/ChifiSource/Toolips.jl">toolips</a></li>
<li><code>next!</code> method to set next animations and changes.</li>
<li><code>observe!</code> has become <code>script</code>and <code>script!</code> in accordance with new syntax.</li>
<li><code>append_first!</code> allows us to append a child to the top of the children.</li>
<li><code>push!</code> <code>ComponentModifier</code> bindings to put scripts into documents.</li>
<li><code>update!</code> allows any Julia type to be written to the "text/html" MIME. Eventually, this function will include an auto-mime algorithm similar to the one seen in Olive.jl <a href="https://github.com/ChifiSource/Olive.jl/blob/main/src/Core.jl">here</a> (towards the bottom of the file).</li>
</ul>

<p dir="auto">Use the following code to start Pluto instead of the standard <code>Pluto.run</code> way.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Pluto, PlutoLiveExport
Pluto.run(; on_export=live_export())"><pre><span class="pl-k">using</span> Pluto, PlutoLiveExport
Pluto<span class="pl-k">.</span><span class="pl-c1">run</span>(; on_export<span class="pl-k">=</span><span class="pl-c1">live_export</span>())</pre></div>
<p dir="auto"><em>Make sure to call <code>live_export</code> as a function (<code>live_export()</code>)</em></p>
<p dir="auto">By default this will automatically make exports to <code>~/Documents/Pluto Exports</code> every time you make a change in a notebook. You can change where these exports get placed by providing a path to the <code>export_path</code> keyword argument.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Pluto, PlutoLiveExport

<p dir="auto">The only positional argument of <code>live_export</code> is <code>format</code>, which can be used to switch export formats from the default of HTML. The currently supported export formats are:</p>
<ul dir="auto">
<li>html</li>
<li>statefile</li>
</ul>
<p dir="auto">As an example, to export to statefiles we would use <code>live_export("statefile")</code>.</p>
<blockquote>
<p dir="auto"><em><a href="https://plutopublish.com/docs/statefiles.md" rel="nofollow">what's a statefile??</a></em></p>
</blockquote>
<p dir="auto">To export into multiple formats simultaneously, pass a vector of formats instead.</p>

<h2><a id="user-content-non-windows-operating-systems" class="anchor" aria-hidden="true" href="#non-windows-operating-systems"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Non-Windows Operating Systems</h2>
<p><em>JavaCall and its derivatives do not work correctly on Julia 1.1 and Julia 1.2. On Julia 1.3, please set the environment variable <code>JULIA_COPY_STACKS</code>. On 1.1 and 1.2, and on 1.3 without <code>JULIA_COPY_STACKS</code> set, you may see segfaults or incorrect results. This is typically due to stack corruption. The Julia long-term-support version of 1.0.x continues to work correctly as before.</em></p>
<p>For Julia pre-1.5, consider the <a href="https://github.com/mkitti/RootTaskREPL.jl">RootTaskRepl.jl</a> package. With RootTaskREPL.jl, JavaCall is able to execute fine without the need of <code>JULIA_COPY_STACKS=1</code> with the exception of <code>@async</code> calls. Starting with the Julia 1.5, the REPL backend now runs on the root Task by default.</p>

<p>Do not set the environmental variable <code>JULIA_COPY_STACKS</code>. To use <code>jcall</code> with <code>@async</code> start Julia in the following way:</p>
<pre><code>$ julia -i -e "using JavaCall; JavaCall.init()"
</code></pre>

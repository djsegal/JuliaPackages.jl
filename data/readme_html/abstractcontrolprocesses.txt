<ul dir="auto">
<li>
<p dir="auto"><code>AbstractProcess</code><br>
Base abstract type for all lab processes. This should not be inherited from directly.</p>
</li>
<li>
<p dir="auto"><code>PhysicalProcess &lt;: AbstractProcess</code><br>
Pysical processes should inherit from this abstract type.</p>
</li>
<li>
<p dir="auto"><code>SimulatedProcess &lt;: AbstractProcess</code><br>
Simulated processes should inherit from this abstract type.</p>
</li>
</ul>

<ul dir="auto">
<li>
<p dir="auto"><code>ny = num_outputs(P::AbstractProcess)</code><br>
Return the number of outputs (measurement signals) of the process.</p>
</li>
<li>
<p dir="auto"><code>nu = num_inputs(P::AbstractProcess)</code><br>
Return the number of inputs (control signals) of the process.</p>
</li>
<li>
<p dir="auto"><code>yrange = outputrange(P::AbstractProcess)</code><br>
Return the range of outputs (measurement signals) of the process. <code>yrange</code> is a vector of
tuples,  <code>length(yrange) = num_outputs(P), eltype(yrange) = Tuple(Real, Real)</code></p>
</li>
<li>
<p dir="auto"><code>urange = inputrange(P::AbstractProcess)</code><br>
Return the range of inputs (control signals) of the process. <code>urange</code> is a vector of
tuples,  <code>length(urange) = num_inputs(P), eltype(urange) = Tuple(Real, Real)</code></p>
</li>
<li>
<p dir="auto"><code>isstable(P::AbstractProcess)</code><br>
Return true/false indicating whether or not the process is stable</p>
</li>
<li>
<p dir="auto"><code>isasstable(P::AbstractProcess)</code><br>
Return true/false indicating whether or not the process is asymptotically stable</p>
</li>
<li>
<p dir="auto"><code>h = sampletime(P::AbstractProcess)</code><br>
Return the sample time of the process in seconds.</p>
</li>
<li>
<p dir="auto"><code>b = bias(P::AbstractProcess)</code><br>
Return an input bias for the process. This could be, i.e., the constant input uâ‚€ around which a nonlinear system is linearized, or whatever other bias might exist on the input.<br>
<code>length(b) = num_inputs(P)</code></p>
</li>
<li>
<p dir="auto"><code>control(P::AbstractProcess, u::AbstractVector)</code><br>
Send a control signal to the process where <code>u</code> is an <code>AbstractVector</code> with dimension equal to <code>num_inputs(P)</code>.<br>
There is a default method which accepts <code>u</code> as a <code>Number</code> and forwards it as a <code>Vector</code> to the implemented version.</p>
</li>
<li>
<p dir="auto"><code>y = measure(P::AbstractProcess)</code><br>
Return a measurement from the process where <code>y</code> has length <code>num_outputs(P)</code></p>
</li>
<li>
<p dir="auto"><code>last_time = periodic_wait(P::AbstractProcess, last_time, dt)</code><br>
Should continue execution at start of next period, i.e. when <code>dt</code> seconds has passed since <code>last_time</code>.<br>
For a <code>PhysicalProcess</code> this is has a default implementation which sleeps from <code>time()</code> until
<code>last_time + dt</code> using <code>Libc.systemsleep</code>. Though this is more precise than using the standard
<code>sleep</code> in Julia, it also blocks the entire Julia thread it is running on so one has to be careful
with that sleep.<br>
For a <code>SimulatedProcess</code> it can either be running in realtime, in which case it should be implemented
similarly to the physical process, or it runs at full simulation speed, in which case the environment
should simulate a step corresponding to <code>dt</code> time when <code>periodic_wait</code> is called.</p>
</li>
<li>
<p dir="auto"><code>initialize(P::AbstractProcess)</code><br>
This function is called before any control or measurement operations are performed. During a call to <code>initialize</code>, one might set up external communications etc. After control is done,
the function <code>finalize</code> is called.</p>
</li>
<li>
<p dir="auto"><code>finalize(P::AbstractProcess)</code><br>
This function is called after any control or measurement operations are performed. During a call to <code>finalize</code>, one might finalize external communications etc. Before control is done,
the function <code>initialize</code> is called.</p>
</li>
</ul>

<p><strong>Note:</strong> This package should still work, but development has moved to <a href="https://github.com/JuliaRobotics/MeshCatMechanisms.jl">MeshCatMechanisms.jl</a> which offers more features and fewer binary dependencies. New users are encouraged to use MeshCatMechanisms.jl instead of this package.</p>
<hr>
<p>This package provides a simple tool for inspecting and manipulating the kinematic structure of a robot inside of an <a href="https://github.com/JuliaLang/IJulia.jl">IJulia</a> notebook. It relies on several other packages to provide this functionality:</p>
<ul>
<li><a href="https://github.com/tkoolen/RigidBodyDynamics.jl">tkoolen/RigidBodyDynamics.jl</a> to parse and represent kinematic structures</li>
<li><a href="https://github.com/RobotLocomotion/director">RobotLocomotion/director</a> for 3D visualization</li>
<li><a href="https://github.com/rdeits/DrakeVisualizer.jl">rdeits/DrakeVisualizer.jl</a> to communicate with <code>director</code></li>
<li><a href="https://github.com/rdeits/LCMCore.jl">rdeits/LCMCore.jl</a> for message-passing and communication</li>
</ul>

In each case, we'll be looking at the NASA Valkyrie robot, which we loaded from the URDF files in <a href="https://github.com/RobotLocomotion/drake/tree/master/drake/examples/Valkyrie">https://github.com/RobotLocomotion/drake/tree/master/drake/examples/Valkyrie</a>. <h3><a id="user-content-pure-kinematic-skeleton" class="anchor" aria-hidden="true" href="#pure-kinematic-skeleton"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Pure kinematic skeleton</h3>
<p>This visualization shows every joint in the model as a sphere, with a cylindrical rod between every pair of connected joints. It's useful primarily for inspecting the way various joints and bodies can move.</p>

<p>When rendered inside an IJulia notebook, a slider is created for each joint in the model:</p>

<h3><a id="user-content-inertial-ellipsoids" class="anchor" aria-hidden="true" href="#inertial-ellipsoids"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Inertial ellipsoids</h3>
<p>The inertial visualization adds to every link on the robot an ellipsoid which would have the same mass and rotational inertia as the robot's actual link. Note that different ellipsoids may have different densities, so this may produce strange results for robot links that have large rotational inertias but small masses (like the fingers in the Valkyrie model):</p>
<p><a target="_blank" rel="noopener noreferrer" href="img/val_inertia.png"><img src="img/val_inertia.png" alt="valkyrie inertia" style="max-width:100%;"></a></p>

<p dir="auto">DuckDB is a high-performance analytical database system. It is designed to be fast, reliable and easy to use. DuckDB provides a rich SQL dialect, with support far beyond basic SQL. DuckDB supports arbitrary and nested correlated subqueries, window functions, collations, complex types (arrays, structs), and more. For more information on the goals of DuckDB, please refer to <a href="https://duckdb.org/why_duckdb" rel="nofollow">the Why DuckDB page on our website</a>.</p>

<p dir="auto">For development, DuckDB requires <a href="https://cmake.org" rel="nofollow">CMake</a>, Python3 and a <code>C++11</code> compliant compiler. Run <code>make</code> in the root directory to compile the sources. For development, use <code>make debug</code> to build a non-optimized debug version. You should run <code>make unit</code> and <code>make allunit</code> to verify that your version works properly after making changes. To test performance, you can run <code>BUILD_BENCHMARK=1 BUILD_TPCH=1 make</code> and then perform several standard benchmarks from the root directory by executing <code>./build/release/benchmark/benchmark_runner</code>. The detail of benchmarks is in our <a href="benchmark/README.md">Benchmark Guide</a>.</p>

<p dir="auto">This package is used to set up a Julia Package Storage Server for mirror sites. The protocol details are
described in <a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="492482336" data-permission-text="Title is private" data-url="https://github.com/JuliaLang/Pkg.jl/issues/1377" data-hovercard-type="issue" data-hovercard-url="/JuliaLang/Pkg.jl/issues/1377/hovercard" href="https://github.com/JuliaLang/Pkg.jl/issues/1377">JuliaLang/Pkg.jl#1377</a>.</p>
<p dir="auto">TL;DR; A storage server contains all the static contents you need to download when you do <code>]add PackageName</code>.</p>
<p dir="auto">If you just want a cache layer service, <a href="https://github.com/JuliaPackaging/PkgServer.jl">PkgServer.jl</a> is a
better choice. This package is made to <em>permanently</em> keep the static contents.</p>
<p dir="auto">To set up a storage server, you'll need to:</p>
<ol dir="auto">
<li>get/update the static contents</li>
<li>serve them as a HTTP(s) service using nginx or whatever you like</li>
</ol>
<p dir="auto">This package is written to make step 1 easy and stupid.</p>

<ol dir="auto">
<li>add this package <code>]add StorageMirrorServer</code></li>
<li>modify the <a href="examples/gen_static_full.example.jl">example script</a> and save it as <code>gen_static.jl</code></li>
<li>pull/build data <code>julia gen_static.jl</code></li>
</ol>
<p dir="auto">You can read the not-so-friendly docstrings for advanced usage, but here are something you may want:</p>
<ul dir="auto">
<li>Redirect output <code>julia gen_static.jl &gt; log.txt 2&gt;&amp;1</code></li>
<li>Utilize multiple threads, set environment variable <code>JULIA_NUM_THREADS</code>. For example,
<code>JULIA_NUM_THREADS=8 julia gen_static.jl</code> would use 8 threads to pull data.</li>
</ul>

<p dir="auto">There are some environment variables that you can use to help configure the download worker <code>curl</code>:</p>
<ul dir="auto">
<li><code>BIND_ADDRESS</code> that passes to <code>curl --interface $BIND_ADDRESS</code>, this can be useful when multiple
network cards are available (newly added in <code>v0.2.1</code>)</li>
</ul>

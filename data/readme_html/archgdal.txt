<p><a href="http://gdal.org/" rel="nofollow">GDAL</a> is a translator library for raster and vector geospatial data formats that is released under an <a href="https://trac.osgeo.org/gdal/wiki/FAQGeneral#WhatlicensedoesGDALOGRuse" rel="nofollow">X/MIT</a> license by the <a href="http://www.osgeo.org/" rel="nofollow">Open Source Geospatial Foundation</a>. As a library, it presents an abstract data model to drivers for various <a href="http://www.gdal.org/formats_list.html" rel="nofollow">raster</a> and <a href="http://www.gdal.org/ogr_formats.html" rel="nofollow">vector</a> formats.</p>
<p>This package aims to be a complete solution for working with GDAL in Julia, similar in scope to <a href="https://pypi.python.org/pypi/GDAL/" rel="nofollow">the SWIG bindings for Python</a>. It builds on top of <a href="https://github.com/JuliaGeo/GDAL.jl">GDAL.jl</a>, and provides a high level API for GDAL, espousing the following principles.</p>
<h2><a id="user-content-principles-the-arch-way" class="anchor" aria-hidden="true" href="#principles-the-arch-way"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Principles (The Arch Way)</h2>
<p>(adapted from: <a href="https://wiki.archlinux.org/index.php/Arch_Linux#Principles" rel="nofollow">https://wiki.archlinux.org/index.php/Arch_Linux#Principles</a>)</p>
<ul>
<li><strong>simplicity</strong>: ArchGDAL tries to avoid unnecessary additions or modifications. It preserves the GDAL Data Model and requires minimal dependencies.</li>
<li><strong>modernity</strong>: ArchGDAL strives to maintain the latest stable release versions of GDAL as long as systemic package breakage can be reasonably avoided.</li>
<li><strong>pragmatism</strong>: The principles here are only useful guidelines. Ultimately, design decisions are made on a case-by-case basis through developer consensus. Evidence-based technical analysis and debate are what matter, not politics or popular opinion.</li>
<li><strong>user-centrality</strong>: Whereas other libraries attempt to be more user-friendly, ArchGDAL shall be user-centric. It is intended to fill the needs of those contributing to it, rather than trying to appeal to as many users as possible.</li>
<li><strong>versatility</strong>: ArchGDAL will strive to remain small in its assumptions about the range of user-needs, and to make it easy for users to build their own extensions/conveniences.</li>
</ul>

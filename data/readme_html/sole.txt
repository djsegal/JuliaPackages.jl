<h2 dir="auto"><a id="user-content-manifesto" class="anchor" aria-hidden="true" href="#manifesto"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Manifesto</h2>
<p dir="auto">Symbolic learning is machine learning based on symbolic logic. Its peculiarity lies in the fact that the learned models enclose an <strong>explicit knowledge representation</strong>, which offers many opportunities:</p>
<ul dir="auto">
<li>Verifying that the model's thought process is adequate for a given task;</li>
<li>Learning of new insights by simple inspection of the model;</li>
<li>Manual refinement of the model at a later time.</li>
</ul>
<p dir="auto">These levels of <strong>transparency</strong> (or <em>interpretability</em>) are generally not available with standard machine learning methods, thus, as AI permeates more and more aspects of our lives, symbolic learning is becoming increasingly popular. In spite of this, implementations of symbolic algorithms (e.g, extraction of decision trees or rules) are mostly scattered across different languages and machine learning frameworks.</p>
<p dir="auto"><em>Enough with this!</em> The lesser and lesser minoritarian theory of symbolic learning deserves a programming framework of its own!</p>
<h2 dir="auto"><a id="user-content-solejl" class="anchor" aria-hidden="true" href="#solejl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><em>Sole.jl</em></h2>
<p dir="auto"><em>Sole.jl</em> is a collection of Julia packages for symbolic learning and reasoning; <em>Sole.jl</em> covers <strong>a relatively wide range of functionality</strong> that is of interest for the symbolic community, but it also fills some gaps with a few functionalities for standard machine learning pipelines (e.g., feature selection on multimodal (un)structured data). At the time of writing, the framework comprehends the following packages:</p>
<ul dir="auto">
<li><a href="https://github.com/aclai-lab/SoleLogics.jl"><em>SoleLogics.jl</em></a> lays the <strong>logical foundations</strong> for symbolic learning. It provides a useful codebase for <a href="https://en.wikipedia.org/wiki/Computational_logic" rel="nofollow"><em>computational logic</em></a>, which features easy manipulation of:
<ul dir="auto">
<li>Propositional and (multi)modal logics (propositions, logical constants, alphabet, grammars, fuzzy algebras);</li>
<li><a href="https://en.wikipedia.org/wiki/Well-formed_formula" rel="nofollow">Logical formulas</a> (random generation, parsing, minimization);</li>
<li><a href="https://en.wikipedia.org/wiki/Interpretation_(logic)" rel="nofollow">Logical interpretations</a> (or models, e.g., <a href="https://en.wikipedia.org/wiki/Kripke_structure_(model_checking)" rel="nofollow">Kripke structures</a>);</li>
<li>Algorithms for <a href="https://en.wikipedia.org/wiki/Model_checking" rel="nofollow">model checking</a> (that is, checking that a formula is satisfied by an interpretation).</li>
</ul>
</li>
<li><a href="https://github.com/aclai-lab/SoleData.jl"><em>SoleData.jl</em></a> provides a <strong>data layer</strong> built on top of DataFrames.jl/Tables.jl. Its codebase is machine learning oriented and allows to:
<ul dir="auto">
<li>Instantiate and manipulate <a href="https://en.wikipedia.org/wiki/Multimodal_learning" rel="nofollow"><em>multimodal</em></a> datasets for (un)supervised machine learning;</li>
<li>Deal with <a href="https://en.wikipedia.org/wiki/Unstructured_data" rel="nofollow"><em>(un)structured</em> data</a> (e.g., graphs, images, time-series, etc.);</li>
<li>Describe datasets via basic statistical measures;</li>
<li>Save to/load from <em>npy/npz</em> format, as well as a custom CSV-based format (with interesting features such as <em>lazy loading</em> of datasets);</li>
<li>Perform basic data processing operations (e.g., windowing, moving average, etc.).</li>
</ul>
</li>
<li><a href="https://github.com/aclai-lab/SoleFeatures.jl"><em>SoleFeatures.jl</em></a> provides tools for filter-based <strong>feature selection</strong> on <a href="https://en.wikipedia.org/wiki/Unstructured_data" rel="nofollow"><em>(un)structured</em> data</a>. At this time, the package provides:
<ul dir="auto">
<li>3 native feature selection methods, plus a wrapper around python implementations from <em>scikit-learn</em> and <em>scikit-feature</em>;</li>
<li>Generalized feature selection methods that also apply to dimensional data (e.g., images or time-series), via a step of window-based flattening;</li>
<li>Specific methods for time-series feature selection, based on <a href="https://github.com/brendanjohnharris/Catch22.jl/">Catch22.jl</a>;</li>
<li>An easily extendible codebase, with abstraction layers similar to those of <em>scikit-learn</em>.</li>
</ul>
</li>
<li><a href="https://github.com/aclai-lab/SoleModels.jl"><em>SoleModels.jl</em></a> defines the building blocks of <strong>symbolic modeling and learning</strong>. It is the core of the framework, and it features:
<ul dir="auto">
<li>Definitions for symbolic models (decision trees/forests, rules, etc.);</li>
<li>Optimized data structures, useful when learning models from datasets;</li>
<li>Support for mixed, neuro-symbolic computation.</li>
</ul>
</li>
<li><a href="https://github.com/aclai-lab/SolePostHoc.jl"><em>SolePostHoc.jl</em></a> provides some tools for <strong>analyzing and post-processing the learned symbolic models</strong>. It features:
<ul dir="auto">
<li>Tools for inspecting and simplifying the models;</li>
<li>Rule extraction from decision forests;</li>
<li>Feature importance estimation.</li>
</ul>
</li>
</ul>
<p dir="auto">Altogether, <em>Sole.jl</em> makes for a powerful tool built with an eye to <strong>formal correctness</strong>, and is of use for both machine learning practitioners and computational logicians.</p>
<p dir="auto"><strong>Q:</strong> Ok, so what symbolic learning methods do you people provide?
<strong>A:</strong> At the moment, <a href="https://github.com/giopaglia/ModalDecisionTrees.jl"><em>ModalDecisionTrees.jl</em></a> is the only package compatible with Sole.jl, and it provides novel decision tree algorithms based on multimodal temporal and spatial logics for time-series and image classification. Checkout the related <a href="https://live.juliacon.org/talk/RQP9TG" rel="nofollow">talk at JuliaCon22</a>.</p>
<p dir="auto"><strong>Q:</strong> Why the name?
<strong>A:</strong> <em>Sole</em> stands for SymbOlic LEarning; it also means "sun" in Italian, a hint to the enlightening power of transparent modeling.</p>

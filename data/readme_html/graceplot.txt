<h2><a id="user-content-description" class="anchor" aria-hidden="true" href="#description"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Description</h2>
<p>The GracePlot.jl module is a simple control interface for Grace/xmgrace - providing more publication-quality plotting facilities to Julia.</p>
<ul>
<li>GracePlot.jl is ideal for seeding a Grace session with plot data before fine-tuning the output with Grace itself.</li>
<li>Grace "templates" (.par) files can then be saved/re-loaded to maintain a uniform appearance in publication.</li>
<li>The user is encouraged to pre-process data using math facilities from Julia instead of those built-in to Grace.</li>
</ul>

<p>The <a href="sample/">sample</a> directory contains a few demonstrations on how to use GracePlot.jl.</p>
<p>The <a href="sample/template/">template</a> directory contains a repository of sample Grace template (parameter) files.</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p>The GracePlot.jl module requires the user to install the Grace plotting tool:<br>
<a href="http://plasma-gate.weizmann.ac.il/Grace/" rel="nofollow">http://plasma-gate.weizmann.ac.il/Grace/</a></p>
<p>More detailed instructions can be found <a href="https://github.com/ma-laforge/HowTo/blob/master/grace/grace_install.md#Installation">here</a></p>
<h2><a id="user-content-configuration" class="anchor" aria-hidden="true" href="#configuration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Configuration</h2>
<p>By default, GracePlot.jl assumes Grace is accessible from the environment path as <code>xmgrace</code>.  To specify a different command/path, simply set the <code>GRACEPLOT_COMMAND</code> environment variable.</p>
<p>The value of <code>GRACEPLOT_COMMAND</code> can therefore be set from <code>.juliarc.jl</code> with the following:</p>
<pre><code>ENV["GRACEPLOT_COMMAND"] = "/home/laforge/bin/xmgrace2"
</code></pre>

<h3><a id="user-content-axes" class="anchor" aria-hidden="true" href="#axes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Axes</h3>
<p>Objects describing axis types are created with the <code>paxes</code> function:</p>
<pre><code>log_lin = paxes(xscale = :log, yscale = :lin)
</code></pre>
<p><strong>Supported scales:</strong> <code>:lin</code>, <code>:log</code>, <code>:reciprocal</code>.</p>
<p>The <code>paxes</code> function also allows the user to specify axis ranges:</p>
<pre><code>ax_rng = paxes(xmin = 0.1, xmax = 1000, ymin = 1000, ymax = 5000)
</code></pre>
<h3><a id="user-content-line-style" class="anchor" aria-hidden="true" href="#line-style"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Line Style</h3>
<p>Objects describing line style are created with the <code>line</code> function:</p>
<pre><code>default_line = line(style=:ldash, width=8, color=1)
</code></pre>
<p><strong>Supported styles:</strong> <code>:none</code>, <code>:solid</code>, <code>:dot</code>, <code>:dash</code>, <code>:ldash</code>, <code>:dotdash</code>, <code>:dotldash</code>, <code>:dotdotdash</code>, <code>:dotdashdash</code>.</p>
<h3><a id="user-content-glyphs" class="anchor" aria-hidden="true" href="#glyphs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Glyphs</h3>
<p>Objects describing display glyphs (markers/symbols) are created with the <code>glyph</code> function:</p>
<pre><code>glyph(shape=:diamond, color=5)
</code></pre>
<p><strong>Supported shapes:</strong> <code>:circle</code>, <code>:o</code>, <code>:square</code>, <code>:diamond</code>, <code>:uarrow</code>, <code>:larrow</code>, <code>:darrow</code>, <code>:rarrow</code>, <code>:cross</code>, <code>:+</code>, <code>:diagcross</code>, <code>:x</code>, <code>:star</code>, <code>:*</code>, <code>:char</code> (see demo2 for use of <code>:char</code>).</p>
<h2><a id="user-content-known-limitations" class="anchor" aria-hidden="true" href="#known-limitations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Known Limitations</h2>
<p>GracePlot.jl currently provides a relatively "bare-bones" interface (despite offering significant functionality).</p>
<ul>
<li>Does not currently provide much in terms of input validation.</li>
<li>Does not support the entire Grace control interface.</li>
<li>In particular: GracePlot.jl does not support Grace math operations.  Users are expected to leverage Julia for processing data before plotting.</li>
<li>On certain runs, Grace will complain that some commands cannot be executed... almost like commands are sent too fast for Grace, or something...  Not sure what this is.  Try re-running.</li>
</ul>

<p>GracePlot.jl will post-process SVG files in an attempt to support the W3C 1999 standard.  The changes enable most new web browsers to display the SVG outputs.  Note, however, that text will not appear correctly on these plots.</p>
<p>The EPS format is therefore suggested if high-quality vector plots are desired.</p>

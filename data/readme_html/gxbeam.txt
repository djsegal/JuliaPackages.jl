<p dir="auto"><em>Pure Julia Implementation of Geometrically Exact Beam Theory</em></p>
<p dir="auto">Author: Taylor McDonnell</p>
<p dir="auto"><strong>GXBeam</strong> is a pure Julia implementation of Geometrically Exact Beam Theory, originally based on the open source code <a href="https://cdmhub.org/resources/367" rel="nofollow">GEBT</a> and its associated papers<a href="#1">[1]</a><a href="#2">[2]</a>, though it has since been augmented with a number of additional features.</p>

<ul dir="auto">
<li>Performs multiple types of analyses including:
<ul dir="auto">
<li>Linear/Nonlinear static analyses</li>
<li>Linear/Nonlinear steady-state analyses</li>
<li>Linear/Nonlinear eigenvalue analyses (by linearizing about a steady state condition)</li>
<li>Linear/Nonlinear time-marching dynamic analyses</li>
</ul>
</li>
<li>Accurately models arbitrary systems of interconnected highly flexible composite beams.
<ul dir="auto">
<li>Captures all geometric nonlinearities due to large deflections and rotations (subject to a small strain assumption)</li>
<li>Models angular displacements of any magnitude using only three parameters</li>
<li>Uses the full 6x6 Timoshenko beam stiffness matrix</li>
</ul>
</li>
<li>Calculate section compliance and inertia matrices and compute strain recovery
<ul dir="auto">
<li>Uses quadrilateral finite elements rather than classical lamiante theory for much better accuracy and cross coupling</li>
<li>Allows for general geometry with inhomogenous properties and anisotropic behavior (computes full 6x6 matrix)</li>
<li>Ply materials are general orthotropic</li>
<li>Provides convenience method for paramterizing airfoil layups</li>
<li>Strain recovery functionality to compute stresses and strains on the mesh using the results of the beam solution</li>
<li>Tsai-wu failure criteria</li>
</ul>
</li>
<li>Models time-varying distributed forces/moments including
<ul dir="auto">
<li>Point and distributed loads which remain fixed in the body-frame</li>
<li>Point and distributed loads which rotate with the structure</li>
<li>Loads due to known body frame velocities and accelerations</li>
<li>Gravitational loads acting on beam elements and point masses</li>
<li>Loads resulting from stiffness-proportional structural damping</li>
</ul>
</li>
<li>Optional <a href="https://github.com/SciML/DifferentialEquations.jl">DifferentialEquations</a> interface.
<ul dir="auto">
<li>Constant mass matrix differential algebraic equation formulation</li>
<li>Fully implicit differential algebraic equation formulation</li>
</ul>
</li>
<li>Provides derivatives with ForwardDiff (including overloading internal solvers with implicit analytic methods)</li>
<li>Result visualization using <a href="https://github.com/jipolanco/WriteVTK.jl">WriteVTK</a></li>
<li>Verified and validated against published analytical and computational results.  See the examples in the <a href="https://flow.byu.edu/GXBeam.jl/dev/" rel="nofollow">documentation</a>.</li>
</ul>

<p dir="auto">By using the simplest possible shape functions (constant or linear shape functions), this package avoids using numerical quadrature except when integrating applied distributed loads (which can be pre-integrated).  As a result, element properties are approximated as constant throughout each beam element and a relatively large number of beam elements may be necessary to achieve grid-independent results.  More details about the convergence of this package may be found in the <a href="https://flow.byu.edu/GXBeam.jl/dev/examples/tipmoment" rel="nofollow">examples</a>.</p>
<p dir="auto">This package does not currently model cross section warping, and therefore should not be used to model open cross sections (such as I, C, or L-beams).  The one exception to this rule is if the beam's width is much greater than its height, in which case the beam may be considered to be strip-like (like a helicopter blade).</p>
<p dir="auto">This package relies on the results of linear cross-sectional analyses.  Most notably, it does not model the nonlinear component of the Trapeze effect, which is the tendency of a beam to untwist when subjected to axial tension.  This nonlinear effect is typically most important when modeling rotating structures such as helicopter blades due to the presence of large centrifugal forces.  It is also more important when modeling strip-like beams than for modeling closed cross-section beams due to their low torsional rigidity.</p>

<p dir="auto"><a href="https://cdmhub.org/resources/367" rel="nofollow">GEBT</a>: Open source geometrically exact beam theory code developed in Fortran as a companion to the proprietary cross sectional analysis tool <a href="https://analyswift.com/vabs-cross-sectional-analysis-tool-for-composite-beams/" rel="nofollow">VABS</a>.  The theory for this code is provided in references <a href="#1">1</a> and <a href="#2">2</a>.  GXBeam was originally developed based on this package and its associated papers, but has since been augmented with additional features.</p>
<p dir="auto"><a href="https://www.nrel.gov/wind/nwtc/beamdyn.html" rel="nofollow">BeamDyn</a>: Open source geometrically exact beam theory code developed in Fortran by NREL as part of the OpenFAST project.  This code was also developed based on <a href="https://cdmhub.org/resources/367" rel="nofollow">GEBT</a>, but uses Legendre spectral finite elements.  This allows for exponential rather than algebraic convergence when the solution is smooth.  This makes this code a good candidate for use when analyzing beams with smoothly varying properties.  Unfortunately, the code is limited to analyzing a single beam, rather than an assembly of beams.</p>
<p dir="auto">The cross sectional analysis uses the same underlying theory as in <a href="https://becas.dtu.dk" rel="nofollow">BECAS</a>, but was written to be fast and optimization-friendly.  <a href="https://analyswift.com/vabs-cross-sectional-analysis-tool-for-composite-beams/" rel="nofollow">VABS</a> and <a href="https://www.nrel.gov/wind/nwtc/precomp.html" rel="nofollow">PreComp</a> are other popular tools for composite cross sectional analysis.  The former is not freely available, whereas the latter is lower fidelity as it is based on classical laminate theory.</p>

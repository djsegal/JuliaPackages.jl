<p dir="auto">RemoteHPC attempts to wrap all the usual interactions one might have with a remote HPC cluster in a restAPI webserver that runs on the frontend of the cluster.</p>
<h2 dir="auto"><a id="user-content-features" class="anchor" aria-hidden="true" href="#features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Features</h2>
<ul dir="auto">
<li>Store locally connection information to remote servers using <code>Server</code></li>
<li>Remotely store information for available executables and execution environments using <code>save(server, Exec(...))</code> and <code>save(server, Environment(...))</code>.</li>
<li>Load remotely stored info using <code>load(server, Exec("&lt;label&gt;"))</code> and <code>load(server, Environment("&lt;label&gt;"))</code>.</li>
<li>Represent a line in a jobscript using <code>Calculation(exec::Exec, infile::String, outfile::String, run::Bool, parallel::Bool)</code>.</li>
<li>Save and submit a job with <code>save(server, jobdir, jobname, environment, calculations)</code> and <code>submit(server, jobdir)</code> or combine both steps with <code>submit(server, jobdir, jobname, environment, calculations)</code>.</li>
<li>The state of a job can be retrieved by <code>state(server, jobdir)</code>.</li>
<li>A job can be aborted using <code>abort(server, jobdir)</code>.</li>
<li>Support for running jobs with <code>SLURM</code>, <code>HyperQueue</code>, or <code>Bash</code>.</li>
<li>remote file operations: <code>read</code>,<code>write</code>, <code>rm</code>, <code>mtime</code>, <code>link</code>, etc.</li>
<li>Starting a remote server with <code>start(server)</code> and automatic creation of ssh tunnels by specifying <code>server.local_tunnel</code>, useful when the frontend of a cluster is behind a proxy.</li>
</ul>

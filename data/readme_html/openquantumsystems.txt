<p dir="auto"><strong>OpenQuantumSystems.jl</strong> is a numerical framework written in Julia that makes it easy to simulate various kinds of open quantum systems with main focus on quantum biology in finite basis. It is inspired by the
<a href="https://github.com/tmancal74/quantarhei">quantarhei</a> and
<a href="https://github.com/qojulia/QuantumOptics.jl">QuantumOptics.jl</a>.</p>

<p dir="auto">What is OpenQuantumSystems.jl capable of right now?</p>
<ul dir="auto">
<li>It only supports systems with a finite basis.</li>
<li>Calculate Hamiltonian for Aggregates of molecules. Molecules have system Hamiltonian that consists only of relevant degrees of freedom (electronic basis) and bath degrees of freedom. Bath consists of LHOs with shifted potentials for excited local electronic states.</li>
<li>Trace over bath degrees of freedom using Franck-Condon Factors and reduced density matrix.</li>
<li>It can calculate dynamics for aggregate named as follows
<ul dir="auto">
<li>Exact dynamics (calculate U(t) operator)</li>
<li>Schrodinger dynamics</li>
<li>Liuville-von Neumann dynamics</li>
<li>Quantum Master Equation dynamics</li>
</ul>
</li>
<li>OQS can also prepare the initial condition for density matrix as a laser-excited pulse.</li>
<li>It can evaluate the remaining part of the bath. The majority of my master thesis proposes various kinds of ansatzes that aim to find a closed-form of QME with RDM.</li>
<li>Scoring of obtained RDM in time in comparison to another RDM.</li>
<li>Calculate memory kernel as superoperator.</li>
</ul>
<p dir="auto">What can we expect in the near future?</p>
<ul dir="auto">
<li>Iterative approach for bath part of the reduced density matrix.</li>
<li>Calculation of rate constants from dynamics and memory kernel.</li>
<li>Better documentation with examples of how to use this package.</li>
<li>QME with iterative correction of bath part.</li>
<li>Calculation of dipoles for molecules given coordinates.</li>
<li>Implementation of Foerster and modified Redfield theories of excitation energy transfer for finite systems (and perhaps special infinite cases).</li>
</ul>
<p dir="auto">What would be nice to have?</p>
<ul dir="auto">
<li>Decomposition of mixed states into a linear combination of pure states using linear programming.</li>
<li>Loading Hamiltonian and data storage for aggregate.</li>
<li>GPU support for Schrodinger equation and possibly state decomposition.</li>
<li>Anharmonic oscillators.</li>
<li>Double excited states.</li>
<li>Interface with quantarhei, or creating an interface in quantarhei package.</li>
</ul>

<p dir="auto">You can use <code>make.bat</code> or <code>make</code> file for testing, but it will install all the dependencies before executing tests. I advise using jupyter notebook <code>src/local_test.ipynb</code> with julia kernel for tests. Package <code>Revise.jl</code> will load all the functions and by using <code>include("src/test_file.jl")</code> you can quickly debug and add new tests.</p>

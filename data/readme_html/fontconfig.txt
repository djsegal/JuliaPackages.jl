<p dir="auto"><code>Pattern</code> corresponds to the fontconfig type <code>FcPattern</code>. It respresents a set
of font properties used to match specific fonts.</p>
<p dir="auto">It can be constructed in two ways. First with zero or more keyword arguments
corresponding to fontconfig font properties.</p>

<p dir="auto">For example</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="Fontconfig.Pattern(family=&quot;Helvetica&quot;, size=10, hinting=true)"><pre>Fontconfig<span class="pl-k">.</span><span class="pl-c1">Pattern</span>(family<span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>Helvetica<span class="pl-pds">"</span></span>, size<span class="pl-k">=</span><span class="pl-c1">10</span>, hinting<span class="pl-k">=</span><span class="pl-c1">true</span>)</pre></div>

<p dir="auto">The primary functionality fontconfig provides is matching font patterns. In
Fontconfig.jl this is done with the <code>match</code> function, corresponding to <code>FcMatch</code>
in fontconfig.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="match(pat::Pattern)"><pre><span class="pl-c1">match</span>(pat<span class="pl-k">::</span><span class="pl-c1">Pattern</span>)</pre></div>
<p dir="auto">It takes a <code>Pattern</code> and return a <code>Pattern</code> corresponding to the nearest
matching installed font.</p>

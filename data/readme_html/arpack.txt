<p>Julia wrapper for the <a href="https://github.com/opencollab/arpack-ng/">arpack</a> library
designed to solve large scale eigenvalue problems.</p>

<p>You can install Arpack.jl through the Julia package manager:</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>Arpack<span class="pl-pds">"</span></span>)</pre></div>
<p>Arpack.jl will use <a href="https://github.com/JuliaPackaging/BinaryProvider.jl">BinaryProvider.jl</a>
to automatically install the Arpack binaries.</p>

<p>If you get</p>
<pre><code>ERROR: LoadError: LibraryProduct(nothing, ["libarpack"], :libarpack, "Prefix(~/.julia/packages/Arpack/cu5By/deps/usr)") is not satisfied, cannot generate deps.jl!
</code></pre>
<p>when building Arpack, it may be because your Julia installation uses your system
blas for which the symbols are not suffixed by <code>_64_</code> while this is required by
the compiled binaries provided by <a href="https://github.com/JuliaPackaging/BinaryProvider.jl">BinaryProvider.jl</a>.
This is notably the case on <a href="https://www.archlinux.org/" rel="nofollow">ArchLinux</a>.
In these case, compile binaries that do not require this suffix as follows.
Download the source of the <a href="https://github.com/opencollab/arpack-ng/releases/tag/3.5.0">v3.5.0 of arpack-ng</a>,
extract it in some <code>&lt;directory&gt;</code>, build it and do (note that you may need to
update <code>cu6By</code> to match the one printed in the error message printed above).</p>
<pre><code>$ cp &lt;directory&gt;/arpack-ng-3.5.0/SRC/.libs/libarpack.so.2.0.0 ~/.julia/packages/Arpack/cu5By/deps/usr/lib/
$ julia -e 'import Pkg; Pkg.build("Arpack")'
  Building Arpack â†’ `~/.julia/packages/Arpack/UiiMc/deps/build.log`
</code></pre>

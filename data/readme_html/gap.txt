<h1 dir="auto"><a id="user-content-gapjl-julia-package" class="anchor" aria-hidden="true" href="#gapjl-julia-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>GAP.jl Julia package</h1>
<p dir="auto">This repository contains the <a href="src/GAP.jl">GAP.jl</a> Julia package, as well as the GAP packages
<a href="pkg/JuliaInterface"><code>JuliaInterface</code></a> and <a href="pkg/JuliaExperimental"><code>JuliaExperimental</code></a>
developed for the <a href="https://www.gap-system.org/" rel="nofollow">GAP</a>-<a href="https://julialang.org/" rel="nofollow">Julia</a> integration
as part of the <a href="https://www.oscar-system.org" rel="nofollow">OSCAR project</a>.</p>
<p dir="auto"><em>WARNING</em>: GAP.jl is intended as a low-level interface between GAP
and Julia. Therefore, for the most part it does not attempt (besides some
general conveniences) to provide a very “Julia-ish” interface to GAP
objects and functions, nor a “GAP-ish” interface to Julia objects and
functions. Instead, this is left to higher-level code, for example in the
<a href="https://github.com/oscar-system/Oscar.jl">Oscar.jl</a> package.</p>

<h2 dir="auto"><a id="user-content-basic-usage" class="anchor" aria-hidden="true" href="#basic-usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Basic usage</h2>
<p dir="auto">After entering the following in Julia,</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using GAP"><pre><span class="pl-k">using</span> GAP</pre></div>
<p dir="auto">one may access any global GAP variable by prefixing its name with <code>GAP.Globals.</code>, and
call GAP functions like Julia functions. For example:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; GAP.Globals.SymmetricGroup(3)
GAP: SymmetricGroup( [ 1 .. 3 ] )"><pre>julia<span class="pl-k">&gt;</span> GAP<span class="pl-k">.</span>Globals<span class="pl-k">.</span><span class="pl-c1">SymmetricGroup</span>(<span class="pl-c1">3</span>)
GAP<span class="pl-k">:</span> <span class="pl-c1">SymmetricGroup</span>( [ <span class="pl-c1">1</span> <span class="pl-k">..</span> <span class="pl-c1">3</span> ] )</pre></div>
<p dir="auto">The Julia types <code>Int64</code> and <code>Bool</code> are automatically converted to GAP
objects when passed as arguments to GAP functions. Several others basic
types of objects can be converted using the <code>GapObj</code> constructor. For
example, here we convert a <code>Vector{Int}</code> to a GAP list:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; x = GapObj([1,2,3])
GAP: [ 1, 2, 3 ]"><pre>julia<span class="pl-k">&gt;</span> x <span class="pl-k">=</span> <span class="pl-c1">GapObj</span>([<span class="pl-c1">1</span>,<span class="pl-c1">2</span>,<span class="pl-c1">3</span>])
GAP<span class="pl-k">:</span> [ <span class="pl-c1">1</span>, <span class="pl-c1">2</span>, <span class="pl-c1">3</span> ]</pre></div>
<p dir="auto">For nested objects, one can use the optional <code>recursive</code> keyword argument:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; GapObj([1,2,[3,4,5]])
GAP: [ 1, 2, &lt;Julia: [3, 4, 5]&gt; ]

julia&gt; GapObj([1,2,[3,4,5]]; recursive=true)
GAP: [ 1, 2, [ 3, 4, 5 ] ]"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">GapObj</span>([<span class="pl-c1">1</span>,<span class="pl-c1">2</span>,[<span class="pl-c1">3</span>,<span class="pl-c1">4</span>,<span class="pl-c1">5</span>]])
GAP<span class="pl-k">:</span> [ <span class="pl-c1">1</span>, <span class="pl-c1">2</span>, <span class="pl-k">&lt;</span>Julia<span class="pl-k">:</span> [<span class="pl-c1">3</span>, <span class="pl-c1">4</span>, <span class="pl-c1">5</span>]<span class="pl-k">&gt;</span> ]

julia<span class="pl-k">&gt;</span> <span class="pl-c1">GapObj</span>([<span class="pl-c1">1</span>,<span class="pl-c1">2</span>,[<span class="pl-c1">3</span>,<span class="pl-c1">4</span>,<span class="pl-c1">5</span>]]; recursive<span class="pl-k">=</span><span class="pl-c1">true</span>)
GAP<span class="pl-k">:</span> [ <span class="pl-c1">1</span>, <span class="pl-c1">2</span>, [ <span class="pl-c1">3</span>, <span class="pl-c1">4</span>, <span class="pl-c1">5</span> ] ]</pre></div>
<p dir="auto">Converting GAP objects to native Julia objects is done using suitable Julia constructors.
For example, to convert the GAP list of integers we defined earlier back to Julia,
we might do this:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; Vector{Int}(x)
3-element Vector{Int64}:
 1
 2
 3"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">Vector</span><span class="pl-c1">{Int}</span>(x)
<span class="pl-c1">3</span><span class="pl-k">-</span>element Vector{Int64}<span class="pl-k">:</span>
 <span class="pl-c1">1</span>
 <span class="pl-c1">2</span>
 <span class="pl-c1">3</span></pre></div>
<p dir="auto">You can temporarily switch to a GAP prompt using <code>GAP.prompt()</code>:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; GAP.prompt();
gap&gt; G := SymmetricGroup(5);
Sym( [ 1 .. 5 ] )
gap&gt; quit;

julia&gt; GAP.Globals.G
GAP: Sym( [ 1 .. 5 ] )"><pre class="notranslate"><code>julia&gt; GAP.prompt();
gap&gt; G := SymmetricGroup(5);
Sym( [ 1 .. 5 ] )
gap&gt; quit;

julia&gt; GAP.Globals.G
GAP: Sym( [ 1 .. 5 ] )
</code></pre></div>
<p dir="auto">For more information on these and other capabilities of this package, please
consult the <a href="https://oscar-system.github.io/GAP.jl/stable" rel="nofollow">GAP.jl manual</a>.</p>

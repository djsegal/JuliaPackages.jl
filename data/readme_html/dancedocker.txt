<p dir="auto">Package can be installed with Julia's package manager, either by using Pkg REPL mode (<em>press ]</em>):</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="pkg&gt; add DanceDocker"><pre class="notranslate"><code>pkg&gt; add DanceDocker
</code></pre></div>
<p dir="auto">or by using Pkg functions</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Pkg; Pkg.add(&quot;DanceDocker&quot;)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Pkg; Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>DanceDocker<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">Compatibility is with Julia 1.1 and Dance 0.2.1 upward.</p>
<h2 dir="auto"><a id="user-content-2---setup" class="anchor" aria-hidden="true" href="#2---setup"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>2 - Setup</h2>
<p dir="auto"><strong>Please make sure your project contains a <code>Project.toml</code> file in its root, at same level as <code>dance.jl</code> file.</strong></p>
<p dir="auto"><strong>If testing Docker on <code>localhost</code>, is recommended to set DanceJL setting's <code>:server_host</code> parameter to <code>0.0.0.0</code>.</strong>
Else connections from host to Docker container might not work smoothly.</p>
<p dir="auto">Invoke terminal in project root directory and:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using DanceDocker
setup()"><pre><span class="pl-k">using</span> DanceDocker
<span class="pl-c1">setup</span>()</pre></div>
<p dir="auto">Optionally if you wish to use different Julia version than one used on your machine, one can do so via <code>version</code> kwarg.
For example:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using DanceDocker
setup(;version=1.4)"><pre><span class="pl-k">using</span> DanceDocker
<span class="pl-c1">setup</span>(;version<span class="pl-k">=</span><span class="pl-c1">1.4</span>)</pre></div>
<h2 dir="auto"><a id="user-content-3---docker-integration" class="anchor" aria-hidden="true" href="#3---docker-integration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>3 - Docker Integration</h2>
<p dir="auto">Following section requires Docker to be installed on system.</p>
<h3 dir="auto"><a id="user-content-31---building-docker-image" class="anchor" aria-hidden="true" href="#31---building-docker-image"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>3.1 - Building Docker Image</h3>
<p dir="auto">To build Docker image:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="docker build -t &lt;name&gt;:&lt;version&gt; ."><pre class="notranslate"><code>docker build -t &lt;name&gt;:&lt;version&gt; .
</code></pre></div>
<p dir="auto">where</p>
<ul dir="auto">
<li><code>name</code> is name of output DOcker image.</li>
<li><code>version</code> is optional, as Docker will default tag to <code>latest</code>.</li>
</ul>
<h3 dir="auto"><a id="user-content-32---running-docker-image" class="anchor" aria-hidden="true" href="#32---running-docker-image"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>3.2 - Running Docker Image</h3>
<p dir="auto">To run built Docker image:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="docker run -d -p &lt;host port&gt;:&lt;docker port&gt; &lt;build name&gt;:&lt;build version&gt;"><pre class="notranslate"><code>docker run -d -p &lt;host port&gt;:&lt;docker port&gt; &lt;build name&gt;:&lt;build version&gt;
</code></pre></div>
<p dir="auto">where:</p>
<ul dir="auto">
<li><code>host port</code> and <code>docker port</code> correspond to ports on host and what was specified under Dockerfile respectively. For example: <code>8000:8000</code>.</li>
<li><code>build name</code> is the name of Docker image, as specified under previous section.</li>
<li><code>build version</code> is optional, depending on if specified under previous section.</li>
</ul>
<p dir="auto">Above command will output a Docker container id.
Hence when finished, simply end Docker instance by running:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="docker container stop &lt;container id&gt;"><pre class="notranslate"><code>docker container stop &lt;container id&gt;
</code></pre></div>

<p dir="auto">This package contains definitions for common functions that are useful for symbolic expression manipulation.
Its purpose is to provide a shared interface between various symbolic programming Julia packages, for example
<a href="https://github.com/JuliaSymbolics/SymbolicUtils.jl">SymbolicUtils.jl</a>, <a href="https://github.com/JuliaSymbolics/Symbolics.jl">Symbolics.jl</a> and <a href="https://github.com/0x0f0f0f/Metatheory.jl">Metatheory.jl</a>.</p>

<p dir="auto">You should define the following methods for an expression tree type <code>T</code> with symbol types <code>S</code> to  work
with TermInterface.jl, and therefore with <a href="https://github.com/JuliaSymbolics/SymbolicUtils.jl">SymbolicUtils.jl</a>
and <a href="https://github.com/0x0f0f0f/Metatheory.jl">Metatheory.jl</a>.</p>
<h4 dir="auto"><a id="user-content-istreext-or-istreextypet" class="anchor" aria-hidden="true" href="#istreext-or-istreextypet"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>istree(x::T)</code> or <code>istree(x::Type{T})</code></h4>
<p dir="auto">Check if <code>x</code> represents an expression tree. If returns true,
it will be assumed that <code>operation(::T)</code> and <code>arguments(::T)</code>
methods are defined. Definining these three should allow use
of <code>SymbolicUtils.simplify</code> on custom types. Optionally <code>symtype(x)</code> can be
defined to return the expected type of the symbolic expression.</p>
<h4 dir="auto"><a id="user-content-exprheadx" class="anchor" aria-hidden="true" href="#exprheadx"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>exprhead(x)</code></h4>
<p dir="auto">If <code>x</code> is a term as defined by <code>istree(x)</code>, <code>exprhead(x)</code> must return a symbol,
corresponding to the head of the <code>Expr</code> most similar to the term <code>x</code>.
If <code>x</code> represents a function call, for example, the <code>exprhead</code> is <code>:call</code>.
If <code>x</code> represents an indexing operation, such as <code>arr[i]</code>, then <code>exprhead</code> is <code>:ref</code>.
Note that <code>exprhead</code> is different from <code>operation</code> and both functions should
be defined correctly in order to let other packages provide code generation
and pattern matching features.</p>

<p dir="auto">Returns the head (a function object) performed by an expression
tree. Called only if <code>istree(::T)</code> is true. Part of the API required
for <code>simplify</code> to work. Other required methods are <code>arguments</code> and <code>istree</code></p>
<h4 dir="auto"><a id="user-content-argumentsxt" class="anchor" aria-hidden="true" href="#argumentsxt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>arguments(x::T)</code></h4>
<p dir="auto">Returns the arguments (a <code>Vector</code>) for an expression tree.
Called only if <code>istree(x)</code> is <code>true</code>. Part of the API required
for <code>simplify</code> to work. Other required methods are <code>operation</code> and <code>istree</code></p>
<p dir="auto">In addition, the methods for <code>Base.hash</code> and <code>Base.isequal</code> should also be implemented by the types for the purposes of substitution and equality matching respectively.</p>
<h4 dir="auto"><a id="user-content-similartermtmytype-f-args-symtypet-metadatanothing-exprheadexprheadt" class="anchor" aria-hidden="true" href="#similartermtmytype-f-args-symtypet-metadatanothing-exprheadexprheadt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>similarterm(t::MyType, f, args, symtype=T; metadata=nothing, exprhead=exprhead(t))</code></h4>
<p dir="auto">Or <code>similarterm(t::Type{MyType}, f, args, symtype=T; metadata=nothing, exprhead=:call)</code>.</p>
<p dir="auto">Construct a new term with the operation <code>f</code> and arguments <code>args</code>, the term should be similar to <code>t</code> in type. if <code>t</code> is a <code>SymbolicUtils.Term</code> object a new Term is created with the same symtype as <code>t</code>. If not, the result is computed as <code>f(args...)</code>. Defining this method for your term type will reduce any performance loss in performing <code>f(args...)</code> (esp. the splatting, and redundant type computation). T is the symtype of the output term. You can use <code>SymbolicUtils.promote_symtype</code> to infer this type. The <code>exprhead</code> keyword argument is useful when creating <code>Expr</code>s.</p>

<h4 dir="auto"><a id="user-content-unsorted_argumentsx" class="anchor" aria-hidden="true" href="#unsorted_argumentsx"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>unsorted_arguments(x)</code></h4>
<p dir="auto">If x is a term satisfying <code>istree(x)</code> and your term type <code>T</code> orovides
and optimized implementation for storing the arguments, this function can
be used to retrieve the arguments when the order of arguments does not matter
but the speed of the operation does. Defaults to <code>arguments(x)</code>.</p>
<h4 dir="auto"><a id="user-content-symtypex" class="anchor" aria-hidden="true" href="#symtypex"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a><code>symtype(x)</code></h4>
<p dir="auto">The supposed type of values in the domain of x. Tracing tools can use this type to
pick the right method to run or analyse code.</p>
<p dir="auto">This defaults to <code>typeof(x)</code> if <code>x</code> is numeric, or <code>Any</code> otherwise.
For the types defined in this SymbolicUtils.jl, namely <code>T&lt;:Symbolic{S}</code> it is <code>S</code>.</p>
<p dir="auto">Define this for your symbolic types if you want <code>SymbolicUtils.simplify</code> to apply rules
specific to numbers (such as commutativity of multiplication). Or such
rules that may be implemented in the future.</p>

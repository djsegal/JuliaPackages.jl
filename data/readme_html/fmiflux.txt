<p dir="auto"><a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a> is a free-to-use software library for the Julia programming language, which offers the ability to set up NeuralFMUs just like NeuralODEs: You can place FMUs (<a href="http://fmi-standard.org/" rel="nofollow">fmi-standard.org</a>) simply inside any feed-forward ANN topology and still keep the resulting hybrid model trainable with a standard (or custom) FluxML training process.</p>

<p dir="auto">1. Open a Julia-REPL, switch to package mode using <code>]</code>, activate your preferred environment.</p>
<p dir="auto">2. Install  <a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a>:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="(@v1.x) pkg&gt; add FMIFlux"><pre lang="julia-repl" class="notranslate"><code>(@v1.x) pkg&gt; add FMIFlux
</code></pre></div>
<p dir="auto">3. If you want to check that everything works correctly, you can run the tests bundled with <a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a>:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="(@v1.x) pkg&gt; test FMIFlux"><pre lang="julia-repl" class="notranslate"><code>(@v1.x) pkg&gt; test FMIFlux
</code></pre></div>
<p dir="auto">4. Have a look inside the <a href="https://github.com/ThummeTo/FMIFlux.jl/tree/examples/examples">examples folder</a> in the examples branch or the <a href="https://thummeto.github.io/FMIFlux.jl/dev/examples/overview/" rel="nofollow">examples section</a> of the documentation. All examples are available as Julia-Script (<em>.jl</em>), Jupyter-Notebook (<em>.ipynb</em>) and Markdown (<em>.md</em>).</p>

<p dir="auto"><a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a> is tested (and testing) under Julia versions <em>1.6</em> (LTS) and <em>1.8</em> (latest) on Windows (latest) and Ubuntu (latest). MacOS should work, but untested.
However, please use Julia versions <em>&gt;= 1.7</em> if possible, because <a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a> runs <strong>a lot</strong> faster with these newer Julia versions.
<a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a> currently only works with FMI2-FMUs.
All shipped examples are tested under Julia version <em>1.8</em> (latest) on Windows (latest).</p>

<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="https://github.com/ThummeTo/FMI.jl/blob/main/docs/src/assets/FMI_JL_family.png?raw=true"><img src="https://github.com/ThummeTo/FMI.jl/raw/main/docs/src/assets/FMI_JL_family.png?raw=true" alt="FMI.jl Family" title="FMI.jl Family" style="max-width: 100%;"></a>
To keep dependencies nice and clean, the original package <a href="https://github.com/ThummeTo/FMI.jl"><em>FMI.jl</em></a> had been split into new packages:</p>
<ul dir="auto">
<li><a href="https://github.com/ThummeTo/FMI.jl"><em>FMI.jl</em></a>: High level loading, manipulating, saving or building entire FMUs from scratch</li>
<li><a href="https://github.com/ThummeTo/FMIImport.jl"><em>FMIImport.jl</em></a>: Importing FMUs into Julia</li>
<li><a href="https://github.com/ThummeTo/FMIExport.jl"><em>FMIExport.jl</em></a>: Exporting stand-alone FMUs from Julia Code</li>
<li><a href="https://github.com/ThummeTo/FMICore.jl"><em>FMICore.jl</em></a>: C-code wrapper for the FMI-standard</li>
<li><a href="https://github.com/ThummeTo/FMIBuild.jl"><em>FMIBuild.jl</em></a>: Compiler/Compilation dependencies for FMIExport.jl</li>
<li><a href="https://github.com/ThummeTo/FMIFlux.jl"><em>FMIFlux.jl</em></a>: Machine Learning with FMUs (differentiation over FMUs)</li>
<li><a href="https://github.com/ThummeTo/FMIZoo.jl"><em>FMIZoo.jl</em></a>: A collection of testing and example FMUs</li>
</ul>

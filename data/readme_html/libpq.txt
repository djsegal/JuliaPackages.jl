<h3 dir="auto"><a id="user-content-current" class="anchor" aria-hidden="true" href="#current"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Current</h3>
<ul dir="auto">
<li>Build
<ul dir="auto">
<li>Installs <code>libpq</code> via <code>BinaryBuilder.jl</code> for MacOS, GNU Linux, and Windows</li>
</ul>
</li>
<li>Connections
<ul dir="auto">
<li>Connect via DSN</li>
<li>Connect via PostgreSQL connection string</li>
<li>UTF-8 client encoding</li>
</ul>
</li>
<li>Queries
<ul dir="auto">
<li>Create and execute queries with or without parameters</li>
<li>Execute queries asynchronously</li>
<li>Stream results using <a href="https://github.com/JuliaData/Tables.jl">Tables</a></li>
<li>Configurably convert a variety of PostgreSQL types to corresponding Julia types (see the <strong>Type Conversions</strong> section of the docs)</li>
</ul>
</li>
<li>Prepared Statements
<ul dir="auto">
<li>Create and execute prepared statements with or without parameters</li>
<li>Stream table of parameters to execute the same statement multiple times with different data</li>
</ul>
</li>
</ul>

<p dir="auto"><em>Note that these are goals and do not represent the current state of this package</em></p>
<p dir="auto">LibPQ.jl aims to wrap <code>libpq</code> as documented in the PostgreSQL documentation, including all non-deprecated functionality and handling all documented error conditions.
Where possible, asynchronous functionality will be wrapped in idiomatic Julia control flow.
All Oids returned in query results will have type conversions (to String by default) defined, as long as I can find documentation on their structure.
Some effort will be made to integrate with other packages (e.g., Tables, already implemented) to facilitate conversion from query results to a malleable format.</p>

<p dir="auto">LibPQ.jl will not attempt to conform to a standard database interface, though anyone is welcome to write a PostgreSQL.jl library to wrap this package.</p>
<p dir="auto">This package will not:</p>
<ul dir="auto">
<li>parse SQL</li>
<li>emit SQL</li>
<li>provide an interface for handling transactions or cursors</li>
<li>provide abstractions over common SQL patterns</li>
</ul>

<p dir="auto">This package may not:</p>
<ul dir="auto">
<li>test on multiple install configurations</li>
<li>aim to support any particular versions of libpq or PostgreSQL</li>
<li>support conversion from some Oid to some type</li>
<li>provide easy access to every possible connection method</li>
<li>be as memory-efficient as possible</li>
</ul>
<p dir="auto">While I may never get to any of these, I welcome tested, documented contributions!</p>

<p>A Julia package for easy access to the <a href="https://mba.tuck.dartmouth.edu/pages/faculty/ken.french/data_library.html" rel="nofollow">Ken French Data Library</a>. The Ken French Data Library is one of the most highly used publicly available data sources for financial investments and asset pricing research.</p>
<p>Working with the data is sometimes tedious because the downloadable files come (1) compressed in <code>zip</code> archives and (2) having non-standard <code>csv</code> layouts. This package handles both of those hurdles, allowing users to get to the data faster.</p>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Usage</h2>
<p>To add the package, type <code>] add FamaFrenchData</code> at the Julia REPL.</p>
<p>Once added, type <code>using FamaFrenchData</code> to import the package.</p>
<p>The package exports 3 functions: <code>readFamaFrench</code>, <code>downloadFamaFrench</code>, and <code>listFamaFrench</code>.</p>
<p>For help with any of these functions, use <code>?</code> at the REPL (eg. <code>?readFamaFrench</code>).</p>
<h2><a id="user-content-example" class="anchor" aria-hidden="true" href="#example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Example</h2>
<p>The Fama-French 3 factor model is a commonly used empirical asset pricing model. This example retrieves the full time series of FF3 monthly and annual returns.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> FamaFrenchData, DataFrames

tables, tablenotes, filenotes <span class="pl-k">=</span> <span class="pl-c1">readFamaFrench</span>(<span class="pl-s"><span class="pl-pds">"</span>F-F_Research_Data_Factors<span class="pl-pds">"</span></span>)

FF3_monthly <span class="pl-k">=</span> tables[<span class="pl-c1">1</span>]
FF3_annual <span class="pl-k">=</span> tables[<span class="pl-c1">2</span>]</pre></div>
<p>If you would rather store the file locally (perhaps so your code is reproducible later on), use <code>downloadFamaFrench</code>.</p>
<div class="highlight highlight-source-julia"><pre>savename <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>path/to/my/file.csv<span class="pl-pds">"</span></span>
<span class="pl-c1">downloadFamaFrench</span>(savename,<span class="pl-s"><span class="pl-pds">"</span>F-F_Research_Data_Factors<span class="pl-pds">"</span></span>)
tables, tablenotes, filenotes <span class="pl-k">=</span> <span class="pl-c1">readFamaFrench</span>(savename) <span class="pl-c"><span class="pl-c">#</span> also reads local files</span></pre></div>
<p>To get a list of all of the available table names, use <code>listFamaFrench</code>.</p>
<div class="highlight highlight-source-julia"><pre>FFnames <span class="pl-k">=</span> <span class="pl-c1">listFamaFrench</span>()</pre></div>
<h2><a id="user-content-additional-notes" class="anchor" aria-hidden="true" href="#additional-notes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Additional Notes</h2>
<ul>
<li>Original files use <code>-99.99</code> or <code>-999</code> to encode missing values, I attempt to replace these with <code>missing</code>.</li>
<li>Original files have no column heading for column 1, I name this column <code>:Date</code>. It maintains the parsed type of <code>Int</code> because it can take several different forms: <code>20011231</code>,<code>200112</code>,<code>2001</code>.</li>
<li>I have not verified that all tables are parsed correctly.</li>
<li>Breakpoints files require the keyword argument <code>header=false</code></li>
</ul>

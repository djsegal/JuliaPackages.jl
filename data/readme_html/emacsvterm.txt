<p dir="auto">Better integration of Julia REPL with Emacs vterm terminal.</p>
<h2 dir="auto"><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Installation</h2>
<ol dir="auto">
<li>
<p dir="auto">In Julia prompt type:</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; ]add EmacsVterm"><pre class="notranslate"><code>julia&gt; ]add EmacsVterm
</code></pre></div>
</li>
<li>
<p dir="auto">Add the following to <code>~/.julia/config/startup.jl</code>:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="atreplinit() do repl
    @eval using EmacsVterm
    # Optionally set EmacsVterm.options as you like (see below)
end"><pre><span class="pl-c1">atreplinit</span>() <span class="pl-k">do</span> repl
    <span class="pl-c1">@eval</span> <span class="pl-k">using</span> EmacsVterm
    <span class="pl-c"><span class="pl-c">#</span> Optionally set EmacsVterm.options as you like (see below)</span>
<span class="pl-k">end</span></pre></div>
</li>
<li>
<p dir="auto">Configure <a href="https://github.com/tpapp/julia-repl">julia-repl</a> to use
the <code>vterm</code> backend by putting:</p>
<div class="highlight highlight-source-emacs-lisp notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="(julia-repl-set-terminal-backend 'vterm)"><pre>(julia-repl-set-terminal-backend <span class="pl-c1">'vterm</span>)</pre></div>
<p dir="auto">to your Emacs config.</p>
</li>
</ol>
<h2 dir="auto"><a id="user-content-features" class="anchor" aria-hidden="true" href="#features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Features</h2>
<ul dir="auto">
<li>
<p dir="auto">You can jump between prompts in <code>*julia*</code> REPL buffers with <code>C-c C-p</code> and <code>C-c C-n</code>.</p>
</li>
<li>
<p dir="auto">Julia REPL informs Emacs about its working directory. Therefore,
after changing directory in Julia, opening file in Emacs (e.g. <code>C-x C-f</code>) starts in that directory.</p>
</li>
<li>
<p dir="auto">Documentation (<code>@doc ...</code> invocation or <code>C-c C-d</code> in <code>julia-mode</code>
buffers with <code>julia-repl</code> enabled) is shown in a separate Emacs
buffer.</p>
<p dir="auto">To disable this functionality, run:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="EmacsVterm.options.markdown = false"><pre>EmacsVterm<span class="pl-k">.</span>options<span class="pl-k">.</span>markdown <span class="pl-k">=</span> <span class="pl-c1">false</span></pre></div>
<p dir="auto">If you are not happy with where Emacs chooses to display the
<code>*julia-doc*</code> buffer, you can configure it via a "display action".
For example, the following piece of code in <code>init.el</code> ensures that
if the <code>*julia-doc*</code> buffer is already shown somewhere, the same
buffer is reused; otherwise, a right side window with an appropriate
width will be created.</p>
<div class="highlight highlight-source-emacs-lisp notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="(add-to-list 'display-buffer-alist '(&quot;\\*julia-doc\\*&quot;
  			       (display-buffer-reuse-window display-buffer-in-side-window)
  			       (side . right) (window-width . 80)))"><pre>(<span class="pl-c1">add-to-list</span> <span class="pl-c1">'display-buffer-alist</span> '(<span class="pl-s"><span class="pl-pds">"</span><span class="pl-cce">\\</span>*julia-doc<span class="pl-cce">\\</span>*<span class="pl-pds">"</span></span>
  			       (<span class="pl-c1">display-buffer-reuse-window</span> display-buffer-in-side-window)
  			       (side <span class="pl-k">.</span> right) (<span class="pl-c1">window-width</span> <span class="pl-k">.</span> <span class="pl-c1">80</span>)))</pre></div>
</li>
<li>
<p dir="auto">Images can be shown in the <code>*julia-img*</code> Emacs buffer. This
functionality is not enabled by default. Enable it with:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="EmacsVterm.options.image = true"><pre>EmacsVterm<span class="pl-k">.</span>options<span class="pl-k">.</span>image <span class="pl-k">=</span> <span class="pl-c1">true</span></pre></div>
</li>
<li>
<p dir="auto">In addition to <code>EmacsVterm.options</code>, the whole Emacs-based
Multimedia I/O can be disabled (resp. enabled) with:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="EmacsVterm.display_off()
EmacsVterm.display_on()"><pre>EmacsVterm<span class="pl-k">.</span><span class="pl-c1">display_off</span>()
EmacsVterm<span class="pl-k">.</span><span class="pl-c1">display_on</span>()</pre></div>
</li>
</ul>

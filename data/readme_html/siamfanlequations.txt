<p dir="auto">The book is finished and this project is <strong>DONE</strong>. So I take the sacred book author oath ...</p>
<ul dir="auto">
<li>I will only make updates to the package and notebooks to fix bugs or typos.</li>
<li>I will not be adding new functionality to this package or new material to the notebooks.</li>
<li>I will make no changes to the user interface for the codes in the package.</li>
</ul>
<p dir="auto">This is a sequel to my book</p>
<p dir="auto">(Kel03) C. T. Kelley, <a href="https://my.siam.org/Store/Product/viewproduct/?ProductId=841" rel="nofollow"><em><strong>Solving Nonlinear Equations with Iterative Methods:</strong></em></a> , Fundamentals of Algorithms 1, SIAM, Philadelphia, 2003.</p>
<p dir="auto">Hence the notebook and this package all have SIAMFANL in their names.</p>
<p dir="auto">The new book has a different algorithm mix and the solvers and examples are in Juila. The project has three parts.</p>
<ol dir="auto">
<li>A print book: (Kel22) C. T. Kelley, <a href="https://my.siam.org/Store/Product/viewproduct/?ProductId=44313635" rel="nofollow"><em><strong>Solving Nonlinear Equations with Newton's Method: Solvers and Examples in Julia</strong></em></a>, Fundamentals of Algorithms 20, SIAM, Philadelphia, 2022.</li>
<li><a href="https://github.com/ctkelley/NotebookSIAMFANL">A suite of IJulia notebooks</a> (open source, MIT License, Creative Commons License)
The latest releases of the notebook suite and package run correctly. The notebooks and package from the master branches also run correctly
together. Bug fixes prior to 1.0 may, with an absurdly low probablilty, break things in older releases.</li>
<li>This package (MIT License)<br></li>
</ol>
<p dir="auto">Content changes from (Kel03):</p>
<ul dir="auto">
<li>New solvers: <strong>pseudo-transient continuation</strong> and <strong>Anderson acceleration</strong></li>
<li>Deletions: <strong>Broyden's method</strong>
<ul dir="auto">
<li>Quasi-Newton methods are not used much for nonlinear equations any more. Newton-Krylov has taken over.</li>
</ul>
</li>
<li>New Case Studies chapter</li>
</ul>

<p dir="auto">The best documentation for this package lives in the <a href="https://github.com/ctkelley/NotebookSIAMFANL">notebook</a> and the print book. They have detailed algorithmic descriptions, examples for you to play with, and guidance on tweaking the algorithmic paramenters to solve your problems. The notebook was built in parallel with the print book and the content is <strong>roughly</strong> the same. The differences are mostly to accommodate the two formats. For example, docstrings need some work after the map from notebook to print and notebook has to make sense as an interactive resource.</p>

<p dir="auto">The solvers are designed to be stand-alone codes. The reason for this is the education mission of the project. I want the codes to be as easy to understand as possible. I have deliberately sacrificed a lot of abstraction and some performance in this effort. The reward for the reader (ie you) is that the algorithmic parameters are completely exposed so  you can play with them. Someday, not soon, I may write a wrapper for all this that hides the parameters as a separate package. However, the stand-alone, keyword-infested codes are what you need if you want to really understand how these methods work. My students became experts in this field by fiddling with the Matlab version of these solvers.</p>
<p dir="auto">The linear solvers are tuned to communicate well with nonlinear solvers. My old Matlab codes are a good illustration of this idea. My <a href="https://ctk.math.ncsu.edu/knl.html" rel="nofollow">new Mablab codes</a> were designed in response to the need to do this better than I had been. In particular, the linear solver and the matrix-vector/preconditioner-vector product function need information on the nonlinear iteration and any precomputed data. While I could use global variables (and did in Kel95) and put these things in a module to simplify the interface, I won't do that anymore. Global varaibles make debugging harder and break parallelism. I like to avoid them.</p>
<p dir="auto">The algorithms, listed by book chapter are</p>
<ul dir="auto">
<li>
<p dir="auto">Chapter 1: Newton-Armijo and Pseudo-transient continuation for scalar equations: <strong>nsolsc.jl</strong> and <strong>ptcsolsc.jl</strong></p>
</li>
<li>
<p dir="auto">Chapter 2: Newton-Armijo and Pseudo-transient continuation for systems with direct linear solvers: <strong>nsol.jl</strong> and <strong>ptcsol.jl</strong></p>
</li>
<li>
<p dir="auto">Chapter 3: Newton-Armijo and Pseudo-transient continuation for systems with iterative linear solvers: <strong>nsoli.jl</strong> and <strong>ptcsoli.jl</strong></p>
</li>
<li>
<p dir="auto">Chapter 4: Anderson acceleration: <strong>aasol.jl</strong></p>
</li>
<li>
<p dir="auto">Chapter 5: Case studies:  <strong>Conductive-Radiative heat transfer</strong> and <strong>Continuation for H-equation.</strong></p>
</li>
</ul>

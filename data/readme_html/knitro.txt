<p dir="auto"><a href="https://github.com/jump-dev/KNITRO.jl">KNITRO.jl</a> is a wrapper for the
<a href="https://www.artelys.com/knitro" rel="nofollow">Artelys Knitro solver</a>.</p>
<p dir="auto">It has two components:</p>
<ul dir="auto">
<li>a thin wrapper around the <a href="https://www.artelys.com/tools/knitro_doc/3_referenceManual/callableLibraryAPI.html" rel="nofollow">C API</a></li>
<li>an interface to <a href="https://github.com/jump-dev/MathOptInterface.jl">MathOptInterface</a>.</li>
</ul>

<p dir="auto">First, obtain a license and install a copy of KNITRO from
<a href="https://www.artelys.com/knitro" rel="nofollow">Artelys</a>.</p>
<p dir="auto">Then, install <code>KNITRO.jl</code> using the Julia package manager:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="import Pkg
Pkg.add(&quot;KNITRO&quot;)"><pre><span class="pl-k">import</span> Pkg
Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>KNITRO<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">If you are having trouble installing KNITRO.jl, here are several things to try:</p>
<ul dir="auto">
<li>Make sure that you have defined your global variables correctly, for example
with <code>export KNITRODIR="/path/to/knitro-vXXX-$OS-64"</code> and <code>export LD_LIBRARY_PATH="$LD_LIBRARY_PATH:$KNITRODIR/lib"</code>. You can check that
<code>KNITRO.jl</code> sees your library with <code>using KNITRO; KNITRO.has_knitro()</code>.</li>
<li>If <code>KNITRO.has_knitro()</code> returns <code>false</code> but you are confident that your paths
are correct, try running <code>Pkg.build("KNITRO")</code> and restarting Julia. In at
least one user's experience, installing and using KNITRO in a temporary Julia
environment (activated with <code>] activate --temp</code>) does not work and the need to
manually build is likely the reason why.</li>
</ul>

<p dir="auto">KNITRO.jl implements most of Knitro's functionalities. If you aim at using part
of Knitro's API that are not implemented in the MathOptInterface/JuMP ecosystem,
you can refer to the low-level API, which wraps Knitro's C API (whose templates
are specified in the file <code>knitro.h</code>).</p>
<p dir="auto">Extensive examples using the C wrapper can be found in <code>examples/</code>.</p>

<p dir="auto">Due to limitations in the interaction between Julia and C, KNITRO.jl disables
multi-threading if the problem is nonlinear. This will override any options such
as <code>par_numthreads</code> that you may have set. Read <a href="https://github.com/jump-dev/KNITRO.jl/issues/93" data-hovercard-type="issue" data-hovercard-url="/jump-dev/KNITRO.jl/issues/93/hovercard">GitHub issue #93</a>
for more details.</p>

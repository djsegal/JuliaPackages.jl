<p dir="auto">Currently, there are two utilities:</p>
<ul dir="auto">
<li>A <code>GifSimulator</code> to be used with the <code>simulate</code> function.</li>
<li>A <code>makegif</code> convenience function to (1) run a simulation and create a gif, or (2) create a gif from a model.</li>
</ul>

<p dir="auto"><code>GifSimulator(kwargs...)</code>. Create a simulator for producing a gif output by calling <code>POMDPModelTools.render</code> at each step.</p>
<p dir="auto"><strong>Keyword Arguments</strong></p>
<ul dir="auto">
<li><code>filename::String=tempname()*".gif"</code></li>
<li><code>fps::Int=2</code>: frames per second</li>
<li><code>spec::Any</code>: specification for which elements of a step to render (see <code>POMDPSimulators.eachstep</code>)</li>
<li><code>max_steps::Int=nothing</code></li>
<li><code>rng::AbstractRNG=GLOBAL_RNG</code></li>
<li><code>show_progress::Bool</code></li>
<li><code>extra_initial::Bool</code> if set to true, the simulator adds an extra step at time 0 (before first transition)</li>
<li><code>extra_final::Boll</code> if set to true, the simulator adds an extra setp at the end (after the last transition)</li>
<li><code>render_kwargs</code>: keyword args to be fed to <code>POMDPModelTools.render</code></li>
</ul>

<p dir="auto">Create a gif of a single simulation of a POMDP or MDP by calling <code>POMDPModelTools.render</code> at each step.</p>
<p dir="auto"><strong>Arguments:</strong></p>
<ul dir="auto">
<li><code>m::Union{POMDP,MDP}</code>: the model to be simulated</li>
</ul>
<p dir="auto">All other positional arguments, for instance a policy, updater, initial state, etc. will be fed to the <code>simulate</code> function. See <a href="https://juliapomdp.github.io/POMDPSimulators.jl/latest/" rel="nofollow">POMDPSimulators documentation</a> for more info.</p>
<p dir="auto"><strong>Keyword Arguments</strong>
All keyword arguments are fed to the <code>GifSimulator</code> constructor. See its documentation for more info.</p>

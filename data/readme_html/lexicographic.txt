<h2 dir="auto"><a id="user-content-design-rationale" class="anchor" aria-hidden="true" href="#design-rationale"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Design rationale</h2>
<p dir="auto">Julia's great support for multiple dispatch means that there are roughly two possibilities for encoding new behavior:</p>
<ol dir="auto">
<li>define new types and new methods that dispatch on those types</li>
<li>define new functions</li>
</ol>
<p dir="auto">This package defines new types, and extends existing ordering functions in <code>Base</code>: (<code>isequal</code>, <code>==</code>, <code>isless</code>)</p>
<p dir="auto">This choice is consistent with defining new methods for</p>
<p dir="auto"><code>*(::Complex{T}, ::Complex{T})</code></p>
<p dir="auto">instead of  defining</p>
<p dir="auto"><code>complex_times(::NTuple{2, T}, ::NTuple{2,T})</code></p>
<p dir="auto">This is important not just for brevity, but also for consistency, because there's an implicit interface with <code>one</code>, <code>zero</code>, <code>:*</code>, <code>:+</code>, <code>inv</code>, <code>:\</code>. The alternative of defining <code>complex_plus</code>, <code>complex_one</code> means that an algorithm that could otherwise be generic cannot.</p>
<p dir="auto">The implicit "ordering" interface in Julia is smaller, so there wouldn't be many <code>lexico_isless</code>, etc., but by defining a few methods, then <code>:&lt;</code>, <code>:&gt;</code>, <code>:&lt;=</code>, <code>:&gt;=</code> all work in a consistent manner.</p>

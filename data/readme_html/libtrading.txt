<p dir="auto"><a href="https://travis-ci.org/StefanKarpinski/LibTrading.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/b6c61868310cf20126ccb8054b31f0532451e76205877bf00f38914199697cee/68747470733a2f2f7472617669732d63692e6f72672f53746566616e4b617270696e736b692f4c696254726164696e672e6a6c2e737667" alt="Build Status" data-canonical-src="https://travis-ci.org/StefanKarpinski/LibTrading.jl.svg" style="max-width: 100%;"></a></p>
<p dir="auto">The Julia LibTrading package is a wrapper for the <a href="https://github.com/libtrading/libtrading">libtrading library</a>, which "is an open source API for high-performance, low-latency trading applications."</p>
<h2 dir="auto"><a id="user-content-example-usage" class="anchor" aria-hidden="true" href="#example-usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Example Usage</h2>
<p dir="auto">After installing libtrading and running <code>make</code> to generate the dynamic shared library object – either named <code>libtrading.so</code> (UNIX) or <code>libtrading.dylib</code> (OS X), make sure that this library can be found by <code>dlopen</code> – for example, but changing directories to the libtrading top-level directory. In that top-level directory, run the <code>fix_server</code> example program with this command line:</p>
<div class="highlight highlight-source-shell notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="$ tools/fix/fix_server -m 1 -p 7070
Server is listening to port 7070..."><pre>$ tools/fix/fix_server -m 1 -p 7070
Server is listening to port 7070...</pre></div>

<p>The <code>PCRE2</code> package implements a low-level API for accessing the PCRE libraries (8, 16, and 32-bit)
It was intended to replace <code>Base.PCRE</code> which was not threadsafe, as well as missing non-UTF-8 support.</p>
<p><code>Base.PCRE</code> still only supports UTF-8, which is why this is needed for the Strs package.
It is my intention to change this to use the libraries now created by the BinaryBuilder.</p>

<p dir="auto">This package leverages the <a href="https://github.com/FourierFlows/FourierFlows.jl">FourierFlows.jl</a> framework to provide modules for solving problems in
Geophysical Fluid Dynamics on periodic domains using Fourier-based pseudospectral methods.</p>

<p dir="auto">To install, use Julia's  built-in package manager (accessed by pressing <code>]</code> in the Julia REPL command prompt) to add the package and also to instantiate/build all the required dependencies</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; ]
(v1.6) pkg&gt; add GeophysicalFlows
(v1.6) pkg&gt; instantiate"><pre>julia<span class="pl-k">&gt;</span> ]
(v1.<span class="pl-c1">6</span>) pkg<span class="pl-k">&gt;</span> add GeophysicalFlows
(v1.<span class="pl-c1">6</span>) pkg<span class="pl-k">&gt;</span> instantiate</pre></div>
<p dir="auto">The most recent version of GeophysicalFlows.jl requires Julia v1.6  (the current long-term-release) or later. <em>We strongly urge you to use this version.</em></p>
<p dir="auto">The latest version that is compatible with Julia v1.5 is GeophysicalFlows.jl v0.13.1.</p>

<h2 dir="auto"><a id="user-content-modules" class="anchor" aria-hidden="true" href="#modules"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Modules</h2>
<ul dir="auto">
<li><a href="https://fourierflows.github.io/GeophysicalFlowsDocumentation/stable/modules/twodnavierstokes/" rel="nofollow"><code>TwoDNavierStokes</code></a>: the
two-dimensional vorticity equation.</li>
<li><a href="https://fourierflows.github.io/GeophysicalFlowsDocumentation/stable/modules/singlelayerqg/" rel="nofollow"><code>SingleLayerQG</code></a>: the barotropic
or equivalent-barotropic quasi-geostrophic equation, which generalizes <code>TwoDNavierStokes</code> to cases with topography, Coriolis
parameters of the form <code>f = f‚ÇÄ + Œ≤y</code>, and finite Rossby radius of deformation.</li>
<li><a href="https://fourierflows.github.io/GeophysicalFlowsDocumentation/stable/modules/multilayerqg/" rel="nofollow"><code>MultiLayerQG</code></a>: a multi-layer
quasi-geostrophic model over topography allowing to impose a zonal flow <code>U_n(y)</code> in each layer.</li>
<li><a href="https://fourierflows.github.io/GeophysicalFlowsDocumentation/stable/modules/surfaceqg/" rel="nofollow"><code>SurfaceQG</code></a>: a surface
quasi-geostrophic model.</li>
<li><a href="https://fourierflows.github.io/GeophysicalFlowsDocumentation/stable/modules/barotropicqgql/" rel="nofollow"><code>BarotropicQGQL</code></a>: the
quasi-linear barotropic quasi-geostrophic equation.</li>
</ul>

<p dir="auto">For now, GeophysicalFlows.jl is restricted to run on either a single CPU or single GPU. These
restrictions come from FourierFlows.jl. Multi-threading can enhance performance for the Fourier
transforms. By default, FourierFlows.jl will use the maximum number of threads available on
your machine. You can set the number of threads used by FourierFlows.jl by setting the
environment variable, e.g.,</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="$ export JULIA_NUM_THREADS=4"><pre class="notranslate"><code>$ export JULIA_NUM_THREADS=4
</code></pre></div>
<p dir="auto">For more information on multi-threading users are directed to the <a href="https://docs.julialang.org/en/v1/manual/multi-threading/" rel="nofollow">Julia Documentation</a>.</p>
<p dir="auto">If your machine has more than one GPU available, then functionality within CUDA.jl package
enables the user to choose the GPU device that FourierFlows.jl should use. The user is referred
to the <a href="https://juliagpu.github.io/CUDA.jl/stable/lib/driver/#Device-Management" rel="nofollow">CUDA.jl Documentation</a>;
in particular, <a href="https://juliagpu.github.io/CUDA.jl/stable/lib/driver/#CUDA.devices" rel="nofollow"><code>CUDA.devices</code></a>
and <a href="https://juliagpu.github.io/CUDA.jl/stable/lib/driver/#CUDA.CuDevice" rel="nofollow"><code>CUDA.CuDevice</code></a>.
The user is also referred to the <a href="https://fourierflows.github.io/FourierFlowsDocumentation/stable/gpu/" rel="nofollow">GPU section</a> in the FourierFlows.jl documentation.</p>

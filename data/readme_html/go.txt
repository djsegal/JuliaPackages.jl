<p dir="auto"><a href="https://travis-ci.org/dmrd/go.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/e1d64cb3e164776eb5d7fc3431e0a825c1e0be9130e230bf889f81baacc3050f/68747470733a2f2f7472617669732d63692e6f72672f646d72642f676f2e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/dmrd/go.jl.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">This repository implements a Go bot framework end-to-end in Julia, including
functionality to maintain board state, extract features, read SGF files, and
communicate with other Go programs over GTP.</p>
<p dir="auto">We use Keras over PyCall to train machine models to predict moves given a board state.  All the functionality is in place to implement more complex approaches, such as tree search or combining multiple policies together. Julia makes it practical to implement these speed intensive search operations.</p>

<p dir="auto">A GTP interface is located at <code>examples/gtp_bot.jl</code>.  It is usable with <a href="http://www.sente.ch/software/goban3/" rel="nofollow">Goban</a> on OSX by creating a GTP player with executable path <code>/path/to/go/examples/gtp_bot.jl</code>, and arguments <code>keras model_name</code> (models are read from the <code>models</code> directory).  Be sure that the correct board size <code>N</code> is set at the top of src/board.jl</p>

<p dir="auto">Example training script is located at <code>examples/training.jl</code>.  There are two main ways of training a model:</p>
<ol dir="auto">
<li>Extract examples from sgf files and start training a model immediately</li>
</ol>
<ul dir="auto">
<li>e.g. <code>julia training.jl cgs_example_model --cgs --epochs 20</code></li>
</ul>
<ol start="2" dir="auto">
<li>Extract examples to an h5 file with <code>examples/extract.jl</code>, then train a model by reading from the h5 file.  Then is necessary to train large models, the data can take many GB (all 170k KGS games are 52gb in h5 format, for example).</li>
</ol>
<ul dir="auto">
<li><code>julia extract.jl ../data/cgs/ cgs2500.h5</code></li>
<li><code>julia training.jl cgs_example_model --h5path ../data/cgs2500.h5 --n_train 100000 --n_validation 5000</code></li>
</ul>
<p dir="auto">Look in each script for more training options and settings.  The default model is a fairly large convolutional neural network which takes a long time to train. Model logic is located in <code>src/policy.jl</code>, and you can change the default clf for each training script (e.g. to <code>CLF_LINEAR</code> instead of <code>CLF_DCNN</code>, which will use a linear softmax instead of a many layer CNN).</p>
<p dir="auto">Models are checkpointed whenever validation loss decreases by default.</p>

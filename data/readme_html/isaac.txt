<p dir="auto">ISAAC (Indirection, Shift, Accumulate, Add, and Count) generates pseudorandom numbers.
The original C reference version was developed by
<a href="https://burtleburtle.net/bob/" rel="nofollow">Robert J. Jenkins Jr.</a>
and is available from
the <a href="https://burtleburtle.net/bob/rand/isaacafa.html" rel="nofollow">ISAAC home page</a>.</p>
<p dir="auto">How secure is it? See the home page, and pay attention to Jenkins' warning:</p>
<blockquote>
<p dir="auto">Seeding a random number generator is essentially the same problem as
encrypting the seed with a block cipher. ISAAC should be initialized
with the encryption of the seed by some secure cipher.</p>
</blockquote>
<p dir="auto">I am definitely not an expert, but there appears to be no public evidence
(see e.g. <a href="https://en.wikipedia.org/wiki/ISAAC_(cipher)" rel="nofollow">Wikipedia</a> that the
cipher has been broken.</p>

<p dir="auto">Two versions of ISAAC generator are provided:</p>
<ul dir="auto">
<li><strong>Isaac32</strong> generates 32-bit unsigned integers</li>
<li><strong>Isaac64</strong> generates 64-bit unsigned integers</li>
</ul>
<p dir="auto">The two generators use different algorithms, and will produce different
numbers given the same seed. Both generators are subtypes of <code>Random.AbstractRNG</code>
and can be used (I hope) with the same methods provided by <a href="https://docs.julialang.org/en/v1/stdlib/Random/#Random-generation-functions" rel="nofollow"><code>Random</code></a> in the Julia
standard library (Thanks to <a href="https://github.com/JuliaRandom/StableRNGs.jl">StableRNGs</a>
for showing me how to do this).</p>

<p dir="auto">Three construction methods are available for each generator:</p>
<ul dir="auto">
<li><code>Isaac32()</code>, <code>Isaac64()</code> are default constructors (equivalent to an empty seed)</li>
<li><code>Isaac32(seed::AbstractVector{UInt8})</code>, <code>Isaac64(seed::AbstractVector{UInt8})</code> use
the items in <code>seed</code> to seed the generator</li>
<li><code>Isaac32(seed::AbstractString)</code>, <code>Isaac64(seed::AbstractString)</code> use the bytes in <code>seed</code>
to seed the generator</li>
</ul>
<h3 dir="auto"><a id="user-content-methods" class="anchor" aria-hidden="true" href="#methods"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Methods</h3>
<ul dir="auto">
<li><code>isaac_rand(rng::Union{Isaac32, Isaac64})</code> provides underlying access to the next
value generated (either <code>UInt32</code> or <code>UInt64</code>. This method is not exported.</li>
<li><code>Random.seed!(rng::Union{Isaac32, Isaac64}, seed)</code> will re-seed <code>rng</code> with <code>seed</code>.
This is used by the constructors, and <code>seed</code> can have the same types as in the constructors.</li>
<li><code>rand(rng::Union{Isaac32, Isaac64}, ::Type{T}, n = (1, ))</code> where <code>T</code> can be any of the
types available to the standard Julia <code>Random.rand(...)</code> methods.</li>
<li>Other methods such as <code>randn(rng::Union{Isaac32, Isaac64},...)</code>,
<code>shuffle(rng::Union{Isaac32, Isaac64},...)</code> are accessible if <code>Random</code> is <code>use</code>d
or <code>import</code>ed.</li>
</ul>

<h2><a id="user-content-about-intervalcontractorsjl" class="anchor" aria-hidden="true" href="#about-intervalcontractorsjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>About <code>IntervalContractors.jl</code></h2>
<p><code>IntervalContractors.jl</code> provides contractors and reverse functions (also called backward functions or relational functions) for interval arithmetic.</p>
<h3><a id="user-content-reverse-functions" class="anchor" aria-hidden="true" href="#reverse-functions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Reverse functions</h3>
<p>The reverse function of a function <code>f</code> calculates the (interval hull of) its inverse function, <code>f⁻¹</code>.</p>
<p>For example, <code>sin_rev(Y::Interval, X::Interval)</code> calculates the (interval hull of) those <code>x ∈ X</code> such that <code>sin(x) ∈ Y</code>. This can also be thought of as an inverse function, calculating <code>X_new := sin⁻¹(Y) ∩ X</code>.
The return value is <code>(Y, X_new)</code>.</p>
<p>Functions such as <code>mul_rev(C::Interval, A::Interval, B::Interval)</code> take three arguments, and correspond to <code>C = A * B</code>; they return <code>(C, A_new, B_new)</code>, with <code>A_new</code> and <code>B_new</code> similarly defined to be the corresponding inverse images of the multiplication operator in each component.</p>
<h3><a id="user-content-contractors" class="anchor" aria-hidden="true" href="#contractors"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Contractors</h3>
<p>Functions like <code>sin!</code> (currently non-exported) are contractors for the set <code>{(x, y): y = sin(x)}</code>.</p>
<p>The list of available functions may be found in the documentation.</p>
<p>These functions are designed to be used inside the <a href="https://github.com/JuliaIntervals/IntervalConstraintProgramming.jl"><code>IntervalConstraintProgramming.jl</code></a> library,
and (eventually) to satisfy the section on reverse functions in the IEEE-1788 2015 standard on interval arithmetic.</p>

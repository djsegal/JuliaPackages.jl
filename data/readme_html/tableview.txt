<p><a href="https://travis-ci.org/JuliaComputing/TableView.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/706927136a42850e48d6d07f54153b8ff6f4f453/68747470733a2f2f7472617669732d63692e6f72672f4a756c6961436f6d707574696e672f5461626c65566965772e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/JuliaComputing/TableView.jl.svg?branch=master" style="max-width:100%;"></a></p>
<p>TableView.jl is an <a href="https://www.ag-grid.com/" rel="nofollow">ag-grid</a> based table viewer built on <a href="https://github.com/JuliaGizmos/WebIO.jl">WebIO.jl</a>. It can display arbitrarily large tables by lazy-loading additional data when scrolling (this is the default for datasets with more than 10k rows).</p>

<p><code>showtable(yourtable)</code> returns a <code>WebIO.Scope</code> which can be displayed with multiple frontends (e.g. IJulia, Blink, Juno...). See the WebIO readme for information on that.</p>
<h3><a id="user-content-limitations" class="anchor" aria-hidden="true" href="#limitations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Limitations</h3>
<p>When trying to display big tables (&gt;10k rows) we switch to lazy-loading additional rows while scrolling, which disables the filtering/sorting that's possible for smaller datasets. It's possible (but not trivial) to write proper backend support for those operations -- PRs would be very welcome.</p>

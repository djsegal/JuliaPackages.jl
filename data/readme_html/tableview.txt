<p dir="auto"><a href="https://github.com/JuliaComputing/TableView.jl/actions/workflows/CI.yml"><img src="https://github.com/JuliaComputing/TableView.jl/actions/workflows/CI.yml/badge.svg" alt="CI" style="max-width: 100%;"></a></p>
<p dir="auto">TableView.jl is an <a href="https://www.ag-grid.com/" rel="nofollow">ag-grid</a> based table viewer built on <a href="https://github.com/JuliaGizmos/WebIO.jl">WebIO.jl</a>. It can display arbitrarily large tables by lazy-loading additional data when scrolling (this is the default for datasets with more than 10k rows).</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer nofollow" href="https://user-images.githubusercontent.com/6735977/53032222-b9d06500-346e-11e9-8b7e-c18cbeb563f6.png"><img src="https://user-images.githubusercontent.com/6735977/53032222-b9d06500-346e-11e9-8b7e-c18cbeb563f6.png" alt="demo" style="max-width: 100%;"></a></p>

<p dir="auto"><code>showtable(yourtable)</code> returns a <code>WebIO.Scope</code> which can be displayed with multiple frontends (e.g. IJulia, Blink, Juno...). See the WebIO readme for information on that.</p>
<h3 dir="auto"><a id="user-content-limitations" class="anchor" aria-hidden="true" href="#limitations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Limitations</h3>
<p dir="auto">When trying to display big tables (&gt;10k rows) we switch to lazy-loading additional rows while scrolling, which disables the filtering/sorting that's possible for smaller datasets. It's possible (but not trivial) to write proper backend support for those operations -- PRs would be very welcome.</p>

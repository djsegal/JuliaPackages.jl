<h4 dir="auto"><a id="user-content-idw" class="anchor" aria-hidden="true" href="#idw"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>IDW</h4>
<p dir="auto">This solver provides a high-performance implementation of the inverse distance weighting scheme
introduced in the very early days of geostatistics (see <a href="https://dl.acm.org/citation.cfm?id=810616" rel="nofollow">Shepard 1968</a>).
It is perhaps the simplest first attempt in the literature to perform estimation based on the
notion of proximity to data locations.</p>
<p dir="auto">This implementation makes use of k-d trees from the <a href="https://github.com/KristofferC/NearestNeighbors.jl">NearestNeighbors.jl</a>
package, which leads to a fast estimation method for large or high-resolution spatial domains.
Although this method is recommended for fast assessment of a new field, it has poor statistical
properties (lacks covariance model) and should mainly be used for qualitative purposes.</p>

<p dir="auto">This solver provides an implementation of locally weighted regression (a.k.a. LOESS) introduced by
<a href="http://www.stat.washington.edu/courses/stat527/s13/readings/Cleveland_JASA_1979.pdf" rel="nofollow">Cleveland 1979</a>.
It is the most natural generalization of inverse distance weighting in which one is allowed to use a
custom weight function instead of distance-based weights.</p>
<p dir="auto">Like in the inverse distance weighting solver, this solver makes use of k-d trees from the
<a href="https://github.com/KristofferC/NearestNeighbors.jl">NearestNeighbors.jl</a> package for fast data
lookup. Locally weighted regression (LWR or LOESS) is a popular non-parametric method, however
it still has poor statistical properties compared to other estimation methods such as Kriging
that explicitly model spatial correlation.</p>
<p dir="auto">In the current implementation, the estimation variance is computed assuming Gaussian residuals.</p>

<p dir="auto">This polyalgorithm solver provides an implementation of various forms of Kriging introduced by
<a href="https://books.google.com.br/books/about/The_Theory_of_Regionalized_Variables_and.html" rel="nofollow">Matheron 1971</a>.
Kriging is a popular method in various industries due to its good statistical properties and flexibility.
Unlike the previous solvers, Kriging relies on the specification of a variogram model, which can be
fit to geospatial data.</p>

<h4 dir="auto"><a id="user-content-lugs" class="anchor" aria-hidden="true" href="#lugs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>LUGS</h4>
<p dir="auto">This solver provides an implementation of direct Gaussian simulation (a.k.a. LU simulation)
as described in <a href="https://link.springer.com/article/10.1007/BF00897191" rel="nofollow">Alabert 1987</a>. In this
method, the full covariance matrix is built to include all locations of the simulation domain,
and samples from the multivariate Gaussian are drawn via LU factorization.</p>
<p dir="auto">The method, which is widely implemented in many packages for Gaussian processes (e.g.
<a href="https://github.com/STOR-i/GaussianProcesses.jl">GaussianProcesses.jl</a>),
is appropriate for relatively small simulation domains (e.g. 100x100 grids) where it is feasible
to factorize the full covariance. For larger domains (e.g. 3D grids), other methods are available
such as sequential Gaussian simulation, spectral methods, and FFT moving averages.</p>
<h4 dir="auto"><a id="user-content-fftgs" class="anchor" aria-hidden="true" href="#fftgs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>FFTGS</h4>
<p dir="auto">This solver provides an implementation of spectral Gaussian simulation (a.k.a. FFT simulation)
as described in <a href="https://link.springer.com/article/10.1007/BF02769641" rel="nofollow">Gutjahr 1997</a>.
In this method, the covariance function is perturbed in the frequency
domain after a fast Fourier transform. White noise is added to the phase
of the spectrum, and a realization is produced by an inverse Fourier transform.</p>
<p dir="auto">The method is limited to simulations on regular grids, and care must be taken
to make sure that the correlation length is small enough compared to the grid
size. As a general rule of thumb, avoid correlation lengths greater than 1/3
of the grid. The method is extremely fast, and can be used to generate large
3D realizations.</p>

<p dir="auto">This solver provides an implementation of sequential Gaussian simulation as described in
<a href="https://link.springer.com/chapter/10.1007/978-94-011-1739-5_8" rel="nofollow">Gomez-Hernandez &amp; Journel 1993</a>.
The method traverses all locations of the spatial domain according to a path, approximates the
conditional distribution function within a neighborhood using Kriging, and assigns a value to
the center of the neighborhood by sampling from this distribution.</p>

<p dir="auto">This solver provides an implementation of the stochastic partial differential equations
(SPDE) approach to Gaussian simulation <a href="https://rss.onlinelibrary.wiley.com/doi/10.1111/j.1467-9868.2011.00777.x" rel="nofollow">Lindgren et al. 2011.</a>.
The method relies on a discretization of the Laplace-Beltrami operator on meshes and is
adequate for highly curved domains (e.g. surfaces).</p>

<p dir="auto">A geostatistical learning solver that takes a classical (non-spatial)
statistical learning model from the literature and applies it to each
point of the domain. Although this solver is widely used in geospatial
problems, it has various issues as described in
<a href="https://arxiv.org/abs/2102.08791" rel="nofollow">Hoffimann et al. 2021</a>.</p>

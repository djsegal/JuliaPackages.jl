<p dir="auto">This package contains abstracts type definition to ensure compatibility of the package <a href="https://github.com/JuliaGeo/GRIBDatasets.jl">GRIBDatasets</a> and <a href="https://github.com/Alexander-Barth/NCDatasets.jl">NCDatasets</a> for manipulating GRIB and NetCDF files. This package aims to follow the <a href="https://docs.unidata.ucar.edu/netcdf-c/current/netcdf_data_model.html" rel="nofollow">Common Data Model</a> and the <a href="https://cfconventions.org/" rel="nofollow">CF (climate and forecast models) Metadata Conventions</a>.</p>
<p dir="auto">This package is at a very early state of developpement.</p>
<p dir="auto">Here is minimal example for loading GRIB or NetCDF files.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="import CommonDataModel as CDM

<p dir="auto">Most users would typically import <a href="https://github.com/JuliaGeo/GRIBDatasets.jl"><code>GRIBDatasets</code></a> and <a href="https://github.com/Alexander-Barth/NCDatasets.jl"><code>NCDatasets</code></a> directly and not <code>CommonDataModel</code>. One should import <code>CommonDataModel</code> only to extent the functionality of <code>GRIBDatasets</code> and <code>NCDatasets</code>.</p>
<p dir="auto">As a proof-of-concept, there is also an <a href="https://github.com/Alexander-Barth/TIFFDatasets.jl"><code>TIFFDatasets</code></a> package for GeoTIFF files.</p>

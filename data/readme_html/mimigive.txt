<p dir="auto">The relevant arguments above are described as:</p>
<p dir="auto"><strong>Socioeconomic</strong></p>
<ul dir="auto">
<li>
<p dir="auto">socioeconomics_source (default :RFF) - The options are :RFF, which uses data from
the RFF socioeconomic projections, or :SSP, which uses data from one of the
Shared Socioeconomic Pathways</p>
</li>
<li>
<p dir="auto">SSP_scenario (default to nothing) - This setting is used only if one is using
the SSPs as the socioeconomics_source, and the current options are "SSP119",
"SSP126", "SSP245", "SSP370", "SSP585", and this will be used as follows.
See the SSPs component here: <a href="https://github.com/anthofflab/MimiSSPs.jl">https://github.com/anthofflab/MimiSSPs.jl</a> for more information.</p>
<p dir="auto">(1) Select the population and GDP trajectories for 2020 through 2300, mapping
each RCMIP scenario to the SSP (SSP1, 2, 3, 5 respectively)</p>
<p dir="auto">(2) Choose the ar6 scenario for data from 1750 - 2019 and the RCMIP emissions
scenario from the MimiSSPs component to pull Leach et al. RCMIP scenario
data for 2020 to 2300 for CO2, CH4, and N2O.</p>
<p dir="auto">(NOTE) that if the socioeconomics_source is :RFF this will not be consequential
and ssp245 will be used for the ar6 data from 1750 - 2019 and trace gases
from 2020 onwards, while emissions for CO2, CH4, and N2O will come from
the MimiRFFSPs component.</p>
</li>
</ul>
<ul dir="auto">
<li>RFFSPsample (default to nothing, which will pull the in MimiRFFSPs) - choose
the sample for which to run the RFF SSP. See the RFFSPs component here:
<a href="https://github.com/rffscghg/MimiRFFSPs.jl">https://github.com/rffscghg/MimiRFFSPs.jl</a>.</li>
</ul>
<p dir="auto"><strong>Agriculture</strong></p>
<ul dir="auto">
<li>Agriculture_gtap (default midDF) - specify the <code>Agriculture_gtap</code> input parameter as one of
<code>["AgMIP_AllDF", "AgMIP_NoNDF", "highDF", "lowDF", "midDF"]</code>, indicating which
gtap damage function the component should use.</li>
<li>Agriculture_floor_on_damages (default true) - If <code>Agriculture_gtap_floor_on_damages</code> = true, then
the agricultural damages (negative values of the <code>agcost</code> variable) in each
timestep will not be allowed to exceed 100% of the size of the  agricultural
sector in each region.</li>
<li>Agriculture_ceiling_on_benefits (default false) - If <code>Agriculture_gtap_ceiling_on_benefits</code> = true,
then the agricultural benefits (positive values of the <code>agcost</code> variable) in
each timestep will not be allowed to exceed 100% of the size of the agricultural
sector in each region.</li>
</ul>
<p dir="auto"><strong>Other</strong></p>
<ul dir="auto">
<li>vsl (default :epa) - Specify the soruce of the value of statistical life (VSL) being used in the model. The default <code>:epa</code> uses the 2017 VSL used in U.S. EPA anlayses. Alternatively, one could use <code>:fund</code>. Both are described in the <code>DataExplainer</code> along with references to the underlying values.</li>
</ul>

<p dir="auto">This function returns the results of a Monte Carlo Simulation with the defined number of <code>trials</code>, and save data into the <code>output_dir</code> folder, optionally also saving the parameter settings for each individual trial if <code>save_trials</code> is set to <code>true</code> If no model <code>m</code> is provided we will run with the default model from <code>get_model()</code>. The rest of the arguments are described as follows:</p>
<ul dir="auto">
<li><code>trials</code> (default 10,000) - number of trials to be run, used for presampling</li>
<li><code>output_dir</code> (default constructed folder name) - folder to hold results</li>
<li><code>save_trials</code> (default false) - whether to save all random variables for all trials to trials.csv</li>
<li><code>fair_parameter_set</code> (default :random) - :random means FAIR mcs samples will be chosen randomly from the provided sets, while :deterministic means they will be  based on the provided vector of to <code>fair_parameter_set_ids</code> keyword argument.</li>
<li><code>fair_parameter_set_ids</code> - (default nothing) - if <code>fair_parameter_set</code> is set to :deterministic, this <code>n</code> element vector provides the fair parameter set ids that will be run, otherwise it is set to <code>nothing</code> and ignored.</li>

<p dir="auto">This function computes the social cost of a gas for an emissions pulse in <code>year</code> <strong>in $2005 USD</strong> for the provided MimiGIVE model, which can be specified as above with a particular settings. If no model is provided, the default model from MimiGIVE.get_model() is used. Furthermore, the <code>DamagesAggregator</code> component allows users to decide which damages are included in the aggregated damages used for the SCC.  The rest of the arguments are described as follows:</p>
<ul dir="auto">
<li><code>m</code> (default get_model()) - if no model is provided, the default model from MimiGIVE.get_model() is used.</li>
<li>'year` (default nothing) - year of which to calculate SC (year of pulse)</li>
<li><code>last_year</code> (default 2300) - last year to include in damages summation</li>
<li><code>prtp</code> (default 0.015) and <code>eta</code> (default 1.45) - Ramsey discounting parameterization</li>
<li><code>discount_rates</code> (default nothing) - a vector of Named Tuples ie. [(prpt = 0.03., eta = 1.45), (prtp = 0.015, eta = 1.45)] - required if running n &gt; 1</li>
<li><code>certainty_equivalent</code> (default false) - whether to compute the certainty equivalent or expected SCC</li>
<li><code>fair_parameter_set</code> (default :random) - :random means FAIR mcs samples will be chosen randomly from the provided sets, while :deterministic means they will be  based on the provided vector of to <code>fair_parameter_set_ids</code> keyword argument.</li>
<li><code>fair_parameter_set_ids</code> - (default nothing) - if <code>fair_parameter_set</code> is set to :deterministic, this <code>n</code> element vector provides the fair parameter set ids that will be run, otherwise it is set to <code>nothing</code> and ignored.</li>
<li><code>rffsp_sampling</code> (default :random) - which sampling strategy to use for the RFF  SPs, :random means RFF SPs will be chosen randomly, while :deterministic means they will be based on the provided vector of to <code>rffsp_sampling_ids</code> keyword argument.</li>
<li><code>rffsp_sampling_ids</code> - (default nothing) - if <code>rffsp_sampling</code> is set to :deterministic,  this <code>n</code> element vector provides the RFF SP ids that will be run, otherwise it is set to <code>nothing</code> and ignored.</li>
<li><code>n</code> (default 0) - If <code>n</code> is 0, the deterministic version will be run, otherwise, a monte carlo simulation will be run.</li>
<li><code>gas</code> (default :CO2) - the gas for which to compute the SC, options are :CO2, :CH4, and :N2O</li>
<li><code>save_list</code> (default []) - which parameters and varaibles to save for each trial, entered as a vector of Tuples (:component_name, :variable_name)</li>
<li><code>output_dir</code> (default constructed folder name) - folder to hold results</li>
<li><code>save_md</code> (default is false) - save and return the marginal damages from a monte carlo simulation</li>
<li><code>save_cpc</code> (default is false) - save and return the per capita consumption from a monte carlo simulation</li>
<li><code>save_slr_damages</code>(default is false) - save global sea level rise damages from CIAM to disk</li>
<li><code>compute_sectoral_values</code> (default is false) - compute and return sectoral values as well as total</li>
<li><code>compute_domestic_values</code> (default is false) - compute and return domestic values in addition to global</li>

<li><code>CIAM_GDPcap</code> (default is false) - Limit SLR damages to country-level annual GDP</li>
<li><code>pulse_size</code> (default 1.) - This determines the size of the additional pulse of emissions. Default of <code>1.</code> implies the standard pulse size of 1Gt of C for CO2, 1Mt of CH4, and 1Mt of N2O.</li>
</ul>
<p dir="auto"><strong>Discount Rate Note</strong>: In scc.jl , the rate of pure time preference <code>prtp</code> is treated as a discrete variable, such that <code>prtp</code> is applied in the discrete form <code>1/(1+prtp)^(t-year)</code>. Note that if using a <code>prtp</code> value calculated as a continuous time rate, like those in Rennert et al. (2021), one must transform this discount factor with <code>prtp_discrete = exp(prtp)-1)</code>.  Please be in touch with the developers if you need assistance or further explanation!</p>
<p dir="auto">The social cost of a gas can be calculated either deterministically or through a Monte Carlo Simulation which incorporates parametric uncertainty and is the recommended approach. Details follow in sections 4a and 4b.</p>

Monte Carlo Simulation (MCS) SCC Calculation</h2>
<p dir="auto">As described in Section 3, you can run the model using a Monte Carlo Simulation. This functionality can be used to compute a distribution of SCCs using the <code>n</code> parameter of <code>compute_scc</code>.  If <code>n</code> is 0, as is default, the deterministic SCC will be calculated.  If <code>n</code> is 2 or greater, a Monte Carlo Simulation will be run.  Note that currently for this option <em>you must use the <code>discount_rates</code> argument</em>.</p>
<p dir="auto">The simplest case of running a Monte Carlo Simulation would look like:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="discount_rates = [(label=&quot;Ramsey&quot;, prtp=0.015, eta=1.45), (label=&quot;Constant 2%&quot;, prtp=0.02, eta=0.)]

<p dir="auto"><strong>Optional Extra Output Specifications</strong></p>
<ul dir="auto">
<li><strong>Marginal Damages</strong> (only relevant for Monte Carlo Simulation): Set keyword argument <code>save_md</code> to <code>true</code> to include undiscounted marginal damages in the returned results.</li>
<li><strong>Net Per Capita Consumption</strong> (only relevant for Monte Carlo Simulation): Set keyword argument <code>save_cpc</code> to <code>true</code> to include net per capita consumption in the returned results.</li>
<li><strong>Sectorally Disaggregated Values</strong> (only relevant for Monte Carlo Simulation): Set keyword argument <code>compute_sectoral_values</code> to <code>true</code> to compute sectorally disaggregated values. Calculations of the disaggregated sectoral SCCs will use global consumption to calculate discount factors, and thus the discount factors are consistent between the global and sectoral calculations of the SCC. To compute an isolated sectoral SCC one may run a separate simulation with only that sector's damages turned on.</li>
<li><strong>Within U.S. Borders Values</strong> - Set keyword argument <code>compute_domestic_values</code> to <code>true</code> to include SCC (and optional marginal damage) values disaggregated to the within-borders USA damages.  Calculations of the disaggregated within U.S. borders SCC will use global consumption to calculate discount factors, and thus the discount factors are consistent between the global and within borders calculations of the SCC.</li>
</ul>

<p dir="auto"><strong>Returned <code>result</code> Object Structure</strong></p>
<p dir="auto">The object returned by <code>result = MimiGIVE.compute_scc(...)</code> for a MCS is a <code>Dictionary</code> with 1-3 keys: <code>scc</code> (always), <code>:mds</code> (if <code>save_md</code> is set to <code>true</code>) and <code>:cpc</code> (if <code>save_cpc</code> is set to <code>true</code>). <ul dir="auto">
<li><code>results[:scc]</code> accesses a Dictionary with keys being <code>NamedTuples</code> with elements (region, sector, dr_label, prtp, eta) and values which are <code>NamedTuples</code> with elements (expected_scc, se_expected_scc, and scc) as well as ce_scc and ce_sccs if certainty_equivalent=true</li>

<li><code>results[:cpc]</code> accesses a Dictionary with keys being <code>NamedTuples</code> with elements (region, sector) and values which are matrices of size num trials x 281 years (2020:2300) of net per capita consumption in USD $2011</li>
</ul>

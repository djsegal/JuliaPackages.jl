<div id="readme" class="md" data-path="README.md"><article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-snowballstemmerjl" class="anchor" aria-hidden="true" href="#snowballstemmerjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>SnowballStemmer.jl</h1>
<p>The SnowballStemmer.jl package extracts the stemmer functionality of the <code>TextAnalysis.jl</code> package, which is also a wrapper for <a href="http://snowball.tartarus.org/" rel="nofollow">libstemmer</a>.
The idea is to expose the stemming functions without forcing your programs to follow the interfaces of <code>TextAnalysis.jl</code>.</p>

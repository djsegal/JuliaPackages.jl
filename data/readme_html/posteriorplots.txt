<p dir="auto">R has excellent packages for the plotting and analysis of Bayesian models fit in probabilistic programming languages such as <a href="https://mc-stan.org" rel="nofollow"><code>Stan</code></a>. Examples of these packages include <a href="http://mc-stan.org/bayesplot/" rel="nofollow"><code>bayesplot</code></a> and <a href="http://mjskay.github.io/tidybayes/" rel="nofollow"><code>tidybayes</code></a>. The functionality afforded by these packages greatly enables researchers to automatically obtain informative and clean graphical summaries of various posterior properties of interest. While packages such as <a href="https://turinglang.github.io/MCMCChains.jl/dev/" rel="nofollow"><code>MCMCChains.jl</code></a>, <a href="https://arviz-devs.github.io/ArviZ.jl/stable/" rel="nofollow"><code>ArviZ.jl</code></a>, and others exist in Julia for models built in the <a href="https://en.wikipedia.org/wiki/Probabilistic_programming" rel="nofollow">PPLs</a> <a href="https://turing.ml/stable/" rel="nofollow"><code>Turing.jl</code></a> and <a href="https://github.com/cscherrer/Soss.jl"><code>Soss.jl</code></a>, the clean, inference-ready output aesthetics produced by <code>bayesplot</code> are not easily available by default. <code>PosteriorPlots.jl</code> seeks to bridge this gap. Since <code>PosteriorPlots.jl</code> functions can take standard object types such as Arrays as inputs (as well as special object types such as Chains), it can flexibly accommodate models from <code>Stan</code> and other PPLs.</p>

<p dir="auto"><code>PosteriorPlots.jl</code> provides intuitive and simple functionality for both statistical inference of model parameter posterior distributions as well as visualisation and interpretation of model fits and posterior diagnostics. Package functionality can be summarised across these two domains:</p>

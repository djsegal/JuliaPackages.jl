<p dir="auto">A Julia package for interacting with the Twitter API.</p>

<p dir="auto">Twitter.jl is a Julia package to work with the Twitter API v1.1. Currently, only the REST API methods are supported; streaming API endpoints aren't implemented at this time.</p>
<p dir="auto">All functions have required arguments for those parameters required by Twitter and an <code>options</code> keyword argument to provide a <code>Dict{String, String}</code> of optional parameters <a href="https://developer.twitter.com/en/docs/twitter-api/v1" rel="nofollow">Twitter API documentation</a>. Most function calls will return either a <code>Dict</code> or an <code>Array &lt;: TwitterType</code>. Bad requests will return the response code from the API (<code>403</code>, <code>404</code>, etc).</p>
<p dir="auto">DataFrame methods are defined for functions returning composite types: <code>Tweets</code>, <code>Places</code>, <code>Lists</code>, and <code>Users</code>.</p>

<p dir="auto">Before one can make use of this package, you must create an application on the <a href="https://dev.twitter.com" rel="nofollow">Twitter's Developer Platform</a>.</p>
<p dir="auto">Once your application is approved, you can access your dashboard/portal to grab your authentication credentials from the "Details" tab of the application.</p>
<p dir="auto">Note that you will also want to ensure that your App has Read / Write OAuth access in order to post tweets. You can find out more about this <a href="https://stackoverflow.com/a/70958807/7619808" rel="nofollow">on Stack Overflow</a>.</p>

<p>Based on julias <code>digits()</code> function.</p>

<h5><a id="user-content-reversedigitsnreversedigitsl" class="anchor" aria-hidden="true" href="#reversedigitsnreversedigitsl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>reversedigits(n)/reversedigits!(l)</h5>
<p>Returns the digits of n in reversed order, i.e. <code>reversedigits(n) == undigit(reverse!(digits(n)))</code>. For a list of digits this is equivalent to <code>reverse()</code> and <code>reverse!()</code>.</p>

<p>The inverse function to julias <code>digits()</code>. Accepting a vector of integers where more significant digits are at higher indexes, according to <code>digits()</code> and a base. Default is 10.</p>

<p>Creates a digit histogram for a given number. The result is a 1-dimensional 10-element array containing the count of each digit. The first element contains the count of 0, while the last represents the count of 9.</p>

<p>Checks whether a or b is part of the other, e.g. <code>contains(1356,35)</code> will result in <code>true</code>.</p>

<p>Checks wether a or b is the first/last part of the other, e.g. <code>startswith(1356,1)</code>, <code>startswith(32,3236)</code>, <code>endswith(31,9831)</code> will all return <code>true</code>.</p>

<p>Returns true if a and b are valid permutations of their digits.</p>

<p>Returns true if n is a palindromic number.</p>
<h5><a id="user-content-cropni" class="anchor" aria-hidden="true" href="#cropni"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>crop(n,i)</h5>
<p>Cuts off i digits from n. If i is positive the first (most significant) digits are cropped. If i is negative the last digits are cut off.</p>
<h5><a id="user-content-combineab" class="anchor" aria-hidden="true" href="#combineab"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>combine(a,b)</h5>
<p>Return the combination of a and b, similar to a string concatenation, e.g. <code>combine(13,56)</code> -&gt; <code>1356</code>.</p>

<p>Returns the digits of n with index idx. idx can be an array or range, e.g. <code>select(64247,[2:4])</code> -&gt; <code>424</code></p>
<h5><a id="user-content-replacenidxvals--replacelidxvals" class="anchor" aria-hidden="true" href="#replacenidxvals--replacelidxvals"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>replace(n,idx,vals) / replace!(l,idx,vals)</h5>
<p>Replaces the digits of n with index idx by the values in vals. vals has to be same size as idx.
For arrays of digits there is also an inplace version of replace.
Example: <code>replace(6215,[2,4],[8,7])</code> -&gt; <code>6817</code></p>

<p>Replaces the all digits of n with value olddigit with newdigit. There is also an inplace operation for arrays of digits.

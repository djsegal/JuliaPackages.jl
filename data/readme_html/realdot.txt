<p dir="auto">This package only contains and exports a single function <code>realdot(x, y)</code>.
It computes <code>real(LinearAlgebra.dot(x, y))</code> while avoiding computing the imaginary part of <code>LinearAlgebra.dot(x, y)</code> if possible.</p>
<p dir="auto">The real dot product is useful when one treats complex numbers as embedded in a real vector space.
For example, take two complex arrays <code>x</code> and <code>y</code>.
Their real dot product is <code>real(dot(x, y)) == dot(real(x), real(y)) + dot(imag(x), imag(y))</code>.
This is the same result one would get by reinterpreting the arrays as real arrays:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="xreal = reinterpret(real(eltype(x)), x)
yreal = reinterpret(real(eltype(y)), y)
real(dot(x, y)) == dot(xreal, yreal)"><pre>xreal <span class="pl-k">=</span> <span class="pl-c1">reinterpret</span>(<span class="pl-c1">real</span>(<span class="pl-c1">eltype</span>(x)), x)
yreal <span class="pl-k">=</span> <span class="pl-c1">reinterpret</span>(<span class="pl-c1">real</span>(<span class="pl-c1">eltype</span>(y)), y)
<span class="pl-c1">real</span>(<span class="pl-c1">dot</span>(x, y)) <span class="pl-k">==</span> <span class="pl-c1">dot</span>(xreal, yreal)</pre></div>
<p dir="auto">In particular, this function can be useful if you define pullbacks for non-holomorphic functions (see e.g. <a href="https://github.com/JuliaDiff/ChainRulesCore.jl/pull/474" data-hovercard-type="pull_request" data-hovercard-url="/JuliaDiff/ChainRulesCore.jl/pull/474/hovercard">this discussion in the ChainRulesCore.jl repo</a>).
It was implemented initially in <a href="https://github.com/JuliaDiff/ChainRules.jl">ChainRules.jl</a> in <a href="https://github.com/JuliaDiff/ChainRules.jl/pull/216" data-hovercard-type="pull_request" data-hovercard-url="/JuliaDiff/ChainRules.jl/pull/216/hovercard">this PR</a> as <code>_realconjtimes</code>.</p>

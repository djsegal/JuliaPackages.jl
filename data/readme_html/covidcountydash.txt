<p>Visualization of <a href="https://github.com/nytimes/covid-19-data">data</a> from <a href="https://www.nytimes.com/interactive/2020/us/coronavirus-us-cases.html" rel="nofollow">The New York Times</a>, based on reports from state and local
health agencies; inspired by <a href="https://twitter.com/jburnmurdoch" rel="nofollow">John Burn-Murdoch</a>'s <a href="https://www.ft.com/coronavirus-latest" rel="nofollow">FT analyses</a> of national-level data.</p>
<p>This has become significantly more complicated with successive iterations; see an earlier
version in the history (like <a href="https://github.com/mbauman/CovidCountyDash.jl/blob/2633d5b665b3e053b8a01411b6adb270ef2fe60f/dashboard.jl">2633d5b</a>) for a simpler example.</p>
<h2><a id="user-content-data" class="anchor" aria-hidden="true" href="#data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Data</h2>
<p>Note the geographical <a href="https://github.com/nytimes/covid-19-data#geographic-exceptions">caveats in the NYT dataset</a>; the most annoying of which are the fact
that the 5 NYC boroughs are all lumped together and Kansas City, MO reports its numbers
separately of the 4 counties it touches.  Further, note that US territories and the District
of Columbia are listed as "states."</p>
<p>This dataset is joined with the US Census <a href="https://www2.census.gov/programs-surveys/popest/datasets" rel="nofollow">population estimates for 2019</a> to facilitate
rates per population, but special care is needed for those caveats. In particular, the city
of KCMO is split between portions of 4 different counties (Cass, Clay, Jackson and Platte)
and does not wholly encompass any one of them. Since KCMO is reported separately, we pull
its population estimate from the <a href="https://www.census.gov/quickfacts/fact/table/kansascitycitymissouri/PST045218" rel="nofollow">census' 2018 estimate</a>
(2019 is not yet available), and then remove population as appropriate from each of those
four counties (using the 2018 breakdown <a href="https://www.marc.org/Data-Economy/Metrodataline/Population/Current-Population-Data" rel="nofollow">available at MARC</a>).</p>
<p>Population estimates are not applied to US territories (so population normalization is not
available for them), and some states report cases from "Unknown" counties (with 0 population).</p>

<p>This is setup to be hosted on free-tier Heroku dyno. In the free tier, the app gets killed
after 30 minutes of inactivity and we're given terribly anemic access to the CPU. To reduce
startup costs, a special <a href="https://github.com/mbauman/heroku-buildpack-julia">heroku buildpack</a> that uses <a href="https://github.com/JuliaLang/PackageCompiler.jl">PackageCompiler.jl</a> is under development.</p>

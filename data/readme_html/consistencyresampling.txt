<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Overview</h2>
<p>This package implements consistency resampling in the Julia language, as described by <a href="https://doi.org/10.1175/WAF993.1" rel="nofollow">Bröcker and Smith (2007)</a>.
It is based on the <a href="https://github.com/juliangehring/Bootstrap.jl"><code>Bootstrap.jl</code></a> package for statistical bootstrapping in Julia.</p>
<p>Consistency resampling is a resampling technique that generates calibrated predictions from a data set of predictions
and corresponding labels. First a set of predictions is sampled from the data set with replacement. In a second step
artificial labels are sampled with the predicted probabilities. This resampling procedure ensures that the predictions
are calibrated for the artificial labels.</p>
<h2><a id="user-content-example" class="anchor" aria-hidden="true" href="#example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Example</h2>
<p>The predictions have to be provided as a matrix of size <code>(m, n)</code>, in which each of the <code>n</code> columns corresponds to
predicted probabilities of the labels <code>1,…,m</code>. The corresponding labels have to be provided as a vector of length <code>n</code>,
in which every element is from the set <code>1,…,m</code>.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> Distributions

predictions <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">Dirichlet</span>(<span class="pl-c1">10</span>, <span class="pl-c1">1</span>), <span class="pl-c1">500</span>)
labels <span class="pl-k">=</span> <span class="pl-c1">rand</span>(<span class="pl-c1">1</span><span class="pl-k">:</span><span class="pl-c1">10</span>, <span class="pl-c1">500</span>)</pre></div>
<p>Consistency resampling is performed similar to the other bootstrapping approaches in <code>Bootstrap.jl</code>. A random number
generator can be provided as optional argument.</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> ConsistencyResampling
<span class="pl-k">using</span> Distances
<span class="pl-k">using</span> Flux<span class="pl-k">:</span> onehotbatch

b <span class="pl-k">=</span> <span class="pl-c1">bootstrap</span>((predictions, labels), <span class="pl-c1">ConsistentSampling</span>(<span class="pl-c1">100_000</span>)) <span class="pl-k">do</span> (x, y)
  <span class="pl-c1">totalvariation</span>(x, <span class="pl-c1">onehotbatch</span>(y, <span class="pl-c1">1</span><span class="pl-k">:</span><span class="pl-c1">10</span>)) <span class="pl-k">/</span> <span class="pl-c1">500</span>
<span class="pl-k">end</span></pre></div>
<p>The bootstrapped samples can be explored and used for estimation of confidence intervals, as explained
in the documentation of <code>Bootstrap.jl</code>.</p>

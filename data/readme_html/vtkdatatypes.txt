<p dir="auto">VTKDataTypes.jl presents a Julia type system for representing and manipulating VTK data natively in Julia. <strong>VTKDataTypes.jl only supports Julia v1.</strong></p>

<p dir="auto">You can use VTKDataTypes.jl to create your own VTK data object of any of the following types: <code>VTKUnstructuredData</code>, <code>VTKPolyData</code>, <code>VTKStructuredData</code>, <code>VTKRectilinearData</code>, <code>VTKImageData</code>, <code>VTKMultiblockData</code>, and <code>VTKTimeSeriesData</code>. 2D and 3D data are both supported and conversion functions are defined where they make sense.</p>

<p dir="auto">You can use the <code>decompose</code> function to decompose your VTK data object to faces or lines averaging cell data of all participating cells in the face or edge. The <code>triangulate</code> function is also defined for interfacing with other rendering and visualization modules which require a triangular mesh.</p>

<p dir="auto">You can use the <code>GLMesh</code> function to change a VTK data object to a triangulated <code>GLNormalVertexcolorMesh</code> using one of the point data variable names to define the vertex color specified by the <code>color</code> option. Other options include <code>opacity</code>, and <code>component</code> to specify a vector data component, as opposed to the magnitude which is used by default.</p>

<p dir="auto"><code>point_coords</code> : this is a <code>Matrix{T}</code> of size <code>(dims, points)</code> where <code>dims</code> is the number of dimensions and <code>points</code> is the number of points.</p>
<p dir="auto"><code>cell_types</code> : this is a <code>Vector{Int}</code> holding the cell types using VTK's convention for cell ids.</p>
<p dir="auto"><code>cell_connectivity</code> : this is a <code>Vector{Vector{Int}}</code> holding the cell connectivity of each cell as per VTK's convention for each cell type.</p>
<p dir="auto"><code>point_data</code> : this is a <code>Dict{String, Array}</code> holding point-specific scalar and vector data. Scalar data arrays are <code>Vector{T}</code> and vector data arrays are <code>Matrix{T}</code> of size <code>(var_dims, points)</code> where <code>var_dims</code> is the number of dimensions of the vector variable.</p>
<p dir="auto"><code>cell_data</code> : this is a <code>Dict{String, Array}</code> holding cell-specific scalar and vector data. Scalar data arrays are <code>Vector{T}</code> and vector data arrays are <code>Matrix{T}</code> of size <code>(var_dims, cells)</code> where <code>var_dims</code> is the number of dimensions of the vector variable and <code>cells</code> is the number of cells.</p>

<p dir="auto">Has the same fields as <code>VTKUnstructuredData</code> but all cells are assumed to be of types: 1, 2, 3, 4, 5, 6, 7, 8, and/or 9.</p>
<h3 dir="auto"><a id="user-content-vtkstructureddatat" class="anchor" aria-hidden="true" href="#vtkstructureddatat"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>VTKStructuredData{T}</h3>
<p dir="auto"><code>point_coords</code> : this is an <code>Array{T,dims+1}</code>. The size of the array is <code>(dims, extents(dataset)...)</code> where <code>extents</code> is a function that returns the number of grid markers along each dimension.</p>
<p dir="auto"><code>point_data</code> : this is a <code>Dict{String, Array}</code> holding point-specific scalar and vector data. Scalar data arrays are stored in <code>Array{T,dims}</code>. The size of a scalar data array is <code>extents(dataset)</code>. Vector data arrays are stored in <code>Array{T,dims+1}</code> of size <code>(var_dims, extents(dataset)...)</code>.</p>
<p dir="auto"><code>cell_data</code> : this is a <code>Dict{String, Array}</code> holding cell-specific scalar and vector data. Scalar data arrays are stored in <code>Array{T,dims}</code>. The size of a scalar data array is <code>cell_extents(dataset)</code>, where <code>cell_extents</code> is a function that returns <code>extents(dataset) .- 1</code> resembling the number of cells along each dimension. Vector data arrays are stored in <code>Array{T,dims+1}</code> of size <code>(var_dims, cell_extents(dataset)...)</code>.</p>
<p dir="auto">All cells are of type 9 (Quad) in 2D and 12 (Hexa) in 3D.</p>

<p dir="auto"><code>blocks</code> : a <code>Tuple{Vararg{AbstractStaticVTKData}}</code> which can store a mixture of all previous types as well as <code>VTKMultiblockData</code> recursively.</p>

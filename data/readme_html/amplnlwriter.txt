<p>This <a href="https://github.com/JuliaLang/julia">Julia</a> package is an interface between <a href="https://github.com/JuliaOpt/MathOptInterface.jl">MathOptInterface.jl</a> and <a href="http://www.ampl.com" rel="nofollow">AMPL-enabled</a> <a href="http://ampl.com/products/solvers/all-solvers-for-ampl/" rel="nofollow">solvers</a>. It is similar in nature to <a href="https://github.com/tkelman/CoinOptServices.jl">CoinOptServices.jl</a>, but instead uses AMPL's low-level <a href="https://en.wikipedia.org/wiki/Nl_%28format%29" rel="nofollow">.nl</a> file format.</p>
<p>A list of AMPL-enabled solvers is available <a href="http://ampl.com/products/solvers/all-solvers-for-ampl/" rel="nofollow">here</a>.</p>
<p><em>Development of AmplNLWriter.jl is community driven and has no official connection with the AMPL modeling language or AMPL Optimization Inc.</em></p>

<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<p>AmplNLWriter.jl provides <code>AmplNLWriter.Optimizer</code> as a usable solver in JuMP. The following Julia code uses the Bonmin solver in JuMP via AmplNLWriter.jl:</p>
<pre><code>julia&gt; using JuMP, AmplNLWriter
julia&gt; m = Model(with_optimizer(AmplNLWriter.Optimizer, "bonmin"))
</code></pre>
<p>You can then model and solve your optimization problem as usual. See <a href="https://github.com/JuliaOpt/JuMP.jl/blob/master/README.md">JuMP.jl</a> for more details.</p>
<p>The <code>AmplNLWriter.Optimizer()</code> constructor requires as the first argument the name of the solver command needed to run the desired solver. For example, if the <code>bonmin</code> executable is on the system path, you can use this solver using <code>AmplNLWriter.Optimizer("bonmin")</code>. If the solver is not on the path, the full path to the solver will need to be passed in. This solver executable must be an AMPL-compatible solver.</p>
<p>The second (optional) argument to <code>AmplNLWriter.Optimizer()</code> is a <code>Vector{ASCIIString}</code> of solver options. These options are appended to the solve command separated by spaces, and the required format depends on the solver that you are using. Generally, they will be of the form <code>"key=value"</code>, where <code>key</code> is the name of the option to set and <code>value</code> is the desired value. For example, to set the NLP log level to 0 in Bonmin, you would run <code>AmplNLWriter.Optimizer("bonmin", ["bonmin.nlp_log_level=0"])</code>. For a list of options supported by your solver, check the solver's documentation, or run <code>/path/to/solver -=</code> at the command line e.g. run <code>bonmin -=</code> for a list of all Bonmin options.</p>
<p>Note that some of the options don't seem to take effect when specified using the command-line options (especially for Couenne), and instead you need to use an <code>.opt</code> file. The <code>.opt</code> file takes the name of the solver, e.g. <code>bonmin.opt</code>, and each line of this file contains an option name and the desired value separated by a space. For instance, to set the absolute and relative tolerances in Couenne to 1 and 0.05 respectively, the <code>couenne.opt</code> file should be</p>
<pre><code>allowable_gap 1
allowable_fraction_gap 0.05
</code></pre>
<p>In order for the options to be loaded, this file must be located in the current working directory whenever the model is solved.</p>
<p>A list of available options for the respective <code>.opt</code> files can be found here:</p>
<ul>
<li><a href="http://www.coin-or.org/Ipopt/documentation/node39.html#app.options_ref" rel="nofollow">Ipopt</a></li>
<li><a href="https://github.com/coin-or/Bonmin/blob/master/Bonmin/test/bonmin.opt">Bonmin</a> (plus Ipopt options)</li>
<li><a href="https://github.com/coin-or/Couenne/blob/master/Couenne/src/couenne.opt">Couenne</a> (plus Ipopt and Bonmin options)</li>
</ul>

<h3><a id="user-content-bonmincouenneipopt" class="anchor" aria-hidden="true" href="#bonmincouenneipopt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Bonmin/Couenne/Ipopt</h3>
<p><strong>NOTE: AmplNLWriter v0.4.0 introduced a breaking change by removing <code>BonminNLSolver</code>, <code>CouenneNLSolver</code>, and <code>IpoptNLSolver</code>. Users are now expected
to pass the path of the solver executable to <code>AmplNLWriter.Optimizer</code>.</strong></p>
<p>If you have <a href="https://github.com/JuliaOpt/Ipopt.jl">Ipopt.jl</a> installed, you can use Ipopt by using the solver <code>AmplNLWriter.Optimizer(Ipopt.amplexe, options)</code>.</p>
<h3><a id="user-content-scip" class="anchor" aria-hidden="true" href="#scip"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>SCIP</h3>
<p>To use SCIP with AmplNLWriter.jl, you must first compile the <code>scipampl</code> binary which is a version of SCIP with support for the AMPL .nl interface. To do this, you can follow the instructions <a href="http://zverovich.net/2012/08/07/using-scip-with-ampl.html" rel="nofollow">here</a>, which we have tested on OS X and Linux.</p>
<p>After doing this, you can access SCIP through <code>AmplNLWriter.Optimizer("/path/to/scipampl")</code>. Options can be specified for SCIP using a <code>scip.set</code> file, where each line is of the form <code>key = value</code>. For example, the following <code>scip.set</code> file will set the verbosity level to 0:</p>
<pre><code>display/verblevel = 0
</code></pre>
<p>A list of valid options for the file can be found <a href="http://plato.asu.edu/milp/scip.set" rel="nofollow">here</a>.</p>
<p>To use the <code>scip.set</code> file, you must pass the path to the <code>scip.set</code> file as the first (and only) option to the solver:</p>
<pre><code>AmplNLWriter.Optimizer("/path/to/scipampl", ["/path/to/scip.set"])
</code></pre>

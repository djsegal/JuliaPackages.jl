<p dir="auto">Extends <a href="https://painterqubits.github.io/Unitful.jl/" rel="nofollow">Unitful.jl</a> to include <em>angle</em> as a dimension and allow <a href="https://docs.julialang.org/en/v1/manual/methods/" rel="nofollow">dispatching</a> on <em>angles</em>.</p>

<p dir="auto">Use <em>DimensionfulAngles.jl</em> for units containing angles and <em>Unitful.jl</em> for all other units.
The units in <em>DimensionfulAngles.jl</em> are differentiated from their dimensionless counterparts with a subscript <code>·µÉ</code>.
Simply add this subscript to any angle units to make it dimensionful.
In Julia environments this can be typed as <code>\^a&lt;tab&gt;</code>.</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Unitful

julia&gt; using DimensionfulAngles

julia&gt; dimensionless_angle = 1u&quot;rad&quot;

1 rad

julia&gt; dimensionful_angle = 1u&quot;rad·µÉ&quot;
1 rad

julia&gt; typeof(dimensionless_angle)
Quantity{Int64, NoDims, Unitful.FreeUnits{(rad,), NoDims, nothing}}

<p dir="auto">When defining quantities with units from <em>DimensionfulAngles.jl</em> you can use the <code>ua</code> string instead and omit the subscript <code>·µÉ</code>.
But when defining quantities with units from both it is more convenient to use the <code>u</code> string.</p>

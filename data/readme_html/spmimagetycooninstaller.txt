<p dir="auto"><em>Please only use the app if you read the disclaimer on the <a href="https://github.com/alexriss/SpmImageTycoon.jl">SpmImage Tycoon project page</a>.</em></p>
<p dir="auto">Usage is simple:</p>
<ol dir="auto">
<li>Install <a href="https://julialang.org/" rel="nofollow">Julia</a></li>
<li>Start Julia and type the following two commands:</li>
</ol>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Pkg
Pkg.add(&quot;SpmImageTycoonInstaller&quot;)
using SpmImageTycoonInstaller
install()"><pre><span class="pl-k">using</span> Pkg
Pkg<span class="pl-k">.</span><span class="pl-c1">add</span>(<span class="pl-s"><span class="pl-pds">"</span>SpmImageTycoonInstaller<span class="pl-pds">"</span></span>)
<span class="pl-k">using</span> SpmImageTycoonInstaller
<span class="pl-c1">install</span>()</pre></div>
<p dir="auto">This will install a compiled version of <a href="https://github.com/alexriss/SpmImageTycoon.jl">SpmImage Tycoon</a>.
The installation will typically take 10 to 20 minutes and take up around 1 GB of space.
The same procedure can be used to update to the latest version.</p>
<p dir="auto">The compiled version will start fast, even on the first run.</p>
<p dir="auto"><em>Tested on Windows, Linux, and MacOS. However, so far, on MacOS no shortcuts/aliases are created.</em></p>

<p dir="auto">Please make sure to use Julia 1.8. I have tested the installer on Windows 11, Ubuntu/WSL, Manjaro 22, as well as macOS x86.
Installations worked well, except for certain configurations of Manjaro Linux.
Here, the package compilation errored, likely related to <a href="https://github.com/JuliaLang/PackageCompiler.jl/issues/16" data-hovercard-type="issue" data-hovercard-url="/JuliaLang/PackageCompiler.jl/issues/16/hovercard">this bug</a>.
As a workaround, install Julia via the <a href="https://julialang.org/downloads/" rel="nofollow">official binaries</a> (instead of via the distro's package manager).</p>

<p dir="auto">
ExifViewer.jl is a Julia wrapper of the C library libexif that provides EXIF support. EXIF is short for Exchangeable Image File, a format that is a standard for storing interchange information in digital photography image files using JPEG compression. 
</p>

<p dir="auto">ExifViewer.jl provides method to read EXIF tags from images using <code>read_tags</code> methods which can
take input in form of Filepath, IO, and bytes sequence(<code>Vector{UInt8}</code>)</p>
<p dir="auto"><code>read_tags</code> reads EXIF tags from the input source data and it returns an empty
dictionary if the source data doesn't contain EXIF tags.
There are couple of keyword arguments that are used by <code>read_tags</code> which have been
described below:</p>
<h4 dir="auto"><a id="user-content-keyword-arguments" class="anchor" aria-hidden="true" href="#keyword-arguments"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Keyword Arguments</h4>
<ul dir="auto">
<li>
<p dir="auto"><code>ifds::Union{Int,NTuple,UnitRange}</code> : Defines which IFD(Image file directory) to search in for the EXIF tags. Default is all ifds i.e. 1:5.</p>
</li>
<li>
<p dir="auto"><code>read_all::Bool</code> : Defines if all EXIF tags are to be read or not. By default, <code>read_all</code> is true.</p>
</li>
<li>
<p dir="auto"><code>tags::Vector{LibExif.ExifTag}</code> : Defines which tags to search, in case <code>read_all</code> is false. When <code>read_all</code> is false, tags that need to be searched need to defined manually. Tags can be provided using bunch of methods but its suggested to supply a vector of strings with each string representing a EXIF tag i.e. ["<code>EXIF_TAG_FLASH_PIX_VERSION</code>", "<code>EXIF_TAG_ORIENTATION</code>"]</p>
</li>
<li>
<p dir="auto"><code>extract_thumbnail::Bool</code> : Defines whether to read the thumbnail data or not. By default, <code>extract_thumbnail</code> is false.</p>
</li>
<li>
<p dir="auto"><code>read_mnote::Bool</code> : Defines whether to read the mnote(MakerNote) tags data or not. By default, <code>read_mnote</code> is false.</p>
</li>
</ul>
<p dir="auto">List of all available tags to search is available here: <a href="https://libexif.github.io/internals/exif-tag_8h.html" rel="nofollow">https://libexif.github.io/internals/exif-tag_8h.html</a></p>

<ul dir="auto">
<li><code>filepath::AbstractString</code> : Name of the file to which image and exif is written.</li>
<li><code>img::AbstractArray</code> : Image Array whose exif data is being written to the filepath mentioned above.</li>

</ul>

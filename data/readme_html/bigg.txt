<div id="readme" class="md" data-path="README.md"><article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-biggjl" class="anchor" aria-hidden="true" href="#biggjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>BigG.jl</h1>
<p>A minimalistic Static Site Generator in Julia
Inspired by <a href="https://blog.thea.codes/a-small-static-site-generator/" rel="nofollow">this post</a>.</p>
<p>Demo can be found <a href="https://github.com/Moelf/BigG_demo">here</a></p>
<p>Markdown rendering is handled by Julia <code>stdlib</code>, docs <a href="https://pkg.julialang.org/docs/julia/THl1k/1.1.1/stdlib/Markdown.html" rel="nofollow">here</a></p>

<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p>The package is not yet registered in <code>METADATA.jl</code> and can be installed with <code>Pkg.add</code>, or in <code>REPL</code> by pressing <code>]</code>:</p>
<div class="highlight highlight-source-julia"><pre>pkg<span class="pl-k">&gt;</span> add https<span class="pl-k">:</span><span class="pl-k">//</span>github<span class="pl-k">.</span>com<span class="pl-k">/</span>Moelf<span class="pl-k">/</span>BigG<span class="pl-k">.</span>jl</pre></div>
<h2><a id="user-content-basic-usage" class="anchor" aria-hidden="true" href="#basic-usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Basic Usage</h2>
<p>Make two folders, <code>source</code> and <code>output</code>, put all <code>.md</code> files in <code>source</code> then:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> BigG

<span class="pl-c1">render_posts</span>(<span class="pl-s"><span class="pl-pds">"</span>./source<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>./output<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>BigG.jl/src/post.hbs<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>BigG.jl/src/head_injection.html<span class="pl-pds">"</span></span>)
<span class="pl-c1">render_index</span>(<span class="pl-s"><span class="pl-pds">"</span>./source<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>./output<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>BigG.jl/src/index.hbs<span class="pl-pds">"</span></span>)</pre></div>
<p><code>"BigG.jl/src/post.hbs"</code> is a provided template, you can change templates as you want, at the moment only <code>{{{content}}}</code>
in posts are used. Index render uses <code>{{#:posts}}</code> <code>{{/:posts}}</code> for loop and in between are <code>{{bname}}</code> and <code>{{title}}</code> for link
generation.</p>
<p>This will generate an <code>index.html</code> where each <code>.md</code> will be rendered into a html and
linked in the index page.</p>

<p dir="auto">Note: There are two different RxNorm databases.  The more complete one, RxNorm, contains
medications including veterinary-use-only medications and medications no longer in use or
which are not available in United States pharmacies. The Julia functions default to this
database. In order to confine search results to generally available human medications,
the RxNorm database also supports the "Prescribable" RxNorm API, which gives results only
within medications currently available for medical prescription in the US. If you want the
"Prescribable" database used for your RxNorm API calls, you should first call the function
<br><br>
prescribable(true)
<br><br>
after which all calls to the RxNorm API will use the somewhat smaller Prescribable database.
To set this back to using the more general database, call <code>prescribable(false)</code>.
<br><br></p>
<p dir="auto">Some of the API functions take optional arguments. For details of the values for such arguments
you should consult the NLM documentation (links are below). If the function takes an optional argument
called <code>extra</code>, this means that the function's optional argument <code>extra</code> should be provided as a <code>Dict</code>
or as a <code>Vector</code> of <code>Pairs</code>, with the keys to the Dict being the label for the optional term and the
values for that key as either a string or a vector of strings to be assigned to that value in the
final URL request. For example, <code>extra = Dict("sources" =&gt; ["ACTIVE", "OBSOLETE"], "toReturn" =&gt; 25)</code>
would be translated to <code>"&amp;sources=ACTIVE+OBSOLETE&amp;toReturn=25"</code> in the REST call request string sent by HTTP.</p>
<p dir="auto">The list of API functions is extensive. The API function names are not exported from RxNav, so to call,
for example, <code>getSpellingSuggestions("asprin")</code> you must call this as <code>RxNav.getSpellingSuggestions("asprin")</code>.</p>

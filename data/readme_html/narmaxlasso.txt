<p>This package provides a method for computing the parameters of NARMAX
(<em>Nonlinear autoregressive moving average with exogenous inputs</em>)
models subject to L1 penalty using pathwise coordinate optimization algorithm.</p>

<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview</h2>
<p>This package allows the estimation of parameters of discrete dynamic models from observed data.</p>
<p>For instance, consider vectors of observed inputs and
outputs: <code>u</code> and <code>y</code>. Assume, for instance, we want
to fit the following model to the observed data:</p>
<pre><code>y[k] = β[1]*y[k-1] + β[2]*u[k-1] + β[3]*v[k-1]
</code></pre>
<p>where the noise term <code>v[k-1]</code> is included to model the random effects that
affects the observation. The estimation can be done by the following command
sequence:</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> NarmaxLasso
julia<span class="pl-k">&gt;</span> ny <span class="pl-k">=</span> <span class="pl-c1">1</span>; nu <span class="pl-k">=</span> <span class="pl-c1">1</span>; nv <span class="pl-k">=</span> <span class="pl-c1">1</span>; order <span class="pl-k">=</span> <span class="pl-c1">1</span>;
julia<span class="pl-k">&gt;</span> mdl <span class="pl-k">=</span> <span class="pl-c1">generate_all</span>(NarmaxRegressors, Monomial, ny, nu, nv, order)
    u[k<span class="pl-k">-</span><span class="pl-c1">1</span>]
    y[k<span class="pl-k">-</span><span class="pl-c1">1</span>]
    v[k<span class="pl-k">-</span><span class="pl-c1">1</span>]
julia<span class="pl-k">&gt;</span> result <span class="pl-k">=</span> <span class="pl-c1">narmax_lasso</span>(y, u, mdl);</pre></div>
<p>Be <code>e</code> the error between the model prediction and the observed values,
the result of the above command sequence provides the solution of
the following minimization problem:</p>
<pre><code>min_β ||e||^2 + λ * ∑ |β[i]|
</code></pre>
<p>for a grid of values of λ.</p>
<p>The output can be visualized using:</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Plots
julia<span class="pl-k">&gt;</span> <span class="pl-c1">plot</span>(result)</pre></div>
<p>A possible output would be:
<a target="_blank" rel="noopener noreferrer" href="example.png"><img src="example.png" alt="example.png" style="max-width:100%;"></a></p>
<p>Usually, the value of λ is chosen by testing on a validation set.</p>
<p>So far the above estimation procedure is implemented
only for linear and polynomial models. More options should
be included latter.</p>

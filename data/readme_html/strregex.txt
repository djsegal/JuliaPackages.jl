<p>The <code>StrRegex</code> package adds Regex support to the <code>Strs</code> package, as well as fix some issues present in the base Regex support.</p>
<ul>
<li>Thread-safe support</li>
<li>Allows the whole range of compile and match options</li>
<li>Supports both UTF and non-UTF strings</li>
<li>Supports strings with 8, 16, and 32-bit codeunit sizes</li>
<li>Correctly sets the NO_CHECK_UTF flag based on the string type</li>
</ul>
<p>It is working on both the release version (v0.6.2) and the latest master (v0.7.0-DEV).</p>
<p>This uses a <code>R"..."</code> macro, or <code>RegexStr</code> constructor, instead of <code>r"..."</code> and <code>Regex</code> as in Base.</p>
<p>Some changes might be needed in Base to make it work with the <code>r"..."</code> regex string macro and <code>Regex</code> type, because there are some fields missing that would be needed to handle arbitrary abstract string types).</p>

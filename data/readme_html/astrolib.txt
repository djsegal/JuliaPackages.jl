<h2><a id="user-content-goal" class="anchor" aria-hidden="true" href="#goal"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Goal</h2>
<p>The aim of this project is to provide users with a set of small generic routines
useful above all in astronomical and astrophysical context, written in
<a href="http://julialang.org/" rel="nofollow">Julia</a>.</p>
<p>Included are also translations of some
<a href="http://idlastro.gsfc.nasa.gov/homepage.html" rel="nofollow">IDL Astronomy User's Library</a>
procedures, which are released under terms of
<a href="http://idlastro.gsfc.nasa.gov/idlfaq.html#A14" rel="nofollow">BSD-2-Clause License</a>.
AstroLib.jl's functions are not drop-in replacement of those procedures, Julia
standard data types are often used (e.g., <code>DateTime</code> type instead of generic
string for dates) and the syntax may slightly differ.  Refer to the
documentation of the functions for details.</p>
<p><strong>Note:</strong> this project is a work-in-progress, only few procedures have been
translated so far.  In addition, function syntax may change from time to time.
Check <a href="https://github.com/JuliaAstro/AstroLib.jl/blob/master/TODO.md">TODO.md</a> out
to see how you can help.  Volunteers are welcome!</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<p><code>AstroLib.jl</code> is available for Julia 1.0 and later versions, and can be
installed with <a href="https://julialang.github.io/Pkg.jl/stable/" rel="nofollow">Julia built-in package
manager</a>.  In a Julia session, after
entering the package manager mode with <code>]</code>, run the command</p>
<div class="highlight highlight-source-julia"><pre>pkg<span class="pl-k">&gt;</span> add AstroLib</pre></div>
<p>Older versions of this package are also available for Julia 0.4-0.7.</p>
<p>Note that, in order to work, a few functions require external files, which are
automatically downloaded when building the package.  Should these files be
missing for some reason, you will be able to load the package but some functions
may not work properly.  You can manually build the package with</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> Pkg<span class="pl-k">.</span><span class="pl-c1">build</span>(<span class="pl-s"><span class="pl-pds">"</span>AstroLib<span class="pl-pds">"</span></span>)</pre></div>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<p>After installing the package, you can start using it with</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> AstroLib</pre></div>
<p>Many functions in <code>AstroLib.jl</code> are compatible with
<a href="https://github.com/giordano/Measurements.jl"><code>Measurements.jl</code></a> package, which
allows you to define quantities with uncertainty and propagate the error when
performing calculations according to
<a href="https://en.wikipedia.org/wiki/Propagation_of_uncertainty" rel="nofollow">propagation of uncertainty rules</a>.
For example:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> AstroLib, Measurements
<span class="pl-c1">mag2flux</span>(<span class="pl-c1">12.54</span> ± <span class="pl-c1">0.03</span>)
<span class="pl-c"><span class="pl-c">#</span> =&gt; 3.499451670283562e-14 ± 9.669342299577655e-16</span></pre></div>

<h3><a id="user-content-observatory" class="anchor" aria-hidden="true" href="#observatory"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Observatory</h3>
<p><code>AstroLib.jl</code> defines a new <code>Observatory</code> type.  This can be used to define a
new object holding information about an observing site.  It is a
<a href="http://docs.julialang.org/en/stable/manual/types/#composite-types" rel="nofollow">composite type</a>
whose fields are</p>
<ul>
<li><code>name</code> (<code>String</code> type): the name of the site</li>
<li><code>latitude</code> (<code>Float64</code> type): North-ward latitude of the site in degrees</li>
<li><code>longitude</code> (<code>Float64</code> type): East-ward longitude of the site in degrees</li>
<li><code>altitude</code> (<code>Float64</code> type): altitude of the site in meters</li>
<li><code>tz</code> (<code>Float64</code> type): the number of hours of offset from UTC</li>
</ul>
<p>The type constructor <code>Observatory</code> can be used to create a new <code>Observatory</code>
object.  Its syntax is</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">Observatory</span>(name, lat, long, alt, tz)</pre></div>
<p><code>name</code> should be a string; <code>lat</code>, <code>long</code>, and <code>tz</code> should be anything that can
be converted to a floating number with <code>ten</code> function; <code>alt</code> should be a real
number.</p>
<p>A predefined list of some observing sites is provided with
<code>AstroLib.observatories</code> constant.  It is a dictionary whose keys are the
abbreviated names of the observatories.  For example, you can access information
of the European Southern Observatory with</p>
<div class="highlight highlight-source-julia"><pre>julia<span class="pl-k">&gt;</span> obs <span class="pl-k">=</span> AstroLib<span class="pl-k">.</span>observatories[<span class="pl-s"><span class="pl-pds">"</span>eso<span class="pl-pds">"</span></span>]

longitude<span class="pl-k">:</span>   <span class="pl-k">-</span><span class="pl-c1">70.73</span>°E
altitude<span class="pl-k">:</span>    <span class="pl-c1">2347.0</span> m

julia<span class="pl-k">&gt;</span> obs<span class="pl-k">.</span>longitude
<span class="pl-k">-</span><span class="pl-c1">70.73</span></pre></div>
<p>You can list all keys of the dictionary with</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">keys</span>(AstroLib<span class="pl-k">.</span>observatories)</pre></div>
<p>Feel free to contribute new sites or adjust information of already present ones.</p>
<h3><a id="user-content-planet" class="anchor" aria-hidden="true" href="#planet"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Planet</h3>
<p>The package provides <code>Planet</code> type to hold information about Solar System
planets.  Its fields are</p>
<ul>
<li>
<p>Designation:</p>
<ul>
<li><code>name</code>: the name</li>
</ul>
</li>
<li>
<p>Physical characteristics:</p>
<ul>
<li><code>radius</code>: mean radius in meters</li>
<li><code>eqradius</code>: equatorial radius in meters</li>
<li><code>polradius</code>: polar radius in meters</li>
<li><code>mass</code>: mass in kilogram</li>
</ul>
</li>
<li>
<p>Orbital characteristics (epoch J2000):</p>
<ul>
<li><code>ecc</code>: eccentricity of the orbit</li>
<li><code>axis</code>: semi-major axis of the orbit in meters</li>
<li><code>period</code>: sidereal orbital period in seconds</li>
</ul>
</li>
</ul>
<p>The constructor has this syntax:</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c1">Planet</span>(name, radius, eqradius, polradius, mass, ecc, axis, period)</pre></div>
<p>The list of Solar System planets, from Mercury to Pluto, is available with
<code>AstroLib.planets</code> dictionary.  The keys of this dictionary are the lowercase
names of the planets.  For example:</p>

<a href="http://juliaastro.github.io/AstroLib.jl/latest/" rel="nofollow">http://juliaastro.github.io/AstroLib.jl/latest/</a>.  There you can find the

<p>You can contribute to the project in number of ways.  You can translate more

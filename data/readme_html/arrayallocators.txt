<p dir="auto">ArrayAllocators.jl is a Julia language package that provides various methods to allocate memory for arrays. It also implements various ways of calculating the total number of bytes in order to detect integer overflow conditions when multiplying the dimensions of an array and the size of the array elements.</p>
<p dir="auto">For example, <a href="https://en.cppreference.com/w/c/memory/calloc" rel="nofollow"><code>calloc</code></a> is a standard C function that allocates memory while initializing the bytes in the memory to <code>0</code>, which may be done lazily by some operating sytems as needed. Contrast this with Julia's <code>Base.zeros</code> which eagerly fills sets all the bytes in memory to <code>0</code>. This often means that allocating memory via <code>calloc</code> may be initially faster than using <code>Base.zeros</code>.</p>
<p dir="auto">The Python package NumPy for example, implements <a href="https://numpy.org/doc/stable/reference/generated/numpy.zeros.html" rel="nofollow"><code>numpy.zeros</code></a> with <code>calloc</code>.
At times, <code>numpy.zeros</code> and Python code using this method may seem to outperform Julia code using <code>Base.zeros</code>. See the Discourse link under
the discussion below for further details.</p>
<p dir="auto">Other examples of specialized array allocation techniques include aligned memory on POSIX systems or virtual allocations on Windows systems.</p>
<p dir="auto">Multiple processor socket systems may also implement Non-Uniform Memory Access (NUMA) memory architecture. To optimally use the NUMA architecture, memory must be explicitly allocated on a specific NUMA node. The subpackage <a href="NumaAllocators">NumaAllocators.jl</a> implements this functionality for Windows and Linux operating systems.</p>
<p dir="auto"><code>AbstractArrayAllocator</code> can be provided as first argument when constructing any subtype of <code>AbstractArray</code> where <code>undef</code> is usually provided.</p>
<p dir="auto">Any C function that returns a pointer can be wrapped by the <code>AbstractArrayAllocator</code> interface by implementing the <code>allocate</code> method and overriding
<code>Base.unsafe_wrap</code>.</p>

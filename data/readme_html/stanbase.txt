<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Introduction</h2>
<p>This package contains all the common components for several application packages wrapping Stan's <code>cmdstan</code> executable.</p>
<p>Stan <code>methods</code> supported are in StanSample, StanOptimize, StanVariational, StanDiagnose and <code>generated_quantities</code> (included in StanSample.jl).</p>
<p><code>StanBase.jl</code>, <code>HelpModel.jl</code>, <code>help_types.jl</code> and <code>cmdline.jl</code> are the templates that have been used in the cmdstan application packages to support cmdstan options such as <code>stan_sample</code>, <code>stan_variational</code>, etc.</p>
<p>Note that <code>help_types.jl</code> is a bit degenerated, a more complete version can be found <a href="https://github.com/StanJulia/StanSample.jl/blob/master/src/stanrun/cmdline.jl">here</a>.</p>
<p>These five application packages support all features and options currently available in the <code>cmdstan</code> exacutable.</p>
<p>The intention is that users of Stan's <code>cmdstan</code> executable will never have to install Stanbase.jl, they can simply install any of the application packages listed <a href="https://github.com/StanJulia">here</a>.</p>
<p>The currenly existing package <code>CmdStan.jl v5.x.x</code> will continue to exists. A new version of <code>Stan.jl, v6</code> using this newer setup is in the pipeline.</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<p>This package is registered yet. Install with</p>
<pre><code>pkg&gt; add StanBase
</code></pre>
<p>This package is loaded automatically when <code>using ...</code> any of the application packages.</p>
<p>This package is derived from Tamas Papp's <a href="https://github.com/tpapp/StanRun.jl">StanRun.jl</a> package. It also uses StanDump.jl and StanSamples.jl.</p>
<p>This and all the application packages need a working <a href="https://mc-stan.org/users/interfaces/cmdstan.html" rel="nofollow">cmdstan</a> installation, the path of which you should specify in <code>JULIA_CMDSTAN_HOME</code>, eg in your <code>~/.julia/config/startup.jl</code> have a line like</p>
<div class="highlight highlight-source-julia"><pre><span class="pl-c"><span class="pl-c">#</span> CmdStan setup</span>

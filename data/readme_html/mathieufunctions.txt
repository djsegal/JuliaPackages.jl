<p>Julia package for Mathieu functions.</p>
<p>Mathieu functions are the eigenvalues and eigenfunction of <em>Mathieu's
equation</em> (for more details, see <a href="http://dlmf.nist.gov/28" rel="nofollow">NIST's Digital Library of
Mathematical Functions</a>).</p>
<h2><a id="user-content-implementation" class="anchor" aria-hidden="true" href="#implementation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Implementation</h2>
<p>This implementation is based on
<a href="http://dl.acm.org/citation.cfm?id=155796" rel="nofollow">an algorithm described by Shirts</a>, which
revolves around an application of Floquet's theorem.</p>
<p>Interestingly, an
equivalent description can be obtained by considering the eigenstates
and eigenenergies of a particular type of artificial atom based on
superconducting circuits
(see <a href="http://www.phys.ens.fr/~cottet/ACottetThesis.pdf" rel="nofollow">Cotet's Ph.D. thesis</a>
for a discussion of this connection, or
<a href="https://arxiv.org/abs/cond-mat/0703002" rel="nofollow">a recent paper by Koch et al.</a>).
In this physical picture, Mathieu
functions can be approximated by computing the eigenvectors and
eigenvalues of a Hamiltonian truncated to a sufficiently high
dimension (this Hamiltonian corresponds to a symmetric tri-diagonal
matrix, so this approximation is rather convenient for numerics).</p>
<p>The current tests consist of checks against a table generated by
Mathematica (which, to our knowledge, has the most complete
implementation of Mathieu functions amongst commonly accessible
mathematical software packages).</p>

</code></pre>

<h2><a id="user-content-application-eigenstates-and-energies-for-transmon-and-quantronium-qubits" class="anchor" aria-hidden="true" href="#application-eigenstates-and-energies-for-transmon-and-quantronium-qubits"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Application: Eigenstates and energies for transmon and quantronium qubits</h2>
<p>Thanks to the way that the Mathieu characteristic values are computed, the
order indexing convention used here is particularly convenient for computing
the energy spectrum of a broad class of superconducting qubits including the
<a href="http://www.phys.ens.fr/~cottet/ACottetThesis.pdf" rel="nofollow">quantronium</a> and the
<a href="https://arxiv.org/abs/cond-mat/0703002" rel="nofollow">transmon</a>.</p>
<p>As an example of how this package may be used, here is the Julia code
that reproduces the energy spectrum illustrated in <a href="https://arxiv.org/abs/cond-mat/0703002" rel="nofollow">Koch et al.</a></p>
<div class="highlight highlight-source-julia"><pre><span class="pl-k">using</span> MathieuFunctions, PyPlot

EJoverEC <span class="pl-k">=</span> <span class="pl-c1">30</span>;
EC       <span class="pl-k">=</span> <span class="pl-c1">0.35</span>; <span class="pl-c"><span class="pl-c">#</span> GHz</span>
nLevels  <span class="pl-k">=</span> <span class="pl-c1">10</span>;
nCharge  <span class="pl-k">=</span> <span class="pl-c1">2</span>; 
nData    <span class="pl-k">=</span> <span class="pl-c1">1001</span>;

Nstart <span class="pl-k">=</span> <span class="pl-c1">5</span>;

q <span class="pl-k">=</span> <span class="pl-k">-</span>(<span class="pl-c1">1</span><span class="pl-k">/</span><span class="pl-c1">2</span>)<span class="pl-k">*</span>EJoverEC;
tol <span class="pl-k">=</span> <span class="pl-c1">1e-12</span>;

ng <span class="pl-k">=</span> <span class="pl-c1">LinRange</span>(<span class="pl-k">-</span>nCharge,nCharge,nData);
nu <span class="pl-k">=</span> <span class="pl-k">-</span><span class="pl-c1">2</span><span class="pl-k">*</span>ng;

a <span class="pl-k">=</span> <span class="pl-c1">zeros</span>(nData,nLevels);
N <span class="pl-k">=</span> <span class="pl-c1">zeros</span>(nData,<span class="pl-c1">1</span>);
Err <span class="pl-k">=</span> <span class="pl-c1">zeros</span>(nData,<span class="pl-c1">1</span>);
<span class="pl-k">for</span> i<span class="pl-k">=</span><span class="pl-c1">1</span><span class="pl-k">:</span>nData
    <span class="pl-c"><span class="pl-c">#</span>println(nu[i])</span>
    a[i,:] <span class="pl-k">=</span> charλ(q,nu[i],k<span class="pl-k">=</span><span class="pl-c1">1</span><span class="pl-k">:</span>nLevels);
<span class="pl-k">end</span>

E <span class="pl-k">=</span> EC<span class="pl-k">*</span>a;
Eng_half <span class="pl-k">=</span> charλ(q,<span class="pl-k">-</span><span class="pl-c1">1</span>,k<span class="pl-k">=</span><span class="pl-c1">1</span><span class="pl-k">:</span>nLevels);
Eng_0    <span class="pl-k">=</span> charλ(q, <span class="pl-c1">0</span>,k<span class="pl-k">=</span><span class="pl-c1">1</span><span class="pl-k">:</span>nLevels);
E01      <span class="pl-k">=</span> Eng_half[<span class="pl-c1">2</span>] <span class="pl-k">-</span> Eng_half[<span class="pl-c1">1</span>];

<span class="pl-c1">plot</span>(ng,(E <span class="pl-k">.-</span> Eng_0[<span class="pl-c1">1</span>])<span class="pl-k">/</span>E01);<span class="pl-c1">xlabel</span>(<span class="pl-s"><span class="pl-pds"><span class="pl-c1">L</span>"</span>$n_g$<span class="pl-pds">"</span></span>);<span class="pl-c1">ylabel</span>(<span class="pl-s"><span class="pl-pds"><span class="pl-c1">L</span>"</span>$E$<span class="pl-pds">"</span></span>);
<span class="pl-c1">ylim</span>(<span class="pl-k">-</span><span class="pl-c1">.5</span>,<span class="pl-c1">5</span>)</pre></div>

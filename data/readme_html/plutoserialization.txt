<p dir="auto">This package provides <code>serialize</code> and <code>deserialize</code> functions which are
compatible with the standard library <code>Serialization</code> module's function, but are
able to handle serializing/deserializing data from/to <code>Pluto</code> notebooks.</p>
<h2 dir="auto"><a id="user-content-details" class="anchor" aria-hidden="true" href="#details"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Details</h2>
<p dir="auto">Standard <code>Serialization</code> does not work inside <code>Pluto</code> notebooks since
evaluation of cells will increment the current "workspace" <code>Module</code> into which
definitions are evaluated. This is done to allow for re-definition of constants
such as type definitions, but means that <code>Serialization.serialize</code> will save
<code>Module</code> references that may not exist when we <code>deserialize</code> the file in a new
notebook session since the workspace number will not likely be the same.</p>
<p dir="auto">This package implements a custom <code>Serialization.AbstractSerializer</code> that
searches all defined workspaces for valid definitions during <code>deserialize</code>
allowing it to not have to rely on fixed <code>Module</code> references. Aside from the
changes required for searching workspaces it should replicate the behaviour of
<code>Serialization</code>.</p>

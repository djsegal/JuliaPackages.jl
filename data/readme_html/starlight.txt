<p align="center" dir="auto">
  <a target="_blank" rel="noopener noreferrer" href="https://github.com/jhigginbotham64/Starlight.jl/blob/main/docs/src/assets/logo.png"><img src="https://github.com/jhigginbotham64/Starlight.jl/raw/main/docs/src/assets/logo.png" alt="Starlight.jl logo" style="max-width: 100%;"></a>
</p>
<p dir="auto">Julia is a <a href="https://julialang.org/blog/2012/02/why-we-created-julia/" rel="nofollow">greedy language</a>, surrounded by a community of greedy people, who deserve greedy frameworks to make their greedy applications.</p>
<p dir="auto">With a focus on flexibility and code quality, Starlight aims to be such a framework. It includes a suite of components and integrations that make it particuarly well-suited for video games, so it is not a stretch to call it a "game engine". However, Starlight is most fundamentally a scripting layer for <a href="http://www.libsdl.org/" rel="nofollow">SDL</a>, <a href="https://www.vulkan.org/" rel="nofollow">Vulkan</a>, and <a href="https://pybullet.org/wordpress/" rel="nofollow">Bullet</a> (via the <a href="https://github.com/jhigginbotham64/Telescope">Telescope</a> backend), meaning it can be used for any application that needs high-performance rendering and physics. Furthermore, there are plans to allow selective enabling of different subsystems, meaning it could be used for GUI apps or pure physics simulation or anything else you can imagine.</p>

<h3 dir="auto"><a id="user-content-basic-usage" class="anchor" aria-hidden="true" href="#basic-usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Basic Usage</h3>
<p dir="auto">Starlight projects are simply Julia projects, no special structure or anything, so you simply declare that you are</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using Starlight"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> Starlight</pre></div>
<p dir="auto">To take advantage of the magic of Starlight you must first create an App:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; a = App()"><pre>julia<span class="pl-k">&gt;</span> a <span class="pl-k">=</span> <span class="pl-c1">App</span>()</pre></div>
<p dir="auto">This gets you an internal clock, message bus, entity component system, rendering, physics, input, and sound, although it doesn't do anything with them yet. To open a window and start running the initialized subsystems, you can call</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; awake!(a)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">awake!</span>(a)</pre></div>
<p dir="auto">To close everything down, call</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; shutdown!(a)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">shutdown!</span>(a)</pre></div>
<p dir="auto">If running in a script you will need to keep the Julia process alive, so instead of <code>awake!(a)</code> you can use</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; run!(a)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">run!</span>(a)</pre></div>
<p dir="auto">...which, as shown, works in the REPL as well.</p>
<p dir="auto">Before you try doing anything interesting with the library, it is recommended that you read the docs.</p>

<p dir="auto">We welcome issues, pull requests, everything. If there's a feature or use case we don't support and you don't have time to implement it yourself, create an issue. If you do have time, create a pull request. There's a ton of work to be done and only one active contributor. Anything you have to offer will be appreciated. Authors of pull requests will also get a special mention and a description of the work they did further down here in the README.</p>
<p dir="auto">The maintainer is willing to personally mentor anyone who wants to either contribute or to learn Starlight for their own use cases. Contact information below. Please reach out.</p>

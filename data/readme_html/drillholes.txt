<p dir="auto">Given a <em>collar table</em>, a <em>survey table</em> and at least one <em>interval
table</em> (such as assay and lithology), the function <code>desurvey</code> can
be used for desurveying and compositing. Examples of these tables
are shown bellow:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="docs/example.png"><img src="docs/example.png" alt="tables" style="max-width: 100%;"></a></p>
<ul dir="auto">
<li><em>Collar table</em>: stores the coordinates (X, Y, Z) of each drill
hole with given ID (HOLEID).</li>
<li><em>Survey table</em>: stores the arc length (AT) and azimuth (AZM) and
dip (DIP) angles along the drill hole trajectory. Together with the
collar table it fully specifies the trajectory.</li>
<li><em>Interval table</em>: stores the actual measurements taken on cylinders
of rock defined by an interval of arc lenghts (FROM and TO). Usually,
there are multiple interval tables with different types of measurements.</li>
</ul>
<p dir="auto">Assuming that each of these tables was loaded into a
<a href="https://github.com/JuliaData/Tables.jl">Tables.jl</a> table
(e.g. CSV.File, DataFrame), we can use the following constructors
to automatically detect the columns:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using DrillHoles

collar = Collar(CSV.File(&quot;collar.csv&quot;))

assay  = Interval(CSV.File(&quot;assay.csv&quot;))
litho  = Interval(CSV.File(&quot;litho.csv&quot;))"><pre><span class="pl-k">using</span> DrillHoles
<span class="pl-k">using</span> CSV

<p dir="auto">If the columns of the tables follow an exotic naming convention,
users can manually specify the names with keyword arguments:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="# manually specify column with hole ID
Collar(CSV.File(&quot;collar.csv&quot;), holeid = :MYID)"><pre><span class="pl-c"><span class="pl-c">#</span> manually specify column with hole ID</span>
<span class="pl-c1">Collar</span>(CSV<span class="pl-k">.</span><span class="pl-c1">File</span>(<span class="pl-s"><span class="pl-pds">"</span>collar.csv<span class="pl-pds">"</span></span>), holeid <span class="pl-k">=</span> <span class="pl-c1">:MYID</span>)</pre></div>
<p dir="auto">Please check the documentation of <code>Collar</code>, <code>Survey</code> and <code>Interval</code>
for more details.</p>
<p dir="auto">By default, the <code>desurvey</code> function returns a
<a href="https://juliaearth.github.io/GeoStats.jl/stable/data.html" rel="nofollow">geospatial table</a>
compatible with <a href="https://github.com/JuliaEarth/GeoStats.jl">GeoStats.jl</a>.
It supports different dip angle conventions from open pit and underground
mining as well as different stepping methods:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="samples = desurvey(collar, survey, [assay, litho])"><pre>samples <span class="pl-k">=</span> <span class="pl-c1">desurvey</span>(collar, survey, [assay, litho])</pre></div>
<p dir="auto">The option <code>geom</code> can be used to control the output format, and the option
<code>len</code> can be used for compositing. Please check the documentation for more
details.</p>

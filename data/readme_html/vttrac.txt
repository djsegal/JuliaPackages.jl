<h2 dir="auto"><a id="user-content-vttrac-velocimetry-by-template-tracking" class="anchor" aria-hidden="true" href="#vttrac-velocimetry-by-template-tracking"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>VTTrac: Velocimetry by Template Tracking</h2>
<p dir="auto">This library provides the julia-language implementation for <code>VTTrac</code> (<a href="https://github.com/thorinouchi/VTTrac">https://github.com/thorinouchi/VTTrac</a>). It does not use module variables, so it should be good for parallel execution.</p>
<p dir="auto">The algorithm used in this library is the simple template matching of PIV (particle image velocimetry) for monochromatic image-like data, but the matching is conducted multiple times in a Lagrangian manner as in PTV (particle tracking velocimetry) over a number of times specified by the parameter named <code>ntrac</code>. The default scoring method for template matching is the cross correlation coefficient, as in the basic PIV. Both forward and backward tracking is available. Use the parameter <code>itstep</code>; tracking is backward along time sequence, if it is negative.</p>
<h3 dir="auto"><a id="user-content-available-scoring-methods" class="anchor" aria-hidden="true" href="#available-scoring-methods"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Available scoring methods</h3>
<ul dir="auto">
<li>XCOR: cross-correlation, <code>cov(x',y')/sig(x)/sig(y)</code>, where <code>x</code> represents the template sub-image and <code>y</code> represents the target sub-image that are slided.</li>
<li>NCOV: normalized covariance, <code>cov(x',y')/sig(x)^2</code>: covariance normalized the variance of the template sub-image <code>x</code>.</li>
</ul>

<p dir="auto">A check (result screening) based on velocity change along trajectory is available (by using the threshold parameter named <code>vxch</code> and <code>vych</code>), so it is recommended to always set <code>ntrac &gt;= 2</code>. Further screening is available for initial templates (e.g., in terms of the complexity and contrast) and the quality of the results (e.g., score threshold); see the source code.</p>

<p dir="auto">Spatial coordinates are based on array indices, with the distance between adjacent grid points always being 1, so they are non-dimensional. The velocities are based on non-dimensional spacial displacement over time difference, where, time can either be dimensional or non-dimensional.</p>

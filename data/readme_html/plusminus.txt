<p>Julia core maintainers suggested in this <a href="https://github.com/JuliaLang/julia/issues/14498">2015 original issue</a> that a plusminus function was not worthy of inclusion in Base Julia, much to the disappointment of statisticians everywhere.</p>
<p>Fear not!</p>
<p>This package implements the <a href="https://en.wiktionary.org/wiki/plus-minus_sign" rel="nofollow">plus-minus</a> function (and that's it!).</p>
<div class="highlight highlight-source-julia position-relative" data-snippet-clipboard-copy-content="plusminus(x::Number,y::Number) = (x-y,x+y)

±(x,y) = plusminus(x,y)     # thats \pm followed by TAB

2 ± 1       # returns (1,3)
"><pre><span class="pl-en">plusminus</span>(x<span class="pl-k">::</span><span class="pl-c1">Number</span>,y<span class="pl-k">::</span><span class="pl-c1">Number</span>) <span class="pl-k">=</span> (x<span class="pl-k">-</span>y,x<span class="pl-k">+</span>y)

±(x,y) <span class="pl-k">=</span> <span class="pl-c1">plusminus</span>(x,y)     <span class="pl-c"><span class="pl-c">#</span> thats \pm followed by TAB</span>

<span class="pl-c1">2</span> ± <span class="pl-c1">1</span>       <span class="pl-c"><span class="pl-c">#</span> returns (1,3)</span></pre></div>
<p>The convention I use here is for the <code>x-y</code> to return first and <code>x+y</code> second. Because the output is a tuple, you can apply <code>reverse</code> to it to get the items in the other order. Alternatively, use direct assignment: <code>minusitem,plusitem = 2 ± 1</code>.</p>
<p>The package is registered, so can be added with <code>]add PlusMinus</code> and available for use after <code>using PlusMinus</code>.</p>
<p>Broadcasting is automatic because Julia rocks.</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="(±).(1:3,1) == [(i-1,i+1) for i in 1:3]
"><pre><code>(±).(1:3,1) == [(i-1,i+1) for i in 1:3]
</code></pre></div>
<p>Flawlessly works with rationals as well.</p>

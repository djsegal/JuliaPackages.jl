<p dir="auto">During the <strong>combinatorial engineering of biosystems</strong>, such as proteins, genetic circuits and genomes, diverse libraries are generated by <strong>assembling and recombining modules</strong>. The variants with the optimal phenotypes are selected with screening techniques. However, when the number of available modules to compose biological designs increases, a combinatorial explosion of design possibilities arises, allowing only for a part of the libary to be analyzed. In this case, it is important for a researcher to get insight in which (minimum) sample size sufficiently covers the design space, <em>i.e.</em> what is the expected <strong>minimum number of designs so that all modules are observed at least once</strong>.</p>

<p dir="auto">BioCCP contains functions for calculating (expected) minimum sample sizes and related statistics:</p>
<table>
<thead>
<tr>
<th>Function name</th>
<th>Short description</th>
</tr>
</thead>
<tbody>
<tr>
<td><code>expectation_minsamplesize</code></td>
<td>Calculates the expected minimum number of designs to observe all modules at least <em>m</em> times</td>
</tr>
<tr>
<td><code>std_minsamplesize</code></td>
<td>Calculates standard deviation on the minimum number of designs</td>
</tr>
<tr>
<td><code>success_probability</code></td>
<td>Calculates the probability that the minimum number of designs <em>T</em> is smaller than or equal to a given sample size <em>t</em></td>
</tr>
<tr>
<td><code>expectation_fraction_collected</code></td>
<td>Returns the fraction of the total number of modules in the design space that is expected to be observed for a given sample size <em>t</em></td>
</tr>
<tr>
<td><code>prob_occurrence_module</code></td>
<td>Calculates for a module with specified module probability <em>p</em>, the probability that this module occurs <em>k</em> times when a given number of designs has been collected</td>
</tr>
</tbody>
</table>
<p dir="auto">For more info about the implementation of the functions, please consult the <a href="https://kirstvh.github.io/BioCCP.jl/" rel="nofollow">docs</a> or <a href="/src/BioCCP.jl">source code</a>.</p>

<p dir="auto">(*)
<em>When exact probabilities are known</em>, define your custom module probability/abundance vector or load them in the notebook from an external file.
<em>When probabilities and/or their distribution are unknown</em>, the user can either:</p>
<blockquote>
<ol dir="auto">
<li>Assume the probabilities of all modules to be equal (uniform distribution), or</li>
<li>Assume the module probabilities to follow <em>Zipf's law</em>, specifying the ratio p<sub>max</sub>/p<sub>min</sub>, or</li>
<li>Assume the histogram of the module probabilities to behave like a <em>bell curve</em>, specifying the ratio p<sub>max</sub>/p<sub>min</sub></li>
</ol>
</blockquote>

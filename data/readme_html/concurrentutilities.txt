<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="ConcurrentUtilities.init(nworkers=Threads.nthreads() - 1)"><pre class="notranslate"><code>ConcurrentUtilities.init(nworkers=Threads.nthreads() - 1)
</code></pre></div>
<p dir="auto">Initialize background workers that will execute tasks spawned via
<a href="@ref"><code>ConcurrentUtilities.@spawn</code></a>. If <code>nworkers == 1</code>, a single worker
will be started on thread 1 where tasks will be executed in contention
with other thread 1 work. Background worker tasks can be inspected by
looking at <code>ConcurrentUtilities.WORKER_TASKS</code>.</p>

<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="ConcurrentUtilities.@spawn expr
ConcurrentUtilities.@spawn passthroughstorage expr"><pre class="notranslate"><code>ConcurrentUtilities.@spawn expr
ConcurrentUtilities.@spawn passthroughstorage expr
</code></pre></div>
<p dir="auto">Similar to <code>Threads.@spawn</code>, schedule and execute a task (given by <code>expr</code>)
that will be run on a "background worker" (see <a href="(@ref)"><code>ConcurrentUtilities.init</code></a>).</p>
<p dir="auto">In the 2-argument invocation, <code>passthroughstorage</code> controls whether the task-local storage of the
<code>current_task()</code> should be "passed through" to the spawned task.</p>

<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="Lockable(value, lock = ReentrantLock())"><pre class="notranslate"><code>Lockable(value, lock = ReentrantLock())
</code></pre></div>
<p dir="auto">Creates a <code>Lockable</code> object that wraps <code>value</code> and
associates it with the provided <code>lock</code>.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="lock(f::Function, l::Lockable)"><pre class="notranslate"><code>lock(f::Function, l::Lockable)
</code></pre></div>
<p dir="auto">Acquire the lock associated with <code>l</code>, execute <code>f</code> with the lock held,
and release the lock when <code>f</code> returns. <code>f</code> will receive one positional
argument: the value wrapped by <code>l</code>. If the lock is already locked by a
different task/thread, wait for it to become available.
When this function returns, the <code>lock</code> has been released, so the caller should
not attempt to <code>unlock</code> it.</p>

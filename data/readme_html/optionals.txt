<p dir="auto">This Package provides the <code>Optional{T}</code> type, this type is a simple wrapper
around <code>Union{T, Missing}</code> at least semantically.</p>
<h2 dir="auto"><a id="user-content-why-should-i-use-this-package" class="anchor" aria-hidden="true" href="#why-should-i-use-this-package"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Why should I use this package?</h2>
<p dir="auto">In Julia 0.7 and above the <code>Union{T, Missing}</code> type makes it quite convenient to
use fields with optional values. The Missings package tries to backport this
feature to 0.6, unfortunately v0.6 is not very efficient in using Union fields.
In the interim <code>Optional</code> provides a somewhat more efficient way for packages
which need to support both v0.6 and newer. After dropping support for v0.6 and
older, <code>Optional{T}</code> can simply be replaced with <code>Union{T, Missing}</code> (hopefully)
without further changes.</p>

<p dir="auto">Convert an <code>Optional{T}</code> to a <code>Union{T, Missing}</code>: <code>coalesce(foo.x)</code></p>

<p dir="auto"><a href="https://github.com/simonp0420/PkgOnlineHelp.jl/actions/workflows/CI.yml?query=branch%3Amain"><img src="https://github.com/simonp0420/PkgOnlineHelp.jl/actions/workflows/CI.yml/badge.svg?branch=main" alt="Build Status" style="max-width: 100%;"></a></p>
<p dir="auto">A package to help quickly open (via a few keystrokes at the Julia REPL)
package documentation web sites or other user-selected sites.</p>

<p dir="auto">The main convenience macro is <code>@docs</code>.  An example of usage:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; using PkgOnlineHelp  # If not included in your startup.jl

julia&gt; @docs StaticArrays
&quot;https://github.com/JuliaArrays/StaticArrays.jl.git&quot;"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-k">using</span> PkgOnlineHelp  <span class="pl-c"><span class="pl-c">#</span> If not included in your startup.jl</span>

julia<span class="pl-k">&gt;</span> <span class="pl-c1">@docs</span> StaticArrays
<span class="pl-s"><span class="pl-pds">"</span>https://github.com/JuliaArrays/StaticArrays.jl.git<span class="pl-pds">"</span></span></pre></div>
<p dir="auto">The <code>StaticArrays</code> repo site is returned and opened in the browser. By default, when
invoked with a package name as shown above, <code>@docs</code> will return and open the package's
repo web site.  However, this site choice can be overridden via a call to <code>add_pkg_docs</code> as
shown below.  The repo site was determined by searching through all Julia registries
in <code>DEPOT_PATH</code>, implying that unregistered packages will not be found, by default.</p>
<p dir="auto">However, <code>@docs</code> can also be used to access unregistered packages or even arbitrary web sites.
Suppose you wish to have rapid access to the portion of the online Julia documentation discussing
macros.  Then you would (one time only) enter the following at the Julia prompt:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; add_pkg_docs(&quot;macros&quot;, &quot;https://docs.julialang.org/en/v1/manual/metaprogramming/#man-macros&quot;)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">add_pkg_docs</span>(<span class="pl-s"><span class="pl-pds">"</span>macros<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>https://docs.julialang.org/en/v1/manual/metaprogramming/#man-macros<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">Later in this Julia session, or in any future session, the site can be quickly opened by entering</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; @docs macros"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">@docs</span> macros</pre></div>
<p dir="auto">Of course, this example assumes that <code>using PkgOnlineHelp</code> has already been entered.</p>
<p dir="auto">Similarly, entering</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="julia&gt; add_pkg_docs(&quot;StaticArrays&quot;, &quot;https://juliaarrays.github.io/StaticArrays.jl/stable/&quot;)"><pre>julia<span class="pl-k">&gt;</span> <span class="pl-c1">add_pkg_docs</span>(<span class="pl-s"><span class="pl-pds">"</span>StaticArrays<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>https://juliaarrays.github.io/StaticArrays.jl/stable/<span class="pl-pds">"</span></span>)</pre></div>
<p dir="auto">will cause future invocations of <code>@docs StaticArrays</code> to open the hosted documentation for the package, rather than the repo site.</p>
<p dir="auto">Each call to <code>add_pkg_docs</code> adds an entry to a tiny TOML database file stored in a
<a href="https://github.com/JuliaPackaging/Scratch.jl">Scratch</a> space, providing permanence across Julia sessions.</p>
<p dir="auto">The contents of the <code>PkgOnlineHelp</code> database can be shown by typing <code>list_pkg_docs()</code>
at the REPL prompt. Items can be removed using, e.g. <code>remove_pkg_docs("macros")</code>.</p>

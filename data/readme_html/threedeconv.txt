<p dir="auto"><strong>ThreeDeconv.jl</strong> is a 3D deconvolution software for fluorescence microscopy written in <a href="https://julialang.org" rel="nofollow">Julia</a>. Currently, it supports Julia v.1.6.0 but may not be going to be supported in the future releases.
If interested, please feel free to make a PR.
The detail of the algorithm is available in <a href="https://www.nature.com/articles/s41598-018-29768-x" rel="nofollow">our paper</a> and <a href="http://www.computationalimaging.org/publications/2d-deconvolution-for-low-photon-count-fluorescence-imaging-scientific-reports-2019/" rel="nofollow">our website</a>.
While the deconvolution algorithm is the same as described in the paper, we made some improvements in our software along with the major update of Julia and related packages. Thanks to this update, this package is significantly faster than the speed reported on the paper and deconvoles a 256 x 256 x 57 image in 5 seconds!  Please refer to <a href="NEWS.md">NEWS.md</a> to catch up the updates.</p>

<p dir="auto">To run the example script, you need <a href="https://github.com/JuliaPy/PyCall.jl"><code>PyCall.jl</code></a> and <a href="https://github.com/JuliaPy/Conda.jl"><code>Conda.jl</code></a>. In addition, you need <a href="https://scikit-image.org/" rel="nofollow">scikit-image</a> available through <code>PyCall</code>. You can run the following in Julia's REPL to install them.</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia&gt; import Pkg; Pkg.add(&quot;Conda&quot;)
julia&gt; import Conda; Conda.add_channel(&quot;conda-forge&quot;); Conda.update(); Conda.add(&quot;scikit-image&quot;)
julia&gt; ENV[&quot;PYTHON&quot;] = joinpath(Conda.PYTHONDIR, &quot;python&quot;)
julia&gt; Pkg.add(&quot;PyCall&quot;)"><pre lang="julia-repl" class="notranslate"><code>julia&gt; import Pkg; Pkg.add("Conda")
julia&gt; import Conda; Conda.add_channel("conda-forge"); Conda.update(); Conda.add("scikit-image")
julia&gt; ENV["PYTHON"] = joinpath(Conda.PYTHONDIR, "python")
julia&gt; Pkg.add("PyCall")
</code></pre></div>
<p dir="auto">If you already have a Python environment that you want to use from Julia, please consult with <code>PyCall.jl</code>.
Since these packages are required just for image I/O, you can also use other packages if you want.
However, I personally found that <code>scikit-image</code> is the most convenient package to read multi-stack tiff files from Julia.</p>
<p dir="auto">If you have a CUDA-compatible NVIDIA GPU, the example script runs on a GPU. You might want to use <a href="https://julialang.github.io/PackageCompiler.jl/dev/examples/plots/" rel="nofollow">`PackageCompiler.jl'</a> to create a sysimage.</p>
<p dir="auto">Then, after downloading the dataset (described below) in the <code>example</code> directory, run</p>
<div class="snippet-clipboard-content notranslate position-relative overflow-auto" data-snippet-clipboard-copy-content="julia example.jl"><pre class="notranslate"><code>julia example.jl
</code></pre></div>
<p dir="auto">from your terminal in the <code>example</code> directory. The script simulates the PSF, deconvolves the fluorescent bead and saves the deconvolved image.</p>

<p dir="auto">The dataset used in our paper is available at <a href="https://drive.google.com/a/stanford.edu/file/d/1lWlvngb5iJkToFKLSA3N1FuScVPTe-42/view?usp=sharing" rel="nofollow">Google Drive (150MB)</a>.
The images are processed with darkfield and flatfield correction as described in the paper. Its raw dataset is also available at <a href="https://drive.google.com/a/stanford.edu/file/d/1pg_OG5GxjcKMSvwi4Si0HTyWT0XCW4Kt/view?usp=sharing" rel="nofollow">Google Drive (1GB)</a>.
This dataset comes with corresponding darkfield and flatfield images.</p>
<p dir="auto">This dataset can be used for academic and other non-commercial purposes.
If you use this dataset in your research paper, please cite our paper.</p>

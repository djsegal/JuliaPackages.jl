<p dir="auto"><a href="https://travis-ci.org/JuliaGraphics/Immerse.jl" rel="nofollow"><img src="https://camo.githubusercontent.com/cbcb7dd82f32ba9b8f03fd118330e0db11c4e5e456373c0a9e350540a7979d26/68747470733a2f2f7472617669732d63692e6f72672f4a756c696147726170686963732f496d6d657273652e6a6c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/JuliaGraphics/Immerse.jl.svg?branch=master" style="max-width: 100%;"></a></p>
<p dir="auto">Immerse is a wrapper that adds graphical interactivity to Julia plots.
Currently, Immerse supports
<a href="https://github.com/dcjones/Gadfly.jl">Gadfly</a>.</p>
<h1 dir="auto"><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Usage</h1>
<p dir="auto">By and large, you plot just as you would in Gadfly:</p>
<div class="highlight highlight-source-julia notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="using Immerse, Distributions

<p dir="auto">However, rather than being displayed in a browser window, you'll see your figure in a Gtk window:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="images/zoom_hexbin_snapshot.png"><img src="images/zoom_hexbin_snapshot.png" alt="window" style="max-width: 100%;"></a></p>
<p dir="auto">The toolbar at the top supports saving your figure to a file, zooming and panning, and lasso selection.</p>
<p dir="auto">Zooming and panning uses the defaults set by <a href="https://github.com/timholy/GtkUtilities.jl">GtkUtilities</a>.  The left mouse button allows you to rubberband-select a zoom region.  Use your mouse wheel or arrow-keys to pan or change the zoom level.  Double-click, or press the 1:1 button, to restore the full view.</p>

<p dir="auto">The right-most button on the toolbar allows you to select a group of points for further analysis by drawing a "lasso" around them:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="images/lasso_snapshot.png"><img src="images/lasso_snapshot.png" alt="lasso" style="max-width: 100%;"></a></p>
<p dir="auto">By default, this pops up a dialog asking you which variable in <code>Main</code> you want to save the selected indexes to:</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="images/lasso_dialog_snapshot.png"><img src="images/lasso_dialog_snapshot.png" alt="lassodialog" style="max-width: 100%;"></a></p>
<p dir="auto">You can alternatively define a custom callback function; see the help for <code>lasso_initialize</code> by typing <code>?lasso_initialize</code> at the REPL.</p>
<p dir="auto">Lasso selection is currently implemented only for <code>Geom.point</code> and <code>Geom.line</code>.  If you want to try this feature, the demonstration in <code>test/faces.jl</code> can be fun.</p>

<p dir="auto">Each figure is addressed by an integer; for a window displaying a
single Gadfly figure, by default this integer appears in the window
title.</p>
<p dir="auto">There are a few simple utilities for working with figure windows:</p>
<ul dir="auto">
<li><code>figure()</code> opens a new figure window. This will become the default
plotting window.</li>
<li><code>figure(3)</code> raises the corresponding window and makes it the default.</li>
<li><code>gcf()</code> returns the index of the current default figure.</li>
<li><code>scf()</code> shows the current figure (raising the window to the top).</li>
<li><code>closefig(3)</code> destroys Figure 3, closing the window.</li>
<li><code>closeall()</code> closes all open figure windows.</li>
</ul>

<p dir="auto">Your window manager may have "focus stealing prevention" enabled. For
example, under KDE, go to the Kmenu-&gt;System Settings-&gt;Window
behavior-&gt;Window behavior (pane)-&gt;Focus (tab) and set "Focus stealing
prevent" to "None".  Alternatively, if you want to limit this change
to julia, use the "Window rules" pane and add a new setting where
"Window class (application)" is set to "Regular Expression" with value
"^julia.*".</p>

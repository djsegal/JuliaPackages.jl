<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview</h2>
<p>If <code>A</code> is a zero-one (integer) matrix, then <code>life_step(A)</code> gives the next
generation from <code>A</code> in Conway's cellular automata Game of Life.
The <code>life_run</code> function shows the progression of the cellular
automata on the screen. Use <code>life_movie</code> to create an animated GIF.</p>

<h3><a id="user-content-life_step" class="anchor" aria-hidden="true" href="#life_step"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>life_step</code></h3>
<p>Cellular automata are represented by an integer matrix with <code>1</code>s representing
live cells and <code>0</code>s dead cells. Given such a matrix <code>A</code>, the function
<code>life_step(A)</code> returns a new matrix that is the one-step progression
of the cellular automata. Optionally, <code>life_step(A,true)</code> signals that
the matrix wraps around (left and right edges are identified,
as are top and bottom edges) and so the domain is a torus.</p>
<p>We also provide <code>life_step!(A,wrap=false)</code> which overwrites <code>A</code> with the next
generation. <code>life_step!</code> returns <code>true</code> if the new <code>A</code> is different from
the old <code>A</code> and <code>false</code> if there was no change.</p>
<h3><a id="user-content-random_life" class="anchor" aria-hidden="true" href="#random_life"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>random_life</code></h3>
<p>The function <code>A = random_life(n,m)</code> creates a random <code>n</code>-by-<code>m</code> zero-one
matrix as a starting point for the cellular automata.</p>
<p><code>random_life(n,m,p)</code> creates an <code>n</code>-by-<code>m</code> random matrix in which the
the probability a cell is a one is <code>p</code>.</p>
<ul>
<li><code>random_life(n)</code> is equivalent to <code>random_life(n,n)</code>.</li>
<li><code>random_life(n,p)</code> is equivalent to <code>random_life(n,n,p)</code>.</li>
</ul>
<h3><a id="user-content-life_run" class="anchor" aria-hidden="true" href="#life_run"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>life_run</code></h3>
<p>The <code>life_run(A)</code> function is used to run and visualize the cellular
automata. The run goes on endlessly, but will stop if there are no
live cells, if two consecutive
generations are identical, or if it enters a period-2 oscillation.
If <code>life_run</code> halts, it returns the number of iterations.</p>
<p>This function takes the following optional named arguments:</p>
<ul>
<li><code>pause=0.0</code> is the number of extra seconds between iterations. Note: The
first image may take a while to appear as the plotting software
initializes.</li>
<li><code>wrap=false</code> determines if the cellular automata field wraps. If <code>true</code>
then the domain is a torus.</li>
<li><code>counter=false</code> shows the iteration number under the image.</li>
<li><code>max_steps</code> places a limit on the number of steps to run.</li>
</ul>
<h3><a id="user-content-life_movie" class="anchor" aria-hidden="true" href="#life_movie"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>life_movie</code></h3>
<p><code>life_movie(A)</code> creates an animated GIF file. Use the following named
arguments to control the result:</p>
<ul>
<li><code>file_name="life.gif"</code>: name of the GIF file in which to save</li>
<li><code>wrap::Bool=false</code>: edge wrapping (same as <code>life_run</code>)</li>
<li><code>max_steps</code>: limit on the number of steps (frames)</li>
<li><code>rate=5</code>: animation frames per second</li>
</ul>

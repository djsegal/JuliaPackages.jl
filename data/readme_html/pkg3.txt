<h4><a id="user-content-using-the-development-version-of-pkgjl" class="anchor" aria-hidden="true" href="#using-the-development-version-of-pkgjl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Using the development version of Pkg.jl</h4>
<p>If you want to develop this package do the following steps:</p>
<ul>
<li>Clone the repo anywhere.</li>
<li>Remove the <code>uuid = </code> line from the <code>Project.toml</code> file.</li>
<li>Change the current directory to the Pkg repo you just cloned and start julia with <code>julia --project</code>.</li>
<li><code>import Pkg</code> will now load the files in the cloned repo instead of the Pkg stdlib .</li>
<li>To test your changes, simply do <code>include("test/runtests.jl")</code>.</li>
</ul>
<p>If you need to build Julia from source with a git checkout of Pkg, then instead use <code>make DEPS_GIT=Pkg</code> when building Julia. The <code>Pkg</code> repo is in <code>stdlib/Pkg</code>, and created initially with a detached <code>HEAD</code>. If you're doing this from a pre-existing Julia repository, you may need to <code>make clean</code> beforehand.</p>

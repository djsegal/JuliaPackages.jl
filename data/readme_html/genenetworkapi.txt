<h1 dir="auto"><a id="user-content-genenetworkapi" class="anchor" aria-hidden="true" href="#genenetworkapi"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>GeneNetworkAPI</h1>
<p dir="auto">Provides access to the <a href="http://genenetwork.org" rel="nofollow">GeneNetwork</a> database
and analysis functions using the <a href="https://github.com/genenetwork/gn-docs/blob/master/api/GN2-REST-API.md">GeneNetwork REST
API</a>.</p>
<p dir="auto">Karl Broman wrote the
<a href="https://github.com/kbroman/GNapi/blob/main/README.md">GNapi</a> R
package for providing access to GeneNetwork from R.  This package
follows the structure and function of that package closely.</p>
<h2 dir="auto"><a id="user-content-note-on-terminology" class="anchor" aria-hidden="true" href="#note-on-terminology"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg></a>Note on terminology</h2>
<p dir="auto">GeneNetwork collects data on genetically segregating populations
(called <em>groups</em>) in a number of <em>species</em> including humans.  Most of
the phenotype data is "omic" data which are organized as <em>datasets</em>.</p>

<p dir="auto">This function performs a one-dimensional genome scan.  The arguments
are</p>
<ul dir="auto">
<li>db (required) - DB name for trait above (Short_Abbreviation listed
when you query for datasets)</li>
<li>trait (required) - ID for trait being mapped</li>
<li>method - hk (default) | ehk | em | imp | mr | mr-imp |
mr-argmax ; Corresponds to the "method" option for the R/qtl scanone
function.</li>
<li>model - normal (default) | binary | 2-part | np ; corresponds
to the "model" option for the R/qtl scanone function</li>
<li>n_perm - number of permutations; 0 by default</li>
<li>control_marker - Name of marker to use as control; this relies on
the user knowing the name of the marker they want to use as a
covariate</li>
<li>interval_mapping - Whether to use interval mapping; "false" by default</li>
</ul>

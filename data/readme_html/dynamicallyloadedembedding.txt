<p dir="auto">This package serves two purposes.</p>
<ul dir="auto">
<li>
<p dir="auto">Make Julia embedding more accessible and easier to use.</p>
</li>
<li>
<p dir="auto">Provide a regression test (via
<a href="https://github.com/JuliaComputing/NewPkgEval.jl">PkgEval</a>) so that
changes in new versions of Julia that break this form of embedding
do not pass unnoticed.</p>
</li>
</ul>
<p dir="auto">With "Dynamically Loaded" it is understood that <code>libjulia</code> is not
linked into the embedding application but is loaded at runtime with
<code>dlopen</code>/<code>LoadLibrary</code>. This comes with both advantages and
disadvantages as discussed at some length in the
<a href="docs/background.md">background</a> section. To summarize the main
advantages are:</p>
<ul dir="auto">
<li>
<p dir="auto">Not tied to a specific version of Julia.</p>
</li>
<li>
<p dir="auto">Julia is not needed in the build environment.</p>
</li>
<li>
<p dir="auto">The application can run without having Julia bundled or installed,
for users who do not need the Julia functionality.</p>
</li>
<li>
<p dir="auto">All interaction is made via <code>cfunction</code>s - simpler interface than
the "regular" embedding.</p>
</li>
</ul>
<p dir="auto">The main disadvantages are:</p>
<ul dir="auto">
<li>
<p dir="auto">More difficult setup of the embedding environment.</p>
</li>
<li>
<p dir="auto">All interaction is made via <code>cfunction</code>s - less powerful interface
than the "regular" embedding.</p>
</li>
</ul>
<p dir="auto">The idea of this package is to take care of most of the difficulties
of this approach.</p>
